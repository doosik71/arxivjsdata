# Learning to Prompt Segment Anything Models

Jiaxing Huang, Kai Jiang, Jingyi Zhang, Han Qiu, Lewei Lu, Shijian Lu, Eric Xing (2024)

## 🧩 Problem to Solve

본 논문은 Segment Anything Models (SAMs), 특히 SAM 및 SEEM과 같은 모델들이 가진 'Promptable Segmentation' 능력을 극대화하는 방법을 다룬다. SAMs는 사용자가 제공한 프롬프트(Spatial prompt 및 Semantic prompt)를 입력받아 해당하는 세그멘테이션 마스크를 생성한다. 하지만 실제 다운스트림 데이터셋에서 기본 프롬프트(단순 클래스 이름이나 격자 형태의 포인트)를 사용하는 것은 최적이 아니며, 매우 작은 프롬프트 변화만으로도 결과가 크게 달라지기 때문에 최적의 프롬프트를 획득하는 것이 매우 어렵다.

저자들은 SAMs의 아키텍처를 분석하여 효과적인 프롬프트 학습을 방해하는 두 가지 핵심 과제를 정의한다. 첫째, **Spatial Prompt Learning의 제한된 탐색 공간(Limited Search Space)** 문제이다. 공간 프롬프트는 이미지의 $XY$ 좌표계(2차원)에서 최적화되는데, 이는 저차원 공간이므로 최적의 프롬프트를 찾기에 탐색 범위가 너무 좁다. 둘째, **Text Prompt Encoder의 부작용(Side Effects)** 문제이다. CLIP과 같은 텍스트 인코더는 주로 객체 중심(Object-centric) 데이터로 학습되어 전경(Foreground)에 대한 지식은 풍부하지만, 배경(Background)에 대한 지식은 상대적으로 부족하여 배경 세그멘테이션 시 부정적인 영향을 줄 수 있다.

따라서 본 논문의 목표는 소수의 데이터(few-shot)만을 사용하여 다운스트림 작업에 최적화된 공간 및 시맨틱 프롬프트를 학습하는 **SSPrompt (Spatial-Semantic Prompt Learning)** 방법론을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 프롬프트를 입력 공간(좌표나 토큰)이 아닌, **고차원 임베딩 공간(Embedding Space)에서 직접 최적화**하는 것이다.

1. **임베딩 공간 최적화**: 2차원 좌표계나 이산적인 텍스트 토큰 대신, 고차원(예: 512차원) 임베딩 공간에서 직접 프롬프트를 학습함으로써 탐색 공간을 획기적으로 확장한다.
2. **선택적 지식 활용**: 사전 학습된 프롬프트 인코더의 지식을 무조건 사용하는 것이 아니라, 학습 가능한 가중치를 통해 기본 임베딩과 새롭게 학습된 임베딩을 적응적으로 융합함으로써 인코더의 편향(예: 전경 편향)을 완화한다.

## 📎 Related Works

최근 SAM 및 SEEM과 같은 모델들이 'Promptable Segmentation'을 통해 뛰어난 성능을 보여주었으나, 대부분의 연구는 수작업으로 생성된 프롬프트에 의존했다.

기존의 프롬프트 학습(Prompt Learning) 연구들은 주로 이미지 분류 파운데이션 모델(CFMs)을 대상으로 진행되었다. 대표적으로 CoOp와 같은 방법론은 텍스트 인코더에 입력될 컨텍스트 토큰을 학습하여 클래스 이름을 보완하는 방식을 취한다. 그러나 이러한 방식은 여전히 텍스트 인코더 내부의 토큰 수준에서 작동하며, 세그멘테이션 모델의 특수한 구조(공간 프롬프트의 존재 및 마스크 디코더의 역할)를 충분히 고려하지 않았다.

본 논문의 접근 방식은 텍스트 인코더 자체를 최적화하는 대신, 인코더 이후의 임베딩 공간에서 직접 최적화를 수행함으로써 계산 효율성을 높이고 탐색 공간의 제약을 해결했다는 점에서 기존 연구와 차별화된다.

## 🛠️ Methodology

### 전체 시스템 구조

SSPrompt는 크게 **SpaPrompt (Spatial Prompt Learning)**와 **SemPrompt (Semantic Prompt Learning)** 두 가지 구성 요소로 이루어져 있으며, 이 둘이 상호 보완적으로 작용하여 마스크 디코더에 입력된다. 전체 파이프라인에서 이미지 인코더, 텍스트 인코더, 공간 인코더 및 마스크 디코더는 모두 고정(Frozen)되며, 오직 학습 가능한 임베딩과 가중치만이 업데이트된다.

### 1. Spatial Prompt Learning (SpaPrompt)

공간 프롬프트의 2차원 좌표 제한을 해결하기 위해 고차원 임베딩 공간에서 최적화를 수행한다.

- $\hat{Z}^S = \{\hat{z}^S_n\}_{n=1}^N$: $N$개의 학습 가능한 공간 임베딩
- $\hat{W}^S = \{\hat{w}^S_n\}_{n=1}^N$: 각 임베딩에 적용될 학습 가능한 가중치 ($\hat{w}^S_n \in [0, 1]$)
- $Z^{S}_{default}$: 기존 공간 인코더를 통해 생성된 기본 공간 임베딩

최종 공간 프롬프트 $Z^{SpaPrompt}_S$는 다음과 같이 계산된다:
$$Z^{SpaPrompt}_S = \{ \hat{w}^S_n \hat{z}^S_n + (1 - \hat{w}^S_n) z^S_n \}_{n=1}^N$$
이 구조를 통해 모델은 기본 인코더의 지식을 유지하면서도 고차원 공간에서의 유연한 최적화를 가능하게 한다.

### 2. Semantic Prompt Learning (SemPrompt)

텍스트 인코더의 전경 편향 문제를 해결하기 위해 기본 시맨틱 임베딩과 학습 가능한 임베딩을 가중 합산하여 융합한다.

- $\hat{Z}^T = \{\hat{z}^T_c\}_{c=1}^C$: $C$개의 학습 가능한 시맨틱 임베딩
- $\hat{W}^T = \{\hat{w}^T_c\}_{c=1}^C$: 학습 가능한 가중치
- $Z^{T}_{default}$: 고정된 텍스트 인코더를 통해 생성된 기본 시맨틱 임베딩

최종 시맨틱 프롬프트 $Z^{SemPrompt}_T$는 다음과 같다:
$$Z^{SemPrompt}_T = \{ \hat{w}^T_c \hat{z}^T_c + (1 - \hat{w}^T_c) z^T_c \}_{c=1}^C$$
이 방식은 거대한 텍스트 인코더를 직접 튜닝하지 않고 임베딩 수준에서 조절하므로 매우 효율적이며, 가중치 $\hat{w}^T_c$를 통해 인코더의 지식을 선택적으로 수용한다.

### 3. 통합 학습 절차 (SSPrompt)

최종적으로 마스크 디코더는 이미지 임베딩 $z^I$와 함께 최적화된 두 프롬프트를 입력받아 마스크와 신뢰도 점수를 예측한다:
$$\{M, C\} = \text{Decoder}(z^I | Z^{SpaPrompt}_S, Z^{SemPrompt}_T)$$
학습 목표는 표준 세그멘테이션 손실 함수 $L_{seg}$를 최소화하는 학습 가능한 변수들을 찾는 것이다:
$$\arg \min_{\{\hat{Z}^S, \hat{W}^S, \hat{Z}^T, \hat{W}^T\}} \frac{1}{|X^I|} \sum_{x^I \in X^I} L_{seg}(\{M, C\}, y^I)$$

## 📊 Results

### 실험 설정

- **데이터셋**: Cityscapes, BDD100K, Mapillary (거리 뷰), ADE20K, Pascal Context (일반 장면), ACDC (악천후 환경) 등 6개 데이터셋.
- **비교 대상**: SEEM-T/L (Baseline), CoOp, LOCN.
- **평가 지표**: mIoU (mean Intersection over Union).
- **조건**: 클래스당 16장의 이미지만 사용하는 16-shot 설정.

### 주요 결과

1. **정량적 성능**: SSPrompt는 모든 데이터셋에서 baseline 및 기존 프롬프트 학습 방법(CoOp, LOCN)보다 일관되게 높은 mIoU를 기록하였다. 특히 SEEM-L 모델이 PASCAL Context에서 예상외로 낮은 성능을 보였음에도 불구하고, SSPrompt를 적용했을 때 가장 큰 폭의 성능 향상이 나타나 SAMs의 실패 사례를 효과적으로 보완함을 입증하였다.
2. **악천후 강인성**: ACDC 데이터셋 실험 결과, 안개, 밤, 비, 눈 등의 극한 환경에서도 타 방법론 대비 압도적인 성능 향상을 보였다.
3. **학습 효율성**: 텍스트 인코더를 거치지 않고 임베딩 공간에서 직접 학습하므로, ADE20K 기준 학습 시간은 약 36% 감소하고, 학습 메모리는 약 53.5% 절감되었다.
4. **태스크 일반화**: 시맨틱 세그멘테이션뿐만 아니라 인스턴스 세그멘테이션 및 파놉틱 세그멘테이션에서도 성능 향상이 확인되었다.

## 🧠 Insights & Discussion

### 분석 및 강점

- **전경-배경 편향 확인**: 저자들은 LAION 데이터셋의 통계를 통해 텍스트 인코더가 전경 객체에 편향되어 있음을 확인하였다. 실제로 학습된 가중치 $\hat{w}^T_c$를 시각화했을 때, 배경 클래스는 낮은 가중치를, 전경 클래스는 높은 가중치를 갖는 경향이 나타났다. 이는 모델이 인코더의 지식이 부족한 배경 부분은 학습 가능한 임베딩 $\hat{z}^T_c$에 더 의존하고, 지식이 풍부한 전경 부분은 인코더의 지식을 더 많이 활용하고 있음을 시사한다.
- **탐색 공간의 중요성**: 좌표계(2D)에서 직접 최적화하는 VSPL(Vanilla Spatial Prompt Learning)과 비교했을 때, 임베딩 공간(512D)에서 최적화하는 SpaPrompt의 성능이 월등히 높았다. 이는 고차원 공간에서의 최적화가 SAMs의 성능을 끌어올리는 데 필수적임을 보여준다.

### 한계 및 비판적 해석

- **SAM 모델의 제약**: 원본 SAM의 경우 텍스트 인코더 버전이 공개되지 않아 클래스 불가지론적(class-agnostic) 세그멘테이션만 가능하며, 이로 인해 SAM 기반 실험에서는 성능 향상 폭이 상대적으로 적게 나타났다. 이는 제안 방법론의 한계라기보다 기본 모델의 기능 제약에 가깝다.
- **데이터 의존성**: Few-shot 설정에서 매우 효율적이지만, 샷 수(4-shot $\to$ 16-shot)가 증가함에 따라 성능이 계속 향상되는 경향을 보여, 데이터 양에 따른 성능 포화 지점이 어디인지에 대한 추가 분석이 필요해 보인다.

## 📌 TL;DR

본 논문은 SAMs의 프롬프트 최적화 시 발생하는 **'제한된 탐색 공간'**과 **'텍스트 인코더의 전경 편향'** 문제를 해결하기 위해, 프롬프트를 고차원 임베딩 공간에서 직접 학습하고 가중치를 통해 선택적으로 융합하는 **SSPrompt**를 제안한다. 이 방법은 계산 비용을 대폭 줄이면서도 다양한 환경(특히 악천후)과 다양한 세그멘테이션 태스크에서 SOTA 성능을 달성하였다. 이는 거대 모델을 특정 도메인에 적응시킬 때 인코더 자체를 튜닝하는 것보다 임베딩 수준의 정교한 조절이 더 효율적이고 효과적일 수 있음을 시사한다.
