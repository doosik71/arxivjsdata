# A REVIEW OF OPEN SOURCE SOFTWARE TOOLS FOR TIME SERIES ANALYSIS

Yunus Parvej Faniband, Iskandar Ishak, Sadiq Sait (2022)

## 🧩 Problem to Solve

본 논문은 시계열 분석(Time Series Analysis, TSA) 분야에서 사용되는 오픈소스 소프트웨어 도구들의 파편화 문제를 해결하고자 한다. 시계열 데이터는 금융, 마케팅, 엔지니어링, IoT, 기후 변화 등 매우 다양한 도메인에서 활용되며, 특히 예측(Forecasting)과 이상 탐지(Anomaly Detection)는 이벤트의 행동 양식을 이해하고 미래를 대비하는 데 매우 중요하다.

하지만 현재의 시계열 분석 생태계는 매우 파편화되어 있다. 많은 전문 도구들이 존재하지만, 서로 호환되지 않는 경우가 많으며 특정 모델 클래스(예: ARIMA, 신경망)에만 국한된 인터페이스를 제공하거나, 특징 추출(Feature Extraction)과 같은 특정 단계만 수행하는 라이브러리가 많다. 결과적으로 연구자와 개발자는 데이터를 준비하고, 모델을 선택하며, 평가하는 전체 파이프라인을 구축하는 데 과도한 시간과 노력을 소모하게 된다. 따라서 본 논문의 목표는 시계열 분석을 위한 주요 오픈소스 도구들을 체계적으로 검토하고, 이를 통합할 수 있는 전형적인 TSA 프레임워크의 아키텍처와 요구사항을 제시하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 다음과 같다.

1. **TSA 프레임워크의 표준 요구사항 정의**: 데이터 전처리, 예측 프레임워크, 이상 탐지 프레임워크, 그리고 이를 외부에서 사용할 수 있게 하는 REST API 노출까지 포함하는 기능적 명세(Functional Specifications)를 제안한다.
2. **일반적인 TSA 프레임워크 아키텍처 설계**: 데이터 입력부터 전처리, 분석 셀렉터(Forecasting/Anomaly Detection Selector)를 거쳐 최적화된 모델을 출력하는 시스템 구조를 설계하여 제시한다.
3. **오픈소스 도구의 체계적 분석 및 분류**: 60개의 시계열 분석 도구를 대상으로 분석 작업(Tasks), 데이터 준비 방법(Data Preparation), 평가 방법(Evaluation) 기준에 따라 정량적 분석을 수행하고 이를 카테고리화한다.

## 📎 Related Works

논문에서는 시계열 분석과 관련된 기존 연구들을 다음과 같이 분류하여 설명한다.

- **연구 주제 및 응용 도메인**: 예측, 이상 탐지, 패턴 인식, 차원 축소, 변화 지점 분석(Change Point Analysis) 등 다양한 주제의 리뷰 논문들이 존재한다. 또한 IoT, Industry 4.0, 의료, 금융과 같은 특정 도메인에 특화된 리뷰들이 수행되었다.
- **언어별 패키지 리뷰**: Python이나 R 언어의 특정 도메인 패키지에 대한 리뷰는 있었으나, 시계열 분석 전체를 아우르는 소프트웨어 도구들에 대한 체계적인 리뷰는 부족한 실정이다.

기존 접근 방식과의 차별점은 단순히 알고리즘의 성능을 비교하는 것이 아니라, 소프트웨어 공학적 관점에서 도구들의 기능성, 유지보수 상태, 상호 운용성 및 프레임워크로서의 구조를 분석했다는 점에 있다.

## 🛠️ Methodology

### 1. 도구 선정 기준 (Selection Criteria)

저자들은 Kitchenham과 Brereton의 가이드라인을 수정하여 다음과 같은 기준으로 도구를 선정하였다.

- **분석 작업(TSK)**: 예측(TSK1), 분류(TSK2), 클러스터링(TSK3), 이상 탐지(TSK4), 세그멘테이션(TSK5), 패턴 인식(TSK6), 변화 지점 탐지(TSK7) 중 하나 이상을 구현해야 한다.
- **구현 언어(SC1)**: Python, Java, C++, Julia, R 중 하나로 구현되어야 한다.
- **유지보수 상태(SC2)**: 최신 커밋이 6개월 이내여야 하며(SC2.1), GitHub의 경우 별(Star) 수가 100개 이상이어야 한다(SC2.2).
- **범용성(IC4)**: 특정 도메인에 국한되지 않은 범용적인 도구여야 한다.

### 2. TSA 프레임워크 아키텍처 및 기능 명세

본 논문이 제안하는 전형적인 TSA 프레임워크의 구조는 다음과 같다.

**전체 파이프라인 흐름**:
$$\text{Input Time Series} \rightarrow \text{Data Processing} \rightarrow \text{Selector (Forecasting/Anomaly Detection)} \rightarrow \text{Optimized Model}$$

- **데이터 전처리 모듈 (Data Processing)**:
  - 물리적 데이터 처리: 데이터를 정형 테이블 형식으로 정리한다.
  - 시계열 특성 처리: 비정상성(Non-stationarity) 제거, 계절성 및 추세 분석, 결측치 보간(Interpolation), 노이즈 제거 등을 수행한다.
  - 데이터 변환: 정규화(Normalization) 및 시각적 가독성을 위한 변환을 처리한다.
- **예측 프레임워크 (Forecasting Framework)**:
  - 단변량 시계열(UTS)과 다변량 시계열(MTS) 모델을 모두 통합한다.
  - **자율적 접근 방식 선택(Autonomous Approach Selection)**: 데이터의 특성을 분석하여 가장 적합한 모델을 선택하고 최적의 하이퍼파라미터를 결정한다.
- **이상 탐지 프레임워크 (Anomaly Detection Framework)**:
  - 예측 프레임워크와 유사한 구조를 가지며, 다변량 시계열(MTS) 데이터에서 이상치를 탐지하는 특화 모델을 관리한다.
- **REST API**: 위 모든 연산 과정을 REST API 형태로 제공하여 다른 서비스와의 통합이 가능하도록 한다.

## 📊 Results

### 1. 정량적 분석 결과

총 60개의 도구를 분석한 결과는 다음과 같다.

- **작업별 도구 수**: 예측(32개) $\rightarrow$ 이상 탐지(21개) $\rightarrow$ 분류(14개) $\rightarrow$ 클러스터링(13개) 순으로 많이 구현되어 있다.
- **데이터 준비 기능**: 결측치 보간(23개), 분해 방법(Decomposition, 24개), 일반 변환 및 특징 생성(24개) 기능은 비교적 흔하지만, 차원 축소(7개)나 유사도 측정(9개) 기능은 상대적으로 적게 제공된다.
- **평가 기능**: 시각화 방법(39개)과 평가 지표/통계 테스트(41개) 제공 도구가 많았으며, 모델 선택 및 하이퍼파라미터 검색 기능(27개)은 그보다 적었다.

### 2. 주요 도구 분석

- **예측 도구**:
  - **Prophet**: 페이스북 개발, 일 단위 데이터에 강점.
  - **GluonTS**: 아마존 개발, 딥러닝 기반의 확률적 예측 모델 제공.
  - **sktime/tslearn**: 시계열 분석을 위한 통합 인터페이스 및 머신러닝 툴킷 제공.
- **이상 탐지 도구**:
  - **PyOD**: 다변량 데이터 이상 탐지를 위한 광범위한 알고리즘 제공.
  - **PySAD**: 스트리밍 데이터 환경에서의 온라인 이상 탐지 특화.
  - **ADTK**: 규칙 기반 및 비지도 학습 기반의 이상 탐지 제공.

## 🧠 Insights & Discussion

### 1. 강점 및 분석

본 논문은 파편화된 시계열 도구 생태계를 정량적으로 가시화했다는 점에서 강점이 있다. 특히 단순한 리스트 작성이 아니라, "자율적 모델 선택"과 "데이터 전처리 $\rightarrow$ 분석 $\rightarrow$ 배포"로 이어지는 표준 프레임워크 아키텍처를 제시함으로써 향후 통합 도구 개발의 가이드라인을 제공했다.

### 2. 한계 및 비판적 해석

- **실제 성능 평가 부재**: 본 논문은 도구들이 제공하는 "기능의 유무"를 분석했을 뿐, 벤치마크 데이터셋을 통해 각 도구의 실제 예측 정확도나 탐지 성능을 직접 비교 평가하지 않았다.
- **상호 운용성 문제**: 분석 결과에서 나타나듯, 대다수의 도구가 Python 기반임에도 불구하고 서로 다른 API 구조를 가지고 있어, 제안된 통합 프레임워크를 실제로 구현하기 위해서는 상당한 수준의 래퍼(Wrapper) 작업이 필요할 것으로 보인다.
- **스트리밍 vs 배치**: 이상 탐지 도구에서 배치(Batch) 방식과 스트리밍(Streaming) 방식의 명확한 구분과 그에 따른 아키텍처 차이를 논의한 점은 고무적이나, 예측 도구에서도 실시간 스트리밍 예측에 대한 논의가 더 보완될 필요가 있다.

## 📌 TL;DR

본 연구는 60개의 오픈소스 시계열 분석 도구를 체계적으로 리뷰하고, 이들이 가진 파편화 문제를 해결하기 위한 **표준 TSA 프레임워크 아키텍처와 기능 명세**를 제안한다. 분석 결과 예측 도구가 가장 많았으며, Python 언어가 지배적이지만 도구 간 상호 운용성은 낮음을 확인하였다. 이 연구는 향후 데이터 전처리부터 모델 선택, 배포까지 통합된 시계열 분석 플랫폼을 구축하는 데 중요한 기초 자료로 활용될 가능성이 높다.
