# TOTEM: TOkenized Time Series EMbeddings for General Time Series Analysis

Sabera Talukder, Yisong Yue, Georgia Gkioxari (2024)

## 🧩 Problem to Solve

본 논문은 다양한 데이터 도메인에 걸쳐 적용 가능한 일반주의자(Generalist) 또는 파운데이션 모델(Foundation Model)을 통한 시계열 분석 문제를 다룬다. 기존의 시계열 모델들은 주로 특정 단일 도메인에 최적화된 전문가(Specialist) 모델이거나, 예측(Forecasting)과 같은 특정 작업만을 수행하도록 설계된 작업 특화형 모델인 경우가 많다. 또한, 많은 기존 모델들이 시계열 데이터를 패치(Patch) 단위로 처리하여 하위 모델의 레이어를 통해 동적으로 임베딩을 생성하는 방식을 취하고 있다.

이러한 전문가 중심의 접근 방식은 도메인 간의 일반화 능력을 저하시키며, 새로운 도메인에 적용하기 위해서는 상당한 양의 데이터와 미세 조정(Fine-tuning)이 필요하다는 한계가 있다. 따라서 본 연구의 목표는 특정 도메인이나 작업에 의존하지 않고, 다양한 시계열 분석 작업(결측치 보간, 이상치 탐지, 예측)을 수행할 수 있는 도메인 불가지론적(Domain-agnostic)인 일반주의자 모델을 설계하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 대규모 언어 모델(LLM)과 컴퓨터 비전 모델의 성공 사례에서 영감을 받아, 시계열 데이터를 이산적인 토큰(Discrete Tokens)으로 변환하는 토큰화(Tokenization) 전략을 도입하는 것이다.

핵심 기여 사항은 다음과 같다.

1. **TOTEM 프레임워크 제안**: 자기지도 학습(Self-supervision)을 통해 다중 도메인 코퍼스에서 고정된 수의 이산적 토큰을 학습하는 단순하고 강력한 토큰화 방법을 개발하였다.
2. **단변량 파형 기반 토큰화**: 토큰이 단변량 파형의 형태(Shape)만을 인코딩하도록 설계함으로써, 서로 다른 크기의 다변량 데이터를 단변량 토큰의 집합으로 단순 적층하여 처리할 수 있게 하여 범용성을 확보하였다.
3. **광범위한 실험적 검증**: 결측치 보간(Imputation), 이상치 탐지(Anomaly Detection), 예측(Forecasting)의 세 가지 주요 작업에 대해 수백 번의 실험을 수행하여, TOTEM이 전문가 모델 및 기존 일반주의자 모델보다 우수하거나 대등한 성능을 보임을 입증하였다. 특히 제로샷(Zero-shot) 성능에서 강력한 우위를 보였다.

## 📎 Related Works

기존 연구들은 크게 세 가지 관점에서 분류될 수 있다.

1. **전문가 vs 일반주의자 학습**: 대부분의 연구는 단일 도메인에서 학습하고 테스트하는 전문가 학습 패러다임에 치중되어 있다. 최근 Google이나 Amazon에서 일반주의자 모델을 탐색하기 시작했으나, 여전히 특정 작업(예: 예측)에 국한된 경우가 많다.
2. **패치 vs 이산 표현**: 현재 시계열 모델의 주류는 데이터를 시간적 또는 공간적으로 패치(Patch)하여 선형 투영하는 방식이다. 패치 표현은 하위 모델의 학습 과정에서 결정되는 동적(Dynamic) 표현인 반면, 토큰화는 하위 모델 학습 전 고정된 "어휘집(Vocabulary)"을 결정하는 정적 표현이다.
3. **다중 작업 수행 능력**: 많은 모델이 단일 작업에 집중하고 있으며, 다중 작업을 다루는 모델이라 하더라도 전문가 방식으로 학습된 경우가 많다.

TOTEM은 일반주의자 학습, 이산적 토큰화, 그리고 다중 작업 수행 능력을 동시에 갖춘 최초의 모델이라는 점에서 기존 연구와 차별화된다.

## 🛠️ Methodology

### 1. 전체 파이프라인 및 구조

TOTEM은 크게 두 단계로 구성된다. 첫 번째는 다중 도메인 데이터셋을 이용해 이산적 토큰을 학습하는 **자기지도 토큰화 단계**이며, 두 번째는 학습된 고정 코드북(Frozen Codebook)을 활용하여 하위 작업을 수행하는 **다운스트림 작업 단계**이다.

### 2. 토큰화 구현 (VQVAE)

TOTEM은 Vector Quantized Variational Autoencoder(VQVAE)를 기반으로 토큰화 도구를 구현하였다.

- **Encoder ($E$)**: 스트라이드 1D CNN(Strided 1D CNN) 스택을 사용하여 입력 시계열 $\text{x} \in \mathbb{R}^T$를 잠재 변수 $\text{z} = E(\text{x}) \in \mathbb{R}^{T/F \times D}$로 압축한다. 여기서 $F$는 압축 계수(Compression factor), $D$는 잠재 특징 차원이다.
- **Quantizer**: 잠재 변수 $\text{z}$를 고정된 코드북 $\mathcal{C} = \{\text{c}_i\}_{i=1}^K$ 내에서 가장 가까운 코드워드(Codeword)로 양자화한다.
  $$\hat{z} = c_\ell, \text{ where } \ell = \arg \min_i ||z - c_i||_2^2$$
- **Decoder ($D$)**: 양자화된 임베딩 $\hat{z}$를 다시 원래의 시계열 공간으로 복원하여 $\hat{x} = D(\hat{z}) \in \mathbb{R}^T$를 생성한다.

### 3. 학습 목표 및 손실 함수

VQVAE의 학습을 위해 다음과 같은 통합 손실 함수를 최적화한다.
$$\mathcal{L} = \underbrace{||x - \hat{x}||_2^2}_{\mathcal{L}_{rec}} + \underbrace{||sg[z] - \hat{z}||_2^2}_{\mathcal{L}_{vq}} + \underbrace{\beta||z - sg[\hat{z}]||_2^2}_{\mathcal{L}_{cmt}}$$
여기서 $\mathcal{L}_{rec}$는 복원 손실, $\mathcal{L}_{vq}$는 코드북 업데이트를 위한 손실, $\mathcal{L}_{cmt}$는 인코더가 코드북에 적응하도록 하는 commitment loss이며, $sg[\cdot]$는 stop-gradient 연산자이다.

### 4. 주요 설계 결정

- **배타적 시간 토큰화(Exclusively-Temporal Tokenization)**: 센서 차원이나 샘플 차원을 제외하고 오직 시간 차원만을 토큰화한다. 이는 센서의 의미가 도메인마다 다르기 때문에, 단변량 파형의 형태만을 학습함으로써 도메인 일반화 능력을 극대화하기 위함이다.
- **Reverse Instance Normalization (RevIN)**: 데이터의 통계적 특성에 구애받지 않도록 RevIN을 사용하여 정규화된 공간에서 파형을 학습한다.
- **단순한 아키텍처**: Dilated Convolution 대신 일반 Strided 1D CNN을 사용하여 모든 타임스텝을 고려하도록 설계하였다.

### 5. 하위 작업 구현

- **보간(Imputation) 및 이상치 탐지(Anomaly Detection)**: 별도의 추가 모델 없이 VQVAE의 복원 능력을 그대로 활용한다. 마스킹된 데이터를 입력하여 복원하거나, 복원 오차가 큰 지점을 이상치로 판별한다.
- **예측(Forecasting)**: 토큰화된 시퀀스를 Transformer Encoder에 입력하여 정규화된 예측값 $\bar{y}$를 도출한다. 이후 별도의 예측 헤드가 예측한 평균 $\mu$와 표준편차 $\sigma$를 이용하여 최종 예측값 $y = \sigma \cdot \bar{y} + \mu$를 계산한다.

## 📊 Results

### 1. 실험 설정

- **데이터셋**: Imputation(12개), Anomaly Detection(25개), Forecasting(12개) 데이터셋 사용.
- **비교 대상**: 전문가 모델(Specialists) 및 일반주의자 모델(Generalists, 예: GPT2, MOMENT).
- **평가 지표**: MSE, MAE (보간/예측), Precision, Recall, F1-score (이상치 탐지).
- **테스트 체계**: In-domain(학습 도메인 내 테스트) 및 Zero-shot(학습하지 않은 새로운 도메인 테스트)으로 구분.

### 2. 주요 결과

- **전문가 설정**: TOTEM은 특정 도메인에 특화된 SOTA 모델들과 비교했을 때 대등하거나 더 우수한 성능을 보였다. (예: 보간 작업에서 AvgWins 52.1% 달성)
- **일반주의자 설정**: TOTEM은 GPT2와 같은 기존 일반주의자 모델을 압도하였다. 특히 **Zero-shot** 상황에서 그 차이가 극명하게 나타났다.
  - **Imputation Zero-shot**: TOTEM (80.0% AvgWins) vs GPT2 (20.0% AvgWins)
  - **Forecasting Zero-shot**: TOTEM (90.0% AvgWins) vs GPT2 (12.5% AvgWins)
- **효율성**: TOTEM은 GPT2 대비 파라미터 수가 훨씬 적으며 학습 시간 또한 획기적으로 단축되었다 (몇 주 $\to$ 수 시간).

## 🧠 Insights & Discussion

### 1. 강점 및 발견

- **토큰 vs 패치**: Ablation study를 통해 동일한 아키텍처에서 이산적 토큰 표현이 패치 표현보다 월등히 우수함을 확인하였다. 이는 시계열 데이터에서도 LLM과 같은 토큰화 전략이 유효함을 시사한다.
- **데이터 다양성의 중요성**: 데이터셋의 규모보다 도메인의 다양성이 일반화 성능에 더 큰 영향을 미친다는 것을 발견하였다. (Electricity 전문가 모델보다 데이터 규모는 작지만 다양한 도메인을 학습한 일반주의자 모델의 제로샷 성능이 더 높음)

### 2. 한계 및 논의사항

- **코드북 크기**: 코드북 크기 $K$가 증가함에 따라 복원 성능은 향상되지만, 하위 작업의 효율성을 위해 $K=256$으로 설정하였다. 더 큰 코드북이 실제 성능 향상으로 이어지는지에 대한 추가 연구가 필요하다.
- **단순한 구조**: 본 연구는 매우 단순한 CNN 및 Transformer 구조를 사용하였다. 이는 아키텍처 개선의 여지가 많음을 의미하며, 더 복잡한 모델과의 결합 가능성이 열려 있다.

### 3. 비판적 해석

본 논문은 토큰화의 효용성을 입증하는 데 성공했으나, 제로샷 성능의 향상이 단순히 "파형의 형태"를 잘 포착했기 때문인지, 아니면 VQVAE의 압축 과정에서 일종의 정규화 효과가 발생했기 때문인지에 대한 심층적인 분석이 더 보완되어야 할 것으로 보인다.

## 📌 TL;DR

TOTEM은 시계열 데이터를 이산적인 토큰으로 변환하는 VQVAE 기반의 토큰화 방법을 통해, 특정 도메인에 종속되지 않는 **범용 시계열 파운데이션 모델**의 가능성을 제시한다. 실험 결과, TOTEM은 보간, 이상치 탐지, 예측의 세 가지 작업 모두에서 기존 SOTA 전문가 모델 및 일반주의자 모델을 능가했으며, 특히 학습하지 않은 도메인에 적용하는 **제로샷(Zero-shot) 성능**에서 압도적인 효율성과 정확도를 보였다. 이는 시계열 분석 분야에서도 NLP의 토큰화 전략을 도입하는 것이 도메인 일반화를 달성하는 핵심 열쇠가 될 수 있음을 시사한다.
