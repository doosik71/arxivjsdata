# Preventing Rogue Agents Improves Multi-Agent Collaboration

Ohav Barbi, Ori Yoran, Mor Geva (2025)

## 🧩 Problem to Solve

본 논문은 다중 에이전트 시스템(Multi-Agent Systems, MAS)에서 발생하는 치명적인 취약점, 즉 단 한 명의 에이전트가 시스템 전체의 실패를 유발하는 문제를 해결하고자 한다. 특히 LLM 기반의 에이전트들이 협력하여 복잡한 과제를 수행할 때, 특정 에이전트가 잘못된 정보를 생성하거나(Hallucination), 무관한 메시지에 주의를 빼앗기거나, 혹은 논리적 오류를 범하는 'Rogue Agent'가 될 가능성이 높다.

이러한 Rogue Agent가 생성한 노이즈는 통신 채널을 통해 다른 에이전트들에게 전파되며, 단계가 진행될수록 증폭되어 결국 시스템 전체의 실패로 이어진다. 기존의 연구들은 통신 프로토콜의 변경이나 추론 모듈의 추가 등을 통해 전반적인 성능을 높이려 했으나, Rogue Agent 한 명으로 인해 시스템이 붕괴되는 근본적인 상황을 방지하는 데에는 한계가 있었다. 따라서 본 연구의 목표는 에이전트의 상태를 실시간으로 모니터링하고, 실패가 예상되는 시점에서 적절히 개입(Intervention)하여 시스템의 안정성과 성공률을 높이는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 에이전트가 완전히 기능 불능 상태가 되어 과제에 실패하기 전, 에이전트의 내부 상태와 환경 관찰 데이터를 통해 미래의 실패를 조기에 감지하고 개입하는 것이다. 이를 위해 다음과 같은 기여를 한다.

1. **실시간 모니터링 및 개입 프레임워크 제안**: 에이전트의 액션 선택 과정에서 나타나는 불확실성(Uncertainty) 신호를 감지하여 Rogue Agent를 식별하고, 통신 채널을 되돌리는 등의 개입을 통해 오류 전파를 차단한다.
2. **WhoDunitEnv 환경 구축**: 에이전트 간의 통신 구조(대칭/비대칭)와 과제 복잡도를 모듈식으로 조절할 수 있는 새로운 협력 환경인 WhoDunitEnv를 도입하여 제안 방법론을 검증한다.
3. **다양한 도메인에서의 성능 입증**: WhoDunitEnv뿐만 아니라 코드 생성(CodeGen), 자원 지속 가능성 시뮬레이션(GovSim) 환경에서도 성능 향상을 입증하여 제안 방법론의 범용성을 보여준다.

## 📎 Related Works

논문에서는 다음과 같은 관련 연구들을 언급하며 본 연구와의 차별점을 제시한다.

- **다중 에이전트 통신**: 에이전트 간의 토론(Debate)이나 협력을 통해 추론 능력을 높이려는 시도가 많았으나, 본 연구는 통신 내용의 질을 실시간으로 감시하고 제어하는 관점에 집중한다.
- **시스템 강건성(Robustness)**: 적대적 공격(Adversarial Attacks)에 대한 연구가 진행되었으나, 본 연구는 외부 공격뿐만 아니라 LLM 자체의 내부적 결함(Hallucination 등)으로 인해 발생하는 Rogue Agent 문제를 다룬다.
- **불확실성 추정(Uncertainty Estimation)**: LLM의 할루시네이션 감지를 위해 엔트로피 등을 사용하는 기법들이 존재했다. 본 연구는 이러한 불확실성 지표를 다중 에이전트 협력이라는 맥락으로 확장하여, 시스템 전체의 성공 확률을 예측하는 모니터로 활용한다.
- **생성 결과의 집계(Aggregating Generations)**: Self-consistency와 같이 여러 결과를 생성해 다수결로 결정하는 방식은 사후적인 처리이며, 액션이 되돌릴 수 없는(Irreversible) 에이전트 환경에서는 적용하기 어렵다. 반면 본 연구의 개입 방식은 문제가 발생하기 전 통신 단계에서 즉각적으로 수행된다.

## 🛠️ Methodology

### 1. 전체 시스템 구조

시스템은 에이전트 집합 $G=\{g_i\}$, 메시지가 저장되는 통신 채널 $C$, 그리고 공유 과제 $T$로 구성된다. 각 에이전트는 자신의 지식 $K_i$를 바탕으로 액션 $A_i$를 수행하며, 이때 액션 선택 확률분포는 다음과 같이 정의된다.
$$P(Action=a|K_i, C_j, T)$$
여기서 $C_j$는 $j$번째 턴의 통신 채널 상태를 의미한다.

### 2. 모니터링 (Monitoring)

모니터는 에이전트의 액션 예측 분포를 관찰하여 해당 턴에서 과제에 성공할 확률 $P(\text{success}|P_{A_i}, j, g_i, T)$을 추정하는 함수이다. 추정된 확률이 특정 임계값 $\tau$보다 낮을 경우 ($\text{probability} < \tau$), Rogue Agent로 판단하고 개입을 트리거한다.

**불확실성 기반 감지 방법:**
에이전트가 액션을 선택할 때 생성하는 확률 분포 $p_i$에서 다음의 세 가지 통계적 지표를 추출한다.

- **Entropy**: 분포의 전반적인 불확실성을 측정한다.
- **Varentropy**: 엔트로피의 변동성을 측정하여 예측의 일관성을 평가한다.
- **Kurtosis**: 분포의 뾰족한 정도(첨도)를 측정하여 특정 토큰으로의 집중도를 확인한다.

이 세 가지 지표의 최댓값과 현재 턴 수를 특성(Feature)으로 사용하여 **Polynomial Ridge Classifier**를 학습시킨다. 이 분류기는 각 턴의 상태를 입력받아 성공 확률을 출력한다.

### 3. 개입 (Intervention)

모니터가 위험 신호를 감지하면 개입 메커니즘이 작동한다. 본 연구에서는 액션을 두 가지 유형으로 구분한다.

- **Reversible Actions**: 단순 정보 공유와 같이 취소 가능한 액션.
- **Irreversible Actions**: 최종 정답 제출이나 자원 소비와 같이 취소 불가능한 액션.

개입이 트리거되면, 시스템은 가장 최근의 Irreversible Action 시점으로 환경을 롤백(Rollback)한다. 즉, 잘못된 정보가 공유된 이후의 Reversible Actions를 모두 삭제하여 에이전트들이 다시 협력할 기회를 제공한다.

## 📊 Results

### 1. 실험 설정

- **WhoDunitEnv**: 용의자 라인업에서 범인을 찾는 게임. 비대칭(Asym: 추궁자와 정보원) 및 대칭(Sym: 정보 공유) 버전으로 구성.
- **CodeGen**: HumanEval 및 LiveCodeBench 데이터셋을 사용하여 다중 에이전트 토론 기반 코드 생성 수행.
- **GovSim**: 자원 지속 가능성을 위한 협력 환경.
- **모델**: LLAMA-3.1-70B, QWEN-2.5-72B, GPT-4O 등 사용.

### 2. 정량적 결과

- **WhoDunitEnv**: Base 모델 대비 성공률(Success-Rate)이 최대 **17.4%** 향상되었다. 특히 비대칭 환경(Asym)에서 두 번의 리셋(Double Reset)을 허용했을 때 더 높은 성능 향상을 보였다.
- **GovSim**: 생존율(Survival-Rate)이 최대 **20.0%** 증가하였으며, GPT-4O의 경우 효율성(Efficiency)이 6.9% 향상되었다.
- **CodeGen**: Pass@1 지표에서 Zero-shot(80.5%) 및 단순 다중 에이전트(81.6%)보다 높은 **83.5%**를 달성하였다. 또한 HumanEval에서 학습된 모니터가 LiveCodeBench에서도 유효함을 보여 일반화 능력을 입증하였다.

### 3. 분석 결과

모니터가 트리거된 사례를 정성 분석한 결과, 다음과 같은 Rogue Agent의 행동이 감지되었다.

- **Hallucination (48%)**: 잘못된 정보를 공유하는 경우.
- **Collapse (16%)**: 이미 정보를 얻었음에도 동일한 질문을 반복하는 경우.
- **Role Loss (8%)**: 자신의 역할(Role)을 잊어버리는 경우.
- **Recall Failure (4%)**: 이전에 공유된 핵심 정보를 기억하지 못하는 경우.

## 🧠 Insights & Discussion

**강점 및 유효성:**
본 연구는 복잡한 프롬프트 수정이나 모델 재학습 없이, 단순한 통계적 지표(Entropy 등)와 경량 분류기만으로도 다중 에이전트 시스템의 실패를 효과적으로 방지할 수 있음을 보여주었다. 특히 할루시네이션이 시스템 붕괴의 가장 큰 원인임을 밝혀냈으며, 이를 실시간으로 감지하여 롤백하는 전략이 매우 유효함을 입증하였다.

**한계 및 논의사항:**

- **컴퓨팅 비용**: 개입(리셋)이 발생할 때마다 턴 수가 증가하므로 추론 시간 비용(Inference-time compute)이 상승한다. 실험 결과 평균 턴 수가 최대 1.9배 증가하였다.
- **데이터 의존성**: 모니터를 학습시키기 위해 성공/실패 라벨이 있는 훈련 데이터셋이 필요하다. 실제 환경에서 이러한 라벨을 수집하는 비용이 높을 수 있으므로, 향후 온라인 학습(Online Learning) 방식의 도입이 필요하다.
- **성능 하한선**: 에이전트 자체가 해당 과제를 전혀 풀 수 없는 수준(Zero accuracy)이라면, 리셋을 하더라도 성능 향상이 없다는 한계가 있다.

## 📌 TL;DR

본 논문은 다중 에이전트 협력 시스템에서 단 한 명의 잘못된 에이전트(Rogue Agent)가 전체 시스템을 무너뜨리는 문제를 해결하기 위해, **실시간 불확실성 모니터링 및 환경 롤백 개입** 방법론을 제안한다. 에이전트의 액션 확률 분포에서 엔트로피 등의 지표를 추출해 실패 가능성을 예측하고, 위험 시 통신 내용을 되돌림으로써 할루시네이션 등의 오류 전파를 차단한다. 이 방법은 WhoDunitEnv, CodeGen, GovSim 등 다양한 환경에서 최대 20%의 성능 향상을 보였으며, 특히 할루시네이션과 같은 전형적인 LLM 오류를 효과적으로 잡아내는 것으로 나타났다. 이는 향후 고신뢰성이 요구되는 자율 에이전트 시스템 설계에 중요한 기여를 할 것으로 보인다.
