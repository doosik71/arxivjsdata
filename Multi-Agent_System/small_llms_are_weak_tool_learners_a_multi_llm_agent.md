# Small LLMs Are Weak Tool Learners: A Multi-LLM Agent

Weizhou Shen, Chenliang Li, Hongzhan Chen, Ming Yan, Xiaojun Quan, Hehong Chen, Ji Zhang, Fei Huang

## 🧩 Problem to Solve

거대 언어 모델(LLM) 에이전트는 외부 도구(예: API, 함수)를 활용하여 LLM의 능력을 확장하지만, 이는 태스크 계획, 도구 호출, 결과 요약과 같은 복합적인 능력을 요구합니다. 특히 작은 규모의 오픈소스 LLM(Small LLM)의 경우, 이러한 모든 능력을 단일 모델로 학습시키는 데 성능 한계가 명확하며, 도구가 자주 업데이트될 때 전체 모델을 재학습해야 하는 비효율성이 발생합니다. 기존의 단일 LLM 접근 방식은 이러한 복잡한 도구 사용 시나리오에서 특히 작은 모델의 경우 성능 제약을 보입니다.

## ✨ Key Contributions

* 작은 LLM이 도구 학습에 취약하다는 점을 입증하고, 기존의 단일 LLM 에이전트보다 우수한 다중 LLM 프레임워크인 α-UMi를 제안합니다.
* 다중 LLM 프레임워크의 효과적인 훈련을 위한 글로벌-투-로컬 점진적 미세 조정(GLPFT) 전략을 제안하며, 이 전략이 프레임워크 성공의 핵심임을 입증합니다.
* 데이터 스케일링 법칙에 대한 심층 분석을 수행하고, 프레임워크의 우수한 성능에 대한 근본적인 이유를 탐구합니다.
* 각 능력에 특화된 작은 LLM들을 활용하여, 더 큰 단일 LLM 에이전트의 성능을 능가할 수 있음을 보여줍니다.

## 📎 Related Works

* **도구 학습 (Tool Learning):** Toolformer (Schick et al., 2023)는 도구 학습의 선구적인 연구로, 이후 실시간 지식, 멀티모달 이해 및 생성, 코드/수학 추론, 특정 도메인 지식 향상을 위해 다양한 외부 도구가 LLM에 통합되었습니다 (e.g., Yang et al., 2023a; Gou et al., 2023; Qin et al., 2023b).
* **LLM 기반 에이전트 (LLM-powered Agents):** ChatGPT 및 GPT-4와 같은 LLM의 역량을 활용하여 BabyAGI, AutoGPT, Voyager, MetaGPT, ChatDev, AutoGen과 같은 다양한 목적의 에이전트 시스템이 개발되었습니다 (e.g., Nakajima, 2023; Wang et al., 2023; Hong et al., 2023).
* **에이전트 능력 향상 기술:** Chain-of-Thought (Wei et al., 2022) 및 Reflexion (Shinn et al., 2023)은 추론 능력을 강화하고, MemoryBank (Zhong et al., 2023)는 장기 기억을 부여하며, FiREACT (Chen et al., 2023)와 AgentTuning (Zeng et al., 2023)은 오픈소스 LLM을 에이전트로 미세 조정하는 데 중점을 둡니다.

## 🛠️ Methodology

본 연구는 복잡한 도구 학습 작업을 세 가지 하위 태스크로 분해하고, 각 하위 태스크를 전담하는 LLM을 할당하는 α-UMi 프레임워크를 제안합니다. 이를 효과적으로 훈련하기 위해 글로벌-투-로컬 점진적 미세 조정(GLPFT) 전략을 사용합니다.

### α-UMi 프레임워크

α-UMi는 플래너($M_{plan}$), 콜러($M_{call}$), 요약기($M_{sum}$) 세 가지 LLM 구성 요소로 이루어져 있으며, 각 모델은 고유한 역할과 훈련 목표를 가집니다.

* **플래너 (Planner):**
  * **역할:** 태스크 계획 및 의사 결정의 "두뇌" 역할을 합니다. 현재 시스템 상태를 기반으로 합리적인 근거($r_t$)를 생성하고, 다음 단계(콜러 호출, 요약기 호출, 실행 종료)를 결정합니다.
  * **입력:** 시스템 프롬프트 $P_{plan}$, 사용자 지시 $q$, 이전 실행 궤적 $\tau_{t-1}$.
  * **수식:** $r_t = M_{plan}(P_{plan}, \tau_{t-1}, q)$
* **콜러 (Caller):**
  * **역할:** 플래너의 지시에 따라 특정 도구를 호출하기 위한 유효한 액션($a_t$)을 생성합니다. 도구 사용에 필요한 요청 작성에 집중합니다.
  * **입력:** 시스템 프롬프트 $P_{call}$, 사용자 지시 $q$, 이전 실행 궤적 $\tau_{t-1}$, 플래너의 현재 근거 $r_t$.
  * **수식:** $a_t = M_{call}(P_{call}, \tau_{t-1}, q, r_t)$
* **요약기 (Summarizer):**
  * **역할:** 플래너의 안내를 받아 전체 실행 궤적을 요약하여 최종 사용자 답변($a_n$)을 생성합니다.
  * **입력:** 시스템 프롬프트 $P_{sum}$, 이전 실행 궤적 $\tau_{n-1}$, 사용자 지시 $q$, 최종 근거 $r_n$.
  * **수식:** $a_n = M_{sum}(P_{sum}, \tau_{n-1}, q, r_n)$

### 글로벌-투-로컬 점진적 미세 조정 (GLPFT) 전략

GLPFT는 두 단계로 구성되어, 전체 태스크에 대한 이해를 먼저 높인 후 각 하위 태스크에 대한 능력을 특화시킵니다.

1. **글로벌 미세 조정 (Global Fine-tuning):**
    * 백본 LLM (예: LLaMA-2-chat)을 전체 도구 학습 데이터셋에 대해 미세 조정합니다. 이 단계에서는 하위 태스크를 구분하지 않고 모델이 합리적인 근거, 액션, 최종 답변을 순차적으로 출력하도록 학습시켜 태스크에 대한 포괄적인 이해를 형성합니다.
2. **로컬 미세 조정 (Local Fine-tuning):**
    * 1단계에서 미세 조정된 백본 LLM의 세 사본을 각각 플래너, 콜러, 요약기로 인스턴스화합니다.
    * 훈련 데이터셋을 각 LLM의 역할에 맞춰 재구성하고, 각 모델을 해당 하위 태스크 데이터셋에 대해 추가로 미세 조정합니다. 이때, 사용자 지시는 1단계와 동일한 것을 재사용하여 데이터 분포의 영향을 완화하고, 각 역할에 특화된 시스템 프롬프트와 마스킹을 통한 목표(rationale, action, final answer 생성)를 사용하여 미세 조정합니다.

## 📊 Results

* **전반적인 성능 향상:** α-UMi는 ToolBench 및 ToolAlpaca 벤치마크에서 기존 단일 LLM 접근 방식보다 뛰어난 성능을 보였습니다. 특히 계획 정확도(Plan ACC), 액션 정확도(Act. EM), 환각(Hallu.) 감소, 인자 F1 점수(Arg. F1)에서 상당한 개선을 보였습니다.
* **작은 LLM의 효율성:** 7B 백본을 사용하는 α-UMi가 13B 백본을 사용하는 Single-LLM보다 우수한 성능을 달성하여, 더 작은 LLM으로도 경쟁력 있는 전체 성능을 낼 수 있음을 입증했습니다. 이는 Multi-LLM 시스템 설계의 핵심 이점입니다.
* **GLPFT 전략의 효과:** GLPFT 없이 단일 단계로 각 LLM을 하위 태스크에 대해 직접 미세 조정한 Multi-LLM$_{one-stage}$나 멀티태스크 방식으로 훈련한 Single-LLM$_{multi-task}$은 성능이 저조하거나 불안정하여, GLPFT 전략이 프레임워크 성공에 필수적임을 보여주었습니다.
* **데이터 스케일링 법칙:** α-UMi는 훈련 데이터 규모가 증가함에 따라 모든 메트릭에서 일관되게 성능이 향상되는 반면, Single-LLM은 각 메트릭에서 최적의 데이터 규모가 달랐습니다.
* **실시간 평가:** ToolBench의 실시간 평가에서 α-UMi (7B)는 ChatGPT 및 ToolLLaMA를 Pass Rate(+6.1% 및 +10.2%) 및 Win Rate(+1.6% 및 +5.9%)에서 크게 능가했으며, GPT-4와 비슷한 Pass Rate를 달성했습니다.
* **수학 추론 능력:** MATH 및 GSM8K 벤치마크에서 프로그램 지원 에이전트 설정에서도 α-UMi가 기존 기준 모델들을 능가하는 정확도를 보였습니다.
* **비용 효율성:** α-UMi는 Single-LLM보다 3배의 저장 공간, 1.3배의 훈련 컴퓨팅 자원, 1.5배의 훈련 시간을 요구하지만, 7B α-UMi는 13B Single-LLM보다 낮은 총 비용으로 더 높은 성능을 달성할 수 있음을 보여주었습니다. 추론 시간은 두 방식이 유사했습니다.

## 🧠 Insights & Discussion

* **성능 향상 메커니즘:** α-UMi의 성공은 복잡한 도구 학습 태스크를 플래너, 콜러, 요약기로 분해하여 각 작은 LLM이 특정 하위 태스크에 집중하도록 함으로써, 단일 LLM의 능력 상한선을 넘어설 수 있게 했다는 점에 있습니다. 특히 플래너와 콜러의 분리가 주요 성능 향상 요인으로 분석됩니다.
* **모듈성 및 유연성:** 이 모듈식 프레임워크는 각 구성 요소를 개별적으로 업데이트하고, 각 LLM의 역할에 맞춰 프롬프트 디자인을 최적화할 수 있는 유연성을 제공합니다. 이는 실제 시나리오에서 도구가 자주 변경될 때 효과적일 수 있습니다.
* **작은 LLM의 가능성:** 본 연구는 작은 LLM이 전담 역할을 수행할 경우, 더 큰 단일 LLM에 의존하는 에이전트 프레임워크를 능가하는 성능을 발휘할 수 있음을 강력하게 시사합니다. 이는 자원 효율적인 LLM 에이전트 개발에 중요한 함의를 가집니다.
* **제한 사항:** 향후 연구에서는 GPT-4와 같은 강력한 폐쇄형 LLM과 작은 LLM을 결합하는 "대형 + 소형" 협업 모델을 탐색하고, 프레임워크의 유연성을 더욱 최적화하여 더 광범위한 에이전트 태스크에 적용할 수 있는 방안을 모색할 수 있습니다.

## 📌 TL;DR

**문제:** 작은 LLM은 태스크 계획, 도구 호출, 결과 요약 등 복합적인 도구 학습 능력에서 성능 한계를 보인다.

**제안 방법:** 본 논문은 이러한 도구 학습 태스크를 `플래너`, `콜러`, `요약기` 세 가지 전담 LLM으로 분해하는 다중 LLM 에이전트 프레임워크 `α-UMi`를 제안한다. 훈련에는 `글로벌-투-로컬 점진적 미세 조정(GLPFT)` 전략을 사용하여, 먼저 백본 LLM을 전체 태스크에 대해 학습시킨 후, 각 전담 LLM을 해당 하위 태스크에 대해 추가로 미세 조정하여 효율성과 전문성을 극대화한다.

**주요 결과:** `α-UMi`는 ToolBench 및 ToolAlpaca 벤치마크에서 7B 모델로도 13B 단일 LLM 에이전트를 능가하는 뛰어난 성능을 보였다. 특히 계획 정확도, 액션 정확도, 환각 감소 등 도구 활용 능력을 크게 향상시켰다. 이는 복잡한 도구 학습 작업을 여러 전문 LLM에 분산함으로써 각 모델의 한계를 극복하고, 작은 LLM들을 활용하여 비용 효율적이면서도 강력한 에이전트 시스템을 구축할 수 있음을 입증한다.
