# MAC: Multi-Agent Constitution Learning

Rushil Thareja, Gautam Gupta, Francesco Pinto, Nils Lukas (2026)

## 🧩 Problem to Solve

본 논문은 LLM의 동작을 제어하기 위해 자연어 규칙 집합을 사용하는 Constitutional AI의 규칙 생성 과정을 자동화하는 문제를 다룬다. 일반적으로 이러한 규칙(Constitution)은 인간 전문가가 직접 작성하지만, 이는 매우 느리고 비용이 많이 들며, 도메인이나 작업 정의가 변경될 때마다 반복해야 하는 번거로움이 있다.

기존의 Automatic Prompt Optimization (APO) 방법들은 이를 자동화하려 시도했으나, 몇 가지 결정적인 한계가 존재한다. 첫째, 많은 양의 라벨링된 데이터가 필요하다. 둘째, 최적화된 프롬프트가 구조화되지 않은 텍스트 형태여서 해석 가능성이 떨어지며, 프롬프트의 길이가 길어질수록 성능 향상 폭이 줄어드는 한계가 있다. 결과적으로 기존 방식들은 Constitutional AI의 핵심 가치인 '감사 가능성(Auditability)'과 '투명성'을 훼손한다. 따라서 본 연구의 목표는 제한된 데이터만으로도 인간이 읽고 수정할 수 있는 구조화된 규칙 집합을 자동으로 도출하는 'Constitution Learning'을 구현하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 프롬프트를 하나의 거대한 텍스트 덩어리로 최적화하는 대신, 이를 개별 규칙들의 집합인 모듈형 구조로 취급하고, 이를 관리하는 전문 에이전트 네트워크를 구축하는 것이다.

중심적인 설계 아이디어는 다음과 같다.

1. **구조화된 규칙 최적화**: 프롬프트를 단순한 텍스트가 아닌, 추가(Add), 수정(Edit), 삭제(Remove)가 가능한 개별 규칙의 집합으로 정의하여 투명성을 유지한다.
2. **Multi-Agent 협업 체계**: 최적화 과정을 결정(Decision), 생성(Creator), 수정(Editor), 실행(Annotator)이라는 네 가지 전문 역할로 분해하여, 각 에이전트가 단순하고 명확한 하위 작업에 집중하게 함으로써 모델의 추론 부담을 줄이고 신뢰성을 높인다.
3. **에이전트 전문화 및 증강**: 더 큰 모델의 성공적인 궤적을 학습시켜 작은 모델의 성능을 높이는 MAC+와, 추론 시점에 유사 사례를 참조하는 reMAC을 제안하여 실용성을 극대화한다.

## 📎 Related Works

본 논문은 다음과 같은 관련 연구들을 배경으로 한다.

- **Constitutional AI**: 자연어 규칙을 통해 모델의 가이드라인을 설정하는 방법론이다. 하지만 규칙을 직접 작성하는 과정의 고비용 문제가 존재한다.
- **Automatic Prompt Optimization (APO)**: GEPA나 MIPRO와 같이 LLM을 이용해 프롬프트를 반복적으로 개선하는 방식이다. 이들은 성능 향상을 꾀하지만, 결과물이 구조화되지 않아 사람이 이해하거나 수정하기 어렵다는 한계가 있다.
- **Private Information Tagging (PII)**: 문서 내 민감 정보(이름, 주소 등)를 식별하는 작업이다. 도메인마다 개인정보의 정의가 다르고, 데이터 보안 문제로 인해 모델 가중치를 직접 수정하는 파인튜닝(Fine-tuning)이 위험할 수 있어 추론 시점의 규칙 적용이 매우 중요하다.

## 🛠️ Methodology

### 전체 시스템 구조 및 최적화 목표

Constitution Learning의 목적은 검증 데이터셋 $D_{val}$에 대해 작업 성능 지표 $\mu$를 최대화하는 최적의 규칙 집합 $C^*$를 찾는 것이다. 이를 수식으로 나타내면 다음과 같다.

$$C^* = \arg \max_{C \in \mathcal{C}} \mu (A_{\theta, \pi}(D_{val}; C), Y_{val}^*)$$

여기서 $A_{\theta, \pi}$는 고정된 가중치 $\theta$와 역할 프롬프트 $\pi$를 가진 LLM 에이전트이며, $C$는 학습 대상인 규칙 집합이다.

### Multi-Agent 구성 요소 및 역할

MAC은 다음과 같은 네 가지 전문 에이전트로 구성된다.

1. **ANNOTATOR(B, C)**: 현재 규칙 집합 $C$를 적용하여 배치 $B$에 대해 예측 결과 $\hat{Y}$를 생성한다.
2. **DECISION(FN, FP, C, $R_{max}$)**: 예측 결과와 정답을 비교하여 발생한 False Negative(FN)와 False Positive(FP) 오류를 분석한다. 이후 규칙을 추가(ADD), 수정(EDIT), 또는 삭제(REMOVE)할지 결정하고 그 근거(Rationale)를 제시한다.
3. **CREATOR(FN, FP, r)**: DECISION 에이전트가 ADD를 결정했을 때, 오류 패턴과 근거 $r$을 바탕으로 새로운 규칙을 작성한다.
4. **EDITOR($c_i$, FN, FP, r)**: DECISION 에이전트가 EDIT를 결정했을 때, 기존의 특정 규칙 $c_i$를 수정하여 오류를 해결한다.

### 학습 및 최적화 절차

MAC의 최적화 과정은 다음과 같은 반복 루프를 따른다.

1. **실행 및 평가**: ANNOTATOR가 규칙을 적용해 결과를 내면, `MATCH` 함수가 F1 스코어를 계산하고 FN/FP 샘플을 추출한다.
2. **전략 결정**: DECISION 에이전트가 오류 프로파일을 분석해 업데이트 연산을 선택한다.
3. **규칙 제안**: 선택된 연산에 따라 CREATOR가 새 규칙을 만들거나 EDITOR가 기존 규칙을 수정하여 후보 규칙 집합 $C^{prop}$를 생성한다.
4. **검증 및 수용**: $C^{prop}$를 동일 배치에 적용해 성능이 개선($\mu^{prop} > \mu$)되면 업데이트를 수용하고, 그렇지 않으면 온도를 높여 재시도하거나 기존 규칙을 유지한다.

### 확장 변형 모델

- **MAC+**: 작은 모델(예: 3B)이 복잡한 규칙 생성 및 진단에 어려움을 겪는 문제를 해결하기 위해, 큰 모델(예: 14B)이 생성한 성공적인 실행 궤적(Trajectories)을 수집하여 작은 모델의 각 에이전트를 Supervised Fine-Tuning(SFT) 하는 방식이다. 이는 도메인 지식이 아닌 '작업 구조'를 학습시키는 데 목적이 있다.
- **reMAC (Retrieval-augmented MAC)**: 학습된 규칙만으로는 특수 용어나 문맥 의존적 정의를 처리하기 어려울 수 있다. 이를 위해 추론 시점에 학습 데이터에서 유사한 예시 $k$개를 검색하여 퓨샷(few-shot) 예제로 추가 제공한다.

## 📊 Results

### 실험 설정

- **데이터셋**: 법률(ECHR), 의료(MACCROBAT), 금융(PUPA) 도메인의 PII 태깅 데이터셋을 사용하였다. 각 도메인당 192개의 문서를 사용했으며, 160개 학습, 16개 검증, 16개 테스트로 분할하였다.
- **모델**: Qwen 2.5 Instruct (3B, 7B, 14B) 모델을 사용하였다.
- **비교 대상**: 사전 학습된 태거(Microsoft Presidio, Nvidia GLiNER), 프롬프트 최적화 도구(GEPA, MIPRO), 학습 기반 방법(SFT, GRPO)과 비교하였다.
- **지표**: 정확한 스팬 레벨 매칭을 통한 F1 스코어를 측정하였다.

### 주요 결과

1. **PII 태깅 성능**: MAC(14B)은 법률(+17%), 금융(+2%), 의료(+123%) 모든 도메인에서 사전 학습된 태거들보다 높은 성능을 보였다. 특히 의료 도메인에서 큰 격차를 보였는데, 이는 사전 학습 모델이 세밀한 의료 프라이버시 유형을 잡지 못하는 반면 MAC은 규칙을 통해 이를 학습했기 때문이다.
2. **프롬프트 최적화 대비 우위**: MAC은 대부분의 설정에서 GEPA와 MIPRO를 압도하였다. 특히 3B 모델의 법률 데이터셋에서 GEPA(13.15 F1) 대비 MAC(36.00 F1)이 174% 이상의 성능 향상을 보였다.
3. **reMAC의 효과**: 유사도 기반 검색을 통한 예시 추가 시 성능이 크게 향상되었으며, 특히 의료 도메인 7B 모델에서 F1 스코어가 9.91에서 36.97로 급증하였다.
4. **MAC+의 효율성**: 3B 모델에 MAC+를 적용했을 때, SFT(56.59 F1)와 GRPO(39.68 F1)보다 높은 59.02 F1을 달성하였다. 이는 에이전트 수준의 전문화(Distillation)가 단순 모델 튜닝보다 효과적임을 시사한다.
5. **작업 일반화**: 툴 호출(Tool-calling) 벤치마크(BFCL)에 적용한 결과, Baseline(82%) 대비 87.5%의 정확도를 기록하며 GEPA 및 MIPRO보다 우수한 성능을 보여, 분류 작업 외의 에이전트 작업으로도 확장 가능함을 입증하였다.

## 🧠 Insights & Discussion

### 분석 및 강점

MAC의 가장 큰 강점은 **'문제의 분해(Decomposition)'**에 있다. 전체 프롬프트를 다시 쓰는 대신, 규칙의 추가/수정/삭제라는 단순한 작업으로 쪼개어 전문 에이전트에게 맡김으로써, 추론 능력이 낮은 소형 모델(3B)에서도 고성능의 규칙 집합을 찾아낼 수 있었다. 또한, 결과물이 명시적인 규칙 형태이므로 사람이 직접 검토하고 수정할 수 있어 고위험 도메인(의료, 법률)에서 매우 유리하다.

### 한계 및 비판적 해석

1. **규칙 수와 투명성의 트레이드-오프**: 규칙의 수가 많아질수록 성능(Utility)은 올라가지만, 사람이 검토해야 할 양이 많아져 투명성이 낮아지는 문제가 발생한다. 논문에서는 이에 대한 정량적 분석이나 사용자 연구가 부족하다.
2. **지표 의존성**: MAC은 $\mu$라는 작업 지표에 전적으로 의존하여 규칙을 업데이트한다. 만약 지표 자체가 노이즈가 많거나 실제 목적과 정렬되지 않았다면, 모델은 실제 성능 향상이 아닌 '지표 최적화'만을 수행하는 과적합(Overfitting) 상태에 빠질 위험이 있다.
3. **규칙 간 상충 가능성**: 규칙 집합이 커질수록 새로운 규칙이 기존 규칙과 충돌하거나 예상치 못한 부작용을 일으킬 수 있으나, 이에 대한 충돌 해결 메커니즘은 명시되지 않았다.

## 📌 TL;DR

본 논문은 LLM의 동작을 제어하는 '헌법(Constitution)' 규칙을 자동으로 학습하는 **Multi-Agent Constitutional Learning (MAC)** 프레임워크를 제안한다. MAC은 프롬프트를 모듈형 규칙 집합으로 정의하고, 이를 관리하는 전문 에이전트(Decision, Creator, Editor, Annotator)를 통해 반복적으로 최적화한다. 실험 결과, MAC은 기존의 비구조적 프롬프트 최적화 방식보다 월등히 높은 성능과 투명성을 보였으며, 특히 소형 모델에서도 효과적으로 작동함을 증명하였다. 이 연구는 향후 LLM의 제어 가능성을 높이고, 사람이 감사 가능한 투명한 AI 시스템을 구축하는 데 중요한 기반이 될 것으로 보인다.
