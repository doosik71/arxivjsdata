# ARCANE: A Multi-Agent Framework for Interpretable and Configurable Alignment

Charlie Masters, Marta Grzeskiewicz, Stefano V. Albrecht (2025)

## 🧩 Problem to Solve

본 논문은 대규모 언어 모델(LLM) 기반의 에이전트가 장기 작업(long-horizon tasks)을 수행할 때, 이해관계자(stakeholder)의 선호도에 맞게 에이전트를 정렬(alignment)하는 문제에 집중한다.

기존의 정렬 방식은 주로 불투명한 보상 모델(reward models)을 사용하여, 이해관계자가 모델의 목표를 이해하거나 감사(audit)하기 어렵다는 한계가 있다. 또한, 대부분의 정렬 기법은 학습 단계에서 고정된 선호도를 최적화하므로, 추론 시점(test-time)에 선호도가 변경되었을 때 모델을 재학습시키지 않고는 대응할 수 없다는 경직성을 가진다.

따라서 본 연구의 목표는 이해관계자가 이해할 수 있고, 감사 가능하며, 재학습 없이도 추론 시점에 동적으로 구성(configurable) 가능한 정렬 프레임워크를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 정렬 문제를 **다중 에이전트 협업 문제**로 재정의하고, 이해관계자의 선호도를 자연어 형태의 **루브릭(Rubrics)**으로 표현하는 것이다. 루브릭은 검증 가능한 기준들의 가중치 집합으로, 이를 통해 보상 모델을 해석 가능한 형태로 변환한다.

주요 기여 사항은 다음과 같다.

- **ARCANE 프레임워크 제안**: 매니저 에이전트가 이해관계자와의 상호작용을 통해 루브릭을 생성하고, 이를 통해 워커 에이전트들을 제어하는 구조를 설계하였다.
- **2단계 학습 절차**: 합성 데이터를 이용한 지도 미세 조정(SFT)과, 정규화된 Group-Sequence Policy Optimization(GSPO)을 통한 강화 학습 단계를 도입하여 루브릭 생성 능력을 최적화하였다.
- **동적 구성 가능성**: 학습된 루브릭은 추론 시점에 가중치를 조정하거나 기준을 수정함으로써, 재학습 없이도 '정확성 vs 간결성'과 같은 트레이드오프를 제어할 수 있게 한다.

## 📎 Related Works

본 논문은 기존의 정렬 접근 방식을 세 가지 관점에서 분석하고 차별점을 제시한다.

1. **학습 시점 정렬 (Training-Time Alignment)**: RLHF 및 그 변형들은 고정된 학습 선호도를 최적화하므로 선호도 드리프트(preference drift)에 취약하며, 보상 과최적화(reward over-optimization) 문제로 인해 불투명한 프록시 보상을 이용하는 경향이 있다.
2. **추론 시점 보상 모델링 (Test-Time Reward Modeling)**: GenRM이나 GRAM과 같은 생성적 보상 모델은 유연성을 제공하지만, 어떤 기준이 평가를 주도하는지, 가중치가 어떻게 설정되었는지 알 수 없는 불투명성(opacity) 문제가 여전하다.
3. **해석 가능한 정렬 (Interpretable Alignment)**: 기존의 루브릭 기반 방식(Open-Rubrics 등)은 루브릭을 정적인 도구로 취급한다. 반면, ARCANE은 루브릭 생성 자체를 정책 최적화 문제로 다루어, 이해관계자와의 상호작용을 통해 루브릭을 동적으로 학습하고 적응시킨다.

## 🛠️ Methodology

### 전체 시스템 구조 및 역할 정의

ARCANE은 세 가지 역할의 에이전트로 구성된다.

- **이해관계자 (Stakeholder, $S$)**: 잠재적인 유틸리티 함수 $U^*(y | x)$를 가지고 있으며, 출력물 $y$에 대한 선호도를 결정한다.
- **매니저 (Manager, $M$)**: 이해관계자와 상호작용하여 선호도를 이끌어내고, 이를 루브릭 형태로 변환하여 워커에게 전달한다.
- **워커 (Workers, $W$)**: 매니저가 제공한 루브릭을 지침으로 삼아 최종 출력물 $y$를 생성한다.

### 루브릭 표현 및 보상 계산

루브릭 $R$은 다음과 같이 가중치가 부여된 검증 기준들의 집합으로 정의된다.
$$R = \{(c_j, w_j)\}_{j=1}^M$$
여기서 $c_j$는 자연어로 된 측정 가능한 기준이며, $w_j$는 해당 기준의 상대적 중요도를 나타내는 가중치($\sum w_j = 1$)이다. 각 기준은 검증기(verifier) $\nu_j(c_j, x, y) \in [0, 1]$를 통해 점수화되며, 전체 프록시 유틸리티 $\hat{u}$는 다음과 같이 선형 합산으로 계산된다.
$$\hat{u}_\phi(y | x) = \sum_{j=1}^M w_j \nu_j(c_j, x, y)$$

### 학습 절차: 2단계 커리큘럼

매니저의 루브릭 생성기 $D_\phi$는 다음의 과정을 통해 학습된다.

**1단계: 지도 미세 조정 (SFT)**
합성된 매니저-이해관계자 대화 데이터를 사용하여 루브릭의 구조와 대화 형식을 익히게 한다. 표준적인 언어 모델링 손실 함수를 사용하여 학습한다.
$$L_{SFT}(\phi) = -\mathbb{E}_{(x, R^*)} \left[ \sum_t \log \pi_\phi(r_\tau | r_{<t}, x) \right]$$

**2단계: Group-Sequence Policy Optimization (GSPO)**
SFT 이후, 루브릭의 유용성을 극대화하기 위해 강화 학습을 수행한다. $K$개의 루브릭 후보 $\{R_k\}$를 생성하고, 각 루브릭 하에서 워커가 생성한 결과물의 실제 유틸리티 $r_k = U^*(y_k | x)$를 보상으로 사용한다.
최적화 목표 함수 $J_{GSPO}(\phi)$는 다음과 같다.
$$J_{GSPO}(\phi) = \mathbb{E}_x \left[ \frac{1}{K} \sum_{k=1}^K \min \left( s_k(\phi) \hat{A}_k, \text{clip}(s_k(\phi), 1-\epsilon, 1+\epsilon) \hat{A}_k \right) - \beta D_{KL}(\pi_\phi \| \pi_{ref}) - C_{clarify}(q_{1:T}) - C_{compute}(R_k) \right]$$

여기서 $s_k(\phi)$는 시퀀스 수준의 중요도 비율(importance ratio)이며, $\hat{A}_k$는 그룹 내 표준화된 이점(advantage)이다. 특히 본 모델은 루브릭의 효율성을 위해 다음의 페널티 항을 추가한다.

- **질의 비용 ($C_{clarify}$)**: 이해관계자에게 질문하는 횟수와 난이도에 따른 인지적 부담을 페널티로 부여한다.
- **계산 비용 ($C_{compute}$)**: 루브릭 검증에 소요되는 API 비용과 실행 시간을 페널티로 부여한다.

### 추론 시점 제어 (Test-Time Steering)

학습된 매니저는 루브릭 $R^*$를 생성하고, 워커는 이를 프롬프트로 받아 출력을 생성한다. 또한, 생성된 루브릭의 점수 $\hat{u}_\phi$를 이용해 다음과 같은 전략을 사용할 수 있다.

- **Best-of-K Sampling**: $K$개의 후보를 생성한 후 루브릭 점수가 가장 높은 것을 선택한다.
- **Importance Reweightng**: 루브릭 점수에 비례하여 샘플을 재추출한다.

## 📊 Results

### 실험 설정

- **데이터셋**: GDPVal 벤치마크 (실제 세계의 복잡한 작업 219개 중 44개 평가 세트 사용).
- **평가 지표**: 평균 실제 리턴($U^*$), NDCG@8(루브릭 기반 순위와 실제 선호도 순위의 일치도), 루브릭의 간결성 및 가독성.
- **비교 대상**: No Rubric(기본 생성), SFT-based Rubric, GSPO-based Rubric, Oracle Rubric(정답 루브릭 사용).

### 주요 결과

1. **유용성 (Usefulness)**: 루브릭 기반 가이드가 없는 경우보다 성능이 일관되게 높았다. $N=8$ 샘플링 시, 평균 리턴은 No Rubric (0.58) $\rightarrow$ SFT (0.68) $\rightarrow$ GSPO (0.74) $\rightarrow$ Oracle (0.81) 순으로 상승하였다. 특히 GSPO는 SFT 대비 통계적으로 유의미한 성능 향상을 보였다.
2. **충실도 (Faithfulness)**: NDCG@8 측정 결과, GSPO 루브릭(0.8722)이 SFT(0.8103) 및 Base(0.7998)보다 훨씬 높게 나타나, 실제 이해관계자의 선호 순위를 더 정확하게 모사함을 확인하였다.
3. **도메인별 경향**: 법률/컴플라이언스(+12.5%) 및 콘텐츠/커뮤니케이션(+11.5%) 등 주관적이고 다차원적인 품질 평가가 필요한 도메인에서 GSPO의 효과가 가장 컸다. 반면 데이터 분석과 같은 객관적 작업에서는 성능 향상이 미미했다.
4. **해석 가능성 (Interpretability)**: GSPO로 생성된 루브릭은 Oracle 루브릭과 유사하게 루브릭당 약 12개의 기준을 가지며, 문장이 더 간결하고 명령형으로 작성되어 가독성과 감사 가능성이 높았다.

## 🧠 Insights & Discussion

### 강점 및 의의

ARCANE은 보상 모델을 '블랙박스'에서 '자연어 루브릭'으로 전환함으로써, AI 정렬의 투명성을 획기적으로 높였다. 특히 매니저 에이전트가 이해관계자와의 대화를 통해 루브릭을 최적화하는 과정은, 정렬을 단순한 파라미터 튜닝이 아닌 **통신과 추론의 과정**으로 정의했다는 점에서 학술적 가치가 크다.

### 한계 및 비판적 해석

1. **단일 워커 중심의 평가**: 이론적으로는 다중 워커 협업이 가능하도록 설계되었으나, 실제 실험은 단일 워커를 대상으로 수행되었다. 다중 에이전트 환경에서의 조정(coordination) 역동성은 아직 검증되지 않았다.
2. **가짜 기준(Spurious Criteria)의 위험**: 매니저가 실제 유틸리티와 인과 관계가 없지만 통계적으로 상관관계가 높은 '가짜 기준'을 학습할 가능성이 있다. 이를 방지하기 위한 인과적 제약 조건(causal penalties)의 도입이 필요하다.
3. **계산 비용**: 루브릭 기반의 Best-of-K 샘플링과 검증기 실행은 추론 시점의 계산 비용과 지연 시간을 증가시킨다.

## 📌 TL;DR

본 논문은 이해관계자의 선호도를 가중치 기반의 자연어 루브릭으로 변환하여 에이전트를 정렬하는 **ARCANE** 프레임워크를 제안한다. 매니저 에이전트가 SFT와 GSPO를 통해 최적의 루브릭을 생성하도록 학습되며, 이를 통해 **재학습 없는 추론 시점의 동적 정렬**과 **보상 모델의 해석 가능성**을 동시에 확보하였다. 특히 주관적인 품질 판단이 중요한 복잡한 작업에서 탁월한 성능을 보이며, 향후 인간-AI 팀의 투명한 협업 체계를 구축하는 데 중요한 기여를 할 것으로 기대된다.
