# Agent WARPP: Workflow Adherence via Runtime Parallel Personalization

Maria Emilia Mazzolenis, Ruirui Zhang (2025)

## 🧩 Problem to Solve

본 논문은 작업 지향 대화(Task-Oriented Dialogue, TOD) 시스템에서 거대 언어 모델(LLM)이 겪는 **긴 조건부 워크플로우(long, conditional workflows) 준수 문제**를 해결하고자 한다. 실제 환경의 TOD 시스템은 외부 도구 호출(tool calls)이 포함된 복잡한 절차를 따르며, 사용자의 속성이나 상태에 따라 실행 경로가 달라지는 조건부 로직을 가진다.

이러한 복잡한 워크플로우에서 LLM은 다음과 같은 한계를 보인다.

1. **추론 오버헤드 및 성능 저하**: 입력 컨텍스트가 길어질수록 추론 성능이 떨어지며, 특히 다단계 추론(multi-hop reasoning)에서 어려움을 겪는다.
2. **신뢰성 부족**: 도구를 잘못된 순서로 호출하거나, 불필요한 도구를 사용하고, 때로는 환각(hallucination) 현상으로 인해 실행 불가능한 작업을 수행하려 한다.
3. **개인화의 어려움**: 사용자의 계정 유형, 선호도 등 실시간 속성에 따라 워크플로우를 동적으로 조정하는 능력이 부족하며, 이를 위해 추가 학습이 필요한 경우가 많다.

따라서 본 연구의 목표는 추가 학습 없이(training-free) 런타임에서 사용자의 특성에 맞게 워크플로우를 최적화하여, 실행 정확도를 높이고 추론 비용(토큰 사용량)을 줄이는 프레임워크를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **런타임 병렬 개인화(Runtime Parallel Personalization)**를 통해 LLM이 탐색해야 할 추론 공간을 획기적으로 줄이는 것이다.

중심적인 설계 아이디어는 다음과 같다.

- **동적 워크플로우 가지치기(Dynamic Workflow Pruning)**: 전체 워크플로우를 그대로 제공하는 대신, 런타임에 사용자의 속성을 기반으로 불필요한 조건부 분기(branch)를 제거한 '최적화된 경로'만을 생성하여 전달한다.
- **병렬 에이전트 구조**: 사용자 인증(Authentication)이 진행되는 동안 개인화 에이전트(Personalizer Agent)가 병렬로 워크플로우를 가지치기함으로써, 전체 시스템의 지연 시간(latency)을 증가시키지 않으면서 효율성을 높인다.
- **모듈형 오케스트레이션**: 의도 파악, 인증, 개인화, 실행을 각각의 전문 에이전트에게 분산시켜 각 에이전트의 컨텍스트 윈도우를 좁히고 전문성을 높였다.

## 📎 Related Works

본 논문은 기존의 LLM 워크플로우 준수 및 개인화 접근 방식의 한계를 지적하며 차별점을 제시한다.

1. **워크플로우 준수 및 도구 활용**: ReAct, Toolformer와 같은 도구 증강 방법론은 LLM의 능력을 확장했지만, 액션 공간(action space)이 커질수록 도구 오용이나 환각 문제가 발생한다. CoT(Chain-of-Thought) 등의 추론 기법 역시 실행 제약 조건과 완전히 일치하지 않는 경우가 많다.
2. **개인화 방법론**: 기존의 개인화는 모델 중심(Fine-tuning, In-context learning)이나 상호작용 수준(정적 프로필 기반 경로 선택)으로 나뉜다. AFLOW와 같은 최신 연구는 워크플로우를 최적화하지만, 이는 오프라인 과정이며 런타임에 동적으로 구조를 변경하지는 않는다.
3. **에이전트 기반 구조**: Multi-agent 시스템(CAMEL, AutoGen)은 복잡한 작업을 분산 처리하여 신뢰성을 높였으나, 본 논문은 여기서 더 나아가 **런타임 병렬화**와 **개인화된 워크플로우 생성**을 결합하여 동적인 적응력을 확보했다.

## 🛠️ Methodology

### 전체 시스템 구조 (Architecture)

WARPP는 OpenAI Agents SDK를 기반으로 하며, 네 가지 핵심 에이전트가 협력하는 구조이다.

1. **Orchestrator Agent**: 사용자의 입력에서 의도(Intent)를 파악하고, 해당 도메인의 전체 워크플로우와 도구 세트를 호출한 뒤 제어권을 Authenticator와 Personalizer에게 넘긴다.
2. **Authenticator Agent**: 보안을 위해 사용자 신원을 확인한다. MFA(다요소 인증) 도구를 사용하여 인증을 수행하며, 이 과정은 Personalizer의 작업과 병렬로 진행된다.
3. **Personalizer Agent (Parallelized)**: 본 프레임워크의 핵심으로, 정보 수집 도구로 얻은 사용자 데이터를 바탕으로 전체 워크플로우를 다음의 3단계로 변환한다.
    - **Static Pruning**: 사용자 속성과 맞지 않는 분기와 도구 호출을 제거하고, 이미 알고 있는 값은 인라인(inline)으로 처리한다.
    - **Fidelity Preservation**: 도구 호출 주변의 성공, 실패, 사용자 응답 등의 결과 분기는 그대로 유지하여 실행 신뢰성을 보장한다.
    - **Cleanup and Formatting**: 설명 단계를 병합하고 지침 번호를 재정렬하여 간결한 마크다운 형식으로 출력한다.
4. **Fulfillment Agent**: Personalizer가 생성한 '가지치기 된 워크플로우'와 '필터링 된 도구 세트'만을 전달받아 최종 과업을 수행한다.

### 추론 복잡도 감소의 수학적 근거

전체 워크플로우의 토큰 수를 $T$, 결정 지점(decision point)의 평균 토큰 수를 $t$, 각 지점에서의 최대 분기 수를 $b$라고 가정할 때, 결정 지점의 수 $n$은 다음과 같다.
$$n \approx \frac{T}{t}$$

최악의 경우, 가지치기 되지 않은 워크플로우의 잠재적 실행 경로는 $b^n \approx b^{T/t}$로 기하급수적으로 증가한다. WARPP는 런타임에 사용자 속성을 평가하여 만족되지 않는 분기를 즉시 제거함으로써, LLM이 탐색해야 할 조합 공간(combinatorial space)을 획기적으로 줄인다.

개인화 단계의 시간 복잡도는 워크플로우 스캔에 $O(T)$, 도구 필터링에 $O(m)$이 소요되어 총 $O(T+m)$의 선형 복잡도를 가지므로 매우 효율적이다.

## 📊 Results

### 실험 설정

- **도메인 및 복잡도**: Banking(단순), Flights(중간), Hospital(복잡)의 3개 도메인에서 5가지 의도를 테스트하였다. 특히 Hospital 도메인은 15개 이상의 도구와 깊은 조건부 분기를 가진다.
- **비교 대상**: $\text{ReAct baseline}$, $\text{WARPP without personalization}$ (단순 오케스트레이션만 적용), $\text{Full WARPP}$ (개인화 포함).
- **평가 모델**: GPT-4o, Llama 3, Claude Sonnet 3.5.
- **측정 지표**: Exact Match, LCS Tools(최장 공통 부분 수열 기반 도구 호출 일치도), Tool F1, Parameter Match(파라미터 정확도), Token Usage.

### 주요 결과

1. **작업 복잡도에 따른 성능 향상**: 단순한 작업에서는 전략 간 차이가 적었으나, 복잡도가 증가할수록 Full WARPP의 성능 향상폭이 뚜렷해졌다. 특히 가장 복잡한 `Process Payment` 작업에서 ReAct 대비 Exact Match와 Tool F1에서 압도적인 성능 향상을 보였다.
2. **모델별 영향**: 성능이 낮은 모델(Llama 3 등)일수록 구조적 오케스트레이션과 개인화의 혜택을 크게 입었으나, 가장 강력한 모델인 Sonnet조차 복잡한 작업에서는 WARPP를 통해 파라미터 일관성과 도구 협응 능력이 개선되었다.
3. **효율성**: 토큰 사용량이 ReAct 대비 현저히 감소하였다. 복잡한 워크플로우의 경우 WARPP가 ReAct 대비 토큰 사용량을 약 절반 수준으로 줄였다.
4. **워크플로우 품질**: LLM Judge(Gemini-2.0-flash)를 통해 평가한 결과, 개인화된 워크플로우의 Relevance(관련성)와 Completeness(완전성) 점수가 모두 4.5 이상(5점 만점)으로 높게 나타나, Personalizer 에이전트가 신뢰할 수 있는 경로를 생성함을 확인하였다.

## 🧠 Insights & Discussion

### 강점 및 분석

본 논문은 LLM의 성능을 높이기 위해 모델을 직접 튜닝하는 대신, **입력되는 지침(Instruction) 자체를 사용자에 맞게 최적화**하는 전략을 취했다. 이는 추가 학습 비용이 없는 Training-free 방식이므로 확장성이 매우 뛰어나다. 특히, 복잡한 조건부 분기를 미리 제거함으로써 LLM이 겪는 '결정 마비' 또는 '경로 이탈' 현상을 원천적으로 차단한 점이 주효했다.

### 한계 및 비판적 해석

1. **시뮬레이션 환경의 제약**: 대부분의 실험이 시뮬레이션된 도구 호출을 통해 이루어졌다. 실제 운영 환경에서 발생할 수 있는 네트워크 지연(latency)이나 API 실패 모드 등이 완전히 반영되지 않았을 가능성이 있다.
2. **의도 파악의 단순함**: Orchestrator Agent의 평가가 주로 의도 식별에 집중되어 있으며, 모호한 사용자 쿼리에 대한 중의성 해소(disambiguation) 능력에 대한 심층적인 분석은 부족하다.
3. **개인화 품질의 편차**: 저자들은 일부 개인화된 워크플로우에서 모범 사례(best practice) 단계가 누락되는 경우가 있음을 언급하였다. 이는 Personalizer의 프롬프트 엔지니어링을 더욱 정교화하거나, 더 강력한 모델을 Personalizer로 배치해야 함을 시사한다.

## 📌 TL;DR

**요약**: Agent WARPP는 TOD 시스템에서 LLM이 긴 조건부 워크플로우를 정확히 따르지 못하는 문제를 해결하기 위해, 런타임에 사용자 속성에 맞춰 워크플로우를 가지치기(pruning)하고 병렬로 처리하는 프레임워크이다.

**가치 및 향후 전망**: 추가 학습 없이도 복잡한 작업의 도구 호출 정확도를 높이고 토큰 비용을 절감할 수 있음을 입증하였다. 이 연구는 향후 실시간으로 변하는 사용자 컨텍스트에 따라 동적으로 실행 경로를 수정해야 하는 고도화된 AI 에이전트 시스템 설계에 중요한 기초가 될 것으로 보인다.
