# MONCE Tracking Metrics: a comprehensive quantitative performance evaluation methodology for object tracking

Kenneth Rapko, Wanlin Xie, and Andrew Walsh (2022)

## 🧩 Problem to Solve

본 논문은 다중 객체 추적(Multi-Object Tracking, MOT) 모델의 성능을 정량적으로 평가하는 체계적인 방법론의 부재를 해결하고자 한다. 특히 국방 분야와 같이 객체가 화면에서 사라졌다가 다시 나타나는 비연속적(non-contiguous) 특성과 장기간(long-term) 추적이 필요한 환경에서 기존의 벤치마크들은 몇 가지 한계를 가지고 있다.

가장 큰 문제는 기존의 추적 성능 지표들이 특정 데이터셋에 지나치게 의존적이거나, 단순히 최종 점수만을 제공할 뿐 성능 저하의 근본 원인(root cause)을 분석할 수 있는 진단 도구를 제공하지 않는다는 점이다. 따라서 본 연구의 목표는 장기간 비연속적 다중 객체 추적 모델의 성능을 객관적으로 측정함과 동시에, 개발자가 모델의 취약점을 파악하고 개선 방향을 설정할 수 있도록 돕는 진단적 통찰력을 제공하는 MONCE(Multi-Object Non-Contiguous Entities) 메트릭 세트를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 추적 성능 평가를 세 가지 계층(Top, Mid, Low level)으로 구조화하여, 단순한 성능 수치 제시를 넘어 모델의 내부 동작을 분석할 수 있는 진단 프레임워크를 설계했다는 점이다.

단일 수치로 표현되는 요약 지표(Top-level)부터 시퀀스 길이에 따른 성능 변화를 보여주는 그래프(Mid-level), 그리고 구체적인 실패 모드를 격리하여 분석하는 세부 지표(Low-level)까지 단계적으로 구성함으로써, 사용자는 "모델이 얼마나 잘 작동하는가"뿐만 아니라 "어느 지점에서, 왜 실패하는가"를 정밀하게 파악할 수 있다. 특히 Re-Identification(REID) 능력과 Absence Prediction(부재 예측) 능력을 분리하여 측정함으로써 장기 추적 모델의 핵심 성능 요소를 정량화하였다.

## 📎 Related Works

논문은 단일 객체 추적 및 다중 객체 추적 성능 평가를 위한 기존의 다양한 방법론과 VOT(Visual Object Tracking) 챌린지 등의 벤치마크를 언급한다. 기존 연구들은 전반적인 추적 성능을 측정하는 데는 유용하지만, 다음과 같은 한계가 명시되어 있다.

첫째, 데이터셋의 특성, 특히 시퀀스 길이의 차이에 따라 지표 값이 크게 왜곡될 수 있다. 둘째, 추적 실패가 발생했을 때 이것이 localization(위치 추정)의 문제인지, UID(Unique ID) 유지의 문제인지, 혹은 부재 객체를 잘못 예측한 문제인지 구분하여 분석할 수 있는 수단이 부족하다. MONCE는 이러한 한계를 극복하기 위해 분석의 수준을 세분화하고, 데이터셋의 길이 변화에 강건한 분석 방식을 도입하여 기존 접근 방식과 차별화를 꾀한다.

## 🛠️ Methodology

MONCE 메트릭 세트는 분석의 정밀도에 따라 세 가지 레벨로 구성된다. 분석에 앞서 $\text{entity-frame}$은 단일 프레임 내 객체의 인스턴스를, $\text{UID}$는 객체 고유 식별자를, $\text{sequence}$는 객체의 첫 등장부터 비디오 종료 시점까지의 모든 프레임을 의미한다.

### 1. Top-Level Metrics (요약 지표)

모델의 전반적인 성능을 단일 수치로 평가하며, 모델 간 비교 및 학습 추세 확인에 사용된다.

- **EAO (Expected Average Overlap):** VOT 챌린지에서 채택된 지표로, Ground Truth(GT) 엔티티 프레임에 대한 예상 중첩도를 측정한다.
- **$\text{EAO}_P$ (Expected Average Overlap Precision):** 저자들이 확장 제안한 지표로, Tracking Precision의 평균을 계산하여 False Positive(FP) 관점의 localization 성능을 분석한다.
- **Longevity Statistics:** 특정 비율의 유니크 엔티티가 성공적으로(FP/FN 없이) 추적된 시퀀스 길이를 측정한다.
- **Short/Long Term REID:** 객체가 프레임을 벗어났다가 재진입했을 때, 원래의 UID를 얼마나 정확하게 다시 할당하는지를 측정한다. 부재 기간을 사용자 정의 임계값으로 단기/장기로 나누어 각각의 성공률을 계산한다.

### 2. Mid-Level Metrics (경향성 분석)

시퀀스 길이에 따른 성능 변화를 플롯으로 나타내어 데이터셋 의존성을 줄이고 데이터 품질(밝기, 해상도 등)의 영향을 분석한다.

- **Tracking Recall Plot:** 시퀀스 길이에 따른 GT 엔티티 프레임의 평균 중첩도를 측정하며, Localization 성능과 False Negative(FN)를 평가한다.
- **Tracking Precision Plot:** 시퀀스 길이에 따른 예측 엔티티 프레임의 평균 중첩도를 측정하며, Localization 성능과 False Positive(FP)를 평가한다.

### 3. Low-Level Metrics (정밀 진단 지표)

구체적인 실패 모드를 격리하여 분석한다.

- **Tracking Longevity:** Localization 성능을 배제하고, 오직 TP/TN의 일치 여부만을 평가한다. $\text{IoU} = 0$ 임계값을 사용하여, 주어진 길이 $T$까지 FP나 FN이 전혀 없는 트랙의 비율을 측정한다.
- **Tracking Localization:** Longevity 분석에서 성공한 트랙들만을 대상으로, 예측 박스가 GT 박스와 얼마나 잘 일치하는지를 $\text{IoU}$ 임계값(0~1)에 따라 측정한다.
- **Absence Prediction Rate:** 객체가 부재한 상황에서 모델이 정확하게 예측하지 않음(True Negative)을 출력하는 능력을 평가한다. 부재 길이 $T_a$에 대해 모든 부재 프레임에서 예측값이 없는 비율을 계산한다.

## 📊 Results

본 논문은 실제 데이터와 모델을 사용하여 MONCE가 어떻게 구체적인 진단 도구로 활용되는지를 네 가지 사례를 통해 보여준다.

1. **개발 중인 모델 분석:** 완벽한 Detection 결과가 주어졌음에도 불구하고, Recall 곡선의 초기 부분에 'tail' 현상이 발견되어 모델이 UID를 할당하기 전 몇 프레임의 관찰 시간이 필요함을 발견했다. 또한, Longevity 플롯을 통해 30프레임 이후부터 UID 유지 능력이 급격히 떨어진다는 점을 확인하여 학습 데이터셋의 시퀀스 길이 부족 가능성을 시사했다.
2. **결함 있는 Detection 모델 식별:** $\text{EAO}_P$와 Tracking Precision이 매우 낮게 측정되는 것을 통해 Detection 모델이 과도하게 많은 FP를 생성하고 있음을 즉각적으로 진단하였다.
3. **미세한 추적 오류 포착:** EAO 점수가 $0.990$으로 매우 높았음에도 불구하고, Longevity 플롯에서 특정 시점(20프레임) 이후 성공률이 소폭 하락하는 것을 발견했다. 이를 통해 두 객체의 UID가 일시적으로 서로 바뀌었다가 돌아오는 미세한 오류를 정밀하게 짚어냈다.
4. **잘못된 라벨링 데이터 발견:** 모델 성능이 비정상적으로 낮게 측정되었으나, Longevity 플롯에서 총 트랙 수가 900프레임마다 계단식으로 감소하는 패턴을 발견했다. 이를 통해 데이터 라벨링 과정에서 900프레임 단위로 UID가 리셋된 오류가 있었음을 밝혀냈다.

## 🧠 Insights & Discussion

MONCE 프레임워크의 가장 큰 강점은 추적 성능을 단순한 '점수'가 아닌 '진단 프로세스'로 전환했다는 점이다. 기존 지표들은 Localization 오류와 ID 스위칭 오류, 그리고 부재 예측 오류가 모두 뒤섞여 하나의 수치로 나타나기 때문에 개발자가 무엇을 수정해야 할지 알기 어려웠다. 하지만 MONCE는 이를 계층적으로 분리함으로써, 예를 들어 Localization은 훌륭하지만 ID 유지가 안 되는 경우인지, 혹은 부재 상황에서 가짜 객체를 계속 생성하는 경우인지를 명확히 구분해 준다.

다만, 본 논문에서는 matching technique으로 IoU 기반의 커스텀 기법을 사용했다고 언급하고 있으나, 구체적인 매칭 알고리즘의 수식이나 절차는 상세히 명시되지 않았다. 또한, REID 분석 시 '단기'와 '장기'를 나누는 임계값 설정 기준이 사용자 정의에 맡겨져 있어, 객관적인 표준 임계값에 대한 논의가 추가될 필요가 있다. 그럼에도 불구하고, 데이터 라벨링 오류까지 잡아낼 수 있을 정도로 지표의 민감도가 높다는 점은 실무적으로 매우 가치 있는 결과이다.

## 📌 TL;DR

본 연구는 장기간 비연속적 다중 객체 추적(MOT) 모델을 평가하기 위한 **MONCE 메트릭 세트**를 제안한다. 이 방법론은 요약 지표(Top), 경향성 분석(Mid), 세부 진단(Low)의 3단계 구조를 통해 **Localization, UID 유지 능력, 부재 예측 능력**을 독립적으로 평가할 수 있게 한다. 이를 통해 단순한 성능 측정을 넘어, 모델의 구체적인 실패 원인을 분석하고 데이터셋의 오류까지 찾아낼 수 있는 강력한 진단 도구를 제공하며, 향후 고성능 MOT 모델 개발 및 검증 과정에서 핵심적인 역할을 할 것으로 기대된다.
