# Object-Adaptive LSTM Network for Real-time Visual Tracking with Adversarial Data Augmentation

Yihan Du, Yan Yan, Si Chen, and Yang Hua (2020)

## 🧩 Problem to Solve

본 논문은 실시간 비주얼 트래킹(Visual Tracking)에서 발생하는 **정확도와 속도 사이의 트레이드오프(Trade-off)** 문제를 해결하고자 한다. 기존의 딥러닝 기반 트래킹 방법은 크게 두 가지 방향으로 나뉜다.

첫째, **분류 기반(Classification-based) 방법**은 객체와 배경을 구분하는 온라인 분류기를 학습시키므로 정확도가 높지만, 수많은 후보 영역(Proposal)에 대해 특성 추출(Feature Extraction)을 수행해야 하므로 계산 비용이 매우 커 실시간 구현이 어렵다.

둘째, **매칭 기반(Matching-based) 방법**(예: Siamese networks)은 사전 학습된 모델을 사용하여 템플릿과 후보 영역을 직접 비교하므로 속도가 매우 빠르다. 그러나 온라인 업데이트 과정이 없거나 부족하여, 객체의 외형이 크게 변하거나 배경에 유사한 객체가 나타날 경우 추적 경로가 이탈하는 드리프트(Drift) 현상이 빈번하게 발생한다.

따라서 본 논문의 목표는 **매칭 기반 방법의 빠른 속도와 분류 기반 방법의 높은 적응성을 동시에 갖춘 실시간 비주얼 트래커를 개발**하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **사전 학습된 Siamese 네트워크를 통한 효율적인 후보 선택**과 **온라인으로 학습되는 LSTM 네트워크를 통한 정교한 분류**를 결합하는 것이다.

1. **Object-Adaptive LSTM Network**: 비디오의 순차적 의존성(Sequential Dependencies)을 캡처하고 객체의 외형 변화를 적응적으로 학습하기 위해 LSTM을 도입하였다. LSTM의 내부 상태를 통해 객체의 표현 모델을 유지하며 실시간으로 업데이트한다.
2. **Fast Proposal Selection Strategy**: 모든 후보 영역을 조사하는 대신, SiamFC를 사용하여 대략적인 위치를 먼저 예측하고, 신뢰도가 높은 소수의 후보 영역만을 선택해 LSTM에 입력함으로써 계산량을 획기적으로 줄였다. 특히, 검색 영역의 전체 특성 맵(Feature Map)에서 필요한 부분만 크롭(Crop)하여 특성 추출 중복 계산을 방지하였다.
3. **Adversarial Data Augmentation (GAN)**: 온라인 학습 시 발생하는 샘플 부족 및 클래스 불균형(양성 샘플 부족) 문제를 해결하기 위해 DCGAN을 도입하였다. 이를 통해 현실적인 양성 샘플을 생성하여 LSTM 네트워크의 학습 성능을 높였다.

## 📎 Related Works

논문에서는 다음과 같은 관련 연구들의 한계를 지적하며 차별점을 제시한다.

- **Siamese Networks (SiamFC, CFNet 등)**: 매우 빠르지만 온라인 적응력이 부족하여 외형 변화에 취약하다. 본 논문은 Siamese 네트워크를 최종 추적기가 아닌, LSTM을 위한 '거친(Coarse) 전처리 단계'로 활용하여 이 문제를 해결한다.
- **RNN/LSTM 기반 트래킹**: 기존 일부 연구(RFL, MemTrack 등)가 LSTM을 템플릿 관리용으로 사용했으나, 본 논문은 LSTM을 직접적인 **객체 전용 분류기(Object-specific Classifier)**로 사용하여 배경 정보와 전경 정보를 모두 활용하고 적응력을 높였다.
- **GAN 기반 트래킹 (VITAL 등)**: GAN을 사용하여 견고한 특성을 추출하려 했으나, 방대한 특성 추출과 복잡한 튜닝으로 인해 속도가 매우 느리다. 반면, 본 논문은 GAN을 **이미지 공간에서의 데이터 증강(Data Augmentation)** 도구로만 활용하여 실시간 속도를 유지하였다.

## 🛠️ Methodology

### 1. 전체 파이프라인

시스템은 크게 두 단계로 구성된다.

- **Stage 1 (Fast Proposal Selection)**: 사전 학습된 Siamese 네트워크를 통해 템플릿과 검색 영역을 매칭하여 응답 맵(Response Map)을 생성하고, 점수가 높은 $N$개의 후보 영역을 선택한다.
- **Stage 2 (Object Classification)**: 선택된 후보들의 특성을 온라인으로 학습된 LSTM 네트워크에 입력하여, 최종적으로 가장 높은 양성 점수를 가진 후보를 추적 결과로 선택한다.

### 2. Fast Proposal Selection

SiamFC를 사용하여 다음과 같은 유사도 함수를 계산한다.
$$F(z,x) = \phi(z) * \phi(x) + kI$$
여기서 $z$는 템플릿, $x$는 검색 영역, $\phi$는 합성곱 임베딩 함수, $*$는 교차 상관(Cross-correlation) 연산이다. 이 결과로 나온 스코어 맵에서 신뢰도가 높은 지점들을 선택하며, 이때 특성 추출을 다시 하는 것이 아니라 이미 계산된 검색 영역의 특성 맵에서 해당 위치의 특성을 그대로 가져와 LSTM에 전달한다.

### 3. Object-Adaptive LSTM Network

LSTM은 이전 프레임의 객체 상태를 기억하여 현재 프레임의 후보가 객체인지 배경인지 분류한다.

**전방 전파(Forward Pass) 과정:**

1. **입력층**: 후보 특성 벡터 $x_t$를 선형 변환하여 $z_t$를 생성한다.
    $$z_t = W_{Input}^T x_t + b_{Input}$$
2. **LSTM 블록**: 이전 프레임의 셀 상태 $\hat{c}_{t-1}$와 은닉 상태 $\hat{h}_{t-1}$를 입력받아 현재 상태를 계산한다.
    - **입력 게이트($i_t$), 망각 게이트($f_t$), 출력 게이트($o_t$)**가 정보의 흐름을 제어한다.
    - **셀 상태($c_t$)**: $\hat{c}_{t-1}$에서 불필요한 정보를 버리고 새로운 정보를 더해 업데이트한다.
    - **은닉 상태($h_t$)**: 업데이트된 셀 상태를 바탕으로 현재의 출력값을 결정한다.
3. **출력층**: Softmax를 통해 해당 후보가 객체일 확률($p^+$)과 배경일 확률($p^-$)을 출력한다.
    $$r_t = \text{Softmax}(W_{Output}^T h_t + b_{Output})$$

**학습 절차(Backward Pass):**

- 매 프레임마다 현재 프레임에서 수집된 샘플 $S_t$와 이전 상태 $\hat{State}_{t-1}$를 사용하여 온라인 학습을 수행한다.
- 손실 함수로는 **교차 엔트로피(Cross-entropy) 손실**을 사용하며, 모든 시점의 그래디언트를 전파하는 대신 현재 프레임의 결과에 집중하여 수렴 속도를 높였다.

### 4. Adversarial Data Augmentation (GAN)

온라인 학습 시 양성 샘플(객체)이 매우 적은 문제를 해결하기 위해 DCGAN을 사용한다.

- **Generator ($G$)**: 랜덤 노이즈 $z$를 입력받아 실제 객체 이미지와 유사한 가짜 이미지 $G(z)$를 생성한다.
- **Discriminator ($D$)**: 입력 이미지가 실제 이미지인지 $G$가 만든 가짜 이미지인지 판별한다.
- **목표 함수**: 두 네트워크는 다음과 같은 Minimax 게임을 수행한다.
    $$\min_{G} \max_{D} F(D,G) = \mathbb{E}_{x \sim P_{img}(x)}[\log D(x)] + \mathbb{E}_{z \sim P_{noise}(z)}[\log(1 - D(G(z)))]$$
이렇게 생성된 다양한 양성 샘플들을 LSTM의 학습 데이터셋에 추가하여 클래스 불균형을 해소하고 일반화 성능을 높인다.

## 📊 Results

### 실험 설정

- **데이터셋**: OTB-2013, OTB-2015, TC-128, UAV-123, VOT-2017.
- **비교 대상**: SiamFC, CFNet, MemTrack, TRACA, ACFN 등 최신 트래커.
- **지표**: Precision(정밀도), Success rate(AUC), EAO(VOT-2017).
- **속도**: NVIDIA GeForce GTX Titan X GPU 기준, 평균 **32.5 fps**로 실시간 동작 확인.

### 주요 결과

1. **정량적 결과 (OTB-2015)**:
    - Precision $87.2$, AUC $62.8$를 기록하여 비교 대상 중 최상위권의 성능을 보였다.
    - 특히 Siamese 기반 모델(SiamFC, CFNet 등)보다 높은 정확도를 기록하여 LSTM의 적응력이 유효함을 증명하였다.
2. **VOT-2017 실시간 챌린지**:
    - **EAO 0.216**으로 1위를 차지하였으며, 이는 베이스라인인 SiamFC 대비 비약적인 향상이다.
3. **속성별 분석**:
    - **Fast Motion, Occlusion, Out of View** 상황에서 타 모델 대비 매우 강력한 성능을 보였다. 이는 LSTM이 이전의 객체 외형을 기억하여 일시적인 가려짐이나 빠른 움직임 속에서도 추적을 유지할 수 있기 때문이다.
    - 다만, **In-plane Rotation**과 **Low Resolution**에서는 MemTrack보다 다소 낮은 성능을 보였는데, 이는 템플릿 업데이트 메커니즘의 차이로 분석된다.

## 🧠 Insights & Discussion

### 강점

본 연구는 **"효율적 후보 선택 $\rightarrow$ 순차적 의존성 학습 $\rightarrow$ 데이터 증강"**으로 이어지는 파이프라인을 통해 실시간성과 적응성이라는 두 마리 토끼를 잡았다. 특히 LSTM을 단순한 메모리가 아닌 온라인 분류기로 활용하고, GAN을 통해 데이터 부족 문제를 해결한 점이 독창적이다.

### 한계 및 해석

- **템플릿 업데이트의 부재**: 논문에서 언급되었듯, 회전이나 저해상도 환경에서의 성능 저하는 템플릿 자체가 동적으로 업데이트되지 않는 구조적 한계에서 기인한 것으로 보인다.
- **GAN 학습 비용**: GAN을 온라인으로 업데이트하는 과정이 추가되므로, GAN이 없는 버전(OA-LSTM+)보다는 속도가 약간 느려진다. 하지만 이를 통해 얻는 정확도 이득이 훨씬 크다.

### 비판적 해석

LSTM의 내부 상태가 객체 표현 모델로서 역할을 수행하지만, 이 상태가 얼마나 오랫동안 유효한지, 혹은 급격한 외형 변화 시 $\hat{State}_{t-1}$가 오히려 노이즈로 작용할 가능성에 대한 심층적인 분석은 부족해 보인다.

## 📌 TL;DR

본 논문은 **SiamFC의 빠른 후보 선택 능력**과 **LSTM의 시계열 적응 능력**, 그리고 **GAN의 데이터 증강 능력**을 결합한 실시간 비주얼 트래커 **OA-LSTM-ADA**를 제안한다. 이 방법은 실시간 속도(32.5 fps)를 유지하면서도 객체의 외형 변화와 복잡한 배경 환경에 매우 강건하게 대응하며, 특히 VOT-2017 실시간 챌린지에서 1위를 기록하며 그 우수성을 입증하였다. 향후 어텐션 메커니즘을 도입한다면 더욱 정교한 후보 선택이 가능할 것으로 기대된다.
