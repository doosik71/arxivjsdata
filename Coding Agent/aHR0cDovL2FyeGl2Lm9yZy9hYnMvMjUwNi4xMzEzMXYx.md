# AlphaEvolve: A coding agent for scientific and algorithmic discovery

Alexander Novikov et al. (2025)

## 🧩 Problem to Solve

본 논문은 새로운 과학적 발견이나 상업적 가치가 높은 알고리즘을 개발하는 과정에서 요구되는 아이디어 구상, 탐색, 가설 검증 및 실험의 반복적인 과정을 자동화하고자 한다. 최근 대규모 언어 모델(LLM)이 벤치마크 성능을 높이고 가설 생성 단계까지는 진전되었으나, 완전히 새로운 과학적 또는 실용적 발견을 이끌어내는 단계까지는 도달하지 못했다는 한계가 존재한다.

연구의 주된 목표는 LLM 기반의 코딩 에이전트를 통해, 자동 평가가 가능한 과학 및 공학 문제 영역에서 기존의 최첨단(SOTA) 솔루션을 뛰어넘는 새로운 알고리즘과 최적화된 코드를 자율적으로 발견하는 시스템을 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 **진화적 계산(Evolutionary Computation)**과 **LLM 기반의 코드 생성**을 결합한 **AlphaEvolve**라는 코딩 에이전트를 제안한 것이다. 

중심적인 설계 아이디어는 솔루션 자체를 직접 찾는 대신, 솔루션을 찾아내거나 구성하는 '알고리즘'을 진화시키는 **초최적화(Superoptimization)** 전략을 사용하는 것이다. AlphaEvolve는 LLM 앙상블을 사용하여 코드를 생성, 비판 및 진화시키며, 프로그램 실행 결과라는 객관적인 피드백을 통해 LLM의 환각(Hallucination) 문제를 회피하고 실제적으로 작동하는 고성능 알고리즘을 탐색한다.

## 📎 Related Works

AlphaEvolve는 Google DeepMind의 이전 연구인 **FunSearch**를 기반으로 하며, 이를 대폭 확장하였다. FunSearch는 LLM 기반의 진화적 탐색을 통해 수학적 발견을 이룬 사례가 있으나, 다음과 같은 한계가 있었다.

- **규모 및 범위:** FunSearch는 단일 Python 함수(10~20라인)만을 진화시켰으나, AlphaEvolve는 수백 라인에 달하는 전체 코드 파일과 다양한 프로그래밍 언어를 처리할 수 있다.
- **모델 성능:** FunSearch는 상대적으로 작은 LLM을 사용했으나, AlphaEvolve는 최신 SOTA LLM(Gemini 2.0 등)의 성능을 활용하여 더 복잡한 논리를 생성한다.
- **최적화 목표:** 단일 지표만을 최적화하던 방식에서 벗어나 다중 지표(Multiple metrics)를 동시에 최적화할 수 있도록 개선되었다.
- **평가 효율:** 매우 빠른 평가 속도가 필요했던 이전 방식과 달리, 병렬 가속기를 활용하여 수 시간의 평가 시간을 소요하는 복잡한 문제까지 다룰 수 있다.

## 🛠️ Methodology

### 1. 전체 파이프라인 및 시스템 구조
AlphaEvolve는 사용자가 정의한 평가 기준과 초기 코드를 바탕으로 다음과 같은 자율 루프를 수행한다.

1.  **Task Specification:** 사용자는 솔루션을 스칼라 값으로 매핑하는 평가 함수 $h$와 초기 프로그램, 그리고 진화 대상이 될 코드 블록(`# EVOLVE-BLOCK-START` 및 `END`)을 정의한다.
2.  **Prompt Sampling:** 프로그램 데이터베이스에서 과거의 시도와 성공적인 사례들을 샘플링하여 LLM에게 풍부한 컨텍스트(과거 시도, 아이디어, 평가 결과 등)를 제공한다.
3.  **Creative Generation:** LLM 앙상블(Gemini 2.0 Flash 및 Pro)이 기존 코드를 개선하기 위한 수정 사항(Diff)을 제안한다.
4.  **Evaluation:** 제안된 코드를 실행하여 평가 함수 $h$를 통해 점수를 매긴다. 이때 효율성을 위해 점진적으로 난이도를 높이는 **Evaluation Cascade(가설 검정)** 방식을 사용한다.
5.  **Evolution:** 평가 결과가 우수한 프로그램은 데이터베이스에 저장되어 다음 세대의 모태가 된다.

### 2. 핵심 구성 요소 및 절차
- **LLM 앙상블 전략:** 처리량이 높은 Gemini 2.0 Flash는 많은 수의 후보군을 빠르게 생성하고, 성능이 뛰어난 Gemini 2.0 Pro는 가끔씩 고품질의 획기적인 제안을 생성함으로써 탐색 효율과 질을 동시에 잡는다.
- **코드 수정 방식:** LLM은 전체 코드를 다시 쓰는 대신, 특정 부분을 정밀하게 수정하는 `SEARCH/REPLACE` 블록 형식을 사용하여 변경 사항을 제안한다.
- **추상화 수준의 유연성:** 문제 특성에 따라 원시 문자열(Raw string), 생성자 함수(Constructor function), 또는 맞춤형 탐색 알고리즘(Custom search algorithm) 중 최적의 추상화 단계를 선택하여 진화시킨다.

### 3. 주요 방정식 및 학습 목표
시스템의 목표는 평가 함수 $h$의 결과값(스칼라 지표)을 최대화하는 프로그램을 찾는 것이다.
$$ \text{Maximize } h(\text{program}) $$
단순한 단일 지표 최적화뿐만 아니라, 여러 지표를 동시에 최적화함으로써 탐색 공간의 다양성을 확보하고 타겟 지표의 성능을 간접적으로 끌어올리는 전략을 취한다.

## 📊 Results

### 1. 행렬 곱셈 알고리즘 발견 (Matrix Multiplication)
행렬 곱셈의 계산 복잡도를 결정하는 텐서 분해(Tensor Decomposition) 문제에 적용하였다.
- **성과:** 총 54개의 타겟 중 14개에서 SOTA를 경신하였다.
- **핵심 발견:** 56년 동안 깨지지 않았던 Strassen 알고리즘의 기록을 넘어, $4 \times 4$ 복소수 행렬 곱셈을 단 48번의 스칼라 곱셈으로 수행하는 알고리즘을 발견하였다(기존 SOTA 49회).

### 2. 수학적 난제 해결 (Mathematical Discovery)
50개 이상의 수학적 문제(해석학, 조합론, 기하학 등)에 적용하였다.
- **성과:** 약 75%의 문제에서 기존 SOTA와 일치하는 결과를 냈으며, 약 20%의 문제에서는 SOTA를 능가하는 새로운 구성을 발견하였다.
- **주요 사례:** 11차원 Kissing Number 문제에서 기존 592개를 넘어 593개의 구를 배치하는 구성을 발견하였으며, Erdős의 최소 중첩 문제(Minimum Overlap Problem)의 상한선을 개선하였다.

### 3. Google 인프라 최적화 (Engineering Application)
- **데이터 센터 스케줄링:** Borg 스케줄러의 휴리스틱 함수를 최적화하여, 방치되는 컴퓨팅 자원(Stranded resources)을 평균 0.7% 회복하였다.
- **Gemini 커널 엔지니어링:** Pallas 커널의 타일링 전략을 최적화하여 커널 실행 속도를 평균 23% 향상시켰으며, 이는 Gemini 전체 학습 시간의 1% 감소로 이어졌다.
- **하드웨어 회로 설계:** TPU 산술 회로의 Verilog 구현에서 불필요한 비트를 제거하는 최적화를 찾아내어 면적과 전력 소비를 줄였다.
- **컴파일러 코드 최적화:** XLA 생성 중간 표현(IR)을 직접 최적화하여 FlashAttention 커널 속도를 32% 향상시켰다.

## 🧠 Insights & Discussion

### 강점 및 가치
AlphaEvolve는 LLM의 광범위한 사전 지식과 프로그램 실행을 통한 엄격한 검증 메커니즘을 결합함으로써, 단순한 코드 작성을 넘어 '과학적 발견'의 영역으로 진입하였다. 특히, 인간 전문가가 수개월에 걸쳐 수행하던 커널 최적화 작업을 단 며칠 만에 자동화하여 실질적인 비용 절감과 성능 향상을 이끌어냈다는 점이 매우 고무적이다. 또한, 이를 **Test-time Compute**의 확장으로 해석할 수 있으며, 추론 단계에서의 연산량을 늘림으로써 기본 모델의 성능을 초과하는 결과물을 낼 수 있음을 보여주었다.

### 한계 및 비판적 해석
본 시스템의 가장 큰 제약은 **자동 평가 함수(Automated Evaluator)의 존재 여부**이다. 수학이나 컴퓨터 과학처럼 정답을 기계적으로 검증할 수 있는 영역에서는 탁월하지만, 실험적 검증이 필수적인 자연 과학 분야로 확장하기 위해서는 LLM 기반의 고수준 아이디어 평가와 실제 구현 단계를 유연하게 연결하는 추가적인 메커니즘이 필요하다.

또한, 논문에서 제시된 결과들이 매우 인상적이지만, 일부 수학적 발견의 경우 개선 폭이 매우 미미한 경우가 있다. 이는 LLM이 기존 문헌의 패턴을 매우 정교하게 모방하여 미세 조정(Fine-tuning)하는 능력은 뛰어나지만, 완전히 새로운 패러다임의 수학적 원리를 창조하는 것에는 여전히 한계가 있을 수 있음을 시사한다.

## 📌 TL;DR

AlphaEvolve는 LLM 앙상블과 진화적 알고리즘을 결합하여, 프로그램 실행 피드백을 통해 스스로 코드를 개선하는 자율적 코딩 에이전트이다. 이 시스템은 56년 된 행렬 곱셈 알고리즘 기록 경신, 11차원 Kissing Number 문제 해결, 그리고 Google의 실제 데이터 센터 및 TPU 하드웨어/소프트웨어 스택 최적화라는 놀라운 성과를 거두었다. 본 연구는 AI가 단순한 도구를 넘어, 수학적 발견과 시스템 엔지니어링 분야에서 인간 전문가의 능력을 보완하거나 능가하는 'AI 과학자'로 발전할 수 있는 가능성을 강력하게 제시한다.
