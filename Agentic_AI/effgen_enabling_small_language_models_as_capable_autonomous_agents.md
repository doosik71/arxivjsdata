# EFFGEN: Enabling Small Language Models as Capable Autonomous Agents

Gaurav Srivastava, Aafiya Hussain, Chi Wang, Yingyan (Celine) Lin, Xuan Wang (2026)

## 🧩 Problem to Solve

현재 대부분의 LLM 에이전트 시스템(예: LangChain, AutoGen)은 GPT, Claude, Gemini와 같은 대규모 언어 모델(LLM)의 성능에 최적화되어 있으며, 주로 API 호출을 통해 작동한다. 이러한 접근 방식은 높은 토큰 비용과 민감한 데이터 처리에 따른 개인정보 보호 문제라는 실질적인 한계를 가진다.

반면, 소규모 언어 모델(Small Language Models, SLM)은 로컬 배포가 가능하여 효율성과 보안성 면에서 유리하지만, LLM에 비해 컨텍스트 윈도우(Context Window)가 제한적이고, 지시어 이행(Instruction-following) 능력이 낮으며, 추론 깊이가 얕다는 제약이 있다. 기존 프레임워크들은 이러한 SLM의 제약 사항을 체계적으로 다루지 않으며, 단순히 LLM용 컴포넌트를 SLM에 맞게 수동으로 조정하는 방식에 의존하고 있다.

따라서 본 논문의 목표는 SLM의 제약 조건을 설계의 최우선 요구 사항으로 설정하여, SLM이 LLM 기반 에이전트에 필적하는 성능을 낼 수 있도록 최적화된 오픈소스 에이전트 프레임워크인 EFFGEN을 개발하는 것이다.

## ✨ Key Contributions

EFFGEN의 핵심 아이디어는 SLM의 하드웨어 및 소프트웨어적 제약을 극복하기 위해 '모델 크기 인식(Model-size-aware)' 최적화 계층을 도입하는 것이다. 주요 기여 사항은 다음과 같다.

1. **SLM 특화 프롬프트 최적화**: 모델 크기에 따라 압축률을 다르게 적용하여 태스크의 의미론적 구조를 유지하면서도 컨텍스트를 70~80%까지 압축한다.
2. **사전 실행 복잡도 분석 및 라우팅**: 실행 전 5가지 요소를 통해 쿼리의 복잡도를 수치화하고, 이에 따라 단일 에이전트 실행 또는 다중 에이전트 분해(Decomposition) 경로를 결정하여 불필요한 계산 낭비를 줄인다.
3. **다중 프로토콜 통합**: MCP(Model Context Protocol), A2A(Agent-to-Agent), ACP(Agent Communication Protocol)를 통합 구현하여 이기종 에이전트 생태계 간의 상호운용성을 확보했다.
4. **3단계 통합 메모리 시스템**: 단기 메모리, 장기 에피소드 저장소, 벡터 기반 시맨틱 검색을 결합하여 SLM의 제한된 컨텍스트 윈도우 문제를 해결했다.

## 📎 Related Works

기존 연구들은 주로 Toolformer, ReAct, Tree-of-Thoughts와 같이 모델의 도구 사용 능력이나 추론 경로를 확장하는 데 집중했다. 또한 AutoGen이나 MetaGPT와 같은 다중 에이전트 프레임워크는 에이전트 간의 협업 패턴을 설계하는 데 주력했으나, 대부분 LLM의 강력한 추론 능력을 전제로 한다.

최근 Smolagents와 같이 소형 모델을 타겟팅한 프레임워크가 등장했으나, 이는 모든 태스크를 코드 실행으로 변환하는 방식에 의존하여 비코딩 작업에서 불필요한 오버헤드를 발생시키며, 체계적인 프롬프트 최적화나 사전 라우팅 메커니즘이 부족하다는 한계가 있다. EFFGEN은 이러한 기존 접근 방식과 달리 SLM의 제약 사항을 설계 단계부터 반영하여, 모델 크기에 따른 적응형 최적화를 제공한다는 점에서 차별화된다.

## 🛠️ Methodology

EFFGEN 에이전트는 $\text{A} = (\text{M}, \text{T}, \text{R}, \text{D}, \text{M}, \phi)$라는 튜플로 정의되며, 여기서 $\text{M}$은 모델, $\text{T}$는 도구 레지스트리, $\text{R}$은 라우팅 함수, $\text{D}$는 분해 연산자, $\text{M}$은 메모리 시스템, $\phi$는 프롬프트 최적화 함수이다.

### 1. SLM 인식 프롬프트 최적화 ($\phi$)

모델 크기를 TINY(<1B), SMALL(1B-3B), MEDIUM(3B-7B), LARGE(≥7B)의 네 단계로 구분하고, 각 단계에 맞는 최적화 파라미터를 적용한다. 최적화 과정은 다음의 5단계 순차 변환으로 이루어진다.

- **압축(Compression)**: 30가지의 패턴 교체 쌍을 사용하여 장황한 문구를 간결한 표현으로 변경한다 (예: "in order to" $\rightarrow$ "to").
- **단순화(Simplification)**: 20단어가 넘는 긴 문장을 접속사 기준으로 분리하여 SLM의 파싱 능력을 높인다.
- **중복 제거(Redundancy Removal)**: 중복 문장 및 "please", "kindly"와 같은 정중한 표현을 제거한다.
- **불렛 포맷팅(Bullet Formatting)**: 지시 사항을 불렛 포인트 형식으로 변환하여 지시어 이행률을 높인다.
- **컨텍스트 절단(Context Truncation)**: 최적화 후에도 최대 토큰 수를 초과할 경우, 문장 경계에서 안전하게 절단한다.

### 2. 복잡도 분석 및 라우팅 ($\text{R}$)

실행 전 쿼리 $q$에 대해 다음과 같은 복잡도 점수 $C(q)$를 계산한다.
$$C(q) = \sum_{i=1}^{5} w_i \cdot f_i(q)$$
여기서 $f_i$는 (1) 태스크 길이, (2) 요구사항 개수, (3) 도메인 폭, (4) 도구 필요성, (5) 추론 깊이를 측정하는 함수이며, $w$는 경험적으로 튜닝된 가중치 벡터이다.

- $C(q) < \tau$ (기본값 7.0): 단일 에이전트(Single Agent)가 $\text{ReAct}$ 루프를 통해 실행한다.
- $C(q) \ge \tau$: 태스크 분해 $\text{D}$를 통해 하위 작업으로 나누며, 의존성 여부에 따라 병렬(Parallel), 순차(Sequential), 또는 계층적(Hierarchical, $C(q) \ge 8.5$) 전략을 선택한다.

### 3. 지능적 태스크 분해 ($\text{D}$)

복잡한 쿼리를 하위 작업 집합 $S = \{s_1, ..., s_n\}$과 의존성 그래프 $G=(V, E)$로 변환한다.

- **병렬 분해**: 의존성이 없는 작업을 동시에 실행하고 최종 결과를 합성한다.
- **순차 분해**: 선행 작업의 결과가 다음 작업의 컨텍스트로 전달되는 체인 구조를 형성한다.
- **계층적 분해**: 매니저-워커 구조를 통해 전문 에이전트에게 작업을 할당하고 매니저가 결과를 통합한다.
- **하이브리드 분해**: 위 패턴들을 DAG(Directed Acyclic Graph) 구조에 따라 혼합하여 실행한다.

### 4. 3단계 메모리 시스템 ($\text{M}$)

SLM의 좁은 컨텍스트 윈도우를 극복하기 위해 세 가지 저장소를 운영한다.

- **단기 메모리 ($\text{M}_s$)**: 최근 대화 기록을 유지하며, 전체 토큰이 컨텍스트 한도의 85%에 도달하면 오래된 메시지를 자동으로 요약한다.
- **장기 메모리 ($\text{M}_l$)**: 중요도 점수($I$)를 기반으로 중요한 사건을 저장하며, 최근성, 빈도, 중요도를 결합하여 검색한다.
- **벡터 메모리 ($\text{M}_v$)**: FAISS 또는 ChromaDB를 사용하여 임베딩 기반의 시맨틱 검색을 수행한다.

## 📊 Results

### 실험 설정

- **데이터셋**: GSM8K, GAIA, SimpleQA 등 도구 사용, 추론, 메모리를 평가하는 13개 벤치마크를 사용했다.
- **모델**: Qwen2.5(1.5B~32B), Gemma 3, GPT-OSS 20B 모델을 평가했다.
- **비교 대상**: Raw Model, LangChain, AutoGen, Smolagents.

### 주요 결과

- **성능 향상**: Qwen2.5-7B 기준, EFFGEN은 13개 벤치마크에서 평균 63.07%의 정확도를 기록하여 Raw 모델 대비 12.3% 향상된 성능을 보였다. 특히 1.5B 모델에서 가장 큰 폭의 개선(13.1%)이 나타났다.
- **효율성**: 소형 모델에서 실행 속도가 최대 18배 향 uma(Smolagents 대비) 향상되었으며, 프롬프트 압축을 통해 토큰 사용량을 1.8배 감소시켰다.
- **규모별 영향**: 프롬프트 최적화는 소형 모델(1.5B)에서 더 큰 이득(11.2% $\uparrow$)을 주었으며, 복잡도 라우팅은 대형 모델(32B)에서 더 큰 이득(7.9% $\uparrow$)을 주어 두 기법이 상호 보완적임을 입증했다.
- **도구 실행**: BeyondBench-Hard와 같은 복잡한 코드 실행 태스크에서 EFFGEN은 Qwen2.5-7B의 성능을 15.42%에서 28.57%로 크게 끌어올렸다.

## 🧠 Insights & Discussion

### 강점 및 분석

EFFGEN은 프레임워크 설계가 모델의 크기가 작을수록 더 결정적인 영향을 미친다는 점을 증명했다. 대형 모델은 자체적인 능력으로 프레임워크의 비효율성을 일부 상쇄할 수 있지만, SLM은 최적화된 프롬프트와 정교한 컨텍스트 관리가 없으면 제 성능을 내지 못한다. 특히 불렛 포맷팅과 같은 구조적 가이드가 SLM의 지시어 이행 능력을 크게 개선함을 확인했다.

### 한계 및 비판적 해석

실험 결과에서 흥미로운 점은 SLM의 '과도한 저신뢰(Underconfidence)' 현상이다. 소형 모델은 검색된 정보가 자신의 내부 지식과 약간만 충돌해도 정답을 알고 있었음에도 불구하고 검색 결과를 맹신하여 오답으로 수정하는 경향이 강했다. 이는 SLM을 위한 RAG(Retrieval-Augmented Generation) 설계 시 단순한 정보 제공을 넘어, 모델이 정보를 비판적으로 수용하도록 하는 신뢰도 조절 프롬프트가 필수적임을 시사한다.

또한, 복잡도 분석기 $C(q)$가 95%의 정확도를 보이지만, 나머지 5%의 오분류가 발생할 경우 불필요한 오버헤드가 발생하거나 최적화 기회를 놓치는 문제가 발생한다. 이는 향후 휴리스틱 기반이 아닌 학습 기반의 라우팅 모델로 발전시켜야 할 필요성을 보여준다.

## 📌 TL;DR

EFFGEN은 SLM의 제한된 컨텍스트와 낮은 추론 능력을 극복하기 위해 설계된 **SLM 특화 에이전트 프레임워크**이다. 모델 크기에 따른 **적응형 프롬프트 압축**, **사전 복잡도 분석 기반 라우팅**, **3단계 하이브리드 메모리**를 통해 SLM이 LLM에 근접한 에이전트 성능을 내도록 돕는다. 특히 소형 모델일수록 프레임워크 최적화의 혜택이 극대화되어, 실행 속도 향상과 정확도 개선이라는 두 마리 토끼를 모두 잡았다. 이 연구는 향후 모델-프레임워크 공동 설계(Co-design) 및 효율적인 로컬 AI 에이전트 배포에 중요한 기초를 제공한다.
