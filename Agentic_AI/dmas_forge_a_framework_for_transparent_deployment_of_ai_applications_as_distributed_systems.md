# DMAS-Forge: A Framework for Transparent Deployment of AI Applications as Distributed Systems

Alessandro Cornacchia, Vaastav Anand, Muhammad Bilal, Zafar Qazi, Marco Canini (2025)

## 🧩 Problem to Solve

최근의 Agentic AI 애플리케이션은 복잡한 작업을 해결하기 위해 서로 다른 역할, 전문 도구, 메모리 계층을 가진 여러 에이전트가 협력하는 Multi-Agent Systems (MAS) 형태로 진화하고 있다. 이러한 구조는 본질적으로 서비스 지향 아키텍처(Service-Oriented Architectures)와 유사하며, 시스템의 규모와 복잡성이 증가함에 따라 이를 분산 시스템으로 배포하고 통합하려는 수요가 높아지고 있다.

분산형 MAS (Distributed MAS, 이하 DMAS)는 마이크로서비스와 유사하게 각 에이전트를 독립적인 서비스나 컨테이너로 배포함으로써 다음과 같은 이점을 제공한다. 첫째, 이기종 런타임 사용이 가능하며 의존성 관리가 단순해진다. 둘째, 컨테이너화를 통해 보안 경계를 강화하고 PII(개인식별정보) 처리 및 외부 API 통합 시 세밀한 제어가 가능하다. 셋째, 장애 격리를 통해 꼬리 지연 시간(tail latency)을 줄이고 시스템의 응답성을 예측 가능하게 만들 수 있다. 마지막으로, 에이전트별로 상이한 CPU, 메모리, I/O 요구사항에 맞춰 독립적인 스케일링과 리소스 최적화가 가능하다.

그러나 LangGraph, CrewAI, AutoGen과 같은 기존 프로그래밍 프레임워크들은 애플리케이션 로직과 실행 환경을 밀접하게 결합(tightly couple)하고 있다. 이들은 주로 단일 시스템(monolithic) 배포를 가정하여 통신 인터페이스를 하드코딩하고 있어, 이를 분산 시스템으로 전환하려면 그래프 분할 로직 결정, 서브 그래프 생성, 프로토콜 어댑터 구현, 인프라 설정 등 막대한 수동 작업이 필요하다. 특히 배포 환경이나 통신 프로토콜이 변경될 때마다 이러한 작업을 반복해야 한다는 점이 가장 큰 문제이다.

본 논문의 목표는 이러한 간극을 메우기 위해 애플리케이션 로직을 배포 선택지로부터 분리하고, 최소한의 수동 작업으로 다양한 시나리오에서 DMAS를 구축할 수 있도록 돕는 프레임워크인 DMAS-Forge를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 "한 번 작성하여 어디서나 배포하는(write-once, deploy-everywhere)" 패러다임을 구현하는 것이다. 이를 위해 핵심 에이전트 로직과 하위 통신 프로토콜 및 배포 인프라를 완전히 분리하는 컴파일러 기반 접근 방식을 제안한다.

핵심적인 설계 직관은 AI 애플리케이션의 핵심 계산 모델(computation model)이 해당 계산이 '어떻게' 그리고 '어디서' 수행되는지와는 완전히 직교(orthogonal)한다는 점이다. 이러한 분리를 통해 개발자는 배포 방식에 구애받지 않고 에이전트의 구조적 워크플로우를 정의할 수 있으며, 이후 컴파일러가 대상 환경에 맞는 glue code와 설정 파일을 자동으로 생성함으로써 배포의 투명성을 확보한다.

## 📎 Related Works

논문에서는 다음과 같은 기존의 접근 방식과 그 한계를 언급한다.

- **기존 MAS 프레임워크**: LangGraph, CrewAI, AutoGen, LlamaIndex, Agno 등이 에이전트 조정, 도구 통합, 메모리 관리 기능을 제공한다. 하지만 이들은 통신 인터페이스가 프레임워크 내부에 하드코딩되어 있어, 분산 배포를 위해서는 개발자가 직접 프로토콜 어댑터를 작성하고 인프라를 구축해야 하는 한계가 있다.
- **통신 프로토콜**: Google의 Agent2Agent (A2A) 프로토콜과 Anthropic의 Model Context Protocol (MCP)과 같은 표준화된 프로토콜이 등장하고 있으나, 기존 프레임워크에서 이러한 프로토콜을 적용하려면 여전히 상당한 수동 엔지니어링이 필요하다.
- **차별점**: DMAS-Forge는 단순한 라이브러리가 아니라 컴파일러로서 동작한다. 사용자가 정의한 워크플로우와 배포 명세(deployment specification)를 입력받아, 대상 환경(Kubernetes, Serverless, VM 등)에 최적화된 실행 아티팩트를 자동으로 생성한다는 점에서 기존 프레임워크와 차별화된다.

## 🛠️ Methodology

### 전체 시스템 구조

DMAS-Forge는 컴파일러 기반의 프레임워크로, 크게 두 가지 입력을 받아 최종 배포 아티팩트를 생성하는 구조를 가진다.

1. **구조적 에이전트 워크플로우 (Structural Agentic Workflow)**: 에이전트 구현체, 관련 도구, 그리고 에이전트 간의 연결 관계를 포함하는 그래프 형태의 계산 워크플로우이다. 본 연구에서는 LangGraph의 계산 그래프 모델을 따라 노드는 에이전트를, 엣지는 에이전트 간 통신을 나타낸다.
2. **배포 명세 (Deployment Specification)**: 각 에이전트와 도구가 실행될 방식(프로세스, 컨테이너, 서버리스 등), 사용될 통신 프로토콜, 런타임 제약 조건(하드웨어 유형, 복제본 수, 액세스 정책), 그리고 각 에이전트가 사용할 LLM에 대한 정보를 포함한다.

### 컴파일 과정 및 작동 원리

컴파일러는 위 두 입력을 바탕으로 다음과 같은 작업을 수행한다.

- **Glue Code 생성**: 연결된 에이전트와 도구가 사용자가 지정한 통신 프로토콜(예: A2A, MCP)을 준수하도록 만드는 인터페이스 코드를 자동으로 삽입한다.
- **구성 파일 생성**: 대상 배포 환경에 따라 Dockerfile, Kubernetes 설정 파일 등을 생성하여 액세스 정책을 강제하고 지정된 LLM 모델에 바인딩한다.

### 프로토타입 구현

논문에서는 Go 언어로 구현된 DMAS-Forge 프로토타입을 제시하며, 이는 마이크로서비스 컴파일러인 Blueprint를 확장하여 구축되었다.

- **Programming Interfaces**:
  - **Workflow API**: 사용자가 LangGraph와 유사하게 구조적 워크플로우를 구현할 수 있도록 지원한다. (예: `.Connect(agent)`, `.AddTool(tool)`)
  - **Wiring API**: 배포 명세를 입력하기 위한 API이다. (예: `a2a.Deploy()`, `mcp.Deploy()`)
- **주요 플러그인**:
  - `AgentPlugin`: OpenAI 호환 모델과 연결된 에이전트를 투명하게 생성한다.
  - `vLLMPlugin`: vLLM을 통한 모델 배포를 자동화한다.
  - `kagentPlugin`: Kubernetes 네이티브 에이전트 프레임워크인 kagent 기반의 분산 배포를 지원한다.
  - `RPC-over-HTTP`: 프로토타입에서 에이전트 간 통신 프로토콜의 예시로 사용된다.

## 📊 Results

본 논문은 정량적인 성능 평가(벤치마크)보다는 프레임워크의 개념 증명(Proof of Concept)과 프로토타입 구현에 집중하고 있다.

- **구현 결과**: Go 언어를 기반으로 한 프로토타입을 통해, 사용자가 간단한 API 호출만으로 두 개의 에이전트를 생성하고 이들을 Linux 컨테이너 환경에 분산 배포하는 과정을 시연하였다(Figure 2 참조).
- **결과적 의의**: 개발자가 인프라의 세부 사항(포트 설정, 네트워크 정책, 프로토콜 어댑터 작성 등)을 직접 다루지 않고도, 고수준의 워크플로우 정의와 배포 명세만으로 DMAS를 구축할 수 있음을 보여주었다.

## 🧠 Insights & Discussion

### 강점 및 기회

DMAS-Forge는 배포의 추상화를 통해 **폐루프 최적화 파이프라인(Closed-loop optimization pipeline)** 구축을 가능하게 한다. 즉, 애플리케이션을 배포한 후 통신 패턴과 리소스 사용량을 프로파일링하고, 그 결과에 따라 수동 재배포 없이 워크플로우 구조를 자동으로 재구성하여 성능을 최적화할 수 있다. 예를 들어, 통신 비용이 높은 에이전트들을 동일한 컨테이너에 배치(co-location)하거나, 부하가 높은 에이전트의 복제본 수를 조정하는 결정이 자동으로 이루어질 수 있다.

### 한계 및 향후 과제

- **자동 보안 경계 설정**: 에이전트들이 최적화 파이프라인에 의해 재배치될 때, 각 컨테이너의 공격 표면을 최소화하기 위해 최소 권한 원칙에 기반한 보안 정책을 자동으로 조정하는 연구가 필요하다.
- **엔지니어링 챌린지**: DMAS-Forge가 독자적인 에이전트 기능을 내장할 것인지, 아니면 기존 프레임워크(LangGraph 등) 위의 추상화 계층으로 작동할 것인지에 대한 결정이 필요하다. 후자의 경우, 프레임워크 고유의 통신 프리미티브를 프로토콜 준수 채널로 리다이렉션하기 위해 몽키 패칭(monkey patching)과 같은 기법이 필요할 수 있다.
- **환경 확장**: 현재 프로토타입은 Linux 컨테이너를 주로 지원하지만, 향후 더 다양한 런타임 환경으로 확장할 계획이다.

## 📌 TL;DR

본 논문은 Multi-Agent Systems (MAS)를 분산 시스템(DMAS)으로 배포할 때 발생하는 막대한 수동 엔지니어링 비용을 해결하기 위해, 애플리케이션 로직과 배포 인프라를 분리하는 컴파일러 기반 프레임워크 **DMAS-Forge**를 제안한다. 이 프레임워크는 워크플로우 정의와 배포 명세를 입력받아 통신 glue code와 인프라 설정 파일을 자동으로 생성함으로써 "Write-once, deploy-everywhere"를 가능하게 한다. 이는 향후 AI 에이전트 시스템의 런타임 최적화와 보안 강화 및 효율적인 클라우드 네이티브 배포에 중요한 기여를 할 것으로 기대된다.
