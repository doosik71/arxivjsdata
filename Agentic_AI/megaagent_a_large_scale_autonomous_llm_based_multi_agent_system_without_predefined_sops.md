# MegaAgent: A Large-Scale Autonomous LLM-based Multi-Agent System Without Predefined SOPs

Qian Wang, Tianyu Wang, Zhenheng Tang, Qinbin Li, Nuo Chen, Jingsheng Liang, Bingsheng He (2025)

## 🧩 Problem to Solve

본 논문은 LLM 기반의 Multi-Agent System(MAS)이 복잡한 작업을 수행할 때 겪는 두 가지 핵심적인 한계점을 해결하고자 한다.

첫째, 기존의 MAS는 작업의 규모가 커지고 복잡해질 때 적응적인 작업 조정(Adaptive Task Coordination) 능력이 부족하며, 특히 수백 명 이상의 에이전트가 투입되는 대규모 시뮬레이션 환경에서 에이전트 간의 효율적인 협업 체계를 구축하는 데 어려움이 있다.

둘째, 대부분의 기존 시스템은 사용자가 사전에 정의한 에이전트의 역할, 표준 운영 절차(Standard Operating Procedures, SOPs), 그리고 정적인 통신 그래프에 과도하게 의존한다. 이러한 방식은 시스템의 유연성을 저해할 뿐만 아니라, 대규모 에이전트를 배포할 때 막대한 인간의 설계 비용(Human effort)을 요구한다는 문제가 있다.

따라서 본 연구의 목표는 사전 정의된 SOP 없이도 스스로 작업을 분해하고, 에이전트를 동적으로 생성하며, 대규모 협업을 자율적으로 수행할 수 있는 확장 가능한 자율형 MAS인 MegaAgent를 개발하는 것이다.

## ✨ Key Contributions

MegaAgent의 핵심 아이디어는 운영체제(Operating System, OS)의 프로세스와 스레드 관리 방식에서 영감을 얻은 것이다. OS가 프로세스 내에서 여러 스레드를 생성해 작업을 처리하고, 프로세스 간 통신(IPC)을 통해 데이터를 주고받는 것처럼, MegaAgent는 대규모 작업을 계층적 구조로 분해하여 관리한다.

중심적인 설계 아이디어는 다음과 같다.

1. **자율적 SOP 생성**: 인간이 SOP를 설계하는 대신, LLM 에이전트가 작업의 복잡도에 따라 스스로 작업을 분할하고 하위 에이전트들을 위한 SOP를 동적으로 생성하게 한다.
2. **계층적 작업 관리 및 모니터링**: Boss Agent, Admin Agent, Ordinary Agent로 이어지는 3단계 계층 구조를 통해 작업 분배와 실행, 검증을 체계화한다.
3. **병렬 실행 및 효율적 통신**: 동일 레벨의 에이전트 그룹들이 병렬로 작업을 수행하게 함으로써 시간 복잡도를 낮추고, 메시지 큐(Message Queue) 기반의 비동기 통신을 통해 효율성을 극대화한다.

## 📎 Related Works

기존의 MAS 연구들(MetaGPT, AutoGen, AgentVerse 등)은 특정 페르소나를 부여하거나 계획 및 메모리 메커니즘을 도입하여 단일 에이전트보다 뛰어난 성능을 보여주었다. 그러나 논문은 다음과 같은 한계점을 지적한다.

- **MetaGPT**: 소프트웨어 개발 프로세스를 시뮬레이션하기 위해 매우 정교한 SOP를 사용하지만, 이는 사용자가 사전에 역할을 정의해야 하며 고정된 파이프라인을 따른다는 한계가 있다.
- **AutoGen 및 AgentVerse**: 에이전트 간의 협업을 지원하지만, 대부분 순차적인(Sequential) 파이프라인을 사용하며 대규모 에이전트 확장 시 발생하는 통신 및 조정 문제를 충분히 해결하지 못한다.

MegaAgent는 이러한 기존 방식과 달리 사전 정의된 SOP를 완전히 제거하고, 동적인 에이전트 생성과 병렬 실행을 지원함으로써 자율성과 확장성 측면에서 차별점을 가진다.

## 🛠️ Methodology

### 1. 계층적 작업 관리 (Hierarchical Task Management)

MegaAgent는 작업을 효율적으로 처리하기 위해 세 가지 레벨의 계층 구조를 가진다.

- **Boss Agent Level**: 사용자의 Meta-prompt를 받아 전체 작업을 작은 서브태스크로 분해하고, 이를 관리할 Admin Agent들을 생성 및 할당한다.
- **Dynamic Hierarchical Group Formation**: Admin Agent는 할당된 서브태스크가 자신의 처리 용량을 초과한다고 판단할 경우, 추가적인 에이전트를 모집하여 하위 그룹을 형성한다. 이 과정은 재귀적으로 일어나므로 작업 복잡도에 따라 계층이 동적으로 확장된다.
- **System-Level Coordination**: 동일 레벨의 그룹들은 병렬로 작업을 수행하며, 외부 시스템과는 Function Call을 통해 상호작용한다.

### 2. 계층적 조정 메커니즘 (Hierarchical Coordination Mechanism)

통신 효율을 높이기 위해 두 가지 층위의 채팅 구조를 도입한다.

- **Intra-group Chat**: 동일한 작업 그룹 내의 에이전트들이 프롬프트 기반으로 업데이트 내용을 공유하며 협업한다.
- **Inter-group Chat**: 서로 다른 그룹의 Admin Agent들끼리만 통신하여 작업 간의 의존성을 해결하고 조율한다. 일반 에이전트(Ordinary Agent)는 그룹 외부와 직접 통신할 수 없도록 제한하여 통신 오버헤드를 줄인다.

### 3. 메시지 큐 메커니즘 (Message Queue Mechanism)

비동기 통신을 위해 생산자-소비자(Producer-Consumer) 패턴 기반의 메시지 큐를 사용하며, 에이전트는 다음 세 가지 상태를 가진다.

1. **Idle State**: 메시지가 없는 상태로, 계산 비용이 발생하지 않는다.
2. **Processing State**: 큐에서 메시지 배치를 가져와 LLM 추론을 수행하는 원자적 실행 단계이다.
3. **Response State**: 결과물의 유효성을 검증한 후 지정된 함수 호출을 통해 응답을 전송한다.

### 4. 파일 및 메모리 관리

- **Git-Based Version Control**: 여러 에이전트가 동시에 파일을 수정할 때 발생하는 충돌을 막기 위해 Git의 커밋 해시 기반 버전 관리와 글로벌 뮤텍스 락(Global Mutex Lock)을 사용하여 동기화를 유지한다.
- **Vector Database**: LLM의 토큰 제한으로 인한 망각 문제를 해결하기 위해 에이전트의 출력물을 임베딩하여 벡터 데이터베이스에 저장하고, 필요 시 관련 메모리를 검색하여 문맥을 유지한다.

### 5. 계층적 모니터링 (Hierarchical Monitoring)

환각(Hallucination)의 전파를 막기 위해 3단계 모니터링 체계를 구축한다.

- **Agent-Level**: 각 에이전트는 생성 시 부여받은 체크리스트(Checklist)를 통해 자신의 진행 상황을 스스로 검증한다.
- **Group-Level**: Admin Agent가 소속 에이전트들의 작업 진행 상황을 감독하고 조정한다.
- **System-Level**: Boss Agent가 최종 출력물의 형식과 정확성을 검토하여 시스템 전체의 일관성을 보장한다.

## 📊 Results

### 1. 표준 벤치마크 성능 (RQ1)

MBPP, HumanEval, MATH, GSM-8K 데이터셋에서 GPT-4o 모델을 사용하여 비교 실험을 진행하였다. MegaAgent는 모든 지표에서 AutoGen, MetaGPT, CAMEL, AgentVerse보다 높은 정확도를 기록하였다. 특히 GSM-8K에서는 $93.0\%$의 정확도를 달성하며 가장 뛰어난 기본 문제 해결 능력을 보였다.

### 2. 오목 게임 개발 실험 (RQ2)

소프트웨어 개발 능력을 평가하기 위해 오목(Gobang) 게임 제작 작업을 수행하였다.

- **결과**: MegaAgent는 자율적으로 7명의 에이전트로 구성된 SOP를 생성하고, 800초 만에 실행 가능한 인터페이스를 포함한 완전한 게임을 개발하는 데 성공하였다.
- **비교**: AutoGen은 코드가 중간에 끊기거나 실행되지 않았으며, MetaGPT는 틱택토 게임을 만드는 등 엉뚱한 결과물을 생성하거나 무한 루프에 빠졌다. CAMEL과 AgentVerse는 실행 불가능한 코드나 플레이스홀더가 포함된 결과를 출력하였다.

### 3. 국가 정책 생성 실험 (RQ3)

대규모 확장성을 평가하기 위해 교육, 보건, 금융 등 여러 도메인의 국가 정책을 수립하는 시뮬레이션을 수행하였다.

- **결과**: MegaAgent는 약 590명의 에이전트를 동적으로 생성하고 조율하여 2,991초 만에 완성도 높은 정책 문서를 생성하였다. 에이전트당 평균 처리 시간은 5초에 불과했다.
- **평가**: LLM-as-a-Judge(Claude-3.5, GPT-4o 등 5종의 모델)를 통해 검증한 결과, 생성된 31개 정책 중 평균 27.4개가 합리적이라는 평가를 받았다.
- **비교**: baseline 모델들은 대부분 10명 미만의 에이전트만을 활용하였으며, 구체적인 정책 내용 없이 단순한 개요(Outline)나 파이썬 코드만을 출력하는 데 그쳤다.

### 4. 절제 연구 (Ablation Study)

- **계층 구조 제거**: 에이전트 수는 줄어드나 작업 완료 시간이 증가하며, 복잡한 작업(국가 정책 등)에서는 불완전한 결과가 생성된다.
- **병렬성 제거**: 시간 복잡도가 $O(\log n)$에서 $O(n)$으로 증가하여 실행 시간이 급격히 늘어난다. (오목 게임의 경우 800초 $\rightarrow$ 4,505초)
- **모니터링 제거**: 실행 시간은 단축되지만, 치명적인 버그가 포함되거나 플레이스홀더가 남은 불완전한 결과물이 생성된다.

## 🧠 Insights & Discussion

### 강점

MegaAgent는 기존 MAS의 고질적인 문제였던 '인간 설계 기반의 SOP 의존성'을 완전히 제거함으로써 진정한 의미의 자율성을 확보하였다. 특히 OS의 구조를 차용한 계층적 관리와 병렬 처리 덕분에 에이전트 수를 수백 명 단위로 확장하더라도 시스템이 붕괴되지 않고 효율적으로 동작함을 입증하였다.

### 한계 및 비판적 해석

1. **통신 오버헤드**: 비용 분석 결과, 전체 토큰 사용량의 대부분이 에이전트 간의 대화(Input tokens)에서 발생한다. 특히 Input 대비 Output 토큰 비율이 약 23:1에서 25:1에 달하는데, 이는 대규모 시스템일수록 통신 효율화가 필수적임을 시사한다.
2. **환각의 전파**: 계층적 모니터링으로 완화하고는 있으나, 체크리스트 자체를 LLM이 생성하므로 초기 단계에서 잘못된 기준이 설정될 경우 오류가 하위 단계로 전파될 위험이 여전히 존재한다.
3. **비용 문제**: GPT-4o와 같은 고성능 모델에 의존하므로 API 비용 부담이 크다. 특정 도메인에 특화된 소형 LLM(sLLM)과의 혼합 구성이 필요해 보인다.

## 📌 TL;DR

MegaAgent는 사전 정의된 SOP 없이도 LLM이 스스로 역할을 분담하고 작업을 조율하는 **대규모 자율형 Multi-Agent System**이다. OS의 계층적 구조와 병렬 실행, 메시지 큐 기반 통신, 3단계 모니터링 체계를 통해 수백 명의 에이전트를 효율적으로 관리할 수 있으며, 이를 통해 복잡한 소프트웨어 개발 및 대규모 사회 시뮬레이션에서 기존 시스템을 압도하는 성능과 확장성을 보여주었다. 이 연구는 향후 금융 시장 시뮬레이션이나 의료 시스템 최적화와 같이 극도로 복잡한 협업이 필요한 분야에 적용될 가능성이 매우 높다.
