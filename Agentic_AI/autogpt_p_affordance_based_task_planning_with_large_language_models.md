# AutoGPT+P: Affordance-based Task Planning using Large Language Models

Timo Birr, Christoph Pohl, Abdelrahman Younes and Tamim Asfour

## 🧩 Problem to Solve

LLM(Large Language Model) 기반 로봇 작업 계획은 일반화 능력 향상에 기여했으나, 고전적인 플래너의 **닫힌 세계 가정(closed-world assumption)**과 **작업 초기 상태의 동적 파악 능력 부족**이라는 한계를 가집니다. 특히, 작업에 필요한 객체가 없거나 정보가 불완전한 상황에서는 효과적인 계획을 수립하기 어렵습니다.

## ✨ Key Contributions

* **어포던스(Affordance) 기반 장면 표현:** 객체 탐지(Object Detection)와 ChatGPT로 자동 생성된 객체 어포던스 매핑(Object Affordance Mapping, OAM)을 결합한 새로운 장면 표현 방식을 제안합니다.
* **LLM 기반 도구 선택을 통한 유연한 작업 계획:** 사용자의 자연어 명령을 바탕으로 완전/부분 계획 수립, 누락된 객체 탐색 및 대체 방안 제안 등 불완전한 정보 상황에 대처하는 작업 계획 접근 방식을 개발합니다.
* **LLM+P 계획 방식 확장:** 동적 PDDL 도메인 생성과 자동화된 의미론적(semantic) 및 구문론적(syntactic) 오류 수정을 통해 LLM 기반 계획의 견고성을 향상시킵니다.
* **실세계 검증:** 휴머노이드 로봇을 이용한 실제 환경에서의 시스템 실행 가능성을 검증하여 제안된 접근 방식의 실용성을 입증합니다.

## 📎 Related Works

* **어포던스 기반 계획:** PDDL 도메인에서 어포던스 활용([31], [9], [10]), 관계형 어포던스 학습([32]), 기능적 어포던스 기반 객체 대체([3], [4], [5]) 연구들이 어포던스를 활용한 계획을 다루었습니다.
* **LLM 기반 작업 계획:**
  * **하위 작업 평가 (Subtask Evaluation):** SayCan ([1])과 같이 LLM이 각 단계에서 가능한 행동을 평가하여 계획을 세웁니다.
  * **전체 자율 회귀 계획 생성 (Full Autoregressive Plan Generation):** LLM이 사용자 지정 작업에 따라 전체 계획을 한 번에 생성합니다 ([44], [39], [37], [34], [49]).
  * **단계별 자율 회귀 계획 생성 (Step-By-Step Autoregressive Plan Generation):** LLM이 한 번에 한 가지 행동만 생성하며, 각 단계 실행 후 피드백을 받아 계획을 개선합니다 ([19], [20], [36], [26], [43], [6]).
  * **플래너와 결합된 LLM (LLM with Planner):** LLM이 자연어 명령을 PDDL 목표로 번역하고, 고전적인 플래너가 이를 기반으로 계획을 수립합니다 ([45], LLM+P [28], [30], [17], [8], [12]).
* 본 연구는 **단계별 자율 회귀 계획 생성**과 **플래너와 결합된 LLM** 패러다임을 결합한 하이브리드 접근 방식을 취합니다.

## 🛠️ Methodology

* **어포던스 기반 장면 표현 ($S$):**
  * 장면 $S$를 객체-어포던스 쌍의 집합인 $S = \{p_1,...,p_n\}$으로 상징적으로 표현합니다. 여기서 $p_i = (o_i, k_i, a_i, b_i)$는 각각 객체 클래스, 객체 인스턴스 ID, 어포던스, 바운딩 박스를 나타냅니다. (여기서 $O$는 모든 객체 클래스의 집합, $N_0$는 객체 인스턴스 ID, $A$는 가능한 어포던스의 집합, $[0,1]^4$는 정규화된 바운딩 박스를 의미합니다.)
  * **객체 어포던스 탐지(OAD):** RGB 이미지 $I$로부터 이러한 장면 표현 $S$를 추출하며, 이는 객체 탐지(Object Detection)와 객체 어포던스 매핑(Object Affordance Mapping, OAM)의 두 단계로 구성됩니다. OAD는 다음과 같이 표현됩니다: $OAD: I \to S$.
  * **OAM 생성 (ChatGPT 활용):** ChatGPT를 사용하여 객체 클래스(예: 칼, 컵)와 어포던스(예: 자르기, 잡기, 담기, 붓기 등 40가지)를 연결하는 OAM을 자동 생성합니다. 이는 주로 `Yes/No-Questions + Logical Combinations` 전략을 사용하여 정확도를 높였습니다.
* **AutoGPT+P 피드백 루프:**
  * **도구 선택(Tool Selection):** LLM은 현재 로봇의 메모리 $M = (S, R, L, l_{\Pi}, \hat{\lambda}, Alt, P)$ (장면 표현 $S$, 객체 관계 $R$, 탐색 가능한 위치 $L$, 현재 에이전트 위치 $l_{\Pi}$, 명령어 이력 $\hat{\lambda}$, 알려진 대체 객체 $Alt$, 최신 계획 $P$)와 사용자 명령 $\lambda$를 기반으로 다음 네 가지 도구 중 하나를 선택합니다:
    * `Plan`: 닫힌 세계 계획 문제 해결.
    * `Partial Plan`: 현재 $S$의 제약 조건 내에서 가능한 최선의 부분 계획 수립.
    * `Suggest Alternative`: 누락된 객체에 대한 대체 객체 제안.
    * `Explore`: 로봇이 미탐색 위치 $l \in L$로 이동하여 카메라 이미지로부터 객체-어포던스 쌍을 추출하고 $S$를 업데이트합니다: $S \leftarrow S \cup OAD(I)$.
  * **어포던스 기반 대체 제안:** 누락된 객체 클래스 $m$에 대해, LLM에 해당 객체의 어떤 어포던스 $aff_m$가 사용자 작업 $\lambda$에 관련이 있는지 질의합니다. 이 정보를 바탕으로 관련 어포던스를 모두 가진 장면 내의 객체들 $O_{filtered}$를 필터링하고, 가장 유사한 대체 객체 $alt$를 LLM에 질의하여 제안합니다.
  * **어포던스 기반 계획 (LLM with Planner):**
    * 사용자 명령 $\lambda$를 PDDL 목표 상태 $\Gamma$로 변환합니다.
    * **동적 PDDL 도메인 및 문제 생성:** 어포던스 기반 장면 표현 및 에이전트의 능력에 따라 PDDL 도메인 $\Delta$와 초기 상태 $\hat{\Xi}$를 동적으로 생성합니다. (예: `sub(object) = A` (객체의 하위 타입은 어포던스), `sub(a) = {o | a ∈ OAM(o)}` (특정 어포던스 $a$의 하위 타입은 해당 어포던스를 가진 객체 $o$)).
    * **외부 피드백을 통한 자가 수정:** 생성된 PDDL 목표 $\Gamma$의 구문론적 및 의미론적 오류를 확인하고, 오류 발생 시 LLM이 스스로 수정하도록 구체적인 오류 메시지를 피드백으로 제공합니다 (의미론적 오류 검출에는 `Prolog`를 활용하여 논리적 모순 여부를 검사).

## 📊 Results

* **OAM 정확도:** `GPT-4`와 `Yes/No-Questions + Logical Combinations` 전략을 사용했을 때 가장 높은 F1-score인 0.89를 달성했습니다.
* **대체 제안 도구:** 어포던스 기반 접근 방식은 객체 수가 많은 복잡한 장면에서 나이브(naive) 접근 방식보다 `GPT-4`에서 약 13% 높은 정확도(complex 시나리오에서 0.80 vs 0.67)로 뛰어난 성능을 보였습니다.
* **계획 도구 (`Plan Tool`):**
  * `SayCan` 명령어 세트에서 `GPT-4` 사용 시 98%의 성공률을 달성했습니다 (자가 수정 없는 버전은 78%). 이는 나이브 `LLM as Planner` 구현보다 훨씬 우수하며, 어포던스 정보를 직접 LLM에 제공하는 것은 오히려 성능을 저하시켰습니다.
  * `Simple Task` 및 `Simple Goal`에서는 거의 100% 성공률을 보였으나, `Knowledge` 및 `Implicit`와 같은 모호한 작업에서는 어려움을 겪었습니다 (GPT-4 Auto 기준 0.57 및 0.47).
  * 오류 자가 수정 기능은 `SayCan` 데이터셋에서 성공률을 78%에서 98%로 크게 향상시키는 중요한 요소였습니다.
* **AutoGPT+P (전체 시스템):** 새로 생성된 150개 시나리오 데이터셋에서 평균 79%의 성공률을 기록했습니다. `GPT-3`는 `GPT-4`에 비해 도구 선택 task에서 현저히 낮은 성공률을 보였습니다.
* **실세계 검증:** 휴머노이드 로봇 `ARMAR-6` 및 `ARMAR-DE`에서 20개 실제 시나리오 중 15개를 성공적으로 계획하고 실행했습니다. 실패 사례는 주로 객체 탐지 오류나 로봇의 잡기 능력 부족에서 기인했습니다.

## 🧠 Insights & Discussion

* **강점:** AutoGPT+P는 어포던스 기반 장면 표현을 통해 누락된 객체 등 복잡한 계획 상황을 처리할 수 있으며, 높은 성공률로 자연어 명령을 유효한 계획으로 변환할 수 있습니다. 명시적으로 잘 정의된 사용자 목표에 대해서는 거의 100%에 가까운 성공률을 보입니다.
* **한계:**
  * 사용자 의도의 모호성 처리 능력 부족: 사용자 의도의 불확실성을 평가하고 명확화를 요청하는 기능이 현재는 없습니다.
  * 결정론적 모델: 객체 인식 오류, LLM 생성 OAM의 불확실성, 불명확한 사용자 명령 등 실세계의 본질적인 불확실성을 모델에 통합하지 못했습니다.
  * 실행 피드백 부재: 실행 중 발생하는 실패(예: 기술 실패)로부터 피드백을 받아 계획을 수정하는 기능이 없어 실세계 적용 시 오류에 취약합니다.
  * 장기 계획 제한: 20단계 이상의 장기 계획 문제에 대한 효과적인 해결 능력에 한계가 있을 수 있습니다.
* **의미:** AutoGPT+P는 도구 선택 프로세스를 통해 누락된 객체와 부분적으로 미탐색된 장면이 있는 작업을 해결할 수 있는 잠재력을 보여주었습니다.

## 📌 TL;DR

AutoGPT+P는 LLM 기반 로봇 작업 계획의 닫힌 세계 가정과 동적 환경 파악의 한계를 극복하기 위해 제안된 시스템입니다. 이 시스템은 **어포던스 기반 장면 표현** (객체 탐지 + ChatGPT 생성 OAM)을 활용하여 로봇의 메모리를 동적으로 업데이트하고, LLM 기반 **도구 선택 메커니즘**을 통해 `Plan`, `Partial Plan`, `Suggest Alternative`, `Explore` 중 적절한 도구를 선택합니다. 특히, `Suggest Alternative` 도구는 누락된 객체의 어포던스를 기반으로 대안을 제안하며, `Plan` 도구는 PDDL 도메인 및 문제의 **동적 생성**과 **구문/의미론적 자가 수정 기능**을 통해 LLM+P를 확장합니다. `SayCan` 데이터셋에서 98%, 새로운 복합 시나리오 데이터셋에서 79%의 성공률을 달성했으며, 실세계 로봇에서 실행 가능함을 검증하여 **불완전한 정보 상황에서의 로봇 작업 계획의 유연성과 견고성**을 크게 향상시켰습니다.
