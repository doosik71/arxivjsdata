# AGENTSTUDIO: A TOOLKIT FOR BUILDING GENERAL VIRTUAL AGENTS

Longtao Zheng, Zhiyuan Huang, Zhenghai Xue, Xinrun Wang, Bo An, Shuicheng Yan (2025)

## 🧩 Problem to Solve

현재의 가상 에이전트(Virtual Agents) 연구는 특정 도메인에 국한된 환경(예: 웹 브라우저 전용)과 복잡한 설정 과정으로 인해 실세계의 일반적인 환경에서 에이전트를 개발하고 평가하는 데 한계가 있다. 특히 기존의 벤치마크는 단순한 성공률(Success Rate) 지표에만 의존하여, 에이전트가 갖추어야 할 근본적인 역량을 세분화하여 분석하지 못하는 문제가 있다.

본 논문은 일반적인 가상 에이전트가 갖추어야 할 세 가지 핵심 요구사항인 **일반적 그라운딩(General Grounding)**, **광범위한 디지털 세계 지식(Extensive Digital World Knowledge)**, 그리고 **개방형 환경에서의 자기 개선 능력(Open-ended Learning)**을 충족시키기 위한 통합 툴킷인 `AgentStudio`를 제안한다.

## ✨ Key Contributions

`AgentStudio`는 환경(Environments), 도구(Tools), 벤치마크(Benchmarks)의 세 가지 요소가 결합된 '트리니티(Trinity)' 구조를 통해 일반 가상 에이전트의 구축과 평가를 지원한다.

- **범용 관찰 및 행동 공간 제공**: 텍스트, 이미지뿐만 아니라 비디오 관찰을 지원하며, GUI 조작과 API 호출을 동시에 수행할 수 있는 통합 행동 공간을 정의하였다.
- **데이터 구축 도구 통합**: GUI 요소 어노테이션, 비디오 내 행동 라벨링, 벤치마크 태스크 생성 및 검증을 위한 효율적인 도구를 제공한다.
- **세분화된 평가 데이터셋 구축**: 단순 성공률 측정을 넘어 UI 그라운딩(`GroundUI`), 비디오 기반 행동 학습(`IDMBench`), 성공 여부 판단(`CriticBench`)이라는 세 가지 핵심 능력을 평가하는 데이터셋을 구축하였다.

## 📎 Related Works

기존의 가상 에이전트 환경인 World of Bits, WebArena, OSWorld 등은 각각 웹이나 데스크톱 환경을 시뮬레이션하지만, 다음과 같은 한계가 존재한다.

- **도메인 제한**: 대부분의 환경이 웹 기반 조작($A_{API}$)에 치중되어 있으며, 실세계의 다양한 소프트웨어를 아우르지 못한다.
- **관찰 공간의 협소함**: 스크린샷(이미지)이나 HTML(텍스트) 수준의 관찰만 제공하며, 실시간 동적 변화를 포착할 수 있는 비디오 관찰($O_{Video}$)이 결여되어 있다.
- **정적 데이터셋의 한계**: 많은 벤치마크가 정적인 데이터셋 형태로 제공되어, 에이전트가 시행착오를 통해 학습할 수 있는 온라인 인터랙티브 환경을 제공하지 못한다.

`AgentStudio`는 이러한 한계를 극복하기 위해 비디오 관찰과 GUI/API 하이브리드 행동 공간을 지원하는 가볍고 인터랙티브한 환경을 제공함으로써 차별점을 갖는다.

## 🛠️ Methodology

### 1. 시스템 모델링 (POMDP)

컴퓨터 작업은 부분 관측 마르코프 결정 과정(Partially Observable Markov Decision Process, POMDP)으로 모델링되며, 이는 다음과 같은 튜플로 정의된다: $(S, A, O, T, R, F, U)$

- $S$: 상태 공간 (State Space)
- $A$: 행동 공간 (Action Space)
- $O$: 관찰 공간 (Observation Space)
- $T$: 전이 함수 (Transition Function)
- $R$: 보상 함수 (Reward Function)
- $F$: 피드백 공간 (Feedback Space)
- $U$: 지시어 공간 (Instruction Space)

### 2. 범용 관찰 및 행동 공간

`AgentStudio`는 실세계의 복잡한 상호작용을 모사하기 위해 관찰 및 행동 공간을 다음과 같이 정의한다.

- **관찰 공간 ($O$)**: 텍스트, 이미지, 비디오 모달리티의 합집합으로 정의된다.
  $$O = O_{Text} \cup O_{Image} \cup O_{Video}$$
  이를 통해 에이전트는 스크린 레코딩 비디오, 스크린샷, 코드 실행 결과 등을 모두 수신할 수 있다.

- **행동 공간 ($A$)**: GUI 상호작용과 API 호출의 합집합으로 정의된다.
  $$A = A_{GUI} \cup A_{API}$$
  $A_{GUI}$는 키보드 및 마우스 제어와 같은 저수준 조작을 의미하며, $A_{API}$는 고수준의 함수 호출을 의미한다. 에이전트는 파이썬 커널을 통해 코드를 실행함으로써 이 두 공간을 자유롭게 활용할 수 있다.

### 3. 인터랙티브 환경 및 피드백

본 환경은 실제 장치에서 실시간으로 동작하며, 전이 함수 $T: S \times A \to S \times O$는 실제 컴퓨터의 동적인 변화를 반영한다. 특히, 에이전트의 자기 수정을 돕기 위해 이진 보상 $R: S \to \{0, 1\}$뿐만 아니라, 실패 원인을 설명하는 자연어 피드백 $F: S \to \{\text{failure reason, success}\}$를 제공한다.

### 4. 구축 도구 (Tools)

- **태스크 생성 및 검증**: $S, A, O, U$를 정의하고 자동 평가기($R$ 및 $F$)를 구축하여 인간의 성공률과 작업 시간을 측정한다.
- **GUI 요소 어노테이션**: 스크린샷 내 UI 요소에 바운딩 박스를 그리고 단계별 지시어를 추가하는 파이프라인을 제공한다.
- **비디오 행동 기록 및 정제**: 사용자의 작업 궤적(비디오 및 행동)을 기록한 후, 에디터를 통해 불필요한 마우스 움직임을 제거하고 원자적 행동을 고수준 행동으로 병합하여 데이터의 품질을 높인다.

## 📊 Results

### 1. 온라인 벤치마크 태스크 (Online Task Suite)

VS Code, Google Workspace, Office 등 다양한 앱을 포함한 205개의 실세계 태스크를 통해 GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro 등을 평가하였다.

- **정량적 결과**: 인간의 성공률은 $72.2\%$인 반면, 최상위 모델인 GPT-4o와 Claude 3.5 Sonnet조차 전체 성공률이 $30\%$대에 머물렀다. 특히 전문 소프트웨어(GIMP, Office 등) 및 여러 앱을 넘나드는 compositional 태스크에서 성능이 급격히 저하되었다.
- **시각적 정보의 역설**: API만으로 해결 가능한 작업임에도 불구하고 스크린샷($O_{Image}$)을 제공했을 때, 일부 모델(특히 Gemini)은 효율적인 API 호출 대신 복잡한 GUI 조작을 시도하여 오히려 성능이 떨어지는 현상이 발견되었다.

### 2. 세분화된 능력 평가 (Fine-grained Evaluation)

- **GroundUI (UI Grounding)**: 모델이 지시어에 맞는 정확한 UI 좌표를 예측하는 능력을 평가하였다. 전문 모델인 `SeeClick`이 가장 높은 성능을 보였으며, 일반 VLM들은 해상도가 높아질수록(모바일 $\to$ 데스크톱) 정확도가 떨어지는 경향을 보였다.
- **IDMBench (Learning from Videos)**: 비디오에서 행동을 추론하는 Inverse Dynamics Models(IDM) 능력을 평가하였다. 단일 단계 행동 예측(IDM-Single)에서는 어느 정도 성능을 보였으나, 전체 행동 시퀀스를 예측하는 IDM-Multiple에서는 성능이 매우 낮게 나타나, 현재 모델들이 비디오 튜토리얼로부터 학습하기에는 역부족임을 확인하였다.
- **CriticBench (Success Detection)**: 궤적을 보고 작업 성공 여부를 판단하는 능력을 평가한 결과, 모델 간의 성능 차이가 크지 않았으며 여전히 실용적인 수준의 정확도에 도달하지 못했다.

## 🧠 Insights & Discussion

### 강점 및 분석

본 연구는 단순히 에이전트의 성능을 측정하는 것을 넘어, 관찰 공간($O$)과 행동 공간($A$)의 확장성이 에이전트의 유연성에 미치는 영향을 분석하였다. 특히, **'시각적 방해'** 현상을 발견하여, 에이전트가 언제 API를 사용하고 언제 GUI를 사용해야 할지 결정하는 전략적 판단 능력이 중요함을 시사하였다.

### 한계 및 비판적 해석

- **자동 평가기의 신뢰성**: 실세계 환경의 복잡성으로 인해 자동 평가기가 간혹 오판할 가능성이 있으며, 이는 저자들도 인정한 한계점이다.
- **데이터 누수 가능성**: 정적인 데이터셋(GroundUI 등)의 경우, 최신 VLM들의 학습 데이터에 포함되었을 가능성이 있어 성능이 과대평가되었을 수 있다.
- **실행 환경의 제약**: 재현성을 위해 Ubuntu Docker 환경에서 실험을 진행하였으나, 이는 실제 Windows나 macOS의 모든 복잡성을 완전히 반영하지 못했을 수 있다.

## 📌 TL;DR

`AgentStudio`는 범용 가상 에이전트 개발을 위해 **범용 관찰/행동 공간을 갖춘 환경**, **데이터 구축 도구**, 그리고 **세분화된 능력 평가 벤치마크**를 통합 제공하는 툴킷이다. 실험 결과, 최신 VLM들조차 정밀한 UI 그라운딩과 비디오 기반 행동 추론, 그리고 효율적인 API/GUI 선택 전략에서 큰 한계를 보였으며, 이는 향후 일반 가상 에이전트 연구가 집중해야 할 방향(그라운딩 강화, 비디오 학습, 성공 판단 능력 개선)을 명확히 제시한다.
