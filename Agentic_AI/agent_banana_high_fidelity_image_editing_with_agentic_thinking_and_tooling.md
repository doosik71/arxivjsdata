# Agent Banana: High-Fidelity Image Editing with Agentic Thinking and Tooling

Ruijie Ye, Jiayi Zhang, Zhuoxin Liu, et al. (2026)

## 🧩 Problem to Solve

본 논문은 전문적인 워크플로우(Professional Workflows) 관점에서의 명령어 기반 이미지 편집(Instruction-based image editing)이 직면한 세 가지 핵심 과제를 해결하고자 한다.

첫째, **과도한 편집(Over-editing)** 문제이다. 기존 모델들은 사용자의 의도를 벗어나 타겟 외의 영역까지 수정하는 경향이 있으며, 이는 전문적인 작업에서 요구되는 정밀한 국소 수정 능력의 부족을 의미한다.

둘째, **멀티턴 편집의 일관성(Multi-turn consistency)** 결여이다. 대부분의 기존 모델은 단일 턴(Single-turn) 작업에 최적화되어 있으며, 여러 단계의 수정이 이어지는 멀티턴 환경에서는 객체의 정체성이나 이미지의 충실도가 저하되는 문제가 발생한다.

셋째, **해상도 불일치(Resolution Gap)** 문제이다. 실제 전문 작업은 4K 이상의 초고해상도(Ultra HD) 이미지에서 수행되지만, 기존 모델들은 대부분 1K 수준의 저해상도에서 작동하거나 다운샘플링에 의존하여 미세한 텍스처와 경계선을 보존하지 못한다.

따라서 본 연구의 목표는 복잡한 요청을 논리적으로 분해하고, 원본 해상도를 유지하며, 멀티턴 상호작용에서도 배경 드리프트(Background drift) 없이 고충실도 편집을 수행할 수 있는 에이전트 기반 시스템을 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 고충실도, 객체 인식 기반의 이미지 편집을 위해 **Planner-Executor** 구조의 계층적 에이전트 프레임워크인 **Agent Banana**를 제안한 것이다. 중심적인 설계 아이디어는 다음과 같다.

1. **Context Folding**: 긴 상호작용 이력을 구조화된 메모리로 압축하여 LLM의 컨텍스트 제한 문제를 해결하고, 장기적인 제어 안정성을 확보한다.
2. **Image Layer Decomposition (ILD)**: 이미지를 레이어 단위로 분해하여 타겟 영역만 국소적으로 편집하고 다시 합성함으로써, 비타겟 영역의 픽셀을 완벽하게 보존하고 네이티브 4K 해상도 편집을 가능하게 한다.
3. **HDD-Bench 구축**: 전문적인 편집 워크플로우를 모사하기 위해 네이티브 4K 이미지와 검증 가능한 단계별 타겟을 포함한 고해상도 대화형 벤치마크를 제안한다.

## 📎 Related Works

기존의 명령어 기반 이미지 편집 연구들은 주로 Diffusion 모델이나 Autoregressive Transformer를 활용하여 사진의 사실성과 명령어 수행 능력을 높이는 데 집중해 왔다. 그러나 이러한 방식들은 주로 단일 턴 편집에 치중되어 있으며, 고해상도 이미지의 국소적 제어 능력이 부족하다는 한계가 있다.

최근에는 LLM의 추론 능력을 활용한 에이전트 시스템(예: ReAct 프레임워크)이나 외부 도구 호출을 위한 MCP(Model Context Protocol) 등이 등장하며 복잡한 작업 수행 능력이 향상되었다. 하지만 이미지 편집 분야에서 이러한 에이전틱 접근 방식이 4K 이상의 초고해상도 유지와 멀티턴 일관성이라는 전문적 요구사항과 결합된 사례는 부족했다. Agent Banana는 단순한 생성 모델을 넘어, 추론-계획-도구 사용-검증으로 이어지는 폐쇄 루프(Closed-loop) 시스템을 구축함으로써 기존 접근 방식과 차별화된다.

## 🛠️ Methodology

### 전체 시스템 구조

Agent Banana는 글로벌 작업 추론을 담당하는 **Planner**와 저수준 실행을 담당하는 **Executor**로 구성된 계층적 구조를 가진다.

- **Planner**: 사용자의 모호한 요청을 해석하여 실행 가능한 원자적 하위 목표(Atomic sub-goals)로 분해하고, 전체 진행 상황을 모니터링하며, 필요 시 이전 상태로 롤백(Rollback)하거나 계획을 수정한다.
- **Executor**: 분해된 목표를 수행하기 위해 MCP 서버를 통해 적절한 도구를 선택하고, ILD 메커니즘을 사용하여 국소 영역을 편집하며, 결과물의 품질을 검증(Quality Test)한다.

### Context Folding (컨텍스트 폴딩)

긴 상호작용 이력으로 인한 토큰 효율성 저하와 노이즈 문제를 해결하기 위해, 이력을 세 가지 수준의 스키마로 압축한다.

1. **Asset Level (ImageContext)**: 이미지를 고차원 토큰 대신 URI, VLM 생성 텍스트 설명, 부모 URI, 변환 유형을 포함하는 가벼운 세만틱 노드로 추상화하여 이미지 진화 그래프를 관리한다.
2. **Execution Level (ToolContext)**: 도구 선택, 파라미터 설정, 추론 과정(Thought) 등 단일 단계의 세부 실행 내역을 담는 일시적 작업 메모리이다. 단계 완료 후에는 '폴딩'되어 글로벌 메모리에 영향을 주지 않는다.
3. **Planning Level (ActionContext)**: 검증된 최종 의도와 핵심 ImageContext 시퀀스만을 저장하는 영구 메모리이다. 이를 통해 수십 번의 턴이 지나도 전체 작업 상태를 명확하게 파악할 수 있다.

### Image Layer Decomposition (ILD)

전체 이미지를 다시 생성할 때 발생하는 픽셀 드리프트와 해상도 저하를 막기 위해 **"분해-편집-합성(decompose-edit-fuse)"** 패러다임을 사용한다.

- **절차**: 객체 인식 마스크를 사용하여 타겟 영역을 정밀하게 식별하고, 이를 원본 고해상도 이미지에서 손실 없이 크롭(Crop)하여 독립적인 레이어 패치로 만든다. 편집은 이 패치의 국소 좌표계 내에서만 수행되며, 완료 후 가우시안 블렌딩(Gaussian blending) 알고리즘을 통해 원본 이미지에 다시 합성한다.
- **효과**: 비편집 영역의 픽셀 상태를 동결(Freeze)함으로써 배경 보존 능력을 극대화하며, 모델의 네이티브 해상도 제한을 넘어 4K 이상의 초고해상도 편집을 지원한다.

### Action Space 및 동작 원리

시스템은 다음과 같은 5가지 원자적 명령어 집합을 사용하여 복잡한 편집을 수행한다.

- $\text{replace}$: 인페인팅 기술을 사용하여 타겟 레이어를 새 객체로 교체한다.
- $\text{remove}$: 타겟 레이어를 제거하고 배경 완성 알고리즘으로 빈 공간을 채운다.
- $\text{add}$: 지정된 위치에 새 레이어를 생성하고 중첩시킨다.
- $\text{adjust}$: 기하학적 구조 변경 없이 색상이나 스타일 등의 속성을 변환한다.
- $\text{undo}$: Context Folding에서 유지되는 상태 그래프를 기반으로 이전 이미지 상태로 즉시 되돌린다.

환경의 동역학은 다음과 같은 전이 연산자 $E$로 추상화할 수 있다.
$$o_i = E(o_{i-1}, a_i)$$
여기서 $o_i$는 $i$번째 이미지 상태이며, $a_i$는 에이전트가 선택한 행동(추론 및 도구 호출)이다.

## 📊 Results

### 실험 설정

- **데이터셋**: 자체 구축한 **HDD-Bench** (4K 해상도, 3-턴 세션으로 구성된 96개 큐레이션 세션).
- **비교 대상**: Flux.1 Kontext, Nano Banana Pro, GPT-Image-1 등 상용 및 오픈소스 모델.
- **평가 지표**:
  - **Instruction Following (IF)**: 타겟 객체의 속성이 요청대로 변경되었는지 측정.
  - **Image Consistency (IC)**: 비타겟 객체가 변경 없이 유지되었는지 측정.
  - **Otsu-Masked Metrics**: Otsu 임계값 기반으로 배경 마스크 $M_{bg}$를 생성하여, 배경 영역에 대해서만 $\text{PSNR}_{OM}, \text{SSIM}_{OM}, \text{LPIPS}_{OM}$을 계산한다.

### 주요 결과

정량적 분석 결과, Agent Banana는 멀티턴 일관성과 배경 충실도에서 SOTA 성능을 달성하였다.

- **배경 보존**: $\text{IC}$ 0.871, $\text{SSIM}_{OM}$ 0.84, $\text{LPIPS}_{OM}$ 0.12를 기록하며 타 모델 대비 압도적인 배경 유지 능력을 보였다.
- **해상도**: 분석 결과, 대다수의 베이스라인 모델이 1K로 다운샘플링 후 업샘플링하는 과정에서 텍스처 손실이 발생하는 반면, Agent Banana는 네이티브 4K 해상도를 완벽하게 보존하였다.
- **단일 턴 성능**: ImgEdit-Bench 평가에서도 타 모델과 대등하거나 우월한 정밀도를 보여, 에이전트 구조가 단순 편집 성능을 저해하지 않음을 입증하였다.

## 🧠 Insights & Discussion

### 강점 및 분석

본 연구의 가장 큰 강점은 **에이전틱 추론(Agentic Thinking)**과 **레이어 기반 실행(Layer-aware Execution)**을 결합하여 전문 작업 수준의 제어력을 확보한 점이다. 특히, 단순한 이미지 생성 모델에 의존하지 않고 '계획 $\rightarrow$ 도구 사용 $\rightarrow$ 검증'의 루프를 구축함으로써 복잡한 명령어를 정확하게 수행할 수 있었다.

### Prior-Induced Editing Drift (PIED) 발견

저자들은 멀티턴 편집에서 발생하는 **Prior-Induced Editing Drift (PIED)**라는 현상을 지적한다. 이는 각 턴의 결과물이 개별적으로는 사실적으로 보일지라도, 비편집 영역이 반복적으로 재생성되면서 점차 생성 모델이 선호하는 통계적 텍스처와 스타일로 변해가는 현상을 말한다.
흥미로운 점은 PIED가 발생할 때 $\text{PSNR}_{OM}$이 오히려 상승하는 경향이 있어, 기존 지표가 장기적인 충실도 저하를 제대로 포착하지 못할 수 있다는 점을 시사한다. Agent Banana는 ILD를 통해 픽셀을 동결함으로써 이러한 드리프트 현상을 효과적으로 억제하였다.

### 한계 및 가정

시스템의 성능이 기반 LLM(GPT-5-mini 등)의 추론 능력에 강하게 의존한다. Ablation Study를 통해 더 작은 모델(Qwen-3-8B)을 사용했을 때, 모호한 명령어 처리 및 장기 계획 수립 능력이 급격히 떨어지며 툴 파라미터 생성 오류가 빈번하게 발생하는 것을 확인하였다. 이는 고성능 에이전트 시스템을 위해서는 강력한 기본 추론 능력이 필수적임을 의미한다.

## 📌 TL;DR

Agent Banana는 **Context Folding**을 통한 효율적인 메모리 관리와 **Image Layer Decomposition**을 통한 국소적 고해상도 편집을 결합한 에이전틱 이미지 편집 프레임워크이다. 4K 네이티브 해상도에서 배경 드리프트 없이 정밀한 멀티턴 편집을 수행하며, 특히 반복적인 재생성으로 인한 이미지 변질(PIED) 문제를 해결하였다. 이 연구는 단순한 이미지 생성을 넘어 전문적인 VFX 및 그래픽 디자인 워크플로우에 통합 가능한 수준의 고충실도 편집 도구의 가능성을 제시한다.
