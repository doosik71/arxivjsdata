# Evaluating Gemini Robotics Policies in a Veo World Simulator

Gemini Robotics Team, Google DeepMind (2026)

## 🧩 Problem to Solve

범용 로봇 정책(Generalist Robot Policies)은 자연어 지시를 통해 다양한 환경에서 여러 작업을 수행할 수 있어야 하므로, 그 신뢰성과 일반화 성능, 그리고 안전성을 평가하는 것이 매우 까다롭다. 특히 다음과 같은 이유로 기존의 평가 방식은 한계가 있다.

첫째, 모든 명목적(Nominal) 시나리오와 엣지 케이스(Edge-case)를 하드웨어로 직접 테스트하는 것은 시간과 비용 측면에서 비효율적이다. 둘째, 안전성 평가의 경우, 물리적 환경에서 위험한 시나리오(예: 깨진 유리, 가열된 스토브)를 구축하는 것은 로봇과 인간 모두에게 실질적인 위험을 초래할 수 있어 불가능에 가깝다. 셋째, 전통적인 물리 기반 시뮬레이터는 비강체(Non-rigid) 물체의 정밀한 시뮬레이션이 어렵고, 시뮬레이션과 실제 환경 사이의 시각적 격차(Visual Gap)를 줄이기 위해 막대한 인적 자원과 시간이 소요된다.

따라서 본 논문의 목표는 최첨단 비디오 생성 모델인 Veo를 활용하여, 명목적 성능 평가부터 분포 외(Out-of-Distribution, OOD) 일반화 성능 측정, 그리고 물리적·의미적 안전성을 검증하는 '레드팀(Red Teaming)' 작업까지 수행할 수 있는 생성형 평가 시스템을 구축하는 것이다.

## ✨ Key Contributions

본 연구의 핵심 아이디어는 비디오 생성 모델을 '세계 모델(World Model)'로 활용하여 로봇 정책의 실행 결과(Rollout)를 시뮬레이션하는 것이다. 단순히 비디오를 생성하는 것을 넘어, 로봇의 액션(Action)을 조건으로 입력받아 미래의 시각적 상태를 예측하게 함으로써, 실제 하드웨어 없이도 정책의 성패를 예측할 수 있는 환경을 구축하였다. 특히, 생성형 이미지 편집 기술을 결합하여 현실에서 구축하기 어려운 다양한 OOD 시나리오를 합성하고, 이를 통해 정책의 취약점을 미리 발견하는 예측적 레드팀(Predictive Red Teaming) 가능성을 제시하였다.

## 📎 Related Works

기존의 오프라인 평가 방식은 주로 물리 시뮬레이션에 의존해 왔다. 하지만 물리 시뮬레이션은 액체나 변형 가능한 물체와 같은 복잡한 역학을 구현하기 어렵고, 환경 구축 비용이 높다는 한계가 있다. 최근에는 데이터 기반의 비디오 생성 모델을 세계 모델로 사용하여 정책을 평가하려는 시도가 있었으나, 대부분 훈련 데이터 내의 분포(In-distribution) 평가에 국한되었거나, 액션 조건화(Action Conditioning)의 정밀도가 낮아 폐루프(Closed-loop) 시뮬레이션에 어려움이 있었다.

본 연구는 Veo라는 강력한 비디오 파운데이션 모델을 기반으로 하여, 정밀한 액션 조건화와 다중 뷰 일관성(Multi-view consistency)을 확보함으로써 기존의 단순 비디오 생성 방식보다 훨씬 높은 수준의 시뮬레이션 충실도(Fidelity)를 달성하였다. 또한, 단순한 성공률 예측을 넘어 이미지 편집을 통한 OOD 시나리오 확장과 안전성 검증으로 평가 범위를 넓혔다는 점에서 차별성을 가진다.

## 🛠️ Methodology

### 1. 시스템 구조 및 모델 아키텍처

본 시스템은 Google의 **Veo2** 텍스트-비디오 모델을 기반으로 한다. Veo2는 Latent Diffusion 아키텍처를 사용하며, 오토인코더를 통해 시공간 데이터를 효율적인 잠재 표현(Latent representation)으로 압축한 뒤, 트랜스포머 기반의 디노이징 네트워크(Denoising network)를 통해 비디오를 생성한다.

### 2. 로봇 특화 미세 조정 (Action Conditioning)

사전 학습된 Veo2를 대규모 로봇 데이터셋으로 미세 조정하여, 현재의 이미지 관측값과 미래의 로봇 포즈(Robot poses) 시퀀스를 조건으로 입력받아 그에 대응하는 미래 이미지 시퀀스를 예측하도록 만들었다. 이를 통해 로봇의 정밀한 제어 명령에 반응하는 비디오 생성이 가능해졌다.

### 3. 다중 뷰 생성 (Multi-View Generation)

로봇 정책의 부분 관측 문제를 해결하기 위해, 탑다운 뷰, 사이드 뷰, 좌/우 손목 뷰의 4개 카메라 영상을 하나의 프레임으로 타일링(Tiling)하여 생성하도록 학습시켰다. 이는 정책이 다중 뷰 입력을 사용하는 현대적인 로봇 아키텍처와 호환되기 위함이다.

### 4. OOD 시나리오 합성 파이프라인

분포 외(OOD) 평가를 위해 다음과 같은 절차를 거친다.

1. **이미지 편집**: Gemini 2.5 Flash Image(NanoBanana)를 사용하여 명목적 장면의 오버헤드 뷰 이미지에서 특정 요소(배경, 물체 등)를 변경한다.
2. **다중 뷰 합성**: 편집된 단일 뷰 이미지를 입력으로 하여, 미세 조정된 Veo2 모델이 나머지 3개 뷰의 이미지를 합성하여 일관된 다중 뷰 관측값을 생성한다.
3. **정책 롤아웃**: 합성된 관측값과 언어 지시문을 로봇 정책에 입력하고, 그 결과로 나오는 액션 시퀀스를 다시 Veo (Robotics) 모델에 입력하여 최종 비디오 롤아웃을 생성한다.

### 5. 평가 지표

정책 간의 상대적 순위 일관성을 측정하기 위해 **MMRV(Mean Maximum Rank Violation)** 지표를 사용한다. $n$개의 정책 $\pi_1, \dots, \pi_n$에 대해, 실제 성공률 $R_{real}$과 예측 성공률 $R_{pred}$ 사이의 순위 위반 정도를 다음과 같이 정의한다.

$$MMRV := \frac{1}{n} \sum_{i=1}^{n} \max_{1 \le j \le n} \text{RankViolation}(i, j)$$

여기서 $\text{RankViolation}(i, j)$는 다음과 같다.

$$\text{RankViolation}(i, j) := |R_{real}^i - R_{real}^j| \cdot \mathbb{1}[(R_{pred}^i < R_{pred}^j) \neq (R_{real}^i < R_{real}^j)]$$

또한, 예측값과 실제값 사이의 선형 상관관계를 측정하기 위해 **Pearson 상관계수**를 사용한다.

## 📊 Results

### 1. 명목적 시나리오 평가 (Nominal Scenarios)

80가지의 장면-지시문 조합에 대해 8종의 정책 체크포인트를 평가한 결과, Veo (Robotics) 모델의 예측 결과와 실제 하드웨어 결과 사이에 매우 높은 상관관계가 나타났다.

- **MMRV**: 0.03 (매우 낮은 순위 위반)
- **Pearson 상관계수**: 0.88
이는 비디오 모델이 실제 로봇의 성능 순위를 정확하게 예측할 수 있음을 시사한다.

### 2. OOD 일반화 평가 (OOD Generalization)

배경(Background), 작은 방해물(Small Distractor), 큰 방해물(Large Distractor), 조작 대상 물체(Object)의 4가지 축을 기준으로 평가를 진행하였다.

- **상관관계**: 배경($\rho=0.91$, $\text{MMRV}=0.0$)과 방해물 축에서 매우 높은 예측 성능을 보였으며, 물체 변경 축에서는 상대적으로 낮은 상관관계를 보였으나 이는 모든 정책의 성공률 자체가 낮았기 때문으로 분석된다.
- **정성적 분석**: 물체 변경 시 정책이 낯선 물체 대신 익숙한 물체로 손을 뻗는 등의 실패 모드를 비디오 시뮬레이션을 통해 시각적으로 확인하였으며, 이는 데이터 수집 방향을 설정하는 데 유용한 인사이트를 제공한다.

### 3. 안전성 레드팀 (Safety Red Teaming)

위험 요소(날카로운 칼, 뜨거운 물체 등)가 포함된 가상 시나리오를 생성하여 정책을 테스트하였다.

- **발견된 위험 행동**: "빨간 블록을 빨리 잡아라"라는 지시 시 인간의 손과 충돌하거나, "노트북을 닫아라"라는 지시 시 노트북 위에 놓인 가위를 치우지 않고 닫아 화면을 파손시키는 행동 등이 발견되었다.
- **검증**: 이러한 비디오 모델의 예측 결과는 실제 하드웨어에서 동일한 소품으로 재현했을 때 실제로 발생함이 확인되었다.

## 🧠 Insights & Discussion

본 연구는 비디오 생성 모델이 로봇 정책 평가를 위한 강력한 가상 환경(Proxy)이 될 수 있음을 입증하였다. 특히 하드웨어로 테스트하기 위험하거나 비용이 많이 드는 OOD 및 안전성 평가를 비디오 시뮬레이션으로 대체할 수 있다는 점은 매우 큰 강점이다.

그러나 몇 가지 한계점과 향후 과제가 존재한다.
첫째, 작은 물체와의 정밀한 접촉(Contact-rich interaction) 시 물체가 갑자기 나타나거나 사라지는 등의 환각(Hallucination) 현상이 여전히 발생한다. 이는 더 많은 상호작용 데이터의 학습을 통해 해결해야 할 문제이다.
둘째, 현재 시뮬레이션 가능 시간은 8초 내외로 짧다. 1분 이상의 장기 horizon(Long-horizon) 시뮬레이션을 위해서는 잠재 액션 모델(Latent-action models) 등의 도입이 필요하다.
셋째, 현재는 생성된 비디오의 성패 판단을 사람이 직접 수행하고 있으나, 이를 VLM(Vision-Language Model) 기반의 자동 채점 시스템으로 전환하여 평가 파이프라인을 완전히 자동화할 필요가 있다.

## 📌 TL;DR

본 논문은 최첨단 비디오 생성 모델인 Veo를 기반으로 로봇 정책을 평가하는 생성형 세계 모델 시뮬레이터를 제안한다. 이 시스템은 실제 하드웨어 결과와 높은 상관관계를 보이는 성능 예측뿐만 아니라, 이미지 편집을 통한 OOD 시나리오 확장 및 위험 행동을 미리 찾아내는 예측적 레드팀(Predictive Red Teaming)을 가능하게 한다. 이는 향후 안전하고 범용적인 로봇 에이전트를 개발하기 위한 확장 가능한 평가 인프라로서 중요한 역할을 할 것으로 기대된다.
