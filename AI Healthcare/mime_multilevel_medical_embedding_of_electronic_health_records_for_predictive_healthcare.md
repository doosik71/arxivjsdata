# MiME: Multilevel Medical Embedding of Electronic Health Records for Predictive Healthcare

Edward Choi, Cao Xiao, Walter F. Stewart, Jimeng Sun (2018)

## 🧩 Problem to Solve

전자 건강 기록(Electronic Health Records, EHR)을 활용한 딥러닝 모델들은 다양한 예측 의료 작업에서 최첨단 성능을 보여주고 있다. 그러나 이러한 모델들이 최적의 성능을 내기 위해서는 대부분의 의료 시스템이 보유한 수준을 훨씬 상회하는 방대한 양의 학습 데이터가 필요하다는 한계가 있다.

데이터 부족 문제를 해결하기 위해 의료 온톨로지(Medical Ontologies)와 같은 외부 자원을 활용하는 방법이 제안되었으나, 실제 의료 현장에서 사용하는 용어가 표준과 일치하지 않거나 병원마다 고유한 용어 체계를 사용하는 경우가 많아 외부 온톨로지를 직접 적용하는 데 어려움이 있다. 따라서 본 논문은 외부 레이블이나 온톨로지에 의존하지 않고, EHR 데이터가 본질적으로 가지고 있는 다층적 구조(Multilevel Structure)를 활용하여 데이터 부족 문제를 해결하고 예측 성능을 높이는 것을 목표로 한다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 EHR 데이터의 계층적 구조, 특히 진단 코드(Diagnosis codes)와 처치 코드(Treatment codes, 약물 및 처치) 사이의 관계를 명시적으로 모델링하는 것이다. 이를 위해 다음과 같은 두 가지 핵심 설계를 제안한다.

1. **Multilevel Medical Embedding (MiME):** EHR의 구조를 '처치 수준 $\rightarrow$ 진단 수준 $\rightarrow$ 방문 수준 $\rightarrow$ 환자 수준'으로 이어지는 하향식(Bottom-up) 임베딩 방식으로 변환하여, 진단과 처치 사이의 상호작용을 정교하게 캡처한다.
2. **Auxiliary Prediction Tasks:** 외부 레이블 없이도 EHR 내부 구조를 활용해 진단 코드와 처치 코드를 예측하는 보조 작업을 함께 학습함으로써, 모델이 일반적인 의료 지식을 학습하도록 유도하고 주 작업(Main task)의 예측력을 보완한다.

## 📎 Related Works

기존의 의료 개념 임베딩 연구들은 주로 희소하고 고차원인 의료 개념을 저차원 벡터로 압축하거나, RNN 등을 이용해 시간적 순서로 표현하는 방식에 집중하였다. 일부 연구에서는 의료 온톨로지를 활용해 임베딩을 학습시키기도 하였으나(예: GRAM), 이는 앞서 언급한 용어 불일치 문제로 인해 실제 적용에 제약이 많다.

최근에는 EHR 내의 다양한 코드 유형을 동시에 다루려는 시도가 있었으나, 단순히 코드들을 집합(Set) 형태로 평탄화(Flatten)하여 처리하는 경우가 많았다. 이는 진단과 처치 사이의 구체적인 계층 관계를 무시하는 접근 방식이며, 본 논문의 MiME는 이러한 평탄화 방식의 한계를 극복하고 진단-처치 간의 상호작용을 명시적으로 모델링한다는 점에서 차별점을 가진다.

## 🛠️ Methodology

MiME는 EHR의 계층 구조를 반영하여 하위 수준에서 상위 수준으로 임베딩을 생성한다.

### 1. Multilevel Embedding Process

**Treatment Level (처치 수준):**
특정 진단 코드 $d_i$와 그에 연관된 처치 코드 $m_{i,j}$ 사이의 상호작용을 캡처한다. 이때 저차원 Bilinear Pooling 기법을 사용하여 다음과 같이 계산한다.
$$g(d_i, m_{i,j}) = \sigma(W_m r(d_i)) \odot r(m_{i,j})$$
여기서 $r(\cdot)$은 각 코드의 임베딩 벡터이며, $\odot$은 요소별 곱셈(Element-wise multiplication)을 의미한다. $W_m$은 진단 임베딩을 상호작용 캡처에 적합한 잠재 공간으로 투영하는 가중치 행렬이다.

**Diagnosis Level (진단 수준):**
진단 코드 $d_i$와 관련된 모든 처치 코드들의 상호작용 합을 더해 진단 객체 임베딩 $o_i$를 생성한다.
$$o_i = \sigma(W_o (r(d_i) + \sum_{j} g(d_i, m_{i,j}))) + G$$
여기서 $G$는 기울기 소실 문제를 방지하기 위한 Skip-connection이다. 이는 진단 표현을 연관된 처치 정보에 따라 조정하는 과정으로 해석될 수 있다.

**Visit Level (방문 수준):**
한 번의 방문(Visit) 내에 존재하는 여러 진단 객체 임베딩 $o_i$들을 합산하여 방문 임베딩 $v$를 생성한다.
$$v = \sigma(W_v (\sum_{i} f(d_i, M_i))) + F$$
여기서 $F$ 역시 Skip-connection이며, $f(d_i, M_i)$는 위에서 계산한 $o_i$와 동일하다. 최종적으로 이 $v$는 RNN 등의 모델에 입력되어 환자 수준의 임베딩 $h$로 변환된다.

### 2. Joint Training with Auxiliary Tasks

모델의 강건함을 높이기 위해, 학습 과정에서 진단 객체 임베딩 $o_i$를 이용해 실제 진단 코드와 처치 코드를 예측하는 보조 작업을 수행한다.

- **진단 코드 예측:** $\hat{d}_i = \text{softmax}(U_d o_i)$
- **처치 코드 예측:** $\hat{m}_{i,j} = \sigma(U_m o_i)$

최종 손실 함수는 주 작업의 손실과 보조 작업의 교차 엔트로피(Cross-Entropy) 손실의 합으로 구성된다.
$$L_{aux} = -\lambda_{aux} \sum_{t} (\sum_{i} (CE(d_i, \hat{d}_i) + \sum_{j} CE(m_{i,j}, \hat{m}_{i,j})))$$
이 보조 작업은 모델이 특정 타겟 작업에 과적합되지 않고, EHR 데이터에 내재된 일반적인 의료 지식을 학습하도록 가이드하는 역할을 한다.

## 📊 Results

### 1. 실험 설정

- **데이터셋:** Sutter Health의 EHR 데이터 (시니어 환자 30,764명).
- **예측 작업:** 심부전(Heart Failure, HF) 발생 예측 및 순차적 질병 예측(SDP).
- **평가 지표:** 불균형 데이터 특성을 고려하여 PR-AUC(Precision-Recall AUC)를 주요 지표로 사용하였다.
- **비교 대상:** Raw binary, Linear, Non-linear(tanh, relu), MLP, Med2Vec, GRAM.

### 2. 주요 결과

- **데이터 크기에 따른 성능:** 데이터 규모를 네 단계($E_1 \sim E_4$)로 나누어 실험한 결과, 모든 설정에서 MiME가 가장 우수한 성능을 보였다. 특히 데이터 규모가 가장 작은 $E_1$에서 최선의 baseline 대비 PR-AUC 기준 약 15%의 상대적 성능 향상을 보여, 데이터 부족 문제 해결에 매우 효과적임을 입증하였다.
- **방문 복잡도에 따른 성능:** 진단-처치 상호작용이 많은 '복잡한 방문'의 비중이 높을수록($D_1 \rightarrow D_3$) MiME의 성능 향상 폭이 컸다. 이는 MiME가 실제로 진단-처치 간의 관계를 효과적으로 활용하고 있음을 의미한다.
- **순차적 질병 예측 (SDP):** 미래의 진단 코드를 예측하는 작업에서도 MiME가 모든 지표에서 baseline을 앞섰으며, 특히 희귀 질환 예측에서 baseline 대비 최대 11.6%의 성능 향상을 보였다.

## 🧠 Insights & Discussion

본 논문은 EHR의 내재적 계층 구조를 모델링하는 것이 외부 온톨로지를 도입하는 것보다 더 실질적인 성능 향상을 가져올 수 있음을 보여주었다. 특히 PR-AUC와 ROC-AUC의 차이를 통해, MiME가 단순히 음성 샘플을 잘 맞추는 것(Specificity)을 넘어, 진단-처치의 세부 상호작용을 파악함으로써 양성 샘플을 정확히 찾아내는 능력(Precision/Recall)을 크게 향상시켰음을 분석하였다.

또한, 실험 과정에서 소규모 데이터셋의 경우 ReLU보다 tanh나 sigmoid와 같이 출력이 제한된(Bounded) 활성화 함수가 정규화 효과를 주어 더 나은 성능을 보였다는 점은 주목할 만한 관찰 결과이다. 이는 데이터 규모에 따라 모델의 하이퍼파라미터 및 활성화 함수 선택이 달라져야 함을 시사한다.

한계점으로는, 본 연구가 주로 진단-처치 관계에 집중했다는 점이다. 향후 연구에서는 처치 결과, 인구통계학적 정보, 투약 지침 등 더 세밀한 의료 이벤트들을 모델에 통합함으로써 표현력을 더욱 높일 수 있을 것이다.

## 📌 TL;DR

MiME는 EHR 데이터의 '진단 $\rightarrow$ 처치' 계층 구조를 반영한 다층 임베딩 기법과 자가 지도 학습 방식의 보조 예측 작업을 통해, 데이터가 부족한 의료 환경에서도 강건한 예측 성능을 내는 모델이다. 특히 데이터 규모가 작고 복잡도가 높을수록 기존의 평탄화(Flattening) 방식이나 외부 온톨로지 기반 방식보다 월등한 성능을 보이며, 이는 실제 의료 AI 시스템의 데이터 부족 문제를 해결하는 실용적인 방향성을 제시한다.
