# Advances in Privacy Preserving Federated Learning to Realize a Truly Learning Healthcare System

Ravi Madduri, Zilinghan Li, Tarak Nandi, Kibaek Kim, Minseok Ryu, Alex Rodriguez (2024)

## 🧩 Problem to Solve

본 논문은 의료 데이터의 특성상 발생하는 데이터 사일로(Data Silos) 문제와 엄격한 개인정보 보호 규정(HIPAA, GDPR 등)으로 인해 진정한 의미의 학습형 헬스케어 시스템(Learning Healthcare System, LHS)을 구축하는 데 따르는 어려움을 해결하고자 한다.

현대 의료 데이터는 전자 건강 기록(EHR), 유전체 정보, 의료 영상 등 매우 다양한 모달리티(Multimodal)로 구성되어 있으며, 이는 서로 다른 기관이나 지역에 분산 저장되어 있다. 이러한 데이터를 중앙 집중식으로 수집하여 AI 모델을 학습시키는 기존 방식은 인프라 비용이 막대할 뿐만 아니라, 법적·윤리적 제약으로 인해 실현 불가능한 경우가 많다. 또한, 단일 기관의 제한된 데이터로만 학습된 모델은 실제 환경에서 데이터 분포가 달라질 때 성능이 급격히 떨어지는 모델 시프트(Model Shift) 현상이 발생한다. 따라서 데이터 프라이버시를 보장하면서도 분산된 다기관의 멀티모달 데이터를 활용해 일반화 성능이 높은 모델을 학습시킬 수 있는 체계적인 프레임워크가 필요하다.

## ✨ Key Contributions

본 논문의 핵심 기여는 프라이버시 보존 연합 학습(Privacy-Preserving Federated Learning, PPFL)을 헬스케어 생태계에 통합하여 진정한 LHS를 구현하기 위한 포괄적인 비전과 프레임워크를 제시한 것이다.

단순한 연합 학습을 넘어, 의료 현장의 실질적인 제약 사항을 해결하기 위해 다음과 같은 설계 아이디어를 제안한다:
1. **멀티모달 연합 학습**: 다양한 의료 데이터 소스를 통합하여 진단 및 예후 예측의 정확도를 높이는 방법론을 제시한다.
2. **계층적 연합 학습(Hierarchical FL)**: 신뢰 경계(Trust Boundary)의 문제를 해결하기 위해 중간 서버를 도입한 계층적 구조를 제안한다.
3. **연합 지속 학습(Federated Continuous Learning)**: 실시간 데이터 변화와 의료 트렌드에 대응하기 위해 평가-감지-재학습으로 이어지는 피드백 루프를 설계한다.
4. **비용 인식 연합 학습(Cost-aware FL)**: 클라우드 환경에서 Spot Instance와 스케줄러를 활용해 학습 비용을 최적화하는 방안을 제시한다.

## 📎 Related Works

논문에서는 기존의 연합 학습(Federated Learning, FL)이 데이터 직접 공유 없이 전역 모델을 생성할 수 있다는 점을 언급하지만, FL 자체만으로는 모델 가중치나 그래디언트 통신 과정에서 로컬 데이터를 역추적할 수 있는 프라이버시 취약점이 존재함을 지적한다. 이를 보완하기 위해 Differential Privacy(DP)와 같은 프라이버시 강화 기술(PET)의 필요성을 강조한다.

또한, 저자들이 이전에 개발한 다음과 같은 도구들을 소개하며 기존 접근 방식의 한계를 극복하려 한다:
- **APPFL**: 분산된 이질적 컴퓨팅 자원 환경에서 프라이버시 보존 연합 학습을 가능하게 하는 오픈소스 프레임워크이다.
- **APPFLx**: 전문 지식이 부족한 도메인 전문가도 웹 인터페이스를 통해 PPFL 실험을 관리하고 배포할 수 있도록 서비스 형태로 제공하는 플랫폼이다.
- **AIDRIN**: AI 모델 학습 전, 데이터의 품질과 준비 상태를 정량적으로 평가하여 불필요한 컴퓨팅 자원 낭비를 막는 툴킷이다.

## 🛠️ Methodology

본 논문에서 제안하는 PPFL 프레임워크는 다음과 같은 주요 구성 요소로 이루어져 있다.

### 1. 멀티모달 학습 및 연합 학습 통합
의료 데이터의 복잡성을 해결하기 위해 각 모달리티(영상, 텍스트, 수치 데이터 등)를 공통된 특징 공간(Shared Feature Space)으로 변환하는 임베딩(Embedding) 과정을 거친다. 이때 데이터 융합(Data Fusion) 방식은 세 가지로 구분된다:
- **Early Fusion**: 각 모달리티의 임베딩을 먼저 생성한 후 결합하여 예측 모델의 입력으로 사용한다.
- **Joint Fusion**: 학습 과정에서 임베딩 생성 모델과 예측 모델을 동시에 업데이트하여 상호작용을 학습한다.
- **Late Fusion**: 각 모달리티별 단일 모델의 예측 결과들을 최종적으로 결합한다.

연합 학습 환경에서의 데이터 분포에 따라 세 가지 시나리오를 정의한다:
- **시나리오 (a)**: 모든 클라이언트가 동일한 특징 공간을 공유하는 경우 $\rightarrow$ 전통적인 Horizontal FL 적용.
- **시나리오 (b)**: 샘플 공간은 동일하지만 모달리티가 다른 경우 $\rightarrow$ Vertical FL(VFL) 적용.
- **시나리오 (c)**: 샘플과 특징 공간이 모두 다른 경우 $\rightarrow$ 중간/후기 융합(Intermediate/Late Fusion)이나 생성 모델을 이용한 합성 데이터 활용 방안을 탐색한다.

### 2. 계층적 연합 학습 (Hierarchical FL, HierFL)
중앙 서버를 완전히 신뢰하지 못하는 소규모 클리닉들을 위해, 신뢰할 수 있는 지역 거점 병원을 중간 서버(Intermediate Server)로 두는 구조를 제안한다. 
- **절차**: 소규모 클리닉 $\rightarrow$ 중간 서버(1차 집계) $\rightarrow$ 루트 서버(최종 전역 집계) $\rightarrow$ 다시 클리닉으로 배포.
- 이를 통해 개별 클리닉의 정보가 직접 루트 서버로 전달되는 것을 방지하고, 더 넓은 범위의 데이터 다양성을 확보하여 모델의 일반화 성능을 높인다.

### 3. 연합 지속 학습 및 모니터링
모델의 성능 저하(Performance Degradation)를 실시간으로 방지하기 위해 다음과 같은 워크플로우를 구축한다:
- **연합 평가(Federated Evaluation)**: 주기적으로 분산된 환경에서 Accuracy, Precision, Recall 등의 지표를 측정한다.
- **감지 및 재학습**: 성능 저하가 감지되면 자동으로 연합 지속 학습 프로세스를 트리거하여 새로운 데이터셋으로 모델을 업데이트한다.

### 4. 비용 인식 클라우드 스케줄링
클라우드 비용 절감을 위해 AWS, Azure 등의 Spot Instance(미사용 자원을 저렴하게 제공하는 인스턴스)를 활용한다.
- **Cost-aware Scheduler**: 실시간 학습 비용을 기록하고, 설정된 예산 내에서 자원을 관리하며, 인스턴스가 강제 종료될 경우 체크포인트를 통해 학습 상태를 복구하고 자원을 재할당한다.

## 📊 Results

본 논문은 특정 알고리즘의 수치적 성능을 측정하는 실험 논문이라기보다, 미래의 LHS 구현을 위한 **아키텍처 제안 및 비전 제시**에 집중하고 있다. 따라서 구체적인 벤치마크 결과나 정량적 수치는 명시되지 않았다.

다만, 저자들이 제안한 APPFL 및 APPFLx 프레임워크가 이미 단일 모달리티의 생물 의학 AI 모델 학습에 적용되어 그 유효성이 입증되었음을 언급하고 있으며, 이를 확장하여 멀티모달 및 계층적 구조로 나아가고자 하는 방향성을 제시하고 있다.

## 🧠 Insights & Discussion

본 논문은 의료 AI의 고질적인 문제인 '데이터 프라이버시'와 '데이터 다양성' 사이의 트레이드오프를 PPFL이라는 기술적 도구로 해결하려 한다는 점에서 강점이 있다. 특히, 단순한 알고리즘 제안에 그치지 않고 계층적 신뢰 구조(HierFL)나 클라우드 비용 최적화와 같은 실무적인 인프라 관점의 고민을 담고 있다는 점이 인상적이다.

하지만 몇 가지 한계점과 논의 사항이 존재한다:
- **프라이버시 예산(Privacy Budget) 설정**: DP를 적용할 때 노이즈 추가로 인한 성능 저하와 프라이버시 보호 수준 사이의 최적 지점을 찾는 구체적인 방법론이 아직 미비하며, 이는 향후 연구 과제로 남아 있다.
- **멀티모달 정렬 문제**: 시나리오 (c)와 같이 샘플과 특징 공간이 모두 다를 때, 고유 식별자 없이 어떻게 데이터를 정렬하고 학습시킬 것인가에 대한 구체적인 해결책은 여전히 탐색 단계에 있다.
- **통신 오버헤드**: 계층적 구조와 지속적인 평가 루프는 필연적으로 네트워크 통신 횟수를 증가시키며, 이에 따른 레이턴시 문제에 대한 분석이 필요하다.

## 📌 TL;DR

본 논문은 데이터 프라이버시 보호와 멀티모달 데이터 활용이라는 두 마리 토끼를 잡기 위해, **멀티모달 학습, 계층적 구조(HierFL), 지속적 모니터링 및 비용 최적화 스케줄러**가 통합된 **프라이버시 보존 연합 학습(PPFL) 프레임워크**를 제안한다. 이 연구는 파편화된 의료 데이터를 안전하게 통합하여, 실시간으로 진화하고 개인화된 진단이 가능한 **진정한 학습형 헬스케어 시스템(LHS)**을 구축하는 데 핵심적인 기반이 될 것으로 기대된다.
