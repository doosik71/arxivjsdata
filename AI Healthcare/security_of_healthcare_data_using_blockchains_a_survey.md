# Security of Healthcare Data Using Blockchains: A Survey

Mayank Pandey, Rachit Agarwal, Sandeep K. Shukla, Nishchal K. Verma (2020/2021)

## 🧩 Problem to Solve

현대 의료 산업은 사이버 물리 시스템(CPS), 빅데이터, 클라우드 컴퓨팅, 머신러닝 등이 통합된 'Health 4.0' 시대로 진입하였다. 이러한 기술적 진보로 인해 데이터 기반의 학습과 시스템 간 상호 연결성이 향상되어 효율성이 증대되었으나, 동시에 의료 데이터의 막대한 유입과 공유, 저장 과정에서 복잡성이 증가하고 새로운 보안 취약점이 발생하였다.

특히 의료 데이터는 개인의 민감한 진단 정보와 생체 정보를 포함하고 있어, 무단 접근 및 데이터 변조와 같은 사이버 공격에 매우 취약하다. 기존의 중앙 집중식 저장 방식은 단일 실패 지점(Single Point of Failure) 문제를 가지며, 데이터 소유권의 모호성으로 인해 프라이버시 보호 및 권한 관리에 어려움이 있다. 따라서 본 논문은 이러한 의료 데이터 보안 문제를 해결하기 위해 블록체인(Blockchain) 기술을 적용하는 방안을 탐색하고, 그 효용성과 한계점을 분석하는 것을 목표로 한다.

## ✨ Key Contributions

본 논문의 중심적인 기여는 의료 데이터 보안을 위한 블록체인 적용 가능성을 **4W1H (What, Why, Who, When, How)** 방법론을 통해 체계적으로 분석한 것에 있다. 주요 기여 사항은 다음과 같다.

- **체계적 서베이 수행**: 의료 데이터의 유형, 필요한 이유, 필요 대상, 적용 시점 및 구체적인 구현 방법(state-of-the-art 기술)을 분석하여 블록체인이 의료 보안에 어떻게 기여할 수 있는지 제시한다.
- **기술적 및 규제적 한계 식별**: 블록체인의 불변성(Immutability)과 개인의 '잊혀질 권리(Right to be forgotten)' 사이의 충돌, 확장성 문제, 표준화 부족 등의 실제적인 제약 사항을 논의한다.
- **팬데믹 사례 연구**: COVID-19 상황에서 접촉자 추적, 면역 증명서 발급, 협력 연구 등 대규모 의료 데이터 보안 관리에 블록체인이 어떻게 활용될 수 있는지 분석한다.
- **향후 연구 방향 제시**: 순수 탈중앙화 달성, 지속 가능한 인센티브 모델 설계, 리소스 효율적인 블록체인 구조 등 미래 연구를 위한 가이드라인을 제공한다.

## 📎 Related Works

논문에서는 의료 데이터 보안을 위해 블록체인을 활용한 기존의 다양한 연구들을 소개하며, 각 접근 방식의 특징과 한계를 분석한다.

- **환자 중심 데이터 교환**: 환자가 자신의 데이터 제어권을 갖도록 하는 프레임워크(Zhuang et al., 2020) 등이 제안되었으나, 각 의료 시설마다 개별적인 설정이 필요하다는 한계가 있다.
- **분산 저장 시스템 결합**: 데이터의 크기가 큰 의료 데이터의 특성을 고려하여 IPFS(InterPlanetary File System)와 같은 분산 저장소에 데이터를 저장하고 블록체인에는 해시(Hash) 값만을 저장하는 방식(Shahnaz et al., 2019)이 사용된다.
- **특수 목적 보안 시스템**: 응급 상황에서의 일시적 데이터 접근 제어(EACMS), 제약 공급망의 위조 방지, 건강보험 사기 탐지 등을 위한 특화된 솔루션들이 제안되었다.

기존 연구들의 공통적인 한계점은 대부분의 시스템이 소규모 노드 기반의 실험에 그치거나, 실제 환경에서의 실시간 성능 및 구축 비용에 대한 분석이 부족하다는 점이다. 또한, 데이터의 기밀성을 위해 추가적인 암호화 기법을 사용하지만, 이는 계산 복잡도를 증가시키는 트레이드-오프 관계에 있다.

## 🛠️ Methodology

본 논문은 특정 알고리즘을 제안하는 대신, 의료 데이터 보안을 위한 블록체인 아키텍처의 구성 요소와 작동 원리를 분석하는 방법론을 취한다.

### 1. 의료 데이터 분류 및 보안 요구사항

의료 데이터를 크게 세 가지 카테고리로 분류하고 각 데이터별 보안 파라미터를 정의한다.

- **개인 데이터**: 전자 건강 기록(EHR), 개인 건강 기록(PHR), 의료 전문가 정보.
- **제약 데이터**: 임상 시험 데이터, 제조 정보, 공급망 데이터.
- **보험 데이터**: 보험 정책, 결제 정보, 이용 서비스 데이터.
- **핵심 보안 파라미터**: 접근 제어(Access Control), 데이터 무결성(Data Integrity), 추적 가능성(Traceability), 사이버 공격 저항성.

### 2. 블록체인 구성 요소 및 메커니즘

의료 보안 구현을 위한 블록체인의 핵심 기술적 요소를 다음과 같이 설명한다.

- **네트워크 유형**:
  - Permissionless(공공): 누구나 참여 가능.
  - Permissioned(허가형): 중앙 권한자가 참여자와 역할을 제어. 의료 분야에는 보안과 효율성 때문에 이 방식이 더 적합하다고 분석한다.
- **합의 알고리즘(Consensus Protocols)**:
  - $PoW$(Proof of Work) 및 $PoS$(Proof of Stake): 일반적인 합의 방식.
  - $PBFT$(Practical Byzantine Fault Tolerance) 및 $Raft$: 허가형/컨소시엄 블록체인에서 주로 사용되며, 악의적인 노드가 존재해도 합의에 도달할 수 있게 한다.
- **데이터 보호 기술**:
  - **해싱(Hashing)**: $SHA-256$ 또는 $ethash$를 사용하여 데이터 블록을 순차적으로 연결함으로써 무결성을 보장한다.
  - **스마트 계약(Smart Contracts)**: 사전 정의된 조건에 따라 자동으로 실행되는 프로그램으로, 데이터 접근 권한 부여 및 역할 관리를 자동화한다.
  - **암호화**: 속성 기반 암호화(Attribute-Based Encryption), 영지식 증명(Zero-Knowledge Proof), 프록시 재암호화(Proxy Re-encryption) 등을 통해 프라이버시를 강화한다.

### 3. 하이브리드 저장 모델 (Off-chain Storage)

의료 데이터(X-ray, CT 스캔 등)는 용량이 매우 크기 때문에 모든 데이터를 블록체인에 저장하는 것은 불가능하다. 따라서 다음과 같은 절차를 제안한다.

1. 실제 데이터는 클라우드나 분산 저장소(IPFS 등)에 저장한다.
2. 해당 데이터의 해시 값, 메타데이터, 포인터만을 블록체인 원장에 기록한다.
3. 데이터 접근 시 블록체인의 해시 값과 실제 데이터의 해시 값을 비교하여 변조 여부를 확인한다.

## 📊 Results

본 논문은 개별 실험 결과보다는 기존 제안된 솔루션들을 분석한 종합적인 결과를 제시한다.

### 1. 정성적 분석 결과 (Table 1 기반)

다양한 최신 솔루션들을 분석한 결과, 다음과 같은 경향성이 확인되었다.

- **저장 방식**: 대부분의 최신 연구는 오직 해시와 트랜잭션만을 온체인(On-chain)에 저장하는 방식($\text{$\text{⸸}$}$)을 채택하고 있다.
- **네트워크 유형**: 프라이빗($Pr$) 및 컨소시엄($C$) 블록체인이 지배적이며, 이는 의료 데이터의 민감성 때문이다.
- **기능적 구현**: 접근 제어와 무결성 보장은 대부분 구현되었으나, 인센티브 체계나 응급 상황 대응 기능은 일부 연구에서만 다루어졌다.

### 2. COVID-19 사례 적용 결과

팬데믹 상황에서 블록체인의 실질적 효용성을 다음과 같이 분석하였다.

- **접촉자 추적**: BeepTrace와 같은 솔루션을 통해 익명성을 유지하면서 효율적인 추적이 가능함을 확인하였다.
- **면역 증명서**: 스마트 계약을 이용해 위변조가 불가능한 디지털 의료 여권을 발행함으로써 국경 이동 시의 보안성을 높일 수 있다.
- **협력 연구**: 가명 처리된 데이터를 블록체인 기반 플랫폼에서 공유함으로써, 프라이버시 침해 없이 전 세계 연구소 간의 신속한 백신 개발 협력이 가능함을 보였다.

## 🧠 Insights & Discussion

### 1. 강점 및 기회

블록체인은 중앙 집중식 관리자의 개입 없이도 데이터의 무결성과 추적 가능성을 보장할 수 있다. 특히 환자가 자신의 의료 데이터에 대한 완전한 제어권을 갖게 함으로써(Patient-centric), 데이터 소유권 문제를 해결하고 의료 서비스의 투명성을 높일 수 있다.

### 2. 기술적 및 규제적 한계

- **불변성 vs 잊혀질 권리**: 블록체인의 핵심 특징인 '한번 기록되면 삭제할 수 없다'는 점은 GDPR 등 현대 프라이버시 법령의 '데이터 삭제 요청권'과 정면으로 충돌한다.
- **확장성 및 비용**: 노드 수가 증가함에 따라 합의에 걸리는 시간이 늘어나고 저장 공간의 부담이 증가한다.
- **표준화 부족**: 각 기관마다 서로 다른 블록체인 프레임워크를 사용하여 상호 운용성(Interoperability)이 낮으며, 이는 환자가 병원을 옮길 때 데이터 마이그레이션을 어렵게 만든다.

### 3. 비판적 해석

논문은 허가형 블록체인이 의료 분야에 적합하다고 주장하지만, 이는 역설적으로 관리자 노드라는 새로운 중앙 집중 권한을 생성하는 결과를 초래한다. 따라서 '완전한 탈중앙화'와 '효율적인 관리' 사이의 균형을 맞추는 것이 여전히 해결되지 않은 과제이다. 또한, 많은 제안 시스템들이 실제 대규모 네트워크에서의 성능 평가 없이 소규모 시뮬레이션에 의존하고 있다는 점이 한계로 지적된다.

## 📌 TL;DR

본 논문은 의료 데이터 보안을 위해 블록체인을 적용하는 방안을 4W1H 방법론으로 분석한 종합 서베이 보고서이다. 기존 중앙 집중식 시스템의 보안 취약점을 해결하기 위해 **허가형 블록체인, 스마트 계약, 하이브리드 저장 모델(Off-chain)**의 조합이 가장 효율적임을 제시한다. 특히 COVID-19와 같은 팬데믹 상황에서 데이터 무결성과 추적 가능성이 매우 중요함을 입증하였다. 향후 연구로는 **프라이버시 전용 모듈의 통합, 리소스 효율적인 합의 알고리즘 개발, 그리고 글로벌 표준 가이드라인 수립**이 필수적임을 강조한다.
