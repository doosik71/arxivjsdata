# Technical Report for ICRA 2025 GOOSE 3D Semantic Segmentation Challenge: Adaptive Point Cloud Understanding for Heterogeneous Robotic Systems

Xiaoya Zhang (2025)

## 🧩 Problem to Solve

본 논문은 다양한 비정형 야외 환경에서 서로 다른 로봇 플랫폼(Heterogeneous Robotic Systems)을 통해 수집된 3D Point Cloud의 시맨틱 세그멘테이션(Semantic Segmentation) 문제를 해결하고자 한다.

이 문제는 다음과 같은 이유로 매우 중요하다. 야외 비정형 환경에서 동작하는 로봇은 안전한 주행, 경로 계획, 환경 상호작용 및 자율적 의사결정을 위해 주변 환경에 대한 세밀한 이해가 필수적이다. 그러나 각 로봇 플랫폼마다 센서 구성이 다르고 수집되는 데이터의 특성이 다르기 때문에, 기존의 단일 모델 접근 방식으로는 다음과 같은 어려움이 발생한다.

1. **도메인 시프트(Domain Shift):** 정형화된 도시 데이터셋(예: SemanticKITTI, nuScenes)으로 학습된 모델은 비정형 자연 지형에서 성능이 크게 저하된다.
2. **부정적 전이(Negative Transfer):** 서로 다른 데이터셋을 단순히 병합하여 학습할 경우, 포인트 밀도, 장면 복잡도, 시맨틱 정의의 차이로 인해 오히려 개별 데이터셋에 대한 성능이 떨어지는 현상이 발생한다.

따라서 본 연구의 목표는 다양한 플랫폼의 데이터 특성을 적응적으로 처리하면서도, 플랫폼 간의 공통된 시맨틱 특징을 학습할 수 있는 적응형 포인트 클라우드 이해 모델을 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 강력한 범용 포인트 클라우드 백본인 **Point Transformer v3 (PTv3)**에 **Point Prompt Tuning (PPT)** 프레임워크를 결합하여 플랫폼 간의 이질성을 극복하는 것이다.

중심적인 설계 아이디어는 다음과 같다.

- **플랫폼별 컨디셔닝(Platform-specific Conditioning):** 학습 가능한 도메인별 프롬프트를 통해 모델이 데이터의 출처(플랫폼)를 인식하게 하여, 공유된 가중치를 유지하면서도 각 플랫폼의 고유한 특성에 맞게 파라미터를 조정한다.
- **교차 데이터셋 클래스 정렬(Cross-dataset Class Alignment):** 서로 다른 라벨 체계를 가진 데이터셋들을 통합하기 위해 언어 모델 기반의 정렬(Language-Driven) 또는 경로 분리 정렬(Decoupled) 전략을 사용하여 시맨틱 일관성을 유지한다.

## 📎 Related Works

최근 3D 시맨틱 세그멘테이션 분야에서는 MinkUNet과 같은 희소 합성곱(Sparse Convolution) 기반 모델이나 Point Transformer 시리즈와 같은 어텐션 기반 모델들이 기하학적 특징 추출에서 우수한 성능을 보여왔다. 또한, 2D 이미지 특징을 결합하거나 DINOv2와 같은 2D 비전 파운데이션 모델(VFM)에서 지식을 증류(Distillation)하는 방식 등이 제안되었다.

그러나 이러한 기존 연구들은 다음과 같은 한계가 있다.

- **데이터셋 편향:** 대부분의 모델이 정형화된 도시 환경 데이터로 학습되어, 비정형 야외 지형으로 확장했을 때 일반화 능력이 떨어진다.
- **다중 데이터셋 학습의 비효율성:** 여러 데이터셋을 동시에 학습할 때 발생하는 포인트 밀도 차이와 라벨 공간의 불일치 문제를 효과적으로 해결하지 못해 부정적 전이 효과가 나타난다.

본 논문은 이러한 한계를 극복하기 위해 PPT 프레임워크를 도입하여, 추가적인 외부 데이터 없이도 도메인별 적응과 클래스 정렬을 수행함으로써 성능을 향상시켰다.

## 🛠️ Methodology

### 전체 파이프라인

시스템은 **PTv3 백본**을 통해 특징을 추출하고, **PPT 프레임워크**를 통해 플랫폼별 적응 및 클래스 정렬을 수행하는 구조로 이루어져 있다.

### 주요 구성 요소 및 역할

#### 1. PTv3 Backbone

특징 추출기로 사용되는 PTv3는 트랜스포머 기반 아키텍처로, 어텐션 메커니즘과 계층적 특징 추출을 통해 포인트 클라우드의 복잡한 지역적 및 전역적 기하 관계를 캡처한다.

#### 2. Point Prompt Tuning (PPT) Framework

이질적인 데이터를 처리하기 위해 두 가지 핵심 메커니즘을 사용한다.

- **데이터 기반 문맥 적응(Data-Driven Context Adaptation):**
  플랫폼별 특성을 처리하기 위해 **조건부 정규화(Conditional Normalization)**를 구현하였다. 백본 내의 정규화 층을 플랫폼 조건(예: "car", "alice", "spot")에 따라 분리하여, 공유 백본이 각 플랫폼의 고유한 센서 특성과 환경적 속성에 최적화된 특징을 처리할 수 있도록 파라미터를 조정한다.

- **교차 데이터셋 클래스 정렬 전략(Cross-Dataset Class Alignment Strategies):**
  다양한 클래스 체계를 통합하기 위해 두 가지 접근 방식을 제안한다.
  - **언어 기반 정렬(PPT-LA, Language-Driven Alignment):** 사전 학습된 CLIP (ViT-B/16)을 사용하여 클래스 이름을 텍스트 프롬프트로 변환하고 텍스트 임베딩을 생성한다. 포인트 특징을 CLIP 임베딩 공간으로 투영하여, 특징과 텍스트 간의 유사도를 비교함으로써 세그멘테이션을 수행한다.
  - **분리 정렬(PPT-DA, Decoupled Alignment):** 공유 백본 특징은 유지하되, 각 플랫폼 조건에 대해 별도의 세그멘테이션 헤드(Segmentation Head)를 둔다. 각 경로가 동일한 타겟 슈퍼클래스(Superclass) 세트를 예측하도록 학습하여 명시적인 매핑 없이도 정렬을 수행한다.

### 학습 절차 및 손실 함수

모델은 모든 플랫폼의 데이터를 통합하여 학습하며, 입력 샘플은 조건 인식 백본을 통과한 후 선택된 정렬 전략(LA 또는 DA)에 따라 최종 경로를 거친다.

학습 시 사용된 손실 함수는 다음과 같이 Cross Entropy Loss와 Lovasz Loss를 동일한 가중치로 결합하여 사용하였다.
$$Loss = \text{CrossEntropyLoss} + \text{LovaszLoss}$$

- **CrossEntropyLoss:** 픽셀 단위의 분류 정확도를 높이기 위해 사용한다.
- **LovaszLoss:** mIoU 지표를 직접적으로 최적화하기 위한 대리 손실 함수로, 특히 클래스 불균형 문제 해결에 도움을 준다.

## 📊 Results

### 실험 설정

- **데이터셋:** GOOSE 및 GOOSE-Ex (MuCAR-3, ALICE, Spot 플랫폼 포함).
- **평가 지표:** mean Intersection over Union (mIoU), mean Accuracy (mAcc), overall Accuracy (allAcc).
- **슈퍼클래스(7종):** 인공 구조물, 인공 지면, 자연 지면, 장애물, 차량, 식생, 인간.
- **하이퍼파라미터:** AdamW 옵티마이저, OneCycleLR 스케줄러, 50 Epoch 학습, 배치 사이즈 4.

### 주요 결과

실험 결과, PTv3에 PPT를 결합한 방식이 베이스라인인 PTv3 단독 모델보다 모든 플랫폼에서 성능 향상을 보였다. 특히 mIoU 기준 다음과 같은 향상이 관찰되었다.

- **Spot 플랫폼:** 최대 $22.59\%$ 증가 (가장 극적인 향상)
- **ALICE 플랫폼:** $12.71\%$ 증가
- **MuCAR-3 플랫폼:** $4.95\%$ 증가

### 전략별 비교 분석

- **PPT-LA (언어 기반):** ALICE와 Spot 플랫폼에서 특히 우수한 성능을 보였다. 이는 CLIP의 시맨틱 임베딩이 도메인 간 지식 전이에 매우 효과적임을 시사한다.
- **PPT-DA (분리 정렬):** MuCAR-3 플랫폼에서 가장 좋은 결과를 냈다. 이는 플랫폼의 특성이 매우 뚜렷할 경우 명시적인 경로 분리가 더 유리할 수 있음을 의미한다.

## 🧠 Insights & Discussion

### 강점 및 해석

본 연구는 단순한 모델 크기 확장이나 외부 데이터 추가 없이, **프롬프트 튜닝과 클래스 정렬**이라는 효율적인 기법만으로 이질적인 로봇 플랫폼 간의 데이터 간극을 성공적으로 메웠다. 특히 CLIP을 이용한 PPT-LA 방식은 텍스트 기반의 시맨틱 앵커를 제공함으로써, 서로 다른 환경에서도 "지면"이나 "식생"과 같은 개념을 일관되게 이해하도록 돕는 강력한 도구가 됨을 확인하였다.

### 한계 및 논의사항

논문에서 명시적으로 언급되지는 않았으나, 데이터셋 통계(Table I)에서 확인되듯 '인간(Human)'이나 '차량(Vehicle)' 클래스의 비율이 극히 낮아($0.03\% \sim 1.99\%$), 이러한 희소 클래스에 대한 일반화 성능은 여전히 도전적인 과제로 남아 있을 가능성이 크다. 또한, 추론 시에는 이미지가 필요 없다는 점이 실용적이지만, 학습 단계에서 CLIP과 같은 대형 모델의 임베딩에 의존하는 점이 계산 비용에 어떤 영향을 미쳤는지에 대한 상세 분석은 부족하다.

## 📌 TL;DR

본 연구는 이질적인 로봇 플랫폼에서 수집된 3D 포인트 클라우드의 세그멘테이션 성능을 높이기 위해 **PTv3 백본과 Point Prompt Tuning (PPT)**을 결합한 프레임워크를 제안하였다. 플랫폼별 조건부 정규화와 CLIP 기반의 언어 정렬 및 분리 정렬 전략을 통해 도메인 시프트와 부정적 전이 문제를 해결하였으며, 그 결과 베이스라인 대비 mIoU를 최대 **$22.59\%$** 향상시켰다. 이 결과는 다양한 센서 구성과 환경을 가진 필드 로보틱스(Field Robotics) 시스템의 실용적인 3D 인지 능력을 높이는 데 중요한 기여를 할 것으로 보인다.
