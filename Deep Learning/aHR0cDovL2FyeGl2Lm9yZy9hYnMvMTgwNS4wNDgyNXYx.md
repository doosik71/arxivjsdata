# Deep Learning in Software Engineering

Xiaochen Li, He Jiang, Zhilei Ren, Ge Li, Jingxuan Zhang (2018)

## 🧩 Problem to Solve

최근 소프트웨어 공학(Software Engineering, 이하 SE) 분야에서 딥러닝(Deep Learning)의 활용이 급격히 증가하고 있으나, 이를 체계적으로 정리한 분석 연구가 부족한 상황이다. 구체적으로 연구자들은 딥러닝을 SE 문제에 어떻게 통합하고 있는지, SE의 어떤 단계들이 딥러닝을 통해 촉진되고 있는지, 그리고 실제 산업 현장의 실무자들이 딥러닝의 혜택을 실제로 얻고 있는지에 대한 명확한 답이 필요하다.

본 논문의 목표는 SE 분야에서 딥러닝을 사용한 연구 논문들을 대상으로 서지 분석(Bibliography Analysis)을 수행하여, 딥러닝의 통합 방식, 적용 범위, 그리고 산업적 관점에서의 실용성을 분석하는 것이다.

## ✨ Key Contributions

본 연구의 핵심 기여는 SE 분야의 딥러닝 적용 현황을 정량적으로 분석하여 전체적인 조망(Overview)을 제공했다는 점이다. 주요 직관은 많은 연구가 새로운 아키텍처 설계보다는 기존의 표준 딥러닝 모델과 그 변형들을 SE 작업에 직접 적용하는 경향이 강하며, 학계의 관심사와 산업계의 실제 수요 사이에 간극이 존재한다는 것을 밝혀낸 것이다.

## 📎 Related Works

논문에서는 딥러닝이 데이터 마이닝과 패턴 인식 분야에서 거둔 성공을 바탕으로 SE 작업(요구사항 추출, 소스 코드 생성, 결함 예측 등)에 통합되는 추세라고 설명한다. 

기존의 문헌 리뷰(Literature Review)와 달리, 본 연구에서 채택한 서지 분석(Bibliography Analysis)은 통계적 데이터를 통해 전반적인 트렌드, 기술, 주제를 반영한다는 점에서 차별점을 가진다. 또한, 딥러닝의 불투명한 내부 표현으로 인해 발생하는 실용성 문제와 관련하여 효율성, 이해 가능성, 테스트 가능성에 관한 초기 연구들을 언급하며 본 분석의 배경으로 삼고 있다.

## 🛠️ Methodology

### 1. 데이터 수집 및 분석 절차
연구진은 Web of Science, ACM Digital Library, IEEE Xplore, Scopus의 4개 디지털 라이브러리에서 다음 세 가지 기준을 적용해 논문을 수집하였다.
- **C1**: SE 관련 키워드(software engineer*, software develop* 등) 포함 여부
- **C2**: 딥러닝 관련 키워드(deep learn*, neural network*) 포함 여부
- **C3**: 컴퓨터 과학 주제의 영어로 작성된 컨퍼런스 또는 저널 논문 여부

초기 4,443편의 후보 논문 중 중복 및 단편 논문을 제거하고, 수동 검토를 통해 SE와 딥러닝 모두에 집중한 63편의 시드 논문을 선정하였다. 이후 스노우볼링(Snowballing, 인용 및 참조 문헌 추적) 기법을 통해 최종적으로 98편의 논문을 분석 대상으로 확정하였다.

### 2. 딥러닝의 SE 적용 예시: 버그 리포트 요약 (Bug Report Summarization)
논문은 딥러닝이 SE에 어떻게 적용되는지 보여주기 위해 AutoEncoder를 이용한 버그 리포트 요약 사례를 제시하며, 이를 6단계 파이프라인으로 설명한다.

1.  **SE 데이터 수집**: 버그 리포트(제목, 설명, 댓글)를 수집한다.
2.  **SE 데이터 전처리**: 불용어(stop words) 및 프로그래밍 특화 노이즈, 너무 짧은 문장을 제거한다.
3.  **모델 선택 및 설정**: AutoEncoder와 같은 모델을 선택하고 층(layer)의 수와 뉴런 수를 결정한다. AutoEncoder는 입력과 출력이 대칭적인 구조를 가지며, 은닉층(hidden layer)을 통해 입력 데이터를 압축된 표현으로 학습한다.
4.  **입력 구성**: 버그 리포트의 단어 빈도를 계산하여 벡터로 변환하고, 이를 AutoEncoder의 학습 세트로 사용한다.
5.  **모델 훈련**: 입력층과 출력층의 차이를 최소화하는 방식으로 파라미터를 튜닝한다. 이는 비지도 학습(Unsupervised Learning) 방식으로 진행된다.
    $$\text{Loss} = \text{minimize}(\| \text{Input} - \text{Output} \|)$$
6.  **모델 적용**: 훈련된 모델을 통해 새로운 버그 리포트의 단어 빈도 벡터를 은닉 상태로 인코딩하고, 벡터 차원별 변화량을 추적하여 단어의 가중치를 산출함으로써 정보량이 많은 핵심 문장을 추출한다.

## 📊 Results

### 1. 딥러닝의 보급 현황
분석 결과, 2015년 이전에는 연간 3편 미만으로 관심이 적었으나, 2016년(28편), 2017년(39편)으로 급격히 증가하는 추세를 보였다. 또한, ICSE, ASE와 같은 SE 전문 학회뿐만 아니라 AAAI, NIPS, ICML 등 AI 분야의 권위 있는 학회에서도 SE 관련 딥러닝 연구가 발표되고 있어 두 커뮤니티 모두의 높은 관심을 확인하였다.

### 2. 딥러닝 통합 방식 및 데이터 유형
- **사용 모델**: 전체의 $84.7\%$가 표준 딥러닝 모델(AutoEncoder, CNN, RNN, DNN) 및 그 변형(LSTM, Bi-LSTM 등)을 그대로 사용하고 있다. 자체적인 특수 아키텍처를 설계한 경우는 극소수였다.
- **입력 데이터**:
    - 정의된 소프트웨어 메트릭(Predefined software metrics): 25편
    - 동적 소프트웨어 상태(Dynamic software status): 14편
    - 비순차적 텍스트/코드(Raw text/source code without sequence): 32편
    - 순차적 텍스트/코드(Raw text/source code in sequence): 22편
    - 기타(이미지 등): 5편

### 3. 적용된 SE 단계 및 작업
딥러닝은 요구사항 분석, 설계, 개발, 테스트, 유지보수, 프로젝트 관리의 6개 단계 전반에서 총 41개의 작업에 적용되었다. 
- **가장 많이 연구된 Top 5 작업**: 프로그램 학습 및 합성(Program learning and synthesis), 악성코드 탐지(Malware detection), 결함 예측(Defect prediction), 신뢰성/변경 가능성 추정(Reliability/changeability estimation), 개발 비용/노력 추정(Development cost/effort estimation).

### 4. 산업적 관심도
분석 대상 논문의 약 $21\%$($21$편)에 산업계 실무자가 저자로 참여하였다. 특히 '프로그램 학습 및 합성' 분야에서는 DeepMind, Facebook, Google, Microsoft와 같은 기업들이 활발히 참여하고 있었다. 그러나 학계의 Top 5 연구 주제와 산업계의 관심사 사이에는 불일치가 존재하며, 실무자들은 여전히 복잡한 딥러닝 모델보다는 가벼운 방법론을 선호하는 경향이 있다.

## 🧠 Insights & Discussion

### 1. 실용성(Practicability)의 한계
본 논문은 딥러닝이 SE에 적용될 때 직면하는 네 가지 주요 장벽을 제시한다.
- **효과성 및 효율성(Effectiveness & Efficiency)**: 특정 작업(예: Stack Overflow 지식 단위 연결)에서는 단순한 SVM 최적화 방식이 딥러닝보다 84배 빠르면서도 유사한 성능을 낸다. 이는 모든 SE 작업에 딥러닝이 최선은 아니며, 도메인 지식을 결합한 가벼운 모델과의 트레이드오프(trade-off) 고려가 필요함을 시사한다.
- **이해 가능성(Understandability)**: 딥러닝의 '블랙박스' 특성은 소프트웨어 엔지니어가 모델을 제어하는 것을 어렵게 만든다. 이를 해결하기 위해 Facebook의 ActiVis와 같은 시각화 도구가 제안되고 있다.
- **테스트 가능성(Testability)**: 복잡한 모델 구조는 보안과 신뢰성 문제를 야기한다. 이를 위해 커버리지 테스트(Coverage testing)나 메타모픽 테스트(Metamorphic testing)와 같은 SE 기법을 AI 모델 검증에 도입하려는 시도가 이루어지고 있다.

### 2. 비판적 해석
본 연구는 딥러닝의 SE 적용 현황을 넓게 훑어주었으나, 대부분의 연구가 기존 모델을 단순 적용하는 수준에 머물러 있다는 점을 꼬집었다. 이는 SE 데이터의 특성(구조적 특성, 희소성 등)을 반영한 전용 아키텍처 연구가 향후 SE-DL 융합의 핵심이 될 것임을 암시한다.

## 📌 TL;DR

본 논문은 98편의 논문을 분석하여 딥러닝이 SE의 전 단계에 걸쳐 41개 이상의 작업에 적용되고 있음을 밝혔다. 대부분의 연구가 표준 모델을 그대로 사용하는 단계에 머물러 있으며, 실제 산업 현장에서는 효율성과 불투명성 문제로 인해 적용에 신중한 모습이다. 향후 연구는 단순한 모델 적용을 넘어, SE 도메인 지식을 통합한 효율적이고 해석 가능한 모델 설계에 집중해야 할 것으로 보인다.
