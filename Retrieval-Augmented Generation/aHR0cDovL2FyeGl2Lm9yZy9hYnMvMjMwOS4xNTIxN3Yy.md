# Ragas: Automated Evaluation of Retrieval Augmented Generation

Shahul Es, Jithin James, Luis Espinosa-Anke, Steven Schockaert (2023)

## 🧩 Problem to Solve

본 논문은 Retrieval Augmented Generation (RAG) 파이프라인의 자동화된 평가 프레임워크의 부재라는 문제를 해결하고자 한다. RAG 시스템은 외부 데이터베이스에서 관련 문서를 검색(Retrieval)하고, 이를 기반으로 거대 언어 모델(LLM)이 답변을 생성(Generation)하는 구조를 가진다. 이러한 구조는 LLM의 고질적인 문제인 환각(Hallucination)을 줄이고 최신 정보를 제공할 수 있다는 장점이 있다.

그러나 RAG 시스템을 평가하는 것은 매우 까다롭다. 시스템의 성능은 검색 모듈이 얼마나 정확한 문맥(Context)을 찾아내는지, 생성 모듈이 해당 문맥을 얼마나 충실하게 활용하여 답변을 생성하는지 등 여러 차원에서 결정되기 때문이다. 기존의 평가 방식은 주로 정답지(Ground Truth)가 필요한 인간의 주석(Human Annotation)에 의존하거나, 모델의 내부 확률값(Perplexity 등)을 측정하는 방식이었다. 하지만 전자는 비용과 시간이 너무 많이 소요되며, 후자는 ChatGPT나 GPT-4와 같은 폐쇄형 API 모델에서는 확률값에 접근할 수 없다는 치명적인 한계가 있다. 따라서 본 연구의 목표는 정답지 없이도(Reference-free) RAG 파이프라인의 다양한 품질 차원을 자동 측정할 수 있는 프레임워크를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 RAG 시스템의 평가를 위해 정답지가 필요 없는 세 가지 핵심 지표(Faithfulness, Answer Relevance, Context Relevance)를 제안하고, 이를 구현한 Ragas 프레임워크를 개발한 것이다.

중심적인 설계 아이디어는 LLM을 평가자로 활용하되, 단순히 점수를 매기게 하는 것이 아니라 답변을 원자적 단위(Atomic statements)로 분해하여 검증하거나, 생성된 답변으로부터 역으로 질문을 생성하여 유사도를 측정하는 등의 구조적 접근 방식을 취함으로써 평가의 객관성과 신뢰도를 높이는 것이다. 또한, 이러한 지표들을 검증하기 위해 인간의 판단이 포함된 새로운 데이터셋인 WikiEval을 구축하여 제안 방법론의 유효성을 입증하였다.

## 📎 Related Works

논문에서는 LLM의 생성 텍스트 내 환각을 탐지하고 텍스트 생성 시스템을 평가하는 기존 연구들을 소개한다.

1. **Faithfulness 및 환각 탐지**: 퓨샷 프롬프팅(Few-shot prompting)을 통한 팩트 체크, 외부 지식 베이스와의 연결, 토큰 생성 확률 분석(BARTScore 등), 또는 모델의 내부 은닉 상태(Hidden states)를 이용한 분류기 학습 등이 연구되었다. 그러나 확률 기반 방식은 API 기반 모델에 적용 불가능하며, 은닉 상태 접근 방식 역시 동일한 한계를 가진다. SelfCheckGPT와 같이 여러 답변을 샘플링하여 일관성을 확인하는 방법도 제안되었으나, 이는 연산 비용이 높다.
2. **텍스트 생성 자동 평가**: GPTScore와 같이 특정 측면(유창성 등)을 프롬프트로 지정해 점수를 매기거나, BERTScore, MoverScore와 같이 임베딩 기반의 유사도를 측정하는 방식이 사용되었다. 하지만 이러한 방식들은 대부분 참조 답변(Reference answer)이 존재해야 한다는 전제가 있어, 실시간으로 변하는 데이터나 정답지가 없는 환경에서는 사용하기 어렵다.

Ragas는 이러한 기존 연구들과 달리, 참조 답변 없이 오직 질문, 문맥, 생성된 답변만으로 구성된 "RAG Triplet"만을 사용하여 평가를 수행한다는 점에서 차별점을 가진다.

## 🛠️ Methodology

Ragas는 RAG 시스템의 품질을 측정하기 위해 세 가지 핵심 지표를 제안한다. 모든 지표는 `gpt-3.5-turbo-16k` 모델을 기반으로 구현되었다.

### 1. Faithfulness (충실도)

답변이 주어진 문맥에 얼마나 근거하고 있는지를 측정한다. 이는 환각 현상을 방지하기 위해 필수적인 지표이다.

- **절차**:
    1. LLM을 사용하여 생성된 답변 $a_s(q)$를 짧고 명확한 개별 진술(Statements)의 집합 $S(a_s(q))$로 분해한다.
    2. 각 진술 $s_i$에 대해, 주어진 문맥 $c(q)$를 통해 해당 진술이 추론 가능한지 여부를 LLM의 검증 함수 $v(s_i, c(q))$를 통해 판별(Yes/No)한다.
- **방정식**:
  $$F = \frac{|V|}{|S|}$$
  여기서 $|V|$는 문맥에 의해 지지되는 진술의 수이며, $|S|$는 전체 진술의 수이다.

### 2. Answer Relevance (답변 관련성)

생성된 답변이 원래의 질문에 적절하게 응답하고 있는지를 측정한다. 이때 사실 여부(Factuality)는 고려하지 않고, 답변의 불완전함이나 불필요한 정보 포함 여부에 집중한다.

- **절차**:
    1. 생성된 답변 $a_s(q)$를 기반으로 LLM이 역으로 $n$개의 잠재적 질문 $q_i$를 생성하게 한다.
    2. 원래의 질문 $q$와 생성된 질문 $q_i$들의 임베딩을 추출(`text-embedding-ada-002` 모델 사용)하여 코사인 유사도 $\text{sim}(q, q_i)$를 계산한다.
- **방정식**:
  $$AR = \frac{1}{n} \sum_{i=1}^{n} \text{sim}(q, q_i)$$

### 3. Context Relevance (문맥 관련성)

검색된 문맥이 질문에 답하는 데 필요한 정보만을 효율적으로 포함하고 있는지, 불필요한 노이즈가 얼마나 적은지를 측정한다.

- **절차**:
    1. LLM을 사용하여 주어진 문맥 $c(q)$에서 질문 $q$에 답하는 데 결정적인(Crucial) 문장들의 부분 집합 $S_{ext}$를 추출한다.
    2. 추출된 문장의 수와 전체 문맥 문장 수의 비율을 계산한다.
- **방정식**:
  $$CR = \frac{\text{number of extracted sentences}}{\text{total number of sentences in } c(q)}$$

## 📊 Results

### 실험 설정 및 WikiEval 데이터셋

제안된 지표를 검증하기 위해 저자들은 **WikiEval** 데이터셋을 구축하였다. 2022년 이후의 최신 위키피디아 페이지 50개를 선정하여 질문-문맥-답변 쌍을 만들고, 두 명의 인간 주석자가 Faithfulness, Answer Relevance, Context Relevance의 세 차원에서 인간의 판단을 기록하였다.

### 정량적 결과

Ragas를 기존의 단순 LLM 평가 방식인 **GPT Score**(0-10점 척도 부여) 및 **GPT Ranking**(두 답변 중 하나를 선택)과 비교 분석하였다. 결과는 인간 주석자의 판단과 얼마나 일치하는지를 나타내는 정확도(Accuracy)로 측정되었다.

| 지표 | Faithfulness | Answer Relevance | Context Relevance |
| :--- | :---: | :---: | :---: |
| **Ragas** | **0.95** | **0.78** | **0.70** |
| GPT Score | 0.72 | 0.52 | 0.63 |
| GPT Ranking | 0.54 | 0.40 | 0.52 |

실험 결과, Ragas는 모든 지표에서 베이스라인보다 월등히 높은 인간 일치도를 보였다. 특히 Faithfulness에서 가장 높은 성능(0.95)을 기록하여, LLM을 이용한 원자적 진술 분해 및 검증 방식이 매우 효과적임을 입증하였다.

## 🧠 Insights & Discussion

본 논문의 실험 결과는 RAG 시스템의 자동 평가에 있어 단순한 점수 부여나 순위 매기기보다, 분석적인 파이프라인(분해 $\to$ 검증 또는 역질문 생성 $\to$ 유사도 측정)을 구축하는 것이 훨씬 더 신뢰할 수 있는 지표를 제공한다는 것을 보여준다.

**강점 및 시사점**:

- 정답지가 없는 실제 운영 환경에서도 RAG 파이프라인의 병목 지점을 파악할 수 있다. 예를 들어, $CR$은 낮고 $F$가 높다면 검색 모듈의 정밀도를 개선해야 하며, $CR$은 높지만 $F$가 낮다면 생성 모델의 프롬프트나 모델 자체를 개선해야 함을 알 수 있다.

**한계 및 비판적 해석**:

- **Context Relevance의 측정 난이도**: 실험 결과에서 $CR$의 정확도가 가장 낮게 나타났다. 이는 LLM이 긴 문맥에서 정답에 결정적인 문장을 정확히 추출하는 데 어려움을 겪기 때문인 것으로 분석된다. 이는 "Lost in the Middle" 현상과 같이 LLM이 긴 컨텍스트의 중간 정보를 처리하는 능력의 한계와 관련이 있을 수 있다.
- **LLM 의존성**: 평가 도구 자체가 LLM(GPT-3.5)에 의존하고 있으므로, 평가 모델 자체가 가지는 편향(Bias)이나 오류가 결과에 반영될 수 있다. 또한 API 비용이 발생한다는 현실적인 제약이 있다.

## 📌 TL;DR

본 연구는 정답지 없이 RAG 시스템을 평가할 수 있는 프레임워크인 **Ragas**를 제안하였다. Ragas는 **Faithfulness**(문맥 근거 여부), **Answer Relevance**(질문 응답 적절성), **Context Relevance**(문맥 효율성)라는 세 가지 지표를 통해 RAG 파이프라인을 정밀하게 진단한다. WikiEval 데이터셋을 통한 검증 결과, 단순 LLM 점수 측정 방식보다 인간의 판단과 훨씬 더 높은 일치도를 보였다. 이 연구는 RAG 시스템 개발자가 빠르게 반복 실험(Iteration)을 수행하고 성능을 정량적으로 개선하는 데 매우 중요한 도구가 될 것으로 기대된다.
