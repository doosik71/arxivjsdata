# MONET: Mixture of Monosemantic Experts for Transformers

Jungwoo Park, Young Jin Ahn, Kee-Eung Kim, Jaewoo Kang (2025)

## 🧩 Problem to Solve

본 논문은 거대 언어 모델(LLM)의 내부 연산 과정을 이해하려는 '기계적 해석 가능성(Mechanistic Interpretability)'의 핵심 장애물인 **다의성(Polysemanticity)** 문제를 해결하고자 한다. 다의성이란 개별 뉴런이 서로 관련 없는 여러 개념에 동시에 반응하는 현상을 말하며, 이는 신경망이 제한된 뉴런 수보다 더 많은 특징을 압축하여 표현하려는 '중첩 가설(Superposition Hypothesis)'에서 기인한다.

기존의 해결책인 희소 오토인코더(Sparse Autoencoders, SAEs)는 사후 학습(post-hoc) 방식의 재구성 손실(reconstruction loss)에 의존하기 때문에, LLM의 원래 성능을 저하시키거나 학습 데이터셋과 실제 코퍼스 간의 괴리로 인한 문제를 야기한다. 또한, 기존의 전문가 혼합(Mixture-of-Experts, MoE) 구조는 전문가의 수가 너무 적어 각 전문가가 여전히 다양한 개념을 동시에 다루는 지식 하이브리드(knowledge hybridity) 문제가 발생하며, 전문가 수를 무작정 늘릴 경우 파라미터 수가 선형적으로 증가하여 메모리 병목 현상이 발생한다는 한계가 있다.

따라서 본 논문의 목표는 모델 성능 저하 없이 전문가의 수를 획기적으로 늘려 개별 전문가가 단일한 의미만을 갖게 하는 **단일 의미성(Monosemanticity)**을 확보하고, 이를 통해 LLM의 내부 지식을 투명하게 관찰하고 제어할 수 있는 아키텍처를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **희소 사전 학습(Sparse Dictionary Learning)을 엔드투엔드(end-to-end) MoE 사전 학습 과정에 직접 통합**하고, 새로운 **전문가 분해(Expert Decomposition)** 방법론을 통해 파라미터 효율적으로 전문가의 수를 극단적으로 확장하는 것이다.

주요 기여 사항은 다음과 같다:
1. **파라미터 효율적인 전문가 확장**: 전문가의 수를 레이어당 262,144개까지 늘리면서도, 전체 파라미터 수는 전문가 수의 제곱근($\sqrt{N}$)에 비례하여 증가하도록 설계하여 메모리 제약을 해결하였다.
2. **단일 의미적 전문가를 통한 해석 가능성**: 매우 세분화된 전문가들의 라우팅 패턴을 분석함으로써, 개별 전문가가 특정 도메인이나 개념(예: 화학 화합물, 미국 주 이름 등)에만 특화되어 반응하는 단일 의미성을 가짐을 증명하였다.
3. **강건한 지식 조작(Knowledge Manipulation)**: 특정 도메인, 언어 또는 독성(toxicity)과 관련된 전문가만을 선택적으로 제거함으로써, 일반적인 성능 저하 없이 특정 지식만을 정밀하게 삭제하거나 모델의 행동을 조정하는 것이 가능함을 보여주었다.

## 📎 Related Works

논문에서 언급한 관련 연구와 그 한계는 다음과 같다:
- **Sparse Autoencoders (SAEs)**: 활성화 공간을 희소하고 과완전한 기저(overcomplete bases)로 분해하여 특징을 얽힘 없이 추출하려 한다. 하지만 사후 학습 방식이므로 LLM의 원래 기능이 훼손될 수 있고, 재구성 오차가 누적되어 모델 전체의 안정성이 떨어지는 문제가 있다.
- **Sparse Mixture-of-Experts (SMoE)**: 일부 전문가만 활성화하여 계산 효율성을 높인다. 그러나 대부분의 SMoE는 전문가 수가 제한적이어서 단일 의미성을 달성하기 어렵다.
- **Parameter Efficient Expert Retrieval (PEER)**: Product Key 알고리즘을 사용하여 전문가 검색 비용을 $O(\sqrt{N})$으로 줄였으나, 여전히 전문가 개별 파라미터를 모두 저장해야 하므로 전체 파라미터 수는 전문가 수 $N$에 따라 선형적으로 증가하는 메모리 병목 문제가 존재한다.

MONET은 PEER의 효율적인 검색 메커니즘을 계승하되, 전문가를 '구성(composition)'하는 방식을 도입하여 메모리 사용량을 $O(\sqrt{N})$으로 낮춤으로써 기존 접근 방식과 차별화된다.

## 🛠️ Methodology

### 전체 시스템 구조 및 전문가 분해
MONET은 전문가를 독립적인 네트워크로 저장하는 대신, 레이어를 분해하여 동적으로 조합하는 **Product Key Composition** 방식을 제안한다. 두 가지 분해 방법이 제시된다.

**1. 수평적 전문가 분해 (Horizontal Decomposition, HD)**
각 전문가를 하위 선형 레이어(Bottom layer)와 상위 선형 레이어(Top layer)로 분해한다.
- 하위 레이어 집합: $\{U_i\}_{i=1}^{\sqrt{N}} \subset \mathbb{R}^{m \times d}$
- 상위 레이어 집합: $\{V_j\}_{j=1}^{\sqrt{N}} \subset \mathbb{R}^{d \times m}$
- $(i, j)$번째 조합된 전문가는 다음과 같이 정의된다:
  $$E_{ij}(x) = V_j \sigma(U_i x + b_{1i}) + b_{2j}$$
이 방식을 통해 $\sqrt{N}$개의 하위 레이어와 $\sqrt{N}$개의 상위 레이어만으로 $N$개의 고유한 전문가를 생성할 수 있다.

**2. 수직적 전문가 분해 (Vertical Decomposition, VD)**
전문가 네트워크를 수직 방향(차원 방향)으로 좌우 세그먼트로 분할하여 조합한다. 수평적 분해와 마찬가지로 공간 복잡도를 $O(\sqrt{N})$으로 유지하며, 실험적으로 HD보다 더 우수한 성능을 보였다.

### 라우팅 및 계산 최적화
- **Dense Routing**: 토큰마다 활성화되는 전문가가 달라 효율적인 계산이 어려운 문제를 해결하기 위해, 희소 라우팅 점수를 모든 전문가로 확장하여 연산을 재구성하는 Dense Routing을 채택하였다. 이를 통해 메모리 집약적인 연산을 라우팅 단계 전후로 미리 계산(precompute)할 수 있다.
- **Batch Normalization**: 하드웨어적으로 비효율적인 Top-k 정렬 작업을 피하기 위해, 배치 정규화를 사용하여 라우팅 분위수(quantile)를 추정함으로써 학습 시간을 단축하였다.

### 학습 목표 및 손실 함수
모델은 일반적인 언어 모델링 손실($L_{LM}$) 외에 전문가의 효율적인 활용을 위해 두 가지 보조 손실 함수를 사용한다.
1. **균일성 손실 (Uniformity Loss, $L_{unif}$)**: 라우팅 확률 분포와 균등 분포 사이의 KL 발산을 최소화하여 특정 전문가에게만 쏠림 현상이 발생하는 것을 방지한다.
  $$L_{unif} = -\frac{1}{2H\sqrt{N}} \sum_{h=1}^{H} \sum_{i=1}^{\sqrt{N}} \log \hat{g}_{1hi} - \frac{1}{2H\sqrt{N}} \sum_{h=1}^{H} \sum_{j=1}^{\sqrt{N}} \log \hat{g}_{2hj}$$
2. **모호성 손실 (Ambiguity Loss, $L_{amb}$)**: 각 토큰이 가장 높은 확신을 가진 특정 전문가에게 할당되도록 유도하여 전문가의 전문화를 촉진한다.
  $$L_{amb} = \frac{1}{2H} \sum_{h=1}^{H} (1 - \max g_{1h}) + \frac{1}{2H} \sum_{h=1}^{H} (1 - \max g_{2h})$$

최종 손실 함수는 다음과 같다:
$$L = L_{LM} + \lambda L_{unif} + \lambda L_{amb}$$

## 📊 Results

### 실험 설정
- **모델 규모**: 850M, 1.4B, 4.1B 파라미터 모델을 구축하여 확장성을 테스트하였다.
- **데이터셋**: FineWeb-Edu(100B 토큰)에서 사전 학습하였으며, 코딩 모델인 CODEMONET은 StarCoder 데이터를 사용하였다.
- **비교 대상**: dense LLAMA 모델, OLMoE(MoE), Gemma 2 + SAE(Gemma Scope) 등과 비교하였다.

### 정량적 결과
- **언어 모델링 성능**: Table 2에 따르면 MONET은 파라미터 수가 유사한 dense LLAMA 모델과 경쟁 가능한 수준의 성능을 유지하였다. 특히 수직 분해(VD) 방식이 수평 분해(HD)보다 일관되게 높은 성능을 보였다.
- **SAE와의 비교**: SAE를 적용한 Gemma 2 모델은 재구성 오차로 인해 open-ended 벤치마크에서 성능 저하와 불안정성을 보인 반면, MONET은 안정적인 성능을 유지하였다.

### 정성적 결과 및 지식 조작
- **단일 의미성 확인**: 분석 결과, 개별 전문가가 매우 구체적인 개념(예: 전문가 147,040은 화학 화합물, 전문가 73,329는 미국 주 이름)에만 반응함을 확인하였다. 또한, "Bay"라는 단어가 들어왔을 때 지리적 영역과 수학적 베이지안(Bayesian) 개념을 서로 다른 전문가(Expert 48,936 vs 54,136)가 처리하는 것을 통해 문맥적 단일 의미성을 입증하였다.
- **지식 삭제 (Knowledge Unlearning)**: 특정 도메인에 특화된 전문가만 제거했을 때, 해당 도메인의 성능은 크게 하락하지만 다른 도메인의 성능은 거의 유지되는 것을 확인하였다(Table 11). 이는 OLMoE나 LLAMA에서는 불가능했던 정밀한 지식 조작이다.
- **독성 제거 (Toxicity Mitigation)**: 독성 콘텐츠 생성과 상관관계가 높은 '독성 전문가'를 식별하여 제거한 결과, 일반적인 도움말 성능(Helpfulness)은 유지하면서 생성되는 텍스트의 독성 확률을 유의미하게 낮출 수 있었다(Table 4, 5).

## 🧠 Insights & Discussion

**강점 및 성과**
본 연구는 전문가의 수를 극단적으로 늘리는 것이 기계적 해석 가능성을 확보하는 실질적인 방법임을 보여주었다. 특히, 사후 분석 도구인 SAE와 달리 학습 단계에서부터 단일 의미성을 내재화함으로써 성능 저하 없는 투명한 모델을 구현하였다. 또한, 지식을 '제거'하는 방식의 접근이 모델의 유해성을 제어하는 매우 강력하고 정밀한 수단이 될 수 있음을 시사한다.

**한계 및 향후 과제**
1. **전문가 선택 기준**: 현재는 라우팅 점수의 왜곡(skewness)이나 피어슨 상관계수와 같은 단순한 기준으로 전문분야를 판별하고 있으나, 더 정교한 선택 방법론이 필요하다.
2. **자동 해석의 정량화**: 전문가의 역할을 설명하는 'Self-explained experts' 방식이 제시되었으나, 이에 대한 정량적 평가는 아직 이루어지지 않았다.
3. **지식 조작의 범위**: 현재는 지식의 '삭제'에 집중하고 있으며, 향후에는 환각(Hallucination) 해결이나 지속 학습(Lifelong Learning)으로 연구 범위를 확장할 필요가 있다.

## 📌 TL;DR

MONET은 전문가 수를 262K개까지 획기적으로 늘린 새로운 MoE 아키텍처로, 전문가 분해(Expert Decomposition) 기법을 통해 메모리 효율성을 확보하면서 개별 전문가가 단일한 개념만 담당하는 **단일 의미성(Monosemanticity)**을 달성하였다. 이를 통해 LLM의 내부 지식을 투명하게 관찰할 수 있을 뿐만 아니라, 특정 도메인의 지식이나 독성 요소를 일반 성능 저하 없이 '외과적으로' 제거할 수 있는 강력한 제어 가능성을 입증하였다. 이 연구는 향후 투명하고 정렬된(aligned) 거대 모델 설계에 중요한 이정표가 될 것으로 보인다.
