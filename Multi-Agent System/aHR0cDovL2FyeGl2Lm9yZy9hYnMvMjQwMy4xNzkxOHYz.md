# AGENTSTUDIO: A TOOLKIT FOR BUILDING GENERAL VIRTUAL AGENTS

Longtao Zheng, Zhiyuan Huang, Zhenghai Xue, Xinrun Wang, Bo An, Shuicheng Yan (2025)

## 🧩 Problem to Solve

범용 가상 에이전트(General Virtual Agents)는 다중 모달 관측치(multimodal observations)를 처리하고, 복잡한 액션 공간을 마스터하며, 동적인 오픈 도메인 환경에서 스스로 개선될 수 있어야 한다. 그러나 기존의 에이전트 학습 및 평가 환경은 다음과 같은 한계점을 가지고 있다.

- **도메인 특정적 환경:** 대부분의 환경이 웹 브라우저나 특정 OS 등 특정 도메인에 국한되어 있어, 인간이 수행하는 실제 세계의 광범위한 작업(real-world tasks)을 포괄하지 못한다.
- **설정의 복잡성 및 정적 데이터셋:** 환경 구축이 어렵고, 많은 벤치마크가 정적인 데이터셋 형태로 제공되어 에이전트가 시행착오(trial and error)를 통해 학습할 수 있는 온라인 상호작용 기회를 제공하지 않는다.
- **세부 능력 분석의 부재:** 단순히 성공률(success rate)만 측정할 뿐, GUI 그라운딩(grounding), 비디오로부터의 학습, 성공 여부 판단(success detection)과 같은 에이전트의 근본적인 핵심 능력을 세부적으로 분석하는 지표가 부족하다.

본 논문의 목표는 이러한 문제를 해결하기 위해 환경(Environments), 도구(Tools), 벤치마크(Benchmarks)의 삼위일체(trinity)로 구성된 **AgentStudio** 툴킷을 제안하여, 범용 가상 에이전트의 개발과 정밀한 평가를 가능하게 하는 것이다.

## ✨ Key Contributions

AgentStudio의 핵심 아이디어는 에이전트가 실제 소프트웨어를 자유롭게 활용하기 위해 필요한 세 가지 필수 요소(Desiderata)—**범용 그라운딩(General Grounding), 디지털 세계 지식(Digital World Knowledge), 오픈엔드 학습(Open-ended Learning)**—를 지원하는 통합 생태계를 구축하는 것이다.

- **범용 관측 및 액션 공간 제공:** 텍스트, 이미지뿐만 아니라 비디오 관측치를 포함하며, GUI 조작(키보드, 마우스)과 API 호출(코드 실행)을 동시에 지원하는 통합 인터페이스를 구축하였다.
- **에이전트 개발 도구 제공:** GUI 요소 어노테이션 도구, 비디오 기반 액션 라벨링 도구, 벤치마크 태스크 생성 및 검증 도구를 제공하여 고품질 데이터셋 구축을 용이하게 하였다.
- **다각적 벤치마크 및 데이터셋 구축:** 실제 소프트웨어 205종을 대상으로 한 온라인 태스크 스위트와 더불어, 근본적 능력을 평가하기 위한 세 가지 특화 데이터셋(GroundUI, IDMBench, CriticBench)을 제안하였다.

## 📎 Related Works

기존의 가상 에이전트 환경들은 특정 도메인에 치중되어 있었다. 예를 들어 World of Bits는 웹 시뮬레이터에 집중했고, AndroidWorld는 안드로이드 기기에 국한되었다. OSWorld는 데스크톱 환경으로 확장하였으나 주로 GUI 상호작용에 집중하였다.

AgentStudio는 기존 연구들과 다음과 같은 차별점을 갖는다.

- **관측 공간의 확장:** 기존 환경들이 스크린샷($O_{\text{Image}}$)이나 HTML($O_{\text{Text}}$)에 의존한 반면, AgentStudio는 실시간 비디오($O_{\text{Video}}$)를 포함하여 동적인 환경 변화를 포착할 수 있게 한다.
- **액션 공간의 통합:** API 기반의 고수준 작업과 GUI 기반의 저수준 작업을 동시에 지원함으로써, 에이전트가 상황에 따라 가장 효율적인 수단(API vs GUI)을 선택하는 능력을 평가할 수 있다.
- **능력의 분해 평가:** 단순 성공률 측정을 넘어, GUI 그라운딩-비디오 학습-성공 판단이라는 세부 능력으로 문제를 분해하여 분석한다.

## 🛠️ Methodology

### 1. 시스템 모델링 (POMDP)

AgentStudio는 컴퓨터 작업을 부분 관측 마르코프 결정 과정(POMDP)으로 모델링하며, 이는 다음과 같은 튜플로 정의된다: $$(S, A, O, T, R, F, U)$$
여기서 $S$는 상태 공간, $A$는 액션 공간, $O$는 관측 공간, $T$는 전이 함수, $R$은 보상 함수, $F$는 피드백 공간, $U$는 지시어(instruction) 공간이다.

### 2. 범용 관측 및 액션 공간

- **관측 공간 ($O$):** 텍스트, 이미지, 비디오의 합집합으로 정의된다.
  $$O = O_{\text{Text}} \cup O_{\text{Image}} \cup O_{\text{Video}}$$
  이를 통해 에이전트는 화면 녹화 영상, 스크린샷, 코드 실행 결과뿐만 아니라 HTML 코드나 접근성 트리(accessibility trees)와 같은 비시각적 정보도 함께 활용할 수 있다.
- **액션 공간 ($A$):** GUI 상호작용과 API 호출의 합집합으로 정의된다.
  $$A = A_{\text{GUI}} \cup A_{\text{API}}$$
  에이전트는 Python 커널을 통해 코드를 실행함으로써 API를 호출하거나, 마우스/키보드 제어를 통해 타사 소프트웨어를 조작한다.

### 3. 인터랙티브 환경 및 피드백

실제 장치에서 실시간으로 작동하는 온라인 환경을 구축하였으며, VNC(Virtual Network Computing)를 통해 Windows, Linux, macOS 등 다양한 OS를 지원한다. 특히 에이전트의 자가 수정(self-correction)을 돕기 위해 이진 보상 $R: S \rightarrow \{0, 1\}$과 더불어 실패 원인을 설명하는 언어 피드백 $F: S \rightarrow \{\text{failure reason, success}\}$를 제공한다.

### 4. 벤치마크 구축 절차

총 205개의 태스크를 다음 3단계에 걸쳐 구성하였다.

1. **기초 단계 (Stage I):** 단일 애플리케이션 기반의 Single-API 태스크(50개)와 Single-GUI 태스크(95개)를 설계하고 자동 평가기(auto-evaluators)를 구현한다.
2. **조합 단계 (Stage II):** 기초 태스크들을 조합하여 여러 앱을 넘나드는 복합적인(compositional) 태스크(60개)를 생성한다.
3. **검증 단계 (Stage III):** 인간의 궤적 데이터를 수집하여 자동 평가기의 정확성을 검증하고 난이도를 측정한다.

## 📊 Results

### 1. 온라인 벤치마크 결과

GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro 등 최신 VLM을 평가한 결과, 다음과 같은 사실이 밝혀졌다.

- **GUI 및 복합 작업의 어려움:** 단순한 CLI 작업에서는 높은 성능을 보이지만, 복잡한 API 호출이나 GUI 조작이 필요한 작업, 특히 여러 앱을 사용하는 compositional task에서는 성능이 급격히 하락한다.
- **관측 공간의 역설:** API로 해결 가능한 작업임에도 스크린샷($O_{\text{Image}}$)을 제공했을 때, 일부 모델(특히 Gemini)은 효율적인 API 호출 대신 비효율적인 GUI 조작을 시도하여 오히려 성능이 떨어지는 현상이 발생한다.

### 2. 세부 능력 평가 (Fine-grained Evaluation)

- **GroundUI (UI 그라운딩):** 전용 모델인 SeeClick이 일반 VLM보다 월등한 성능을 보였다. 또한, 화면 해상도가 낮을수록(모바일 환경) 정확도가 높아지는 경향이 확인되었다.
- **IDMBench (비디오 액션 라벨링):** VLM이 비디오를 보고 수행된 액션을 예측하는 능력은 매우 낮았으며, 특히 여러 단계의 액션을 연속적으로 예측하는 IDM-Multiple 설정에서는 성능이 심각하게 저하되었다.
- **CriticBench (성공 판단):** 궤적을 보고 작업 성공 여부를 판단하는 능력은 오픈 소스와 상용 모델 간에 큰 차이가 없었으며, 액션 정보를 추가로 제공하더라도 성능 향상이 제한적이었다.

## 🧠 Insights & Discussion

### 강점 및 시사점

본 연구는 가상 에이전트가 단순히 '작업을 완수하는 것'을 넘어, **'어떻게'** 완수하는지에 대한 근본적인 능력을 분석할 수 있는 프레임워크를 제시하였다. 특히 GUI 그라운딩의 오차가 누적되어 전체 작업 실패로 이어진다는 점과, 현재의 VLM이 인터넷의 방대한 튜토리얼 비디오로부터 액션을 학습(Inverse Dynamics Modeling)하기에는 아직 역부족이라는 점을 정량적으로 증명하였다.

### 한계 및 비판적 해석

- **평가기의 완전성:** 실제 환경의 복잡성으로 인해 자동 평가기(auto-evaluator)가 완벽하지 않을 수 있으며, 이는 결과에 노이즈를 줄 수 있다.
- **데이터 오염 가능성:** 정적인 데이터셋(GroundUI 등)의 경우, 모델이 학습 과정에서 해당 데이터를 접했을 가능성(data leakage)을 배제할 수 없다.
- **환경의 제약:** 재현성을 위해 Ubuntu Docker 환경에서 실험을 진행했으나, 이는 실제 사용자의 매우 다양한 OS 환경을 완벽하게 대변하지 못할 수 있다.

## 📌 TL;DR

AgentStudio는 범용 가상 에이전트 구축을 위한 **환경, 도구, 벤치마크의 통합 툴킷**이다. 이 툴킷은 비디오 관측과 GUI/API 액션을 모두 지원하는 범용 인터페이스를 제공하며, 특히 GUI 그라운딩, 비디오 학습, 성공 판단이라는 세 가지 핵심 능력을 정밀하게 측정할 수 있는 데이터셋을 함께 제안한다. 실험 결과, 최신 VLM조차 실제 GUI 조작과 복합 작업에서 큰 한계를 보였으며, 이는 향후 에이전트 연구가 단순한 프롬프팅을 넘어 정밀한 그라운딩과 비디오 기반 학습에 집중해야 함을 시사한다.
