# RAI: Flexible Agent Framework for Embodied AI

Kajetan Rachwał, Maciej Majek, Bartłomiej Boczek, Kacper Dąbrowski, Paweł Liberadzki, Adam Dąbrowski, and Maria Ganzha (2025)

## 🧩 Problem to Solve

본 논문은 Embodied AI(구체화된 인공지능)를 구현함에 있어 생성형 언어 모델(LLM)을 로봇 시스템과 통합하는 과정에서 발생하는 파편화와 유연성 부족 문제를 해결하고자 한다. 

현재 LLM 기반의 Embodied AI 시스템들은 대부분 외부 API나 특정 AI 모델과의 통합을 위해 맞춤형(custom) 구현에 크게 의존하고 있다. 기존의 일부 프레임워크는 특정 통신 스택(예: ROS 2)에 너무 강하게 결합되어 있거나, 단일 에이전트(single-agent) 배치에 치중되어 있어 다중 에이전트 시스템(Multi-Agent Systems, MAS)을 통한 복잡한 작업 수행에 한계가 있다. 또한, LLM 고유의 문제인 환각(hallucination), 해석 가능성 부족, 보안 취약성 등의 문제가 실제 물리적 환경의 로봇에 적용될 때 심각한 기능적 및 윤리적 우려를 야기한다.

따라서 본 연구의 목표는 LLM, 로봇 스택, 외부 API 및 인간-로봇 상호작용(HRI) 메커니즘을 유연하게 통합할 수 있는 확장 가능한 다중 에이전트 프레임워크인 RAI를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 **M-Agent 모델**을 기반으로 한 모듈식 및 확장 가능한 아키텍처를 통해 Embodied AI를 위한 범용 프레임워크를 설계한 것이다. 

중심적인 설계 아이디어는 에이전트를 **추론 시스템(Reasoning system), 센서(Sensors), 액추에이터(Actuators)**의 세 가지 추상화 단계로 분리하고, 이를 `Agents`, `Connectors`, `Tools`라는 구성 요소로 구체화하여 에이전트가 가상 환경, 물리 환경, 또는 그 중간 단계에서 원활하게 동작하도록 하는 것이다. 특히, RAG(Retrieval Augmented Generation)와 벡터 데이터베이스를 활용하여 로봇의 물리적 정체성(Embodiment)을 정의하고 이를 LLM에 주입하는 메커니즘을 제공함으로써, 에이전트가 자신의 물리적 형태와 능력을 이해할 수 있도록 설계하였다.

## 📎 Related Works

논문에서는 다음과 같은 기존 접근 방식과 그 한계를 언급한다.

- **API 통합 도구 및 프레임워크:** `ModelScope`와 같은 도구가 API 통합을 돕고, `ROSA`와 같은 시스템은 ROS/ROS 2와 밀접하게 결합되어 있다. 그러나 `ROSA`는 다른 통신 스택과의 통합 유연성이 부족하다.
- **에이전트 평가 환경:** `AgentGYM`은 표준화된 환경을 제공하고 LLM 에이전트를 평가하지만, 주로 단일 에이전트 배치에 설계되어 있다.
- **전통적 로봇 에이전트 프레임워크:** `FABRIC`은 ROS와의 통합을 위해 재사용 가능한 컴포넌트를 제공하며, `MARC`는 강화 학습 기반의 MAS를 위해 설계되었다. `G++` 아키텍처는 다양한 로봇 도메인에 적용 가능한 일반화된 프레임워크를 제안하였다.

**RAI와의 차별점:** 기존 프레임워크들은 LLM이 제공하는 원시적인 유연성과 인간-로봇 상호작용(HRI) 능력을 내장하고 있지 않다. RAI는 LLM의 Tool-calling 메커니즘을 로봇의 액추에이터와 직접 연결하고, 다중 에이전트 구조를 통해 추론과 제어를 분리함으로써 기존 시스템보다 더 높은 유연성을 제공한다.

## 🛠️ Methodology

RAI 아키텍처는 고수준의 추상 클래스를 정의하여 에이전트를 간결하고 유연하게 정의할 수 있도록 한다.

### 1. 주요 구성 요소 (Component Definitions)
- **Agents:** MAS의 기본 빌딩 블록이다. `run()`과 `stop()` 메서드를 포함하는 인터페이스를 가지며, 내부적으로 의사결정 프로세스(규칙 기반 또는 LLM 기반)를 갖는다.
- **Connectors:** 에이전트의 센서와 액추에이터를 추상화한 것이다. ROS 2의 통신 방식과 유사하게 세 가지 모드를 지원한다.
    - **Publish-Subscribe:** 효율적인 메시지 송수신을 제공하나 수신 보장이 없다.
    - **Service-based:** 요청-응답 구조로 메시지 전달 및 결과 확인이 가능하여 신뢰성이 높다.
    - **Action-based:** 실행 상태에 대한 지속적인 피드백을 제공한다.
- **Tools:** LLM의 Tool-calling 메커니즘에서 영감을 얻은 추상화 단계이다. `Connector`를 통해 송수신되는 데이터를 생성하거나 파싱하며, `langchain`과 호환되어 LLM이 로봇 기능을 함수처럼 호출할 수 있게 한다.

### 2. 구체화 메커니즘 (Embodiment)
RAI는 에이전트가 자신의 "몸"과 환경에 대한 이해를 갖도록 `RAI_whoami` 패키지를 제공한다. 
- **데이터 처리:** 텍스트, PDF 등의 문서 데이터를 자동으로 벡터 데이터베이스(Faiss 사용)로 변환한다.
- **RAG 통합:** LLM이 Retrieval Augmented Generation(RAG)을 통해 로봇의 사양, 능력, URDF(Unified Robot Description Format) 등의 정보를 검색하여 시스템 프롬프트에 반영하거나 런타임 중에 활용할 수 있게 한다.

### 3. 에이전트 라이브러리
RAI는 즉시 사용 가능한 네 가지 사전 패키지 에이전트를 제공한다.
- **VoiceRecognition / TextToSpeech:** 음성 인식 및 합성 기능을 통한 HRI 구현.
- **Conversational Agent:** ReAct 아키텍처 기반의 LLM 에이전트로, 커넥터를 통해 멀티모달 입출력을 처리한다.
- **StateBased Agent:** 유한 상태 기계(FSM) 기반 에이전트로, 절차적 접근 방식과 LLM 추론을 결합하여 복잡한 작업을 수행한다.

## 📊 Results

본 논문은 세 가지 서로 다른 환경에서 RAI의 실효성을 검증하였다.

### 1. 자율 이동 로봇 (Husarion ROSBot XL)
- **설정:** 물리적 사무실 환경과 O3DE 시뮬레이션 내 디지털 트윈 환경에서 테스트하였다.
- **결과:** 단일 에이전트 구성에서는 미션 수행 중 HRI 응답성이 떨어지는 문제가 발생하였으나, **Conversational Agent(HRI 담당)**와 **Robot Control Agent(미션 수행 담당)**로 역할을 분리한 MAS 구조를 통해 응답성과 미션 성공률을 동시에 높였다.

### 2. 로봇 팔 조작 (Robotic Arm)
- **설정:** O3DE 시뮬레이션에서 물체 분류(Sorting), 쌓기(Stacking), 교체(Replacement) 작업을 수행하였다. GPT-4o, Claude 3.5 Sonnet 등 멀티모달 LLM을 추론 엔진으로 사용하였다.
- **결과:** 기본적인 조작은 성공했으나, **공간 추론(Spatial Reasoning)** 및 **자기 수정(Self-correction)** 능력이 부족함을 확인하였다. 예를 들어, 물체를 위에 쌓지 않고 "내부"에 배치하려 하거나, 오류가 있음에도 성공했다고 판단하는 경향이 있었다. 매우 구체적인 단계별 지침(Prompt Engineering)이 있을 때만 성공률이 높아졌다.

### 3. 농업용 트랙터 (Autonomous Tractor)
- **설정:** O3DE orchard 환경에서 예외 상황(장애물 발견) 처리 능력을 테스트하였다.
- **비교 실험:** 텍스트 설명만 제공한 'Language-only embodiment'와 트랙터의 외형 이미지를 제공한 'Visual embodiment'를 비교하였다.
- **결과:** **Visual embodiment**를 적용했을 때 트랙터의 크기와 내구성에 대한 판단력이 크게 향상되었다. 텍스트만 제공했을 때는 작은 나뭇가지를 위험 요소로 오판하여 멈추는 경우가 많았으나, 이미지를 통해 자신의 물리적 특성을 이해한 에이전트는 작은 장애물을 무시하고 주행하는 등 더 정확한 위험 분류 능력을 보였다.

## 🧠 Insights & Discussion

### 강점 및 성과
RAI 프레임워크는 모듈식 구조를 통해 새로운 에이전트와 도구를 최소한의 오버헤드로 추가할 수 있음을 증명하였다. 특히, HRI와 제어 로직을 분리한 MAS 설계가 실제 로봇 운용에서 응답성 문제를 해결하는 효과적인 전략임을 보여주었다. 또한, 시각적 정보(이미지)를 통한 구체화(Embodiment)가 LLM의 물리적 판단 능력을 향상시키는 핵심 요소임을 확인하였다.

### 한계 및 비판적 해석
실험 결과, 최신 멀티모달 LLM들조차 물리적 경계 이해 및 복잡한 공간 추론 작업에서 심각한 한계를 보였다. 이는 현재의 LLM이 시각적 데이터를 픽셀 단위로 처리할 뿐, 물리 세계의 3차원 기하학적 제약 조건을 내재적으로 이해하지 못하고 있음을 시사한다. 또한, MAS 구조에서 에이전트 간의 상태 동기화 문제와 LLM의 일관성 없는 오류 처리 능력이 관찰되었으며, 이는 단순한 프롬프트 개선보다는 규칙 기반의 동기화 메커니즘이나 전용 공간-시간 데이터베이스와 같은 보조 시스템이 필요함을 의미한다.

## 📌 TL;DR

본 논문은 LLM과 로봇 시스템을 유연하게 연결하는 다중 에이전트 프레임워크 **RAI**를 제안한다. `Agents`, `Connectors`, `Tools`라는 추상화 계층을 통해 ROS 2와 같은 로봇 스택과 LLM의 Tool-calling 기능을 통합하였으며, RAG 기반의 구체화(Embodiment) 메커니즘을 통해 로봇의 정체성을 LLM에 주입하였다. 이동 로봇, 로봇 팔, 농업용 트랙터 실험을 통해 MAS 구조의 효율성과 시각적 구체화의 중요성을 입증하였으나, 동시에 LLM의 공간 추론 능력 부족이라는 근본적인 한계를 드러냈다. 이 연구는 향후 LLM 기반 Embodied AI의 확장 가능한 아키텍처 표준을 제시하고, 물리적 상호작용을 위한 보조 시스템 연구의 필요성을 강조한다는 점에서 중요한 의의를 갖는다.
