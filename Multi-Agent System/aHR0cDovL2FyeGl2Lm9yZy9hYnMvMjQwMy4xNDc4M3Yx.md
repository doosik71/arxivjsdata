# Multi-Agent VQA: Exploring Multi-Agent Foundation Models in Zero-Shot Visual Question Answering

Bowen Jiang, Zhijun Zhuang, Shreyas S. Shivakumar, Dan Roth, Camillo J. Taylor (2024)

## 🧩 Problem to Solve

본 논문은 시각적 질의응답(Visual Question Answering, VQA) 작업에서 파운데이션 모델(Foundation Models)의 제로샷(Zero-shot) 능력을 탐구하고 그 한계를 극복하는 것을 목표로 한다.

현재의 거대 시각-언어 모델(Large Vision-Language Models, LVLM)들은 대부분 특정 VQA 데이터셋에 대해 미세 조정(Fine-tuning)을 거쳐 성능을 높이는 방식을 취하고 있다. 그러나 이러한 방식은 제한된 어휘 집합에 의존하게 만들며, 학습 데이터셋을 벗어난 실제 오픈 월드 환경에서의 일반화 능력을 저하시키는 결과를 초래한다. 특히 LVLM은 이미지 내의 국소적인 씬 그래프(Local Scene Graphs)를 해석하거나, 객체의 세부 속성 및 관계를 파악하는 능력, 그리고 객체의 수를 정확히 세는 Counting 작업에서 취약함을 보인다.

따라서 본 연구의 목표는 모델을 특정 데이터셋으로 미세 조정하지 않고도, 특화된 에이전트들을 도구(Tools)로 활용하는 적응형 멀티 에이전트 시스템을 구축하여 제로샷 VQA 성능을 향상시키는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 LVLM의 한계를 보완하기 위해 **특화된 모델들을 에이전트 형태로 구성하여 필요할 때만 호출하는 적응형(Adaptive) 멀티 에이전트 시스템**을 설계한 것이다.

단순히 모든 질문에 대해 복잡한 파이프라인을 적용하는 것이 아니라, 먼저 LVLM이 직접 답을 시도하게 하고, 실패하거나 특정 작업(예: Counting)이 필요하다고 판단될 때만 객체 탐지(Object Detection)나 카운팅(Counting) 전문 에이전트를 호출한다. 이를 통해 추가적인 학습 없이도 파운데이션 모델의 범용성과 전문 모델의 정밀함을 결합하여 제로샷 환경에서 강건한 성능을 확보하고자 하였다.

## 📎 Related Works

기존의 VQA 접근 방식은 BEiT-3나 VLMo와 같이 대규모 데이터셋으로 사전 학습된 모델을 특정 VQA 벤치마크 데이터셋에 맞춰 미세 조정하는 방향으로 발전해 왔다. 이러한 방식은 벤치마크 성능은 높일 수 있으나, 모델이 데이터셋의 특정 정답 분포나 제한된 어휘에 과적합되는 경향이 있어 제로샷 일반화 능력이 매우 떨어진다는 한계가 있다.

본 연구는 이러한 미세 조정 중심의 접근 방식과 차별화되어, 모델 자체를 수정하는 대신 외부 도구(Tools)를 활용하는 멀티 에이전트 프레임워크를 제안한다. 이는 최근 LLM이 외부 API나 도구를 사용하여 능력을 확장하는 Toolformer 등의 흐름을 VQA 영역으로 확장한 것으로 볼 수 있다.

## 🛠️ Methodology

본 시스템은 GPT-4V를 LVLM으로, GPT-3.5를 LLM으로 사용하며, 전체 파이프라인은 다음과 같은 단계로 구성된다.

### 1. Initial Attempt (초기 시도)

먼저 LVLM이 주어진 질문에 직접 답을 시도한다. 이때 프롬프트를 통해 모델이 과잉 확신을 갖지 않도록 유도하며, 만약 이미지에서 핵심 객체를 찾지 못해 답을 낼 수 없다고 판단하면 `[Answer Failed]`라는 특수 토큰을 출력하도록 설계되었다. 성공적으로 답을 냈다면 추가 연산 없이 프로세스를 종료하여 추론 시간을 최적화한다.

### 2. Reattempt by Adaptively Calling Multi-Agents (적응형 재시도)

초기 시도가 실패했을 경우, 다음과 같은 전문 에이전트들이 순차적으로 개입한다.

- **LLM Parsing Agent**: 초기 응답을 분석하여 `[Answer Failed]` 토큰이 발견되면 누락된 객체 이름을 추출한다. 또한 질문이 카운팅 문제인지 판단하여 카운팅 대상 객체를 식별한다.
- **Object Detection Agent**: 누락된 객체가 식별되면, Grounded Segment Everything 모델을 사용하여 이미지 내에서 해당 객체의 마스크와 바운딩 박스(Bounding Box)를 생성한다.
- **LVLM Object Description Agent**: 탐지된 바운딩 박스 영역을 크롭(Crop)하여 개별 LVLM 인스턴스에 전달한다. 각 에이전트는 해당 국소 영역만을 집중적으로 분석하여 질문 해결에 필요한 세부 속성을 텍스트로 묘사한다.
- **LVLM Reattempt**: 최종적으로 LVLM은 원본 이미지와 질문, 이전의 실패 이유, 그리고 객체 묘사 에이전트가 생성한 상세 텍스트 정보를 모두 입력받아 최종 답변을 도출한다. 이는 텍스트 형태의 암시적 씬 그래프를 구축하는 효과를 준다.
- **Object Counting Agent**: 질문이 카운팅 문제이고 예측 숫자가 3보다 큰 경우, 전문 카운팅 모델인 CLIP-Count를 호출하여 밀도 맵(Density Map) 기반의 제로샷 카운팅 결과를 얻고 이를 최종 답안으로 사용한다.

### 3. LLM Answer Grading Agent (정답 채점 에이전트)

제로샷 설정에서는 정답과 완전히 일치하는 텍스트를 찾는 것보다 의미적인 정답 여부가 중요하다. 이를 위해 세 명의 LLM 채점 에이전트가 각 답변을 평가하고, 다수결(Majority Vote) 방식으로 최종 정답 여부를 결정하는 개방형 평가 방식을 도입하였다.

## 📊 Results

### 실험 설정

- **데이터셋**: VQA-v2 (rest-val subset, 5,228쌍), GQA (subset, 1,000샘플).
- **비교 대상**: BEiT-3, VLMo (미세 조정 버전 및 제로샷 버전), BLIP-2, LessIsMore-local.
- **지표**: Accuracy (정확도).

### 주요 결과

- **VQA-v2 결과**: 미세 조정된 BEiT-3-large는 $88.33\%$의 높은 성능을 보였으나, 미세 조정되지 않은 제로샷 버전의 BEiT-3와 VLMo는 거의 $0\%$에 가까운 정확도를 기록하였다. 반면, 제안된 **Multi-Agent VQA는 제로샷 설정에서 $78.02\%$의 정확도**를 달성하며 압도적인 제로샷 일반화 능력을 입증하였다.
- **GQA 결과**: 제로샷 설정에서 Multi-Agent VQA는 $79.70\%$의 정확도를 기록하며, BLIP-2 ($50.40\%$)나 LessIsMore-local ($58.30\%$) 보다 월등한 성능을 보였다.
- **Ablation Study**:
  - 상세한 Chain-of-Thought(CoT) 추론을 제거했을 때 성능이 하락하였다.
  - CLIP-Count 에이전트를 제거하면 특히 숫자 관련 질문(num)의 정확도가 크게 떨어진다.
  - 멀티 에이전트 파이프라인 전체를 제거하고 단일 LVLM만 사용했을 때 전체 정확도가 약 $10\%$ 가량 감소하였다.

## 🧠 Insights & Discussion

본 연구는 LVLM이 가진 잠재력을 극대화하기 위해 모델의 파라미터를 수정하는 대신, 외부의 전문 도구를 적응적으로 활용하는 구조가 제로샷 VQA에서 매우 효과적임을 보여주었다. 특히, 이미지 전체를 한 번에 처리하는 대신 관심 영역을 크롭하여 분석하게 함으로써 LVLM의 시각적 해상도 한계를 극복하고 세부 속성을 정확히 파악하게 한 점이 주효했다.

**한계점 및 비판적 해석:**

1. **추론 효율성**: API 호출에 의존하는 구조로 인해 추론 속도가 느리며, 비용 문제가 발생한다. 또한 API 응답의 비결정론적 특성으로 인해 결과의 일관성이 완벽하지 않을 수 있다.
2. **카운팅 모델의 의존성**: 카운팅 성능이 CLIP-Count라는 외부 모델의 성능에 종속되어 있으며, 제로샷 카운팅 자체의 기술적 한계가 존재한다.
3. **Overthinking 문제**: 일부 실패 사례에서 모델이 너무 상세하게 분석하려다 오히려 불필요한 정보를 생성하고 잘못된 결론에 도달하는 '과잉 사고' 현상이 관찰되었다. 예를 들어, 단순히 해변인지 묻는 질문에 대해 불필요하게 파라솔의 존재 여부까지 분석하다가 오답을 내는 경우가 이에 해당한다.

## 📌 TL;DR

본 논문은 LVLM의 제로샷 VQA 성능 한계를 극복하기 위해, GPT-4V와 전문 도구(Grounded SAM, CLIP-Count)를 결합한 **적응형 멀티 에이전트 시스템**을 제안한다. 미세 조정 없이도 전문 에이전트를 통해 객체 탐지와 카운팅 능력을 보완함으로써, 기존 제로샷 모델 대비 획기적인 성능 향상을 이루었다. 이 연구는 향후 LLM 기반의 시각적 추론 시스템이 단순한 모델 확장이 아닌, 효율적인 도구 활용(Tool-use) 방향으로 나아가야 함을 시사한다.
