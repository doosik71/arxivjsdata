# Learning Reward Machines in Cooperative Multi-Agent Tasks

Leo Ardon, Daniel Furelos-Blanco, Alessandra Russo (2023)

## 🧩 Problem to Solve

본 논문은 협력적 다중 에이전트 강화학습(Cooperative Multi-Agent Reinforcement Learning, CMARL) 환경에서 복잡한 작업 구조를 효율적으로 학습하고 해결하는 문제를 다룬다. 다중 에이전트 환경은 부분 관측 가능성(partial observability), 비정상성(non-stationarity), 그리고 확장성(scalability)과 같은 고유한 도전 과제를 가지고 있으며, 특히 에이전트들이 공통의 목표를 달성하기 위해 정교하게 조정(coordinate)하고 동기화(synchronize)해야 하는 상황에서 학습 난이도가 급격히 증가한다.

기존의 Reward Machine(RM)은 보상 함수의 구조를 유한 상태 기계(finite-state machine)로 인코딩하여 비마르코프(non-Markovian) 보상 문제를 해결하고 정책의 해석 가능성을 높이는 도구로 사용되어 왔다. 그러나 대부분의 RM 연구는 인간이 직접 RM 구조를 설계(handcrafting)하여 제공하는 방식에 의존한다. 인간의 직관에 의존한 RM 설계는 복잡한 작업에서 한계가 있으며, 잘못 설계된 RM은 오히려 학습에 방해가 되는 adversarial한 영향을 줄 수 있다. 그렇다고 해서 전체 시스템의 글로벌 RM(global RM)을 직접 학습시키는 방식은 상태 공간의 증가로 인해 NP-complete 문제에 해당할 만큼 계산 복잡도가 매우 높아 확장성이 떨어진다는 문제가 있다. 따라서 본 논문은 사람이 직접 설계하지 않고도, 글로벌 RM의 복잡성을 피하면서 각 에이전트가 자신의 하위 작업(sub-task)에 맞는 RM을 자율적으로 학습할 수 있는 방법을 제안하는 것을 목표로 한다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 글로벌 작업을 여러 개의 하위 작업으로 분해하고, 각 에이전트가 자신의 하위 작업에 해당하는 **개별 Reward Machine(individual RMs)을 분산된 방식으로 학습**하는 것이다.

구체적으로, 복잡한 글로벌 RM 하나를 학습시키는 대신, 각 에이전트가 관찰하는 레이블 트레이스(label traces)를 기반으로 하위 RM을 유도(induction)하고, 이를 강화학습 정책 학습과 인터리빙(interleaving)하는 구조를 제안한다. 이러한 접근 방식은 학습해야 할 상태 수를 줄여 계산 복잡도를 낮추며, 학습된 RM을 통해 각 에이전트가 수행해야 할 작업의 구조를 심볼릭하게 이해할 수 있어 해석 가능성을 제공한다. 또한, 이렇게 개별적으로 학습된 RM들의 병렬 조합(parallel composition)을 통해 최종적으로 글로벌 협력 작업을 완수하도록 유도한다.

## 📎 Related Works

본 연구는 다음과 같은 관련 연구들을 바탕으로 차별점을 가진다.

- **Reward Machines (RMs):** Toro Icarte 등이 제안한 RM은 보상 함수의 구조를 상태 전이 그래프로 표현하여 non-Markovian 보상을 처리한다. 기존 연구들은 주로 논리 명세(logic specifications)나 시연(demonstrations)으로부터 RM을 유도하거나, 고정된 RM을 기반으로 정책을 학습하는 데 집중했다.
- **CMARL 및 작업 분해:** Neary 등은 글로벌 RM을 개별 에이전트의 RM으로 투영(projection)하여 작업을 분해하고 분산 학습하는 프레임워크를 제안했다. 하지만 이 방식은 글로벌 RM의 구조가 사전에 알려져 있다는 강한 가정을 전제로 한다.
- **QRM (Q-learning for RMs):** RM의 상태별로 $Q$-함수를 학습하여 작업 구조를 활용하는 알고리즘이다. 본 논문은 이 QRM을 확장하여 RM 구조 자체가 학습 과정 중에 동적으로 변하는 상황에서도 정책을 학습할 수 있도록 설계하였다.

본 논문은 Neary 등의 작업 분해 개념을 수용하되, RM을 사전에 제공받는 것이 아니라 **학습 과정에서 자율적으로 유도**한다는 점에서 결정적인 차별점을 가진다.

## 🛠️ Methodology

### 전체 시스템 구조 및 파이프라인

본 논문은 협력적 마르코프 게임(Cooperative Markov Game) 설정을 사용한다. 각 에이전트 $\mathcal{A}_i$는 자신만의 상태 공간 $S_i$, 행동 공간 $A_i$, 전이 함수 $p_i$, 그리고 레이블링 함수 $\ell_i$를 가진다. 전체 시스템은 각 에이전트가 자신의 하위 RM $\mathcal{M}_i$를 학습하고, 이에 대응하는 $Q$-함수 $q_i$를 업데이트하는 프로세스를 반복한다.

### Reward Machine (RM) 정의

RM $\mathcal{M}$은 다음과 같은 튜플로 정의된다: $\mathcal{M} = \langle U, P, u_0, u_f, \delta_u, \delta_r \rangle$

- $U$: RM의 상태 집합
- $P$: 명제(propositions) 집합
- $u_0, u_f$: 초기 상태와 최종 상태
- $\delta_u: U \times 2^P \rightarrow U$: 레이블 관찰 시 상태를 전이시키는 함수
- $\delta_r: U \times U \rightarrow \mathbb{R}$: 상태 전이 시 발생하는 보상 함수

### 학습 절차: Interleaved RM and Policy Learning

본 논문의 핵심 알고리즘은 **RM 유도(Induction)**와 **Q-학습**을 교차적으로 수행하는 것이다.

1. **경험 수집 및 트레이스 저장:** 에이전트는 환경과 상호작용하며 레이블 시퀀스(trace)를 수집한다.
2. **트레이스 분류:** 수집된 트레이스는 다음과 같이 분류되어 저장된다.
    - $\Lambda^\top$ (Goal Traces): 목표 달성에 성공한 트레이스.
    - $\Lambda^\perp$ (Incomplete Traces): 목표 달성에 실패했거나, RM의 최종 상태 $u_f$에 도달했지만 실제로는 목표를 달성하지 못한 counterexample 트레이스.
3. **RM 유도 (LearnRM):** 유도 논리 프로그래밍(Inductive Logic Programming) 시스템인 **ILASP**를 사용하여 $\Lambda^\top$를 포함하고 $\Lambda^\perp$를 배제하는 가장 작은 크기의 RM을 학습한다. 만약 현재 상태 수 $|U|$로 해결책을 찾지 못하면 상태 수를 하나씩 늘려가며 최소 크기의 RM을 찾는다.
4. **정책 업데이트 (QRM):** 학습된 RM의 각 상태 $u \in U$에 대해 독립적인 $Q$-함수 $q_u$를 학습한다. 업데이트 식은 다음과 같다:
   $$q_u(s, a) = q_u(s, a) + \alpha \left( \delta_r(u, u') + \gamma \max_{a'} q_{u'}(s', a') - q_u(s, a) \right)$$
   여기서 $u'$는 현재 상태 $u$에서 레이블 $\ell(s, s')$을 관찰했을 때의 다음 RM 상태이다. RM 구조가 변경되면 기존의 $Q$-함수는 초기화된다.

### 추론 및 동기화

에이전트들은 분산적으로 행동하지만, 여러 에이전트의 명제 집합에 공통으로 포함된 **공유 명제(shared proposition)**가 관찰될 경우, 다른 팀원들이 동일한 명제를 관찰했는지 확인하는 동기화 과정을 거친다.

## 📊 Results

### 실험 설정

- **환경:** $7 \times 7$ 크기의 그리드 월드 환경인 `ThreeButtons`와 `Rendezvous` 작업.
- **비교 대상:**
  - **RM Provided:** 사람이 설계한 RM을 사전에 제공한 경우 (Upper Bound).
  - **RM Learnt:** 본 논문에서 제안한 자율 학습 방식.
  - **No RM:** RM 프레임워크 없이 독립적으로 학습한 경우.
- **지표:** 목표 달성까지의 평균 단계 수(Number of Steps) 및 집단 보상(Collective Reward).

### 주요 결과

1. **ThreeButtons Task:**
    - **RM Learnt** 방식은 **RM Provided**와 거의 유사한 성능을 보였으며, 최종적으로 모든 에이전트가 협력하여 목표를 달성하는 데 성공했다.
    - RM을 사용하지 않은 경우보다 학습 속도가 현저히 빨랐다. 다만, 수렴 후의 최종 단계 수는 RM을 사용하지 않은 경우가 약간 더 짧게 나타났다.
    - 학습된 RM을 분석한 결과, 실제 RM과 매우 유사했으나 목표 달성에 직접적 영향을 주지 않는 일부 전이(transition)가 생략된 최소화된 형태의 RM이 학습되었다.
2. **Rendezvous Task:**
    - 이 작업에서는 **RM Learnt**와 **RM Provided**만이 목표를 달성할 수 있었다.
    - **No RM** 방식은 보상이 0에 머물며 작업을 해결하지 못했다. 이는 복잡한 순차적 협력 작업에서 RM의 구조적 가이드가 필수적임을 시사한다.
3. **글로벌 RM 학습 시도:**
    - 글로벌 RM을 직접 학습시키려 했을 때, 상태 공간의 폭발로 인해 ILASP 시스템이 타임아웃(1시간 제한)되어 실패했다. 이는 개별 RM으로 분해하여 학습하는 방식의 효율성을 입증한다.

## 🧠 Insights & Discussion

### 강점 및 기여

본 연구는 MARL에서 가장 까다로운 부분 중 하나인 "작업 구조의 정의" 문제를 자율 학습으로 해결했다. 특히, 계산적으로 불가능에 가까운 글로벌 RM 학습 대신, 하위 작업으로 분해하여 개별 RM을 학습함으로써 확장성 문제를 해결하고, 동시에 심볼릭한 RM 구조를 통해 에이전트가 무엇을 학습했는지 사람이 확인할 수 있게 했다는 점이 고무적이다.

### 한계 및 가정

- **레이블링 함수의 의존성:** 각 에이전트가 자신의 작업과 동기화에 필요한 레이블링 함수 $\ell_i$를 이미 알고 있다는 가정이 필요하다. 만약 레이블링 함수 자체가 불확실하거나 노이즈가 있다면 시스템이 붕괴될 가능성이 크다.
- **작업 분해의 수동성:** 글로벌 작업을 하위 작업으로 나누는 과정이 아직은 사전에 정의되어 있다. 즉, 어떤 에이전트가 어떤 하위 작업을 맡을지를 사람이 정해줘야 한다.
- **Q-함수 초기화:** RM 구조가 업데이트될 때마다 $Q$-함수를 초기화해야 하므로, RM의 잦은 변경은 학습 효율을 떨어뜨릴 수 있다.

### 비판적 해석

본 논문은 RM의 유용성을 잘 입증했으나, 실험 환경이 그리드 월드라는 비교적 단순한 환경에 국한되어 있다. 실제 복잡한 연속적 상태 공간이나 고차원 관측치 환경에서도 ILASP와 같은 논리 기반 유도 방식이 유효하게 작동할지는 미지수이다. 또한, RM을 통해 학습 속도를 높였음에도 불구하고 `ThreeButtons`에서 No RM 방식이 최종 경로 효율성(steps)이 더 좋았다는 점은, RM이 제공하는 제약 조건이 때로는 최적의 경로 탐색을 방해할 수 있음을 시사한다.

## 📌 TL;DR

본 논문은 협력적 다중 에이전트 강화학습에서 복잡한 작업 구조를 정의하는 Reward Machine(RM)을 사람이 설계하는 대신, **각 에이전트가 자신의 하위 작업에 맞는 RM을 분산적으로 자율 학습**하는 방법을 제안한다. ILASP를 이용한 RM 유도와 QRM 정책 학습을 인터리빙함으로써, 계산 복잡도를 낮추면서도 협력 작업의 성공률과 학습 속도를 크게 향상시켰다. 이 연구는 향후 복잡한 협력 시스템에서 에이전트들에게 구조적인 가이드를 자동으로 제공하는 방향으로 확장될 가능성이 높다.
