# AR-RAG: Autoregressive Retrieval Augmentation for Image Generation

Jingyuan Qi, Zhiyang Xu, Qifan Wang, Lifu Huang (2025)

## 🧩 Problem to Solve

최근의 이미지 생성 모델들은 매우 사실적인 이미지를 생성하는 능력을 보여주고 있으나, 복잡한 구조를 가진 객체나 객체 간의 상호작용, 그리고 학습 데이터에서 비중이 낮은 희귀한 객체를 생성할 때 국소적인 왜곡(local distortions)과 불일치(inconsistencies) 문제가 빈번하게 발생한다. 이를 해결하기 위해 외부의 실제 이미지를 참조하는 Retrieval-Augmented Generation (RAG) 방식이 제안되었으나, 기존의 이미지 RAG 접근법들은 생성 시작 전에 단 한 번의 정적(static)이고 조립도가 낮은(coarse-grained) 이미지 단위 검색을 수행하는 한계가 있다.

이러한 정적 검색 방식은 생성 과정 전반에 걸쳐 고정된 참조 이미지를 사용하므로, 생성 중 변화하는 국소적 필요에 대응하지 못한다. 결과적으로 참조 이미지의 불필요한 세부 사항을 그대로 복제하는 over-copying 문제, 특정 스타일로 편향되는 stylistic bias, 그리고 프롬프트와 무관한 시각적 요소가 나타나는 hallucination 현상이 발생한다. 따라서 본 논문은 생성 단계마다 동적으로 최적의 시각적 참조를 통합하여 이미지의 정교함과 일관성을 높이는 것을 목표로 한다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 이미지 생성 과정을 자기회귀(Autoregressive) 방식으로 수행하면서, 각 패치(patch)를 생성할 때마다 해당 위치에 가장 적합한 시각적 정보를 실시간으로 검색하여 통합하는 **AR-RAG** 프레임워크를 제안한 것이다.

기존 방식이 '텍스트-이미지' 단위의 단일 검색에 의존했다면, AR-RAG는 이미 생성된 주변 패치들을 쿼리(query)로 사용하여 패치 수준(patch-level)의 $k$-nearest neighbor (k-NN) 검색을 수행한다. 이를 통해 모델은 생성 중인 이미지의 맥락에 맞게 필요한 시각적 요소를 정밀하게 가져올 수 있으며, 이미지 전체를 복제하는 것이 아니라 필요한 부분(patch)만 선택적으로 통합함으로써 기존 RAG의 고질적인 문제인 over-copying과 스타일 편향을 효과적으로 억제한다.

## 📎 Related Works

기존의 RAG 연구는 주로 자연어 처리(NLP) 분야에서 지식 집약적인 작업을 위해 발전해 왔으며, 이미지 생성 분야에서는 최근에야 도입되기 시작했다. 기존의 이미지 기반 RAG 접근법(예: RDM, ImageRAG, RA-CM3)들은 주로 다음과 같은 특성을 가진다.

1. **정적 검색:** 생성 시작 전 프롬프트를 기반으로 참조 이미지를 검색하고 이를 전체 생성 과정의 조건(condition)으로 사용한다.
2. **이미지 단위 검색:** 패치가 아닌 이미지 전체를 검색 단위로 삼으므로, 참조 이미지 내의 불필요한 배경이나 객체까지 함께 생성되는 경향이 있다.

AR-RAG는 이러한 기존 방식과 달리 검색 단위를 패치 수준으로 세분화하고, 검색 시점을 생성 프로세스 내부의 각 단계로 옮긴 '자기회귀적 검색'을 도입함으로써 차별성을 가진다.

## 🛠️ Methodology

### 1. Patch-based Retrieval Database Construction

AR-RAG를 위해 먼저 대규모 이미지 데이터셋(CC12M, JourneyDB 등)을 사용하여 패치 기반 데이터베이스를 구축한다.

- **Value:** 각 이미지 패치의 잠재 표현(latent representation) $V_{ij}$를 저장한다.
- **Key:** 해당 패치를 중심으로 하는 $h$-hop 주변 패치들의 벡터를 연결(concatenation)하여 키로 사용한다.
이러한 구조는 생성 시 주변 맥락(surrounding patches)을 쿼리로 사용하여 대상 패치와 가장 유사한 주변 환경을 가진 패치를 효율적으로 찾기 위함이다.

### 2. Distribution-Augmentation in Decoding (DAiD)

DAiD는 별도의 추가 학습이 필요 없는 plug-and-play 방식의 디코딩 전략이다. 모델이 예측한 다음 패치의 확률 분포와 검색된 패치들의 분포를 직접 병합한다.

- **검색 분포 생성:** 검색된 상위 $K$개 패치의 $\ell_2$ 거리를 기반으로 소프트맥스(softmax)를 적용하여 분포 $D_{\text{retrieval}}$을 생성한다.
$$p(\hat{v}_k) = \frac{\exp(-s_k/\tau)}{\sum_{k=1}^K \exp(-s_k/\tau)}$$
여기서 $s_k$는 $\ell_2$ 거리이며, $\tau$는 온도 하이퍼파라미터이다.

- **분포 병합:** 모델의 원래 예측 분포 $D_{\text{model}}$과 검색 분포를 가중 평균하여 최종 분포 $D_{\text{merge}}$를 도출한다.
$$D_{\text{merge}} = (1-\lambda) \cdot D_{\text{model}} + \lambda \cdot D_{\text{retrieval}}$$
최종 토큰 $v_{ij}$는 이 병합된 분포에서 샘플링된다.

### 3. Feature-Augmentation in Decoding (FAiD)

FAiD는 파라미터 효율적 미세 조정(PEFT)을 통해 검색된 정보를 특징(feature) 수준에서 통합하는 방식이다.

- **Multi-Scale Feature Smoothing:** 검색된 패치가 현재 생성 중인 이미지의 주변 맥락과 자연스럽게 어우러지도록 다중 스케일 컨볼루션($2\times2$부터 $Q\times Q$)을 적용한다. 이를 통해 구조적, 스타일적 일관성을 확보한 정제된 특징 $\hat{h}_k$를 생성한다.
$$\hat{h}_k \leftarrow \sum_{q=2}^Q \text{softmax}(\Omega)_q \cdot \hat{h}_k^q$$

- **Feature Augmentation:** 정제된 특징과 현재 모델의 은닉 상태(hidden state)를 적응적으로 블렌딩한다. 각 검색 패치에 대해 학습 가능한 선형 변환을 통해 호환성 점수(compatibility score) $s_k$를 계산하고 이를 반영한다.
$$h_{ij}^{(l+1)} = h_{ij}^l + \Delta h_{ij}^l + \sum_{k=1}^K s_k \hat{h}_k$$
이 과정은 트랜스포머 디코더의 여러 층에 걸쳐 점진적으로 수행되어 특징을 정교화한다.

## 📊 Results

### 실험 설정

- **백본 모델:** Janus-Pro-1B, Show-o
- **벤치마크 및 지표:**
  - **GenEval:** 객체 속성 및 관계 생성 능력 평가
  - **DPG-Bench:** 복잡하고 세밀한 프롬프트 수행 능력 평가
  - **Midjourney-30K:** FID (분포 유사도), CMMD (인간 인지 정렬), FWD (공간/주파수 일관성) 측정

### 주요 결과

1. **정량적 성과:** AR-RAG(특히 FAiD)는 모든 벤치마크에서 기존 비-RAG 모델 및 이미지 수준 RAG 모델보다 우수한 성능을 보였다. Janus-Pro에 FAiD를 적용했을 때 Midjourney-30K에서 FID 6.67을 기록하며 SOTA 성능을 달성했다.
2. **복잡한 프롬프트 해결:** GenEval의 "Two Obj." 및 "Position" 카테고리에서 큰 성능 향상을 보였다. 이는 서로 다른 두 객체가 동시에 등장해야 하는 상황에서, 이미지 수준 RAG는 한쪽 객체에 편향되는 반면, AR-RAG는 패치 단위로 필요한 정보를 동적으로 가져오기 때문에 가능했다.
3. **시각적 품질 개선:** 정성적 분석 결과, 기존 모델들이 겪던 객체 왜곡(예: 동물의 신체 부위 오류)과 불가능한 구조 생성 문제가 AR-RAG를 통해 현저히 감소했다.
4. **추론 효율성:** DAiD는 추론 시간 오버헤드가 0.22%로 거의 무시할 만한 수준이며, FAiD는 약 36.03%의 증가가 있으나 품질 향상 폭에 비해 수용 가능한 수준이다.

## 🧠 Insights & Discussion

본 연구는 이미지 생성에서 RAG를 적용할 때 '언제(When)'와 '어떤 단위로(What level)' 검색하느냐가 성능의 핵심임을 시사한다. 이미지 전체를 한 번에 검색하여 조건으로 주는 방식은 모델에게 너무 강한 제약을 주어 프롬프트 준수 능력을 떨어뜨리고 불필요한 요소를 복제하게 만들지만, 생성 과정 중에 패치 단위로 검색하는 방식은 모델의 생성 유연성을 유지하면서 필요한 시각적 힌트만 제공하는 효과를 준다.

**강점:**

- 학습 없이 적용 가능한 DAiD와 고성능의 FAiD라는 두 가지 옵션을 제공하여 범용성을 높였다.
- 패치 수준의 자기회귀적 검색이 이미지의 전역적 품질(FID)까지 개선시킨다는 점을 입증했다.

**한계 및 비판적 해석:**

- **모델 제약:** 본 방법론은 이산적 토큰(discrete tokens)을 사용하는 자기회귀 모델에 최적화되어 있어, 잠재 공간에서 작동하는 연속적 확산 모델(Diffusion Models)에 직접 적용하기 어렵다.
- **데이터베이스 의존성:** 검색 데이터베이스의 규모와 다양성이 생성 결과의 한계를 결정짓는다. 데이터베이스에 없는 희귀한 패턴의 경우 여전히 생성 능력이 제한될 수 있다.
- **차원 확장:** 현재는 2D 이미지 생성에 국한되어 있으며, 3D 데이터 등으로의 확장 가능성에 대한 실증적 분석은 부족하다.

## 📌 TL;DR

AR-RAG는 이미지 생성 중 각 패치를 만들 때마다 주변 맥락을 기반으로 최적의 패치를 실시간 검색하여 통합하는 새로운 패러다임이다. 학습이 필요 없는 **DAiD**와 미세 조정 방식인 **FAiD**를 통해 기존 이미지 RAG의 고질적 문제인 '과도한 복제(over-copying)'와 '스타일 편향'을 해결하였으며, 특히 복잡한 객체 배치와 세밀한 묘사가 필요한 작업에서 탁월한 성능 향상을 보였다. 이 연구는 향후 고해상도 및 복잡한 구조의 이미지 생성 모델에서 외부 지식을 정밀하게 통합하는 표준적인 방법론이 될 가능성이 높다.
