# Video Enriched Retrieval Augmented Generation Using Aligned Video Captions

Kevin Dela Rosa (2024)

## 🧩 Problem to Solve

본 논문은 대규모 비디오 코퍼스(corpus)에 포함된 정보를 Retrieval Augmented Generation (RAG) 기반의 챗봇 시스템에 효율적으로 통합하는 문제를 해결하고자 한다.

현재 GPT-4 Vision이나 Google Gemini와 같은 멀티모달 대형 언어 모델(Multimodal LLM)이 비디오 이해 능력을 갖추고 있으나, 이를 대규모 데이터셋의 RAG 시스템에 직접 적용하는 데에는 다음과 같은 어려움이 존재한다. 첫째, 비디오 데이터는 크기가 매우 크고 멀티모달 특성을 가지고 있어 공학적인 처리 비용이 높다. 둘째, 비디오 프레임을 직접 샘플링하여 LLM의 컨텍스트 윈도우(context window)에 삽입할 경우, 토큰 소모가 극심하여 효율성이 떨어진다.

따라서 본 연구의 목표는 비디오의 시각적·청각적 정보를 텍스트 형태로 변환한 "Aligned Visual Captions"를 제안함으로써, LLM이 비디오 내용을 효율적으로 추론하고 활용할 수 있는 RAG 프레임워크를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 비디오의 시각적 캡션(Visual Captions)과 자동 음성 인식(ASR) 및 자막(Subtitles) 데이터를 시간축으로 정렬하여 텍스트 기반의 **Aligned Video Caption Transcripts**를 생성하는 것이다.

이 방식의 중심적인 직관은 비디오의 핵심 정보가 텍스트(자막 및 묘사)로 충분히 표현될 수 있으며, 이렇게 변환된 텍스트 데이터는 멀티모달 데이터를 직접 처리하는 것보다 컨텍스트 윈도우를 훨씬 적게 사용하면서도 유사한 수준의 추론 성능을 낼 수 있다는 점이다. 이는 복잡한 비디오 데이터를 LLM이 처리하기 쉬운 텍스트 포맷으로 추상화하여 RAG 시스템의 확장성과 효율성을 극대화하는 설계 방향을 제시한다.

## 📎 Related Works

논문에서는 비디오 이해와 관련된 여러 최신 연구를 언급한다. EgoInstructor와 같이 외부 비디오를 참조하여 캡션을 생성하는 RAG 기반 멀티모달 캡셔닝 모델이나, 텍스트를 사용하여 입력 비디오에 대한 질문에 답하는 연구, 그리고 CLIP 및 BLIP-2와 같은 멀티모달 임베딩을 통한 이미지-텍스트 검색 연구 등이 이에 해당한다.

기존 접근 방식과의 차별점은, 특정 비디오 하나를 깊게 이해하는 것이 아니라 **대규모 비디오 코퍼스 자체를 RAG의 지식 베이스로 활용**하여 챗봇의 응답에 통합하려 한다는 점이다. 기존 연구들이 주로 비디오-텍스트 간의 정렬이나 단일 비디오의 질의응답에 집중했다면, 본 연구는 대규모 비디오 데이터를 텍스트 기반의 색인(index)으로 변환하여 효율적인 검색 및 생성 파이프라인을 구축하는 데 집중한다.

## 🛠️ Methodology

### 1. Aligned Video Captions의 정의 및 구축

Aligned Video Caption Transcripts는 비디오의 각 장면(scene)에 대해 기계 생성된 시각적 캡션과 해당 시간대의 자막 또는 ASR 텍스트를 시간적으로 동기화하여 결합한 데이터이다.

- **데이터 수집:** Panda-70M 데이터셋에서 각 YouTube 카테고리별로 약 2,000개씩, 총 29,259개의 비디오를 샘플링하였다.
- **구성 요소:** 이 데이터셋은 총 150만 개의 비디오 클립과 그에 대응하는 시각적 캡션으로 구성되며, 여기에 YouTube API를 통해 수집한 자막(Subtitles)을 결합하여 정렬된 전사본(Aligned Transcripts)을 생성하였다.

### 2. RAG 시스템 아키텍처

본 논문에서 제안하는 비디오 강화 챗봇의 전체 파이프라인은 다음과 같은 단계로 진행된다.

1. **도구 선택(Tool Selection):** 사용자 쿼리가 입력되면 LLM이 사전에 정의된 도구 설명 중 적절한 리트리버(retriever) 도구를 선택한다.
2. **벡터 검색(Vector Search):** 선택된 쿼리 엔진이 사용자 쿼리를 벡터화하고, 벡터 데이터베이스에서 정렬된 비디오 캡션 텍스트 블록을 검색한다.
3. **응답 생성 및 포맷팅:** 검색된 결과물을 LLM이 해석하여 Pydantic 포맷(예: '방법'에 대한 질문은 불렛 포인트 리스트로, '장소'에 대한 질문은 설명문으로)에 맞춰 요약하고, 검색된 문서에 포함된 타임스탬프(timestamp)를 함께 제공하여 사용자가 실제 비디오의 해당 구간으로 이동할 수 있게 한다.

## 📊 Results

### 1. 비디오 요약 성능 검증 (Sanity Check)

텍스트 기반의 캡션이 멀티모달 LLM과 유사한 정보를 제공하는지 확인하기 위해 GPT-4 Turbo를 기준으로 BERTScore를 통해 요약 성능을 측정하였다.

- **결과:** ASR 텍스트만 사용하거나 시각적 캡션을 사용한 설정이 GPT-4 Vision(비디오 프레임 + ASR)의 결과와 매우 높은 상관관계를 보였다.
- **효율성:** Gemini 1.5 Pro와 같이 프레임당 1초 단위로 샘플링하여 입력하는 방식과 비교했을 때, Aligned Visual Captions를 사용하면 토큰 소모량을 약 69배 줄일 수 있음을 계산하였다.

### 2. 비디오 RAG 검색 성능 실험

일반 지식 질문 1,000개를 대상으로 텍스트 임베딩(text-embedding-3-small)과 멀티모달 임베딩(BLIP-2, CLIP)의 성능을 비교하였다. 측정 지표로는 $\text{HIT@K}$(상위 $K$개 결과 중 정답 포함 여부)와 $\text{QUALITY@1}$(GPT-3.5 turbo가 평가한 정답의 품질, 1-10점)을 사용하였다.

- **정량적 결과:**
  - **Text Embeddings (Aligned Transcript):** $\text{HIT@1} = 0.741, \text{HIT@5} = 0.934, \text{HIT@10} = 0.971, \text{QUALITY@1} = 7.377$
  - **Multimodal Embeddings (CLIP ViT-L/14):** $\text{HIT@1} = 0.553, \text{HIT@5} = 0.859, \text{HIT@10} = 0.926, \text{QUALITY@1} = 5.889$
- **분석:** 텍스트 기반 임베딩, 특히 Aligned Transcript와 ASR을 사용한 방식이 낮은 $K$ 값에서 훨씬 높은 검색 정확도를 보였다. 멀티모달 임베딩은 $K$ 값이 커질수록 성능이 올라가지만, 초기 검색 정밀도는 텍스트 기반 방식보다 낮았다.

## 🧠 Insights & Discussion

### 강점 및 시사점

본 연구는 비디오라는 무거운 데이터를 직접 처리하는 대신, 이를 정교하게 정렬된 텍스트 표현으로 변환함으로써 RAG 시스템의 효율성을 극적으로 높일 수 있음을 입증하였다. 특히 ASR 데이터가 비디오 이해의 핵심적인 역할을 하며, 시각적 캡션이 이를 보완하여 멀티모달 LLM에 근접한 성능을 낼 수 있다는 점이 고무적이다.

### 한계 및 향후 과제

- **캡셔닝 모델 의존성:** 본 시스템의 성능은 초기 단계에서 비디오를 텍스트로 변환하는 캡셔닝 모델의 품질에 전적으로 의존한다. 사용 사례에 맞는 적절한 캡셔닝 모델을 선택하는 것이 중요하다.
- **세그멘테이션 문제:** 비디오에서 의미 있는 클립 구간을 어떻게 나누어 요약할 것인가에 대한 최적의 전략이 아직 명확히 제시되지 않았다.
- **오디오 정보의 제한:** 현재는 음성 언어(spoken word)에만 집중하고 있으나, 음악, 배경 소음, 감정적 톤 등 비언어적 오디오 신호를 통합한다면 더욱 풍부한 표현이 가능할 것이다.

## 📌 TL;DR

본 논문은 비디오의 시각적 캡션과 ASR 자막을 시간축으로 정렬한 **Aligned Video Captions**를 제안하여, 대규모 비디오 코퍼스를 효율적으로 활용하는 RAG 기반 챗봇 시스템을 구축하였다. 실험 결과, 이 텍스트 기반 접근 방식은 멀티모달 LLM 대비 토큰 소모량을 획기적으로 줄이면서도, 검색 정확도($\text{HIT@K}$)와 응답 품질 면에서 멀티모달 임베딩 방식보다 우수한 성능을 보였다. 이는 향후 대규모 비디오 데이터베이스를 활용한 지식 검색 시스템 구현에 있어 매우 실용적인 경로를 제시한다.
