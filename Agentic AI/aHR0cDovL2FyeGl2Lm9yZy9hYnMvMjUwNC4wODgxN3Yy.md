# Exploring utilization of generative AI for research and education in data-driven materials science

Takahiro Misawa, Ai Koizumi, Ryo Tamura, and Kazuyoshi Yoshimi (2025)

## 🧩 Problem to Solve

데이터 기반 재료 과학(Data-driven materials science) 분야에서는 실험 데이터의 획득, 분석, 그리고 최적화를 위해 다양한 소프트웨어 패키지가 사용된다. 그러나 이러한 전문 소프트웨어들은 학습 곡선이 가파르며, 사용자가 기능을 완전히 익히기 위해서는 방대한 양의 매뉴얼을 읽고 숙지해야 하는 시간이 소요된다는 문제점이 있다. 또한, 많은 유용한 도구들이 Python 스크립트 형태로 제공되어, Python 환경 설정에 익숙하지 않은 연구자나 교육 현장의 입문자들이 이를 실제로 활용하는 데 진입 장벽이 존재한다.

본 논문의 목표는 생성형 AI(Generative AI)를 활용하여 재료 과학 연구 및 교육 프로세스에서 발생하는 이러한 효율성 저하 문제를 해결하는 방안을 탐색하는 것이다. 구체적으로는 소프트웨어의 초기 시도(Trial use), 맞춤형 AI 튜터 구축, 그리고 사용자 친화적인 GUI 애플리케이션 개발이라는 세 가지 관점에서 생성형 AI의 효용성을 검증하고자 한다.

## ✨ Key Contributions

본 연구의 핵심 아이디어는 생성형 AI를 단순한 질의응답 도구를 넘어, 전문 소프트웨어의 진입 장벽을 낮추는 '지식 브릿지' 및 '개발 가속기'로 활용하는 것이다. 주요 기여 사항은 다음과 같다.

- **소프트웨어 학습 비용의 획기적 감소**: 매뉴얼 탐색 대신 AI에게 직접 단계별 가이드를 요청함으로써, 생소한 소프트웨어의 핵심 기능을 빠르게 실행하고 검증하는 워크플로우를 제시하였다.
- **도메인 특화 지식 주입을 통한 AI 튜터 구현**: 단순 URL 참조 방식의 한계를 지적하고, 구조화된 PDF 문서를 직접 제공하는 방식(RAG와 유사한 접근)이 AI 응답의 정확도를 유의미하게 향상시킴을 입증하였다.
- **코드-GUI-배포의 자동화 파이프라인**: Python 스크립트를 Windows 실행 파일(.exe)로 변환하고, 이에 GUI를 입히며, 최종 사용자 매뉴얼까지 작성하는 전 과정을 생성형 AI와의 반복적인 상호작용(Iterative Prompting)을 통해 구현함으로써 소프트웨어 배포의 효율성을 극대화하였다.

## 📎 Related Works

논문에서는 GitHub Copilot 및 Amazon CodeWhisperer와 같은 LLM 기반 코드 완성 도구들이 개발자의 생산성을 높이고 코드의 의미론적 이해(Semantics)를 돕고 있다는 점을 언급한다. 이러한 도구들은 코드 작성 및 디버깅 지원을 넘어, 향후 소프트웨어 사양과 매뉴얼을 통합하여 자동으로 튜토리얼을 생성하는 방향으로 발전할 가능성이 있다.

기존의 접근 방식이 주로 연구자가 직접 매뉴얼을 검색하고 학습하는 수동적 방식이었다면, 본 연구는 생성형 AI를 통해 '필요한 기능의 즉각적인 도출'과 '맞춤형 인터페이스의 빠른 구축'이라는 능동적이고 효율적인 방식으로의 전환을 꾀한다는 점에서 차별점을 가진다.

## 🛠️ Methodology

본 연구는 2024년 7월에 개최된 해커톤(AIMHack2024)의 결과물을 바탕으로 하며, 다음 세 가지 주요 방법론을 제안한다.

### 1. AI 지원 소프트웨어 시도 (AI-assisted software trials)
사용자가 특정 소프트웨어(예: ImageJ)의 기능을 알고자 할 때, AI에게 이미지와 함께 구체적인 질문을 던져 단계별 조작 가이드를 얻는 방식이다.
- **절차**: 이미지 업로드 $\rightarrow$ 프롬프트 입력 $\rightarrow$ AI 생성 가이드 수신 $\rightarrow$ 소프트웨어 적용 및 결과 검증.
- **핵심 기능 활용**: 특히 겹쳐진 입자를 분리하여 계수하기 위해 `Watershed` 기능을 찾는 과정을 통해, AI가 매뉴얼 검색 시간을 얼마나 단축할 수 있는지 확인하였다.

### 2. MyGPTs를 이용한 맞춤형 AI 튜터 구축
베이지안 최적화(Bayesian Optimization) 패키지인 PHYSBO를 대상으로, 특정 소프트웨어에 특화된 AI 튜터를 구축하였다.
- **지식 주입 방식 비교**: 
    - **URL 방식**: 공식 GitHub, 논문, 매뉴얼 URL만 제공. 결과적으로 환각(Hallucination) 현상이 발생하여 구현되지 않은 기능(UCB 등)을 구현되었다고 잘못 응답하였다.
    - **PDF 방식**: 공식 문서와 논문을 PDF 파일로 직접 업로드. 결과적으로 정확한 획득 함수(Acquisition Function) 목록을 제시하며 정확도가 크게 향상되었다.
- **튜토리얼 생성**: Keras 기반 신경망의 하이퍼파라미터 최적화 튜토리얼을 생성하고, 실제 실행 가능한 Python 코드를 도출하였다.

### 3. 생성형 AI 기반 GUI 애플리케이션 개발
Python 스크립트를 일반 사용자가 사용할 수 있는 Windows 실행 파일 형태의 GUI 앱으로 변환하는 반복적 프롬프팅 절차를 밟았다.
- **개발 단계**:
    1. **Executable 변환**: `PyInstaller`를 사용하여 `.exe` 파일 생성 가이드 획득.
    2. **인자 추가**: `argparse` 모듈을 사용하여 `num_proposals` 및 `physbo_score`를 입력받도록 수정.
    3. **GUI 구현**: `tkinter` 라이브러리를 도입하여 입력창 및 다이얼로그 구현.
    4. **기능 확장**: `filedialog`를 통한 입력 파일(`candidates.csv`) 선택 기능 및 결과 출력 여부(`output_res`) 설정 기능 추가.
    5. **문서화**: 생성된 앱의 사용자 매뉴얼을 AI가 자동으로 작성.

## 📊 Results

### 1. 소프트웨어 시도 결과
- **ImageJ 활용**: AI가 제시한 단계(Grayscale 변환 $\rightarrow$ Threshold 조정 $\rightarrow$ Binary 변환 $\rightarrow$ Watershed $\rightarrow$ Particle Analysis)를 따라 수행한 결과, 겹쳐진 원형 입자들을 정확하게 분리하고 계수하는 데 성공하였다.

### 2. AI 튜터 정확도 및 성능
- **정확도 개선**: PDF 문서를 제공했을 때, PHYSBO가 지원하는 획득 함수($TS, EI, PI$ 및 다목적 최적화를 위한 $HVPI, EHVI$)를 정확하게 식별하였다.
- **코드 실행 결과**: AI가 생성한 Keras 하이퍼파라미터 최적화 코드를 MNIST 데이터셋에 적용한 결과, 최종 정확도 $0.9742$를 달성하며 기능적으로 올바른 튜토리얼임을 입증하였다.

### 3. GUI 애플리케이션 구현 결과
- **결과물**: `PHYSBO.exe`라는 단일 실행 파일이 성공적으로 생성되었으며, Python 환경이 없는 윈도우 PC에서도 동작함을 확인하였다.
- **사용성**: 파일 선택 $\rightarrow$ 파라미터 입력 $\rightarrow$ 결과 도출로 이어지는 직관적인 GUI 워크플로우가 완성되었으며, AI가 작성한 매뉴얼이 수정 없이 사용 가능할 정도로 높은 완성도를 보였다.

## 🧠 Insights & Discussion

본 연구는 생성형 AI가 재료 과학의 전형적인 연구 워크플로우(실험 데이터 획득 $\rightarrow$ 최적화 $\rightarrow$ 도구 활용) 전반에 걸쳐 실질적인 도움을 줄 수 있음을 보여준다. 

- **강점**: AI는 특히 '도구의 사용법'이라는 기술적 장벽을 낮추는 데 매우 강력한 성능을 발휘한다. 또한, 구조화된 문서(PDF)를 제공함으로써 도메인 특화 지식의 정확도를 높일 수 있다는 점은 향후 AI 튜터 개발의 핵심 전략이 될 수 있다.
- **한계 및 주의점**: AI가 항상 정확한 답을 제공하지는 않으므로, 반드시 인간의 검증(Human verification)과 실제 테스트 과정이 수반되어야 한다. URL 기반 참조보다 직접적인 문서 업로드가 더 효과적이라는 점은 LLM의 정보 검색 메커니즘에 대한 비판적 이해가 필요함을 시사한다.
- **해석**: 본 논문은 단순한 기술 보고서를 넘어, AI가 연구자의 보조 도구로서 어떻게 통합될 수 있는지에 대한 초기 기록으로서 가치가 있다. 특히 GUI 개발 사례는 전문 개발자가 아닌 연구자들도 AI를 통해 자신의 도구를 배포 가능한 형태로 빠르게 전환할 수 있음을 시사한다.

## 📌 TL;DR

본 논문은 생성형 AI를 활용하여 데이터 기반 재료 과학 연구의 진입 장벽을 낮추는 세 가지 방안(AI 기반 소프트웨어 학습, PDF 기반 맞춤형 AI 튜터 구축, Python 스크립트의 GUI 앱 변환)을 제시하고 실증하였다. 특히 구조화된 지식 제공이 AI의 정확도를 높이며, 반복적 프롬프팅을 통해 소프트웨어 개발 및 배포 과정을 자동화할 수 있음을 보여주었다. 이 연구는 향후 재료 과학 분야의 연구 효율성 증대와 교육 민주화에 중요한 역할을 할 가능성이 높다.
