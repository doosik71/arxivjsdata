# MM-DeepResearch: A Simple and Effective Multimodal Agentic Search Baseline

Huanjin Yao et al. (2026)

## 🧩 Problem to Solve

본 논문은 복잡한 심층 연구(Deep Research) 작업을 수행할 수 있는 멀티모달 리서치 에이전트를 개발하는 것을 목표로 한다. 기존의 Multimodal Large Language Models(MLLMs)는 모델 내부의 고정된 지식에 의존하기 때문에, 학습 데이터 범위를 벗어난 오픈 월드(Open-world)나 정보 집약적인 질문에 답변하는 데 한계가 있다. 

이를 해결하기 위해 기존에는 Retrieval-Augmented Generation(RAG)나 프롬프트 기반의 워크플로우 방식이 사용되었으나, 이러한 방식들은 검색(Retrieval)과 추론(Reasoning) 과정이 분리되어 있어, 모델의 추론 상태에 따라 검색 전략을 동적으로 수정하는 능력이 부족하다는 문제가 있다.

저자들은 진정한 에이전트 기반 검색(Agentic Search) 능력을 갖춘 모델을 구축하기 위해 다음과 같은 세 가지 주요 난제를 제시한다.
1. **검색 집약적 멀티모달 QA 데이터의 부족**: 다회차 검색과 다양한 도구 호출이 필요한 고품질 데이터셋이 매우 희소하다.
2. **효과적인 검색 궤적(Search Trajectory)의 부재**: 기존의 프롬프트 기반 합성 방식은 단발성 검색에 치중되어 있어, 반복적인 추론이 필요한 다회차 시나리오를 생성하기 어렵다.
3. **온라인 검색 API 사용의 막대한 비용**: 강화학습(RL) 과정에서 수많은 롤아웃(Rollout)이 발생하는데, 이때 온라인 API(예: SerpAPI, Jina)를 호출하면 훈련 1회당 수천 달러의 비용이 발생하여 체계적인 실험이 어렵다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 데이터 생성, 궤적 합성, 그리고 학습 환경 구축이라는 세 가지 측면에서 효율적인 파이프라인을 설계하여 비용 효율적이면서도 강력한 멀티모달 검색 에이전트를 구축하는 것이다.

1. **Hyper-Search**: 하이퍼그래프(Hypergraph) 구조를 도입하여 시각적·텍스트적 노드를 연결함으로써, 반드시 다회차 검색과 다양한 도구 사용이 필요한 '검색 집약적' 멀티모달 QA 쌍을 생성한다.
2. **DR-TTS (Decompose–Recompose Tool Tree Search)**: 검색 작업을 도구별로 분해하여 전문 전문가(Expert) 모델을 먼저 학습시킨 뒤, 이들을 다시 결합해 트리 탐색(Tree Search)을 수행함으로써 고품질의 검색 궤적을 합성한다.
3. **오프라인 멀티모달 검색 엔진**: 학습 단계에서 비용이 많이 드는 온라인 API 대신 사용할 수 있는 오프라인 검색 엔진을 구축하여, 비용 부담 없이 에이전트 강화학습을 수행할 수 있게 한다.

## 📎 Related Works

기존의 검색 에이전트 연구는 크게 두 가지 방향으로 진행되었다.
- **워크플로우 기반 검색 에이전트**: RAG 기반이나 프롬프트 기반의 정적인 파이프라인을 사용한다. 하지만 이러한 방식은 검색과 추론이 분리되어 있어 유연성이 떨어지며, 검색량이 너무 적거나 과도하게 많아지는 문제가 발생하여 일반화 성능이 낮다.
- **멀티모달 딥 리서치 에이전트**: 최근 MMSearch-R1이나 WebWatcher와 같이 에이전트 능력을 부여하려는 시도가 있었으나, 대부분 검색 집약적 데이터셋이나 궤적 데이터를 공개하지 않았으며, 온라인 API에 과도하게 의존하여 개발 진입 장벽이 높다는 한계가 있다.

본 연구는 하이퍼그래프 기반의 데이터 생성과 전문가 모델의 분해-결합 방식을 통해 이러한 데이터 부족 문제를 해결하고, 오프라인 엔진으로 학습 효율성을 극대화했다는 점에서 차별점을 가진다.

## 🛠️ Methodology

### 1. Hyper-Search: QA 데이터 생성
Hyper-Search는 웹의 이미지와 텍스트 정보를 하이퍼그래프로 모델링하여 복잡한 QA 데이터를 생성한다.

- **하이퍼그래프 구축**: 이미지 노드 $I$와 텍스트 노드 $T$로 구성된다. 이미지 노드에서는 역이미지 검색(Reverse Search)과 시각적 유사 검색(Visual Search)을 통해 노드를 확장하며, 텍스트 노드에서는 MLLM을 통해 관련 URL과 이미지 링크를 추출하여 노드를 확장한다. 부모 노드와 확장된 자식 노드들을 하나의 하이퍼엣지(Hyperedge)로 묶어 모달리티 간의 관계를 캡처한다.
- **QA 생성**: 
    - **Level 1 (Intra-hyperedge)**: 단일 하이퍼엣지 내에서 정보를 결합해 질문을 생성한다.
    - **Level 2 (Inter-hyperedge)**: 여러 하이퍼엣지에 걸쳐 정보를 통합하여, 더 깊은 탐색과 다회차 검색이 필요한 고난도 질문을 생성한다.
- **필터링**: GPT-5 및 Seed 모델을 사용하여 시각적으로 무관하거나 검색 없이 풀 수 있는 질문을 제거하여 최종적으로 3K의 Hyper-Search-3K 데이터셋을 구축한다.

### 2. DR-TTS: 검색 궤적 합성
단일 모델로 궤적을 생성하면 도구 사용 편향이 발생하므로, '분해 후 재결합' 전략을 사용한다.

- **도구 정의**: 정보 기반 검색 도구(T2T, T2I, I2I)와 지식 기반 검색 도구(전문 모델을 통한 T2T)로 구분한다.
- **분해 학습 (Decomposed Training)**: 각 도구별로 특화된 전문가 모델을 RL을 통해 각각 학습시킨다.
- **재결합 트리 탐색 (Recomposed Tree Search)**: 학습된 전문가들을 결합하여 트리 구조로 탐색을 수행한다. 각 노드에서 서로 다른 전문가가 생성한 경로를 확장하며, 최종 답변이 정답인 경로만을 추출하여 SFT를 위한 궤적 데이터 $\tau = \{t_1, c_1, r_1, \dots, t_n, a_n\}$를 생성한다. (여기서 $t$: 추론 단계, $c$: 도구 호출, $r$: 도구 응답, $a$: 최종 답변)

### 3. MM-DeepResearch 학습 절차
학습은 두 단계로 진행된다.

**단계 1: SFT (Cold Start)**
DR-TTS로 합성한 궤적 데이터를 사용하여 모델이 도구 호출 패턴을 익히게 한다. 손실 함수는 다음과 같다.
$$\mathcal{L}_{SFT} = -\mathbb{E}_{\tau \sim \mathcal{D}} \sum_{i=1}^{n} \log p_{\theta}(t_i, c_i, a_n | Q, t_{<i}, c_{<i}, r_{<i}})$$
이때 도구의 응답($r_i$)은 정답지(Observation)로 처리하여 손실 계산에서 제외한다.

**단계 2: 오프라인 엔진 기반 RL**
비용 절감을 위해 대규모 멀티모달 코퍼스를 구축한 오프라인 검색 엔진을 사용하며, GRPO(Group Relative Policy Optimization) 알고리즘을 적용한다.
- **보상 함수 (Reward)**: 최종 답변의 정확도($\mathcal{R}_{acc}$)와 도구 호출 형식의 준수 여부($\mathcal{R}_{format}$)를 결합하여 계산한다.
$$\mathcal{R} = \alpha \mathcal{R}_{acc} + (1-\alpha) \mathcal{R}_{format}$$
- **최적화**: GRPO를 통해 그룹 내 상대적 이점(Advantage) $A_g$를 계산하고, KL 페널티가 포함된 목적 함수 $J_{GRPO}(\theta)$를 최적화하여 모델을 업데이트한다.

## 📊 Results

### 실험 설정
- **데이터셋**: SimpleVQA, MMSearch, LiveVQA, FVQA-test, InfoSeek, Browsecomp-VL 등 6개 벤치마크 사용.
- **비교 대상**: GPT-4o, GPT-5, Qwen3-VL (8B, 32B) 및 기존 에이전트 모델(MMSearch-R1, WebWatcher 등).

### 주요 결과
- **성능 우위**: MM-DeepResearch-8B는 베이스라인인 Qwen3-VL-8B 대비 평균 17% 성능 향상을 보였으며, 유사 규모의 SOTA 모델인 SenseNova-MARS-8B보다도 높은 성능을 기록했다.
- **확장성**: 32B 모델로 확장했을 때 Qwen3-VL-32B 대비 14.9%의 성능 향상을 보이며, 모든 벤치마크에서 매우 강력한 성능을 입증했다.
- **데이터 효과**: Hyper-Search로 생성한 데이터를 사용했을 때 평균 도구 호출 횟수가 증가하며 더 깊은 탐색이 이루어짐을 확인했다 (Table 2).
- **구성 요소 기여도**: SFT 단독보다 RL을 병행했을 때 성능이 비약적으로 상승하며, 특히 정보 기반 도구(T2T, T2I, I2T)와 지식 기반 도구가 상호 보완적으로 작용함을 확인했다 (Table 3, 4).

### 비용 및 효율성
오프라인 검색 엔진은 온라인 API 대비 훈련 비용을 거의 0으로 낮추었으며, 샘플당 응답 시간을 60초에서 1초로 획기적으로 단축했다.

## 🧠 Insights & Discussion

본 논문은 멀티모달 에이전트 학습에서 데이터의 질과 학습 비용이라는 두 가지 핵심 난제를 영리하게 해결했다. 특히 하이퍼그래프를 통해 '검색이 필수적인' 상황을 인위적으로 설계하여 데이터의 희소성 문제를 해결한 점과, 전문가 모델을 분해하여 학습시킨 후 트리 탐색으로 궤적을 추출한 DR-TTS 방식은 매우 효율적인 접근법이다.

또한, 오프라인 검색 엔진이 온라인 엔진보다 절대적인 성능은 약간 낮지만(Table 6), RL 학습 과정에서 모델이 '검색 행동' 자체를 학습하도록 유도하는 데에는 충분하다는 것을 입증함으로써, 현실적인 학습 프레임워크를 제시했다.

다만, 테스트 단계에서는 다시 온라인 엔진으로 전환하며 LLM을 통해 응답을 요약하는 보조 프로세스를 도입했는데, 이는 오프라인 엔진과 온라인 엔진 사이의 데이터 분포 차이(redundancy 등)를 메우기 위한 조치로 보인다.

## 📌 TL;DR

MM-DeepResearch는 하이퍼그래프 기반의 QA 생성(Hyper-Search), 전문가 모델 분해-결합 기반의 궤적 합성(DR-TTS), 그리고 비용 효율적인 오프라인 검색 엔진을 통해 구축된 강력한 멀티모달 리서치 에이전트이다. 이 연구는 고가의 API 비용 없이도 고품질의 검색-추론 궤적을 생성하고 학습시킬 수 있는 베이스라인을 제공하며, 기존의 정적인 RAG 방식을 넘어선 동적인 에이전트 기반 검색의 가능성을 보여주었다.
