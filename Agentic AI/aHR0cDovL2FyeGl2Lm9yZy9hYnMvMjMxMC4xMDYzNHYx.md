# OpenAgents: AN OPEN PLATFORM FOR LANGUAGE AGENTS IN THE WILD

Tianbao Xie, Fan Zhou, Zhoujun Cheng, Peng Shi, Luoxuan Weng, Yitao Liu, Toh Jing Hua, Junning Zhao, Qian Liu, Che Liu, Leo Z. Liu, Yiheng Xu, Hongjin Su, Dongchan Shin, Caiming Xiong, Tao Yu

## 🧩 Problem to Solve

현재의 언어 에이전트 프레임워크는 주로 개념 증명(proof-of-concept) 구현과 개발자 중심의 콘솔 인터페이스에 초점을 맞춰 일반 사용자의 접근성을 제한하고 애플리케이션 수준의 설계를 간과합니다. 기존 벤치마크는 대부분 통제된 환경에서 에이전트를 평가하여 실제 생활에서 발생하는 다양한 사용자 요구와 예측 불가능한 환경을 반영하지 못합니다. 또한, OpenAI의 성공적인 에이전트 제품(Advanced Data Analysis, Plugins, Browse with Bing)은 비공개 소스로 제공되어 개발자와 연구자들이 이를 기반으로 추가 탐색, 평가 및 개선을 하는 데 한계가 있습니다.

## ✨ Key Contributions

* **오픈 플랫폼 `OpenAgents` 구축:** 일반 사용자가 언어 에이전트를 사용하고 호스팅할 수 있도록 설계된 오픈 플랫폼을 제시합니다.
* **세 가지 핵심 에이전트 제공:**
  * **Data Agent:** Python/SQL 및 다양한 데이터 도구를 활용한 데이터 분석 기능을 제공합니다.
  * **Plugins Agent:** 200개 이상의 일상 API 도구를 통합하여 광범위한 작업을 수행합니다.
  * **Web Agent:** 자율적인 웹 브라우징 기능을 통해 웹 기반 작업을 처리합니다.
* **사용자 친화적 웹 UI:** 일반 사용자가 코딩 지식 없이도 에이전트 기능과 상호작용할 수 있도록 최적화된 웹 사용자 인터페이스를 제공하며, 빠른 응답과 효율적인 오류 처리를 지원합니다.
* **개발자/연구자를 위한 쉬운 배포:** 개발자와 연구자가 로컬 환경에서 프론트엔드와 백엔드를 쉽게 배포할 수 있는 코드를 제공하여 새로운 언어 에이전트 개발 및 실세계 평가를 위한 견고한 기반을 마련합니다.
* **실세계 평가 환경:** 실제 사용자 요구에 따라 에이전트가 작업을 수행하고, 인간-에이전트 상호작용 추적 및 사용자 피드백을 기록하여 "in-the-wild" 환경에서의 포괄적인 평가를 가능하게 합니다.
* **실제 구현 과제 해결:** 실시간 응답 스트리밍, 적응형 데이터 매핑, 전략적 데이터 저장, 시스템 견고성(오류 처리, 토큰 오버플로우), 자동 도구 선택 및 확장, 안전한 실행 환경(샌드박스) 구축 등 실제 배포 시 발생하는 다양한 기술적 도전 과제에 대한 실용적인 해결책을 제시합니다.

## 📎 Related Works

* **전통적인 지능형 에이전트:** Wooldridge & Jennings (1995), Sutton & Barto (2005), Russell (2010) 등.
* **LLM 기반 언어 에이전트 연구:** Yao et al. (2022b)의 ReAct, Yang et al. (2023a), Wang et al. (2023a) 등 LLM을 활용한 에이전트 개념.
* **기존 에이전트 프레임워크:** LangChain (Chase, 2022), AutoGPT (Gravitas, 2023), BabyAGI (Nakajima, 2023), Gentopia (Xu et al., 2023a), AgentVerse (Chen et al., 2023), Agents (Zhou et al., 2023c) 등 개념 증명 또는 개발자 중심의 구현체들.
* **에이전트 벤치마크:** 코딩(Yang et al., 2023a), 도구 활용(Li et al., 2023c; Patil et al., 2023; Qin et al., 2023b), 웹 브라우징(Shi et al., 2017; Yao et al., 2022a; Deng et al., 2023; Zhou et al., 2023b) 또는 이들의 조합(Liu et al., 2023)을 위한 제어된 환경에서의 평가.
* **멀티 에이전트 연구:** Camel (Li et al., 2023b), Generative Agents (Park et al., 2023), MetaGPT (Hong et al., 2023) 등.
* **상용 LLM 기반 에이전트:** OpenAI의 ChatGPT Plus (Advanced Data Analysis, Plugins, Browse with Bing)와 같은 클로즈드 소스 솔루션.

## 🛠️ Methodology

`OpenAgents`의 아키텍처는 크게 **사용자 인터페이스** (프론트엔드 및 백엔드)와 **언어 에이전트** (언어 모델, 도구 및 환경)의 두 부분으로 나뉩니다.

1. **시스템 설계 (System Design)**
    * **사용자 인터페이스:** 사용자 친화적인 UI 구축에 중점을 두어 백엔드 서버 운영, 오류 처리, 데이터 스트리밍 등 재사용 가능한 비즈니스 로직을 견고하게 지원합니다.
    * **언어 에이전트:** 언어 모델, 도구 인터페이스, 환경으로 구성되며, ReAct (Yao et al., 2022b) 접근 방식을 기반으로 `Observation -> Deliberation -> Action`의 순차적 상호작용 과정을 따릅니다.

2. **실질적인 구현 과제 (Practical Implementation Challenges)**
    * **사용자 인터페이스 구현:**
        * **적응형 데이터 매핑 (Adaptive Data Mapping):** `DataModel` 개념을 도입하여 텍스트, 코드, 이미지, 테이블 등 다양한 원시 데이터를 인간, 프론트엔드, 컴퓨팅 시스템, LLM 등 여러 수신자에 최적화된 형식으로 변환합니다.
        * **전략적 데이터 저장 (Strategic Data Storage):** 다중 사용자 환경을 고려하여 임시 변수는 메모리에, 전역 변수는 Redis (Carlson, 2013)에, 사용자 대화 데이터는 MongoDB (Banker et al., 2016)에 저장합니다.
        * **사용자 중심 인터페이스 (User-Centric Interface):** 데이터 에이전트의 경우 코드 스니펫, 콘솔 출력, 대화형 시각화를 통합하고, 플러그인 에이전트는 카드 형태로 정보(예: 웹사이트, 제품)를 표시하며, 웹 에이전트는 Chrome 확장 프로그램을 통해 브라우저를 직접 제어하고 실행 계획을 시각화합니다.
        * **실시간 응답 스트리밍 (Real-time Response Streaming):** Push-down automata (Autebert et al., 1997)에 기반한 스트리밍 방식을 채택하여 LLM의 긴 응답 생성 시 발생하는 지연을 줄이고 실시간으로 토큰을 파싱하여 사용자에게 즉각적인 피드백을 제공합니다. 이는 단순한 채팅 모델($ChatML$)을 넘어 언어 에이전트의 다양한 토큰 역할(tool call, tool parameter, displayed response)을 식별하는 데 중요합니다.
        * **시스템 견고성 (System Robustness):**
            * **오류 처리 (Failure Handling):** LLM API 및 플러그인 API 호출 실패와 같은 예측 불가능한 오류 유형을 식별하고 재시도 또는 오류 메시지를 통해 처리합니다.
            * **실시간 응답 (In-time Response):** LLM 키 풀을 활용하여 API 호출 오버헤드를 완화하고, 사용자에게 생성 중단 및 재시도 옵션을 제공하며, 원격 API 호출에 최대 대기 시간을 설정하여 사용자 경험을 개선합니다.
            * **토큰 오버플로우 관리 (Token Overflow):** `MessageDataModel` 서브클래스를 구현하여 메시지 기록 토큰을 명시적으로 자르고 필요한 정보(예: 액션, 도구 응답)를 추출합니다.
        * **Chrome 확장 프로그램을 통한 브라우저 제어 (Browser Control via Chrome Extension):** 웹 에이전트가 Chrome debugger API를 통해 사용자 브라우저를 직접 제어할 수 있도록 Chrome 확장 프로그램을 개발하여 실시간 사용자 모니터링 및 개입을 가능하게 합니다.
    * **언어 에이전트 구현:**
        * **데이터 근거 (Data Grounding):** 사용자 업로드 파일(Excel, CSV, JSON 등)을 에이전트가 코드를 작성하여 처리할 수 있도록 `DataModel`을 통해 데이터를 선형화하고 파일 이름으로 색인화하여 활용합니다.
        * **자동 도구 선택 (Automatic Tool Selection):** 텍스트 임베딩 기술인 Instructor (Su et al., 2023)를 사용하여 사용자 지침에 가장 적합한 플러그인을 자동으로 감지하고 선택합니다.
        * **자동 도구 확장 (Automatic Tool Scaling):** RapidAPI 및 OpenAI 플러그인 스토어에서 200개 이상의 고품질 플러그인 정보를 소싱하고 LLM 접근성을 테스트합니다.
        * **실행 가능한 환경 (Executable Environments):**
            * **코드 실행 환경:** SQL 쿼리 실행을 위한 SQL 엔진과 Python 코드 실행을 위한 안전하고 격리된 Docker 샌드박스 환경을 제공합니다 (Kaggle Dockerfile 참조).
            * **API 호출 환경:** RESTful 디자인 (Fielding, 2000) 원칙을 따르고, `DataModel`을 통해 문맥 길이 초과와 같은 예외 조건을 관리하며, 프롬프팅 LLM으로 구동되는 파싱 및 디스패칭 모듈이 API 호출을 조정합니다.
            * **웹 환경:** 자체 구축한 Chrome 확장 프로그램을 통해 웹 에이전트가 실제 브라우저(Chrome)를 제어하여 웹 탐색 및 조작 작업을 수행합니다.

## 📊 Results

`OpenAgents`는 다음 세 가지 주요 에이전트를 성공적으로 구현하고 시연했습니다.

* **Data Agent:**
  * 사용자가 테이블 및 이미지와 같은 사용자 지정 파일을 업로드하고 Python 및 SQL을 사용하여 데이터를 쿼리, 시각화 및 조작할 수 있습니다.
  * Kaggle 데이터 검색, 데이터 프로파일링, 대화형 ECharts 플로팅과 같은 통합된 데이터 도구를 활용하여 다양한 데이터 관련 작업을 처리합니다.
  * LLM을 도구 자체에 내장하여 Python, SQL, ECharts와 같은 도구가 코드를 생성하게 함으로써 LLM의 프로그래밍 능력을 최대한 활용하고 에이전트의 부담을 경감시킵니다.
* **Plugins Agent:**
  * Google Search, Wolfram Alpha, Zapier, Klarna 등 200개 이상의 플러그인을 통합하여 쇼핑, 검색, 뉴스 읽기, 날씨 예측, 웹사이트 생성 등 광범위한 일상 업무를 처리합니다.
  * 사용자가 플러그인을 수동으로 선택할 필요 없이, 지침에 따라 가장 관련성 높은 플러그인을 자동으로 선택하는 기능을 제공합니다.
* **Web Agent:**
  * (Chat Agent와 연동하여) 자율적인 웹 브라우징 기능을 제공합니다. 복잡한 사용자 작업을 더 작은 하위 작업으로 분해하여 웹 에이전트가 순차적으로 처리하도록 합니다.
  * Chrome 확장 프로그램을 통해 사용자 브라우저를 직접 제어하여 항공편 검색, IMDb 댓글 요약, Google 지도 탐색, Twitter 게시물 작성, Google Form 작성 지원 등 다양한 웹 기반 작업을 수행합니다.
  * 사용자는 에이전트의 실행 계획과 단계를 실시간으로 모니터링하고 필요시 개입할 수 있습니다.

이러한 결과는 `OpenAgents`가 개발자가 아닌 일반 사용자도 웹 UI를 통해 에이전트와 효과적으로 상호작용할 수 있으며, 연구자들이 실세계 시나리오에서 언어 에이전트를 개발하고 평가할 수 있는 실용적이고 견고한 플랫폼을 제공함을 입증합니다. 또한, GPT-4, Claude와 같은 오픈 언어 API를 기반으로 구현되어 클로즈드 소스 모델에 대한 대안을 제시합니다.

## 🧠 Insights & Discussion

* **LLM 프롬프팅의 도전 과제:** 실제 사용자용 애플리케이션 구축 시 백엔드 로직, 출력 미관, 보안 등을 위한 프롬프트 명령이 수백 토큰에 달하여 LLM의 토큰 제한 및 컨텍스트 처리 능력을 초과할 수 있습니다. 이는 LLM의 부적절한 출력 형식으로 이어져 사용자 경험을 저해하며, 특정 도메인 및 요구사항에 특화된 에이전트 모델 훈련의 필요성을 시사합니다.
* **제어 불가능한 실세계 요인:** 실세계 배포 시 API 서버 충돌, 사용자 불만족으로 인한 LLM 생성 중단, CAPTCHA 및 웹 페이지 광고 변경 등 예측 불가능한 요인들이 발생합니다. 기존 연구들은 이러한 요소들을 충분히 모델링하지 못했으며, 이는 더욱 현실적인 모델링 접근 방식의 필요성을 강조합니다.
* **실세계 시나리오에서의 추가 지표:** 연구는 주로 성능 지표에 초점을 맞추지만, 실세계에서는 스트리밍(응답 속도), 미려한 출력 형식 등 사용자 경험에 필수적인 요소들이 간과됩니다. 성능과 사용자 경험 간의 트레이드오프를 고려한 연구가 중요합니다.
* **시스템 문제로 인한 평가 복잡성:** LLM 기반 애플리케이션은 복잡한 시스템 체인을 가지므로, 실패의 원인이 LLM의 한계인지 애플리케이션 로직의 결함인지 구별하기 어렵습니다. 지능형 에이전트의 설계 및 운영 논리를 간소화하여 평가 복잡성을 줄이는 것이 중요합니다.
* **향후 연구 방향:** `OpenAgents`는 맞춤형 대화 시스템, 멀티모달 상호작용, 자동화된 워크플로우 통합과 같은 혁신적인 에이전트 애플리케이션을 위한 기반을 제공합니다. 또한, 더 다양한 API 소스, 대규모 멀티모달 모델 통합, 새로운 UI 디자인 등 도구 및 구성 요소 통합이 가능하며, 인간-LM 상호작용 및 적응형 UI 생성 연구에도 기여할 수 있습니다. 마지막으로, 제어된 환경을 넘어 실제 환경에서 LLM의 능력과 성능을 정확하게 평가하기 위한 "in-the-wild" 평가 방법론 개발에 중요한 플랫폼이 될 것입니다.

## 📌 TL;DR

`OpenAgents`는 기존 LLM 에이전트 프레임워크가 간과했던 일반 사용자의 접근성과 실제 애플리케이션 배포 문제를 해결하기 위한 오픈 플랫폼입니다. 데이터 분석을 위한 `Data Agent`, 200개 이상의 API 도구를 활용하는 `Plugins Agent`, 자율 웹 브라우징을 위한 `Web Agent`의 세 가지 에이전트를 제공합니다. 사용자 친화적인 웹 UI와 개발자를 위한 쉬운 배포를 지원하며, 실시간 스트리밍, 오류 처리, 자동 도구 선택/확장, 안전한 실행 환경 등 실제 배포 시 발생하는 다양한 기술적 도전 과제에 대한 실용적인 해결책을 제시합니다. `OpenAgents`는 일반 사용자가 LLM 에이전트의 잠재력을 활용하고, 연구자들이 실세계 시나리오에서 에이전트를 개발하고 평가할 수 있는 견고한 기반을 제공하며, LLM 프롬프팅의 한계와 실세계 환경의 예측 불가능한 요소, 사용자 경험의 중요성을 강조합니다.
