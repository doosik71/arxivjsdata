# STOP WASTING YOUR TOKENS: TOWARDS EFFICIENT RUNTIME MULTI-AGENT SYSTEMS

Fulin Lin, Shaowen Chen, Ruishan Fang, Hongwei Wang, Tao Lin (2026)

## 🧩 Problem to Solve

최근 거대 언어 모델(LLM)의 발전으로 복잡한 작업을 수행할 수 있는 Multi-Agent Systems(MAS)의 능력이 크게 향상되었으나, 시스템의 자율성과 복잡성이 증가함에 따라 심각한 비효율성과 취약성이 함께 나타나는 역설적인 상황이 발생하고 있다.

첫째, **운영 효율성 저하(Economic Inefficiency)** 문제이다. 에이전트가 웹 페이지나 도구 출력물과 같은 방대한 관찰 데이터(Observation)를 처리할 때 컨텍스트 윈도우가 불필요하게 팽창하여 Token 소비량이 급증하며, 이는 비용 증가뿐만 아니라 중요한 정보가 희석되어 작업 수행 능력을 떨어뜨리는 원인이 된다. 또한, 에이전트가 최적이지 않은 전략을 선택하여 동일한 행동을 반복하는 루프(Loop)에 빠지는 경우가 빈번하다.

둘째, **시스템 견고성 부족(Lack of Robustness)** 문제이다. MAS 내의 긴 상호작용 체인은 에러 전파(Error Propagation)에 취약하다. 특히 하나의 에이전트가 생성한 잘못된 정보(Misinformation)나 환각(Hallucination)이 메모리에 저장될 경우, 이후의 모든 하위 에이전트들의 추론을 오염시켜 시스템 전체의 실패로 이어질 수 있다.

기존의 연구들은 주로 실패 후 원인을 분석하는 사후 분석(Post-hoc failure attribution)에 집중해 왔으나, 본 논문은 런타임 중에 실시간으로 개입하여 이러한 문제를 선제적으로 해결함으로써 MAS의 견고성과 효율성을 동시에 높이는 것을 목표로 한다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 기존 에이전트의 아키텍처를 변경하지 않고도 실시간으로 감시하고 개입할 수 있는 경량화된 모듈형 프레임워크인 $\text{SUPERVISORAGENT}$를 도입하는 것이다.

가장 중심적인 설계 직관은 **'적응형 필터(Adaptive Filter)'**를 통한 선택적 개입이다. 모든 상호작용을 감시하는 것은 막대한 계산 비용을 초래하므로, LLM을 사용하지 않는 가벼운 휴리스틱 기반의 필터를 통해 고위험 상황(에러 발생, 비효율적 루프, 과도한 관찰 길이)에서만 $\text{SUPERVISORAGENT}$를 활성화한다. 이를 통해 최소한의 오버헤드로 에러를 교정하고, 비효율적인 행동을 가이드하며, 관찰 데이터를 정제(Purification)함으로써 시스템 전체의 성능을 최적화한다.

## 📎 Related Works

논문에서는 MAS의 견고성과 효율성을 다룬 기존 연구들을 세 가지 관점에서 설명한다.

1.  **실패 원인 분석 및 견고성:** Aegis나 SHIELDA와 같은 시스템은 실패의 분류 체계를 제안하고, AgenTracer나 A2P는 실패의 근본 원인을 추적하는 방법을 제시하였다. 그러나 이러한 방식들은 모두 실패가 일어난 후 분석하는 반응형(Reactive) 접근 방식이라는 한계가 있다. 본 연구는 실시간으로 위험을 탐지하고 차단하는 선제적(Proactive) 개입 방식이라는 점에서 차별화된다.
2.  **효율적인 MAS 설계:** AgentDropout이나 SafeSieve와 같이 설계 단계에서 에이전트나 통신 링크를 제거하여 효율성을 높이려는 시도가 있었다. 또한 MetaAgent나 HiVA처럼 처음부터 효율적인 위상(Topology)을 구축하거나, 컨텍스트 압축을 통해 토큰을 줄이는 연구들이 진행되었다. 하지만 본 논문은 정적인 설계나 메시지 내용의 압축에 집중하는 대신, 실행 중인 프로세스 자체를 제어하는 런타임 프로세스 제어(Runtime Process Control) 방식을 취한다.
3.  **기타 감독 에이전트:** AWorld의 Guard 에이전트가 사실 확인을 위해 개입하는 것과 유사하지만, $\text{SUPERVISORAGENT}$는 단순한 정확도 향상을 넘어 시스템 전체의 효율성(토큰 절감)과 운용 견고성을 포괄하는 더 넓은 범위의 목적을 가진다.

## 🛠️ Methodology

### 전체 시스템 구조: Supervised Multi-Agent System (SMAS)

본 논문은 기존 MAS에 메타 수준의 제어 에이전트인 Supervisor를 추가한 $\text{Supervised Multi-Agent System (SMAS)}$ 개념을 제안한다. Supervisor는 에이전트의 핵심 로직을 수정하지 않고 다음과 같은 세 가지 고위험 상호작용 지점을 실시간으로 모니터링한다.

- **Agent-Agent 상호작용:** 에이전트 간 통신 중 발생하는 환각 정보의 전파를 감시한다.
- **Agent-Tool 상호작용:** 외부 도구로부터 들어오는 부정확하거나 불필요하게 긴 데이터의 유입을 감시한다.
- **Agent-Memory 상호작용:** 메모리에서 오래되었거나 잘못된 정보가 인출되는지 감시한다.

### 런타임 작동 절차

$\text{SUPERVISORAGENT}$의 작동 방식은 "무엇을, 언제, 어떻게 감독할 것인가"라는 세 가지 단계로 구성된다.

**1. 언제 감독할 것인가: 적응형 필터 (Adaptive Filter)**
LLM을 사용하지 않는 휴리스틱 기반 필터가 다음 조건 중 하나라도 만족하면 $\text{SUPERVISORAGENT}$를 트리거한다.
- **에러 발생:** 도구 사용이나 코드 실행 중 명시적인 에러가 발생한 경우.
- **비효율적 행동:** 에이전트가 동일한 행동을 반복하는 루프에 빠졌거나, 특정 하위 작업에 대해 과도한 단계(Step)를 소모하는 경우.
- **과도한 관찰 길이:** 도구 출력물이 미리 정의된 임계값 $\tau_{\text{len}}$ (기본값 3,000자)을 초과하는 경우.

**2. 무엇을 감독할 것인가: 컨텍스트 윈도우 (Context Window)**
트리거가 발생하면 $\text{SUPERVISORAGENT}$는 의사결정을 위해 다음과 같은 정보를 포함하는 컨텍스트 윈도우 $W$를 구성한다.
$$W = (N, Q^g, Q^l, T^l, S)$$
여기서 $N$은 검토 대상 에이전트, $Q^g$는 전역 작업 목표, $Q^l$은 지역 작업 목표, $T^l$은 에이전트의 최근 행동 기록(Local Trace), $S$는 최신 상호작용 요약이다. 비효율성 진단이 필요한 경우 모든 에이전트의 상호작용 기록인 전역 기록 $T^g$를 추가로 참조한다.

**3. 어떻게 감독할 것인가: 다단계 개입 전략 (Intervention Actions)**
상황에 따라 $\text{SUPERVISORAGENT}$는 다음과 같은 액션 공간 $A$에서 최적의 조치를 선택한다.

- **$\text{approve}$ (승인):** 반복적인 행동이라도 현재 상태에서 최선이라고 판단될 때 개입 없이 진행시킨다.
- **$\text{provide\_guidance}$ (가이드 제공):** 에이전트의 추론 경로가 잘못되었을 때, 기존 관찰 데이터 끝에 간결하고 구체적인 힌트를 추가하여 방향을 수정한다.
- **$\text{correct\_observation}$ (관찰 정제):** 데이터가 너무 길거나(Purification) 사실적으로 틀렸을 때, 원래의 데이터를 완전히 삭제하고 정제된 버전으로 교체한다.
- **$\text{run\_verification}$ (검증 실행):** 내부 정보만으로 부족한 복잡한 에러 상황에서 별도의 검증 서브 에이전트를 호출하여 외부 팩트 체크를 수행하고 그 결과를 반환한다.

## 📊 Results

### 실험 설정
- **데이터셋:** GAIA (주요 벤치마크), AIME 2024, GSM8k-Hard (수학), HumanEval, MBPP (코드), DROP (질의응답).
- **베이스라인:** Smolagent, OAgents, MetaAgent, AWorld 등 최신 MAS 프레임워크.
- **평가 지표:** 성공률 ($\text{pass@k}$), 평균 토큰 소비량 ($\text{Avg. Tokens}$), F1 스코어.
- **사용 모델:** GPT-4.1, Gemini-2.5-pro, Qwen3 등 다양한 모델을 사용하여 모델-불가지론적(Model-agnostic) 특성을 검증하였다.

### 주요 결과
- **토큰 효율성 향상:** GAIA 벤치마크에서 Smolagent에 $\text{SUPERVISORAGENT}$를 적용했을 때, 성공률을 유지하면서도 평균 토큰 소비량을 **29.68% 감소**시켰다. 특히 난이도가 높은 Level 2, 3 작업에서 토큰 절감 효과가 더 뚜렷하게 나타났다.
- **파레토 개선(Pareto Improvement):** 수학 추론(AIME)에서는 성공률을 6.67% 높이면서 토큰 비용을 18.92% 줄였고, 코드 생성(HumanEval)에서는 정확도를 유지하며 토큰 사용량을 23.74% 절감하였다.
- **견고성 및 일관성:** 토큰 소비량의 분산(Variance)이 크게 줄어들어, 특정 작업에서 갑자기 자원을 과도하게 사용하는 아웃라이어 발생 빈도가 낮아졌으며 이는 시스템의 예측 가능성과 신뢰성을 높였음을 의미한다.
- **범용성 확인:** AWorld와 OAgents 프레임워크에 통합했을 때도 각각 평균 36.54%, 39.36%의 토큰 절감 효과를 보이며 다양한 MAS 아키텍처에 적용 가능함을 입증하였다.

### 어블레이션 연구 (Ablation Study)
- **정제(Purification)** 모듈이 토큰 절감의 가장 큰 기여 요인임을 확인하였다. 이 모듈을 제거할 경우 토큰 절감률이 29.68%에서 15.96%로 급격히 떨어진다.
- **교정(Correction)** 및 **가이드(Guidance)** 모듈은 성공률(Accuracy) 유지에 결정적인 역할을 하며, 이를 제거할 경우 성공률이 유의미하게 하락하였다.

## 🧠 Insights & Discussion

본 연구는 $\text{SUPERVISORAGENT}$가 단순한 최적화 도구가 아니라, 미래의 MAS에서 메모리 뱅크나 도구 사용 프레임워크처럼 필수적인 **기초 구성 요소(Foundational Component)**가 될 수 있음을 시사한다.

특히 주목할 만한 통찰은 **'노이즈와 신호의 트레이드오프'**이다. 저자들은 HTML 구조나 잘림 표시(truncation cues)와 같은 겉보기에 '노이즈' 같은 정보가 ReAct 스타일의 에이전트에게는 중요한 환경적 신호가 될 수 있음을 발견하였다. 따라서 지나치게 공격적인 정제(Purification)는 오히려 성능을 떨어뜨릴 수 있으며, 정보 밀도와 환경적 텍스처 보존 사이의 균형이 중요함을 역설한다.

한계점으로는 감독 에이전트의 호출로 인한 약간의 레이턴시 증가(평균 1.5분 미만)가 발생한다는 점이 언급되었다. 또한, 휴리스틱 기반 필터가 모델마다 다른 성능을 보일 수 있어, 향후에는 학습 기반의 적응형 필터(Learning-based adaptive filter)를 통해 더 정밀한 제어를 수행하는 방향으로 발전시킬 필요가 있다.

## 📌 TL;DR

본 논문은 Multi-Agent Systems의 고질적인 문제인 과도한 토큰 소비와 에러 전파 문제를 해결하기 위해, 런타임 중에 선택적으로 개입하는 경량화된 감독 프레임워크 $\text{SUPERVISORAGENT}$를 제안한다. 이 시스템은 LLM-free 필터를 통해 고위험 상황을 탐지하고, 상황에 맞는 정제, 가이드, 교정 액션을 수행함으로써 **성능 저하 없이 토큰 소비량을 평균 약 30% 절감**하고 시스템의 견고성을 높였다. 이는 실제 서비스 환경에서 MAS의 경제적 비용을 낮추고 신뢰성을 확보하는 데 중요한 기여를 할 것으로 기대된다.
