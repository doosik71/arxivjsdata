# Agentic AI Home Energy Management System: A Large Language Model Framework for Residential Load Scheduling

Reda El Makroum, Sebastian Zwickl-Bernhard, Lukas Kranzl (2025)

## 🧩 Problem to Solve

본 연구는 주거용 에너지 관리 시스템(Home Energy Management Systems, HEMS)의 보급을 가로막는 핵심 장애물인 **사용자 상호작용 장벽(User Interaction Barriers)**을 해결하고자 한다. 전 세계적인 에너지 전환과 재생 에너지 통합을 위해서는 주거 부문의 수요 반응(Demand Response) 용량을 대폭 확대해야 하며, 이를 위해 가전제품의 에너지 소비 시간을 유연하게 조정하는 Load Scheduling이 필수적이다.

그러나 기존의 HEMS는 사용자가 일상적인 선호도를 복잡한 기술적 파라미터로 직접 변환하여 입력해야 하는 구조를 가지고 있다. 이러한 과정은 비전문가나 고령자에게 높은 진입 장벽이 되며, 결과적으로 HEMS의 실제 채택률을 낮추는 원인이 된다. 따라서 본 논문의 목표는 자연어 입력을 통해 복잡한 기술적 설정 없이도 다수의 가전제품 스케줄링과 제어를 자율적으로 수행할 수 있는 **Agentic AI 기반의 HEMS 프레임워크**를 설계하고 구현하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 LLM을 단순한 전처리 도구가 아닌 **자율적 의사결정 주체(Autonomous Decision-Making Entity)**로 활용한 점에 있다. 구체적인 설계 아이디어는 다음과 같다.

1.  **자율적 의사결정 체계 구축**: 기존 연구들이 LLM을 코드 생성기나 파라미터 추출기로 사용하여 전통적인 최적화 솔버에 전달했던 것과 달리, 본 시스템은 LLM이 ReAct(Reason+Act) 패턴을 통해 직접 추론하고 도구를 호출하여 스케줄링 결정을 내린다.
2.  **계층적 멀티 에이전트 아키텍처(Hierarchical Multi-Agent Architecture)**: 하나의 오케스트레이터(Orchestrator) 에이전트와 세 개의 전문 에이전트(Specialist Agents: 세탁기, 식기세척기, 전기차 충전기)로 구성된 구조를 도입하여, 복잡한 다중 가전 조정 작업을 모듈화하고 확장성을 확보하였다.
3.  **통합 도구 프레임워크 개발**: 분석 기능, 외부 API(전기 요금, Google 캘린더) 연동, 전문 에이전트 위임 기능을 하나의 통합된 시스템 내에서 조율하는 다층 도구 체계를 구축하였다.

## 📎 Related Works

기존의 LLM 기반 에너지 시스템 연구들은 주로 다음과 같은 한계를 보였다.
- **전처리 도구로서의 LLM**: 많은 연구가 LLM을 사용하여 최적화 코드를 생성하거나, 사용자 요청에서 파라미터를 추출하여 전통적인 최적화 알고리즘(MILP 등)에 입력하는 인터페이스 역할로 한정하여 사용하였다.
- **단일 작업 자동화**: 특정 가전의 제어나 단일 시스템의 최적화에 집중했을 뿐, 자연어 입력부터 다중 가전의 통합 스케줄링 및 제어까지의 전 과정을 자율적으로 관리하는 에이전트 시스템은 부재했다.

본 연구는 LLM이 외부 환경과 상호작용하며 스스로 계획을 수정하고 실행하는 **Agentic AI** 패러다임을 HEMS에 적용함으로써, 전통적인 제어 로직으로의 중간 전달 단계 없이 LLM이 직접 운영 결정을 내린다는 점에서 차별성을 가진다.

## 🛠️ Methodology

### 전체 시스템 구조
본 시스템은 **계층적 멀티 에이전트 구조**를 따른다. 중앙의 오케스트레이터 에이전트가 사용자 요청을 분석하고, 필요한 데이터를 수집하며, 각 가전제품 전담 전문 에이전트에게 작업을 위임한 뒤 최종 스케줄을 확정한다.

### 주요 구성 요소 및 역할
1.  **Orchestrator Agent**: ReAct 패턴을 사용하여 '추론(Reason) $\rightarrow$ 행동(Act) $\rightarrow$ 관찰(Observe)' 루프를 반복한다. 사용자 요청 해석, API 데이터 페칭, 전문 에이전트 호출 및 최종 setpoint 확정 등의 전략적 조율을 담당한다.
2.  **Specialist Agents**: 각 가전(세탁기, 식기세척기, EV 충전기)에 특화된 에이전트로, 오케스트레이터로부터 받은 데이터와 제약 조건을 바탕으로 비용 최소화 스케줄을 제안한다. 이들은 반복적인 루프 대신 단일 턴(Single-turn) 상호작용을 통해 토큰 효율성을 높였다.
3.  **Tool Ecosystem**:
    - `get_electricity_prices`: ENTSO-E API를 통해 15분 단위의 전일 전기 요금을 가져온다.
    - `get_calendar_ev_constraint`: Google Calendar API를 통해 사용자의 일정에서 EV 충전 마감 시간을 추론한다.
    - `calculate_window_sums`: 특정 윈도우 크기에 대한 요금 합계를 계산하여 최적의 시간대를 찾는다.
    - `call_appliance_agent`: 전문 에이전트에게 작업을 위임한다.
    - `schedule_appliance`: 결정된 스케줄을 binary array 형태로 저장하여 장치를 제어한다.

### 수학적 최적화 및 목표 변수
시스템의 성능 검증을 위해 Mixed-Integer Linear Programming (MILP)을 통해 지상 truth(Ground Truth) 최적해를 계산한다. 목표 함수는 모든 가전제품의 총 전기 요금을 최소화하는 것이다.

$$ \min_{t_{WM}, t_{DW}, t_{EV}} \sum_{a \in A} \sum_{k=0}^{d_a-1} C_{t_a+k} $$

여기서 $A$는 가전제품 집합, $d_a$는 가전 $a$의 작동 지속 시간, $C$는 각 타임슬롯의 전기 요금, $t_a$는 시작 시간이다. 각 가전은 작동 시간과 마감 시간($t_{max,a}$) 제약 조건을 만족해야 한다.

### 학습 및 실행 절차
본 시스템은 Few-shot learning이나 파인튜닝 없이 **Zero-shot**으로 동작한다. 에이전트들에게는 예시 데이터 대신 도구 설명과 작업 지침이 담긴 시스템 프롬프트만 제공된다. 

### 보안 아키텍처
프롬프트 인젝션 공격을 방지하기 위해 3단계 방어 전략을 사용한다.
- **Pre-LLM Validation**: 요청 횟수 제한 및 입력 길이 제한.
- **Pattern-based Detection**: 정규표현식(Regex)을 통해 지침 무시, 시스템 프롬프트 유출 시도 등을 탐지하여 즉시 거부한다.
- **Privilege Separation**: 사용자 입력을 `<user_input>` 태그로 감싸 LLM이 이를 신뢰할 수 없는 데이터로 인식하게 하여 시스템 지침을 덮어쓰지 못하도록 격리한다.

## 📊 Results

### 실험 설정
- **모델**: Llama-3.3-70B, Qwen-3-32B, GPT-OSS-120B.
- **데이터셋**: 오스트리아의 실제 전일 전기 요금 및 Google Calendar 일정.
- **지표**: MILP 최적해와의 일치 여부(Optimality), 성공률, 평균 반복 횟수, 토큰 소모량, 실행 시간.

### 주요 결과
1.  **단일 가전 스케줄링**: 세 모델 모두 100% 최적해를 찾아내어 기본적인 조율 능력은 충분함을 보였다.
2.  **다중 가전 조율 (Multi-appliance Coordination)**: 
    - **Llama-3.3-70B**는 모든 시나리오에서 100% 성공하며 완벽한 조율 능력을 보였다.
    - **Qwen-3-32B**는 일부 성공(20%)했으나 EV 충전기 조율에서 지속적으로 실패했다.
    - **GPT-OSS-120B**는 세탁기만 스케줄링하고 나머지는 시도조차 하지 않아 실패율이 가장 높았다.
3.  **분석적 질의(Analytical Query) 성능**:
    - Baseline(지침 없음)에서는 모든 모델이 도구 사용에 실패했다.
    - Minimal Guidance(최소 지침)에서는 도구 호출은 성공했으나 정답 도출은 불안정했다.
    - **Explicit Workflow**(명시적 워크플로우) 지침이 제공되었을 때만 모든 모델이 100% 정답률을 기록했다.

### 정량적 지표 (Llama-3.3-70B 기준)
- 다중 가전 조율 시 평균 9.0회 반복, 약 32,883 토큰 소모, 실행 시간 14.7초가 소요되었다. 이는 실제 서비스에 적용 가능한 수준의 지연 시간이다.

## 🧠 Insights & Discussion

### 강점 및 가능성
본 연구는 LLM이 단순한 인터페이스를 넘어 복잡한 도구 체계를 조율하는 **의사결정 엔진**으로 작동할 수 있음을 입증했다. 특히 Llama-3.3-70B의 결과는 자연어 기반의 유연한 인터페이스가 전통적인 수학적 최적화 수준의 성능을 낼 수 있음을 보여주며, 이는 HEMS의 사용자 진입 장벽을 획기적으로 낮출 수 있는 경로를 제시한다.

### 한계 및 비판적 해석
1.  **모델 크기 및 추론 능력의 의존성**: 모델의 파라미터 크기와 추론 능력이 다중 에이전트 조율 성공률에 결정적인 영향을 미친다. 이는 소형 모델을 사용할 경우 시스템의 신뢰성이 급격히 떨어질 수 있음을 의미한다.
2.  **프롬프트 엔지니어링의 필요성**: 분석적 질의 결과에서 보듯, LLM이 도구를 자율적으로 선택하는 능력은 여전히 불완전하며, 일관된 성능을 위해서는 매우 상세한 워크플로우 지침이 필요하다.
3.  **지속 가능성 문제**: 수백만 가구가 LLM 기반 스케줄링을 수행할 때 발생하는 데이터 센터의 전력 소모량이 기존 최적화 알고리즘보다 훨씬 크다. 에너지 절감을 위한 시스템이 오히려 에너지 소모를 늘리는 역설적 상황에 대한 고려가 필요하다.
4.  **제약 조건 충돌 검증 부족**: 실험에서 사용된 요금 패턴이 비용 최적해와 캘린더 마감 시간이 충돌하지 않는 구조였다. 실제 환경에서 비용과 마감 시간이 상충할 때 에이전트가 어떤 우선순위로 의사결정을 내리는지에 대한 검증이 추가로 이루어져야 한다.

## 📌 TL;DR

본 논문은 자연어 요청만으로 가전제품의 에너지 소비를 최적화하는 **자율적 Agentic AI HEMS 프레임워크**를 제안한다. 오케스트레이터와 전문 에이전트의 계층적 구조와 ReAct 패턴을 통해 복잡한 기술적 설정 없이도 MILP 최적해에 부합하는 스케줄링을 달성하였다. 특히 **Llama-3.3-70B** 모델이 가장 뛰어난 조율 능력을 보였으며, 이는 향후 HEMS의 사용자 경험을 혁신하고 수요 반응 참여를 확대하는 데 중요한 역할을 할 것으로 기대된다.
