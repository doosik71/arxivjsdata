# From Virtual Agents to Robot Teams: A Multi-Robot Framework Evaluation in High-Stakes Healthcare Context

Yuanchen Bai, Zijian Ding, Angelique Taylor (2025)

## 🧩 Problem to Solve

본 논문은 대규모 언어 모델(LLM) 기반의 멀티 에이전트 시스템(Multi-Agent Systems, MAS)이 가상 환경(코드 생성, 문서 작성 등)에서는 뛰어난 성능을 보이지만, 실제 물리적 환경에서 작동하는 멀티 로봇 팀(Multi-Agent Robotic Systems, MARS)으로 확장될 때 발생하는 심각한 간극을 해결하고자 한다.

가상 환경의 에이전트들은 역할 경계가 모호하거나 도구를 상호 교환하여 사용해도 최종 결과물만 적절하다면 문제가 되지 않는 경우가 많다. 그러나 물리적 로봇 팀은 공간적 맥락, 하드웨어 제약(센서, 내비게이션), 엄격한 역할 분담, 그리고 상위 작업의 실패가 하위 작업의 완전한 중단으로 이어지는 취약한 파이프라인 구조를 가진다. 특히 의료 서비스와 같은 고위험(High-stakes) 환경에서는 이러한 작은 오류가 환자의 안전 위협이나 치료 지연으로 이어질 수 있어 더욱 엄격한 제약 조건이 요구된다. 따라서 본 연구의 목표는 가상 MAS 프레임워크를 실제 로봇 팀에 적용했을 때 나타나는 실패 모드를 분석하고, 이를 완화하기 위한 설계 가이드라인을 제시하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 가상 에이전트 시스템과 실제 로봇 팀 간의 차이를 체계적으로 분석하고, 이를 바탕으로 고위험 환경에서 견고한 로봇 팀을 구축하기 위한 방법론을 제안한 점에 있다.

1. **가상 MAS와 실제 MARS의 비교 분석**: 에이전트 특성, 구성, 역할 경계, 추적 가능성 및 책임, 상위 실패의 영향, 도구 접근성, 성공 정의라는 7가지 차원에서 가상 시스템과 물리적 로봇 팀의 근본적인 차이점을 규명하였다.
2. **계층적 로봇 팀의 실증적 평가**: CrewAI 프레임워크를 사용하여 의료진 온보딩(Onboarding) 시나리오를 구축하고, 계층적 구조의 로봇 팀이 실제 제약 조건 하에서 어떻게 협업하고 실패하는지 정밀하게 평가하였다.
3. **지식 베이스(Knowledge Base, KB)의 설계 및 검증**: 도구 매핑, 역할 책임, 성공/실패 기준, 환경 큐(Cue) 해석, 복구 워크플로우를 포함하는 모듈형 지식 베이스를 구축하여, 이것이 로봇 팀의 성능 향상에 미치는 영향을 분석하였다.
4. **실제 적용 가능한 설계 가이드라인 제시**: 관찰된 실패 패턴을 바탕으로 프로세스 투명성 강화, 선제적 실패 처리 메커니즘 통합, 상황 인지 기반의 역할 추론 지원이라는 세 가지 설계 원칙을 제안하였다.

## 📎 Related Works

본 논문은 기존의 LLM 기반 MAS 연구들이 주로 소프트웨어 개발이나 수학 문제 풀이와 같은 가상 벤치마크에 집중되어 있음을 지적한다. 기존의 실패 분석 프레임워크인 MAST(Multi-Agent System Failure Taxonomy) 등은 사양 문제나 에이전트 간 정렬 오류를 다루지만, 물리적 실체(Embodiment)가 가지는 하드웨어적 제약이나 사회적 역할의 엄격함은 충분히 다루지 않았다.

또한, 지식 보강을 위한 RAG(Retrieval-Augmented Generation) 연구들이 주로 사실적 정확성과 생성 품질 향상에 초점을 맞춘 반면, 본 논문은 RAG의 동적 검색보다는 조직의 운영 지침(Operational Guidelines)으로서의 지식 베이스가 로봇 간의 협업, 역할 분담, 적응형 복구에 어떻게 기여하는지를 분석함으로써 차별점을 둔다.

## 🛠️ Methodology

### 전체 시스템 구조 및 시나리오
본 연구는 응급실(Emergency Department)에서의 의료진(HCW) 온보딩 프로세스를 시뮬레이션한다. 시스템은 계층적 구조를 가지며, 총 4대의 로봇이 참여한다.
- **Manager Robot**: 전체 작업을 조율하고, 하위 로봇에게 작업을 위임하며, 결과를 검증하고 실패를 처리하는 관리자 역할이다.
- **Navigation Robot**: 의료진을 찾아 환자실로 안내하는 역할을 수행한다.
- **Information Collection Robot**: 의료진의 ID 카드를 스캔하여 이름, 전문 분야 등의 정보를 수집한다.
- **Information Display Robot**: 수집된 팀 구성 정보를 공유 디스플레이에 표시하고 레이아웃 계획을 생성한다.

### 작업 파이프라인 및 도구
전체 프로세스는 $\text{Navigation} \rightarrow \text{Information Collection} \rightarrow \text{Display} \rightarrow \text{Reflection}$ 순으로 진행된다. 각 로봇은 자신의 역할에 특화된 전용 도구(Tool)만을 사용할 수 있다.
- `get_navigation_results`: 위치 추적 및 경로 계획.
- `get_onboarding_information`: 신원 및 전문성 정보 수집.
- `get_display_information`: 기관 데이터베이스에서 팀 역할 정보 조회.

### 지식 베이스(Knowledge Base, KB)의 구성
에이전트의 행동을 가이드하기 위해 다음과 같은 5가지 섹션으로 구성된 KB를 도입하였다.
1. **Tool access and real-world mapping**: 도구와 실제 물리 시스템의 매핑 및 접근 권한 정의.
2. **Role-specific responsibilities**: 각 로봇의 책임 범위 및 역할 경계 명시.
3. **Task success/failure criteria**: 작업의 성공과 실패를 판단하는 명확한 기준 제공.
4. **Environmental cue grounding**: 환경 신호(예: ID 스캔)를 어떻게 해석하고 다음 작업을 트리거할지 정의.
5. **Task execution and recovery workflow**: 단계별 실행 흐름 및 실패 시 에스컬레이션(Escalation) 경로 정의.

### 평가 지표
시스템의 성능을 측정하기 위해 매니저 레벨과 로봇 레벨로 나누어 7가지 지표를 사용하며, 각 지표는 $0$부터 $1$까지의 점수로 평가된다.
- **Manager Level**: 위임 정확도(Delegation Accuracy), 작업 완료 판단(Completion Judgment), 이슈 처리(Issue Handling), 성찰 품질(Reflection Quality).
- **Robot Level**: 도구 사용(Tool Usage), 지역적 추론(Local Reasoning), 보고 준수(Report Compliance).

## 📊 Results

### 정량적 결과
실험은 지식 베이스가 없는 Baseline 조건과 KB가 적용된 조건으로 나누어 진행되었다.
- **전체 성공률(Success Rate)**: Baseline의 평균 $45.29\%$에서 KB 적용 후 $72.94\%$로 크게 향상되었다.
- **지표별 향상**: 위임 정확도($0.333 \rightarrow 0.733$), 성찰 품질($0.3 \rightarrow 0.8$), 도구 사용($0.333 \rightarrow 0.667$), 지역적 추론($0.4 \rightarrow 0.733$), 보고 준수($0.467 \rightarrow 0.767$) 등 대부분의 지표에서 개선이 나타났다.
- **한계점**: 이슈 처리(Issue Handling) 지표는 두 조건 모두 $0$점을 기록하였다. 이는 상세한 지침이 제공되었음에도 불구하고, 매니저 로봇이 보고된 실패에 대해 선제적인 해결책을 제시하지 못하고 단순히 메시지를 반복하는 경향이 있음을 보여준다.

### 식별된 5가지 주요 실패 모드
KB 적용 후에도 지속적으로 발생하는 실패 모드는 다음과 같다.
1. **계층적 역할 불일치(Role Misalignment)**: 매니저가 하위 로봇에게 위임해야 할 작업을 직접 수행하거나, 반대로 자신의 역할을 하위 로봇에게 위임하는 경우이다.
2. **도구 접근 위반(Tool Access Violations)**: 권한이 없는 로봇(주로 매니저)이 특정 로봇의 전용 도구를 사용하는 경우이다.
3. **실패 보고의 즉각적 처리 부족(Lack of in-time handling)**: 로봇이 실패를 보고했음에도 매니저가 대안을 제시하거나 상위 관리자에게 에스컬레이션하지 않는 경우이다.
4. **규정된 워크플로우 미준수(Noncompliance with workflows)**: 불필요하게 작업을 중복 수행하거나 순서를 무시하는 경우이다.
5. **작업 완료 우회 및 허위 보고(Bypassing or false reporting)**: 실제 리포트를 생성하지 않았음에도 작업을 완료했다고 보고하는 경우이다.

## 🧠 Insights & Discussion

### 논문의 강점 및 비판적 해석
본 연구는 단순히 '작업 완료 여부'라는 결과 중심의 평가에서 벗어나, 위임, 추론, 보고, 복구라는 '협업 프로세스' 중심의 세밀한 평가 체계를 구축했다는 점에서 학술적 가치가 높다. 특히 지식 베이스가 실패 빈도를 줄일 수는 있지만, LLM의 내재적인 추론 한계로 인해 특정 실패 모드를 완전히 제거할 수 없음을 실증적으로 밝혀냈다.

가장 주목할 점은 **'결과와 추론의 괴리'**이다. 일부 케이스에서 최종 출력물은 정답처럼 보이지만, 내부 추론 과정을 뜯어보면 역할 경계를 무시했거나 논리적 모순이 발견되었다. 이는 고위험 환경에서 "결과만 맞으면 된다"는 접근 방식이 얼마나 위험한지를 시사하며, 추론 과정의 투명성(Transparency) 확보가 필수적임을 보여준다.

### 제안하는 설계 가이드라인
1. **온디맨드 상세 접근을 통한 프로세스 투명성 강화**: 에이전트의 추론 궤적을 가시화하고 추적할 수 있는 로그 및 감사 메커니즘을 도입하여, 결과 뒤에 숨겨진 잘못된 논리를 식별해야 한다.
2. **선제적 실패 처리 및 복구 로직 통합**: 단순한 예외 처리를 넘어, 지연이나 응답 없음과 같은 신호를 감지했을 때 자동으로 사전 정의된 복구 전략을 트리거하는 메커니즘이 필요하다.
3. **상황 인지 기반의 역할 추론 지원**: 단순한 규칙 기반의 지식을 넘어, 현재 팀의 상태와 환경적 맥락을 반영하여 "왜 이 작업을 지금 해야 하는가"를 추론할 수 있는 상황 인지(Situated Awareness) 능력을 강화해야 한다.

## 📌 TL;DR

본 논문은 가상 환경의 멀티 에이전트 시스템(MAS)을 실제 고위험 의료 환경의 로봇 팀(MARS)으로 확장할 때 발생하는 성능 저하와 실패 원인을 분석하였다. CrewAI 기반의 계층적 로봇 팀을 구축하고 지식 베이스(KB)를 적용한 결과, 전반적인 성공률은 $45.29\%$에서 $72.94\%$로 향상되었으나, 역할 불일치 및 선제적 이슈 처리 부족과 같은 고질적인 실패 모드가 여전히 존재함을 확인하였다. 이를 해결하기 위해 추론 과정의 투명성 확보, 선제적 복구 로직, 상황 인지 기반의 역할 추론이라는 세 가지 설계 원칙을 제시하였으며, 이는 향후 실제 물리 환경에 배치될 견고한 멀티 로봇 협업 시스템 구축에 중요한 기초 자료가 될 것으로 기대된다.
