# A Mechanism for Discovering Semantic Relationships among Agent Communication Protocols

Idoia Berges, Jesús Bermúdez, Alfredo Goñi, Arantza Illarramendi (2024)

## 🧩 Problem to Solve

본 논문은 서로 다른 정보 시스템(Information Systems)에 소속된 에이전트들이 서로 다른 통신 프로토콜(Communication Protocols)을 사용할 때 발생하는 상호운용성(Interoperability) 문제를 해결하고자 한다. 일반적으로 에이전트 간 통신은 통신 행위(Communication Acts)의 교환으로 이루어지며, 프로토콜은 이러한 행위들의 허용 범위를 제한하는 규칙을 정의한다.

문제의 핵심은 동일한 목적을 가진 프로토콜이라 할지라도 설계 방식에 따라 구조가 다를 수 있다는 점이다. 기존의 구조적 비교 방식은 너무 경직되어 있어, 에이전트들이 서로 다른 표준 프로토콜을 사용하더라도 그 의미적 본질이 같다면 상호작용이 가능해야 한다는 유연한 상호운용성을 달성하기 어렵다. 따라서 본 연구의 목표는 프로토콜의 외형적 구조가 아닌, 그 내면의 의미적 관계(예: 동등성, 특수화, 제한 등)를 발견할 수 있는 메커니즘을 구축하는 것이다.

## ✨ Key Contributions

본 논문의 중심적인 아이디어는 프로토콜의 의미를 해당 프로토콜의 각 분기(Branch)가 생성하는 **플루언트(Fluents)**들의 집합으로 정의하고, 이를 통해 프로토콜 간의 관계를 분석하는 것이다. 주요 기여 사항은 다음과 같다.

1.  **통신 행위 온톨로지(CommOnt) 도입**: 통신 행위를 계층적 구조(Upper, Standards, Applications layer)로 정의하여 표준 프로토콜의 사용자 정의(Customization)와 명시적 의미 표현을 가능하게 한다.
2.  **이중 의미 표현 체계(Dual Semantic Representation)**: 통신 행위의 정적 구조는 OWL-DL 온톨로지로 표현하고, 행위가 가져오는 동적 효과(사회적 약속, Social Commitments)는 Event Calculus의 프레디케이트(Predicate)로 표현한다.
3.  **SWRL을 이용한 실행 가능 메커니즘**: 정적 온톨로지와 동적 Event Calculus를 SWRL(Semantic Web Rule Language)로 통합하여, 프로토콜의 상태 변화를 실제로 계산하고 추론할 수 있는 체계를 제공한다.
4.  **의미적 관계의 정형 정의**: 프로토콜 간의 관계를 동등성(Equivalence), 특수화(Specialization), 제한(Restriction), 접두사(Prefix), 접미사(Suffix), 인픽스(Infix), 인픽스에 대한 보완(Complement to infix)으로 세분화하여 정의하고 이를 발견하는 알고리즘을 제시한다.

## 📎 Related Works

논문은 관련 연구를 세 가지 기준(의미적 접근 여부, 정형 관계 정의 여부, 프로토콜 합성 고려 여부)으로 분류하여 설명한다.

*   **의미적 접근 방식**: Mallya와 Singh(2007)은 약속(Commitments)을 기반으로 한 전이 시스템을 제안했으나, 관계를 계산하는 구체적인 알고리즘이 부족하다. Desai 등(2005)은 $\pi$-calculus를 통해 동등성을 분석했지만, 분석 범위가 제한적이다.
*   **구조적 접근 방식**: Montes-Rendón 등(2006)은 FSM(Finite State Machine)을 이용한 정렬(Alignment) 방식을 사용하는데, 이는 프로토콜 구조가 매우 유사해야만 통신이 가능하다는 점에서 매우 경직된 방식이다. d'Inverno 등(1998)과 Mazouzi 등(2002) 역시 FSM이나 Petri net을 사용하지만, 통신 행위의 의미적 내용은 고려하지 않는다.
*   **차별점**: 본 논문은 기존의 구조적 비교에서 벗어나, 통신 행위의 내용과 그로 인해 발생하는 사회적 약속(Social Commitment)이라는 의미적 결과물을 기반으로 프로토콜을 비교함으로써 훨씬 유연한 상호운용성을 제공한다.

## 🛠️ Methodology

### 1. 전체 시스템 구조
본 시스템은 통신 행위의 정의 $\rightarrow$ 프로토콜의 구조적 표현 $\rightarrow$ 분기별 의미 추출(Trace 생성) $\rightarrow$ 프로토콜 간 관계 도출의 파이프라인으로 구성된다.

### 2. 통신 행위의 이중 표현
*   **정적 표현 (CommOnt)**: OWL-DL을 사용하여 통신 행위를 `CommunicationAct` 클래스로 정의하며, 발화 행위 이론(Speech Act Theory)에 따라 화자의 태도(Illocutionary force)와 명제 내용(Propositional content)으로 구분한다.
*   **동적 표현 (Event Calculus)**: 통신 행위의 효과를 사회적 약속(Social Commitment)으로 정의한다.
    *   **기본 약속 (Base-level commitment)**: $C(x, y, p)$는 채무자 $x$가 채권자 $y$에게 명제 $p$를 달성하겠다고 약속한 상태이다.
    *   **조건부 약속 (Conditional commitment)**: $CC(x, y, p, q)$는 조건 $p$가 달성되면 $x$가 $y$에게 $q$를 달성하겠다고 약속한 상태이다.
    *   이러한 약속들은 Event Calculus의 플루언트(Fluents)로 취급되며, `Initiates`와 `Terminates` 프레디케이트를 통해 상태 변화를 기술한다.

### 3. 프로토콜의 표현 및 분석
프로토콜은 결정론적이며 사이클이 없는 상태 전이 시스템(State Transition System, STS)으로 모델링된다.
$$STS = (S, s_0, L, T, F)$$
여기서 $S$는 상태 집합, $s_0$는 초기 상태, $L$은 온톨로지에서 가져온 통신 행위 라벨 집합, $T$는 전이 집합, $F$는 최종 상태 집합이다.

**프로토콜 트레이스(Protocol Trace) 생성 과정**:
1.  프로토콜을 초기 상태에서 최종 상태까지의 모든 경로인 분기(Branch) $\Omega(P)$로 분리한다.
2.  각 분기 $B$에 대해 SWRL 규칙을 적용하여, 최종 상태에서 유지되는 플루언트들의 집합 $T(B)$를 계산한다.
3.  이 $T(B)$가 해당 분기의 의미적 결과물이 되며, 프로토콜 $P$의 해석은 $\{T(B) | B \in \Omega(P)\}$가 된다.

### 4. 프로토콜 비교 알고리즘 및 관계 정의
두 프로토콜 $P_1, P_2$를 비교하기 위해 분기 간의 유사도를 측정한다. 두 플루언트 $t_1, t_2$의 관계를 동등($eq$), 일반화($g_1, g_2$), 비교불가($in$)로 구분하고, 다음과 같은 유사도 함수 $f$를 사용한다.
$$f(x_0, x_1, x_2, x_3) = \frac{x_0 + \max(x_1, x_2)}{\sum_{i=0}^{3} x_i}$$
여기서 $x_0$는 동등한 플루언트 쌍의 수, $x_1, x_2$는 각각 일반화 관계의 수, $x_3$는 초과된 플루언트 수이다.

이 유사도를 기반으로 최적의 매칭 $\pi$를 찾아 다음과 같은 관계를 정의한다.
*   **동등성 ($P[E]Q$)**: 모든 분기가 일대일로 대응하며 의미적으로 동등할 때.
*   **특수화 ($P[Z]Q$)**: 모든 분기가 대응하는 $Q$의 분기를 특수화할 때.
*   **제한 ($P[R]Q$)**: $P$의 분기 집합이 $Q$의 분기 집합의 진부분집합과 동등할 때.
*   **접두사/접미사/인픽스 ($P[P]Q, P[S]Q, P[I]Q$)**: 분기의 트레이스가 다른 분기의 일부분(앞, 뒤, 중간)과 일치할 때.
*   **인픽스 보완 ($P[C]Q$)**: $P$의 분기가 $Q$의 분기를 감싸는 형태(앞부분과 뒷부분이 일치)일 때.

## 📊 Results

### 실험 설정 및 시나리오
본 논문은 도로 사고 발생 시 구급대원(Nurse)과 병원 의사(Doctor) 간의 상호작용 시나리오를 통해 제안된 메커니즘을 검증한다.
*   **프로토콜 $P_1$ (공공병원)**: 도움 요청 $\rightarrow$ 수락 $\rightarrow$ 헬리콥터 요청 $\rightarrow$ 사고 정보 전송 $\rightarrow$ 헬리콥터 가능 여부 응답.
*   **프로토콜 $P_2$ (사립병원)**: 도움 요청 $\rightarrow$ 신원 확인 $\rightarrow$ 보험 확인 $\rightarrow$ 수락 $\rightarrow$ 사고 정보 전송 및 헬리콥터 요청 $\rightarrow$ 헬리콥터 가능 여부 응답.

### 결과 분석
1.  **분기 분리 및 플루언트 생성**: $P_1$은 2개의 분기, $P_2$는 3개의 분기로 분리된다. 각 분기의 통신 행위에 따라 사회적 약속(Commitment) 플루언트들이 생성된다.
2.  **분기 매칭**: 유사도 행렬을 통해 $P_1$의 분기 $B_{1.1}$은 $P_2$의 $B_{2.1}$과 동등($\equiv_b$)하며, $B_{1.2}$는 $B_{2.3}$의 인픽스 보완 관계($\equiv_c$)임이 밝혀진다.
3.  **최종 관계 도출**: $P_2$의 3개 분기 중 2개만이 $P_1$과 매칭되므로 이는 '제한($R$)' 관계이며, 매칭된 관계가 동등($E$) 및 인픽스 보완($C$)이므로, 최종적으로 **$P_1[ECR]P_2$**라는 복합 관계(Specialization of a Restriction of a Complement to Infix)가 도출된다.

## 🧠 Insights & Discussion

### 강점
본 연구는 프로토콜 비교의 패러다임을 '구조'에서 '의미(결과물)'로 전환했다는 점에서 매우 강력하다. 특히 OWL-DL과 Event Calculus를 SWRL로 통합함으로써, 이론적인 의미 정의에 그치지 않고 실제로 계산 가능한 알고리즘으로 구현했다는 점이 돋보인다. 이를 통해 서로 다른 표준을 사용하는 에이전트 시스템 간의 유연한 상호운용성을 실질적으로 달성할 수 있는 기반을 마련하였다.

### 한계 및 가정
1.  **모델의 제약**: 분석 대상이 되는 STS가 '결정론적(Deterministic)'이고 '사이클이 없다(Without cycles)'고 가정한다. 실제 복잡한 통신 프로토콜에서는 루프(Loop)나 비결정론적 전이가 빈번하게 발생하는데, 이에 대한 처리는 명시되지 않았다.
2.  **시간 파라미터 제외**: Event Calculus에서는 시간 $t$가 중요하지만, 실제 알고리즘 구현 단계에서는 시간 파라미터를 생략하여 분석하였다. 이는 실시간 제약 조건이 중요한 프로토콜 분석에는 한계가 될 수 있다.
3.  **계산 복잡도**: 프로토콜의 분기 수가 급격히 증가할 경우, 분기 간의 카테시안 곱을 통한 유사도 계산 및 최적 매칭 탐색의 복잡도가 증가할 가능성이 있다.

### 비판적 해석
논문에서 제시한 $P_1[ECR]P_2$와 같은 복잡한 관계 정의는 학술적으로는 정교하지만, 실제 시스템 관리자가 이러한 관계를 직관적으로 이해하고 활용하기에는 다소 어려움이 있을 수 있다. 하지만 이는 프로토콜 간의 '차이'를 정밀하게 측정하기 위한 필수적인 과정으로 보이며, 향후 이를 자동화된 협상 메커니즘과 연결한다면 매우 유용한 도구가 될 것이다.

## 📌 TL;DR

본 논문은 서로 다른 통신 프로토콜을 사용하는 에이전트 간의 상호운용성을 높이기 위해, 프로토콜의 구조가 아닌 **의미적 결과물(플루언트/약속)**을 비교하여 관계를 발견하는 메커니즘을 제안한다. `CommOnt` 온톨로지와 Event Calculus를 SWRL로 통합하여 프로토콜의 의미적 트레이스를 생성하고, 이를 통해 동등성, 특수화, 제한 등 정교한 관계를 도출한다. 이 연구는 향후 이기종 에이전트 시스템이나 웹 서비스 간의 자동화된 프로토콜 매칭 및 변환 연구에 핵심적인 역할을 할 것으로 기대된다.
