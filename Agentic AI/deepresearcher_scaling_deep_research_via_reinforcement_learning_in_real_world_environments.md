# DeepResearcher: 실제 환경에서 강화 학습을 통한 심층 연구 확장

Yuxiang Zheng, Dayuan Fu, Xiangkun Hu, Xiaojie Cai, Lyumanshan Ye, Pengrui Lu, Pengfei Liu

## 🧩 Problem to Solve

현재 웹 검색 기능이 탑재된 대규모 언어 모델(LLM) 기반의 심층 연구 에이전트는 두 가지 주요 문제에 직면해 있습니다.

* **프롬프트 엔지니어링 기반 접근 방식**: 수동으로 설계된 프롬프트에 의존하여 성능이 취약하고, 지시를 따르거나 일관된 추론을 하기가 어려우며, 새로운 작업에 대한 일반화 능력이 떨어지고, 안정적인 성능을 위해 광범위한 수동 작업이 필요합니다.
* **RAG(Retrieval-Augmented Generation) 기반 강화 학습 에이전트**: 통제된 환경(고정된 코퍼스) 내에서 작동하므로 실제 웹 환경의 복잡성(노이즈, 비정형성, 동적 특성)을 제대로 반영하지 못합니다. 필요한 모든 정보가 고정된 지식 기반 내에 존재한다고 가정하며, 검색 품질의 가변성이나 다양한 웹 콘텐츠 형식을 다루지 못합니다.

따라서, 실제 웹 검색 상호작용이 가능한 **실제 환경에서 강화 학습(RL)을 확장하여 LLM 기반 심층 연구 에이전트를 엔드투엔드로 훈련하기 위한 포괄적인 프레임워크가 부재**하다는 것이 핵심 문제입니다.

## ✨ Key Contributions

* **DeepResearcher 도입**: 실제 웹 환경에서 LLM 기반 심층 연구 에이전트의 엔드투엔드 훈련을 위한 최초의 포괄적인 강화 학습 프레임워크를 제안합니다.
* **실제 환경에서의 RL 확장**: 기존 RAG 기반 접근 방식과 달리, 에이전트가 실제 검색 엔진과 직접 상호작용하며 개방형 웹의 내재된 가변성과 복잡성을 다루도록 훈련합니다.
* **엔드투엔드 훈련**: 인간의 선험적 지식 없이 모델을 엔드투엔드로 훈련하여, 에이전트가 자체적인 문제 해결 전략(예: 계획 수립, 교차 검증, 자기 성찰)을 발견하도록 합니다.
* **기술적 난관 극복**: API 호출 속도 제한, 네트워크 지연, 안티-크롤링 메커니즘, 다양한 웹 페이지 구조 처리 등 실제 웹 검색 환경에서 발생하는 고유한 기술적 문제를 해결합니다.
* **멀티 에이전트 아키텍처**: 전용 브라우징 에이전트가 전체 웹 페이지에서 관련 정보를 추출하는 전문화된 멀티 에이전트 아키텍처를 사용합니다.
* **획기적인 성능 향상**: 프롬프트 엔지니어링 기반 기준선 대비 최대 28.9포인트, RAG 기반 RL 에이전트 대비 최대 7.2포인트의 상당한 성능 개선을 달성했습니다.
* **인지 행동의 출현**: 엔드투엔드 RL 훈련을 통해 계획 수립, 여러 출처에서 정보 교차 검증, 연구 방향 재조정을 위한 자기 성찰, 명확한 답을 찾을 수 없을 때 정직하게 응답하는 능력 등 실제 연구자와 유사한 인지 행동이 나타남을 확인했습니다.

## 📎 Related Works

이 논문은 기존 LLM 검색 에이전트를 다음과 같이 분류하고 DeepResearcher의 위치를 설명합니다.

* **프롬프트 기반 검색 에이전트**:
  * **특징**: OpenResearcher, AirRAG, Search-o1 등과 같이 수동으로 설계된 워크플로우를 사용하여 LLM이 외부 지식 소스와 상호작용하는 방식을 정의합니다.
  * **한계**: 인간이 설계한 프롬프트에 의존하여 행동 패턴이 엄격하고 적응성이 제한적입니다.
* **훈련 기반 검색 에이전트**:
  * **지도 미세 조정(SFT)**: RAG 워크플로우를 최적화하는 데 사용되나, 높은 계산 오버헤드와 알려지지 않은 시나리오에 대한 약한 일반화 능력을 가집니다 (예: CoRAG).
  * **강화 학습(RL)**: Search-R1, ReSearch, R1-Searcher 등은 수동으로 지정된 신호 대신 모델이 자율적으로 추론을 개발하도록 합니다. 하지만 대부분 정적 로컬 텍스트 코퍼스에 중점을 둡니다. OpenAI의 심층 연구 에이전트도 RL을 사용하지만 상세 방법론은 공개되지 않았습니다.
* **훈련 환경**:
  * **로컬 RAG 환경**: 사전 구축된 고정 지식 저장소에 의존하여 정보의 적시성 부족, 도메인 적응성 미흡, 저장 효율성 병목 현상과 같은 문제가 발생합니다.
  * **실제 웹 검색 환경**: LLM을 개방형 검색 엔진과 통합하여 실시간 정보에 접근합니다. API 속도 제한, 네트워크 지연, 다양한 웹 페이지 구조 처리와 같은 고유한 문제를 수반하며, 외부 시스템 참여가 필요한 검색 기반 방법은 엔드투엔드 훈련이 거의 이루어지지 않습니다.

DeepResearcher는 기존 RL 방법이 정적 로컬 코퍼스에 집중하는 것과 달리, 강화 학습과 실제 웹 환경 훈련을 결합하여 동적인 웹의 복잡성을 직접 다룹니다.

## 🛠️ Methodology

DeepResearcher는 에이전트가 동적인 실제 웹 환경에서 문제를 해결할 수 있도록 훈련하기 위해 다음과 같은 방법론을 사용합니다.

* **심층 연구 궤적 (Deep Research Trajectory)**:
  * `<think>` 태그: 에이전트는 행동을 취하기 전에 추론 과정을 거칩니다.
  * `websearch` 도구: `websearch`라는 도구 이름과 검색 쿼리를 포함하는 JSON 형식의 요청을 생성하여 웹 검색 도구를 호출합니다. 검색 결과는 제목, URL, 스니펫의 구조화된 형식으로 반환됩니다.
  * `web browsing agent`: 각 쿼리에 대한 단기 메모리 저장소를 유지합니다. `browsewebpage` 요청을 받으면 URL의 첫 페이지 세그먼트를 처리하고, 다음 URL/세그먼트를 계속 읽을지 중지할지를 결정하며, 관련 정보를 단기 메모리에 추가합니다.
  * `<answer>` 태그: 충분한 정보를 확보했다고 판단되면 최종 답변을 생성합니다.
* **동적인 실제 웹 환경의 문제점 해결**:
  * **고동시성 요청 처리**: RL 롤아웃 과정에서 발생하는 대량의 검색 쿼리 및 웹 페이지 크롤링 작업을 관리하기 위해 50개의 노드로 구성된 분산 CPU 서버 클러스터를 구축했습니다.
  * **웹 크롤링 및 API 제한 관리**: API 호출 또는 웹 페이지 크롤링 중 발생하는 예외를 효과적으로 처리하기 위한 강력한 재시도 메커니즘을 구현하고, 동일한 검색 쿼리에 대해 미리 정해진 기간(예: 7일) 내에는 캐시에서 결과를 검색하는 캐싱 전략을 도입하여 API 호출 빈도와 비용을 줄입니다.
  * **멀티 에이전트 접근 방식을 통한 정보 추출 최적화**: 전용 읽기 에이전트를 사용하여 크롤링된 웹 페이지에서 관련 정보를 추출합니다. 긴 웹 페이지를 작은 세그먼트로 분할하고, 인간과 유사하게 콘텐츠를 순차적으로 처리하여 비생산적인 웹 페이지를 건너뜁니다.
* **강화 학습 훈련 프레임워크**:
  * **GRPO (Group Relative Policy Optimization)**: 현재 정책 $\pi_{\theta}$를 최적화하기 위해 기준 정책 $\pi_{\theta_{\text{ref}}}$와 기존 정책 $\pi_{\theta_{\text{old}}}$에서 생성된 롤아웃을 활용합니다. 다음 목적 함수를 최대화하여 정책을 최적화합니다:
    $$J(\theta) = E_{x \sim D, \{y_i\}_{i=1}^G \sim \pi_{\theta_{\text{old}}}(\cdot|x)} \left[ \frac{1}{G} \sum_{i=1}^G \min \left( \frac{\pi_{\theta}(y_i|x)}{\pi_{\theta_{\text{old}}}(y_i|x)} A_i, \text{clip} \left( \frac{\pi_{\theta}(y_i|x)}{\pi_{\theta_{\text{old}}}(y_i|x)}, 1-\epsilon, 1+\epsilon \right) A_i \right) - \beta D_{\text{KL}}(\pi_{\theta}||\pi_{\theta_{\text{ref}}}) \right]$$
  * **관찰 마스킹 (Masking Observations)**: 도구 출력은 모델이 생성해야 하는 결과가 아니므로, 훈련에 포함되지 않도록 마스킹 처리하고 모델의 응답만 훈련 과정에 기여하도록 합니다.
* **보상 (Reward)**:
  * 주요 보상 지표로 F1 점수를 사용합니다 (단답형 개방형 QA 데이터셋의 경우).
  * 형식 패널티: 형식이 올바르지 않으면 -1점의 패널티를 부여합니다.
  * F1 보상: 형식이 올바르면 생성된 답변의 정확도를 측정하는 단어 수준 F1 점수를 기반으로 보상을 부여합니다.
* **검색 종속 훈련 데이터 큐레이션**:
  * **개방형 QA 데이터 활용**: NaturalQuestions(NQ), TriviaQA(TQ)는 단일 홉 시나리오에, HotpotQA, 2WikiMultiHopQA(2Wiki)는 다중 홉 시나리오에 활용됩니다.
  * **데이터 오염 문제 해결**: 모델이 단순히 암기된 정보를 사용하는 것이 아니라 웹 검색 도구를 진정으로 활용하도록 훈련하기 위해 데이터 오염을 방지합니다.
  * **데이터 정제 및 오염 탐지**:
    * **저품질 질문 필터링**: 시기성, 주관성, 잠재적 유해성이 있는 질문을 DeepSeek-R1을 사용하여 필터링합니다.
    * **오염 탐지**: 각 질문에 대해 기본 모델에서 무작위로 10개의 응답을 샘플링하여 정답이 포함된 경우(pass@10) 해당 질문을 훈련 세트에서 제외합니다.
  * 최종 훈련 데이터셋은 80,000개의 예시로 구성되며, NQ:TQ:HotpotQA:2Wiki = 1:1:3:3의 비율로 다중 홉 시나리오에 중점을 둡니다 (75%).

## 📊 Results

* **성능 향상**:
  * DeepResearcher는 프롬프트 엔지니어링 기반 기준선 대비 **최대 28.9포인트**, RAG 기반 RL 에이전트 대비 **최대 7.2포인트** 향상된 연구 작업 완료율을 달성했습니다.
* **동일 도메인(In-domain) 결과 (표 1)**: NQ, TQ, HotpotQA, 2Wiki의 4개 데이터셋에서 MBE(Model-based Evaluation) 측정 시 DeepResearcher가 가장 높은 성능을 보였습니다. 특히 TQ와 2Wiki에서 상당한 격차로 기준선을 능가합니다. (Search-r1-base는 로컬 RAG 환경에서 직접 Wikipedia 코퍼스에 접근하여 훈련된 반면, DeepResearcher는 전체 인터넷을 탐색합니다.)
* **도메인 외(Out-of-domain) 결과 (표 2)**: MuSiQue, Bamboogle, PopQA의 3개 OOD 데이터셋에서 모든 기준선보다 지속적으로 우수한 성능을 보여, 다양한 출처에서 정보를 추론, 검색, 종합하는 일반화 가능한 기술을 성공적으로 학습했음을 입증했습니다. 특히 Bamboogle 데이터셋은 Wikipedia 외의 지식을 요구하는 질문이 많아 DeepResearcher가 로컬 RAG 기반 방법을 크게 능가했습니다.
* **실제 환경 훈련의 중요성**: Bamboogle 벤치마크에서 DeepResearcher는 로컬 RAG 기반 방법보다 월등히 우수하며, 심지어 로컬 RAG로 훈련된 R1-Searcher가 실제 웹 검색을 사용하더라도 DeepResearcher보다 성능이 훨씬 떨어졌습니다. 이는 RL 스케일링 훈련 중 실제 환경 사용이 정보 검색 및 종합 능력 개발에 결정적인 이점을 제공함을 보여줍니다.
* **훈련 동역학 (그림 4)**:
  * F1 점수는 강화 학습을 통해 점진적으로 향상됩니다 (0.375에서 약 0.55로 증가).
  * 훈련이 진행됨에 따라 어려운 질문(4-hop)에 필요한 도구 호출 횟수가 증가하며, 모델이 더 어려운 질문을 처리하기 위해 더 많은 정보를 검색하는 방법을 학습하고 있음을 시사합니다.
  * 응답 길이 또한 추론 복잡성과 함께 증가하며, 모든 설정에서 지속적인 상승 추세를 보여 모델이 훈련 중 추론 과정을 계속 확장하고 있음을 나타냅니다.

## 🧠 Insights & Discussion

DeepResearcher의 엔드투엔드 RL 훈련을 통해 다음과 같은 중요한 인지 행동이 나타났습니다.

* **계획 수립 (Planning)**: 다중 홉 질문을 다룰 때 계획을 수립하고 추론 과정 전반에 걸쳐 동적으로 조정할 수 있습니다. 이는 명시적인 계획 데이터에 대한 SFT 없이도 계획 능력이 자연스럽게 나타날 수 있음을 보여줍니다.
* **교차 검증 (Cross-validation)**: 첫 번째 도구 호출에서 정답을 식별한 후에도 즉시 결과를 확정하지 않고 후속 단계를 통해 정확성을 검증합니다. 이러한 신중한 접근 방식은 모델 응답의 신뢰성을 높여 최종 답변의 견고성을 보장합니다.
* **성찰 (Reflection)**: 검색된 정보가 질문과 완전히 일치하지 않을 때, 환경 피드백을 기반으로 이러한 불일치를 인식하고 후속 도구 호출에서 검색 쿼리를 개선합니다. 이 성찰 능력은 모델이 추론에서 막히는 것을 방지하는 데 필수적입니다.
* **정직성 (Honesty)**: 필요한 지식이 부족할 때 정답을 찾지 못했음을 인식하고 적절하게 답변 제공을 거부하여 환각을 최소화합니다. (현재 QA 평가 지표는 이러한 신뢰성 측면을 고려하지 않고 있습니다.)

이러한 결과는 실제 웹 환경과의 직접적인 상호작용이 견고한 연구 능력 개발에 단순한 구현 세부 사항이 아니라 근본적인 요구 사항임을 강조합니다. 엔드투엔드 훈련은 인간이 설계한 워크플로우를 넘어 에이전트가 자율적으로 문제 해결 전략을 발견하도록 지원하여 실제 응용 프로그램과 일치하는 강력한 연구 능력을 개발합니다.

## 📌 TL;DR

**문제**: 기존 LLM 기반 심층 연구 에이전트(프롬프트 기반 또는 RAG 기반 RL)는 정적 지식 기반이나 취약한 수동 워크플로우 때문에 실제 웹 환경의 복잡성에 취약합니다.
**방법**: DeepResearcher는 실제 웹 환경에서 LLM 에이전트의 엔드투엔드 강화 학습(GRPO)을 위한 최초의 포괄적인 프레임워크를 제안합니다. 이 프레임워크는 전문화된 브라우징 에이전트를 포함하는 멀티 에이전트 아키텍처를 사용하고, API 제한 및 크롤링과 같은 실제 환경의 난관을 관리하며, 데이터 오염 없는 훈련 데이터를 큐레이션합니다.
**발견**: DeepResearcher는 기존 방식 대비 상당한 성능 향상(프롬프트 기반 대비 최대 28.9포인트, RAG 기반 RL 대비 7.2포인트)과 강력한 일반화 능력을 보입니다. 특히, 계획, 교차 검증, 성찰, 정직성과 같은 인지 행동이 출현하며, 실제 웹 환경에서의 강화 학습 훈련이 견고한 연구 에이전트 개발에 필수적임을 입증했습니다.
