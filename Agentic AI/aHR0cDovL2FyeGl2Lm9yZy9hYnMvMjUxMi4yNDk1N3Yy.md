# AMAP Agentic Planning Technical Report

AMAP AI Agent LLM Team (2026)

## 🧩 Problem to Solve

본 연구가 해결하고자 하는 핵심 문제는 실제 세계의 **시공간적 이해(Spatio-temporal understanding)**가 필요한 복잡한 추론 작업의 수행이다. 구체적으로는 제약 조건이 포함된 관심 지점(POI, Point-of-Interest) 탐색이나 복잡한 여행 일정 계획(Itinerary planning)과 같은 과업을 의미한다.

이러한 시공간적 추론 작업은 단순한 지식 검색을 넘어, 여러 외부 도구를 유기적으로 호출하고 그 결과에 따라 계획을 동적으로 수정해야 하는 **System 2** 수준의 인지적 노력이 필요한 과업이다. 기존의 도구 통합 추론(TIR, Tool-Integrated Reasoning) 연구들은 주로 수학적 추론이나 코드 테스트와 같은 정형화된 도메인에 집중되어 있었으며, 실제 세계의 복잡한 제약 조건과 동적인 환경을 반영한 실용적인 해결책은 부족한 실정이다.

따라서 본 논문의 목표는 시공간적 시나리오에 최적화된 에이전틱 거대 언어 모델(Agentic LLM)인 **STAgent**를 구축하여, 복잡한 제약 조건 하에서도 정밀한 도구 호출과 추론을 수행하는 능력을 갖추게 하는 것이다.

## ✨ Key Contributions

STAgent의 핵심 설계 아이디어는 **"안정적인 환경 $\rightarrow$ 고품질 데이터 $\rightarrow$ 단계적 학습"**으로 이어지는 통합 파이프라인을 구축하는 것이다. 주요 기여 사항은 다음과 같다.

1. **강건한 인터랙티브 환경 구축**: 10개 이상의 도메인 특화 도구를 지원하며, 비동기 롤아웃(Asynchronous rollout)과 학습을 가능하게 하는 ROLL 인프라와 FastMCP 프로토콜을 도입하여 학습 효율성을 80% 향상시켰다.
2. **계층적 데이터 큐레이션 프레임워크**: 3,000만 개의 방대한 무감독(Unsupervised) 로그 데이터에서 다양성과 난이도가 높은 상위 1% 미만의 고품질 쿼리만을 정밀하게 추출하는 self-evolving 선택 프레임워크를 제안하였다.
3. **캐스케이드 학습 레시피(Cascaded Training Recipe)**: Seed SFT를 통해 쿼리의 난이도를 측정하고, 확신도가 높은 샘플로 SFT를 수행한 후, 난이도가 높고 불확실한 샘플을 대상으로 RL(강화학습)을 진행하는 **SFT-Guided RL** 패러다임을 설계하였다.

## 📎 Related Works

본 논문은 기존의 도구 통합 추론(TIR) 연구들이 주로 수학 및 코딩 도메인에 치중되어 있다는 점을 한계로 지적한다. 실제 세계의 시공간 추론은 다음과 같은 차별점을 가진다.

- **인지 비용의 차이**: 단순 검색이 아닌, 복잡한 제약 조건을 최적화해야 하는 System 2 추론이 필수적이다.
- **환경의 불확실성**: 실시간 교통, 날씨, 예약 가능 여부 등 동적인 외부 데이터와의 상호작용이 중요하며, 이는 정적인 벤치마크 데이터셋만으로는 해결하기 어렵다.
- **데이터의 특성**: 실제 사용자 로그는 매우 노이즈가 많고 비정형적이며, 학습에 직접 사용하기에는 난이도나 카테고리 정보가 부족하다.

## 🛠️ Methodology

### 1. 환경 구축 (Environment Construction)

STAgent는 **FastMCP** 기반의 샌드박스 환경에서 동작하며, 총 4개 카테고리의 10가지 전문 도구를 사용한다.

- **지도 및 내비게이션**: POI 검색, 경로 계산, 경로 주변 검색, 최적 만남 장소 찾기, 랭킹 리스트 쿼리.
- **여행**: 항공편 검색, 기차 시간표 검색.
- **날씨**: 실시간 기상 및 AQI, 다일 예보.
- **정보 검색**: 오픈 도메인 웹 검색.

### 2. 데이터 큐레이션 (Prompt Curation)

방대한 로그 데이터에서 고품질 데이터를 추출하기 위해 다음과 같은 계층적 과정을 거친다.

- **의도 분류 체계(Intent Taxonomy)**: Seed 프롬프트에서 시작하여 LLM과 전문가의 피드백을 통해 5개 대분류, 16개 중분류, 30개 리프 노드로 구성된 체계를 구축하였다.
- **깔때기 필터링(Funnel Filtering)**:
  - **어휘적 중복 제거**: Locality-Sensitive Hashing(LSH)을 사용하여 중복 문자열 제거.
  - **의미적 중복 제거**: Faiss 기반의 임베딩 유사도 검색을 통해 의미적으로 유사한 샘플 제거.
  - **기하학적 중복 제거**: K-Center-Greedy 알고리즘을 통해 임베딩 공간에서 가장 대표성 있는 샘플을 선택하여 롱테일 데이터를 보존하였다.
- **난이도 측정**: 도구 선택의 모호성, 실행 체인의 깊이, 제약 조건의 복잡성을 기준으로 $-1$부터 $5$까지의 점수를 부여한다.

### 3. 보상 설계 (Reward Design)

**Rubrics-as-reward** 방식을 채택하여 세 가지 차원에서 보상을 산출한다.

- **차원**: 추론 및 선제적 계획(Reasoning & Proactive Planning), 정보 충실도 및 통합(Information Fidelity & Integration), 표현 및 서비스 루프(Presentation & Service Loop).
- **동적 가중치**: 작업 유형(복잡한 계획, 정보 검색, 상담 및 설명)에 따라 각 차원의 가중치 $w$를 다르게 설정한다.
- **환각 거부(Hallucination Veto)**: 사실 관계에 대한 환각이 발견될 경우, 다른 점수와 관계없이 최종 보상을 $0$으로 처리하는 Hard Veto 메커니즘을 적용한다.
$$R = \mathbb{1}_{H=0} \times \sum_{k \in \{reas, info, pres\}} w_k \cdot s_k$$

### 4. 학습 절차 (Training Recipe)

#### 4.1 Supervised Fine-Tuning (SFT)

기본 모델을 전문 에이전트로 전환하는 단계로, DeepSeek-R1을 이용해 생성한 고품질 궤적(Trajectory)을 사용하여 학습한다. 손실 함수는 다음과 같은 Negative Log-Likelihood(NLL)를 사용한다.
$$\mathcal{L}_{SFT}(o_i | \theta) = \frac{1}{C} \sum_{t=1}^{|o_i|} I(o_i, t) \log \pi_\theta(o_i, t | q_i)$$
여기서 $I(o_i, t)$는 도구 관측값(Observation) 부분을 마스킹하여 모델이 도구의 출력값을 예측하는 것이 아니라 추론 과정만을 학습하도록 유도하는 지표 함수이다.

#### 4.2 동적 능력 기반 커리큘럼 (Dynamic Capability-Aware Curriculum)

모델의 현재 능력에 따라 학습 데이터를 선택하는 전략이다.

- **학습 가능성 점수(Learnability Potential Score)**: $\hat{\mu}_i$(평균 보상)와 $\hat{\sigma}_i^2$(보상 분산)를 이용해 $S_i = \hat{\sigma}_i^2 \cdot \hat{\mu}_i$를 계산한다.
- **영역 구분**:
  - **Trivial**: 확신도가 높고 보상이 높은 영역 ($\nabla L \approx 0$).
  - **Noise**: 모델의 능력 밖이거나 데이터가 오염된 영역.
  - **Learnable**: 보상이 존재하지만 분산이 큰 영역. 이 영역의 데이터만을 선택하여 학습 효율을 극대화한다.

#### 4.3 Reinforcement Learning (RL)

SFT 모델을 초기값으로 하여 **GRPO(Group Relative Policy Optimization)**의 변형인 **GSPO(Group Sequence Policy Optimization)**를 적용한다. GSPO는 토큰 단위가 아닌 시퀀스 단위의 최적화 제약을 부여하여 학습을 안정화한다.

- **시퀀스 비율(Importance Ratio)**:
$$s_i(\theta) = \exp \left( \frac{1}{|o_i|} \sum_{t=1}^{|o_i|} \log \frac{\pi_\theta(o_i, t | q, o_{i, <t})}{\pi_{\theta_{old}}(o_i, t | q, o_{i, <t})} \right)$$

## 📊 Results

### 1. 실험 설정

- **내부 평가**: 1,000개의 고품질 쿼리를 이용한 온라인 벤치마크 및 **TravelBench** 오프라인 벤치마크 수행.
- **일반 능력 평가**: Tool Use(ACEBench, $\tau^2$-Bench, BFCL v3), 수학(AIME), 코딩(LiveCodeBench), 일반 지식(MMLU-Pro, C-Eval) 등을 통해 성능 측정.

### 2. 주요 결과

- **도메인 특화 성능**: TravelBench에서 STAgent는 전체 점수 $70.3$을 기록하며, 파라미터 수가 훨씬 많은 DeepSeek R1($64.7$)과 Qwen3-235B-Instruct($69.9$)를 상회하는 성능을 보였다. 특히 Multi-turn 작업에서 $66.6$으로 최고 성능을 달성하였다.
- **온라인 벤치마크**: Qwen3-30B-Thinking 모델 대비 요약/추출(Win rate: 81.9%) 및 표현(Win rate: 73.7%) 차원에서 압도적인 우위를 점하였다.
- **일반화 능력**: 도메인 특화 학습 이후에도 수학, 코딩, 일반 지식 벤치마크에서 성능 저하가 거의 없거나 오히려 일부 향상(BFCL v3: +4.4)되어, 특수 도메인 학습이 일반적인 도구 사용 능력의 전이(Transfer)를 유도했음을 입증하였다.

## 🧠 Insights & Discussion

**강점 및 성과**:
본 연구는 단순히 많은 데이터를 사용하는 대신, 극소수의 고품질 데이터를 정밀하게 큐레이션($<1\%$ 필터링 비율)하고 모델의 현재 능력 수준에 맞춘 **Dynamic Curriculum**을 적용함으로써 학습 효율을 극대화하였다. 특히, 전문 도메인 에이전트를 학습시켰음에도 일반 도메인 벤치마크 성능이 유지되었다는 점은, 시공간 추론 학습이 LLM의 전반적인 추론 및 도구 활용 능력을 향상시키는 일종의 '추론 훈련'으로 작용했음을 시사한다.

**한계 및 논의**:

- **모델 규모의 제약**: 온라인 벤치마크에서 최상위 모델(Gemini-3-Pro 등)과의 성능 격차는 여전히 존재하며, 이는 30B 규모의 모델이 가진 기본 파라미터 용량의 한계로 해석된다.
- **데이터 의존성**: 고품질 궤적 생성을 위해 DeepSeek-R1과 같은 강력한 Teacher 모델에 의존하고 있으며, Teacher 모델의 편향이 STAgent에 전이되었을 가능성이 있다.
- **실시간성**: API 호출 레이턴시를 줄이기 위해 LRU 캐싱을 도입하였으나, 완전한 실시간 환경에서의 응답 속도 최적화에 대한 논의는 부족하다.

## 📌 TL;DR

STAgent는 실제 세계의 복잡한 시공간적 추론 문제를 해결하기 위해 설계된 에이전틱 LLM이다. **강건한 비동기 도구 환경(ROLL/FastMCP)**, **극소수 고품질 데이터 큐레이션**, 그리고 **SFT-Guided RL 커리큘럼**을 통해 구축되었다. 실험 결과, TravelBench 등 시공간 특화 작업에서 초대형 모델들을 능가하는 성능을 보였으며, 동시에 일반적인 추론 및 도구 사용 능력을 보존하거나 향상시키는 성과를 거두었다. 이 연구는 특정 전문 도메인을 위한 에이전트 구축 시 데이터의 양보다 질과 학습 단계의 정밀한 설계가 더 중요하다는 것을 입증하며, 향후 다른 복잡한 실세계 도메인 에이전트 개발의 표준 프레임워크가 될 가능성이 높다.
