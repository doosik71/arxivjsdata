# METAGPT: METAPROGRAMMING FOR A MULTI-AGENT COLLABORATIVE FRAMEWORK

Sirui Hong, Mingchen Zhuge, Jiaqi Chen, Xiawu Zheng, Yuheng Cheng, Ceyao Zhang, Jinlin Wang, Zili Wang, Steven Ka Shing Yau, Zijuan Lin, Liyang Zhou, Chenyu Ran, Lingfeng Xiao, Chenglin Wu, Jürgen Schmidhuber

## 🧩 Problem to Solve

기존의 LLM(Large Language Model) 기반 다중 에이전트 시스템은 단순한 대화형 작업은 해결할 수 있지만, 복잡한 태스크에서는 LLM의 단순한 연결로 인한 연쇄적 환각(cascading hallucinations)과 논리적 불일치(logic inconsistencies) 때문에 효과적이고 일관되며 정확한 문제 해결에 어려움을 겪습니다. 특히 의미 있는 협업이 필요한 경우 이러한 문제가 더욱 두드러집니다.

## ✨ Key Contributions

* **MetaGPT 프레임워크 제안**: LLM 기반 다중 에이전트 협업을 위한 메타 프로그래밍 프레임워크인 MetaGPT를 소개합니다. 이는 역할 정의 및 메시지 공유와 같은 잘 정의된 기능을 통해 LLM 기반 다중 에이전트 시스템 개발을 위한 편리하고 유연한 플랫폼을 제공합니다.
* **인간과 유사한 SOP 통합**: MetaGPT 설계 전반에 걸쳐 인간과 유사한 표준 운영 절차(SOPs)를 혁신적으로 통합하여 시스템의 견고성을 크게 향상시키고 LLM 기반 에이전트 간의 비생산적인 협업을 줄입니다.
* **실행 가능한 피드백 메커니즘 도입**: 런타임에 코드를 디버깅하고 실행하여 코드 생성 품질을 크게 향상시키는(예: MBPP에서 5.4% 절대 개선) 새로운 실행 가능한 피드백 메커니즘을 도입합니다.
* **최첨단 성능 달성**: HumanEval 및 MBPP 벤치마크에서 새로운 최첨단(SoTA) 성능을 달성하여 MetaGPT가 LLM 기반 다중 에이전트 시스템 개발을 위한 유망한 메타 프로그래밍 프레임워크임을 입증합니다.

## 📎 Related Works

* **자동 프로그래밍**: PROW, Balzer, Soloway의 초기 연구부터 CodeBERT, CodeLlama, WizardCoder와 같은 최신 NLP 기반 접근 방식까지 아우릅니다. ReAct, Reflexion, ToolFormer, ToolLLM과 같은 LLM 기반 에이전트 연구들도 자동 프로그래밍 발전에 기여했습니다. 특히 Li et al. (2023)과 Qian et al. (2023)은 역할극(role-play) 및 다중 에이전트 소프트웨어 개발 프레임워크를 제안했지만, MetaGPT는 구조화된 출력 형식과 효과적인 워크플로우를 통해 복잡한 문제를 더 잘 처리합니다.
* **LLM 기반 다중 에이전트 프레임워크**: Generative Agents (Park et al., 2023), NLSOM (Zhuge et al., 2023), Stable-Alignment (Liu et al., 2023a) 등 LLM의 문제 해결 능력을 향상시키기 위해 여러 에이전트 간의 토론 및 상호작용을 통합한 연구들이 있습니다. 그러나 MetaGPT는 "어시스턴트 반복 지시" 또는 "메시지 무한 루프"와 같은 다중 에이전트의 일반적인 문제를 해결하기 위해 SOP 적용에 중점을 둡니다.

## 🛠️ Methodology

MetaGPT는 LLM 기반 다중 에이전트 시스템을 위한 메타 프로그래밍 프레임워크로, 다음 구성 요소들을 통합합니다.

1. **SOP 기반 에이전트 구성**:
    * **역할 전문화 (Specialization of Roles)**: 복잡한 작업을 더 작고 구체적인 하위 작업으로 분해하기 위해 역할이 명확하게 전문화됩니다. 소프트웨어 개발 회사를 시뮬레이션하여 Product Manager, Architect, Project Manager, Engineer, QA Engineer와 같은 5가지 역할을 정의합니다. 각 에이전트는 고유한 프로필(이름, 프로필, 목표, 제약 조건)과 특정 컨텍스트 및 기술(예: Product Manager는 웹 검색, Engineer는 코드 실행)을 가집니다.
    * **에이전트 간 워크플로우 (Workflow across Agents)**: 소프트웨어 개발의 SOP를 따라 모든 에이전트가 순차적으로 작업하도록 기본 워크플로우를 설정합니다.
        * **Product Manager**: 사용자 요구사항 분석 후 사용자 스토리 및 요구사항 풀을 포함하는 PRD(Product Requirements Document)를 생성합니다.
        * **Architect**: PRD를 기반으로 파일 목록, 데이터 구조, 인터페이스 정의 등 시스템 설계 구성 요소를 만듭니다.
        * **Project Manager**: 설계 정보를 바탕으로 프로젝트를 작업 목록으로 분해하고 각 코드 파일을 엔지니어에게 할당합니다.
        * **Engineer**: 할당된 클래스와 함수를 구현하고 코드를 작성합니다.
        * **QA Engineer**: 엔지니어가 생성한 코드에 대한 단위 테스트 코드를 작성하고 버그를 식별 및 수정합니다.

2. **통신 프로토콜 (Communication Protocol)**:
    * **구조화된 통신 인터페이스 (Structured Communication Interfaces)**: 무제한 자연어 대신 구조화된 통신을 사용하여 에이전트 간 정보 왜곡 및 손실을 줄입니다. 각 역할에 대한 스키마와 형식을 설정하고 필요한 출력을 특정 역할과 컨텍스트에 따라 제공하도록 요청합니다 (예: Architect는 시스템 인터페이스 설계 및 시퀀스 흐름 다이어그램 생성).
    * **발행-구독 메커니즘 (Publish-Subscribe Mechanism)**: 모든 에이전트가 구조화된 메시지를 게시하고 다른 에이전트의 메시지에 투명하게 접근할 수 있는 공유 메시지 풀을 도입합니다. 정보 과부하를 방지하기 위해 에이전트는 역할별 관심사에 따라 관련 정보를 구독합니다.

3. **실행 가능한 피드백을 통한 반복 프로그래밍 (Iterative Programming with Executable Feedback)**:
    * LLM 환각으로 인한 오류를 극복하기 위해 초기 코드 생성 후 실행 가능한 피드백 메커니즘을 도입합니다.
    * Engineer 에이전트는 코드를 작성한 후 해당 단위 테스트 케이스를 실행하고 테스트 결과를 받습니다.
    * 오류가 발생하면, 에이전트는 과거 메시지 메모리를 확인하고 PRD, 시스템 설계, 코드 파일과 비교하여 코드를 디버깅하고 개선합니다.
    * 이 반복적인 테스트 과정은 테스트 통과 또는 최대 3회 재시도까지 계속됩니다.

## 📊 Results

* **HumanEval 및 MBPP 벤치마크**: MetaGPT는 단일 시도(Pass@1)에서 HumanEval 85.9%, MBPP 87.7%를 달성하여 모든 이전 접근 방식(GPT-4의 67.0% 및 82.3% 대비)을 능가하는 최첨단 성능을 기록했습니다.
* **SoftwareDev 벤치마크**: 복잡한 소프트웨어 프로젝트에 대한 MetaGPT의 성능은 ChatDev와 비교하여 거의 모든 지표에서 우수합니다 (표 1 참조).
  * **실행 가능성 (Executability)**: MetaGPT는 3.75점(거의 완벽)을 달성했으며, ChatDev의 2.25점보다 훨씬 높습니다.
  * **시간 및 비용 효율성**: MetaGPT는 ChatDev(762초)보다 적은 시간(503초)이 소요됩니다. 비록 더 많은 토큰(24,613-31,255개)을 사용하지만, 코드 라인당 토큰 사용량(생산성)은 MetaGPT가 124.3-126.5개로 ChatDev의 248.9개보다 훨씬 효율적입니다.
  * **인간 수정 비용 (Human Revision Cost)**: MetaGPT는 평균 0.83회로 ChatDev의 2.5회에 비해 인간의 개입이 훨씬 적습니다.
* **능력 분석**: MetaGPT는 PRD 생성, 기술 설계 생성, API 인터페이스 생성, 역할 기반 작업 관리, 코드 검토 등 소프트웨어 엔지니어링 작업을 위한 광범위한 기능을 제공하며, AutoGPT, LangChain, AgentVerse, ChatDev와 같은 다른 프레임워크보다 포괄적입니다 (표 2 참조).
* **어블레이션 스터디 (Ablation Study)**:
  * **역할의 효과**: Engineer 단독일 때보다 Product Manager, Architect, Project Manager 등 다양한 역할을 추가할수록 코드 수정 횟수와 실행 가능성이 꾸준히 향상되어 각 역할의 효과를 입증합니다 (표 3 참조).
  * **실행 가능한 피드백 메커니즘의 효과**: 실행 가능한 피드백을 추가함으로써 HumanEval에서 4.2%, MBPP에서 5.4%의 Pass@1이 크게 개선되었습니다. 또한 SoftwareDev에서 실행 가능성(3.67 $\rightarrow$ 3.75)과 인간 수정 비용(2.25 $\rightarrow$ 0.83)이 향상되었습니다 (표 1, 그림 4 참조).
* **다른 LLM 백엔드 사용**: GPT-3.5 및 Deepseek Coder 33B를 백엔드로 사용해도 작업 완료가 가능하지만, GPT-4를 사용할 때 가장 우수한 성능을 보였습니다 (표 5 참조).
* **명령어 수준의 영향**: 상세한 명령어는 요구사항이 명확하여 더 나은 소프트웨어 프로젝트와 더 낮은 생산성 비율을 이끌었지만, MetaGPT는 높은 수준의 간단한 입력으로도 실행 가능성 3.8점의 충분히 좋은 소프트웨어를 생성할 수 있었습니다 (표 6 참조).

## 🧠 Insights & Discussion

* **SOPs의 중요성**: MetaGPT는 인간의 표준 운영 절차(SOPs)를 LLM 기반 다중 에이전트 시스템에 통합하는 것이 복잡한 문제 해결 능력을 향상시키는 데 매우 효과적임을 입증했습니다. 이는 에이전트 간의 비생산적인 "잡담(idle chatter)"을 줄이고 구조화된 협업을 가능하게 합니다.
* **실행 가능한 피드백의 가치**: 코드 생성 과정에 런타임 디버깅 및 최적화를 위한 실행 가능한 피드백 메커니즘을 통합함으로써 LLM의 환각 문제를 줄이고 코드의 실행 가능성 및 품질을 크게 향상시킬 수 있습니다. 이는 기존의 비실행 코드 리뷰 및 자기 반성 방식의 한계를 극복합니다.
* **정보 과부하 및 컨텍스트 관리**: 공유 메시지 풀과 구독 메커니즘을 통해 정보 과부하 문제를 해결하고, 에이전트가 관련 정보에 집중할 수 있도록 하여 통신 효율성을 높입니다.
* **미래 연구 방향**: 본 연구는 인간에서 영감을 받은 기술을 인공 다중 에이전트 시스템에 적용하는 초기 시도이며, LLM 기반 다중 에이전트 프레임워크를 규제하는 방향으로 나아갈 수 있습니다. 향후 에이전트가 프로젝트 경험으로부터 학습하고 시간이 지남에 따라 더 나은 학습 알고리즘을 메타 학습하는 자기 개선 메커니즘(recursive self-improvement)과 동적 SOP 변경 및 에이전트 경제 모델(multi-agent economies) 통합 가능성을 제시합니다.
* **제한 사항**: 현재 시스템은 UI 및 프론트엔드와 같은 특정 시나리오를 완전히 처리하지 못하며, 실제 애플리케이션의 다양하고 복잡한 요구사항을 충족하기에는 여전히 도전 과제가 있습니다. 또한 사용자 측면에서 에이전트의 실행 과정을 중단하거나 체크포인트를 설정하는 기능은 아직 미흡합니다.
* **윤리적 고려사항**: MetaGPT는 자연어를 통한 프로그래밍을 더 쉽게 만들어 프로그래밍 접근성을 높일 수 있으며, 오픈 소스 프레임워크로 투명성과 책임성을 보장합니다. 또한 사용자 데이터를 로컬에서 처리하여 개인 정보 보호 및 데이터 보안을 강화합니다.

## 📌 TL;DR

MetaGPT는 LLM 기반 다중 에이전트 시스템에서 발생하는 복잡한 작업의 비일관성과 환각 문제를 해결하기 위해 고안된 메타 프로그래밍 프레임워크입니다. 인간의 **표준 운영 절차(SOPs)** 를 에이전트 역할 전문화, 구조화된 통신, 발행-구독 메커니즘에 통합하여 효율적인 협업 워크플로우를 구축합니다. 또한 **실행 가능한 피드백**을 통해 런타임에 코드를 디버깅하고 개선합니다. 이 접근 방식을 통해 MetaGPT는 HumanEval 및 MBPP 벤치마크에서 최첨단 성능을 달성했으며, 복잡한 소프트웨어 개발 작업에서 기존 다중 에이전트 시스템보다 훨씬 뛰어난 효율성과 견고성을 입증했습니다.
