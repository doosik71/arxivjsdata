# Agent Network Protocol (ANP) Technical White Paper

Gaowei Chang, Eidan Lin, Chengxuan Yuan, Rizhao Cai, Binbin Chen, Xuan Xie, Yin Zhang (2025)

## 🧩 Problem to Solve

본 논문은 대규모 언어 모델(LLM)과 자율 의사결정 AI의 발전으로 인해 AI 에이전트가 인터넷의 새로운 주체로 부상하고 있음에도 불구하고, 기존의 인터넷 인프라가 인간의 상호작용을 중심으로 설계되어 발생 하는 한계점을 해결하고자 한다. 구체적인 문제는 다음과 같다.

첫째, 데이터 사일로(Data Silos)와 컨텍스트 제한 문제이다. 현재의 인터넷 플랫폼들은 서로 단절되어 있어 데이터가 개별 애플리케이션이나 서버에 고립되어 있으며, 이는 에이전트가 최적의 의사결정을 내리는 데 필요한 종합적인 컨텍스트 정보에 접근하는 것을 방해한다.

둘째, 비효율적인 인터페이스 문제이다. 기존 웹 서비스는 GUI(Graphic User Interface) 기반의 인간 중심 설계로 되어 있어, 에이전트가 이를 이용하려면 인간의 조작을 시뮬레이션해야 하며, 이는 높은 오류 가능성과 비효율성을 초래한다.

셋째, 에이전트 간 협업 비용의 증가이다. 서로 다른 에이전트 간의 직접적인 통신 메커니즘이 부족하여, 자연어를 통한 협상이 가능함에도 불구하고 표준 프로토콜의 부재로 인해 자율적인 조직화와 협업이 어렵다.

결과적으로 본 논문의 목표는 이러한 제약을 극복하고 에이전트 간의 개방적이고 통일된 연결 메커니즘을 제공하는 'Agent Network Protocol (ANP)'을 제안하여, 이른바 'Agentic Web'의 기반을 마련하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 에이전트 간의 상호운용성과 협업 효율성을 극대화하기 위해 AI-Native하게 설계된 새로운 통신 프로토콜인 ANP를 제안한 것이다. 

중심적인 설계 아이디어는 기존 인터넷 프로토콜과의 호환성을 유지하면서도, 에이전트가 인간의 개입 없이 서로의 신원을 확인하고, 통신 방식을 동적으로 협상하며, 능력을 탐색할 수 있도록 하는 3계층 프로토콜 아키텍처를 구축하는 것이다. 특히, 정적인 인터페이스가 아닌 AI가 직접 생성한 코드와 메타 프로토콜을 통해 통신 방식을 최적화한다는 점이 핵심적인 직관이다.

## 📎 Related Works

논문에서는 기존의 신원 인증 및 통신 방식의 한계를 언급한다. 

우선, OAuth 2.0과 같은 기존의 중앙 집중식 신원 인증 기술은 교차 시스템 간의 인증 과정이 복잡하고 탈중앙화 수준이 낮아 에이전트 네트워크에 적합하지 않다고 분석한다. 또한, 블록체인 기반의 탈중앙화 신원 증명(DID) 솔루션은 잠재력이 크지만, 대규모 적용 시 확장성(Scalability) 문제가 발생한다는 점을 지적한다.

ANP는 이러한 한계를 극복하기 위해 W3C의 Decentralized Identifier (DID) 표준을 채택하되, 기존의 성숙한 웹 인프라(HTTPS, DNS 등)를 활용하는 `did:wba` 방식을 제안함으로써 구현 난이도를 낮추고 상호운용성을 확보하며 기존 접근 방식과 차별화를 둔다.

## 🛠️ Methodology

ANP는 하위 계층부터 상위 계층으로 구성된 3계층 아키텍처를 통해 시스템을 구축한다.

### 1. Identity and Secure Communication Layer (신원 및 보안 통신 계층)
이 계층은 에이전트 간의 상호 신원 확인과 암호화 통신을 담당한다.
- **신원 인증**: W3C DID 표준을 기반으로 하며, 특히 `did:wba` (Web-Based Agent) 메서드를 제안한다. 이는 중앙 집중식 기술(클라우드 등)을 활용하면서도 HTTP 프로토콜을 통해 상호운용성을 확보하는 방식이다.
- **인증 절차**: 클라이언트는 첫 HTTP 요청 시 헤더에 DID와 서명(Signature)을 포함한다. 서버는 DID 문서의 공개키를 통해 이를 검증하고 토큰을 발급하여 이후 통신의 효율성을 높인다.
- **보안 통신**: Elliptic Curve Diffie-Hellman Ephemeral (ECDHE) 프로토콜을 사용하여 종단간 암호화(End-to-End Encryption)를 구현함으로써 중간 노드가 내용을 해독할 수 없도록 설계한다.

### 2. Meta-Protocol Layer (메타 프로토콜 계층)
특정 데이터 전송이 아닌, '어떻게 통신할 것인가'를 결정하는 프로토콜 협상 계층이다.
- **동적 협상 과정**: 
    1. **요청**: 에이전트 A가 자연어로 요구사항, 입력/출력 형태, 후보 프로토콜을 제안한다.
    2. **협상**: 에이전트 B가 AI를 통해 이를 분석하고 수락 여부를 결정하거나 대안을 제시한다.
    3. **코드 생성 및 배포**: 합의된 프로토콜을 바탕으로 양측이 AI 코드 생성 기술을 통해 실제 처리 코드를 생성하고 배포한다.
    4. **테스트 및 적용**: 테스트 데이터를 통해 디버깅 후 정식 통신을 시작한다.
- **최적화**: 매번 협상하는 비용을 줄이기 위해, 유사한 협상 결과는 저장하여 향후 재사용하거나 다른 에이전트와 공유한다.

### 3. Application Protocol Layer (애플리케이션 프로토콜 계층)
에이전트의 능력을 정의하고 발견하는 두 가지 핵심 모듈로 구성된다.
- **Agent Description Protocol (ADP)**: 에이전트의 '명함' 역할을 하는 표준화된 자기 기술 방법이다. JSON-LD 형식을 사용하며, 에이전트 이름, 고유 식별자, 제공 서비스(Capability), 호출 가능한 API 엔드포인트, 보안 및 인증 방법 등을 포함한다.
- **Agent Discovery Protocol**: 에이전트를 찾기 위한 검색 메커니즘이다.
    - **Active Discovery**: 도메인의 특정 경로(`/.well-known/agent-descriptions`)에 접속하여 해당 도메인 내의 모든 에이전트 목록을 가져오는 방식이다.
    - **Passive Discovery**: 에이전트가 전문 검색 서비스(Search Service Agent)에 자신의 AD 문서 URL을 등록하고, 검색 서비스가 이를 인덱싱하여 제공하는 방식이다.

### 4. Security and Privacy (보안 및 개인정보 보호)
- **권한 분리**: `humanAuthorization` 메서드를 도입하여, 단순 조회와 같은 저위험 작업은 에이전트가 자율적으로 처리하게 하고, 자금 이체와 같은 고위험 작업은 반드시 인간의 명시적 승인(생체 인식 등)을 거쳐야만 서명 키에 접근할 수 있도록 설계하였다.
- **개인정보 보호**: 주 DID와 서비스별 서브 DID를 분리하는 Multi-DID 전략을 사용하며, 최소 정보 공개 원칙(Minimal Disclosure Principle)을 준수한다.

## 📊 Results

본 논문은 새로운 프로토콜의 아키텍처를 제안하는 기술 백서(Technical White Paper)의 성격을 띠고 있다. 따라서 특정 데이터셋을 이용한 정량적인 성능 평가나 벤치마크 실험 결과는 제시되지 않았다. 대신, 제안된 프로토콜이 기존 웹 인프라(DNS, HTTPS, DID 등) 위에서 어떻게 동작하고 배포될 수 있는지에 대한 기술적 타당성과 구조적 설계안을 중심으로 설명하고 있다.

## 🧠 Insights & Discussion

본 논문이 제시하는 ANP의 강점은 '실용적인 배포 가능성(Pragmatic Deployability)'에 있다. 완전히 새로운 네트워크 망을 구축하는 대신, 기존의 HTTP, DNS, W3C DID와 같은 검증된 표준을 래핑(Wrapping)하여 AI-Native한 시맨틱 계층을 추가함으로써 도입 장벽을 크게 낮추었다.

다만, 몇 가지 비판적 검토 사항이 존재한다. 첫째, 메타 프로토콜 계층에서 AI가 직접 코드를 생성하고 배포하는 과정의 안정성과 보안성 문제이다. AI가 생성한 코드에 취약점이 있을 경우, 이를 동적으로 실행하는 환경에서 심각한 보안 사고가 발생할 수 있다. 둘째, 메타 프로토콜 협상 결과의 공유를 위한 경제적 인센티브 메커니즘이 아직 구체적으로 정의되지 않았다는 점이다. 에이전트들이 자신의 효율적인 협상 결과를 무상으로 공유할 동기가 부족하므로, 이에 대한 보상 체계 연구가 병행되어야 할 것이다.

결론적으로 본 연구는 인터넷의 중심축을 '플랫폼'에서 '프로토콜'로 이동시키려는 시도로, 에이전트 중심의 개방형 생태계 구축을 위한 이론적, 구조적 토대를 제공하였다는 점에서 의미가 크다.

## 📌 TL;DR

본 논문은 인간 중심의 웹 환경을 AI 에이전트 중심의 'Agentic Web'으로 전환하기 위한 새로운 통신 표준인 **Agent Network Protocol (ANP)**을 제안한다. ANP는 **신원 인증(DID) $\rightarrow$ 프로토콜 협상(Meta-Protocol) $\rightarrow$ 능력 기술 및 발견(ADP/Discovery)**으로 이어지는 3계층 구조를 통해, 에이전트들이 상호 신원을 확인하고 자율적으로 통신 방식을 결정하며 협업할 수 있는 AI-Native 데이터 네트워크를 구축하고자 한다. 이는 미래에 폐쇄적인 플랫폼 생태계를 깨고 개방적인 에이전트 협업 네트워크를 형성하는 핵심 인프라가 될 가능성이 높다.
