# Dingtalk-DeepResearch: A Unified Multi-Agent Framework for Adaptive Intelligence in Enterprise Environments

Industrial Brain Team, Dingtalk, Alibaba Group (2025)

## 🧩 Problem to Solve

본 논문은 실제 기업 환경(Enterprise Environments)에서 발생하는 복잡하고 가변적인 태스크를 수행하기 위한 지능형 에이전트 프레임워크의 부재를 해결하고자 한다. 일반적인 딥 리서치 시스템은 대규모 공개 데이터셋에서는 효율적이지만, 실제 기업 환경에서는 다음과 같은 세 가지 핵심 문제에 직면한다.

첫째, 데이터의 **이질성(Heterogeneity)** 문제이다. 기업 데이터는 긴 형태의 문서, 반정형 표(Table), 지식 그래프, 멀티미디어 콘텐츠 등 다양한 형태로 흩어져 있어, 이를 통합적으로 추론하고 합성하는 능력이 필수적이다. 

둘째, **정적인 아키텍처(Static Architecture)**의 한계이다. 기존 시스템들은 고정된 프롬프트나 스크립트에 의존하며, 실제 사용자의 피드백이나 변화하는 환경에 맞춰 스스로 진화하는 메커니즘이 부족하다.

셋째, **표 추론과 텍스트 합성의 단절**이다. 정형 데이터(표)에 대한 심볼릭 추론 결과가 비정형 텍스트 생성 과정과 유기적으로 결합되지 않아, 정교한 기업용 보고서 작성이 어렵다는 점이다.

결과적으로 본 논문의 목표는 딥 리서치 생성, 이질적 표 추론, 멀티모달 보고서 합성을 통합하고, 스스로 진화하는 적응형 멀티 에이전트 프레임워크를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 LLM의 파라미터를 매번 재학습시키지 않고도 에이전트가 스스로 능력을 향상시킬 수 있도록 하는 **적응형 온라인 학습(Adaptive Online Learning)**과 **데이터 플라이휠(Data Flywheel)** 구조를 도입한 것이다.

가장 중점적인 기여는 엔트로피 가이드 기반의 메모리 인식 온라인 학습 메커니즘을 통해, 에이전트가 과거의 성공 및 실패 사례(Episodic Memory Bank)를 검색하고 활용함으로써 추론 및 계획 능력을 지속적으로 개선하도록 설계한 점이다. 또한, `DingAutoEvaluator`라는 자동 평가 엔진을 도입하여 저성능 케이스를 자동으로 마이닝하고, 이를 다시 강화학습(RL) 및 미세 조정(Fine-tuning) 루프에 투입함으로써 성능의 퇴보를 막고 지속적인 최적화를 달성하는 폐쇄 루프(Closed-loop) 시스템을 구축하였다.

## 📎 Related Works

논문은 OpenAI의 리서치 에이전트, Anthropic의 Claude Research Workbench, Google DeepMind의 Deep Research, Perplexity의 professional research mode 등을 언급하며 기존 연구의 한계를 분석한다.

기존 시스템들은 공개된 정보원으로부터 데이터를 통합하는 효율성은 높으나, 기업 내부의 프라이빗 데이터 통합 능력이 부족하고 동적인 최적화 메커니즘이 결여되어 있다. 특히, 인간-인-루프(Human-in-the-loop) 가이드는 제공하지만, 배포 환경에서의 자동화된 평가 및 지속적 최적화 루프가 부족하며, 복잡한 표 처리 및 기업 리소스 연결 능력이 제한적이라는 점을 지적하며 `Dingtalk-DeepResearch`와의 차별성을 강조한다.

## 🛠️ Methodology

### 1. 시스템 아키텍처 (Three-Layer Structure)
`Dingtalk-DeepResearch`는 다음과 같은 3계층 구조로 설계되었다.
- **Agent Studio Layer**: 딥 리서치, 표 처리, 데이터 분석을 위한 전문 에이전트 및 사용자 맞춤형 개인 에이전트를 구성하는 계층이다.
- **Core Layer**: 컨텍스트 압축, 추론 및 계획, 도구 오케스트레이션, 장/단기 메모리, 그리고 엔트로피 가이드 기반의 자가 진화 엔진이 포함된 핵심 계층이다.
- **Data Layer**: 기업, 산업, 개인 및 합성 데이터 소스를 포괄하는 멀티모달 데이터 백본(지식 그래프, DB, 캐시 등)을 제공한다.

### 2. 다단계 문서 생성 강화학습 (Multi-stage Documentary RL)
문서 생성 능력을 극대화하기 위해 5단계의 훈련 파이프라인을 거친다.
- **Stage 1 (Doc-RM)**: 80만 개의 인간 주석 쌍을 통해 사실 정확성, 논리 구조 등을 평가하는 보상 모델(Reward Model)을 학습시킨다.
- **Stage 2 (Cold-Start SFT)**: 3,200개의 큐레이션된 샘플을 통해 Markdown 기반 PPT, 복잡한 표 요약 등 구조화된 출력 형식에 대해 지도 미세 조정(SFT)을 수행한다.
- **Stage 3 (Static RL)**: 학습된 Doc-RM을 보상 함수로 사용하여 정적 문서 저장소에서 강화학습을 수행한다.
- **Stage 4 (Live RL)**: 실시간 웹 크롤링 및 검색을 포함하는 1만 개의 시간 민감형 쿼리를 통해 최신 정보 반영 능력을 강화한다.
- **Stage 5 (Online DPO)**: 실제 사용자가 수정한 결과물과 모델의 출력을 비교하여 Direct Preference Optimization(DPO)을 통해 개인화된 선호도를 학습한다.

### 3. 엔트로피 가이드 메모리 리트리벌 (Adaptive Online Learning)
LLM 파라미터 수정 없이 성능을 높이기 위해 다음과 같은 메커니즘을 사용한다.
에이전트는 현재 태스크 상태에서 메모리 뱅크에 저장된 과거 케이스들의 확률 분포를 계산한다. 이때 각 케이스의 추정 $Q$-value와 온도 파라미터(Temperature)를 사용하여 탐색(Exploration)과 활용(Exploitation)의 균형을 맞춘다. 이는 모델이 조기에 특정 경험에 과적합되는 것을 방지하고, 새로운 시나리오에 적응할 수 있게 한다.

### 4. 구조 인식 이질적 표 추론 (Structure-Aware Table Reasoning)
표 데이터 처리를 위해 다음과 같은 파이프라인을 적용한다.
1. **데이터 섭취 및 파싱**: 표를 단순 텍스트로 변환하지 않고 계층적 표현(헤더, 병합 셀, 중첩 표 등)으로 파싱하여 구조적 충실도를 유지한다.
2. **시맨틱 이해 및 분해**: 사용자 질문을 텍스트 기반 서브 쿼리와 표 기반 서브 쿼리로 분해한다.
3. **표 추론 (NL2SQL)**: NL2SQL 생성기를 통해 SQL 문을 생성하고 관계형 DB에서 실행한다. 이때 `DingAutoEvaluator`가 발견한 실패 케이스를 통해 NL2SQL 생성기를 반복적으로 재학습시키는 루프를 가진다.
4. **하이브리드 검색**: 헤더에서 시작하는 Top-down 방식과 본문 값에서 시작하는 Bottom-up 방식을 결합하여 최적의 표 영역을 검색한다.

### 5. DingAutoEvaluator (Data Flywheel)
자동 평가 엔진인 `DingAutoEvaluator`는 불확실성 기반 케이스 마이닝(Uncertainty-calibrated case mining)을 통해 모델이 확신하지 못하는 '그레이 존(Grey-zone)' 데이터를 추출한다. 이후 RAG 평가, LLM 평가, 추론 모델 평가, 에이전트 프레임워크 평가, 지식 베이스 평가라는 5개 영역의 다차원 지표를 통해 성능을 측정하고, 이 결과를 다시 학습 루프로 피드백하여 성능 퇴보를 방지하고 지속적인 최적화를 유도한다.

## 📊 Results

### 1. 정량적 평가
논문은 Deep Research Benchmark와 Researcher Bench에서 우수한 성능을 보였음을 Figure 1을 통해 제시하고 있다. 구체적인 수치는 본문 텍스트에 상세히 기재되지 않았으나, 기존 SOTA 모델들과 경쟁 가능한 수준임을 시사한다.

### 2. 정성적 쇼케이스 (Showcases)
- **Case A (복잡한 표 추론)**: 산업 제조 및 공급망 관리 시나리오에서 수천 행에 달하는 복잡한 재고/배송 계획 표를 파싱하고, 특정 기간의 총 생산량을 계산하는 SQL을 정확히 생성하여 정답을 도출하는 과정을 보여준다. 특히, 단순 텍스트 검색이 아닌 구조적 파싱 $\rightarrow$ SQL 실행 $\rightarrow$ 결과 합성의 단계를 통해 높은 정확도를 입증하였다.
- **Case B (멀티모달 보고서 생성)**: Kaggle의 에콰도르 슈퍼마켓 매출 예측 경진대회 데이터를 활용하여, 데이터 로딩부터 EDA, 특성 공학, 모델 선택(LightGBM), 최종 결과 리포트 작성까지 전 과정을 인간의 개입 없이 자동 수행하였다. 이는 코드 생성, 실행, 시각화, 분석이 통합된 딥 리서치 워크플로우의 실효성을 증명한다.

## 🧠 Insights & Discussion

본 논문의 가장 큰 강점은 **'평가-학습-배포'가 하나로 연결된 폐쇄 루프(Closed-loop) 시스템**을 구축했다는 점이다. 단순한 모델 성능 향상이 아니라, `DingAutoEvaluator` $\rightarrow$ `Case Mining` $\rightarrow$ `Targeted Training`으로 이어지는 데이터 플라이휠을 통해 실제 운영 환경에서 모델이 스스로 진화할 수 있는 구조를 만들었다.

또한, 모든 데이터를 LLM의 컨텍스트 윈도우에 밀어넣는 대신, 구조적 파싱과 SQL 기반의 심볼릭 추론을 결합함으로써 기업 데이터의 핵심인 '표'에 대한 추론 정확도를 획기적으로 높였다. 

다만, 제공된 텍스트 내에서는 각 구성 요소(예: 엔트로피 가이드 메모리 vs 단순 RAG)에 대한 개별적인 Ablation Study 결과가 명시적으로 제시되지 않아, 각 모듈이 최종 성능 향상에 기여하는 구체적인 비중을 파악하기 어렵다는 한계가 있다. 또한, 실시간 환경에서의 추론 지연 시간(Latency)과 비용 효율성에 대한 상세 분석이 부족하다.

## 📌 TL;DR

본 논문은 기업 환경의 복잡한 데이터와 가변적인 태스크를 해결하기 위해, **적응형 온라인 학습**과 **자동 평가 엔진(`DingAutoEvaluator`)**을 통합한 멀티 에이전트 프레임워크 `Dingtalk-DeepResearch`를 제안한다. 

핵심 기여는 LLM 재학습 없이 메모리 뱅크와 엔트로피 가이드 검색을 통해 에이전트가 스스로 진화하게 한 점과, NL2SQL 기반의 정교한 표 추론 능력을 결합하여 고품질의 기업용 보고서를 자동 생성할 수 있게 한 점이다. 이 연구는 향후 기업용 AI 에이전트가 정적인 도구를 넘어, 데이터 플라이휠을 통해 지속적으로 성장하는 '적응형 지능'으로 나아가는 방향성을 제시한다.
