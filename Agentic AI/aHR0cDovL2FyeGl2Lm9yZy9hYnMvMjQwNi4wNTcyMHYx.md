# VillagerAgent: A Graph-Based Multi-Agent Framework for Coordinating Complex Task Dependencies in Minecraft

Yubo Dong, Xukun Zhu, Zhengzhe Pan, Linchao Zhu, Yi Yang (2024)

## 🧩 Problem to Solve

본 논문은 LLM(Large Language Model) 기반의 멀티 에이전트 시스템이 공간적(spatial), 인과적(causal), 그리고 시간적(temporal) 제약 조건이 얽힌 복잡한 의존성(complex dependencies)을 가진 과제를 수행할 때 겪는 협업 효율성 저하 문제를 해결하고자 한다.

기존의 멀티 에이전트 프레임워크들은 주로 에이전트들이 매 라운드 병렬로 실행 가능한 하위 작업(subtasks)에 국한되거나, 고정된 파이프라인 및 단계에 묶여 있는 경향이 있다. 이러한 방식은 순차적 실행과 병렬 실행이 동시에 필요한 복잡한 작업에서 유연성과 확장성을 떨어뜨리며, 특히 에이전트 간의 정교한 동기화가 필요한 상황에서 한계를 보인다. 따라서 본 연구의 목표는 이러한 복잡한 의존성을 체계적으로 관리하여 협업 효율을 높이는 프레임워크를 제안하고, 이를 평가할 수 있는 새로운 벤치마크를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 작업을 **방향성 비순환 그래프(Directed Acyclic Graph, DAG)** 형태로 구조화하여 관리하는 것이다.

1.  **VillagerAgent 프레임워크**: 작업을 DAG의 노드로 표현함으로써, 어떤 작업이 선행되어야 하는지(순차적 의존성)와 어떤 작업이 동시에 수행될 수 있는지(병렬성)를 명확히 정의한다. 이를 통해 복잡한 상호 의존성을 해결하고 에이전트 간의 협업 효율을 극대화한다.
2.  **VillagerBench 구축**: Minecraft 환경을 활용하여 공간적 의존성(건축), 인과적 의존성(요리), 시간적 의존성(탈출 룸)을 테스트할 수 있는 다각적인 벤치마크를 제안하여 멀티 에이전트의 협업 능력을 정밀하게 측정한다.
3.  **중앙 집중식 제어 구조**: 에이전트 간의 자유로운 토론 대신, Task Decomposer와 Agent Controller를 통한 중앙 집중식 계획 및 할당 방식을 채택하여 LLM의 고질적인 문제인 환각(hallucination) 현상을 줄이고 작업 성공률을 높였다.

## 📎 Related Works

논문에서는 기존의 LLM 기반 멀티 에이전트 프레임워크인 AgentVerse와 MetaGPT 등을 언급한다. AgentVerse는 전문가 모집, 협동 의사결정, 실행, 평가의 4단계 구조를 가지며, MetaGPT는 조립 라인(assembly line) 방식을 통해 역할을 분담한다.

그러나 저자들은 이러한 기존 방식들이 하위 작업 간의 복잡한 의존성을 간과하고, 단순히 병렬 실행 가능성에만 의존하거나 고정된 단계에 얽매여 있다는 점을 한계로 지적한다. 이로 인해 순차적-병렬적 실행이 혼합된 시나리오에서는 일반성과 확장성이 부족하게 된다. 본 연구는 작업을 DAG로 모델링함으로써 이러한 고정된 파이프라인의 한계를 극복하고 동적인 작업 관리를 가능하게 한다는 점에서 차별점을 가진다.

## 🛠️ Methodology

### 전체 시스템 구조
VillagerAgent는 크게 네 가지 핵심 구성 요소로 이루어져 있다: **Task Decomposer**, **Agent Controller**, **State Manager**, 그리고 **Base Agents**이다.

1.  **Task Decomposer**: 현재 환경 상태와 목표를 바탕으로 하위 작업들의 DAG를 생성한다. 각 노드 $N_j$는 $(T_j, D_j, C_j, F_j)$의 튜플로 정의되며, 여기서 $T$는 작업 설명, $D$는 관련 문서 데이터, $C$는 할당된 에이전트, $F$는 실행 피드백을 의미한다. Zero-shot Chain-of-Thought(CoT) 기법을 사용하여 JSON 형식으로 DAG 구조를 출력한다.
2.  **Agent Controller**: 생성된 DAG에서 선행 작업이 모두 완료되어 즉시 실행 가능한 작업 집합 $N_{ready}$를 식별한다. 이후 LLM을 통해 각 작업에 가장 적합한 에이전트를 매칭하여 할당한다.
3.  **State Manager**: 환경 정보와 에이전트의 상태를 지속적으로 업데이트한다. 에이전트의 액션 히스토리를 요약하여 장기 기억(Long-term memory) 형태의 상태 $S_i$를 유지하며, 전역 환경 상태 $I$에서 현재 작업에 필요한 관련 정보 $E$만을 추출하여 제공한다.
4.  **Base Agents**: 할당된 하위 작업을 ReAct(Reasoning and Acting) 방식을 통해 실행한다. 작업 완료 후에는 자신의 실행 과정을 돌아보는 Self-Reflection 과정을 거쳐 피드백 $F_j$를 생성하고, 이는 다시 DAG의 상태 업데이트에 반영된다.

### 주요 방정식 및 지표
- **작업 완료율 (Completion, $C$):**
  $$C = \frac{\text{# Indicators Detected}}{\text{# Total Indicators Expected}}$$
- **실행 효율성 (Efficiency, $E$):**
  $$E = \frac{\text{Task Completion Rate}}{\text{Total Execution Time}}$$
- **에이전트 활용 균형도 (Balance, $B$):**
  에이전트들의 실제 작동 시간 $t$를 정규화한 $t'$에 대해, 표준편차 $\sigma(t')$를 이용하여 계산한다.
  $$B = 1 - \sigma(t')$$
  여기서 $t' = \frac{t - \min(t)}{\max(t) - \min(t)}$ 이다.

## 📊 Results

### 실험 설정
- **데이터셋 및 모델**: GPT-4-1106-preview, Gemini-Pro, GLM-4 세 가지 모델을 사용하였다.
- **테스트 시나리오**: 건축 협력(Construction), 팜-투-테이블 요리(Farm-to-Table Cooking), 탈출 룸 챌린지(Escape Room Challenge)의 세 가지 과제를 수행하였다.

### 주요 결과
1.  **모델별 성능**: GPT-4가 전반적으로 가장 높은 완료율과 정확도를 보였다. 다만, 단순한 순서와 타이밍이 중요한 탈출 룸 챌린지에서는 Gemini-Pro가 빠른 추론 속도 덕분에 높은 효율성을 나타냈다.
2.  **VillagerAgent vs AgentVerse**: 요리 작업에서 VillagerAgent는 AgentVerse보다 월등히 높은 완료율을 기록하였다. 특히 AgentVerse는 에이전트 간의 토론 과정에서 수행하지 않은 일을 했다고 믿는 환각 현상이 빈번하게 발생하여 작업 실패로 이어졌으나, VillagerAgent는 중앙 집중식 DAG 관리로 이를 억제하였다.
3.  **에이전트 수의 영향**: 에이전트 수가 증가함에 따라 초기에는 성능이 향상되지만, 일정 수준을 넘어서면 자원 경쟁 및 LLM의 관리 복잡도 증가로 인해 성능이 다시 하락하는 피크(peak) 지점이 존재함을 확인하였다.
4.  **능력 다양성의 영향**: 에이전트마다 서로 다른 특수 API를 가진 경우, 오히려 협동 효율이 떨어졌다. 이는 특정 에이전트가 병목 지점이 되어 전체 워크플로우가 중단될 가능성이 높기 때문이다.
5.  **Overcooked-AI 전이 성능**: Overcooked-AI 환경에서도 ProAgent보다 우수한 성능을 보였으며, 특히 단일 프롬프트 세트로 여러 시나리오를 해결함으로써 높은 전이 가능성(Transferability)을 입증하였다.

## 🧠 Insights & Discussion

본 논문은 멀티 에이전트 시스템에서 '자유로운 소통'이 반드시 '효율적인 협업'으로 이어지지 않는다는 점을 시사한다. AgentVerse와 같은 토론 기반 모델은 인간의 협업 방식을 모방하지만, LLM의 특성상 가공의 정보를 생성하는 환각 문제가 발생하며 이는 전체 시스템의 붕괴로 이어진다. 반면, VillagerAgent는 작업을 DAG로 정형화하고 중앙에서 제어함으로써 실행의 결정론적 성격을 강화하고 오류를 줄였다.

**한계점 및 비판적 해석**:
- **낮은 절대적 성공률**: 프레임워크의 개선에도 불구하고 Minecraft의 방대한 API와 환경 복잡성으로 인해 전체적인 작업 완료율 자체가 여전히 낮다는 한계가 있다.
- **확장성 문제**: 에이전트 수가 8명을 초과할 때 성능이 하락하는 현상은, LLM이 처리해야 할 컨텍스트의 길이가 길어지고 DAG의 복잡도가 증가함에 따라 발생하는 '관리 한계'를 보여준다. 이는 향후 계층적 DAG 구조나 분산 제어 방식의 도입이 필요함을 시사한다.
- **능력 다양성의 역설**: 다양한 능력이 협업을 어렵게 만든다는 결과는, 현재의 LLM이 서로 다른 능력을 가진 에이전트들을 정교하게 오케스트레이션하는 능력이 아직 부족함을 의미한다.

## 📌 TL;DR

이 논문은 Minecraft 환경에서 복잡한 의존성을 가진 작업을 해결하기 위해 **방향성 비순환 그래프(DAG) 기반의 멀티 에이전트 프레임워크인 VillagerAgent**를 제안한다. 작업을 그래프 노드로 분해하고 중앙에서 제어함으로써, 기존의 토론 기반 에이전트들이 겪는 환각 문제를 해결하고 작업 효율을 크게 높였다. 또한, 이를 검증하기 위한 **VillagerBench**라는 새로운 벤치마크를 통해 공간적, 인과적, 시간적 제약 조건을 정밀하게 평가하였다. 이 연구는 향후 복잡한 실세계 로보틱스나 시뮬레이션 환경에서 LLM 에이전트들의 정교한 협업 체계를 구축하는 데 중요한 기반이 될 가능성이 크다.
