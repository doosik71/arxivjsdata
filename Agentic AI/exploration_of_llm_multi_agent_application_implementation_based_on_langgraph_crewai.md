# Exploration of LLM Multi-Agent Application Implementation Based on LangGraph+CrewAI

Zhihua Duan, Jialin Wang (2024)

## 🧩 Problem to Solve

현대의 인공지능 시스템에서 단일 에이전트(Single Agent)가 복잡하고 동적인 환경의 모든 과업을 완벽히 수행하는 것은 매우 어렵다. 특히 복잡한 시스템 내에서는 과업의 세분화, 정밀한 제어 흐름의 설계, 그리고 여러 에이전트 간의 효율적인 협업 체계가 필수적이다.

본 논문은 이러한 문제를 해결하기 위해 두 가지 핵심 목표를 설정한다. 첫째, $\text{LangGraph}$를 활용하여 에이전트의 워크플로우를 그래프 구조로 설계함으로써 정밀한 제어 능력을 확보하는 것이다. 둘째, $\text{CrewAI}$를 통해 에이전트에게 구체적인 역할(Role)과 도구를 부여하여 다양한 복잡한 과업을 완수할 수 있는 협업 능력을 강화하는 것이다. 궁극적으로는 이 두 프레임워크의 통합 적용을 통해 대규모 언어 모델(LLM) 기반 멀티 에이전트 시스템의 효율성과 확장성을 높이는 것을 목표로 한다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **'그래프 기반의 제어 흐름(LangGraph)'**과 **'역할 기반의 자율 협업(CrewAI)'**을 결합하는 통합 아키텍처를 제안한 점이다.

$\text{LangGraph}$는 상태 관리와 순환(Loop) 구조를 통해 워크플로우의 전반적인 제어권을 제공하며, $\text{CrewAI}$는 에이전트의 역할 정의, 작업 할당 및 자원 관리를 통해 실제 실행 단계의 효율성을 극대화한다. 이러한 조합을 통해 개발자는 복잡한 다단계 워크플로우를 시각적으로 정의함과 동시에, 각 단계에서 전문화된 에이전트들이 자율적으로 협력하여 최적의 결과물을 도출하게 할 수 있다.

## 📎 Related Works

논문에서는 멀티 에이전트 시스템과 관련된 기존 연구들을 다음과 같이 분류하여 설명한다.

- **역할 기반 협업 프레임워크**: $\text{MetaGPT}$와 $\text{ChatDev}$ 등은 에이전트에게 미리 정의된 역할과 책임을 부여하여 소프트웨어 개발과 같은 협업 과업을 수행하도록 설계되었다.
- **메모리 및 계획 메커니즘**: $\text{RLP}$는 대화 상대방의 내부 상태를 유지하는 단기 메모리 메커니즘을 도입하였으며, $\text{SayPlan}$은 복잡한 과업을 단순한 하위 과업으로 분해하는 계획 능력을 강조한다.
- **추론 및 최적화 기법**: $\text{Chain of Thought (CoT)}$는 단계적인 추론 과정을 프롬프트에 포함시켜 복잡한 문제 해결 능력을 높였으며, 최근에는 $\text{LangGraph}$와 $\text{CoT}$를 결합한 뱅킹 어시스턴트나 그래프 기술을 접목한 $\text{RAG (Retrieval-Augmented Generation)}$ 시스템 연구가 진행되고 있다.

본 연구는 이러한 기존의 역할 정의 및 추론 기법을 넘어, 실제 구현 단계에서 제어 가능성(Controllability)과 협업 효율성을 동시에 잡기 위해 $\text{LangGraph}$와 $\text{CrewAI}$라는 구체적인 프레임워크의 통합 모델을 제시한다는 점에서 차별점을 갖는다.

## 🛠️ Methodology

### 1. LangGraph의 역할

$\text{LangGraph}$는 LLM 기반 에이전트를 위한 상태 머신(State Machine) 역할을 수행한다. 기존의 단순 선형 체인과 달리 다음과 같은 기능을 제공한다.

- **Cycles (순환)**: 작업 결과에 따라 이전 단계로 돌아가 수정하거나 반복하는 루프 구조를 지원한다.
- **Persistence (지속성)**: 에이전트의 상태를 저장하고 복구할 수 있는 지속적 메모리를 제공한다.
- **Granular Control (세밀한 제어)**: 단일 에이전트, 멀티 에이전트, 계층적 구조 등 다양한 제어 흐름을 정밀하게 설계할 수 있다.

### 2. CrewAI의 역할

$\text{CrewAI}$는 실제 과업을 수행하는 '팀'의 운영 체제 역할을 하며, 다음과 같은 핵심 구성 요소로 이루어진다.

- **Agent**: 특정 역할(Role), 목표(Goal), 백스토리(Backstory)를 가진 자율적 단위이다.
- **Task**: 에이전트가 수행해야 할 구체적인 작업 내용과 필요한 도구, 담당 에이전트를 정의한다.
- **Tool**: 에이전트가 외부 데이터에 접근하거나 특정 기능을 실행하기 위해 사용하는 도구이다.
- **Crew**: 에이전트와 태스크를 조합하여 전체적인 프로세스를 조율(Orchestration)하는 실행 단위이다.

### 3. 통합 파이프라인 및 작동 원리

$\text{LangGraph} + \text{CrewAI}$ 통합 시스템은 다음과 같은 흐름으로 동작한다.

1. **워크플로우 정의 ($\text{LangGraph}$)**: 전체 프로세스를 그래프 노드로 정의한다. (예: 이메일 확인 $\rightarrow$ 초안 작성 $\rightarrow$ 전송 대기)
2. **에이전트 팀 구성 ($\text{CrewAI}$)**: 각 노드에서 실행될 전문 에이전트 팀을 구성하고 역할을 부여한다.
3. **상태 공유 및 실행**: $\text{LangGraph}$가 현재 상태를 유지하며 $\text{CrewAI}$ 팀에게 작업을 할당하고, $\text{CrewAI}$는 자율적으로 협업하여 결과를 도출한 뒤 다시 그래프 상태에 반영한다.
4. **피드백 루프**: 결과가 만족스럽지 않을 경우 $\text{LangGraph}$의 루프 구조를 통해 다시 수정 단계로 회귀한다.

## 📊 Results

본 논문은 제안한 프레임워크를 검증하기 위해 다음과 같은 환경에서 두 가지 적용 사례(Application Case)를 구현하였다.

**개발 환경:**

- **Frameworks**: $\text{CrewAI}$, $\text{LangGraph}$
- **Monitoring**: $\text{LangSmith}$ (워크플로우 추적 및 감사)
- **Embedding & DB**: $\text{Ollama}$ embedding model, $\text{Faiss}$ (벡터 데이터베이스)

**실제 적용 사례:**

1. **코드 생성 및 리뷰 시스템**: 코드 생성 에이전트와 코드 리뷰 에이전트를 구성하여, 실시간 상태 공유와 피드백 메커니즘을 통해 코드의 품질과 생성 효율을 높였다.
2. **티켓 감사 및 전달 시스템 (Ticket Auditing & Forwarding)**: 고객 문의나 작업 요청(Ticket) 텍스트를 심층 분석하는 여러 지능형 에이전트를 구축하여, 티켓 내용을 정확히 이해하고 적절한 담당자에게 자동 전달하는 시스템을 구현하였다.

**결과 분석:**
논문은 정량적인 수치(Accuracy, F1-score 등)를 제시하지는 않았으나, 사례 분석을 통해 $\text{LangGraph}$의 그래프 워크플로우와 $\text{CrewAI}$의 역할 기반 협업이 결합되었을 때 작업 관리 효율성, 시스템 유연성 및 확장성이 향상됨을 확인하였다고 기술한다.

## 🧠 Insights & Discussion

### 강점

본 연구는 이론적인 제안에 그치지 않고 $\text{LangGraph}$와 $\text{CrewAI}$라는 실제 사용 가능한 최신 라이브러리를 결합하여 실용적인 아키텍처를 제시하였다. 특히 제어 흐름의 '구조화'($\text{LangGraph}$)와 실행 단계의 '자율화'($\text{CrewAI}$)를 분리하여 설계함으로써, 개발자가 시스템의 예측 가능성을 높이면서도 에이전트의 유연한 대응 능력을 동시에 확보할 수 있게 하였다.

### 한계 및 비판적 해석

1. **정량적 평가의 부재**: 가장 큰 한계점은 실험 결과가 정성적인 사례 분석에만 의존하고 있다는 점이다. 베이스라인(예: 단일 LLM 또는 단일 프레임워크 사용 시) 대비 정확도, 처리 속도, 토큰 소모량 등이 얼마나 개선되었는지에 대한 데이터가 명시되지 않아 객관적인 성능 향상 폭을 알 수 없다.
2. **복잡도 증가**: 두 가지 프레임워크를 동시에 사용함으로써 발생하는 시스템 복잡도와 오버헤드에 대한 논의가 부족하다.
3. **추측 배제**: 논문 내에 구체적인 손실 함수나 학습 알고리즘에 대한 언급이 없는 것으로 보아, 본 연구는 모델을 직접 학습시키는 연구가 아니라 기존 LLM을 활용한 **시스템 엔지니어링 및 오케스트레이션** 연구로 해석된다.

## 📌 TL;DR

본 논문은 정밀한 워크플로우 제어를 위한 $\text{LangGraph}$와 역할 기반 자율 협업을 위한 $\text{CrewAI}$를 통합한 멀티 에이전트 시스템 구현 방안을 제시한다. 이 결합을 통해 코드 생성 및 티켓 처리와 같은 복잡한 과업에서 효율적인 제어와 협업이 가능함을 보였으며, 이는 향후 복잡한 기업용 AI 워크플로우 설계에 있어 중요한 참고 모델이 될 가능성이 높다.
