# AutoGen: Enabling Next-Gen LLM Applications via Multi-Agent Conversation

Qingyun Wu, Gagan Bansal, Jieyu Zhang, Yiran Wu, Beibin Li, Erkang Zhu, Li Jiang, Xiaoyun Zhang, Shaokun Zhang, Jiale Liu, Ahmed Awadallah, Ryen W. White, Doug Burger, Chi Wang

## 🧩 Problem to Solve

최근 대규모 언어 모델(LLM)은 추론, 도구 사용, 새로운 관찰에 대한 적응 능력을 가진 강력한 에이전트를 구축하는 데 핵심적인 요소가 되고 있습니다. 그러나 LLM을 활용한 애플리케이션의 복잡성과 도메인의 광범위한 확장을 고려할 때, 단일 에이전트로는 그 능력을 확장하는 데 한계가 있습니다. 이에 따라 다중 에이전트 협업의 필요성이 대두되지만, 이를 효과적으로 촉진할 수 있는 유연하고 일반화된 프레임워크가 부재했습니다. 특히, 다음 두 가지 핵심 질문에 대한 답이 필요했습니다:

1. **개별 에이전트 설계**: 다중 에이전트 협업에서 유능하고, 재사용 가능하며, 사용자 정의 가능하고, 효과적인 개별 에이전트를 어떻게 설계할 것인가?
2. **대화 인터페이스 개발**: 단일 또는 다중 턴 대화, 다양한 인간 개입 모드, 정적 또는 동적 대화 등 광범위한 에이전트 대화 패턴을 수용할 수 있는 직관적이고 통합된 인터페이스를 어떻게 개발할 것인가?

## ✨ Key Contributions

* **사용자 정의 및 대화 가능 에이전트 설계**: LLM, 인간 입력, 도구 또는 이들의 조합을 활용하는 일반 에이전트 디자인을 제시합니다. 모든 에이전트는 메시지를 주고받으며 반응하고 응답할 수 있도록 '대화 가능(conversable)'하게 설계되어, 개발자가 다양한 역할과 능력을 가진 에이전트를 쉽고 빠르게 생성하고 구성할 수 있습니다.
* **대화 프로그래밍 패러다임 도입**: 복잡한 LLM 애플리케이션 워크플로우를 다중 에이전트 대화로 단순화하고 통합하는 새로운 프로그래밍 패러다임을 제안합니다. 에이전트 간의 상호작용 행동을 자연어 및/또는 프로그래밍 언어로 유연하게 정의할 수 있도록 지원합니다.
* **통합 인터페이스 및 자동 응답 메커니즘**: 에이전트 간의 자동화된 대화를 위한 `send`, `receive`, `generate_reply`와 같은 통합 대화 인터페이스와 에이전트 자동 응답 메커니즘을 제공하여 분산적이고 모듈화된 워크플로우를 가능하게 합니다.
* **다양한 애플리케이션 지원 및 검증**: 수학 문제 해결, 검색 증강 코드 생성 및 질의응답, 텍스트 기반 의사 결정, 다중 에이전트 코딩, 동적 그룹 채팅, 대화형 체스 등 다양한 도메인과 복잡성의 LLM 애플리케이션을 AutoGen으로 성공적으로 구축 및 시연하여 프레임워크의 유연성과 성능을 입증합니다.
* **성능 향상 및 개발 효율성 증대**: AutoGen 기반 솔루션이 기존 최첨단(SOTA) 접근 방식보다 뛰어난 성능을 달성하고, 개발 코드량 감소 및 수동 작업 부담 경감 효과를 가져왔음을 보여줍니다.

## 📎 Related Works

* **단일 에이전트 시스템**:
  * **AutoGPT**: 자율적인 AI 에이전트 구현체. 다중 에이전트 협업을 지원하지 않습니다.
  * **ChatGPT+ (코드 인터프리터 또는 플러그인)**: 코드 실행 및 다양한 도구 활용이 가능한 대화형 AI. 단일 에이전트 패러다임입니다.
  * **LangChain Agents**: LLM 기반 애플리케이션 개발을 위한 일반 프레임워크의 일부. 주로 단일 에이전트 패러다임이며, 대화 및 협업 모드에 최적화되지 않았습니다.
  * **Transformers Agent**: 자연어 해석 및 도구 사용을 위한 실험적 API. 단일 에이전트 패러다임입니다.
* **다중 에이전트 시스템**:
  * **BabyAGI**: LLM 기반 작업 관리 시스템. 미리 정의된(정적) 대화 패턴만 지원하며, 도구 사용 및 인간 개입은 제한적입니다.
  * **CAMEL**: 역할 플레이를 통해 에이전트 간 통신을 시연하는 프레임워크. 도구 사용을 기본적으로 지원하지 않으며, 정적 대화 패턴만 가능합니다.
  * **Multi-Agent Debate**: 다중 LLM 인스턴스를 에이전트로 구성하여 논쟁을 통해 발산적 사고 및 추론 능력을 향상시키는 접근 방식. 도구나 인간 개입이 없고 미리 정의된 대화 순서만 따릅니다.
  * **MetaGPT**: 자동 소프트웨어 개발에 특화된 다중 에이전트 대화 프레임워크. 특정 시나리오에 특화되어 있으며, AutoGen과 같은 일반적인 인프라가 아닙니다.

AutoGen은 위 시스템들과 달리 **제네릭 인프라**로서 **정적 및 동적 대화 패턴을 유연하게 지원**하며, **LLM, 인간, 도구 및 이들의 조합**을 활용하는 **실행 가능하고 유연한 인간 개입**을 제공한다는 점에서 차별화됩니다.

## 🛠️ Methodology

AutoGen 프레임워크는 다음 두 가지 핵심 개념을 중심으로 구축됩니다.

1. **대화 가능 에이전트 (Conversable Agents)**
    * **정의**: `ConversableAgent`는 특정 역할을 가진 엔터티로, 다른 에이전트와 메시지를 주고받아 대화를 시작하거나 계속할 수 있습니다. 송수신 메시지를 기반으로 내부 컨텍스트를 유지하며, LLM, 도구 또는 인간 입력 등 다양한 능력을 갖도록 구성할 수 있습니다.
    * **능력 원천 (LLM, 인간, 도구)**:
        * **LLM 기반**: 고급 LLM의 역할 플레이, 대화 이력을 통한 암시적 상태 추론, 피드백 제공 및 적응, 코딩 등의 능력을 활용합니다. 결과 캐싱, 오류 처리, 메시지 템플릿 등 향상된 LLM 추론 기능을 제공합니다.
        * **인간 기반**: `UserProxyAgent`를 통해 대화의 특정 단계에서 인간 입력을 요청하도록 구성할 수 있습니다. 인간이 입력을 건너뛸 수 있는 옵션 등 유연한 인간 개입 수준을 지원합니다.
        * **도구 기반**: 코드 실행 또는 함수 호출을 통해 도구를 실행할 수 있는 능력을 가집니다. (예: `UserProxyAgent`는 LLM이 제안한 코드를 실행하거나 함수를 호출할 수 있습니다).
    * **에이전트 사용자 정의 및 협력**: `ConversableAgent` 클래스는 최상위 추상화이며, `AssistantAgent` (LLM 기반 AI 어시스턴트)와 `UserProxyAgent` (인간 프록시 및 도구 실행)는 미리 구성된 서브클래스입니다. 내장 에이전트를 재사용하거나 확장하여 애플리케이션별로 특화된 에이전트를 쉽게 생성할 수 있습니다.

2. **대화 프로그래밍 (Conversation Programming)**
    * **개념**: 다중 에이전트 대화를 중심으로 한 프로그래밍 패러다임입니다.
        * **계산 (Computation)**: 에이전트가 대화에서 응답을 계산하기 위해 취하는 행동을 의미하며, 이는 메시지 전달로 이어집니다.
        * **제어 흐름 (Control Flow)**: 이러한 계산이 발생하는 순서 또는 조건을 의미하며, 에이전트 간 대화에 의해 주도됩니다.
    * **주요 설계 패턴**:
        * **통합 인터페이스 및 자동 응답 메커니즘**: 모든 에이전트는 `send`, `receive`, `generate_reply`와 같은 통합 대화 인터페이스를 가집니다. 에이전트가 메시지를 받으면 자동으로 `generate_reply`를 호출하고 응답을 보내는 **에이전트 자동 응답 메커니즘**을 기본적으로 채택하여, 별도의 중앙 제어 모듈 없이도 자연스러운 대화 흐름을 유도합니다.
        * **프로그래밍 및 자연어 융합을 통한 제어**:
            * **자연어 제어 (LLM 활용)**: LLM 기반 에이전트를 프롬프트하여 대화 흐름을 제어합니다 (예: 오류 시 코드 수정 지시, "TERMINATE"로 작업 완료 지시).
            * **프로그래밍 언어 제어**: Python 코드를 사용하여 대화 종료 조건, 인간 입력 모드, 도구 실행 로직 등을 지정하고, 사용자 정의 자동 응답 함수를 등록하여 대화 흐름을 제어할 수 있습니다.
            * **자연어와 프로그래밍 언어 간 제어 전환**: 사용자 정의 응답 함수 내에서 LLM 추론을 호출하거나, LLM이 제안하는 함수 호출을 통해 유연한 제어 전환을 지원합니다.
    * **유연한 대화 패턴**:
        * **사용자 정의 `generate_reply` 함수**: 에이전트가 현재 메시지 및 컨텍스트에 따라 다른 에이전트와 동적으로 대화를 시작할 수 있습니다.
        * **함수 호출 (Function Calls)**: LLM이 대화 상태에 따라 특정 함수 호출 여부를 결정하여 동적인 다중 에이전트 대화를 유도할 수 있습니다.
        * **동적 그룹 채팅**: 내장된 `GroupChatManager`를 통해 다음 발화자를 동적으로 선택하고 응답을 다른 에이전트에게 브로드캐스팅하여 복잡한 동적 그룹 채팅을 관리할 수 있습니다.

## 📊 Results

AutoGen은 6가지 다양한 LLM 애플리케이션에서 그 효과성을 입증하며 뛰어난 성능과 개발 편의성을 보여주었습니다.

* **A1: 수학 문제 해결 (Math Problem Solving)**
  * **자율 문제 해결**: MATH 데이터셋(특히 Level-5 문제 및 전체 데이터셋)에서 AutoGen은 Multi-Agent Debate, LangChain ReAct, vanilla GPT-4, ChatGPT+Code Interpreter, ChatGPT+Plugin 등 기존의 대안적 접근 방식들보다 **최고 성능**을 달성했습니다 (전체 데이터셋에서 69.48% 정확도).
  * **인간 참여 문제 해결**: `human_input_mode='ALWAYS'` 설정으로 인간의 피드백을 효과적으로 통합하여 자율적으로 해결하기 어려운 문제를 해결했습니다.
  * **다중 사용자 문제 해결**: 학생과 전문가와 같은 다중 인간 사용자가 LLM 에이전트와 협력하여 문제를 해결하는 새로운 시나리오를 지원했습니다.

* **A2: 검색 증강 코드 생성 및 질의응답 (Retrieval-Augmented Code Generation and Question Answering)**
  * **Q&A 성능 향상**: `Retrieval-augmented Chat` 시스템을 구축하여 Natural Questions 데이터셋에서 Q&A 작업 시 `interactive retrieval` 기능(25.88%의 F1 점수, 66.65%의 Recall)을 통해 관련 정보가 없을 때 LLM이 "UPDATE CONTEXT"를 요청하여 추가 검색을 유도, 성능을 크게 향상시켰습니다.
  * **최신 API 활용 코드 생성**: GPT-4 학습 데이터에 포함되지 않은 Spark 관련 API를 사용하여 정확한 코드를 성공적으로 생성했습니다.

* **A3: 텍스트 월드 환경에서의 의사 결정 (Decision Making in Text World Environments)**
  * ALFWorld 벤치마크에서 초기 두 에이전트 시스템은 ReAct와 유사한 성능을 보였습니다.
  * 상식적 지식 부족으로 반복 오류에 빠지는 문제를 해결하기 위해 'Grounding Agent'를 추가한 세 에이전트 시스템은 **평균 15%의 성능 향상**을 달성했습니다.

* **A4: 다중 에이전트 코딩 (Multi-Agent Coding)**
  * OptiGuide를 AutoGen으로 재구현하여 핵심 워크플로우 코드를 430줄 이상에서 100줄로 **4배 이상 감소**시켰고, 개발 생산성을 대폭 향상시켰습니다.
  * ChatGPT+Code Interpreter 대비 **사용자 시간 3배 절약, 사용자 상호작용 3-5배 감소**를 시연했습니다.
  * 다중 에이전트 설계가 안전하지 않은 코드 식별 F1 점수를 GPT-4에서 8%, GPT-3.5-turbo에서 35% 향상시켰습니다.

* **A5: 동적 그룹 채팅 (Dynamic Group Chat)**
  * `GroupChatManager`를 통해 동적으로 발화자를 선택하고 응답을 브로드캐스팅하는 동적 그룹 채팅을 구현했습니다.
  * 파일럿 연구에서 역할 기반 프롬프트를 사용한 동적 발화자 선택 정책이 단순한 작업 기반 프롬프트보다 **높은 성공률과 적은 LLM 호출**을 유도했습니다.

* **A6: 대화형 체스 (Conversational Chess)**
  * 인간 또는 LLM 플레이어 에이전트와 규칙 검증을 위한 보드 에이전트로 구성된 자연어 인터페이스 체스 게임을 개발했습니다.
  * AI-AI, AI-인간, 인간-인간 등 유연한 게임 플레이 패턴을 지원하며, 보드 에이전트의 `Grounding` 기능으로 게임 무결성을 유지했습니다. 보드 에이전트 없이 프롬프트에만 의존할 경우 불법적인 움직임으로 게임 중단이 발생함을 확인했습니다.

## 🧠 Insights & Discussion

AutoGen은 다중 에이전트 협업을 통해 LLM 애플리케이션 개발의 새로운 가능성을 열었습니다.

* **주요 장점**:
  * **사용 용이성**: 내장 에이전트들이 별다른 사용자 정의 없이도 강력한 성능을 제공하여 개발 시작의 장벽을 낮춥니다 (A1, A3).
  * **모듈성**: 작업을 개별 에이전트로 분할함으로써 시스템의 모듈성을 높여, 각 에이전트를 독립적으로 개발, 테스트, 유지보수할 수 있게 하여 전체 개발 프로세스를 단순화합니다 (A3, A4, A5, A6).
  * **프로그래밍 가능성**: 기존 에이전트를 쉽게 확장하거나 사용자 정의하여 특정 애플리케이션 요구 사항을 충족시킬 수 있습니다 (A1-A6). OptiGuide 워크플로우 코드를 4배 절감한 사례(A4)는 생산성 향상을 보여줍니다.
  * **인간 개입 허용**: `UserProxyAgent`를 통해 인간이 AI 에이전트와 원활하게 협력하거나 감독할 수 있는 메커니즘을 제공하여, 자율성과 인간 통제 사이의 균형을 이룹니다 (A1, A2, A5, A6).
  * **협력/적대적 에이전트 상호작용**: 에이전트가 정보를 공유하고 능력을 보완하여 더 나은 솔루션을 찾거나(협력적), 가상 적대적 검사기 역할을 수행하여 LLM의 효과적인 활용 및 증강을 가능하게 합니다 (A1, A2, A3, A4, A6).

* **제한 및 향후 연구 방향**:
  * **최적의 다중 에이전트 워크플로우 설계**: 주어진 작업에 대한 최적의 에이전트 수, 역할 할당, 상호작용 방식, 자동화 수준 등을 결정하는 체계적인 연구가 필요합니다.
  * **고도로 유능한 에이전트 생성**: LLM, 도구, 인간의 강점을 통합하여 더욱 유능한 에이전트를 개발하고, 애플리케이션별 에이전트 지침, 대규모 오픈 소스 지식 베이스 구축, 에이전트의 기술 발견 및 업그레이드 메커니즘 연구가 중요합니다.
  * **확장성, 안전성, 인간 에이전시 확보**: 복잡한 다중 에이전트 워크플로우의 로깅 및 디버깅 도구 개발, 연쇄 실패 및 오용 방지를 위한 안전 장치, 보상 해킹 완화, 통제 불능 및 원치 않는 행동 방지, 인간 감독의 적절한 수준 및 패턴 결정 등이 중요합니다.

* **윤리적 고려사항**: 프라이버시 및 데이터 보호, LLM의 편향 완화, 책임 및 투명성, AI 시스템에 대한 신뢰 및 의존도 관리, 의도치 않은 결과 방지 등에 대한 지속적인 연구와 주의가 필요합니다.

## 📌 TL;DR

AutoGen은 LLM, 인간, 도구를 통합하는 사용자 정의 가능하고 대화 가능한 에이전트와 대화 중심의 프로그래밍 패러다임을 통해 복잡한 LLM 애플리케이션 개발을 위한 개방형 프레임워크를 제시합니다. 이 프레임워크는 유연한 대화 패턴(정적 및 동적)을 지원하며, 수학 문제 해결, 검색 증강, 의사 결정, 코딩 등 다양한 분야에서 기존 방법론을 능가하는 성능과 개발 효율성을 입증합니다.
