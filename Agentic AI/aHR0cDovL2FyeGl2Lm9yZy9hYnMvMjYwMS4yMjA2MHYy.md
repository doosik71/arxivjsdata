# Vision-DeepResearch: Incentivizing DeepResearch Capability in Multimodal Large Language Models

Wenxuan Huang et al. (2026)

## 🧩 Problem to Solve

본 논문은 멀티모달 거대 언어 모델(MLLM)이 외부 검색 도구를 사용하여 복잡한 사실 기반의 시각적 질의응답(VQA)을 해결할 때 발생하는 두 가지 핵심적인 한계를 해결하고자 한다.

첫째는 **검색 적중률(Hit-Rate) 문제**이다. 기존의 멀티모달 검색 방식은 이미지 전체(Full-image) 또는 단순한 엔티티 수준의 쿼리를 사용하여 검색을 수행한다. 그러나 실제 웹 환경에서는 배경 소음(Visual noise)이 심하고 검색 엔진의 결과가 쿼리의 스케일에 따라 매우 가변적이기 때문에, 단 한 번의 단순한 쿼리만으로는 정답에 필요한 핵심 증거를 찾지 못하는 경우가 많다.

둘째는 **추론 깊이(Reasoning Depth)와 검색 너비(Search Breadth)의 제한**이다. 기존의 에이전트 기반 방식들은 대개 짧은 추론 궤적(Trajectory)을 생성하며, 몇 번의 검색 시도 후 빠르게 결론을 내리려는 경향이 있다. 이는 복잡한 다단계(Multi-hop) 추론이 필요하거나 다양한 소스에서 증거를 수집해야 하는 딥 리서치(Deep Research) 수준의 과제를 해결하는 데 한계로 작용한다.

따라서 본 연구의 목표는 다회차(Multi-turn), 다엔티티(Multi-entity), 다중 스케일(Multi-scale)의 시각 및 텍스트 검색을 수행할 수 있는 새로운 멀티모달 딥 리서치 패러다임을 제안하고, 이를 MLLM에 내재화하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 인간이 정보를 탐색하는 방식, 즉 **시행착오(Trial-and-error) 기반의 반복적 탐색 과정**을 모델에 학습시키는 것이다. 

주요 기여 사항은 다음과 같다.
1. **다중 스케일 시각 검색 전략**: 이미지 전체가 아닌 다양한 크기의 크롭(Crop) 영역을 생성하여 시각 검색 엔진에 쿼리로 던짐으로써, 노이즈가 많은 환경에서도 타겟 엔티티의 적중률을 높인다.
2. **롱-호라이즌(Long-horizon) 궤적 생성**: 텍스트 기반의 강력한 딥 리서치 LLM의 추론 능력을 시각 영역으로 전이시키기 위해, 이미지-설명(Image-to-Description) 브릿징 기술을 사용하여 수십 단계의 추론과 수백 번의 도구 호출이 포함된 고품질 학습 데이터를 구축하였다.
3. **SFT 및 RL 기반의 능력 내재화**: 지도 미세 조정(SFT)과 강화 학습(RL)을 통해 모델이 스스로 계획을 세우고, 검색 결과를 반영하여 쿼리를 수정하며, 정답을 도출할 때까지 탐색을 지속하는 능력을 갖추게 하였다.

## 📎 Related Works

기존 연구는 크게 텍스트 전용 딥 리서치 LLM과 멀티모달 딥 리서치 MLLM으로 나뉜다.

- **텍스트 전용 딥 리서치 LLM**: Tongyi-DeepResearch, WebDancer 등이 대표적이며, 이들은 "추론-도구 호출-재추론"의 반복 루프를 통해 복잡한 지식 집약적 과제를 해결한다. 그러나 시각적 인지 및 엔티티 지역화(Localization) 능력이 없어 이미지 기반의 정보 탐색이 불가능하다는 한계가 있다.
- **멀티모달 딥 리서치 MLLM**: WebWatcher, MMSearch-R1 등이 제안되었다. WebWatcher는 역이미지 검색을 활용하고, MMSearch-R1은 GRPO를 통해 검색 도구 호출을 장려한다. 하지만 이들은 여전히 단발성(One-shot)의 이미지 검색에 의존하거나, 학습 데이터의 추론 궤적이 너무 짧아 복잡한 실제 환경에서의 강건성이 떨어진다.

본 연구는 이러한 한계를 극복하기 위해 **시각적 쿼리의 다변화(Multi-scale cropping)**와 **추론 궤적의 획기적인 확장(Long-horizon)**을 통해 차별성을 둔다.

## 🛠️ Methodology

### 1. 전체 시스템 구조 및 파이프라인
Vision-DeepResearch는 고품질의 VQA 데이터를 합성하고, 이를 바탕으로 롱-호라이즌 추론 궤적을 생성한 뒤, SFT와 RL로 모델을 학습시키는 파이프라인으로 구성된다.

### 2. 데이터 생성 프로세스 (Data Pipeline)
#### A. 다중 엔티티 및 다중 스케일 시각 검색
입력 이미지 $I$와 질문 $q$가 주어지면, MLLM이 관련 영역을 로컬라이즈하여 여러 개의 바운딩 박스 $S_b = \{I_1^b, \dots, I_n^b\}$를 생성한다. 각 단계 $t$에서의 시각적 액션 $A_v^t$는 다음과 같이 정의된다.
$$A_v^t = \text{Tool-Call}(S_b^t)$$
시각 도구 파이프라인은 [시각 검색 $\rightarrow$ 웹사이트 방문 $\rightarrow$ 웹사이트 요약]의 순서로 작동하며, 결과물인 관찰값(Observation) $O_v^t$는 다음과 같다.
$$O_v^t = \text{Vision-Pipeline}(A_v^t)$$
이때, 외부 판별 모델(Judge Model)이 현재까지 수집된 증거 $V_v^t = \{O_1, \dots, O_{t_v}\}$가 정답 $a_{\text{true}}$를 도출하기에 충분한지 판단하여 검색 종료 여부를 결정한다.
$$h_v^t = \text{Judge}(I, q, V_v^t, a_{\text{true}}) \in \{0, 1\}$$

#### B. 텍스트 브릿징 및 텍스트 전용 딥 리서치
시각적 궤적 $C_{\text{vision}}$을 텍스트 기반 딥 리서치 LLM이 처리할 수 있도록, 이미지 $I$를 상세한 텍스트 설명 $D$로 대체하여 브릿징한다. 이후 강력한 텍스트 전용 딥 리서치 모델이 이를 이어받아 추가적인 텍스트 검색 및 파이썬 코드 실행 등을 통해 최종 답안 $a_{\text{output}}$을 도출하는 텍스트 궤적 $C_{\text{text}}$를 생성한다. 최종적으로 $C_{\text{multimodal}} = \{C_{\text{vision}}, C_{\text{text}}\}$ 형태로 결합된다.

#### C. 검증된 사실 기반 VQA 생성
단순한 질문이 아닌 복잡한 질문을 만들기 위해 **모호화(Obfuscation)** 전략을 사용한다.
- **정답 모호화(Answer Obfuscation)**: 정답 주변에 관계 체인을 형성하여 추론 깊이를 증가시킨다. (예: "고양이 주인 딸의 선생님 이름은?")
- **엔티티 모호화(Entity Obfuscation)**: 웹페이지의 링크를 따라 랜덤 워크(Random Walk)를 수행하여 원래 엔티티를 관련 엔티티로 교체한다.

### 3. 학습 전략
#### A. 지도 미세 조정 (SFT)
약 30K 개의 고품질 궤적(VQA 궤적 16K, 텍스트 QA 궤적 8K, 모호화된 VQA 궤적 6K)을 사용하여 표준 autoregressive cross-entropy loss로 학습한다. 이를 통해 모델은 다회차 검색 패턴과 롱-호라이즌 계획 능력을 습득한다.

#### B. 강화 학습 (RL)
SFT 이후, **GRPO (Group Relative Policy Optimization)**를 적용하여 모델의 행동을 최적화한다.
- **보상 설계**: LLM-as-Judge를 사용하여 최종 정답의 정확도에 따라 $1.0$ 또는 $0.0$의 보상을 주는 Pure Accuracy Reward를 채택한다.
- **비동기 롤아웃(Asynchronous Rollout)**: 긴 추론 단계와 잦은 도구 호출로 인한 지연을 줄이기 위해, 멀티스레드 비동기 롤아웃 파이프라인을 설계하여 처리량을 10배 이상 높였다.
- **엔지니어링 기법**: 
    - 반복되는 텍스트나 잦은 툴 호출 오류 발생 시 궤적을 조기에 종료하는 감지기를 도입하였다.
    - 비정상적으로 종료된 궤적에 대해 과도한 음수 보상이 주어지는 것을 막기 위해, 해당 궤적을 그래디언트 업데이트에서 제외하는 **Mask Trajectory** 기법을 사용하였다.
    - 수치적 안정성을 위해 FP16 대신 BF16을 사용하였다.

## 📊 Results

### 1. 정량적 결과
6개의 벤치마크(VDR, FVQA, MMSearch+, MMSearch, LiveVQA, BC-VL)에서 평가한 결과, Vision-DeepResearch는 오픈 소스 모델 중 최고 성능을 기록했으며, 강력한 폐쇄형 모델(GPT-5, Gemini-2.5 Pro 등)의 에이전트 워크플로우와 경쟁 가능한 수준에 도달하였다.

- **성능 향상**: Qwen3-VL-30B-A3B-Instruct 기반의 에이전트 설정 대비, Vision-DeepResearch-30B-A3B는 평균 성능이 $24.1\% \rightarrow 56.9\%$로 크게 향상되었다.
- 특히 MMSearch+ (+18.5%)와 BC-VL (+11.1%)에서 괄목할 만한 상승을 보였다.

### 2. 절제 연구 (Ablation Study)
- **검색 전략 분석**: '다중 스케일 크롭 시각 검색(CIS) + 텍스트 검색(TS)' 조합이 가장 높은 성능을 보였다. 단일 이미지 검색(WIS)은 배경 소음으로 인해 성능 향상이 제한적이었으나, CIS를 통해 정교한 시각적 앵커를 확보했을 때 텍스트 검색의 효율이 극대화됨을 확인하였다.
- **데이터 영향**: SFT 단계에서 일반 VQA 궤적뿐만 아니라 '모호화된(Fuzzy) VQA 궤적'을 추가했을 때 롱테일 지식 및 다단계 추론 성능이 더욱 향상되었다.
- **RL의 효과**: SFT 모델에 RL을 추가 적용했을 때, 평균 3.1%의 추가 성능 향상이 있었으며, 모델이 더 효율적인(더 짧으면서 정확한) 궤적을 생성하도록 최적화됨을 확인하였다.

## 🧠 Insights & Discussion

본 논문은 MLLM이 단순한 도구 호출자를 넘어 '연구원(Researcher)'과 같은 심층 탐색 능력을 갖추기 위해서는 무엇이 필요한지를 잘 보여준다.

**강점**으로는 텍스트 기반 딥 리서치 모델의 강력한 추론 능력을 시각 영역으로 전이시키기 위해 '텍스트 브릿징'이라는 실용적인 접근법을 사용한 점이 돋보인다. 또한, 실제 웹 환경의 노이즈 문제를 해결하기 위해 다중 스케일 크롭이라는 구체적인 전략을 제시하고 이를 RL로 내재화한 점이 효과적이었다.

**한계 및 논의사항**으로는 여전히 RL 학습 과정에서 발생하는 '롱테일' 문제(일부 궤적이 너무 오래 걸려 전체 학습 속도를 늦추는 현상)와 API 비용 문제가 언급된다. 또한, 본 논문에서는 BF16을 사용했으나, 일부 연구에서 제안된 FP16의 안정성 문제는 긴 컨텍스트(64K) 상황에서 수치적 오버플로우로 인해 적용하기 어려웠다는 점은 향후 해결해야 할 과제이다.

비판적으로 해석하자면, 본 모델의 성능 향상은 상당 부분 고품질의 합성 데이터와 대규모 RL에 의존하고 있다. 모델이 실제로 '이해'하여 검색 전략을 수정하는 것인지, 아니면 학습 데이터에 포함된 정교한 궤적의 패턴을 모방하는 것인지에 대한 더 깊은 분석이 필요할 수 있다.

## 📌 TL;DR

Vision-DeepResearch는 MLLM의 시각 검색 적중률 문제를 해결하기 위해 **다중 스케일 크롭 검색**을 도입하고, 텍스트 기반 딥 리서치 모델의 능력을 전이시킨 **롱-호라이즌 추론 궤적**을 통해 모델을 학습시킨 연구이다. SFT와 GRPO 기반의 RL을 통해 모델이 수십 단계의 추론과 수백 번의 도구 호출을 수행할 수 있게 하였으며, 그 결과 오픈 소스 MLLM 중 최고 수준의 딥 리서치 성능을 달성하였다. 이 연구는 향후 MLLM이 복잡한 웹 환경에서 자율적으로 정보를 탐색하고 분석하는 진정한 의미의 '멀티모달 에이전트'로 발전하는 데 중요한 이정표를 제시한다.
