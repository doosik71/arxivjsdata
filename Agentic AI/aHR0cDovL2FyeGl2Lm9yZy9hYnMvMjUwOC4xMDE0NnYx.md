# Agentic AI Frameworks: Architectures, Protocols, and Design Challenges

Hana Derouiche, Zaki Brahmi, Haithem Mazeni (2025)

## 🧩 Problem to Solve

최근 대규모 언어 모델(Large Language Models, LLMs)의 발전으로 인해 지능형 에이전트가 목표 지향적 자율성, 문맥 추론, 동적 다중 에이전트 협업 능력을 갖춘 Agentic AI라는 새로운 패러다임이 등장하였다. 그러나 이러한 발전 속도에 비해, 현재 등장한 다양한 Agentic AI 프레임워크들이 설계 철학, 기술적 구성 요소, 실질적인 기능 면에서 어떻게 다른지에 대한 체계적인 이해와 분석이 부족한 실정이다.

본 논문은 CrewAI, LangGraph, AutoGen, Semantic Kernel, Agno, Google ADK, MetaGPT 등 주요 프레임워크들의 아키텍처 원칙, 통신 메커니즘, 메모리 관리, 안전 가드레일(Safety Guardrails)을 비교 분석함으로써 이 분야의 기초적인 분류 체계(Taxonomy)를 확립하는 것을 목표로 한다. 또한, 에이전트 간 통신 프로토콜의 현황을 분석하고, 이러한 시스템들이 서비스 컴퓨팅(Service Computing) 생태계에 통합될 수 있는 수준인지 평가하여 향후 연구 방향을 제시하고자 한다.

## ✨ Key Contributions

본 논문의 핵심적인 기여는 파편화되어 있는 Agentic AI 프레임워크들을 체계적으로 분류하고 비교 분석한 종합적인 참조 모델을 제공했다는 점이다. 구체적인 설계 아이디어는 다음과 같다.

- **통합적 비교 분석:** 단순한 기능 나열이 아니라 메모리, 가드레일, 서비스 컴퓨팅 호환성이라는 구체적인 기준을 통해 서로 다른 설계 철학을 가진 프레임워크들을 대조하였다.
- **통신 프로토콜의 체계화:** MCP, A2A, ANP, ACP, Agora와 같은 최신 에이전트 통신 프로토콜들을 분석하여, 에이전트 간 상호운용성을 높이기 위한 표준화된 통신 계층의 필요성을 역설하였다.
- **서비스 지향 관점의 평가:** Agentic AI를 단순한 도구가 아닌 서비스 컴퓨팅의 관점에서 바라보고, W3C 표준(WSDL, BPEL 등)과의 적합성을 분석하여 'Agent-as-a-Service'로의 확장 가능성을 탐색하였다.

## 📎 Related Works

본 논문은 전통적인 인공지능 에이전트와 현대의 LLM 기반 에이전트를 구분하며 논의를 시작한다.

- **전통적인 AI 에이전트 및 MAS:** 과거의 Multi-Agent Systems(MAS)는 센서를 통해 환경을 인식하고 이펙터를 통해 행동하는 자율 엔티티로 정의되었다. 주로 정적인 규칙이나 BDI(Belief-Desire-Intention) 모델에 기반하여 통제된 환경에서 작동하였으며, 통신은 FIPA ACL과 같은 초기 시맨틱 표준을 따랐다.
- **현대적 LLM 기반 에이전트:** ReAct, PRACT, RAISE, Reflexion과 같은 최신 아키텍처들은 LLM을 추론 엔진으로 활용한다. 이들은 단순한 규칙 기반 행동을 넘어 planning, memory, tool use를 반복적인 루프(Iterative Loops)를 통해 수행하며, 보다 유연하고 인간 중심적인 문맥에서 작동한다.
- **기존 연구의 한계:** 기존 문헌들은 특정 도메인(예: 금융 서비스)에 국한된 리뷰이거나 개별 프레임워크의 특정 기능에만 집중하는 경향이 있어, 전체적인 생태계를 관통하는 아키텍처 수준의 비교 분석이 부족하였다.

## 🛠️ Methodology

본 논문은 Agentic AI 시스템을 분석하기 위해 다음과 같은 방법론적 접근을 취한다.

### 1. 현대적 에이전트의 재정의
저자들은 현대적 에이전트를 "추론 및 통신 능력을 갖추고, 구조화된 문맥을 동적으로 해석하며, 도구를 오케스트레이션하고, 분산 시스템 전반에서 메모리와 상호작용을 통해 행동을 적응시킬 수 있는 자율적이고 협력적인 엔티티"로 정의한다.

### 2. 에이전트 통신 프로토콜 분석
에이전트 간의 상호운용성을 위해 다음과 같은 프로토콜들을 상세 분석한다.
- **MCP (Model Context Protocol):** JSON-RPC를 통한 구조화된 도구 호출과 보안 스키마 검증에 집중하며, 클라이언트-서버 모델을 따른다.
- **A2A (Agent-to-Agent):** Google에서 제안하였으며, Agent Cards, Task Objects, Artifacts와 같은 개념을 통해 메모리 관리와 목표 조정을 정형화한다.
- **ANP (Agent Network Protocol):** 분산 식별자(DIDs)와 JSON-LD 시맨틱을 사용하여 탈중앙화된 에이전트 시장과 생태계를 지향한다.
- **ACP (Agent Communication Protocol):** RESTful API를 기반으로 하며, Web3 환경과 호환되는 전송 불가지론적(Transport-agnostic) 설계를 가진다.
- **Agora:** MCP, ANP, ACP 등을 통합하는 메타 조정 계층(Meta-coordination layer)으로, 프로토콜 문서(PDs)를 통해 에이전트가 적절한 통신 방식을 선택하도록 돕는다.

### 3. 프레임워크 설계 분류 및 모델링
다양한 프레임워크를 오케스트레이션 방식에 따라 분류하고 통합 클래스 모델을 제안한다.
- **역할 기반 협업:** CrewAI, MetaGPT (소프트웨어 엔지니어링 팀 모사)
- **대화 기반 오케스트레이션:** AutoGen (다중 에이전트 대화)
- **그래프 기반 흐름 제어:** LangGraph (상태 머신 기반의 결정론적 흐름)
- **엔터프라이즈 지향:** Semantic Kernel (플래너 및 스킬 실행 중심)
- **데이터 중심:** LlamaIndex (인덱싱된 데이터 쿼리 및 지식 기반 애플리케이션)

### 4. 메모리 및 가드레일 분석
- **메모리 구조:** 단기 메모리(즉각적 문맥)와 장기 메모리(사용자 선호도, 과거 이력)로 구분하며, 특히 Semantic, Procedural, Episodic 메모리의 구현 여부를 분석한다.
- **가드레일:** 출력 값 검증, 보안 강제, 워크플로우 무결성 유지 메커니즘을 평가한다.

## 📊 Results

본 논문은 실험적인 수치 결과보다는 체계적인 비교 분석 결과를 제시한다.

### 1. 통신 프로토콜 비교 결과
대부분의 프로토콜이 HTTP 전송 계층을 사용하고 있으나, 시맨틱 표현 방식(Custom performatives vs Goal-oriented messages)에서 큰 차이가 있어 완전한 상호운용성에는 한계가 있음을 발견하였다.

### 2. 프레임워크별 메모리 지원 현황
- **LangGraph:** 그래프 노드 간의 상태(State) 유지를 통해 문맥을 보존한다.
- **CrewAI 및 AutoGen:** 에이전트 수준의 메모리와 공유 메모리 컨텍스트를 지원하여 역할 간 협업을 가능케 한다.
- **Semantic Kernel:** 플래너 및 스킬과 통합된 확장 가능한 메모리 모듈을 제공한다.
- **LlamaIndex:** 임베딩 기반의 대규모 데이터 검색을 통한 외부 메모리 활용에 특화되어 있다.

### 3. 서비스 컴퓨팅 호환성 평가
대부분의 프레임워크가 동적 발견(Discovery)이나 발행(Publishing) 기능을 기본적으로 제공하지 않는다.
- **Semantic Kernel**과 **Google ADK**는 서비스 구성(Composition) 능력이 뛰어나지만, 여전히 외부 레지스트리에 의존한다.
- **LangGraph**는 상태 머신 추상화를 통해 가장 견고한 구성 패턴을 보이며, 외부 카탈로그 어댑터를 통해 발견 기능을 확장할 가능성이 높다.
- W3C 표준(WSDL, BPEL 등)의 경우, CrewAI의 JSON 스키마가 WSDL의 역할을 일부 수행하고 AutoGen의 워크플로우가 BPEL과 유사한 형태를 띠지만, 공식적인 표준 채택 수준은 매우 낮다.

## 🧠 Insights & Discussion

### 강점 및 성과
본 연구는 파편화된 Agentic AI 생태계를 아키텍처, 통신, 메모리, 서비스 컴퓨팅이라는 네 가지 핵심 축으로 정리하여, 개발자와 연구자가 자신의 목적에 맞는 프레임워크를 선택할 수 있는 명확한 기준을 제시하였다.

### 한계 및 비판적 해석
논문에서 지적하는 현재 시스템의 주요 한계는 다음과 같다.
1. **아키텍처의 경직성:** MetaGPT나 CrewAI처럼 역할이 사전에 정의된 경우, 실행 중에 동적으로 역할을 변경하는 것이 어렵다.
2. **런타임 발견의 부재:** 에이전트들이 실행 중에 서로의 능력을 발견하고 협업하는 기능이 부족하며, 대부분 정적으로 정의된 상호작용에 의존한다. 이를 해결하기 위해 '에이전트/스킬 레지스트리' 도입이 시급하다.
3. **코드 실행 안전성:** AutoGen과 MetaGPT처럼 코드를 생성하고 실행하는 경우, 시스템 파일 접근이나 쉘 명령 실행과 같은 심각한 보안 리스크가 존재한다. Docker와 같은 샌드박스 환경 도입이 필수적이다.
4. **상호운용성 격차:** 각 프레임워크가 서로 다른 추상화 모델을 사용하므로, CrewAI의 태스크 모델을 AutoGen 에이전트가 직접 이해할 수 없는 등의 고립(Silo) 현상이 발생한다.

### 향후 방향
저자들은 Agentic AI가 단순한 작업 수행 플랫폼을 넘어 진정한 서비스 컴퓨팅 솔루션이 되기 위해서는, AI 에이전트를 RESTful API로 래핑하여 서비스화하는 SOA(Service-Oriented Architecture) 원칙의 도입과 FIPA-ACL과 같은 표준 통신 프로토콜의 현대적 재해석이 필요하다고 주장한다.

## 📌 TL;DR

본 논문은 현대의 LLM 기반 Agentic AI 프레임워크들을 아키텍처, 통신 프로토콜, 메모리, 가드레일 관점에서 체계적으로 분석한 종합 리뷰 보고서이다. 주요 기여는 서로 다른 설계 철학을 가진 프레임워크들을 비교 분류하고, 에이전트 간 상호운용성을 위한 통신 표준의 필요성을 제기한 것이다. 현재의 프레임워크들은 개별적인 기능은 뛰어나나 상호운용성과 런타임 자율 발견, 보안성 면에서 한계가 있으며, 이를 해결하기 위해 서비스 지향 아키텍처(SOA) 및 W3C 표준의 도입이 중요함을 시사한다. 이 연구는 향후 확장 가능하고 견고한 다중 에이전트 생태계를 구축하려는 연구자들에게 핵심적인 참조 가이드가 될 것이다.
