# An Overview of Deep Semi-Supervised Learning

Yassine Ouali, Céline Hudelot, Myriam Tami (2020)

## 🧩 Problem to Solve

본 논문은 딥러닝 모델을 학습시키기 위해 필수적인 대규모 레이블 데이터셋(Labeled Dataset) 구축에 소요되는 막대한 비용, 시간 및 노력의 문제를 해결하고자 한다. 지도 학습(Supervised Learning)은 대량의 정답 데이터가 있을 때 뛰어난 성능을 보이지만, 실제 많은 응용 분야에서는 레이블이 지정된 데이터 $D_l$은 매우 적고, 레이블이 없는 데이터 $D_u$가 훨씬 더 많이 존재하는 상황에 직면한다.

따라서 본 논문의 목표는 적은 양의 레이블된 데이터와 대량의 레이블되지 않은 데이터를 동시에 활용하여, 레이블된 데이터만 사용했을 때보다 더 정확한 예측 함수 $f_\theta$를 학습시키는 준지도 학습(Semi-Supervised Learning, SSL)의 최신 동향과 방법론을 체계적으로 분석하고 종합적으로 제공하는 것이다.

## ✨ Key Contributions

본 논문은 딥러닝 기반 준지도 학습(Deep SSL)의 방대한 연구 흐름을 분석하여 다음과 같은 핵심 분류 체계를 제시한다.

1. **일관성 규제(Consistency Regularization)**: 입력에 미세한 섭동(Perturbation)을 가해도 모델의 예측 결과가 변하지 않아야 한다는 직관을 이용한다.
2. **엔트로피 최소화(Entropy Minimization)**: 레이블이 없는 데이터에 대해 모델이 확신을 가지고 예측(Low-entropy)하도록 유도한다.
3. **프록시 레이블 방법(Proxy-label Methods)**: 모델이 생성한 가짜 레이블(Pseudo-label)을 정답처럼 활용하여 학습 데이터를 확장한다.
4. **생성 모델(Generative Models)**: VAE나 GAN을 통해 데이터의 분포 $p(x)$를 학습함으로써 판별 모델의 성능을 보조한다.
5. **그래프 기반 방법(Graph-Based Methods)**: 데이터 간의 유사도를 그래프의 엣지로 표현하고, 레이블 정보를 그래프 전체로 전파(Propagation)한다.

## 📎 Related Works

논문은 SSL과 밀접하게 연관된 다음과 같은 연구 분야들을 소개하며 차이점을 설명한다.

- **능동 학습(Active Learning)**: 모델이 학습에 가장 도움이 될 만한 데이터를 선택하여 사용자에게 레이블링을 요청하는 상호작용 방식이다. 반면 SSL은 주어진 $D_u$를 그대로 활용한다.
- **전이 학습 및 도메인 적응(Transfer Learning & Domain Adaptation)**: 소스 도메인의 지식을 타겟 도메인으로 전이하는 것이다. SSL과 유사하게 레이블 없는 데이터를 사용하지만, SSL은 $D_l$과 $D_u$가 동일한 분포에서 왔다고 가정하는 반면, 도메인 적응은 두 분포가 다르다는 점이 핵심이다.
- **약지도 학습(Weakly-Supervised Learning)**: 정교한 레이블 대신 저품질의 레이블(예: 이미지 레벨 레이블로 픽셀 레벨 세그멘테이션 수행)을 사용하는 방식이다.
- **노이즈 레이블 학습(Learning with Noisy Labels)**: 잘못된 레이블이 포함된 데이터셋에서 강건한 모델을 학습하는 연구이다. 일부 SSL 기법은 노이즈 레이블 데이터를 $D_u$로 간주하여 해결책으로 제시되기도 한다.

## 🛠️ Methodology

논문은 Deep SSL의 방법론을 다섯 가지 주요 범주로 나누어 상세히 설명한다.

### 1. 일관성 규제 (Consistency Regularization)

데이터에 노이즈를 추가해도 예측값이 일정해야 한다는 **Cluster Assumption**에 기반한다.

- **$\Pi$-Model**: 동일한 입력 $x$에 대해 서로 다른 드롭아웃이나 증강을 적용해 두 번의 예측 $\tilde{y}_1, \tilde{y}_2$를 수행하고, 이 둘의 거리(MSE 등)를 최소화한다.
- **Mean Teacher**: 학생 모델(Student)과 교사 모델(Teacher)을 둔다. 교사 모델의 가중치는 학생 모델 가중치의 지수 이동 평균(EMA)으로 업데이트하며, 학생은 교사의 예측값을 타겟으로 삼아 학습한다.
- **VAT (Virtual Adversarial Training)**: 무작위 노이즈 대신, 모델의 예측값을 가장 크게 변화시키는 '가장 위험한 방향'의 섭동 $r_{adv}$를 계산하여 일관성을 강제한다.
- **ICT (Interpolation Consistency Training)**: 두 데이터 포인트 사이의 보간(MixUp) 결과가 각 데이터 예측값의 보간 결과와 일치하도록 학습한다.

### 2. 엔트로피 최소화 (Entropy Minimization)

모델이 레이블 없는 데이터에 대해 모호한 예측을 하지 않고, 하나의 클래스로 강하게 예측하도록 유도한다. 손실 함수는 다음과 같다.
$$L_{ent} = -\sum_{k=1}^{C} f_\theta(x)_k \log f_\theta(x)_k$$
다만, 단독 사용 시 오버피팅 위험이 커 다른 기법과 결합하여 사용한다.

### 3. 프록시 레이블 방법 (Proxy-label Methods)

- **Self-training**: 모델이 예측한 값 중 신뢰도가 임계값 $\tau$보다 높은 샘플에 프록시 레이블을 부여하고, 이를 학습 세트에 추가하여 반복 학습한다.
- **Co-training**: 데이터의 서로 다른 뷰(View)를 학습하는 두 모델이 서로에게 신뢰도 높은 레이블을 제공하는 방식이다.
- **MPL (Meta Pseudo Labels)**: 메타 학습을 통해 학생 모델의 검증 손실을 최소화하는 방향으로 교사 모델이 프록시 레이블을 생성하도록 최적화한다.

### 4. 통합 방법론 (Holistic Methods)

위의 기법들을 하나로 합친 최신 프레임워크들이다.

- **MixMatch**: 증강 $\rightarrow$ 레이블 추측(평균) $\rightarrow$ 샤프닝(Sharpening, 엔트로피 감소) $\rightarrow$ MixUp 과정을 거쳐 학습 데이터를 생성하고 지도/준지도 손실을 함께 최적화한다.
- **FixMatch**: 약하게 증강된 이미지($A_w$)의 예측값이 임계값 $\tau$를 넘으면 이를 정답으로 간주하고, 강하게 증강된 이미지($A_s$)의 예측값이 이 정답과 일치하도록 학습한다.

### 5. 생성 모델 및 그래프 기반 방법

- **Generative Models**: VAE는 잠재 공간(Latent Space)을 학습하여 분류 성능을 높이며, GAN은 판별기(Discriminator)가 실제 데이터와 생성 데이터의 경계를 찾게 함으로써 클래스 경계면을 더 정교하게 학습하게 한다.
- **Graph-Based SSL**: 데이터 포인트를 노드로, 유사도를 엣지로 정의한다. **Label Propagation**은 그래프 라플라시안(Graph Laplacian)을 이용하여 레이블 정보를 주변 노드로 전파한다.
$$\hat{Y}_u = (I - P_{uu})^{-1} P_{ul} Y_l$$

## 📊 Results

본 논문은 개별 알고리즘의 성능 수치보다는, SSL 모델을 올바르게 평가하기 위한 **실험적 가이드라인**을 제시하는 데 집중한다.

- **평가 절차**: CIFAR-10, SVHN 등의 데이터셋에서 레이블의 일부(1~10%)만 남기고 나머지를 $D_u$로 설정하여 성능을 측정한다.
- **권장 평가 지표 및 방법**:
    1. **공통 구현체(Shared Implementation)**: 동일한 아키텍처와 하이퍼파라미터를 사용하여 공정하게 비교해야 한다.
    2. **강력한 지도 학습 베이스라인**: SSL의 목표는 지도 학습보다 나은 성능을 내는 것이므로, 주어진 $D_l$만으로 최적화한 모델과 비교해야 한다.
    3. **전이 학습과의 비교**: 대규모 데이터셋에서 사전 학습 후 미세 조정(Fine-tuning)한 결과와 비교하여 SSL의 실질적 이득을 확인해야 한다.
    4. **클래스 분포 불일치 고려**: 실제 환경에서는 $D_l$과 $D_u$의 클래스 분포가 다를 수 있음을 명시하고 이를 테스트해야 한다.

## 🧠 Insights & Discussion

본 논문은 Deep SSL의 발전 과정을 통해 다음과 같은 통찰을 제공한다.

- **강점**: 단순한 데이터 증강을 넘어, 모델의 예측 일관성(Consistency)을 강제하고 생성 모델의 분포 학습 능력을 결합함으로써 매우 적은 레이블만으로도 높은 성능을 낼 수 있음을 보여준다.
- **한계 및 위험요소**: 특히 프록시 레이블 방식에서 발생하는 **Confirmation Bias(확증 편향)**가 주요 문제로 지적된다. 모델이 틀린 예측을 정답으로 믿고 학습하면 그 오류가 증폭되는 현상이다. 이를 해결하기 위해 MixMatch의 샤프닝이나 FixMatch의 임계값 필터링 같은 기법들이 도입되었다.
- **비판적 해석**: 많은 SSL 방법론이 특정 데이터셋(CIFAR 등)의 특성에 의존하는 경향이 있으며, 실제 산업 현장의 '분포가 다른 데이터' 상황에서의 강건성에 대해서는 더 많은 연구가 필요하다. 또한, VAT와 같은 방법은 계산 비용이 매우 높다는 실용적 한계가 있다.

## 📌 TL;DR

이 논문은 딥러닝 기반 준지도 학습(Deep SSL)의 핵심 패러다임인 **일관성 규제, 엔트로피 최소화, 프록시 레이블링, 생성 모델, 그래프 기반 방법**을 집대성한 종합 리뷰 보고서이다. 특히 FixMatch와 같은 최신 통합 방법론과 SSL 모델의 올바른 평가 방법론을 제시함으로써, 향후 데이터 효율적인 딥러닝 모델을 설계하려는 연구자들에게 필수적인 이론적 지도(Map) 역할을 한다.
