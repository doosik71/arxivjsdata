# Metrics Matter in Surgical Phase Recognition

Isabel Funke, Dominik Rivoir, Stefanie Speidel (2023)

## 🧩 Problem to Solve

본 논문이 해결하고자 하는 핵심 문제는 수술 단계 인식(Surgical Phase Recognition) 분야, 특히 가장 널리 사용되는 벤치마크 데이터셋인 Cholec80에서의 **평가 프로토콜의 불일치(Inconsistency)**이다.

최근 많은 연구가 자동 수술 단계 인식 모델을 제안하며 유망한 결과를 보고하고 있으나, 다음과 같은 이유로 서로 다른 방법론 간의 유의미한 비교가 어렵다.

- **데이터 분할의 다양성**: 학습, 검증, 테스트 세트를 나누는 기준이 연구마다 다르다.
- **지표 계산 방식의 차이**: 동일한 지표(예: F1-score)를 사용하더라도 이를 계산하는 세부 구현 방식이 다르다.
- **불완전한 보고**: 평가 세부 사항(데이터 분할, 변동성 측정 방법 등)이 논문에 명시되지 않은 경우가 많다.

이러한 불일치는 결과적으로 모델의 성능이 과대평가될 수 있게 하며, 실제 임상 적용을 위한 신뢰할 수 있는 결론을 도출하는 데 방해가 된다. 따라서 본 논문의 목표는 Cholec80 벤치마크에서 발생하는 일반적인 평가 오류와 편차를 분석하고, 표준화된 평가 체계의 중요성을 제고하는 것이다.

## ✨ Key Contributions

본 논문의 중심적인 기여는 수술 단계 인식 모델의 성능을 측정하는 **평가 지표의 계산 방식이 최종 결과에 얼마나 큰 영향을 미치는지 정량적으로 분석**한 것이다. 주요 기여 사항은 다음과 같다.

1. **평가 불일치 사례 정리**: 데이터 분할 방식, 표준 편차 계산법, F1-score의 정의, 정의되지 않은 값(Undefined values) 처리 전략 등에서 나타나는 흔한 오류와 편차를 체계적으로 정리하였다.
2. **지표 변동성 실증 분석**: Baseline 모델을 구축하고, 다양한 평가 전략(Strategy A vs B, Regular vs Relaxed metrics 등)을 적용하여 동일한 모델이라도 평가 방식에 따라 결과값이 어떻게 달라지는지 보여주었다.
3. **기존 SOTA 모델 재검토**: 기존 연구들의 결과를 구조화된 표로 정리하여, 서로 다른 평가 프로토콜을 사용한 연구들 간의 직접적인 비교가 불가능함을 입증하였다.
4. **평가 가이드라인 제시**: 향후 연구자들이 일관성 있고 비교 가능한 결과를 도출할 수 있도록 구체적인 평가 권장 사항을 제안하였다.

## 📎 Related Works

논문은 Cholec80 데이터셋을 활용한 다양한 기존 연구들을 소개한다.

- **기존 접근 방식**: SV-RCNet, TeCNO, Trans-SVNet 등 CNN-LSTM, TCN, Transformer 기반의 모델들이 제안되었다. 이들은 주로 시각적 특징 추출기(Visual Feature Extractor)와 시계열 모델(Temporal Model)을 결합한 구조를 가진다.
- **한계점**: 대부분의 연구가 모델 아키텍처의 개선에만 집중했을 뿐, 평가 프로토콜의 엄밀함에는 주의를 기울이지 않았다. 특히 '완화된 경계(Relaxed boundaries)' 지표를 사용하면서도 이를 명시적으로 밝히지 않거나, 검증 세트 없이 테스트 세트에서 하이퍼파라미터 튜닝을 진행하여 일반화 성능이 과대평가되는 경향이 있다.
- **차별점**: 본 논문은 새로운 모델 구조를 제안하는 것이 아니라, **"어떻게 측정하는가"**라는 평가론적 관점에서 접근하여 벤치마크의 신뢰성을 회복하려 한다는 점에서 기존 연구들과 명확히 차별화된다.

## 🛠️ Methodology

### 1. 수술 단계 인식 정의

비디오 $v$의 전체 길이 $T$에 대해, 각 시간 $t$에서의 정답 라벨을 $y(v)_t$, 모델의 예측 라벨을 $\hat{y}(v)_t$라고 정의한다. 여기서 라벨은 수술 단계의 집합 $P$에 속한다.

### 2. 핵심 평가 지표 및 방정식

본 논문은 혼동 행렬(Confusion Matrix) $C$를 기반으로 지표를 정의한다. $C_{pq}$는 실제 단계가 $p$이고 예측 단계가 $q$인 프레임의 수이다.

- **Precision, Recall, F1-score, Jaccard Index**:
  각 단계 $p$에 대해 다음과 같이 계산한다.
  $$\text{Precision}_p = \frac{C_{pp}}{\sum_{q} C_{qp}}, \quad \text{Recall}_p = \frac{C_{pp}}{\sum_{q} C_{pq}}$$
  $$\text{F1}_p = \frac{2 \cdot \text{Precision}_p \cdot \text{Recall}_p}{\text{Precision}_p + \text{Recall}_p}, \quad \text{Jaccard}_p = \frac{C_{pp}}{\sum_{q} C_{pq} + \sum_{q} C_{qp} - C_{pp}}$$

- **Macro-averaged Metrics**:
  모든 단계 $p \in P$에 대해 개별 지표를 계산한 후 산술 평균을 구한다.
  $$\text{Macro F1} = \frac{1}{|P|} \sum_{p \in P} \text{F1}_p$$

### 3. 평가상의 주요 쟁점 및 변형 방식

논문은 결과에 영향을 주는 세 가지 주요 변수와 해결책을 제시한다.

- **F1-score 계산의 차이**:
  - $\text{Macro F1}$: 각 단계별 F1을 구한 뒤 평균 (본 논문 권장).
  - $\text{MacroF1}$: 전체 평균 Precision과 Recall을 이용해 조화 평균을 계산.
  - 결과적으로 $\text{F1} \ge \text{MacroF1} \ge \text{Macro F1}$의 관계가 성립하며, 계산 방식에 따라 수치가 크게 달라질 수 있다.

- **정의되지 않은 값(Undefined values) 처리**:
  분모가 0이 되는 경우(예: 특정 비디오에 해당 단계가 없는 경우) 처리 전략이 다르다.
  - **Strategy A**: 정의되지 않은 값만 계산에서 제외한다.
  - **Strategy B**: 해당 단계가 정답 라벨에 없으면, 예측값이 있더라도 해당 비디오의 모든 지표를 제외한다.

- **완화된 지표(Relaxed Metrics)**:
  단계 전환 시점의 $\omega$ 시간(보통 10초) 동안 발생하는 오차를 허용하는 방식이다.
  - 정답 단계 $q$의 시작 $\omega$초 동안 예측값 $\hat{q}$가 $( \hat{q}, q ) \in T$(유효한 전이)라면 정답으로 인정한다.
  - 이는 짧은 단계의 성능을 인위적으로 높이는 효과가 있으며, 특히 기존 MATLAB 구현체에 심각한 버그가 있음을 지적한다.

### 4. 실험적 검증을 위한 Baseline 모델

분석을 위해 다음과 같은 Baseline 모델을 구축하였다.

- **구조**: ResNet-50 (시각 특징 추출) + LSTM (시계열 모델).
- **학습 절차**:
  - ResNet-50은 ImageNet 가중치로 초기화하고 초기 3개 레이어는 동결(Frozen)하였다.
  - AdamW 옵티마이저와 One Cycle Learning Rate 스케줄을 사용하였다.
  - 32:8:40 데이터 분할(학습:검증:테스트)을 적용하여 하이퍼파라미터를 튜닝하였다.

## 📊 Results

### 1. 평가 전략에 따른 수치 변화 (Baseline 모델 결과)

동일한 모델임에도 불구하고 계산 전략에 따라 결과가 크게 변했다.

- **Strategy A vs B**: Strategy B(해당 단계 부재 시 제외)를 사용할 때 지표가 더 높게 나타났다. 이는 데이터셋의 불균형성과 특정 단계의 부재가 결과에 영향을 주기 때문이다.
- **Macro F1 vs MacroF1**: MacroF1 방식이 Macro F1보다 상당히 높은 수치를 기록하였다.
- **Regular vs Relaxed**: 완화된 지표(Relaxed metrics)를 적용했을 때 성능이 비약적으로 상승하였다. 특히 길이가 짧은 단계에서 $\Delta$값이 매우 크게 나타났으며, 이는 완화된 지표가 모델의 실제 성능보다 과하게 관대한 평가를 내리고 있음을 보여준다.

### 2. 기존 연구(SOTA) 비교 분석

논문은 기존 연구들을 정리하여 두 가지 '분파'가 형성되었음을 발견했다.

- **Relaxed-branch**: 완화된 지표를 사용하여 매우 높은 Precision/Recall(90% 이상)을 보고하는 그룹.
- **Regular-branch**: 일반 지표를 사용하여 상대적으로 낮은 수치를 보고하는 그룹.
문제는 많은 연구자가 본인이 완화된 지표를 사용했음을 명시하지 않았다는 점이며, 이로 인해 서로 다른 분파의 결과물을 직접 비교하는 것은 무의미하다는 결론에 도달했다.

## 🧠 Insights & Discussion

### 1. 강점 및 비판적 해석

본 논문은 모델의 구조적 개선보다 **평가의 정밀함**이 학문의 발전에 더 중요하다는 점을 매우 설득력 있게 제시하였다. 특히, 단순한 수치 비교가 아닌, 수식 수준에서의 정의 차이($\text{Macro F1}$ vs $\text{MacroF1}$)와 구현 수준의 버그(MATLAB 스크립트 오류)를 찾아냄으로써 기존 벤치마크의 허점을 날카롭게 지적하였다.

### 2. 한계 및 논의사항

- **데이터셋의 한계**: 분석이 Cholec80이라는 단일 데이터셋에 집중되어 있어, 다른 수술 데이터셋에서도 동일한 현상이 나타나는지에 대한 일반화 가능성은 추가 확인이 필요하다.
- **구현체 의존성**: MATLAB 스크립트의 버그를 지적하였으나, 이미 많은 연구가 이를 기반으로 결과를 냈기에 과거 데이터의 전면적인 재평가가 필요하다는 현실적인 어려움이 있다.

### 3. 결론적 제언

저자들은 향후 연구자들에게 다음과 같은 가이드라인을 제시한다.

- 독립된 검증 세트(Validation set)를 반드시 사용하고, 여러 랜덤 시드(Random seed)로 실험을 반복하여 평균과 표준편차를 보고할 것.
- $\text{M(Precision)}$과 $\text{M(Recall)}$만 보고하지 말고, $\text{M(F1)}$이나 $\text{M(Jaccard)}$를 함께 보고하여 지표의 상한선(Upper bound) 오류를 피할 것.
- **완화된 지표(Relaxed metrics)의 사용을 중단할 것.**

## 📌 TL;DR

본 연구는 수술 단계 인식의 표준 벤치마크인 Cholec80에서 **평가 방식의 불일치로 인해 모델 성능이 왜곡되고 있음**을 폭로한다. 데이터 분할, F1-score 계산법, 완화된 경계 적용 여부에 따라 동일 모델의 결과가 완전히 달라짐을 실증적으로 보여주었으며, 특히 기존에 널리 쓰인 완화된 지표의 심각한 결함과 구현 버그를 지적하였다. 이 연구는 향후 수술 단계 인식 분야에서 신뢰할 수 있는 성능 비교를 위해 **엄격하고 표준화된 평가 프로토콜의 도입이 필수적임**을 시사한다.
