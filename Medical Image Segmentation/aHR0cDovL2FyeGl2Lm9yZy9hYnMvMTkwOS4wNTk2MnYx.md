# SegNAS3D: Network Architecture Search with Derivative-Free Global Optimization for 3D Image Segmentation

Ken C. L. Wong, Mehdi Moradi (2019)

## 🧩 Problem to Solve

본 논문은 3D 의료 영상 분할(Medical Image Segmentation)을 위한 신경망 구조 자동 탐색(Network Architecture Search, NAS) 문제를 해결하고자 한다. 딥러닝의 발전으로 수동 특징 선택의 필요성은 줄었으나, 여전히 최적의 네트워크 구조와 하이퍼파라미터를 설정하는 과정은 전문가의 수작업에 의존하며, 이는 매우 많은 시간과 비용을 소모한다.

특히 3D 영상 분할 분야에서 NAS를 적용하는 데에는 다음과 같은 몇 가지 핵심적인 어려움이 존재한다. 첫째, 3D 데이터는 2D 데이터보다 계산 복잡도가 훨씬 높아 탐색 비용이 기하급수적으로 증가한다. 둘째, 구조 탐색 공간 내에 다수의 지역 최적점(Local Optima)이 존재할 가능성이 커 기존의 경사 하강법 기반 최적화로는 한계가 있다. 셋째, 3D 모델의 크기로 인해 학습 도중 메모리 부족(Out-of-Memory, OOM) 오류가 빈번하게 발생하며, 이를 최적화 과정에서 어떻게 처리할 것인지에 대한 대안이 필요하다. 따라서 본 연구의 목표는 계산 효율성을 확보하면서도 3D 의료 영상 분할에 최적화된 구조를 자동으로 찾을 수 있는 NAS 프레임워크인 SegNAS3D를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 계산 가능성을 확보하기 위해 전체 네트워크를 반복적인 블록 구조로 설계하고, 이를 미분 불가능한 목적 함수에서도 작동하는 전역 최적화(Global Optimization) 기법으로 탐색하는 것이다. 주요 기여 사항은 다음과 같다.

1. **DAG 기반의 학습 가능한 블록 구조**: 전체 네트워크를 반복되는 블록들의 조합으로 구성하며, 각 블록 내부의 연결 관계를 Directed Acyclic Graph(DAG)로 표현하여 탐색 공간을 효율적으로 정의하였다.
2. **미분 불가능한 전역 최적화 적용**: 하이퍼파라미터 공간을 연속적으로 완화(Continuous Relaxation)하여 표현하고, Controlled Random Search(CRS)라는 미분 불필요(Derivative-free) 전역 최적화 알고리즘을 사용하여 지역 최적점 문제를 해결하고 OOM 오류와 같은 비정상적인 상황을 처리하였다.
3. **블록 연결 하이퍼파라미터의 통합 탐색**: 단순히 블록 내부 구조뿐만 아니라, 채널 수, 풀링 횟수, Deep Supervision 적용 여부 등 네트워크의 전체적인 연결 구조를 결정하는 하이퍼파라미터를 함께 최적화하였다.

## 📎 Related Works

기존의 NAS 연구들은 주로 이미지 분류(Image Classification) 작업에 집중되어 왔으며, 의료 영상 분할, 특히 3D 영역에 적용된 사례는 매우 제한적이다. 일부 2D 의료 영상 분할을 위한 NAS 연구에서는 Policy Gradient 강화학습을 사용하여 커널 크기나 채널 수를 학습시켰으나, 이는 레이어 간의 연결 구조(Interconnections)를 학습하는 것이 아니라 단순한 하이퍼파라미터 튜닝에 가까운 수준이었다.

또한, 3D 영상의 경우 연산량과 메모리 요구량이 매우 높기 때문에 기존의 NAS 방식을 그대로 적용하기에는 계산 복잡도가 너무 크다. 본 논문은 이러한 한계를 극복하기 위해 DARTS와 같은 블록 기반 구조를 차용하되, 경사 하강법 대신 전역 최적화 기법을 도입함으로써 불연속적인 목적 함수와 메모리 제약 문제를 동시에 해결하려 하였다.

## 🛠️ Methodology

### 전체 파이프라인 및 시스템 구조
SegNAS3D는 U-Net 및 V-Net과 유사한 인코더-디코더 구조를 기반으로 한다. 네트워크는 여러 개의 **MegaBlock**으로 구성되며, 각 MegaBlock은 학습 가능한 내부 블록 구조, Spatial Dropout, 그리고 선택적인 Residual Connection으로 이루어져 있다.

### 주요 구성 요소 및 상세 설명

**1. 블록 구조 (Block Structure)**
각 블록은 DAG로 표현되며, 이를 구현하기 위해 상삼각 행렬(Upper Triangular Operation Matrix)을 사용한다. 행과 열은 각각 입력과 출력 노드를 의미하며, 행렬의 각 원소는 다음과 같은 연산(Operation) 번호로 매핑된다.
- $\text{Conv}(1, 1), \text{Conv}(3, 1), \text{Conv}(5, 1)$ (Dilation=1)
- $\text{Conv}(3, 2), \text{Conv}(5, 2)$ (Dilation=2)
- Skip connection 및 None
각 컨볼루션 연산 뒤에는 Batch Normalization과 ReLU 활성화 함수가 뒤따른다. 노드 간 결합은 메모리 효율을 위해 Concatenation 대신 Summation을 사용한다.

**2. 네트워크 연결 하이퍼파라미터**
블록 내부 구조 외에 다음과 같은 상위 수준의 하이퍼파라미터를 함께 최적화한다.
- $n$: 첫 번째 블록의 특징 채널 수 (이후 풀링 시 2배 증가, 업샘플링 시 1/2 감소)
- $p$: Max pooling의 횟수
- $\text{sup}$: Deep Supervision 사용 여부 (Binary)
- $\text{res}$: 블록 내 Residual Connection 사용 여부 (Binary)

**3. 전역 최적화 및 연속적 완화 (Continuous Relaxation)**
하이퍼파라미터 조합의 수가 매우 방대하므로 전수 조사는 불가능하다. 따라서 정수 제약을 제거한 연속 공간에서 최적화를 수행하기 위해 continuous relaxation을 적용하고, 실제 네트워크를 구축할 때는 바닥 함수(floor function) $\lfloor x \rfloor$를 통해 정수 값으로 변환한다.

목적 함수 $f$는 검증 데이터셋의 Dice 계수를 이용하여 다음과 같이 정의한다.
$$f = -\ln(\text{Dice})$$

최적화 알고리즘으로는 **Controlled Random Search (CRS)**를 사용한다. CRS는 초기 표본 집합에서 시작하여 무작위화된 Nelder-Mead 알고리즘과 유사한 방식으로 점진적으로 진화하며 최적점을 찾는다.

**4. 예외 상황 처리 (OOM 및 Illegal Structure)**
구조적으로 유효하지 않은 블록(소스/싱크 노드 문제)이나 메모리 부족(OOM)으로 인해 학습이 불가능한 경우, 해당 구조에 대해 매우 높은 패널티 값인 $\max f_e = 10$을 부여하여 최적화 알고리즘이 해당 영역을 피하도록 유도한다.

### 학습 절차
- **데이터 증강**: 회전($\pm 30^\circ$), 이동($\pm 20\%$), 스케일링($[0.8, 1.2]$)을 적용한다.
- **최적화 도구**: Nadam 옵티마이저를 사용하며, 학습률은 $10^{-3}$으로 설정한다.
- **손실 함수**: Dice loss와 Cross-entropy가 결합된 Exponential Logarithmic Loss를 사용한다.

## 📊 Results

### 실험 설정
- **데이터셋**: 43개의 3D 뇌 MRI 영상 (19개 해부학적 구조 라벨링).
- **데이터 분할**: 학습 50%, 검증 20%, 테스트 30%로 분할하여 3회 반복 실험.
- **비교 대상**:
    - **SegNAS11**: 모든 하이퍼파라미터(블록 구조 + 연결 구조) 최적화.
    - **SegNAS4**: 연결 하이퍼파라미터만 최적화 (블록 구조 고정).
    - **SegNAS7**: 블록 구조만 최적화 (연결 구조 고정).
    - **3D U-Net / V-Net**: 기존 수동 설계 모델.

### 주요 결과
정량적 결과는 Table 3에 요약되어 있으며, 주요 내용은 다음과 같다.

- **성능 (Dice %)**: $\text{SegNAS}_{11}$이 평균 $81.7\%$로 가장 높은 성능을 보였으며, $\text{SegNAS}_{4}$는 $81.0\%$로 그 뒤를 이었다. 반면 V-Net은 $47.9\%$로 매우 낮은 성능을 보였다.
- **모델 크기 (Parameters)**: $\text{SegNAS}_{4}$가 가장 작은 모델($3.2\text{M}$)을 생성하였으며, $\text{SegNAS}_{11}$은 $9.7\text{M}$ 수준이었다. V-Net은 $71.1\text{M}$으로 매우 거대했다.
- **계산 비용 (GPU days)**: 각 탐색은 3대의 GPU로 3일 이내에 완료되었다. $\text{SegNAS}_{4}$가 가장 적은 탐색 비용을 소모했다.
- **특이 사항**: 3D U-Net은 메모리 사용량이 너무 많아 OOM 에러로 인해 실험이 불가능하였다.

## 🧠 Insights & Discussion

**1. 하이퍼파라미터의 영향력**
실험 결과, 블록 내부의 세부 구조($\text{ops}$)를 최적화하는 것보다 네트워크의 전체적인 연결 구조($n, p, \text{sup}, \text{res}$)를 최적화하는 것이 성능 향상에 더 효과적임이 밝혀졌다. 이는 $\text{SegNAS}_{4}$가 매우 단순한 블록 구조를 가졌음에도 불구하고 높은 Dice 계수를 기록한 점에서 확인할 수 있다.

**2. 모델 크기와 데이터 양의 상관관계**
V-Net이 매우 낮은 성능을 보인 이유는 모델의 파라미터 수에 비해 학습 데이터가 턱없이 부족하여 과적합(Overfitting)이 발생했기 때문으로 분석된다. 실제로 학습 데이터를 $50\%$에서 $70\%$로 늘렸을 때 V-Net의 성능이 $68.1\%$까지 상승한 점이 이를 뒷받침한다. 반면 SegNAS3D는 데이터 양에 적합한 작은 크기의 모델을 자동으로 탐색함으로써 적은 데이터로도 높은 성능을 낼 수 있었다.

**3. 전역 최적화의 이점**
미분 기반의 NAS는 OOM 에러가 발생했을 때 기울기(Gradient)를 계산할 수 없어 중단되지만, 본 논문의 CRS 기반 접근법은 OOM 상황을 단순한 패널티 값으로 처리함으로써 탐색 과정을 중단 없이 지속할 수 있었다.

## 📌 TL;DR

본 논문은 3D 의료 영상 분할을 위해 **DAG 기반의 블록 구조**와 **미분 불필요 전역 최적화(CRS)**를 결합한 **SegNAS3D** 프레임워크를 제안하였다. 이 방법론은 계산 복잡도가 높은 3D 영상에서도 효율적인 구조 탐색을 가능하게 하며, 특히 **메모리 부족(OOM) 문제를 최적화 과정에 통합하여 해결**한 점이 특징이다. 실험 결과, 수동으로 설계된 V-Net이나 3D U-Net보다 훨씬 작은 모델 크기로도 더 높은 분할 정확도(Dice 82%)를 달성하였다. 이는 향후 데이터가 제한적인 의료 영상 분야에서 효율적인 모델 아키텍처를 자동으로 구축하는 데 중요한 기여를 할 것으로 보인다.
