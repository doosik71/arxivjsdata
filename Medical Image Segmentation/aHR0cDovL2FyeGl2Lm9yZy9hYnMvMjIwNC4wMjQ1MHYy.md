# Federated Cross Learning for Medical Image Segmentation

Xuanang Xu et al. (2023)

## 🧩 Problem to Solve

본 논문은 의료 영상 분할(Medical Image Segmentation) 분야에서 연합 학습(Federated Learning, FL)을 적용할 때 발생하는 성능 저하 문제를 해결하고자 한다. 연합 학습은 여러 병원이 보유한 독립적인 환자 데이터를 공유하지 않고 모델을 협력적으로 학습시킬 수 있는 이점이 있지만, 데이터가 독립 동일 분포(Independent and Identically Distributed, IID)를 따르지 않는 **non-iid** 상황에서 심각한 성능 저하가 발생한다.

특히 의료 영상 데이터는 획득한 클리닉의 장비나 프로토콜에 따라 데이터 분포의 차이가 크기 때문에 non-iid 특성이 매우 강하다. 기존의 연합 학습 방식은 로컬 모델들의 파라미터를 평균 내어 글로벌 모델을 업데이트하는데, 딥러닝의 최적화 목적 함수가 non-convex 특성을 가지므로 이러한 단순 평균화 작업이 파라미터 공간에서 최적이 아닌(sub-optimal) 지점으로 모델을 유도하여 결과적으로 성능을 떨어뜨리는 결과를 초래한다. 따라서 본 논문의 목표는 모델 집계(Model Aggregation) 과정 없이 non-iid 데이터 환경에서도 효과적으로 학습할 수 있는 새로운 연합 학습 전략을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 기존 연합 학습의 고질적인 문제인 **'모델 집계(Aggregation)' 단계를 완전히 제거**하는 것이다. 이를 위해 다음과 같은 두 가지 핵심 설계안을 제시한다.

1.  **FedCross (Federated Cross Learning):** 여러 클라이언트에서 모델을 동시에 학습시키고 이를 합치는 대신, 하나의 글로벌 모델을 여러 클라이언트가 순차적으로(round-robin 방식) 전달받아 학습시키는 전략이다. 이를 통해 집계 과정에서 발생하는 성능 저하 위험을 원천적으로 차단한다.
2.  **FedCrossEns (Federated Cross Ensemble Learning):** FedCross를 기반으로 하되, 여러 개의 모델을 서로 다른 무작위 경로로 학습시켜 앙상블(Ensemble) 하는 방식이다. 이는 중앙 집중식 학습(Centralized Learning) 수준의 정확도를 확보함과 동시에, 모델의 불확실성(Uncertainty)을 정량적으로 추정할 수 있게 한다.

## 📎 Related Works

기존의 연합 학습 연구들은 주로 $\text{FedAvg}$ 알고리즘을 기반으로 하며, non-iid 문제를 해결하기 위해 다음과 같은 접근 방식을 취해왔다.

-   **$\text{FedProx}$:** 로컬 최적화 목적 함수에 추가적인 제약 조건을 더해 로컬 업데이트의 크기를 조절함으로써 클라이언트 간의 그래디언트 차이를 완화하려 한다.
-   **$\text{FedBN}$:** 특성 공간(feature space)에서 non-iid 문제를 해결하기 위해 Batch Normalization 레이어를 글로벌 집계 없이 로컬에서만 업데이트하여 특성을 정렬한다.
-   **NAS 기반 접근:** 신경 구조 탐색(Neural Architecture Search)을 통해 각 클라이언트에 최적화된 서브 네트워크들의 앙상블 형태인 슈퍼 네트워크를 구축한다.

하지만 위 방법들은 여전히 어떤 형태로든 파라미터 평균화(averaging parameters) 과정을 포함하고 있으며, 이는 non-iid 데이터 환경에서 여전히 성능 저하의 위험을 내포하고 있다는 한계가 있다.

## 🛠️ Methodology

### 1. Theoretical Analysis of Non-IID Problem
논문은 $\text{FedAvg}$의 수식 분석을 통해 집계 과정의 문제를 증명한다. 글로벌 목적 함수 $L(\theta) = \sum_{k=1}^{K} w_k l(\theta, D_k)$를 최소화하려 할 때, 클라이언트 $C_k$의 로컬 업데이트는 다음과 같다.

$$\theta_{j+1}^k = \theta_j^k - \alpha_j \nabla l(\theta_j^k, D_k), \quad \text{for } j=1, 2, \dots, J$$

이후 서버에서 수행하는 모델 집계 식은 다음과 같다.

$$\theta' = \sum_{k=1}^{K} w_k \theta_{J+1}^k$$

이 식을 전개하면 $\theta'$는 글로벌 목적 함수의 하강 방향인 $\nabla L(\theta)$ 항과, 클라이언트 간의 $\theta_j^k$ 차이로 인해 발생하는 오차 항으로 나뉜다. non-iid 데이터의 경우 로컬 학습 횟수 $J$가 증가할수록 이 오차 항이 커지며, 결과적으로 집계된 모델 $\theta'$가 글로벌 손실 함수의 최소점이 아닌 지점에 위치하게 되어 성능이 저하된다.

### 2. FedCross (Federated Cross Learning)
FedCross는 모델 집계를 완전히 배제한다. 전체 파이프라인은 다음과 같다.

-   **순차적 학습:** 서버는 매 라운드마다 무작위로 하나의 활성 클라이언트 $C_t^k$를 선택하여 글로벌 모델 $\theta_t$를 전송한다.
-   **로컬 적응:** 선택된 클라이언트는 자신의 로컬 데이터 $D_k$를 사용하여 모델을 학습시킨다. 이때 $\text{FedAvg}$와의 형평성을 위해 로컬 학습 에포크 수를 $K$배(클라이언트 수만큼) 증가시킨다.
-   **모델 전달:** 학습이 완료된 모델 $\theta_{t+1}$은 다시 서버로 전송되며, 서버는 이를 다음 라운드의 선택된 클라이언트에게 전달한다.
-   이 과정은 마치 중앙 집중식 학습에서 데이터를 클라이언트별로 묶어 순차적으로 학습시키는 것과 유사한 효과를 낸다.

### 3. FedCrossEns (Federated Cross Ensemble Learning)
단일 모델의 한계를 극복하기 위해 앙상블 메커니즘을 결합한다.

-   **다중 모델 학습:** $K$개의 로컬 모델을 초기화하고, 각각 서로 다른 무작위 경로(route)를 통해 클라이언트들을 순회하며 FedCross 방식으로 학습시킨다.
-   **추론 및 앙상블:** 최종적으로 얻어진 $K$개의 모델이 각각 예측을 수행하고, 이 예측 결과들의 평균을 내어 최종 분할 마스크를 생성한다.
-   **불확실성 추정:** 앙상블된 모델들 간의 픽셀 단위 표준 편차(pixel-wise standard deviation)를 계산하여 불확실성 맵(Uncertainty Map)을 생성한다.

## 📊 Results

### 실험 설정
-   **데이터셋:** 전립선 MRI 분할을 위한 4개의 공개 데이터셋(MSD, NCI-ISBI, PROMISE12, PROSTATEx)을 사용하였다. 각 데이터셋을 서로 다른 4개의 클라이언트 노드로 간주하여 non-iid 환경을 모사하였다.
-   **모델 및 지표:** 3D U-Net을 사용하였으며, 평가지표로는 Dice Similarity Coefficient (DSC)와 Average Symmetric Surface Distance (ASD)를 사용하였다.
-   **비교 대상:** Localized training, Centralized training, $\text{FedAvg}$, $\text{FedProx}$, $\text{FedBN}$.

### 주요 결과
-   **정량적 성능:** $\text{FedCrossEns}$가 가장 높은 정확도를 기록하였으며, 이는 중앙 집중식 학습(Centralized) 결과에 근접하는 수준이다. $\text{FedCross}$ 단독 모델 또한 다른 FL 방법론들보다 우수한 성능을 보였으며, 특히 데이터 분포 차이가 심한 PROMISE12 데이터셋에서 강점이 나타났다.
-   **로컬 에포크($E$)의 영향:** $\text{FedAvg}$, $\text{FedProx}$, $\text{FedBN}$과 같은 집계 기반 방식들은 $E$가 증가할수록(즉, 로컬 학습을 많이 할수록) 성능이 급격히 저하되는 경향을 보였다. 반면, $\text{FedCross}$는 집계 과정이 없으므로 $E$의 증가에 훨씬 안정적인 성능을 유지하였다. 이는 통신 횟수를 줄이면서도 안정적인 학습이 가능함을 시사한다.
-   **정성적 결과:** $\text{FedCrossEns}$를 통해 생성된 불확실성 맵은 예측 결과의 신뢰도를 시각화하여, 의료진이 수동으로 수정하거나 승인하는 과정에서 중요한 가이드를 제공할 수 있음을 보여주었다.

## 🧠 Insights & Discussion

본 논문은 연합 학습에서 발생하는 non-iid 문제를 해결하기 위해 '집계'라는 고정관념을 버리고 '순차적 학습'이라는 단순하지만 강력한 대안을 제시하였다. 

**강점:**
-   이론적 분석을 통해 $\text{FedAvg}$의 한계를 명확히 짚어내고, 이를 해결하기 위한 구조적 설계를 제안하였다.
-   앙상블 기법을 통해 성능 향상뿐만 아니라 의료 분야에서 매우 중요한 '불확실성 추정' 기능을 추가하여 임상적 활용 가치를 높였다.
-   로컬 학습 횟수가 늘어나도 성능이 유지된다는 점은 실제 통신 비용이 비싼 분산 환경에서 큰 이점이 된다.

**한계 및 논의:**
-   **Catastrophic Forgetting (파괴적 망각):** 모델이 순차적으로 클라이언트를 이동하며 학습하므로, 최근에 학습한 클라이언트의 데이터에 편향되어 이전 클라이언트의 지식을 잊어버릴 위험이 있다. 하지만 저자는 의료 영상 연합 학습의 경우 클라이언트 수($K$)가 일반적으로 적기 때문에 이 영향이 제한적일 것이라고 주장한다.
-   **병렬성 부족:** 모든 모델이 동시에 학습되는 $\text{FedAvg}$와 달리 순차적으로 학습하므로, 전체 학습 시간이 증가할 수 있는 병렬성 저하 문제가 존재한다.

## 📌 TL;DR

이 논문은 의료 영상의 non-iid 데이터 문제로 인한 연합 학습의 성능 저하를 해결하기 위해, 모델 집계 과정을 완전히 제거하고 모델을 클라이언트 간에 순차적으로 전달하며 학습시키는 **FedCross** 전략을 제안한다. 더 나아가 여러 모델을 앙상블하는 **FedCrossEns**를 통해 중앙 집중식 학습에 가까운 성능을 달성하고 모델의 불확실성까지 추정할 수 있게 하였다. 이 연구는 특히 클라이언트 간 데이터 편차가 심한 의료 영상 분야에서 매우 효율적이고 안정적인 협력 학습 프레임워크를 제공한다.
