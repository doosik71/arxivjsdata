# QuickDraw: Fast Visualization, Analysis and Active Learning for Medical Image Segmentation

Daniel Syomichev, Padmini Gopinath et al. (2025)

## 🧩 Problem to Solve

의료 영상(CT, MRI, X-ray) 분석은 질병의 진단과 치료에 있어 핵심적인 역할을 수행한다. 그러나 의료 영상에서 이상 징후를 탐지하고 식별하는 과정은 전문가의 정밀한 분석을 필요로 하며, 이는 매우 많은 시간이 소요되는 작업이다. 또한 분석가 개인의 주관에 따라 결과가 달라지는 interobserver variability 문제가 발생할 가능성이 높다.

최근 이를 자동화하기 위한 머신러닝 기반의 Segmentation 모델들이 제안되었으나, 실제 임상 현장에서는 다음과 같은 한계로 인해 도입이 더딘 상황이다.

1. **워크플로우의 단절**: 최신 모델들이 기존의 의료 영상 뷰어(Medical Image Viewer)와 쉽게 인터페이스되지 않아, 시각화 도구, 모델 추론 도구, 분석 도구가 각각 서로 다른 플랫폼에서 분절되어 운영된다.
2. **데이터 구축의 고비용**: 고품질의 학습 데이터를 구축하기 위해 3D 스캔 하나당 최대 4시간의 수동 세그멘테이션 시간이 소요된다.
3. **기술적 진입장벽**: 기존의 모델 추론 및 분석 과정은 기술적 전문 지식이 부족하거나 컴퓨팅 자원이 제한적인 사용자가 접근하기에 어렵다.

본 논문의 목표는 모델 추론, 평가, 그리고 Active Learning을 위한 재어노테이션(re-annotation) 과정을 하나의 통합된 인터페이스에서 제공하는 오픈소스 프레임워크인 QuickDraw를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 의료 영상의 시각화, 모델 기반 세그멘테이션, 결과 평가 및 수정 과정을 하나의 통합된 워크플로우로 결합한 QuickDraw 프레임워크를 제안한 것이다.

핵심 설계 아이디어는 전문가가 단순히 모델의 결과를 수용하는 것에 그치지 않고, **'추론 $\rightarrow$ 시각적 분석 $\rightarrow$ 수동 수정 $\rightarrow$ 데이터 내보내기 $\rightarrow$ 모델 재학습'**으로 이어지는 Active Learning 루프를 효율적으로 수행할 수 있도록 지원하는 것이다. 이를 통해 모델의 예측 오류를 빠르게 수정하고, 수정된 데이터를 다시 학습에 활용함으로써 모델의 성능을 반복적으로 개선할 수 있는 환경을 제공한다.

## 📎 Related Works

### 기존 의료 영상 뷰어 및 ML 프레임워크
3D Slicer, OHIF Viewer, ITK-SNAP과 같은 도구들은 DICOM 파일의 시각화와 수동 어노테이션을 지원한다. 또한 MONAI, MedSAM, nnU-Net, U-Net 등의 프레임워크는 모델 학습 과정을 단순화하였다. 그러나 이러한 도구들은 개별적인 기능에 특화되어 있으며, 모델의 예측 결과를 확인하고 이를 다시 수정하여 Active Learning 파이프라인으로 연결하는 전체 과정을 단일 인터페이스에서 제공하지 않는다. 대부분의 기존 연구들은 저품질 예측을 식별하거나 마스크를 정교화하기 위해 외부 도구에 의존하는 한계가 있다.

### 의료 영상 분석을 위한 Active Learning
Active Learning은 가장 정보량이 많은 데이터 샘플을 선택적으로 재어노테이션하여 학습함으로써, 적은 양의 데이터로도 모델 성능을 높이는 방법이다. PAAL, Selective Uncertainty-Based Active Learning 등 불확실성(Uncertainty)이나 다양성(Diversity) 기반의 샘플 선택 기법들이 제안되었다. 하지만 이러한 알고리즘적 발전에도 불구하고, 실제 예측 결과를 수정하고 처리하는 과정은 여전히 분절된 도구들에 의존하고 있어 효율성이 떨어진다.

## 🛠️ Methodology

### 전체 시스템 구조 (Software Architecture)
QuickDraw는 웹 기반의 인터페이스와 클라우드 기반의 추론 서버를 결합한 구조를 가진다.

1. **User Interface**: OHIF Viewer를 기반으로 하며, 사용자가 DICOM 영상을 시각화하고 세그멘테이션 마스크를 편집할 수 있는 웹 환경을 제공한다.
2. **Image Storage**: Orthanc DICOM 서버를 사용하여 의료 영상 레코드를 저장하고 관리하며, REST API를 통해 데이터를 주고받는다.
3. **Model Management & Inference**: Google Cloud Services (GCS)를 활용한다.
    - 모델은 Docker 컨테이너 형태로 GCS Artifact Registry에 저장된다.
    - 추론 요청이 발생하면 GCS Compute Engine이 가상 머신(VM)을 동적으로 생성하여 모델을 실행한다.
    - 추론이 완료되면 VM은 자동으로 중지되어 하드웨어 오버헤드와 비용을 최소화한다.
4. **Data Flow**: 사용자가 모델을 선택하면 $\text{OHIF Viewer} \rightarrow \text{Python 중개 서버} \rightarrow \text{Orthanc Server} \rightarrow \text{GCS VM(추론)} \rightarrow \text{Orthanc Server} \rightarrow \text{OHIF Viewer}$ 순으로 데이터가 흐른다. 환자 개인정보 보호를 위해 추론 후 서버 내의 의료 영상은 삭제된다.

### 시각화 및 추론 절차
사용자는 웹 뷰어에서 DICOM 영상을 로드한 후 드롭다운 메뉴에서 원하는 모델을 선택하여 추론을 요청한다. 모델이 생성한 3D 세그멘테이션 마스크는 영상 위에 레이어 형태로 겹쳐지며, 사용자는 OHIF Viewer의 내장 편집 도구를 사용하여 예측된 마스크를 정교하게 수정할 수 있다.

### 영상 분석 도구 (Image Analysis)
QuickDraw는 예측 결과와 Ground Truth(GT) 간의 차이를 정량적, 정성적으로 분석하는 기능을 제공한다.

- **DICE Score**: 예측 마스크와 GT 마스크 간의 유사도를 측정하는 지표이다.
- **Discrepancy Map**: XOR 비트 마스크 연산을 통해 예측 결과와 GT가 일치하지 않는 영역만을 분리하여 시각화한다. 이를 통해 사용자는 모델이 어느 부분에서 오작동했는지 직관적으로 파악할 수 있다.

## 📊 Results

### 실험 설정
- **참가자**: 의료 영상 경험이 있는 사용자 4명과 경험이 없는 사용자 4명, 총 8명이 참여하였다.
- **평가 방법**: 가이드 하에 수행된 타이밍 작업(Timed tasks)과 5점 척도(Likert scale)의 사용자 설문 조사를 실시하였다.
- **측정 지표**: DICE Score, 세그멘테이션 소요 시간, 전체 작업 완료 시간.

### 주요 결과
1. **사용성 및 만족도**: 설문 결과, 응답자들은 전반적인 성능과 응답성(4.5/5.0), DICE Score 계산 기능의 유용성(4.5/5.0)에 매우 높은 만족도를 보였다.
2. **시간 효율성**:
    - 수동으로 CT 스캔을 세그멘테이션할 때 소요되던 시간을 **4시간에서 6분으로 획기적으로 단축**시켰다.
    - 기존의 ML 보조 세그멘테이션 방식과 비교하여 작업 시간을 약 10% 더 단축하였다.
3. **학습 곡선 (Learning Curve)**: 경험이 없는 사용자들의 경우, 두 번째 시도에서 작업 완료 시간이 크게 감소하는 경향을 보였다. 이는 QuickDraw의 인터페이스가 직관적이며 학습 비용이 낮음을 시사한다.
4. **정량적 성능**: 모든 참가자가 초기 모델 예측(DICE Score = 0.78)보다 수정 후 더 높은 DICE Score를 달성하였으며, 특히 경험자들은 높은 정확도를 유지하면서도 매우 빠른 작업 속도를 보였다.

## 🧠 Insights & Discussion

### 강점
QuickDraw는 분절되어 있던 의료 영상 분석 워크플로우를 하나의 플랫폼으로 통합함으로써, 전문가의 수동 수정 작업을 효율화하고 이를 Active Learning으로 연결하는 가교 역할을 수행한다. 특히 클라우드 기반의 동적 VM 할당을 통해 비용 효율적인 모델 추론 환경을 구축한 점이 돋보인다.

### 한계 및 논의사항
사용자 조사 결과, 세그멘테이션 마스크를 수정하는 과정에서 브러시 크기 조절이나 sphere 기능 사용에 어려움을 겪는 경우가 있었다. 이는 툴 자체의 기능적 한계라기보다 OHIF Viewer의 기본 편집 도구에 대한 숙련도 문제일 가능성이 크며, 향후 인터페이스 개선을 통해 해결해야 할 과제이다.

### 비판적 해석
본 논문은 도구의 유용성과 시간 단축 효과를 성공적으로 입증하였으나, 실제 Active Learning 루프가 모델 성능을 얼마나 구체적으로 향상시켰는지에 대한 정량적 모델 성능 지표(예: 수정된 데이터를 학습시킨 후의 mIoU 변화 등)는 구체적으로 제시되지 않았다. 도구의 '사용성'에 초점이 맞춰져 있어, 실제 모델의 성능 향상 폭에 대한 추가적인 실험 데이터가 보완된다면 더 강력한 설득력을 가질 수 있을 것이다.

## 📌 TL;DR

QuickDraw는 의료 영상의 시각화, 모델 추론, 결과 수정 및 평가를 단일 워크플로우로 통합한 오픈소스 프레임워크이다. 수동 세그멘테이션 시간을 4시간에서 6분으로 단축시켰으며, 전문가가 모델의 오류를 쉽게 수정하고 이를 다시 학습에 활용할 수 있는 Active Learning 환경을 제공함으로써 의료 영상 분석의 효율성을 극대화한다. 이 연구는 최신 딥러닝 모델의 임상 적용 장벽을 낮추고, 데이터 구축 비용을 절감하는 데 중요한 기여를 할 것으로 기대된다.
