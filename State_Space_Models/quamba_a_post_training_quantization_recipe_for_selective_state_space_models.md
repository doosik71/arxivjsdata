# Quamba: A Post-Training Quantization Recipe for Selective State Space Models

Hung-Yueh Chiang, Chi-Chih Chang, Natalia Frumkin, Kai-Chiang Wu, and Diana Marculescu (2024)

## 🧩 Problem to Solve

본 논문은 선택적 상태 공간 모델(Selective State Space Models, SSMs)을 자원 제한적인 엣지 디바이스나 요청 집약적인 클라우드 환경에 효율적으로 배포하기 위한 양자화 문제를 해결하고자 한다. SSM은 Transformer 대비 긴 컨텍스트 길이를 처리할 수 있는 상수 메모리 복잡도와 선형 계산 복잡도를 가져 매우 유망한 아키텍처로 주목받고 있다.

그러나 SSM을 실제 하드웨어에 배포하는 데 있어 메모리와 지연 시간(latency) 제약이 큰 걸림돌이 된다. 기존의 Transformer 대상 사후 훈련 양자화(Post-Training Quantization, PTQ) 기법들은 SSM의 특수한 구조적 특성을 반영하지 못한다. 특히 SSM은 선택적 스캔(Selective Scan) 메커니즘 내의 선형 재귀(Linear Recurrence) 과정에서 입력 텐서에 대한 양자화 오차에 매우 민감하게 반응하며, 출력 활성화 함수에서 Transformer의 셀프 어텐션 모듈에서는 나타나지 않는 거대한 아웃라이어(Outliers)가 발생한다는 특징이 있다. 따라서 본 연구의 목표는 이러한 SSM 특유의 민감도와 아웃라이어 문제를 해결하여, 정확도 손실을 최소화하면서 하드웨어 가속을 극대화할 수 있는 8비트 정수 양자화 레시피를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 SSM의 입력과 출력 활성화 텐서가 서로 다른 양자화 특성을 가진다는 점에 착안하여, 각각에 최적화된 서로 다른 전략을 적용한 **Quamba** 방법론을 제안한 것이다.

1. **입력 활성화 텐서($x$)의 정밀도 개선**: 입력 텐서는 수치적으로는 작지만, 선형 재귀 시스템의 특성상 작은 양자화 오차가 출력에서 크게 증폭되는 민감한 성질을 가진다. 이를 위해 백분위수 기반 클리핑(Percentile-based clipping)을 적용하여 소수의 극단적인 아웃라이어가 양자화 스텝 사이즈를 키우는 것을 방지하고, 대다수 값의 양자화 정밀도를 높였다.
2. **출력 활성화 텐서($y$)의 아웃라이어 제거**: 출력 텐서에서 발생하는 거대한 아웃라이어 문제를 해결하기 위해 월시-하다마르 변환(Walsh-Hadamard Transform, WHT)을 도입하였다. 이를 통해 활성화 텐서를 아웃라이어가 없는 공간으로 투영하여 8비트 양자화 시 발생하는 정밀도 저하를 막았다.
3. **하드웨어 친화적 통합 설계**: 하다마르 변환의 역변환 행렬을 출력 선형 투영 층의 가중치와 융합(Fusion)하여 추가적인 계산 오버헤드 없이 연산 불변성(Compute-invariance)을 달성하였다.
4. **이론적 분석**: LTI(Linear Time-Invariant) 시스템 모델을 통해 SSM의 양자화 오차가 입력 시퀀스 길이에 대해 유계(Bounded)임을 수학적으로 증명하였다.

## 📎 Related Works

기존의 모델 양자화 연구는 크게 양자화 인식 훈련(QAT)과 사후 훈련 양자화(PTQ)로 나뉜다. 본 논문은 추가 훈련이 필요 없는 PTQ 방식에 집중한다.

특히 Transformer를 위한 최신 PTQ 기법들인 SmoothQuant나 QuaRot 등은 활성화 텐서의 아웃라이어를 처리하기 위해 혼합 정밀도나 가중치 재조정, 또는 하다마르 회전을 사용한다. 그러나 이러한 기법들을 SSM에 그대로 적용했을 때 성능이 크게 저하되는 현상이 관찰되었다. 이는 Transformer의 셀프 어텐션 층이 양자화 오차에 상대적으로 견고한 반면, SSM의 선택적 스캔 메커니즘은 입력값의 작은 변화가 재귀적으로 누적되어 출력에 막대한 영향을 미치는 인과적 관계(Causal relationship)를 가지기 때문이다. 결과적으로 기존 방식들은 SSM의 민감한 텐서를 적절히 처리하지 못하거나, SSM 구조에 맞지 않는 추가적인 계산 오버헤드를 유발한다는 한계가 있다.

## 🛠️ Methodology

### 전체 파이프라인 및 시스템 구조

Quamba는 가중치와 활성화 텐서를 모두 8비트 부호 있는 정수(INT8)로 양자화하는 대칭 균등 양자화(Symmetric Uniform Quantization)를 기본으로 한다. 전체 시스템은 입력 클리핑 $\rightarrow$ 선택적 SSM 연산 $\rightarrow$ 하다마르 변환 기반 출력 양자화 $\rightarrow$ 융합된 출력 투영 순으로 구성된다.

### 주요 구성 요소 및 방법론

#### 1. SSM 입력 양자화 (Percentile-based Clipping)

입력 텐서 $x$는 수치적 크기는 작지만 양자화 오차에 매우 민감하다. 일반적인 Max-Abs 스케일링을 사용하면 0.1% 미만의 아주 적은 아웃라이어만으로도 스케일 팩터 $s_x$가 커져 대부분의 값들이 0으로 뭉개지는 현상이 발생한다. 이를 해결하기 위해 $p$ 백분위수(예: $p=99.999$)를 사용하여 최대값을 제한한다.

$$s_x = \frac{\max_p |x|}{2^{b-1}-1}$$

이 방식은 추론 시 추가 오버헤드가 없으며, 입력의 양자화 정밀도를 획기적으로 높여 출력에서의 오차 증폭을 억제한다.

#### 2. SSM 출력 양자화 (Hadamard Transform)

출력 텐서 $y$에서 발생하는 거대 아웃라이어를 제거하기 위해 WHT를 적용한다. 출력 $y$를 하다마르 행렬 $H_n$을 통해 변환하여 $\tilde{y} = H_n y$를 얻고, 이 $\tilde{y}$ 공간에서 양자화를 수행한다. 이때, 역변환 과정의 오버헤드를 없애기 위해 출력 가중치 행렬 $W_{out}$에 하다마르 행렬을 미리 곱해 융합한다.

$$W_{rot} = H_n W_{out}$$
$$\text{Mamba Output} = W_{out}^\top y = \frac{1}{n} (W_{rot})^\top \tilde{y}$$

이를 통해 계산 결과는 동일하게 유지하면서, 양자화는 아웃라이어가 제거된 $\tilde{y}$ 공간에서 수행함으로써 정밀도를 확보한다.

#### 3. 기타 연산 최적화

- **Projection Layers**: CUTLASS 라이브러리를 사용하여 8비트 정수 연산 가속(Tensor Cores)을 활용한다.
- **Fused Causal Convolution**: 입력과 가중치를 8비트로 양자화하고, SiLU 활성화 함수를 컨볼루션 연산 내에 융합하여 메모리 접근을 최소화한다.
- **Fused RMSNorm**: 잔차 연결(Residual addition)과 정적 양자화를 RMSNorm 연산과 통합하여 반정밀도(FP16) 연산 후 즉시 8비트로 양자화하여 다음 블록으로 전달한다.

### 수학적 오차 분석

본 논문은 이산 선형 시불변(LTI) 시스템 $h(t) = A(t)h(t-1) + Bx(t)$에서 입력 양자화 오차 $\delta x(t)$가 전체 시스템에 미치는 영향을 분석하였다. 상태 행렬 $A$의 노름이 $\|A(t)\|_2 \le a \cdot e^{t-T} (0 < a < 1)$로 유계라고 가정할 때, 시간 $t$에서의 양자화 오차 $\Delta(t)$의 상한은 다음과 같다.

$$\|\Delta(t)\|_2 \le \frac{\epsilon b}{1 - a e^{t-T}}$$

결과적으로 전체 시퀀스 길이 $T$에 대한 전역 양자화 오차는 $\frac{\epsilon b}{1-a}$로 유계되며, 이는 SSM의 양자화 오차가 시퀀스 길이에 따라 무한히 발산하지 않고 제어될 수 있음을 이론적으로 뒷받침한다.

## 📊 Results

### 실험 설정

- **모델**: Mamba (130M, 370M, 1.4B, 2.8B), Jamba (52B 하이브리드 모델)
- **데이터셋**: LAMBADA, HellaSwag, PIQA, ARC, WinoGrande (Zero-shot 평가)
- **하드웨어**: Nvidia A5000 (클라우드), Nvidia Orin Nano 8G (엣지)
- **지표**: 평균 정확도, Perplexity, 생성 지연 시간(Latency)

### 주요 결과

1. **정확도 유지**: Mamba 2.8B 모델을 8비트로 양자화했을 때, FP16 대비 평균 정확도 하락이 단 0.9%에 불과했다. 또한 52B 파라미터의 대규모 모델인 Jamba에서도 약 1%의 정확도 하락만을 보이며 확장 가능성을 입증했다.
2. **지연 시간 단축**: Nvidia Orin Nano 8G 환경에서 생성 지연 시간(TPOT)을 1.72배 단축시켰으며, A5000 GPU에서는 입력 길이에 따라 1.2배에서 1.27배의 성능 향상을 보였다.
3. **메모리 효율**: 2.8B 모델의 크기를 약 5.29GB(FP16)에서 2.76GB(INT8)로 거의 절반 가까이 줄였다.
4. **Pareto 최적성**: 정확도-지연 시간 트레이드오프 분석 결과, Quamba는 FP16 Mamba 및 기존 Transformer 기반 양자화 기법(Mamba-Had 등)보다 우월한 파레토 전선(Pareto front)에 위치함을 확인했다.

## 🧠 Insights & Discussion

### 강점 및 해석

Quamba는 SSM의 수학적 특성(선형 재귀의 민감도)과 데이터 특성(출력의 아웃라이어)을 정확히 짚어내어 맞춤형 해결책을 제시했다. 특히 단순히 알고리즘만 제안한 것이 아니라, 하다마르 변환을 가중치에 융합하거나 RMSNorm을 통합하는 등 하드웨어 가속을 고려한 구현 최적화를 통해 실질적인 속도 향상을 이끌어낸 점이 돋적으로 평가된다. 또한 이론적 오차 유계 증명을 통해 제안 방법의 안정성을 뒷받침했다.

### 한계 및 논의사항

1. **정밀도 하락**: 매우 낮은 비트수(예: 4비트)로 내려갈 경우, Quamba를 포함한 모든 방법론에서 정확도가 급격히 하락(약 63% $\rightarrow$ 30%)하는 현상이 관찰되었다. 이는 SSM이 Transformer보다 저비트 양자화에 훨씬 취약함을 시사하며, 4비트 이하의 양자화를 위해서는 단순한 PTQ 이상의 새로운 접근법이 필요함을 의미한다.
2. **대칭 양자화의 한계**: 분석 결과 입력 텐서의 분포가 비대칭적임을 확인했으며, 비대칭 양자화가 성능상 이점이 있음을 발견했다. 그러나 현재의 하드웨어 가속 라이브러리(CUTLASS 등) 지원 문제로 대칭 양자화를 채택했다. 향후 하드웨어 지원이 확대된다면 비대칭 양자화를 통해 추가적인 성능 향상이 가능할 것으로 보인다.

## 📌 TL;DR

본 논문은 SSM의 특수한 아웃라이어 패턴과 재귀적 민감도를 해결하는 8비트 PTQ 방법론인 **Quamba**를 제안한다. 입력 텐서에는 **백분위수 기반 클리핑**을, 출력 텐서에는 **하다마르 변환**을 적용하여 정확도 손실을 최소화(Mamba 2.8B 기준 0.9% 하락)하면서도 엣지 디바이스에서 최대 1.72배의 속도 향상을 달성했다. 이 연구는 대규모 SSM 기반 모델을 자원 제한적인 환경에 효율적으로 배포할 수 있는 실무적인 레시피를 제공하며, 향후 온디바이스 AI 서비스 구현에 중요한 역할을 할 것으로 기대된다.
