# Limits of Generalization in RLVR: Two Case Studies in Mathematical Reasoning

Md Tanvirul Alam, Nidhi Rastogi (2025)

## 🧩 Problem to Solve

본 논문은 대규모 언어 모델(LLM)의 수학적 추론 능력을 향상시키기 위해 널리 사용되는 Reinforcement Learning with Verifiable Rewards (RLVR)의 실제 효과와 일반화 한계를 분석한다. RLVR은 정답이나 유닛 테스트와 같이 자동으로 검증 가능한 신호를 보상으로 사용하여 모델을 미세 조정하는 방식이다. 최근 이러한 방식이 수학 및 프로그래밍 벤치마크에서 높은 성능 향상을 보였으나, 이것이 모델이 실제로 새로운 추론 전략을 습득하여 수학적 역량이 강화된 것인지, 아니면 단순히 표면적인 패턴 매칭이나 휴리스틱(heuristics)을 통해 정답에 도달하는 '지름길(shortcut)'을 학습한 것인지에 대해서는 불분명하다.

따라서 본 연구의 목표는 정답뿐만 아니라 추론 과정의 충실성(fidelity)을 완전히 검증할 수 있는 두 가지 조합 최적화 문제(Combinatorial Problems)를 통해 RLVR이 유도하는 성능 향상의 본질을 규명하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 정답이 유일하게 결정되는(unique optima) 데이터셋을 구축하여, 정답의 정확도뿐만 아니라 추론 단계에서 생성되는 시퀀스의 정확성을 정밀하게 측정하는 것이다. 이를 통해 RLVR이 진정한 추론 능력을 배양하는지, 아니면 표면적인 휴리스틱을 강화하는지를 분석한다.

주요 기여 사항은 다음과 같다.

1. **검증 가능한 작업 설계**: Activity Scheduling과 Longest Increasing Subsequence (LIS)라는 두 가지 과제를 설정하고, 각 인스턴스마다 유일한 최적해를 갖도록 데이터셋을 생성하여 추론 과정의 정밀한 측정이 가능하게 하였다.
2. **보상 함수 설계 및 비교**: 정답만 고려하는 보상부터 시퀀스의 부분 일치 및 정렬 상태를 고려하는 보상까지 다양한 Reward Design을 실험하여, 보상의 형태가 모델의 추론 행동에 미치는 영향을 분석하였다.
3. **휴리스틱 분석**: Random Forest 회귀 분석을 통해 RLVR로 학습된 모델이 입력 데이터의 통계적 특징(휴리스틱)에 의존하여 답을 예측한다는 것을 정량적으로 증명하였다.

## 📎 Related Works

최근 DeepSeek-R1이나 OpenAI o1과 같은 모델들은 RLVR를 통해 수학적 추론 능력을 비약적으로 상승시켰다. 기존 연구들은 RLVR가 인간의 주석 없이도 확장 가능한 학습을 가능하게 하며, 도전적인 추론 작업에서 일관된 이득을 준다고 보고하였다.

그러나 본 논문은 기존의 벤치마크들이 추론 과정의 정답 여부를 완전히 검증하기 어렵다는 한계를 지적한다. 특히 $\text{Pass@K}$와 같은 지표는 모델이 운 좋게 정답을 맞춘 경우를 과대평가할 수 있으며, $\text{CoT-Pass@K}$와 같은 엄격한 지표는 확장성이 낮다는 문제가 있다. 또한, 일부 연구에서는 RLVR가 새로운 추론 전략을 유도하기보다 기존에 모델이 가지고 있던 능력을 안정화하거나 재가중치 부여하는 수준에 그친다고 주장한다. 본 연구는 이러한 맥락에서 RLVR의 일반화 한계를 구체적인 사례 연구를 통해 실증적으로 분석한다.

## 🛠️ Methodology

### 1. 분석 대상 과제 (Tasks)

- **Activity Scheduling**: 겹치지 않는 활동의 최대 집합을 찾는 문제이다. 가장 빨리 끝나는 활동을 우선적으로 선택하는 Greedy 알고리즘을 통해 유일한 최적해를 생성하도록 설계하였다.
- **Longest Increasing Subsequence (LIS)**: 주어진 수열에서 가장 긴 엄격하게 증가하는 부분 수열을 찾는 문제이다. 동적 계획법(Dynamic Programming)을 사용하여 유일한 최적해를 갖는 인스턴스를 생성하였다.

### 2. 보상 함수 (Reward Functions)

모델의 출력을 파싱하여 최종 정답 $\hat{a}(y)$와 선택된 ID 시퀀스 $\hat{s}(y)$를 추출하며, 다음과 같은 보상을 정의한다.

- **Answer-only ($r_{ans}$)**: 최종 숫자 정답의 정오표만 확인한다.
$$r_{ans}(y) = \mathbb{I}[\hat{a}(y) = a^\star]$$
- **Answer + Format ($r_{ans+fmt}$)**: LIS 작업에서 추론 과정의 붕괴를 막기 위해 서식(format) 보너스를 추가한다. $\lambda = 0.1$을 가중치로 사용한다.
$$r_{ans+fmt}(y) = (1 - \lambda)r_{ans}(y) + \lambda \text{fmt}(y)$$
- **Exact-IDs ($r_{ids,exa}$)**: 예측된 ID 시퀀스가 최적해와 완전히 일치해야 보상을 준다.
$$r_{ids,exa}(y) = \mathbb{I}[\hat{s}(y) = s^\star]$$
- **Prefix-IDs ($r_{ids,pre}$)**: 정답 시퀀스와의 최대 공통 접두사(longest common prefix) 길이에 비례하여 부분 보상을 준다. 길이 불일치 시 페널티 $\gamma = 0.1$을 적용한다.
$$r_{ids,pre}(y) = \max\left(0, \frac{m(y)}{L} - \gamma \mathbb{I}[\hat{s}(y) = \emptyset \lor |\hat{s}(y)| \neq L]\right)$$
- **Sorting-Match ($r_{sort}$)**: Activity Scheduling에서 첫 단계인 '종료 시간순 정렬'이 제대로 이루어졌는지 확인하는 보조 보상이다.
$$r_{sort}(y) = \mathbb{I}[\hat{s}_{sort}(y) = s^\star_{sort}]$$

### 3. 학습 및 평가 절차

- **모델 및 프레임워크**: Qwen2.5-7B-Instruct 모델을 기반으로 GRPO (Group Relative Policy Optimization) 알고리즘과 `verl` 프레임워크를 사용하여 학습하였다.
- **하이퍼파라미터**: 학습률 $10^{-6}$, 20 에포크(120 업데이트), KL 페널티 $\beta_{KL} = 0$을 적용하였다.
- **평가 지표**: $\text{Pass@k}$ (k개 샘플 중 하나라도 정답일 확률)와 $\text{Self-consistency (SC)}$ (다수결 예측의 정답률)를 사용하여 정답 정확도($\text{Acc}_{ans}$)와 시퀀스 정확도($\text{Acc}_{ids}$)를 측정하였다.

## 📊 Results

### 1. 정답 기반 보상 ($r_{ans}$)의 효과

- **Activity Scheduling**: RLVR 학습 후 $\text{SC}_{ans}$가 $0.24$에서 $0.68$로 크게 상승하였으며, 시퀀스 일치도($\text{Pass@k}_{ids}$) 또한 Base 모델($0.14$) 대비 $0.64$로 향상되었다.
- **LIS**: 정답 보상만으로 학습할 경우, 모델이 중간 추론 과정(Chain-of-Thought)을 생략하고 짧은 정답만 출력하는 '추론 붕괴(reasoning collapse)' 현상이 발생하였다. 서식 보상($r_{ans+fmt}$)을 추가하면 길이는 유지되지만, 시퀀스 일치도($\text{Acc}_{ids}$)는 Base 모델보다 오히려 낮게 나타났다.

### 2. 시퀀스 기반 보상의 효과

- $r_{ids,exa}$와 $r_{ids,pre}$ 모두 두 과제에서 $\text{Acc}_{ids}$를 대폭 향상시켰으며, 부수적으로 $\text{Acc}_{ans}$ 또한 소폭 상승시켰다. 이는 중간 단계의 보상이 최종 정답 도달 능력을 돕는다는 것을 시사한다.

### 3. 정렬 성능 및 휴리스틱 분석

- **Activity Scheduling의 정렬 분석**: 모델들이 응답 서두에 정렬된 리스트를 생성하는 경향이 있었으나, 실제 정밀 분석 결과 정렬 정확도는 약 $2\%$에 불과했다. 즉, 겉으로는 정렬을 하는 것처럼 보이지만 실제로는 정렬 결과가 최종 스케줄링 결정에 영향을 주지 않는 '표면적 출력'임이 밝혀졌다.
- **LIS의 휴리스틱 분석**: Random Forest 회귀 모델을 통해 모델의 예측값과 입력 특징 간의 관계를 분석한 결과, RLVR 학습 모델들은 입력 값의 분포, 증가율, 단순 그리디(greedy) 근사치 등의 휴리스틱 특징에 매우 높은 $R^2$ 값을 보였다. 이는 모델이 논리적 추론이 아닌 데이터의 통계적 특징을 이용해 답을 유추하고 있음을 의미한다.

## 🧠 Insights & Discussion

본 연구는 RLVR가 벤치마크 수치를 높이는 데는 효과적이지만, 그것이 반드시 모델의 '추론 능력' 향상을 의미하지는 않는다는 점을 시사한다.

**강점 및 발견**:

- **표면적 일반화**: RLVR는 정답률을 높이는 방향으로 정책을 최적화하지만, 이는 진정한 알고리즘 습득이 아니라 모델이 이미 가지고 있던 편향을 강화하거나 표면적인 휴리스틱을 이용하는 방식으로 이루어질 가능성이 높다.
- **추론과 출력의 괴리**: Activity Scheduling 사례에서 보듯, 모델이 "정렬을 수행한다"고 말하면서 실제로 정렬되지 않은 리스트를 사용해 답을 내는 현상은 LLM의 '충실하지 않은 설명(unfaithful explanations)' 문제를 극명하게 보여준다.

**한계 및 비판적 해석**:

- 본 실험은 Qwen2.5-7B와 Llama-3.1-8B라는 특정 규모의 모델에서 수행되었으므로, 모델 크기가 훨씬 커졌을 때도 동일한 휴리스틱 의존성이 나타날지는 미지수이다.
- 다만, 정답 기반 보상이 추론 과정을 붕괴시키거나 가짜 추론(pseudo-reasoning)을 유도한다는 점은 RLVR의 보상 설계에 있어 매우 신중한 접근이 필요함을 시사한다.

## 📌 TL;DR

본 논문은 RLVR가 수학적 추론 작업에서 정답 정확도를 높이지만, 실제로는 진정한 추론 전략을 학습하기보다 표면적인 휴리스틱과 데이터의 통계적 특성을 이용하는 '지름길'을 학습한다는 것을 입증하였다. 특히, 모델이 겉으로는 정교한 추론 과정(예: 정렬)을 거치는 것처럼 출력하더라도 실제 내부 결정 과정과는 일치하지 않는 경우가 많음을 보였다. 이 연구는 향후 LLM의 추론 능력을 평가할 때 단순 정답률보다는 추론 과정의 충실성을 검증할 수 있는 정밀한 벤치마크 설계가 필수적임을 강조한다.
