# RLPR: Extrapolating RLVR to General Domains without Verifiers

Tianyu Yu et al. (2025)

## 🧩 Problem to Solve

본 논문은 검증 가능한 보상을 이용한 강화학습(Reinforcement Learning with Verifiable Rewards, RLVR)을 일반 도메인으로 확장하고자 한다. RLVR은 수학이나 코드 생성과 같이 정답 여부를 명확히 판별할 수 있는 영역에서는 LLM의 추론 능력을 비약적으로 향상시켰으나, 일반적인 도메인으로의 확장은 다음과 같은 문제로 인해 제한적이었다.

첫째, 기존의 RLVR은 정답을 판별하는 'Verifier'에 지나치게 의존한다. 수학의 경우 정교한 규칙 기반(Rule-based) Verifier를 사용할 수 있지만, 자연어 기반의 일반 도메인 답변은 그 다양성과 복잡성으로 인해 이러한 규칙을 설계하는 것이 사실상 불가능하다.

둘째, 모델 기반 Verifier(Model-based Verifier)를 사용하는 대안이 제시되었으나, 이는 대규모의 데이터 어노테이션이 필요하고 학습 과정이 복잡하며 추가적인 계산 비용을 발생시킨다는 한계가 있다.

따라서 본 연구의 목표는 외부 Verifier 없이도 일반 도메인에서 LLM의 추론 능력을 향상시킬 수 있는 확장 가능한 강화학습 프레임워크를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 LLM이 정답 토큰을 생성할 때 가지는 **내재적 확률(Intrinsic Probability)** 자체가 해당 추론 과정이 정답으로 얼마나 잘 유도되었는지에 대한 모델 스스로의 평가(Reward)로 작용한다는 직관에서 출발한다. 이를 위해 저자들은 다음과 같은 기여를 하였다.

- **Verifier-free 프레임워크(RLPR) 제안**: 외부 Verifier 없이 모델의 토큰 확률 점수만을 사용하여 보상을 계산하는 RLPR(Reinforcement Learning with Reference Probability Reward)을 제안하였다.
- **Probability-based Reward (PR)**: 단순한 시퀀스 가능도(Sequence Likelihood) 대신, 정답 토큰들의 평균 디코딩 확률을 사용하는 더 견고한 보상 체계를 설계하였다.
- **Reward Debiasing**: 추론 과정 없이 정답을 바로 생성했을 때의 확률을 빼줌으로써, 질문 자체의 난이도나 정답의 희소성으로 인한 편향을 제거하였다.
- **Standard Deviation Filtering**: 보상의 표준편차를 기반으로 너무 쉽거나 어려운 샘플을 동적으로 제거하는 커리큘럼 학습 메커니즘을 도입하여 학습 안정성을 확보하였다.

## 📎 Related Works

기존의 RLVR 접근 방식은 주로 수학 및 코드 도메인에 집중되어 있으며, 구체적으로는 다음과 같은 한계를 가진다.

- **Rule-based Verifiers**: 상징적 검증기(Symbolic Verifier)나 샌드박스 실행 등을 통해 정답을 판별하지만, 이는 도메인 특화적인 설계가 필요하며 일반 도메인의 자유 형식(Free-form) 답변에는 적용할 수 없다.
- **Model-based Verifiers**: 별도의 LLM을 Verifier로 학습시켜 보상을 부여하는 방식(예: General Reasoner)이다. 하지만 이는 데이터 구축 비용이 높고, 모델의 이해도나 파싱 능력에 따라 보상의 질이 결정되는 한계가 있다.
- **Likelihood-based Approaches**: 최근 VeriFree와 같이 정답 시퀀스의 가능도를 보상으로 사용하는 시도가 있었으나, 이는 답변 길이가 길어질수록 분산이 매우 커지는 문제가 있어 짧은 답변(7토큰 미만)으로 제한되는 제약이 있었다.

RLPR은 이러한 제약 없이, 토큰 단위의 평균 확률을 사용함으로써 답변 길이의 제한을 없애고 외부 모델 없이도 높은 보상 품질을 확보하여 기존 방식들과 차별화된다.

## 🛠️ Methodology

### 1. Probability-based Reward (PR) 계산

모델이 생성한 응답 $o$에서 추론 과정 $z$를 제외하고 최종 답변 $y$ 부분만을 추출한다. 이후 $y$를 학습 데이터의 참조 정답 $y^*$로 교체하여 수정된 시퀀스 $o'$를 구성하고, 이를 모델에 입력하여 각 토큰의 확률 $p_i$를 얻는다.

이때, 단순 곱셈 기반의 가능도(Likelihood)는 작은 확률 값 하나에도 결과가 크게 변하는 높은 분산 문제를 가지므로, RLPR은 다음과 같이 **평균 확률**을 보상으로 사용한다.
$$r = \frac{1}{|y^*|} \sum_{o'_i \in y^*} p_i$$
이 방식은 동의어 처리나 미세한 변동에 더 견고하며, 답변의 질과 더 높은 상관관계를 보인다.

### 2. Reward Debiasing

보상 $r$에는 추론 과정 $z$의 영향뿐만 아니라, 질문 $Q$나 정답 $y^*$ 자체가 가지는 내재적 특성(예: 특정 단어가 원래 생성 확률이 낮은 경우)으로 인한 편향이 포함된다. 이를 제거하기 위해, 추론 과정 $z$ 없이 정답 $y^*$를 직접 생성했을 때의 확률 $r'$를 계산하여 다음과 같이 디바이아싱된 보상 $\hat{r}$을 정의한다.
$$\hat{r} = \text{clip}(0, 1, r - r')$$
이는 정답 생성 확률이 추론 과정 $z$를 통해 얼마나 **향상되었는가**를 측정하는 것으로, 보상의 순수성을 높여 학습을 안정화한다.

### 3. Standard Deviation Filtering

연속적인 값인 PR의 특성상 기존의 이진(0/1) 정확도 필터링을 적용하기 어렵다. 대신, 보상 값들의 표준편차($\text{std}$)를 활용한다. 보상의 표준편차가 너무 낮다는 것은 모든 응답이 일관되게 너무 쉽거나(전부 고득점) 너무 어렵다(전부 저득점)는 것을 의미하며, 이는 학습에 효율적이지 않다.
따라서 지수 이동 평균(EMA)을 통해 동적 임계값 $\beta$를 설정하고, $\text{std} < \beta$인 프롬프트를 제거하는 적응형 커리큘럼 학습을 수행한다.

### 4. 학습 목표

최종적으로 다음과 같은 목적 함수를 최적화한다.
$$\nabla J_{RLPR}(\theta) = \mathbb{E}_{o \sim \pi_\theta(\cdot|x)} [\hat{r} \nabla \log \pi_\theta(o|x)]$$

## 📊 Results

### 실험 설정

- **모델**: Qwen2.5-7B-Base, Llama3.1-8B, Gemma2-2B
- **데이터셋**: WebInstruct에서 추출한 고난도 일반 도메인 프롬프트 77k 개
- **평가 지표**: MMLU-Pro, GPQA, TheoremQA, WebInstruct (일반 영역) / MATH-500, Minerva, AIME24 (수학 영역)

### 주요 결과

- **일반 도메인 성능 향상**: Qwen2.5-7B 기준, 일반 도메인 4개 벤치마크에서 평균 성능이 약 24.9% 향상되었다.
- **Verifier 기반 모델 능가**: 1.5B 파라미터의 별도 Verifier 모델을 사용하는 `General Reasoner`보다 7개 벤치마크 평균 1.6점 더 높은 성능을 기록하였다.
- **타 Verifier-free 방법론 대비 우위**: 최근 제안된 `VeriFree`와 비교했을 때, TheoremQA(+7.6점)와 Minerva(+7.5점)에서 압도적인 성능 차이를 보였다.
- **수학 도메인 전이**: 수학 데이터를 직접 학습하지 않았음에도 불구하고, 일반 도메인 RLVR을 통해 수학 벤치마크(Minerva 등)의 성능이 크게 향상되었으며, 이는 수학 전용 RL 방법론들과 대등한 수준이다.

## 🧠 Insights & Discussion

### 강점 및 분석

RLPR의 가장 큰 강점은 **'LLM의 내재적 확률'**이라는 단순한 신호를 정교한 보상 체계로 변환했다는 점이다. 실험 결과, 작은 규모의 모델(Qwen2.5-0.5B)로 계산한 PR조차도 전용으로 학습된 General-Verifier보다 정답 판별 능력(AUC)이 뛰어남을 확인하였다. 이는 LLM이 이미 정답에 대한 강력한 내부 판단 기준을 가지고 있으며, 이를 적절히 추출(Extract)하는 것이 별도의 모델을 학습시키는 것보다 효율적임을 시사한다.

### 한계 및 논의

- **참조 정답 의존성**: 본 방법론은 여전히 ground-truth reference answer($y^*$)가 필요하다. 완전히 정답이 없는 환경에서의 self-improvement 가능성에 대해서는 추가 연구가 필요하다.
- **계산 효율성**: 보상 계산을 위해 참조 정답을 넣고 다시 포워드 패스를 수행해야 하므로, 단순한 규칙 기반 보상보다는 계산량이 많다.

### 비판적 해석

본 논문은 PR이 엔트로피나 답변 길이와 상관관계가 낮음을 입증하여 보상의 견고함을 주장하였다. 특히 $\hat{r} = r - r'$ 식을 통해 '추론의 가치'를 정량화한 점이 인상적이다. 다만, 이 방식이 매우 복잡한 다단계 추론(Multi-step reasoning)에서 중간 단계의 오류를 잡아낼 수 있는지(Process-based reward)보다는 최종 결과의 확률에 집중하는 Outcome-based reward의 성격이 강하므로, 향후 Process-based PR로의 확장이 필요할 것으로 보인다.

## 📌 TL;DR

본 논문은 외부 Verifier 없이 LLM의 **내재적 토큰 생성 확률**을 보상으로 활용하는 **RLPR** 프레임워크를 제안하였다. 평균 확률 기반의 보상(PR), 정답 편향 제거(Debiasing), 동적 표준편차 필터링을 통해 일반 도메인에서도 RLVR을 성공적으로 적용하였으며, 이는 모델 기반 Verifier를 사용하는 기존 방식보다 더 높은 성능과 확장성을 보여주었다. 이 연구는 향후 Verifier 구축이 어려운 다양한 일반 지식 및 복합 추론 영역에서 LLM의 성능을 끌어올리는 핵심적인 방법론이 될 가능성이 높다.
