# A State-of-the-Art SQL Reasoning Model using RLVR

The Databricks Mosaic Research and Genie Teams (2025)

## 🧩 Problem to Solve

본 논문은 기업 환경에서 발생하는 Text-to-SQL 작업의 한계를 해결하고자 한다. 일반적인 대규모 언어 모델(LLM)은 뛰어난 성능을 보이지만, 특정 조직 내에서만 사용되는 전문 용어(jargon), 조직 고유의 개념, 도구 및 API에 대한 이해가 부족하여 맞춤형 엔터프라이즈 작업 수행에 어려움을 겪는다. 특히 데이터 과학 분야에서 자연어 쿼리를 정확한 SQL 실행문으로 변환하는 작업은 데이터베이스 스키마의 복잡성, 모호한 컬럼 이름, 테이블 조인(join)의 필요성 등으로 인해 난이도가 높다.

연구의 목표는 검증 가능한 보상(Verifiable Rewards)을 활용한 강화학습(RLVR)을 통해, 추가적인 학습 데이터 없이 기존의 BIRD 벤치마크 학습 세트만으로 데이터 과학 분야에서 최첨단(SOTA) 성능을 내는 SQL 추론 모델을 구축하는 것이다.

## ✨ Key Contributions

본 연구의 핵심 아이디어는 **오프라인 강화학습(Offline RL)을 통한 웜업(warm-up) 단계와 온라인 강화학습(Online RL) 단계를 결합한 2단계 학습 레시피**를 적용하는 것이다.

단순히 모델의 가중치를 업데이트하는 것에 그치지 않고, 모델이 정답 SQL에 도달하기 위한 추론 과정(Reasoning Trace)을 생성하도록 유도하며, 실행 결과라는 객관적인 피드백을 통해 모델을 최적화한다. 특히, 모델 선택, 프롬프트 최적화, TAO(Test-time Adaptive Optimization)를 통한 사전 학습, 그리고 RLVR을 통한 정밀 튜닝으로 이어지는 파이프라인을 구축하여 효율적인 성능 향상을 달성하였다.

## 📎 Related Works

기존의 Text-to-SQL 시스템은 초기에는 규칙 기반(rule-based) 방식에 의존하였으나, 이후 LLM을 통해 자연어를 직접 SQL로 변환하는 방향으로 발전하였다. 최근에는 스키마 링킹, 자기 성찰(self-reflection), 정교한 프롬프트 엔지니어링을 결합한 에이전트 기반 설계가 주를 이루고 있으며, 많은 연구가 대규모의 정제된 파인튜닝 데이터셋을 구축하는 데 집중해 왔다.

반면, 본 논문은 다음과 같은 차별점을 갖는다.

- **데이터 의존도 감소**: 대규모의 추가 데이터셋을 구축하는 대신, 강화학습을 통해 모델 스스로 정답을 찾아가도록 하는 방식을 채택하였다.
- **보상 체계의 객관성**: 인간의 선호도를 학습한 보상 모델(Reward Model)을 사용하는 일반적인 RLHF와 달리, SQL 실행 결과라는 객관적이고 검증 가능한 보상(Verifiable Rewards)을 직접 사용한다.
- **최적화 대상의 차이**: GEPA와 같이 프롬프트만을 최적화하는 방식과 달리, LLM의 가중치를 직접 최적화하여 모델 자체의 추론 능력을 향상시켰다.

## 🛠️ Methodology

### 1. 모델 및 프롬프트 선택

- **모델**: 다양한 오픈 소스 및 폐쇄형 모델을 평가한 결과, `Qwen 2.5 32B Coder Instruct` 모델이 BIRD 개발 세트에서 가장 높은 성능을 보여 기본 모델로 선정되었다.
- **프롬프트**: OmniSQL 프롬프트를 기반으로 하되, LLM이 데이터셋을 더 쉽게 이해할 수 있도록 데이터베이스 인코딩 방식을 수정하고, 모델이 SQL 생성 전 추론 과정(Reasoning Trace)을 생성하도록 유도하는 지침을 추가하였다.

### 2. 검증 가능한 보상(Verifiable Reward) 설계

학습에 사용되는 보상 함수는 매우 단순하고 객관적으로 정의된다. BIRD 벤치마크의 실행 메트릭(Execution Metric)을 기반으로 하며, 다음과 같이 정의된다.

- 생성된 SQL의 실행 결과가 정답(Gold SQL)의 결과와 일치하는 경우: $+1$
- 실행 결과가 일치하지 않는 경우: $0$
- SQL 문법이 틀려 실행 자체가 불가능한 경우(Syntactically incorrect): $-1$

### 3. 학습 파이프라인

학습은 크게 두 단계로 진행된다.

**단계 1: TAO (Test-time Adaptive Optimization) 학습**
오프라인 RL 방식인 TAO를 사용하여 모델에 좋은 귀납적 편향(inductive bias)을 제공함으로써 웜스타트(warm start)를 수행한다.

- 현재 모델을 사용하여 각 훈련 데이터에 대해 여러 개의 응답을 생성한다.
- 위에서 정의한 검증 가능한 보상을 통해 각 응답의 점수를 계산한다.
- 수집된 데이터를 바탕으로 오프라인 RL 최적화를 수행하여 모델을 업데이트한다.

**단계 2: RLVR (Reinforcement Learning with Verifiable Rewards) 학습**
TAO로 최적화된 모델을 기반으로 온라인 RL 학습을 수행한다.

- GRPO(Group Relative Policy Optimization)와 같은 최신 알고리즘을 지원하는 Databricks의 RLVR 서비스를 사용하였다.
- 학습 과정에서 낮은 학습률(learning rate)과 낮은 KL-divergence 계수를 사용하는 것이 효과적임을 확인하였으며, 일부 설정에서는 KL-divergence 항을 완전히 제거해도 성능 저하가 없었다.

### 4. 추론 단계의 최적화 (Test-time Computation)

추론 시에는 **Self-consistency** 기법을 적용한다. 모델로부터 여러 개의 응답(본 연구에서는 7개)을 생성한 후, 파라미터가 없는 가중치 기반 다수결 투표(weighted majority vote) 방식을 통해 최종 최적의 응답을 선택한다.

## 📊 Results

### 실험 설정

- **데이터셋**: BIRD 벤치마크 (Train: 9,428 / Dev: 1,534 / Test: 1,789)
- **지표**: Exact-match execution accuracy (생성된 SQL과 정답 SQL의 실행 결과가 일치하는지 측정)

### 정량적 결과

- **단일 호출(Single-call) 설정**: `Databricks-RLVR-32B` 모델은 테스트 세트에서 $73.56\%$의 정확도를 달성하며 SOTA를 기록하였다. 이는 TAO만 적용했을 때($67.40\%$)나 기본 Qwen 모델($64.80\%$)보다 훨씬 높은 수치이다.
- **Self-consistency 적용 설정**: 7개의 응답을 생성하여 투표했을 때, 최종 정확도는 $75.68\%$까지 상승하였다.

### 비교 분석

- **타 모델 대비 효율성**: 다른 상위권 모델들이 8~32개의 응답을 생성하여 유사한 성능을 낸 것에 비해, 본 모델은 단 7개의 응답만으로 더 높은 정확도를 달성하였다.
- **폐쇄형 모델과의 비교**: 놀랍게도 GPT-4o, O3, Claude Sonnet과 같은 상용 모델들이 Qwen이나 Llama 기반의 튜닝 모델보다 BIRD 벤치마크에서 낮은 성능을 보였다.

## 🧠 Insights & Discussion

본 연구는 단순한 보상 함수와 2단계 RL 학습 과정만으로도 매우 강력한 SQL 추론 모델을 만들 수 있음을 입증하였다. 특히 주목할 점은 다음과 같다.

첫째, **오프라인 RL(TAO)의 중요성**이다. 온라인 RL 단계에 진입하기 전 TAO를 통한 웜업만으로도 이미 리더보드 상위 10위권에 해당하는 성능 향상이 가능했다. 다만, TAO 단계에서 과도하게 학습할 경우 이후 온라인 RL 단계의 이득이 줄어드는 경향이 있어 적절한 조절이 필요하다.

둘째, **추론 능력의 일반화**이다. 개발 세트($70.80\%$)보다 테스트 세트($73.56\%$)에서 더 높은 성능을 보인 것은 모델이 특정 데이터에 과적합되지 않고 SQL 생성 능력 자체를 효과적으로 학습했음을 시사한다.

셋째, **상용 모델의 한계**이다. 범용 LLM이 반드시 도메인 특화 작업에서 우위에 있지 않다는 점이 확인되었으며, 이는 특정 도메인(Text-to-SQL)에 특화된 RLVR 학습이 매우 효과적임을 보여준다.

한계점으로는, BIRD가 훌륭한 벤치마크이지만 실제 기업의 모든 복잡한 시나리오를 대변하지는 못한다는 점이 있으며, 본 논문에서는 구체적인 하이퍼파라미터의 수치보다는 경향성 위주로 설명하고 있다는 점이 언급된다.

## 📌 TL;DR

본 논문은 **오프라인 RL(TAO) $\rightarrow$ 온라인 RL(RLVR) $\rightarrow$ Self-consistency**로 이어지는 학습 및 추론 파이프라인을 통해 BIRD 벤치마크에서 $75.68\%$라는 SOTA 성능을 달성한 SQL 추론 모델을 제안한다. 추가적인 학습 데이터 없이 실행 결과라는 검증 가능한 보상만을 사용하여 모델의 가중치를 최적화하였으며, 이는 향후 기업용 BI, 데이터 과학 및 코딩 자동화 도구에 광범위하게 적용될 가능성이 매우 높다.
