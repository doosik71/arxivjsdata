# Agent-RLVR: Training Software Engineering Agents via Guidance and Environment Rewards

Jeff Da, Clinton Wang, Xiang Deng, Yuntao Ma, Nikhil Barhate, Sean Hendryx (2025)

## 🧩 Problem to Solve

본 논문은 검증 가능한 보상을 이용한 강화학습(Reinforcement Learning from Verifiable Rewards, RLVR)을 에이전트 환경, 특히 소프트웨어 공학(Software Engineering, SWE) 작업에 적용할 때 발생하는 한계를 해결하고자 한다. RLVR은 수학이나 단순 코딩 문제와 같이 정답이 명확하고 검증이 쉬운 도메인에서는 큰 성과를 거두었으나, 복잡한 에이전트 환경에서는 다음과 같은 문제로 인해 효율성이 급격히 떨어진다.

첫째, 에이전트 환경은 다단계의 복잡한 추론 과정(multi-step reasoning)을 거쳐야 하므로, 최종 단계에서만 보상이 주어지는 '희소한 보상(sparse reward)' 문제가 심각하다. 이로 인해 모델이 우연히 정답에 도달할 확률이 매우 낮아 학습이 어렵다. 둘째, 외부 환경과의 상호작용을 위한 인프라 설정과 실행 시간이 많이 소요되어 학습의 복잡도가 증가한다.

따라서 본 논문의 목표는 인간의 교육 방식에서 영감을 얻은 '에이전트 가이드(Agent Guidance)' 메커니즘을 도입하여, 에이전트가 복잡한 솔루션 공간을 효율적으로 탐색하고 성공적인 궤적(trajectory)을 생성하도록 유도함으로써 RLVR의 효과를 극대화하는 프레임워크인 Agent-RLVR을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 RLVR의 보상 희소성 문제를 해결하기 위해 **Pedagogical(교육적) 요소인 '가이드'를 학습 루프에 통합**한 점이다.

중심적인 설계 아이디어는 정답을 맞히지 못한 에이전트에게 단순한 실패 신호만 주는 것이 아니라, 숙련된 교사나 테크 리드가 주니어 엔지니어에게 힌트를 주는 것과 유사하게 전략적 계획, 환경 피드백, 상호작용 힌트를 제공하는 것이다. 이를 통해 에이전트가 스스로 정답에 도달할 수 있는 경로를 탐색하게 하며, 이렇게 생성된 '가이드 기반의 성공 궤적'을 RLVR 학습 데이터로 활용하여 정책을 업데이트한다.

또한, 본 연구는 817개의 소프트웨어 공학 환경으로 구성된 특화 데이터셋을 구축하였으며, 이를 통해 Qwen-2.5-72B-Instruct 모델의 PASS@1 성능을 9.4%에서 22.4%까지 향상시킴으로써 제안 방법론의 유효성을 입증하였다.

## 📎 Related Works

본 논문은 다음과 같은 관련 연구들을 배경으로 한다.

1. **RLVR 및 추론 모델**: OpenAI의 o1이나 DeepSeek-R1과 같이 검증 가능한 보상을 통해 추론 능력을 강화하는 연구들이 수행되었다. 그러나 이러한 방식은 주로 단일 응답으로 해결 가능한 수학/코딩 문제에 집중되어 있으며, 도구 사용과 환경 상호작용이 필요한 에이전트 설정으로의 확장은 미흡했다.
2. **SWE 에이전트**: SWE-bench와 같은 벤치마크를 통해 GitHub 이슈를 자동으로 해결하는 에이전트 연구가 활발하다. Agentless와 같은 연구는 고정된 모델을 사용한 스캐폴딩(scaffolding) 최적화에 집중했으나, 본 논문은 모델 자체를 환경 보상에 맞게 학습시키는 것에 집중한다.
3. **강화학습 기반 학습**: SWE-RL과 같은 연구가 RL을 적용했으나, 본 논문은 특히 '가이드'를 통한 데이터 증강과 오프라인 DPO(Direct Policy Optimization)를 결합하여 보상 희소성 문제를 구체적으로 해결하려 했다는 점에서 차별점을 가진다.

## 🛠️ Methodology

### 전체 파이프라인 및 시스템 구조

Agent-RLVR의 학습 루프는 크게 세 단계로 구성된다 (Algorithm 1 참조).

1. **초기 시도 (Initial Attempt)**: 에이전트가 가이드 없이 문제 해결을 시도한다. 결과물(patch)은 도커 컨테이너 환경에서 유닛 테스트를 통해 검증된다.
2. **가이드 생성 및 재시도 (Guidance & Re-attempt)**: 실패한 궤적에 대해 외부 LLM(교사 모델)이 환경 정보를 바탕으로 가이드를 생성한다. 에이전트는 이 가이드를 바탕으로 다시 문제 해결을 시도하며, 성공 시 이를 긍정적 궤적으로 저장한다.
3. **정책 업데이트 (Policy Update)**: 수집된 성공/실패 궤적 쌍을 이용하여 SFT(Supervised Fine-Tuning)와 RLVR(DPO 방식)을 통해 모델의 파라미터를 업데이트한다.

### 가이드(Guidance)의 종류

에이전트의 탐색을 돕기 위해 다음 세 가지 유형의 가이드가 제공된다.

- **Plan**: 문제 정의로부터 도출된 해결 단계 또는 핵심 문제에 대한 전략적 힌트이다.
- **Environment Feedback**: 이전 시도에서 발생한 스택트레이스(stacktrace)나 에러 메시지를 분석하여 무엇을 수정해야 하는지 알려주는 피드백이다.
- **Environment Interaction**: 수정이 필요한 정확한 파일 경로 및 위치(함수명, 라인 번호 등)를 명시하여 로컬라이제이션(localization) 단계의 어려움을 줄여준다.

### 학습 목표 및 손실 함수

본 논문은 RLVR의 구현을 위해 KL 발산 페널티가 포함된 보상 함수와 DPO 손실 함수를 사용한다.

먼저, 기준 정책 $\pi_{\text{ref}}$로부터 모델 $\pi_{\theta}$가 너무 멀어지지 않도록 제약하는 보상 함수 $R(x, y)$는 다음과 같다.
$$R(x, y) = r(x, y) - \beta \log \frac{\pi_{\theta}(y|x)}{\pi_{\text{ref}}(y|x)}$$
여기서 $r(x, y)$는 유닛 테스트 통과 여부에 따른 검증 가능한 보상이며, $\beta$는 하이퍼파라미터이다.

이후, 보상이 높은 윈닝 응답 $y_w$와 보상이 낮은 루징 응답 $y_l$의 쌍을 이용하여 DPO 손실 함수를 통해 모델을 최적화한다.
$$\mathcal{L}_{\text{DPO}}(\theta) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} \left[ \log \sigma \left( \beta \left( \log \frac{\pi_{\theta}(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \log \frac{\pi_{\theta}(y_l|x)}{\pi_{\text{ref}}(y_l|x)} \right) \right) \right]$$
여기서 $\sigma$는 시그모이드 함수이다. $\mathcal{D}$는 유닛 테스트를 통과한 궤적($y_w$)과 실패한 궤적($y_l$)으로 구성된 선호도 데이터셋이다.

## 📊 Results

### 실험 설정

- **데이터셋**: SWE-Gym 및 자체 수집한 817개의 SWE 환경 (총 8,186개 궤적).
- **모델**: Qwen-2.5 시리즈 (14B, 32B, 72B).
- **평가 지표**: SWE-bench Verified 데이터셋에서 PASS@1 및 BEST@1 측정.
- **베이스라인**: 가이드 없이 거부 샘플링(rejection sampling)만 사용한 Agent-SFT.

### 정량적 결과

- **성능 향상**: Qwen-2.5-72B-Instruct 모델의 경우, Agent-RLVR 적용 시 PASS@1 성능이 **9.4% $\rightarrow$ 22.4%**로 대폭 상승하였다.
- **SFT 대비 우위**: 동일 환경에서 학습한 Agent-SFT(20.8%)보다 더 높은 성능을 보였다.
- **Reward Model (RM) 효과**: Agent-RLVR로 생성된 데이터를 통해 보상 모델을 추가 학습시키고, 테스트 시 32개의 샘플 중 최적의 패치를 선택(ranking)하게 했을 때 PASS@1이 **27.8%**까지 상승하였다.

### 가이드의 중요성 분석

- **보상 희소성 완화**: 가이드를 적용했을 때 성공적인 롤아웃(rollout) 횟수가 평균 138회에서 165회로 증가하였으며, 빈 패치(empty patch) 생성 비율이 감소하였다.
- **PASS@k 분석**: PASS@k 수치가 높아질수록 가이드 기반 모델과 비가이드 모델 간의 성능 격차가 더 벌어지는 양상을 보였다. 이는 가이드 학습 모델이 더 다양하고 정확한 솔루션을 생성할 수 있음을 시사한다.

## 🧠 Insights & Discussion

본 연구는 에이전트 학습에서 단순히 결과에 대한 보상만 주는 것이 아니라, **성공으로 가는 경로를 알려주는 '가이드'가 보상 지형의 희소성을 극복하는 핵심**임을 보여주었다. 특히, 가이드 모델이 제공하는 힌트가 모델로 하여금 평소라면 시도하지 않았을 새로운 궤적을 탐색하게 함으로써 능동적인 자기 개선(active self-improvement)을 유도했다는 점이 고무적이다.

**한계 및 비판적 해석**:

1. **언어 제한**: 실험이 Python 언어에 국한되어 있어 다른 프로그래밍 언어에서의 일반화 가능성은 확인되지 않았다.
2. **오프라인 RL의 한계**: 본 논문은 구현의 편의성과 인프라 비용 문제로 오프라인 DPO를 사용하였다. 저자들도 언급했듯이, PPO나 GRPO와 같은 온라인 RL 방식을 적용한다면 더 높은 성능 향상을 기대할 수 있을 것이다.
3. **데이터셋 규모**: 817개의 환경은 SWE 작업의 방대함을 고려할 때 상대적으로 작은 규모이다. 더 많은 도메인의 환경이 추가된다면 모델의 범용 능력이 향상될 수 있을 것이다.

## 📌 TL;DR

본 논문은 소프트웨어 공학 에이전트 학습 시 발생하는 보상 희소성 문제를 해결하기 위해, 교사의 힌트와 같은 **'에이전트 가이드'를 도입한 RLVR 프레임워크(Agent-RLVR)**를 제안한다. 가이드를 통해 생성된 성공 궤적을 DPO로 학습시킨 결과, Qwen-2.5-72B 모델의 성능을 9.4%에서 22.4%로 끌어올렸으며, 추가적인 보상 모델 활용 시 27.8%까지 달성하였다. 이 연구는 복잡한 실세계 환경에서 에이전트를 효율적으로 학습시키기 위한 가이드 기반 RL의 가능성을 제시하였다.
