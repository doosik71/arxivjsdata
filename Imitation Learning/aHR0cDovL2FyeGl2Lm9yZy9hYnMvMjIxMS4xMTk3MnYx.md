# imitation: Clean Imitation Learning Implementations

Adam Gleave et al. (2022)

## 🧩 Problem to Solve

본 논문은 강화학습(Reinforcement Learning, RL)의 핵심 난제 중 하나인 보상 함수(Reward Function) 설계의 어려움을 해결하기 위한 소프트웨어 프레임워크의 필요성을 제기한다. 현실 세계의 많은 작업에서 보상 함수를 절차적으로 명시하는 것은 매우 어렵거나 불가능하며, 설령 보상 함수를 정의하더라도 보상이 매우 희소(Sparse)하여 RL 알고리즘이 효율적으로 학습하지 못하는 문제가 발생한다.

이를 해결하기 위해 모방 학습(Imitation Learning, IL)과 보상 학습(Reward Learning)이 대안으로 사용되지만, 기존의 오픈소스 라이브러리들은 다음과 같은 한계를 가지고 있다. 첫째, 지원하는 알고리즘의 수가 매우 제한적이다. 둘째, 많은 라이브러리가 더 이상 유지보수되지 않으며 오래된 프레임워크(TensorFlow 1, Theano 등)를 기반으로 구축되어 최신 환경에서 사용하기 어렵다. 셋째, 모방 학습 알고리즘은 세부적인 구현 방식에 따라 성능 차이가 크게 나타나는데, 기존 라이브러리들의 구현 품질이 낮아 실험 결과의 신뢰성이 떨어진다는 문제가 있다.

따라서 본 연구의 목표는 신뢰할 수 있고, 모듈화되어 있으며, 최신 딥러닝 백엔드를 사용하는 고품질의 모방 및 보상 학습 라이브러리인 `imitation`을 제공하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 PyTorch와 Stable Baselines3를 기반으로 구축된 `imitation` 라이브러리의 설계 및 구현이다. 주요 설계 아이디어는 다음과 같다.

1. **광범위한 알고리즘 구현**: Inverse Reinforcement Learning (IRL) 3종, Imitation Learning (IL) 3종, 그리고 Preference Comparison 알고리즘 1종을 포함하여 총 7가지의 핵심 알고리즘을 구현하였다.
2. **일관된 인터페이스 (Consistent Interface)**: `BaseImitationAlgorithm`이라는 공통 기반 클래스를 통해 모든 알고리즘에 통일된 API를 제공함으로써, 사용자가 여러 알고리즘을 쉽게 교체하며 실험하고 비교할 수 있게 하였다.
3. **높은 모듈성 (Modularity)**: 보상 네트워크, 정책 네트워크 아키텍처, RL 알고리즘 및 옵티마이저를 코드 수정 없이 자유롭게 변경할 수 있도록 설계하였다. 특히 GAIL과 AIRL과 같이 구조가 유사한 알고리즘은 `AdversarialTrainer`를 상속받아 공통 로직을 공유하도록 구현하였다.
4. **검증된 신뢰성**: 전체 코드의 98%에 달하는 테스트 커버리지를 확보하고, `pytype` 및 `mypy`를 이용한 정적 타입 체크를 수행하여 구현의 정확성을 높였다.

## 📎 Related Works

논문에서는 기존의 모방 학습 라이브러리들(OpenAI Baselines, Stable Baselines2, Intel COACH 등)과 `imitation`을 비교한다. 기존 접근 방식의 한계는 다음과 같다.

- **유지보수 부족**: 많은 참조 구현체들이 수년 전 출시된 이후 업데이트되지 않았으며, 특히 GAIL과 AIRL의 원본 코드베이스는 최신 프레임워크와의 호환성이 낮다.
- **구현의 경직성**: 기존 라이브러리들은 특정 RL 알고리즘을 상속받아 구현하는 경우가 많아, 새로운 RL 알고리즘으로 포팅하기 위해 상당한 양의 코드 수정이 필요했다.
- **실험적 기준선(Baseline)의 부재**: Orsini et al. (2021)의 연구에서 밝혀졌듯이, 모방 학습에서는 작은 구현 디테일이 성능에 큰 영향을 미친다. 하지만 기존 라이브러리들은 엄격한 벤치마크 결과나 테스트 코드를 제공하지 않아, 취약한 기준선으로 인해 잘못된 긍정 결과(Spurious positive results)가 보고될 위험이 있었다.

## 🛠️ Methodology

`imitation` 라이브러리는 확장성과 재사용성을 극대화하기 위해 다음과 같은 구조로 설계되었다.

### 1. 전체 시스템 구조 및 파이프라인
라이브러리는 PyTorch를 백엔드로 하며, RL 에이전트의 학습 부분은 Stable Baselines3와 통합되어 있다. 전체적인 흐름은 `BaseImitationAlgorithm` $\rightarrow$ 구체적인 알고리즘 클래스 $\rightarrow$ 학습 및 평가 스크립트로 이어진다.

### 2. 구현된 주요 알고리즘
- **Inverse Reinforcement Learning (IRL)**:
    - **MCE IRL**: Maximum Causal Entropy IRL을 구현하여 tabular 방식의 보상 학습을 지원한다.
    - **Density Estimation baseline**: 밀도 추정 기반의 보상 학습을 제공한다.
    - **AIRL**: Adversarial IRL을 통해 최신 적대적 보상 학습 기능을 제공한다.
- **Imitation Learning (IL)**:
    - **BC (Behavioral Cloning)**: 전문가 데이터를 이용한 단순 지도 학습 기반 모방을 수행한다.
    - **DAgger**: 전문가의 상호작용 피드백을 통해 정책을 개선하는 인터랙티브 학습을 지원한다.
    - **GAIL**: Generative Adversarial Imitation Learning을 통해 전문가의 상태-행동 분포를 모방한다.
- **Preference Learning**:
    - **DRLHP**: Deep RL from Human Preferences를 통해 궤적 조각 간의 비교 데이터를 기반으로 보상 함수를 추론한다.

### 3. 모듈화 및 확장 메커니즘
- **상속 구조**: `AdversarialTrainer` 클래스를 통해 GAIL과 AIRL의 공통 학습 로직을 캡슐화하고, 각 알고리즘은 판별자(Discriminator)의 정의 부분만 오버라이딩하여 구현한다.
- **구성 가능성**: 사용자는 `imitation` 내부에서 Stable Baselines3가 제공하는 7가지 RL 알고리즘 중 하나를 선택하거나, 동일한 인터페이스를 가진 커스텀 알고리즘을 연결할 수 있다.
- **실험 관리**: `Sacred` 라이브러리를 도입하여 설정(Configuration)과 로깅을 체계화함으로써 실험의 재현성을 보장한다.

## 📊 Results

### 1. 실험 설정
- **데이터셋 및 환경**: MuJoCo 기반의 표준 Gym 환경인 Ant, Half Cheetah, Hopper, Swimmer, Walker를 사용하였다.
- **측정 지표**: 전문가 정책의 리턴(Return) 값을 1, 무작위 정책의 리턴 값을 0으로 정규화한 normalized return을 지표로 사용하였다.
- **비교 대상**: 구현된 7가지 알고리즘의 성능을 전문가(Expert) 및 무작위(Random) 정책과 비교하였다.

### 2. 정량적 결과
- **알고리즘 성능**: 대부분의 환경에서 GAIL, BC, DAgger 등이 전문가 수준에 근접한 성능을 보였다. 
- **특이 사항**: 
    - AIRL은 Ant와 Walker 환경에서 성능이 낮게 나타났으며, DAgger는 Hopper 환경에서 낮은 성능을 보였다.
    - 특히 AIRL의 경우 원본 논문에서는 Ant 환경에서 긍정적인 결과가 보고되었으나, 본 구현에서는 무작위 수준에 가깝게 나타났다. 저자들은 이를 원본 논문이 표준 Gym 환경이 아닌 커스텀 환경을 사용했기 때문인 것으로 분석하였다.

### 3. 소프트웨어 비교 결과 (Table 1 기반)
- **알고리즘 수**: `imitation`은 7종의 알고리즘을 지원하여, 1~2종만 지원하는 다른 라이브러리(APReL, Stable Baselines2 등)보다 압도적으로 넓은 범위를 커버한다.
- **품질 지표**: 테스트 커버리지(98%), 타입 어노테이션 적용 여부, 최신 백엔드(PyTorch) 사용 여부 등 모든 소프트웨어 공학적 지표에서 기존 라이브러리를 능가하거나 대등한 수준임을 확인하였다.

## 🧠 Insights & Discussion

본 논문은 단순한 알고리즘 제안이 아니라, 연구 커뮤니티를 위한 '도구(Tooling)'의 중요성을 강조한다. 딥러닝 및 강화학습 분야에서 흔히 발생하는 "구현의 불투명성"과 "재현성 부족" 문제를 소프트웨어 공학적 접근(단위 테스트, 타입 체크, 모듈화)으로 해결하려 했다는 점이 강점이다.

특히, AIRL의 결과가 원본 논문과 다르게 나타난 점은 매우 중요한 시사점을 준다. 이는 동일한 알고리즘이라 할지라도 환경 설정이나 구현 디테일이 달라지면 결과가 완전히 바뀔 수 있음을 보여주며, 신뢰할 수 있는 표준 라이브러리의 존재가 왜 필수적인지를 역설적으로 증명한다.

다만, 본 논문은 소프트웨어 구현에 집중하고 있어 새로운 이론적 방법론이나 알고리즘적 개선을 제시하지는 않는다. 하지만 이러한 기반 인프라가 구축됨으로써 향후 연구자들이 구현의 늪에 빠지지 않고 새로운 알고리즘 설계에만 집중할 수 있는 환경을 제공했다는 점에서 학술적 가치가 크다고 판단된다.

## 📌 TL;DR

본 연구는 PyTorch 기반의 고품질 모방 및 보상 학습 라이브러리인 `imitation`을 제안한다. 총 7가지의 핵심 알고리즘을 일관된 API와 모듈형 구조로 구현하였으며, 98%의 테스트 커버리지를 통해 신뢰성을 확보하였다. MuJoCo 환경 벤치마크를 통해 대부분의 알고리즘이 전문가 수준의 성능을 낼 수 있음을 검증하였으며, 이는 향후 모방 학습 연구에서 표준적인 실험 기준선(Baseline)으로 활용될 가능성이 매우 높다.
