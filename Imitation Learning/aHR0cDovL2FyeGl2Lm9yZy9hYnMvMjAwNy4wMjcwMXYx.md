# Scaling Imitation Learning in Minecraft

Artemij Amiranashvili, Nicolai Dorka, Wolfram Burgard, Vladlen Koltun, Thomas Brox (2020)

## 🧩 Problem to Solve

본 연구가 해결하고자 하는 핵심 문제는 마인크래프트(Minecraft)와 같이 보상이 매우 희소(sparse rewards)하고 액션 공간(action space)이 방대한 몰입형 3D 환경에서 에이전트가 복잡한 작업을 수행하도록 학습시키는 것이다. 일반적으로 강화학습(Reinforcement Learning, RL)은 이러한 환경에서 뛰어난 성능을 보일 수 있으나, 최적의 정책을 찾기 위해 수십억 번의 환경 상호작용이 필요하다는 심각한 샘플 효율성(sample inefficiency) 문제를 가지고 있다. 특히 마인크래프트의 `ObtainIronPickaxe` 작업은 11개의 세부 작업이 순차적으로 연결되어 있어, 단 한 단계만 실패해도 이후의 모든 진행이 불가능해지는 구조적 어려움이 있다. 따라서 본 논문은 환경과의 직접적인 상호작용 없이 전문가의 시연 데이터(expert demonstrations)만을 사용하는 모방 학습(Imitation Learning, IL)을 통해 이러한 문제를 해결하고 성능을 확장(scaling)하는 것을 목표로 한다.

## ✨ Key Contributions

본 논문의 중심적인 설계 아이디어는 대규모 전문가 데이터셋을 활용하여 모방 학습의 성능을 극대화하기 위해 네트워크 아키텍처의 확장, 데이터 증강(data augmentation), 그리고 손실 함수의 최적화를 체계적으로 분석하는 것이다. 구체적으로는 다음과 같은 기여를 한다.

1. **아키텍처 확장**: DQN부터 Impala, Deep Impala, Double Deep Impala에 이르기까지 네트워크 크기와 깊이가 성능에 미치는 영향을 분석하여, 더 큰 모델이 복잡한 환경 제어에 유리함을 입증하였다.
2. **데이터 증강의 효과 검증**: 다양한 이미지 증강 기법 중 수평 뒤집기(horizontal flipping)가 성능 향상에 결정적인 역할을 한다는 것을 발견하였다.
3. **보조 데이터 활용**: 주 작업(`ObtainIronPickaxe`) 외에 하위 작업인 `Treechop` 데이터를 통합하여 학습 데이터를 확장함으로써 성능을 크게 개선하는 방법을 제시하였다.

## 📎 Related Works

기존의 강화학습 접근 방식인 Rainbow 등은 Atari나 Go와 같은 환경에서 성공을 거두었으나, 마인크래프트와 같이 관찰 공간이 부분적으로만 보이고(partially observable) 보상이 희소한 환경에서는 학습 속도가 매우 느리다는 한계가 있다. 이를 해결하기 위해 전문가의 궤적을 직접 학습하는 모방 학습이 대안으로 제시되었으며, 특히 `MineRL-v0`와 같은 대규모 이미지 기반 시연 데이터셋의 등장은 모방 학습을 적용할 수 있는 기반이 되었다. 본 연구는 기존의 단순한 Behavior Cloning을 넘어, 네트워크 아키텍처의 확장성과 데이터 처리 파이프라인의 최적화가 실제 성능에 미치는 영향을 심도 있게 분석함으로써 기존 접근 방식과의 차별점을 둔다.

## 🛠️ Methodology

### 전체 파이프라인 및 시스템 구조

본 시스템은 전문가의 시연 데이터($s_t, a_t$)를 입력받아 상태 $s$가 주어졌을 때 전문가의 행동 $a$를 예측하는 정책 $\pi(s)$를 학습하는 구조이다.

### 주요 구성 요소 및 역할

1. **상태 공간(State Space)**:
    - **시각 정보**: $64 \times 64$ RGB 이미지.
    - **벡터 정보**: 현재 손에 든 아이템(one-hot encoding)과 인벤토리 내 아이템 보유 현황(multi-hot 또는 float 값으로 인코딩)을 포함한다.
2. **액션 공간(Action Space)**:
    - 이동(앞, 뒤, 좌, 우, 점프, 질주, 공격, 잠입)과 카메라 회전(yaw, pitch) 및 아이템 제작/장착/배치 액션으로 구성된다.
    - 연속적인 카메라 제어는 $22.5^\circ$ 단위로 양자화(quantization)하여 이산적인 액션으로 변환하였다. 최종적으로는 총 130개의 이산 액션으로 정의된다.
3. **네트워크 아키텍처**:
    - **Perceptual Part**: 이미지를 임베딩하는 CNN 부분으로, DQN, Impala, Deep Impala(Fixup Initialization 적용), Double Deep Impala 아키텍처를 테스트하였다.
    - **Vectorial Part**: 인벤토리 정보를 처리하는 FC 레이어이다.
    - **Final Part**: 이미지 임베딩과 벡터 임베딩을 결합(concatenate)한 후 두 개의 FC 레이어를 거쳐 최종 액션 확률을 출력한다.

### 훈련 목표 및 손실 함수

본 논문에서는 두 가지 학습 방식을 비교하였다.

1. **Behavior Cloning (Cross-Entropy Loss)**:
    - 정책을 Softmax 함수로 정의한다:
    $$\pi(s,a) = p^\pi(a|s) = \text{Softmax}_a(f(s,a))$$
    - 전문가의 액션을 정답 클래스로 하여 Cross-Entropy 손실 함수를 통해 학습하며, 추론 시에는 이 분포에서 액션을 샘플링한다.

2. **DQfD (Margin Loss)**:
    - Q-learning의 사전 학습 단계에서 사용되는 방식이다. 전문가 액션 $a_E$에 대해 다음과 같은 Margin Loss를 정의한다:
    $$L_S = \max_{a \in A} [Q(s,a) + m(a_E, a)] - Q(s, a_E)$$
    - 여기서 $m(a_E, a)$는 $a = a_E$일 때 0이고, 그렇지 않으면 $b > 0$인 마진 함수이다. 이는 $Q(s, a_E)$의 값이 다른 액션들의 값보다 최소 $b$만큼 더 크게 만들어 전문가의 행동을 유도한다.

### 학습 절차

- **옵티마이저**: Adam ($\text{lr} = 6.25 \times 10^{-5}$, $\text{weight decay} = 10^{-5}$)을 사용하며 최대 $3 \times 10^6$ 스텝까지 학습한다.
- **데이터 보강**: `Treechop` 데이터셋에는 인벤토리 정보가 없으므로, 주 작업 데이터셋에서 무작위 상태를 샘플링하여 인벤토리 벡터를 합성해 넣어 학습에 활용하였다.

## 📊 Results

### 실험 설정

- **데이터셋**: MineRL-v0 데이터셋.
- **작업**: `ObtainIronPickaxe`(철 곡괭이 획득) 및 `Treechop`(나무 베기).
- **지표**: 에피소드당 평균 보상(Average Return).

### 주요 결과

1. **아키텍처 영향**: 네트워크 크기가 커질수록 성능이 향상되었다. Double Deep Impala가 가장 높은 성능을 보였으며, 이는 DQN 대비 비약적인 상승이다.
2. **데이터 증강**: 수평 뒤집기(Horizontal Flipping)가 성능을 $73\%$ 향상시킨 반면, 밝기, 대비, 포스터라이제이션 등의 다른 증강 기법은 효과가 없거나 오히려 성능을 떨어뜨렸다.
3. **손실 함수 비교**: Cross-Entropy(CE) 기반의 샘플링 정책이 Margin Loss 기반의 결정론적 정책보다 우수한 성능을 보였다. 다만, CE 기반 정책에서도 $\text{argmax}$를 사용하여 결정론적으로 행동하게 하면 성능이 Margin Loss보다 낮아지는 특성을 보였다.
4. **추가 데이터의 효과**: `Treechop` 데이터를 통합했을 때 성능이 크게 향상되었으며, 이는 대규모 데이터셋에서도 데이터의 양이 여전히 병목 지점임을 시사한다.
5. **RL과의 비교**: `Treechop` 작업에서 모방 학습은 단 $50,000$ 스텝 만에 거의 최적의 성능에 도달한 반면, 강화학습(Rainbow)은 대부분의 실행에서 유의미한 보상을 얻지 못했다.

### 최종 성능

최적의 설정(Double Deep Impala + CE Loss + Treechop 데이터 + Flipping)을 사용한 에이전트는 에피소드의 $82\%$에서 돌 곡괭이(stone pickaxe)를 제작하는 데 성공하였으며, 철 곡괭이 제작 성공률은 약 $1\%$ 수준이었다.

## 🧠 Insights & Discussion

본 연구의 핵심 인사이트는 모방 학습이 데이터가 충분할 때 매우 강력하지만, 강화학습과 마찬가지로 **훈련 손실(training loss)과 실제 성능(reward) 간의 상관관계가 매우 낮다**는 점이다. 실험 결과, 테스트 손실이 증가함에도 불구하고 실제 보상은 계속 상승하는 현상이 관찰되었다.

또한, 저자들은 **'성능 붕괴(performance collapse)'** 가능성을 언급한다. 이미지 분류와 같은 일반적인 지도 학습에서는 일부 샘플의 오분류가 전체 정확도에 큰 영향을 주지 않지만, 마인크래프트와 같은 시계열 제어 작업에서는 초기 단계의 사소한 잘못된 액션 하나가 이후의 모든 상태를 변화시켜 전체 궤적을 실패로 이끌 수 있다. 이러한 시간적 상관성(temporal correlation) 때문에 훈련 손실만으로는 실제 에이전트의 성능을 예측하기 어렵고, 반드시 환경에서의 직접적인 평가가 필요하다.

## 📌 TL;DR

본 논문은 마인크래프트의 희소 보상 문제를 해결하기 위해 대규모 시연 데이터를 이용한 모방 학습의 확장 가능성을 연구하였다. **Double Deep Impala 아키텍처, 수평 뒤집기 데이터 증강, 그리고 하위 작업 데이터(`Treechop`)의 통합**이 성능 향상의 핵심 요소임을 밝혀냈으며, 이를 통해 강화학습보다 훨씬 빠르게 높은 성능에 도달할 수 있음을 입증하였다. 이 연구는 복잡한 3D 환경에서 전문가 데이터를 어떻게 효율적으로 활용하여 정책을 확장할 수 있는지에 대한 강력한 베이스라인을 제공한다.
