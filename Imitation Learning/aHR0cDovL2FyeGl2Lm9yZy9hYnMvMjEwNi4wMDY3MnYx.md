# What Matters for Adversarial Imitation Learning?

Manu Orsini, Anton Raichuk, Léonard Hussenot, Damien Vincent, Robert Dadashi, Sertan Girgin, Matthieu Geist, Olivier Bachem, Olivier Pietquin, Marcin Andrychowicz (2021)

## 🧩 Problem to Solve

본 논문은 연속 제어(continuous control) 문제에서 널리 사용되는 Adversarial Imitation Learning (AIL) 프레임워크의 구성 요소와 하이퍼파라미터(Hyperparameters, HPs)가 실제 성능에 미치는 영향을 체계적으로 분석하고자 한다.

기존의 AIL 연구들은 판별자(Discriminator)의 손실 함수 변경이나 온-폴리시(on-policy)에서 오프-폴리시(off-policy) 에이전트로의 전환 등 다양한 개선 방안을 제안해 왔다. 그러나 이러한 개별적인 선택지들이 상호 어떻게 작용하는지, 혹은 어떤 구현 세부 사항이 성능에 결정적인 영향을 미치는지에 대한 엄격하고 대규모의 비교 실험은 거의 이루어지지 않았다. 이로 인해 연구자들은 특정 알고리즘의 개선이 실제 설계의 변경 때문인지, 아니면 단순히 하이퍼파라미터 튜닝의 결과인지 판단하기 어려운 상황이었다.

따라서 본 연구의 목표는 50개 이상의 다양한 설정 옵션을 포함하는 범용 AIL 프레임워크를 구현하고, 합성 데이터(synthetic demonstrations)와 인간 생성 데이터(human-generated demonstrations) 모두를 사용하여 대규모 실험을 수행함으로써 AIL의 성능을 결정짓는 핵심 요소를 규명하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 AIL의 고수준 알고리즘 선택지와 저수준 구현 세부 사항이 성능에 미치는 영향을 정량적으로 분석한 대규모 벤치마크 연구라는 점이다. 주요 기여 사항은 다음과 같다.

1. **범용 AIL 프레임워크 구현**: 4가지 RL 알고리즘과 7가지 판별자 정규화 기법을 포함하여 50개 이상의 하이퍼파라미터를 설정할 수 있는 고도로 구성 가능한(highly-configurable) AIL 에이전트를 구현하였다.
2. **대규모 실험 수행**: 10개의 연속 제어 태스크에서 50만 개 이상의 에이전트를 학습시키는 대규모 스윕(sweep) 실험을 통해 각 설정의 영향력을 분석하였다.
3. **실무적 통찰 제공**: 판별자 정규화, 보상 함수, 관측치 정규화, 데이터 소스(인간 vs AI) 등에 따른 성능 차이를 분석하여 AIL 설계 및 사용을 위한 구체적인 권장 사항을 제시하였다. 특히, 합성 데이터가 인간 데이터의 적절한 대리 지표(proxy)가 될 수 없음을 밝혀내어 기존 연구 관행에 경종을 울렸다.

## 📎 Related Works

본 논문은 GAIL(Generative Adversarial Imitation Learning), AIRL(Adversarial Inverse Reinforcement Learning), DAC(Discriminator-Actor-Critic) 등 기존의 AIL 기반 연구들을 기반으로 한다.

기존 연구들은 주로 특정 구성 요소(예: 새로운 손실 함수나 정규화 기법)를 도입하고 이를 통해 성능이 향상되었음을 보여주는 방식에 집중하였다. 그러나 본 논문은 다음과 같은 차별점을 가진다.
- **통합적 비교**: 개별적인 개선안을 따로 보는 것이 아니라, 가능한 모든 조합의 설정 공간 내에서 어떤 선택이 가장 우수한지 분석하였다.
- **데이터 소스의 확장**: 대부분의 기존 AIL 연구가 RL 알고리즘으로 생성된 합성 데이터만을 사용한 반면, 본 연구는 실제 인간이 수집한 데이터셋(Droit 등)을 포함하여 실제 시나리오에서의 성능을 검증하였다.
- **분석 방법론**: 단순히 최고 성능을 찾는 것이 아니라, 95번째 백분위수(95th percentile) 분석과 상위 5% 설정 내 분포 분석을 통해 특정 선택지의 강건성과 중요도를 평가하였다.

## 🛠️ Methodology

### 전체 시스템 구조
본 연구에서 구현한 AIL 시스템은 판별자($D$)와 정책($\pi$)이 서로 경쟁하는 두 플레이어 게임 구조를 가진다. 판별자는 주어진 상태-동작 쌍 $(s, a)$가 전문가의 것인지 에이전트의 것인지 구분하도록 학습되며, 에이전트는 판별자를 속여 전문가와 유사한 궤적을 생성하도록 학습된다.

### 주요 구성 요소 및 학습 절차
1. **판별자(Discriminator) 학습**: 전문가 데이터 $\mathcal{D}_{expert}$와 에이전트가 생성한 데이터 $\mathcal{D}_{\pi}$를 사용하여 교차 엔트로피(cross-entropy) 손실 함수로 학습한다.
2. **보상 함수(Reward Function) 정의**: 판별자의 출력 $D(s, a)$를 기반으로 에이전트가 최대화해야 할 보상 $r(s, a)$를 정의한다. 본 논문에서는 다음과 같은 보상 함수들을 비교하였다.
   - **GAIL**: $r(s, a) = -\ln(1 - D(s, a))$
   - **AIRL**: $r(s, a) = \ln D(s, a) - \ln(1 - D(s, a))$
   - **Natural**: $r(s, a) = \ln D(s, a)$
   - **FAIRL**: $r(s, a) = -h(s, a) e^{h(s, a)}$ (여기서 $h$는 판별자의 로짓)
3. **에이전트(Agent) 학습**: 정의된 보상 함수를 최대화하기 위해 RL 알고리즘(SAC, TD3, D4PG, PPO)을 사용하여 정책 $\pi$를 업데이트한다.

### 주요 방정식 및 기법
- **Absorbing State**: 에피소드 길이가 가변적인 환경에서 발생하는 보상 편향(reward bias)을 줄이기 위해, 에피소드가 종료되는 시점에 특수 상태인 '흡수 상태(absorbing state)'를 추가하여 에이전트가 전문가와 유사한 길이의 에피소드를 생성하도록 유도하였다.
- **판별자 정규화(Regularization)**: 판별자가 너무 빠르게 최적화되어 에이전트에게 유용한 신호를 주지 못하는 현상을 막기 위해 다음 기법들을 적용하였다.
  - **Spectral Normalization**: 가중치 행렬의 최대 고유값으로 나누어 립시츠(Lipschitz) 연속성을 보장한다.
  - **Gradient Penalty (GP)**: $\lambda(||\nabla_{\hat{x}} D(\hat{x})||^2 - k)^2$ 항을 손실 함수에 추가하여 기울기의 크기를 제한한다.
  - **Mixup**: 전문가 데이터와 에이전트 데이터의 볼록 조합(convex combination)을 사용하여 학습한다.

### 학습 절차 및 분석 지표
- **데이터셋**: OpenAI Gym(HalfCheetah, Hopper, Walker2d, Ant, Humanoid) 및 Adroit(Pen, Door, Hammer) 환경을 사용하였다.
- **성능 측정**: 환경의 원래 보상을 사용하여 평가하며, 무작위 정책의 성능을 0, 전문가의 성능을 1로 정규화한 값의 학습 곡선 아래 면적(AUC)을 통해 성능을 측정하였다.

## 📊 Results

### 에이전트 학습 관련 결과
- **RL 알고리즘**: 오프-폴리시 알고리즘(SAC, TD3, D4PG)이 PPO보다 샘플 효율성(sample complexity) 면에서 압도적으로 우수하였다.
- **보상 함수**: 합성 데이터에서는 AIRL 보상 함수가 가장 우수했으나, 인간 데이터에서는 GAIL의 $-\ln(1-D)$ 방식이 더 좋은 성능을 보였다.
- **관측치 정규화(Observation Normalization)**: 입력 데이터를 평균 0, 표준편차 1로 정규화하는 것이 알고리즘 선택보다 더 큰 성능 영향을 미쳤으며, 환경에 따라 고정(fixed) 정규화와 온라인(online) 정규화의 우위가 달랐다.

### 판별자 학습 관련 결과
- **정규화 기법**: Spectral Normalization이 전반적으로 가장 우수하였으나, 놀랍게도 Supervised Learning에서 쓰이는 Dropout과 Weight Decay가 AIL 전용 정규화 기법들과 대등한 성능을 보였다.
- **판별자 구조**: 복잡한 전용 아키텍처보다 단순한 MLP 구조가 대등하거나 더 나은 성능을 보였다.
- **학습률**: 판별자의 최적 학습률은 RL 에이전트의 학습률보다 $10^{-2}$에서 $10^{-2.5}$배 정도 낮은 수준에서 형성되었다.

### 데이터 소스 비교 (Synthetic vs Human)
- **차이점 발견**: 인간 데이터는 합성 데이터보다 판별자의 정규화에 훨씬 더 의존적이었으며, 판별자 네트워크의 크기가 더 작을 때 더 잘 학습되는 경향이 있었다.
- **입력값의 영향**: 합성 데이터에서는 상태-동작 쌍 $(s, a)$를 모두 입력하는 것이 유리했지만, 인간 데이터(특히 Adroit Door 태스크)에서는 동작 $a$를 제외하고 상태 $s$만 사용하는 것이 더 나은 결과를 보였다.

## 🧠 Insights & Discussion

본 연구는 AIL 프레임워크에서 흔히 간과되는 하이퍼파라미터와 구현 세부 사항이 성능에 결정적인 영향을 미친다는 것을 입증하였다.

**강점 및 발견**:
- 기존 연구들이 주장했던 특정 정규화 기법의 필수성이 사실은 하이퍼파라미터 튜닝의 결과일 수 있음을 보여주었다. (예: Dropout이 Gradient Penalty만큼 효과적일 수 있음)
- **합성 데이터의 함정**: 가장 중요한 통찰은 "합성 데이터로 검증된 알고리즘이 반드시 실제 인간 데이터에서도 잘 작동하지 않는다"는 점이다. 인간의 행동은 다중 모드(multi-modal) 특성을 가지며 마르코프 성질(markovian)을 완벽히 따르지 않기에, 합성 데이터만으로 벤치마킹하는 기존의 관행이 현실 세계의 적용력을 떨어뜨릴 수 있음을 시사한다.

**한계 및 비판적 해석**:
- 본 연구는 매우 방대한 실험을 수행했음에도 불구하고, 모든 하이퍼파라미터의 상호작용을 완전히 설명하기에는 설정 공간이 너무 넓어 일부 결과는 통계적 경향성으로 제시되었다.
- 인간 데이터셋의 경우 데이터의 양이 매우 적어(11개 궤적), 여기서 도출된 결론을 일반화하기 위해서는 더 많은 인간 데이터셋에 대한 검증이 필요하다.

## 📌 TL;DR

본 논문은 50만 개 이상의 에이전트 학습을 통해 Adversarial Imitation Learning(AIL)의 성능을 결정짓는 요소들을 분석한 대규모 벤치마크 연구이다.

**핵심 요약**:
1. **알고리즘**: SAC와 같은 오프-폴리시 RL이 효율적이며, Spectral Norm이 강력한 정규화 도구이지만 Dropout 같은 표준 기법도 충분히 효과적이다.
2. **데이터**: 합성 데이터(AI 생성)와 인간 데이터는 특성이 매우 다르며, 합성 데이터 기반의 평가 결과가 실제 인간 데이터를 사용한 환경에서의 성능을 보장하지 않는다.
3. **권장 사항**: 가변 길이 에피소드에서는 Absorbing State를 사용하고, 관측치 정규화를 반드시 적용하며, 데이터 소스에 따라 보상 함수를 다르게 선택($\text{Synthetic} \to \text{AIRL}$, $\text{Human} \to \text{GAIL}$)할 것을 권장한다.

이 연구는 향후 AIL 알고리즘 설계 시 단순한 구조 변경보다는 데이터의 특성과 세부 하이퍼파라미터 튜닝, 정규화 전략의 중요성을 인지해야 함을 시사한다.
