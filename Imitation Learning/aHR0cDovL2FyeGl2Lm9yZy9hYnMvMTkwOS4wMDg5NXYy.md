# Federated Imitation Learning: A Privacy Considered Imitation Learning Framework for Cloud Robotic Systems with Heterogeneous Sensor Data

Boyi Liu, Lujia Wang, Ming Liu and Cheng-Zhong Xu (2019)

## 🧩 Problem to Solve

본 논문은 로봇의 Imitation Learning(모방 학습) 과정에서 발생하는 데이터 효율성 및 일반화 성능 저하 문제를 해결하고자 한다. 전통적인 모방 학습은 전문가의 시연(demonstration) 데이터를 필요로 하지만, 다음과 같은 현실적인 제약이 존재한다.

첫째, 고품질의 대량 데이터셋을 구축하는 데 막대한 비용과 시간이 소요된다. 둘째, 서로 다른 로봇들이 수집한 데이터를 공유하여 학습 효율을 높이고 싶어도, 센서 구성이 서로 다른 데이터의 이질성(heterogeneity) 문제와 민감한 데이터 유출에 따른 개인정보 보호(privacy) 문제가 걸림돌이 된다.

결과적으로 로봇들은 각자 고립된 데이터만을 사용하는 '정보의 섬(information island)' 상태에 놓이게 되며, 이는 학습 시간의 증가와 제한적인 일반화 성능으로 이어진다. 본 논문의 목표는 데이터 프라이버시를 보호하면서도 이질적인 센서 데이터를 가진 로봇들 간의 지식을 효율적으로 공유하고 전이할 수 있는 Federated Imitation Learning(FIL) 프레임워크를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 원본 데이터를 직접 공유하는 대신, 학습된 모델의 파라미터를 클라우드에서 융합하여 '가이드 모델(guide model)'을 생성하고, 이를 개별 로봇이 전이 학습(transfer learning)을 통해 활용하게 하는 것이다.

주요 기여 사항은 다음과 같다.
1. **FIL 프레임워크 제안**: 이질적인 센서 데이터를 사용하고 프라이버시 보호가 필요한 클라우드 로봇 시스템을 위한 새로운 모방 학습 프레임워크를 제시한다.
2. **Knowledge Fusion 알고리즘**: 클라우드 서버가 여러 로봇으로부터 전달받은 프라이빗 모델들의 파라미터를 융합하여, 다양한 센서 타입에 대응하는 가이드 모델을 생성하는 알고리즘을 제안한다.
3. **Knowledge Transfer 스킴**: Transfer Learning을 기반으로 클라우드의 가이드 모델로부터 로컬 로봇이 지식을 빠르게 습득하여 학습 효율과 정확도를 높이는 방안을 제시한다.

## 📎 Related Works

논문에서는 다음 세 가지 분야의 관련 연구를 다룬다.

1. **Imitation Learning**: 전문가의 행동을 모방하는 기법으로, 본 논문은 상태(state)에서 행동(action)으로의 직접적인 매핑을 학습하는 Behavioral Cloning(BC) 방식에 기반한다. 기존 BC는 데이터셋의 제한과 분포 차이로 인해 취약한 모습을 보였으나, 본 논문은 클라우드 프레임워크를 통해 데이터셋을 무한히 확장함으로써 이를 해결하려 한다.
2. **Cloud Robotic System**: 계산 및 저장 자원을 네트워크를 통해 공유하는 시스템이다. 기존 연구들은 매핑, 국지화, 인식 등에 활용되었으나, 이질적인 데이터를 처리하면서 프라이버시를 고려한 지식 공유 방식에 대해서는 구체적인 제안이 없었음을 지적한다.
3. **Federated Learning (FL)**: 원본 데이터를 중앙 서버로 전송하지 않고 로컬에서 학습된 모델 파라미터만을 공유하여 프라이버시를 보호하는 머신러닝 기법이다. 본 논문은 FL의 철학을 모방 학습에 접목하여 데이터 사일로(data silo) 문제를 해결하고자 한다.

## 🛠️ Methodology

### 전체 파이프라인
FIL 프레임워크는 **로컬 로봇 $\rightarrow$ 클라우드 서버 $\rightarrow$ 요청 로봇**의 흐름으로 동작한다.
1. **로컬 학습**: 각 로봇은 자신이 가진 센서 데이터로 모방 학습을 수행하여 프라이빗 모델을 구축한다.
2. **파라미터 전송**: 원본 데이터가 아닌 모델의 파라미터($\theta$)만을 클라우드로 전송한다.
3. **지식 융합**: 클라우드는 수집된 파라미터들을 사용하여 가이드 모델을 생성한다.
4. **지식 전이**: 서비스 요청을 한 로봇은 자신의 센서 타입에 맞는 가이드 모델을 내려받아 전이 학습을 수행하여 최종 정책을 획득한다.

### Knowledge Fusion 알고리즘
클라우드 서버는 레이블이 없는 자체 데이터베이스를 가지고 있다. 이 데이터베이스의 각 씬(scene)에 대해 여러 프라이빗 모델들의 출력값을 이용해 레이블을 생성한다.

이때, 단순 평균이 아닌 **중앙값(Median)**을 사용하여 레이블을 결정하는데, 이는 자율주행과 같은 작업에서 발생할 수 있는 극단적인 예측 오류(extreme values)를 배제하여 보다 안정적인 가이드 모델을 만들기 위함이다.

수식적 절차는 다음과 같다.
- 로컬 로봇의 학습 목표(구조적 위험 최소화):
$$\theta^*_i = \arg \min_\theta R_{\text{struct } D}(\theta)$$
- 클라우드 데이터의 씬($\text{scene}_{in}$)에 대한 각 모델의 출력값:
$$l_{in} = f_{\theta_i}(\text{scene}_{in})$$
- 융합된 레이블 결정:
$$\text{Me}_{in} = \text{Median}(l_{in})$$
- 클라우드 가이드 모델의 학습 목표($L_2$ 정규화 포함):
$$\theta^*_{\text{i(cloud)}} = \arg \min_\theta \frac{1}{M} \sum_{n=1}^M L(\text{Me}_{in}, f(\text{scene}^{(n)}_{in}; \theta)) + \frac{1}{2}\lambda\|\theta\|^2$$

### Knowledge Transfer (Layer Transfer)
클라우드 모델은 모든 로봇에게 적용 가능한 '조심스러운(cautious)' 정책일 뿐, 각 로봇의 환경에 최적화된 모델은 아니다. 따라서 **Layer Transfer** 방식을 사용한다.

- **방법**: 이미지 인식 네트워크의 특성을 고려하여, 하위 계층(Front layers)인 특징 추출기(feature extractor)는 클라우드 모델의 것을 복사하여 고정(freeze)하고, 상위 계층인 전결합층(Fully connected layers)만을 로컬 데이터로 다시 학습시킨다.
- **효과**: 학습해야 할 파라미터 수가 줄어들어 과적합(over-fitting)을 방지하고, 학습 속도와 정확도를 향상시킨다.

## 📊 Results

### 실험 설정
- **시뮬레이터**: Microsoft AirSim 및 CARLA.
- **작업**: 단순화된 자율주행(장애물 회피, 회전, 직진).
- **데이터 타입**: RGB 이미지, Depth 이미지, Semantic Segmentation 이미지 (총 3종의 이질적 데이터).
- **네트워크**: VGG-16과 유사한 구조의 Policy Network 사용.

### 주요 결과
1. **가이드 모델의 성능 (Table I)**: 
   - 로컬 전용 모델보다 클라우드에서 생성된 모델의 정확도가 더 높았다.
   - 특히 RGB 이미지 기반 컨트롤러의 경우, 로컬 모델은 장애물 충돌률 $3.45\%$, 회전 실수 $12\%$였으나, 클라우드 모델은 충돌률 $0.69\%$, 회전 실수 $0\%$로 획기적인 성능 향상을 보였다.

2. **지식 전이 능력 (Table II & Fig 10)**:
   - 일반 모방 학습(General policy) 대비 FIL을 통한 전이 학습(Transferred policy) 모델의 에러율이 현저히 낮았다.
   - **학습 효율**: Fig 10의 학습 곡선을 통해 전이 학습 모델이 더 낮은 초기 에러 값에서 시작하며 빠르게 수렴함을 확인하였다.
   - **일반화 성능**: 비, 눈, 안개, 먼지 등 악천후 환경에서 전이 학습 모델이 일반 모델보다 훨씬 강건한(robust) 성능을 보였다. (예: RGB 이미지의 경우 먼지 환경에서 일반 모델 에러율 $52.17\% \rightarrow$ 전이 모델 $39.13\%$)

## 🧠 Insights & Discussion

본 논문은 프라이버시 보호와 데이터 이질성이라는 두 가지 난제를 해결하면서도 모방 학습의 효율을 높이는 현실적인 프레임워크를 제시하였다. 특히 클라우드에서 $\text{Median}$ 기반의 레이블링을 통해 노이즈에 강건한 가이드 모델을 생성하고, 이를 Layer Transfer로 연결한 지점이 매우 실용적이다.

**강점**:
- 원본 데이터를 공유하지 않아 프라이버시를 완벽히 보호한다.
- 서로 다른 센서를 사용하는 로봇들이 서로의 지식을 간접적으로 공유할 수 있는 메커니즘을 구현하였다.
- 전이 학습을 통해 'Zero-scratch' 학습의 고통을 줄이고 수렴 속도를 높였다.

**한계 및 논의**:
- 본 연구는 모방 학습(특히 Behavioral Cloning)에만 국한되어 있으며, Reinforcement Learning 등 다른 학습 방식에 대한 유연한 적용 가능성은 검토되지 않았다.
- 실험이 시뮬레이션 환경에서만 진행되었으므로, 실제 물리 로봇의 하드웨어 제약이나 네트워크 지연 시간이 성능에 미치는 영향에 대한 분석이 추가될 필요가 있다.
- 지원하는 센서 데이터의 종류가 제한적이며, 더 다양한 센서 타입에 대한 확장 가능한 융합 방법론이 필요하다.

## 📌 TL;DR

본 논문은 서로 다른 센서를 가진 로봇들이 프라이버시를 유지하며 지식을 공유할 수 있는 **Federated Imitation Learning (FIL)** 프레임워크를 제안한다. 로컬 로봇의 모델 파라미터를 클라우드에서 융합하여 가이드 모델을 만들고, 이를 로컬에서 전이 학습(Layer Transfer)함으로써 **학습 속도 향상, 정확도 증대 및 악천후 등 극한 환경에서의 일반화 성능 개선**을 달성하였다. 이 연구는 향후 클라우드 기반 로봇 군집의 효율적인 지식 전파 시스템 구축에 중요한 기초가 될 가능성이 높다.
