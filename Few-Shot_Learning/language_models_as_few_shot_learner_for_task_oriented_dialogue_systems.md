# Language Models as Few-Shot Learner for Task-Oriented Dialogue Systems

Andrea Madotto, Zihan Liu, Zhaojiang Lin, Pascale Fung (2020)

## 🧩 Problem to Solve

모듈화된 태스크 지향 대화(Task-Oriented Dialogue, ToD) 시스템은 자연어 이해(Natural Language Understanding, NLU), 대화 상태 추적(Dialogue State Tracking, DST), 대화 정책(Dialogue Policy, DP), 그리고 자연어 생성(Natural Language Generation, NLG)의 네 가지 핵심 모듈로 구성된다. 이러한 시스템은 스마트 스피커와 같은 최신 기술의 핵심을 이룬다. 각 모듈은 일반적으로 지도 학습(supervised learning) 또는 강화 학습(reinforcement learning)을 통해 개별적으로 훈련되므로, 많은 양의 주석(annotation) 데이터가 요구된다.

이러한 데이터 수집 과정은 특히 대화 정책 주석과 같은 특정 태스크의 경우 전문가, 심지어 전문 언어학자에 의해 이루어져야 하므로 매우 노동 집약적이고 비용이 많이 든다. 따라서, ToD 시스템의 각 모듈을 최소한의 샘플(few-shots)만으로도 효과적으로 학습할 수 있는 모델을 개발하는 것이 중요한 연구 과제이다.

이 논문의 목표는 GPT-2와 같은 대규모 언어 모델(Language Model, LM)의 "프라이밍(priming)" 기반 퓨샷 학습(few-shot learning) 능력을 NLU, DST, DP (간소화된 대화 행동 분류, ACT), NLG의 네 가지 ToD 태스크에서 평가하는 것이다. 기존의 미세 조정(fine-tuning) 기반 접근 방식과 달리, 프라이밍 방식은 모델 파라미터를 업데이트하지 않고 몇 가지 예시를 모델의 입력 접두사(prefix)로 제공하여 퓨샷 학습을 수행하는 방식이다. 이 논문은 이 접근 방식의 현재 한계를 조명하고, 미래 연구에 대한 시사점을 논의한다.

## ✨ Key Contributions

이 논문의 중심적인 아이디어는 대규모 언어 모델(LM)이 가지는 "프라이밍" 능력을 활용하여 태스크 지향 대화(ToD) 시스템의 네 가지 핵심 모듈(NLU, DST, DP, NLG)에 대한 퓨샷 학습을 수행하는 것이다. 기존의 퓨샷 학습 방법론들이 대규모 사전 학습된 모델을 특정 태스크에 맞게 미세 조정(fine-tuning)함으로써 태스크별 파라미터 세트를 필요로 하는 반면, 이 논문은 파라미터 업데이트 없이 소수의 예시를 모델 입력의 접두사(prefix)로 제공하여 학습하는 방식(인-컨텍스트 학습)을 제안하고 평가한다. 이는 단일 모델이 여러 태스크를 동시에 수행할 수 있도록 하는 유연성을 제공한다.

구체적인 기여는 다음과 같다.

- NLU, DST, ACT (간소화된 DP), NLG 등 ToD 시스템의 주요 4가지 태스크에 걸쳐 LM 프라이밍 기반 퓨샷 학습의 효과를 체계적으로 평가한다.
- 각 태스크의 특성에 맞춰 이진(binary), 값 기반(value-based), 생성(generative)의 세 가지 유형의 프라이밍 접두사 방식을 설계하고 적용한다.
- GPT-2 모델의 다양한 크기(SMALL, LARGE, XL)를 사용하여 LM 크기가 퓨샷 학습 성능에 미치는 영향을 분석한다.
- LM 프라이밍 퓨샷 학습의 현재 한계점, 즉 계산 비용과 제한된 컨텍스트 길이 문제를 식별하고 논의한다.

## 📎 Related Works

태스크 지향 대화 시스템에서 퓨샷 학습의 가장 성공적인 접근 방식은 전이 학습(transfer learning)으로, 대규모 모델을 먼저 방대한 코퍼스에 사전 학습(pre-trained)시킨 후 특정 태스크의 소수 샘플에 대해 미세 조정(fine-tuned)하는 방식이다.

논문에서 언급된 관련 연구 및 그 한계는 다음과 같다.

- **TOD-BERT (Wu et al., 2020)**: 태스크 지향 대화 시스템을 위한 대규모 사전 학습 모델로, 퓨샷 NLU, DST, DP에서 BERT (Devlin et al., 2019)보다 우수한 성능을 달성한다.
- **Liu et al. (2020)**: NLU 모듈의 핵심 태스크인 슬롯 채우기(slot-filling)를 위한 투 스텝(two-step) 분류 방법을 제안했다.
- **SC-GPT (Peng et al., 2020b)**: 퓨샷 NLG를 위한 벤치마크와 태스크에 특화된 사전 학습 언어 모델을 소개했다.
- **Kale and Rastogi (2020)**: T5 (Raffel et al., 2019) 기반의 템플릿 재작성(template rewriting) 스키마를 퓨샷 NLG에 적용했다.
- **Soloist (Peng et al., 2020a)**: 파이프라인 형태의 엔드투엔드(end-to-end) 태스크 지향 대화 시스템을 위한 사전 학습 LM을 제안하며, MWoZ 데이터셋에서 유망한 퓨샷 학습 성능을 보여주었다.
- **메타 학습(meta-learning) 접근 방식**: DP (Xu et al., 2020), NLG/ACT (Mi et al., 2019), 파이프라인 엔드투엔드 모델 (Qian and Yu, 2019), 개인화된 대화 시스템 (Madotto et al., 2019) 등 다양한 ToD 태스크에 적용되었다.

**기존 접근 방식의 한계 및 본 논문과의 차별점**:
위에서 언급된 기존의 퓨샷 학습 방법들은 모두 모델을 소수의 샘플로 미세 조정하는 과정을 거치기 때문에, 각 태스크별로 별도의 파라미터 세트가 필요하다는 한계가 있다.

반면, 이 논문에서 제안하는 접근 방식은 GPT-2 및 GPT-3 (Brown et al., 2020)와 같은 언어 모델의 "프라이밍" 기능을 활용하여 퓨샷 학습을 수행한다. 이 방식에서는 어떠한 파라미터도 업데이트되지 않으므로, 단일 모델이 여러 태스크를 동시에 수행할 수 있다는 점에서 기존 미세 조정 기반 방법들과 차별화된다.

## 🛠️ Methodology

이 연구는 태스크 지향 대화 시스템의 네 가지 핵심 모듈인 NLU, DST, ACT (Dialogue Policy의 간소화된 형태), NLG에 대해 언어 모델 프라이밍 기반 퓨샷 학습의 능력을 평가한다. 모델은 GPT-2의 다양한 크기(SMALL, LARGE, XL) 버전을 사용한다.

### 전체 파이프라인 및 시스템 구조

전체 시스템은 특정 태스크를 해결하기 위해 언어 모델에 "프라이밍" 접두사(prefix)를 제공하는 방식으로 작동한다. 이는 소수의 입력-출력 예시를 모델의 입력으로 먼저 제공한 후, 실제 예측하고자 하는 샘플을 뒤에 이어 붙이는 형태이다. 이 과정에서 모델의 파라미터는 업데이트되지 않는다.

### 각 주요 구성 요소 및 역할

1. **태스크 지향 대화 시스템의 네 가지 태스크**:
    - **NLU (Natural Language Understanding)**: 사용자의 발화 ($X$)로부터 슬롯-값 쌍(SLOT-FILLING)과 의도(INTENT)를 추출한다.
        - **SLOT-FILLING**: 사용자 발화 $X$를 입력으로 받아, 슬롯($s_i$)과 해당 값($v_i$)으로 구성된 사전 $M=\{s_1=v_1, \dots, s_n=v_n\}$을 생성한다. 값 $v_i$는 발화에 언급되지 않은 경우 'None'이 될 수 있다.
        - **INTENT**: 사용자 발화 $X$를 입력으로 받아, $Y \in \{y_1, \dots, y_n\}$ 중 하나의 의도 클래스로 분류한다.
    - **DST (Dialogue State Tracking)**: 대화의 현재 턴까지의 모든 사용자 발화를 기반으로 슬롯-값 쌍을 추출하여 대화 상태를 추적한다. NLU의 대화 수준 버전으로 볼 수 있다. 대화 $D=\{X^U_1, X^S_1, \dots, X^U_t\}$ (사용자 발화 $X^U$, 시스템 발화 $X^S$)를 입력으로 받아, 턴 $t$에서의 사전 $M_t=\{s_1=v_1, \dots, s_n=v_n\}$을 예측한다. 기존 DST 모델들은 이전 턴의 $M_{t-1}$을 활용하여 필요한 슬롯만 업데이트한다.
    - **ACT (Speech-Act Classification)**: 현재 대화 상태(대화 또는 슬롯-값 사전 형태)가 주어졌을 때 다음 시스템의 '화행(speech-act)' (예: INFORM, REQUEST)을 예측한다. 본 연구에서는 이를 다중 레이블 분류 태스크로 간소화하여, 사용자 발화 $X$를 입력으로 받아 가능한 화행 집합 $I \in \{I_1, \dots, I_n\}$으로 분류한다.
    - **NLG (Natural Language Generation)**: 화행(speech-act)과 슬롯-값 사전으로 구성된 대화 행동을 자연어로 매핑한다. 입력은 $I(s_1=v_1, \dots, s_n=v_n)$ 형태이며, 출력은 자연어 발화 $X$이다.

2. **LM 프라이밍(Priming)을 위한 접두사(Prefix) 설계**:
    퓨샷 학습을 위해 모델 파라미터를 미세 조정하는 대신, 소수의 예시를 모델 입력의 접두사로 사용하여 모델을 "프라이밍"한다. 일반 입력은 $X$, $i$-번째 퓨샷 샘플은 $X_i$로 표기한다.
    - **이진(Binary) 접두사**: 분류 태스크(INTENT, ACT)에 사용된다. 다중 클래스 분류도 각각의 클래스를 이진 분류로 처리한다.
        $$X_1 \to \text{True } X^*_1 \to \text{False } \dots X \to$$
        여기서 $X_i$는 퓨샷 샘플 중 하나이고, $X^*_i$는 다른 클래스에서 가져온 샘플이거나 'False' 클래스에 해당하는 샘플이다. $n$개의 클래스를 예측하려면 $n$개의 접두사를 사용해야 하며, 따라서 $n$번의 모델 포워드 패스(forward pass)가 필요하다.

    - **값 기반(Value-based) 접두사**: 발화에서 특정 슬롯의 값을 할당하거나, 값이 없는 경우 'None'을 할당하는 태스크(SLOT-FILLING, DST)에 사용된다. 각 슬롯에 대해 별도의 접두사를 정의한다.
        $$X_1 \to s=v_1 \text{ } X^*_1 \to s=\text{None } \dots X \to s=$$
        여기서 $v_1$은 퓨샷 학습에서 할당된 값이다. 이 과정은 사전에 포함될 각 슬롯에 대해 반복되어 사전 $M$을 생성한다. $n$개의 슬롯을 디코딩하기 위해 $n$번의 모델 포워드 패스가 필요하다.

    - **생성(Generative) 접두사**: 소스 정보(예: 대화 행동)가 주어졌을 때 자연어를 생성하는 태스크(NLG)에 사용된다.
        $$X_1 \to Y_1 \dots X_k \to Y_k \text{ } X \to$$
        여기서 $X_i$와 $Y_i$는 일반적인 단어 시퀀스이다.

### 학습 절차 (모델 추론 절차)

이 논문에서는 파라미터 업데이트가 없는 LM 프라이밍 방식을 사용하므로, 별도의 "학습" 절차보다는 "추론" 절차가 핵심이다.

1. **퓨샷 샘플 준비**: 각 태스크에 대해 소수의 입력-출력 쌍을 준비한다.
2. **접두사 구성**: 위에서 설명한 세 가지 접두사 유형 중 해당 태스크에 맞는 유형을 사용하여 퓨샷 샘플과 실제 예측할 입력 $X$를 조합하여 언어 모델의 입력 시퀀스를 구성한다.
3. **모델 입력**: 구성된 접두사 시퀀스를 GPT-2 모델의 입력으로 제공한다.
4. **예측 생성**: 언어 모델은 접두사에 이어서 가장 가능성 있는 토큰 시퀀스를 생성한다. 이 생성된 토큰들이 해당 태스크의 실제 예측값(예: 클래스 레이블, 슬롯-값, 자연어 발화)이 된다. 분류 및 슬롯 채우기 태스크의 경우, 이진 또는 값 기반 접두사 방식 때문에 여러 개의 포워드 패스가 필요할 수 있다.

## 📊 Results

이 논문은 GPT-2 모델(SMALL: 117M, LARGE: 762M, XL: 1.54B)을 사용하여 NLU, DST, ACT, NLG의 네 가지 태스크에서 LM 프라이밍 퓨샷 학습의 성능을 평가하고, 기존 미세 조정(fine-tuning) 기반 모델의 결과와 비교한다. 모든 실험은 단일 NVIDIA 1080Ti GPU에서 실행되었다.

### 데이터셋, 작업, 기준선, 지표

1. **NLU (Natural Language Understanding)**
    - **데이터셋**: SNIPS (Coucke et al., 2018)
    - **SLOT-FILLING 태스크**:
        - **지표**: CoNLL F1-scorer.
        - **기준선**: Liu et al. (2020)의 20-샷 설정 모델(CT, RZT, Coach).
        - **프라이밍**: 값 기반(value-based) LM 접두사, 최대 15샷.
    - **INTENT 분류 태스크**:
        - **지표**: Accuracy.
        - **기준선**: RoBERTa (Liu et al., 2019)를 10 샘플로 미세 조정.
        - **프라이밍**: 이진(binary) LM 접두사, 최대 10샷.

2. **DST (Dialogue State Tracking)**
    - **데이터셋**: MultiWoZ (Budzianowski et al., 2018; Eric et al., 2019; Zang et al., 2020)
    - **입력**: 다른 연구와 달리 마지막 사용자 발화만 입력으로 사용하고, 예측된 DST를 턴마다 업데이트한다.
    - **지표**: Joint Accuracy 및 Slot Accuracy.
    - **기준선**: TOD-BERT (Wu et al., 2020) 및 BERT (Devlin et al., 2019)를 훈련 데이터의 10% (500 예시)로 미세 조정.
    - **프라이밍**: 값 기반(value-based) LM 접두사, 최대 15샷.

3. **ACT (Speech-Act Identification)**
    - **데이터셋**: MultiWoZ
    - **입력**: 다른 연구와 달리 시스템 발화만 입력으로 사용한다.
    - **지표**: F1-score (Wu et al., 2020의 설정).
    - **기준선**: TOD-BERT (Wu et al., 2020) 및 BERT (Devlin et al., 2019)를 훈련 데이터의 10% (500 예시)로 미세 조정.
    - **프라이밍**: 이진(binary) LM 접두사, 최대 15샷.

4. **NLG (Natural Language Generation)**
    - **데이터셋**: FewShotWOZ (Peng et al., 2020b)
    - **지표**: BLEU 및 Slot Error Rate (SLR).
    - **기준선**: SC-LSTM, GPT-2, SC-GPT-2 (Peng et al., 2020b)를 훈련 데이터에서 50 예시로 미세 조정.
    - **프라이밍**: 생성(generative) LM 접두사, 최대 20샷.

### 주요 정량적 또는 정성적 결과 및 중요성

- **NLU 및 NLG 태스크**: 모델의 크기가 클수록 일반적으로 성능이 향상된다. 이는 Brown et al. (2020)의 가설(대규모 LM이 더 나은 퓨샷 학습자라는 가설)을 지지한다. 특히 NLU와 NLG에서 LM 프라이밍 퓨샷 학습은 가장 약한 미세 조정 기반 기준선(더 많은 샷을 사용)과 유사하거나 더 나은 성능을 달성하는 유망한 결과를 보여주었다.

- **DST 및 ACT 태스크**: GPT-2 LARGE (762M)가 XL (1.54B) 버전보다 더 나은 성능을 보이는 예상치 못한 결과가 관찰되었다. 이는 GPT-3에서 보고된 결과와 상반되며, 접두사 변경을 통해 대규모 모델의 성능을 개선할 수 있는지 추가 조사가 필요함을 시사한다. 특히 DST 태스크에서는 기존 기준선과의 성능 격차가 여전히 크다.

- **LM 프라이밍의 한계**:
  - **계산 비용**: 이진 및 값 기반 생성 방식은 클래스 또는 슬롯 수만큼 모델 포워드 패스를 필요로 한다. 이는 직접적으로 클래스나 태그를 생성하는 것만큼 효율적이지 않다. 초기 실험에서 모든 태스크를 생성(generative) 형식으로 전환하려 시도했지만, 결과가 좋지 않았다.
  - **제한된 컨텍스트 길이**: GPT-2의 최대 입력 길이(1024 토큰)는 모델에 제공할 수 있는 샷의 수를 크게 제한한다. 대부분의 태스크에서 15샷 이상을 제공할 수 없었으며, 이는 더 많은 샷을 사용하는 기존 모델과 직접 비교하기 어렵게 만든다.

**결과의 중요성**:
이 연구는 미세 조정 없이 대규모 언어 모델의 인-컨텍스트 학습 능력을 활용하여 ToD 시스템의 여러 핵심 구성 요소에서 퓨샷 학습이 가능하다는 것을 실증한다. NLU, ACT, NLG에서는 경쟁력 있는 성능을 보여주며, 비용이 많이 드는 데이터 수집 문제를 완화할 수 있는 잠재력을 제시한다. 그러나 DST에서의 한계와 특정 태스크에서 가장 큰 모델의 성능이 떨어지는 현상은 향후 연구 방향을 제시한다. 계산 효율성과 컨텍스트 길이 제한은 이 방법론이 실용적인 ToD 시스템에 적용되기 전에 해결해야 할 중요한 문제로 지적된다.

## 🧠 Insights & Discussion

이 논문은 LM 프라이밍 기반 퓨샷 학습이 태스크 지향 대화(ToD) 시스템의 주요 태스크들(NLU, DST, ACT, NLG)에서 지닌 잠재력을 보여준다.

### 논문에서 뒷받침되는 강점

- **퓨샷 학습의 잠재력**: 대부분의 태스크(NLU, ACT, NLG)에서 LM 프라이밍 기반 퓨샷 학습이 유망한 결과를 보였으며, 일부 경우에서는 가장 약한 미세 조정 기반 기준선보다 유사하거나 더 나은 성능을 달성할 수 있음을 입증하였다. 이는 데이터 수집 비용이 높은 ToD 시스템의 현실적인 문제 해결에 기여할 수 있는 가능성을 제시한다.
- **대규모 LM의 효과 확인**: NLU 및 NLG 태스크에서 모델 크기가 클수록 성능이 향상되는 경향을 보여, 대규모 언어 모델이 더 나은 퓨샷 학습자라는 가설(Brown et al., 2020)을 부분적으로 확인한다.
- **파라미터 업데이트 불필요**: 미세 조정과 달리 파라미터 업데이트가 필요 없으므로, 단일 모델로 여러 태스크를 처리할 수 있는 유연성과 효율성을 제공한다. 이는 모델 배포 및 관리 측면에서 이점을 가진다.

### 한계, 가정 또는 미해결 질문

- **계산 비용의 비효율성**: 이진 및 값 기반 접두사 방식은 분류 클래스 수 또는 슬롯 수에 비례하여 여러 번의 모델 포워드 패스를 요구한다. 이는 직접 생성이 가능한 경우보다 계산적으로 비효율적이다. 모든 태스크를 생성 형식으로 전환하려는 시도가 성공적이지 못했다는 점은 이 방식의 일반화에 대한 한계를 보여준다. GPT-3와 같은 더 큰 LM이 이 문제를 해결할 수 있을지에 대한 질문이 남아 있다.
- **제한된 컨텍스트 길이**: GPT-2의 최대 입력 길이(1024 토큰)는 모델에 제공할 수 있는 퓨샷 샘플의 수를 극히 제한한다(대부분의 태스크에서 15샷 이하). 이로 인해 더 많은 샷을 사용하는 기존 미세 조정 기반 모델들과 직접적인 비교가 어려워진다.
- **예상치 못한 모델 성능 저하**: DST 및 ACT 태스크에서는 GPT-2 LARGE (762M)가 GPT-2 XL (1.54B)보다 더 나은 성능을 보였다. 이는 일반적으로 모델 크기가 커질수록 성능이 향상된다는 기대와 상반되며, 프라이밍 접두사 설계 또는 태스크별 특성이 대규모 모델에 다르게 영향을 미칠 수 있음을 시사하는 미해결 질문이다.
- **DST 태스크의 큰 성능 격차**: DST 태스크에서는 LM 프라이밍 방식이 기존 기준선 대비 여전히 큰 성능 격차를 보였다. 이는 대화 상태 추적의 복잡성이나 장기적인 의존성 처리 능력에서 LM 프라이밍이 아직 한계를 가짐을 나타낸다.

### 논문에 근거한 간략한 비판적 해석 및 논의사항

이 논문은 LM 프라이밍을 ToD 시스템에 적용하는 초기 탐색 연구로서 가치가 있다. 특히 미세 조정 없이 대규모 LM을 활용한다는 아이디어는 상당한 잠재력을 가지고 있다. 하지만 결과에서 드러난 한계점들은 이 접근 방식의 실질적인 적용을 위해 해결해야 할 과제가 많음을 보여준다.

계산 비용 문제와 컨텍스트 길이 제한은 GPT-2 모델의 구조적 한계와 관련이 깊다. Transformer XL, LongFormer, BigBird와 같이 더 긴 컨텍스트를 처리할 수 있는 LM들을 벤치마킹하는 것이 이러한 한계를 극복하는 데 중요할 것으로 보인다. 또한, 이진 및 값 기반 프라이밍 방식의 비효율성을 개선하거나, 생성형 프라이밍 방식의 성능을 향상시키는 연구가 필요하다. GPT-2 XL의 예상치 못한 성능 저하는 프라이밍 메커니즘 자체에 대한 심층적인 이해와 최적화가 필요함을 의미할 수 있다. 단순히 모델 크기를 늘리는 것만이 해결책이 아닐 수 있으며, 태스크에 특화된 프라이밍 전략 또는 더 정교한 "트리거(triggers)" 설계를 탐색할 필요가 있다.

전반적으로, 이 연구는 LM 프라이밍이 ToD 퓨샷 학습을 위한 유망한 방향임을 제시하지만, 효율성, 특정 태스크에서의 성능 격차, 그리고 모델 크기-성능 관계에 대한 더 깊은 이해가 필요함을 강조한다.

## 📌 TL;DR

이 논문은 태스크 지향 대화(ToD) 시스템의 핵심 모듈(NLU, DST, ACT, NLG)에서 대규모 언어 모델(LM)의 "프라이밍" 기반 퓨샷 학습 능력을 평가한다. 기존의 미세 조정(fine-tuning) 방식과 달리, 이 방법은 모델 파라미터 업데이트 없이 소수의 예시를 입력 접두사(prefix)로 제공하여 퓨샷 학습을 수행한다.

주요 기여는 다음과 같다.

1. GPT-2 모델(SMALL, LARGE, XL)을 사용하여 NLU, DST, ACT, NLG 등 ToD 시스템의 4가지 주요 태스크에 걸쳐 LM 프라이밍의 효과를 체계적으로 탐색한다.
2. 각 태스크의 특성을 반영하여 이진(binary), 값 기반(value-based), 생성(generative)의 세 가지 유형의 프라이밍 접두사를 설계하고 적용한다.
3. NLU, ACT, NLG 태스크에서 LM 프라이밍이 유망한 퓨샷 학습 결과를 보여주며, 일부 경우 더 많은 샷을 사용하는 가장 약한 미세 조정 기준선과 유사하거나 더 나은 성능을 달성할 수 있음을 입증한다.

그러나 이 연구는 LM 프라이밍의 두 가지 주요 한계점을 밝힌다. 첫째, 이진 및 값 기반 프라이밍 방식은 여러 번의 모델 포워드 패스를 요구하여 계산 비용이 비효율적이다. 둘째, GPT-2의 제한된 최대 입력 길이(1024 토큰)는 제공할 수 있는 퓨샷 샘플의 수를 제한하여, 더 많은 샷을 사용하는 기존 모델과의 비교를 어렵게 만든다. 또한, DST 태스크에서는 기존 기준선과의 성능 격차가 여전히 크며, 일부 태스크(DST, ACT)에서는 가장 큰 모델(GPT-2 XL)이 중간 크기 모델(GPT-2 LARGE)보다 성능이 낮은 예상치 못한 결과도 관찰되었다.

이 연구는 LM 프라이밍이 ToD 퓨샷 학습의 유망한 방향임을 보여주지만, 효율성과 컨텍스트 길이의 한계를 해결하기 위한 추가 연구(예: 더 긴 컨텍스트를 처리하는 LM 활용, 프라이밍 전략 최적화)가 필요함을 시사한다. 이는 실제 ToD 시스템의 데이터 수집 비용 문제를 완화하고, 향후 대규모 언어 모델 기반 대화 시스템 연구의 중요한 발판을 마련할 가능성이 있다.
