# Active Learning for Crowd-Sourced Databases

Barzan Mozafari, Purna Sarkar, Michael Franklin, Michael Jordan, Samuel Madden (2014)

## 🧩 Problem to Solve

본 논문은 크라우드 소싱(Crowd-sourcing) 기반 데이터베이스에서 발생하는 확장성(Scalability) 문제를 해결하고자 한다. 이미지 태깅, 엔티티 해상도(Entity Resolution), 감성 분석과 같이 인간이 컴퓨터보다 더 정확하게 수행할 수 있는 작업의 경우 크라우드 소싱이 널리 사용된다. 그러나 인간의 노동력에 의존하는 방식은 시간과 비용이 많이 소요되므로, 수백만 개의 아이템으로 구성된 웹 규모(Web-scale)의 대규모 데이터셋에 적용하기에는 한계가 있다.

따라서 본 연구의 목표는 머신러닝 분류기(Classifier)의 속도 및 비용 효율성과 인간 라벨링의 정확성을 결합하여, 전체 라벨링 비용을 최소화하면서도 높은 정확도를 유지할 수 있는 Active Learning(AL) 알고리즘을 설계하는 것이다. 특히, 실무자들이 쉽게 사용할 수 있도록 범용성(Generality), 블랙박스 처리(Black-box treatment), 배치 처리(Batching), 병렬성(Parallelism), 그리고 노이즈 관리(Noise management)라는 다섯 가지 실무적 설계 기준을 만족하는 시스템을 구축하는 것을 목표로 한다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **비모수 부트스트랩(Nonparametric Bootstrap)** 이론을 Active Learning에 도입하여, 분류기의 내부 구조를 수정하지 않고도 모델의 불확실성을 추정하고 최적의 학습 데이터를 선택하는 것이다. 주요 기여 사항은 다음과 같다.

1.  **두 가지 AL 랭킹 알고리즘 제안**: 분류기의 불확실성만을 고려하는 `Uncertainty` 알고리즘과, 불확실성과 잠재적 영향력을 동시에 고려하여 기대 오차를 최소화하는 `MinExpError` 알고리즘을 제안하였다.
2.  **PBA(Partitioning-Based Allocation) 기법**: 크라우드 작업자의 라벨링 노이즈를 관리하기 위해, 데이터셋을 하위 그룹으로 나누고 각 그룹의 난이도에 따라 중복 라벨링 횟수를 동적으로 할당하는 정수 선형 계획법(ILP) 기반의 기법을 제안하였다.
3.  **실무적 제약 조건 충족**: 부트스트랩을 통해 분류기를 블랙박스로 처리함으로써 범용성을 확보하였으며, 각 부트스트랩 계산이 독립적이므로 현대적인 멀티코어 및 분산 환경에서 병렬 실행이 가능하도록 설계하였다.

## 📎 Related Works

기존의 Active Learning 연구들은 주로 이론적인 샘플 복잡도(Sample Complexity)의 경계를 설정하는 데 집중하였으며, 다음과 같은 한계를 지닌다.

-   **범용성 및 블랙박스 처리 부족**: 많은 AL 알고리즘이 특정 분류기(예: SVM, Neural Networks)에 종속적이거나, 분류기의 내부 손실 함수를 수정해야 하는 제약이 있다.
-   **배치 처리 및 병렬성 부재**: 이론적 AL은 대개 아이템을 하나씩 처리하는 온라인(Online/Streaming) 시나리오를 가정한다. 이는 대규모 데이터셋에서 계산 비용이 너무 높고, 크라우드 소싱의 특성상 여러 작업을 동시에 요청해야 하는 배치 처리 요구사항과 맞지 않는다.
-   **노이즈 관리 미흡**: 전통적인 AL은 전문가가 제공한 정답(Ground Truth)을 가정하지만, 크라우드 소싱 데이터는 오타, 지식 부족, 스팸 등으로 인해 상당한 노이즈를 포함하고 있다.

본 논문은 이러한 간극을 메우기 위해 이론적 경계보다는 실무적인 설계 기준을 우선시하며, 부트스트랩을 통해 블랙박스 분류기에 대해서도 효율적인 배치 및 병렬 처리를 지원함으로써 기존의 IWAL이나 Bootstrap-LV 등의 알고리즘과 차별화된다.

## 🛠️ Methodology

### 1. 비모수 부트스트랩 (Nonparametric Bootstrap)
분류기 $\theta$가 데이터 $L$로부터 학습되어 아이템 $u$의 라벨 $\theta^L(u)$를 예측할 때, $L$을 모분포 $D$에서 추출된 표본으로 간주한다. 부트스트랩은 $L$ 자체를 분포의 대리물로 사용하여, $L$에서 중복 허용 추출(Resampling)을 통해 $k$개의 복제본 $S_1, \dots, S_k$를 생성한다. 각 복제본으로 학습된 모델들의 예측값 $\theta^{S_1}(u), \dots, \theta^{S_k}(u)$를 통해 예측값의 경험적 분포 $\hat{D}(u)$를 구성하며, 이를 통해 예측의 분산과 불확실성을 추정한다.

### 2. Uncertainty 알고리즘
분류기가 가장 확신하지 못하는(즉, 분산이 큰) 아이템을 요청하는 방식이다. 분류 에러 $e_u$가 분산, 편향, 노이즈의 합으로 분해된다는 이론에 근거하여, 분산을 줄임으로써 전체 에러를 낮추고자 한다.
이진 분류에서 $X(u)$를 $k$개의 부트스트랩 모델 중 라벨 1을 예측한 모델의 비율이라고 할 때, 불확실성 점수는 다음과 같이 계산된다.
$$\text{Uncertainty}(u) = \text{Var}[\theta^L(u)] = X(u)(1 - X(u))$$

### 3. MinExpError 알고리즘
단순히 어려운 질문을 하는 것이 아니라, 정답을 얻었을 때 모델의 성능 향상 폭이 가장 큰 아이템을 선택한다.
-   $\hat{p}(u)$: 부트스트랩 모델들이 현재 예측값 $l$과 일치하는 비율로, 예측이 정답일 확률의 추정치이다.
$$\hat{p}(u) = \frac{1}{k} \sum_{i=1}^k \mathbb{1}(l^i = l)$$
-   $\hat{e}_{\text{right}}, \hat{e}_{\text{wrong}}$: 예측값이 정답일 때와 오답일 때, 각각 해당 데이터를 학습셋에 추가하고 다시 학습시켰을 때의 예상 에러(교차 검증으로 추정)이다.
-   **최종 점수**: 예측이 맞았을 때와 틀렸을 때의 기대 오차를 가중 평균하여 계산한다.
$$\text{MinExpError}(u) = \hat{p}(u)\hat{e}_{\text{right}} + (1 - \hat{p}(u))\hat{e}_{\text{wrong}}$$

### 4. PBA (Partitioning-Based Allocation)
크라우드 라벨의 노이즈를 줄이기 위해 중복도(Redundancy)를 최적화하는 기법이다.
1.  **분할**: 데이터를 $G$개의 하위 그룹으로 나눈다.
2.  **추정**: 각 그룹 $g$에서 작업자가 정답을 맞힐 확률 $p_g$를 추정한다.
3.  **최적화**: 다수결 투표(Majority Vote) 시 정답 확률 $P_{g,b}$를 계산하고, 주어진 예산 $B$ 내에서 전체 오차를 최소화하는 그룹별 투표 수 $V_g$를 정수 선형 계획법(ILP)으로 결정한다.
$$P_{g,b} = \sum_{i=0}^{v} \binom{b}{i} p_g^{b-i} (1-p_g)^i \quad (\text{where } b=2v+1)$$

### 5. 실행 시나리오
-   **Upfront**: 초기 라벨 $L_0$만을 사용하여 한 번에 요청할 아이템들을 결정하고 크라우드에 요청한다.
-   **Iterative**: 소량의 라벨을 요청하고 모델을 재학습하는 과정을 예산 소진 시까지 반복한다.

## 📊 Results

### 실험 설정
-   **데이터셋**: Amazon Mechanical Turk를 통해 수집된 3종의 실제 데이터(엔티티 해상도, 성별 감지, 감성 분석) 및 UCI KDD의 15종 데이터셋.
-   **비교 대상**: Random Baseline, IWAL, Bootstrap-LV, MarginDistance, Entropy 등.
-   **지표**: 학습 곡선의 AUC, $\text{AUC}_{\text{LOG}}$, Questions Saved(베이스라인 대비 질문 절감 비율).

### 주요 결과
-   **질문 수 절감**: 랜덤 베이스라인 대비 질문 수를 평균 $100\times$(Upfront) 및 $7\times$(Iterative) 절감하였으며, 최신 범용 AL 알고리즘인 IWAL 대비로도 $44\times$(Upfront) 및 $4.5\times$(Iterative) 더 적은 질문만으로 동일 성능을 달성하였다.
-   **알고리즘별 특성**:
    -   `MinExpError`는 Upfront 시나리오에서 매우 강력한 성능을 보였으며, 이는 제한된 초기 데이터만으로 최적의 질문을 선택해야 하는 상황에 적합함을 시사한다.
    -   `Uncertainty`는 Iterative 시나리오에서 `MinExpError`와 대등하거나 더 나은 성능을 보였으며, 계산 비용이 훨씬 낮아 효율적이다.
-   **PBA 효과**: 데이터의 난이도에 따라 예산을 다르게 할당하는 PBA 기법이 균등 할당(Uniform allocation)보다 크라우드 에러를 약 $10\%$ 감소시켰다.
-   **범용성 확인**: 특정 도메인에 특화된 알고리즘(CrowdER, CVHull 등)과 비교했을 때도 경쟁력 있거나 오히려 우수한 성능을 기록하였다.

## 🧠 Insights & Discussion

본 논문은 이론적 AL과 실무적 구현 사이의 간극을 부트스트랩이라는 통계적 도구로 성공적으로 메웠다. 특히 분류기를 블랙박스로 취급함으로써, 사용자가 어떤 최신 분류기를 사용하더라도 즉시 적용 가능한 범용 프레임워크를 제공했다는 점이 큰 강점이다.

또한, 계산 복잡도 측면에서 부트스트랩의 각 복제본 학습이 독립적이라는 점을 이용하여 '당황스러울 정도로 병렬 가능한(Embarrassingly Parallel)' 구조를 구현함으로써, AL의 고질적인 문제인 계산 오버헤드를 해결하였다. 

다만, `MinExpError`의 경우 모든 미라벨링 아이템에 대해 케이스 분석을 수행해야 하므로 $\mathcal{O}((|U|+k) \cdot T(|U|))$의 복잡도를 가지며, 이는 데이터셋 크기가 극단적으로 커질 경우 병렬 처리 능력이 있더라도 상당한 자원을 요구할 수 있다는 가정이 필요하다. 또한, PBA 기법에서 $p_g$를 추정하기 위해 소량의 골드 데이터나 사전 라벨링이 필요하다는 점이 실제 적용 시 제약 사항이 될 수 있다.

## 📌 TL;DR

본 논문은 크라우드 소싱 데이터베이스의 확장성 문제를 해결하기 위해 **비모수 부트스트랩 기반의 Active Learning 알고리즘(`Uncertainty`, `MinExpError`)**과 **노이즈 관리 기법(`PBA`)**을 제안하였다. 이 방식은 분류기의 내부 구조를 수정할 필요 없는 블랙박스 접근법을 취하면서도 병렬 처리를 통해 대규모 데이터셋에 적용 가능하도록 설계되었다. 실험 결과, 기존의 범용 및 도메인 특화 AL 알고리즘보다 훨씬 적은 수의 질문으로 높은 정확도를 달성하였으며, 이는 실제 크라우드 소싱 서비스의 비용을 획기적으로 낮추고 데이터 구축 속도를 높이는 데 중요한 기여를 할 것으로 기대된다.
