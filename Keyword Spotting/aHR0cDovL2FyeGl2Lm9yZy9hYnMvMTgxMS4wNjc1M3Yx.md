# Stochastic Adaptive Neural Architecture Search for Keyword Spotting

Tom Véniat, Olivier Schwander, and Ludovic Denoyer (2018)

## 🧩 Problem to Solve

본 논문이 해결하고자 하는 문제는 실시간 오디오 스트림에서 특정 키워드를 식별하는 Keyword Spotting(KWS) 시스템의 계산 효율성 문제이다. KWS는 가상 비서와 같이 항상 주변 소리를 듣고 있어야 하는 시스템에서 필수적이지만, 기존의 딥러닝 모델들은 매우 방대한 파라미터를 가지고 있어 추론 시 계산 비용과 에너지 소비가 매우 높다는 한계가 있다.

특히 기존 방식들은 입력 신호의 난이도(예: 단순한 정적/소음 상태인지, 실제 키워드가 포함된 복잡한 신호인지)와 관계없이 모든 프레임에 대해 동일한 크기의 정적(static) 네트워크 구조를 적용한다. 이에 따라 단순한 처리가 가능한 구간에서도 불필요하게 많은 계산 자원을 낭비하게 된다. 본 연구의 목표는 입력 스트림의 난이도에 따라 추론 시점에 신경망 구조를 동적으로 변경하여, 쉬운 입력에는 작은 구조를, 어려운 입력에는 큰 구조를 사용하는 적응형 아키텍처 탐색 방법을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 추론 단계에서 각 타임스텝마다 신경망의 아키텍처를 동적으로 적응시킬 수 있는 SANAS(Stochastic Adaptive Neural Architecture Search) 알고리즘을 제안한 점이다.

중심적인 설계 아이디어는 모델이 예측 성능(Prediction Performance)과 평균 계산 비용(Average Computational Cost) 사이의 트레이드오프를 최적화하도록 학습시키는 것이다. 이를 통해 모델은 스스로 어떤 상황에서 더 많은 계산 자원을 투입해야 성능이 올라가는지를 학습하며, 결과적으로 정적 모델보다 훨씬 적은 평균 FLOPs(Floating Point Operations)만으로도 동등하거나 더 높은 인식률을 달성할 수 있게 한다.

## 📎 Related Works

기존의 Neural Architecture Search(NAS) 연구들은 주로 예측 품질과 비용 사이의 균형을 맞추는 최적의 구조를 찾는 데 집중해 왔다. 유전 알고리즘(Genetic Algorithms)이나 강화 학습(Reinforcement Learning)을 이용한 방식들이 대표적이며, 최근에는 파라미터 학습과 아키텍처 탐색을 동시에 수행하는 효율적인 기법들이 등장하였다.

그러나 기존의 모든 NAS 접근 방식은 탐색 결과로 하나의 '정적인' 아키텍처를 도출하며, 일단 결정된 구조는 모든 입력 데이터에 대해 동일하게 적용된다. 반면, KWS 분야에서 사용되는 CNN이나 CNN-RNN 결합 모델들 역시 모든 오디오 프레임에 대해 동일한 연산을 수행하므로, 입력 데이터의 특성에 따라 계산량을 조절할 수 없다는 명확한 한계가 존재한다. SANAS는 이러한 정적 구조의 한계를 넘어 타임스텝별 동적 적응을 가능하게 함으로써 차별점을 가진다.

## 🛠️ Methodology

### 전체 시스템 구조

SANAS는 전체 탐색 공간을 대표하는 거대한 그래프인 Super-Network 내에서 최적의 서브 그래프(sub-graph)를 동적으로 찾아내는 방식으로 동작한다. 전체 파이프라인은 다음과 같다.

1. 현재 타임스텝 $t$에서 이전 상태 $z_t$를 바탕으로 아키텍처 분포 $\Gamma_t$를 생성한다.
2. 이 분포로부터 구체적인 아키텍처 $H_t$를 샘플링한다.
3. 샘플링된 아키텍처를 사용하여 입력 $x_t$에 대해 예측 $\hat{y}_t$를 수행하고, 다음 상태를 위한 특징 벡터 $\Phi$를 추출한다.
4. 추출된 특징 벡터를 통해 다음 상태 $z_{t+1}$을 업데이트한다.

### 주요 구성 요소 및 동작 원리

- **Super-Network**: 레이어들의 집합 $L=\{l_1, \dots, l_n\}$과 이들을 연결하는 에지 $E$로 구성된 유향 비순환 그래프(DAG)이다. 각 에지는 작은 신경망 $f_{i,j}$를 포함한다.
- **아키텍처 샘플링**: 모델은 베르누이 분포 $B(\Gamma_t)$를 통해 에지의 활성화 여부를 결정하는 이진 행렬 $H_t$를 샘플링한다. 최종적으로 사용되는 아키텍처 $A_t$는 Super-Network의 에지 행렬 $E$와 샘플링된 행렬 $H_t$의 하다마르 곱(Hadamard product, $\circ$)으로 정의된다 ($A_t = E \circ H_t$).
- **동적 제어**: RNN(구체적으로는 GRU)이 컨텍스트 $z_t$를 유지하며, 선형 레이어와 시그모이드 함수를 통해 각 타임스텝의 아키텍처 분포 $\Gamma_t = h(z_t, \theta)$를 생성한다.

### 학습 목표 및 손실 함수

모델은 예측 오차와 계산 비용의 합을 최소화하는 방향으로 학습된다. 주어진 시퀀스 $(x, y)$에 대한 손실 함수 $L(x, y, \theta)$는 다음과 같다.

$$L(x,y,\theta) = \mathbb{E}_{\{A_t\}} \left[ \sum_{t=1}^{\#x} [\Delta(f(z_t, x_t, \theta, A_t), y_t) + \lambda C(A_t)] \right]$$

여기서 $\Delta$는 미분 가능한 예측 손실 함수이며, $C(A_t)$는 선택된 아키텍처 $A_t$의 계산 비용, $\lambda$는 성능과 비용 사이의 가중치를 조절하는 하이퍼파라미터이다.

### 학습 절차

아키텍처 샘플링 과정이 이산적(discrete)이므로 직접적인 역전파가 불가능하다. 따라서 본 논문은 몬테카를로(Monte-Carlo) 추정법을 이용한 정책 경사(Policy Gradient) 방식의 그래디언트 근사치를 사용하여 $\theta$와 아키텍처 분포를 동시에 업데이트한다.

## 📊 Results

### 실험 설정

- **데이터셋**: Speech Commands 데이터셋을 사용하며, 12개의 카테고리(yes, no, up, down, left, right, on, off, stop, go, bg-noise, unknown)로 분류하는 작업을 수행한다.
- **비교 대상**: 정적 모델인 `cnn-trad-fpool3` 구조와 이에 Shortcut 연결을 추가한 모델을 베이스라인으로 설정하였다.
- **지표**: 예측 정확도(Accuracy), 프레임당 연산량(FLOPs), 그리고 실시간 스트림에서의 매칭률(Matched %), 정답률(Correct %), 오답률(Wrong %), 오경보율(FA)을 측정하였다.

### 주요 결과

실험 결과, SANAS는 정적 모델보다 훨씬 적은 계산 비용으로 더 높은 성능을 보였다.

- **효율성**: SANAS 모델 중 하나는 프레임당 37.7M FLOPs만 사용하고도 86.5%의 단어 매칭률(Correct 80.7%)을 기록하였다. 반면 베이스라인 모델들은 124.6M 및 137.3M FLOPs를 사용했음에도 매칭률이 각각 81.7%, 82.9%에 그쳤다.
- **정확도**: SANAS가 베이스라인보다 절대적인 정확도가 높게 나타났는데, 이는 기본 구조에 Shortcut 연결을 추가하여 표현력을 높였기 때문이다. 이때 발생할 수 있는 과적합 문제는 손실 함수의 계산 비용 제약 항($\lambda C(A_t)$)이 규제 역할을 수행함으로써 방지되었다.
- **동적 행동 분석**: 학습 과정에서 모델은 배경 소음(bg-noise)이 포함된 프레임에서는 매우 작은 아키텍처를 선택하여 비용을 낮추고, 실제 단어가 포함된 프레임에서는 더 큰 아키텍처를 선택하여 정확도를 높이는 전략을 스스로 학습함을 확인하였다. 특히 'yes'와 같이 식별이 쉬운 단어는 상대적으로 적은 비용을 할당하였다.

## 🧠 Insights & Discussion

본 논문은 입력 데이터의 난이도에 따라 계산 자원을 유연하게 배분하는 것이 실시간 스트림 처리 시스템에서 매우 강력한 최적화 도구가 될 수 있음을 입증하였다. 특히, 고정된 모델을 사용하는 대신 적응형 모델을 사용함으로써 에지 디바이스(Edge Device)와 같이 전력과 연산 자원이 제한된 환경에서 실용적인 이점을 얻을 수 있다.

강점으로는 성능과 비용의 트레이드오프를 명시적인 손실 함수에 포함시켜 엔드-투-엔드(end-to-end)로 학습 가능하게 설계했다는 점이다. 다만, 본 연구에서 제안한 방식은 여전히 Super-Network라는 거대한 상위 구조를 정의해야 하며, 샘플링 기반의 학습으로 인해 수렴 속도나 분산 문제가 발생할 가능성이 있다. 또한, 본 논문에서는 FLOPs를 비용으로 정의하였으나, 실제 하드웨어에서의 전력 소비나 메모리 점유율은 FLOPs와 완전히 일치하지 않을 수 있다는 점이 한계로 남는다.

## 📌 TL;DR

본 논문은 Keyword Spotting을 위해 추론 시점에 신경망 구조를 동적으로 변경하는 **SANAS** 알고리즘을 제안하였다. 이 모델은 입력 신호가 단순하면 작은 네트워크를, 복잡하면 큰 네트워크를 사용하도록 스스로 학습하며, 이를 통해 **기존 정적 모델 대비 계산 비용(FLOPs)을 획기적으로 줄이면서도 오히려 더 높은 인식 정확도를 달성**하였다. 이 연구는 향후 전력 효율이 중요한 온디바이스 AI(On-device AI) 및 실시간 신호 처리 시스템의 아키텍처 설계에 중요한 방향성을 제시한다.
