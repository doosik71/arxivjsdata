# Fast ASR-free and almost zero-resource keyword spotting using DTW and CNNs for humanitarian monitoring

Raghav Menon, Herman Kamper, John Quinn, Thomas Niesler (2018)

## 🧩 Problem to Solve

본 논문은 데이터 자원이 극도로 부족한 언어 환경에서 효율적으로 작동하는 키워드 스포팅(Keyword Spotting, KWS) 시스템을 구축하는 것을 목표로 한다. 특히 아프리카 일부 지역의 인도주의적 구호 프로그램을 지원하기 위해, 전사 데이터(transcribed data)가 거의 없는 상황에서도 빠르게 배포 가능한 시스템이 필요하다.

기존의 키워드 스포팅 시스템은 주로 자동 음성 인식(Automatic Speech Recognition, ASR) 기술에 의존하는데, 이는 대량의 전사된 학습 데이터가 필수적이다. 하지만 자원이 부족한 언어(under-resourced languages)의 경우, 이러한 데이터를 구축하는 것이 현실적으로 불가능하거나 막대한 비용과 시간이 소요된다. 또한, 전사 데이터 없이 동작하는 Query-by-Example (QbyE) 방식의 Dynamic Time Warping (DTW) 시스템은 데이터 요구량은 적지만, 추론 속도가 매우 느려 실시간 모니터링에 부적합하다는 문제가 있다.

따라서 본 연구의 목표는 DTW의 낮은 데이터 의존성과 CNN의 빠른 추론 속도를 결합하여, 아주 적은 양의 표본 데이터만으로 학습 가능하며 실시간 처리가 가능한 ASR-free 키워드 스포팅 시스템을 개발하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **DTW를 CNN 학습을 위한 지도 신호(supervision signal)로 활용하는 것**이다.

구체적으로, 소량의 키워드 발화 데이터(seed corpus)와 대량의 라벨링되지 않은 음성 데이터를 사용하여 DTW를 통해 유사도 점수를 계산하고, 이 점수를 소프트 타겟(soft target)으로 삼아 CNN을 학습시킨다. 즉, DTW를 일종의 '교사 모델(teacher model)'로 사용하여 CNN이 DTW의 유사도 판단 기준을 학습하게 함으로써, 추론 단계에서는 DTW의 복잡한 정렬 과정 없이 CNN의 빠른 연산만으로 키워드를 탐지하도록 설계하였다.

## 📎 Related Works

본 논문에서 언급하는 관련 연구 및 기존 방식의 한계는 다음과 같다.

1.  **ASR 기반 키워드 스포팅**: ASR을 통해 생성된 래티스(lattice)를 검색하는 방식이다. 하지만 이는 타겟 도메인의 최소한의 전사 코퍼스가 필요하며, 자원이 부족한 환경에서는 적용이 불가능하다.
2.  **Query-by-Example (QbyE) 및 DTW**: 텍스트가 아닌 오디오 쿼리를 사용하여 검색하는 방식으로, DTW를 통해 쿼리와 검색 대상 간의 시간축을 정렬하여 유사도를 측정한다. 데이터 요구량은 매우 적지만, 모든 구간에 대해 정렬을 수행해야 하므로 계산 비용이 매우 높고 속도가 느리다.
3.  **신경망 기반 QbyE**: RNN이나 Siamese CNN 등을 사용하여 가변 길이의 음성을 고정 차원 벡터로 매핑하여 비교하는 방식이다. 하지만 이러한 신경망 기반 접근법들은 일반적으로 대량의 학습 데이터를 필요로 한다는 한계가 있다.

본 연구는 DTW의 데이터 효율성과 CNN의 연산 효율성을 결합함으로써, 대규모 학습 데이터 없이도 빠른 속도의 KWS를 구현했다는 점에서 기존 연구와 차별화된다.

## 🛠️ Methodology

### 1. 전체 파이프라인
시스템은 크게 두 단계로 구성된다. 첫째, 소량의 키워드 표본을 이용하여 라벨링되지 않은 대규모 코퍼스에 대한 DTW 유사도 점수를 생성한다. 둘째, 생성된 점수를 타겟으로 하여 CNN 분류기를 학습시킨다.

### 2. DTW 기반 타겟 생성 (Supervision)
특정 키워드 타입 $K$에 대해 $N$개의 반복 발화 표본 $K = (k_1, \dots, k_i, \dots, k_N)$이 있다고 가정한다. 여기서 각 $k_i$는 음성 특징 벡터의 시퀀스이다.

검색 대상이 되는 발화 $U$ 내에 키워드 $K$가 포함되었는지 확인하기 위해, 모든 표본 $k_i$와 $U$의 모든 부분 세그먼트 $u_p$ 사이의 DTW 비용을 계산하여 최솟값을 선택한다.

$$c = \min_{i \in 1 \dots N} \left[ \min_{u_p \in U} \text{DTW}\{k_i, u_p\} \right]$$

여기서 $\text{DTW}\{k_i, u_p\}$는 두 시퀀스 간의 정렬 비용이다. 본 연구에서는 프레임 간 비교를 위해 코사인 거리(cosine distance)를 사용하며, 결과 값 $c$의 범위는 $[0, 2]$이다.

CNN의 학습 타겟으로 사용하기 위해, 이 비용 $c$를 확률 값으로 해석 가능한 $y \in [0, 1]$ 범위로 정규화한다.

$$y_j = -\frac{1}{2} c_j + 1$$

이렇게 $L$개의 키워드 타입에 대해 계산된 정규화 점수들을 모아 하나의 타겟 벡터 $y = [y_1, \dots, y_L]$를 구성한다.

### 3. CNN 아키텍처 및 학습
CNN은 음성 특징(MFCC)을 입력받아 위에서 계산된 타겟 벡터 $y$를 예측하도록 학습된다.

-   **구조**: 다수의 Convolutional layers $\rightarrow$ Global Temporal Max-pooling layer $\rightarrow$ Fully Connected layers $\rightarrow$ Sigmoid activation layer.
-   **Global Temporal Max-pooling**: 시간 축에 대해 최대 활성화 값을 추출하여 입력 시퀀스의 길이에 관계없이 고정된 차원의 출력을 생성한다. 이는 발화 내에서 키워드의 존재 여부를 결정짓는 지배적인 특징을 추출하기 위함이다.
-   **활성화 함수**: 마지막 층을 제외한 모든 층에 Leaky ReLU ($\alpha=1/3$)를 사용하였다.
-   **손실 함수**: 타겟이 확률 형태로 정규화되었으므로, Summed Binary Cross-Entropy (BCE) 손실 함수를 사용하여 학습한다.

$$\ell = -\sum_{j=1}^{L} \{y_j \log \hat{y}_j + (1-y_j) \log [1-\hat{y}_j]\}$$

여기서 $\hat{y}_j$는 모델이 예측한 $j$번째 키워드의 존재 확률이다.

## 📊 Results

### 1. 실험 설정
-   **데이터셋**: South African Broadcast News (SABN) 코퍼스를 사용하였다 (총 23시간).
-   **학습 데이터**: 40개 키워드에 대해 24명의 화자가 각각 2회씩 발화한 총 1,920개의 고립된 키워드 데이터만 사용하였다.
-   **비교 대상 (Baselines)**:
    -   **DTW-QbyE**: 모든 표본의 DTW 점수를 평균하여 사용.
    -   **DTW-KS**: 모든 표본 중 최솟값(최적 점수)을 사용.
    -   **CNN**: 고립된 키워드 데이터만으로 직접 지도 학습한 CNN.
-   **평가 지표**: AUC (Area Under the ROC Curve), EER (Equal Error Rate), 실행 시간.

### 2. 정량적 결과
실험 결과, 절대적인 성능(AUC, EER)은 DTW-KS가 가장 높았으나, 제안된 CNN-DTW 방식이 그 뒤를 이었다.

| 시스템 | AUC (test) | EER (test) | 실행 시간 (min) |
| :--- | :---: | :---: | :---: |
| CNN (Supervised) | 0.5448 | 0.4771 | 55 |
| DTW-QbyE | 0.6612 | 0.3885 | 900 |
| DTW-KS | 0.7515 | 0.3162 | 900 |
| **CNN-DTW** | **0.6285** | **0.4161** | **15** |
| **CNN-DTW (+GNL)** | **0.6357** | **0.4092** | **25** |

-   **성능 향상**: 단순 지도 학습 CNN의 AUC(0.54)보다 CNN-DTW의 AUC(0.64)가 상당히 향상되었다.
-   **속도 이점**: CNN-DTW 시스템은 DTW 기반 시스템보다 약 180배 빠르다 (15분 vs 900분).

### 3. 정성적 및 개별 분석
-   **키워드별 성능**: 'Government', 'War', 'HIV'와 같이 빈도수가 높거나 발음이 뚜렷한 키워드에서 높은 성능을 보였다. 반면 'Wounded', 'Attack' 등의 키워드에서는 성능이 낮게 나타났다.
-   **일반화 능력**: 입력과 합성곱 층 사이에 Gaussian Noise Layer (GNL)를 추가했을 때 AUC가 약 1% 절대적으로 향상되어 일반화 성능이 개선됨을 확인하였다.

## 🧠 Insights & Discussion

본 논문은 자원이 극도로 부족한 환경에서 **'느리지만 정확한 교사(DTW)'**와 **'빠른 학생(CNN)'**의 관계를 이용하여 실용적인 KWS 시스템을 구축하였다.

**강점 및 시사점:**
-   전사 데이터 없이 소량의 오디오 샘플만으로 CNN을 학습시킬 수 있는 파이프라인을 제시하였다.
-   추론 속도를 획기적으로 개선하여 실시간 라디오 모니터링이라는 실제 응용 시나리오에 적합함을 입증하였다.
-   인도주의적 모니터링 환경에서는 인간 분석가가 최종 필터링을 수행하므로, 약간의 오탐(False Positive)이 있더라도 빠른 처리 속도가 더 큰 가치를 가진다는 실무적 관점을 잘 반영하였다.

**한계 및 논의사항:**
-   CNN-DTW의 성능이 DTW-KS(교사 모델)의 성능을 완전히 따라잡지는 못했다. 이는 DTW가 생성한 소프트 타겟에 노이즈가 포함되어 있거나, CNN이 DTW의 복잡한 정렬 메커니즘을 완벽하게 근사하지 못했기 때문으로 해석된다.
-   키워드별 발생 빈도에 따라 성능 차이가 크게 나타나는데, 이는 학습에 사용된 라벨링되지 않은 데이터 내의 키워드 분포가 불균형하기 때문으로 보인다.

## 📌 TL;DR

본 연구는 전사 데이터가 없는 저자원 언어 환경을 위해, **DTW로 생성한 유사도 점수를 타겟으로 CNN을 학습시키는 ASR-free 키워드 스포팅 방법**을 제안한다. 이 방식은 소량의 오디오 샘플만으로 학습 가능하며, DTW 대비 추론 속도를 **약 180배 향상**시켜 실시간 인도주의적 라디오 모니터링에 매우 적합한 솔루션을 제공한다.
