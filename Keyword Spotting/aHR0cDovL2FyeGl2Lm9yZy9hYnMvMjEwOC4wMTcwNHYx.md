# BIFOCAL NEURAL ASR: EXPLOITING KEYWORD SPOTTING FOR INFERENCE OPTIMIZATION

Jonathan Macoskey, Grant P. Strimel, Ariya Rastrow (2021)

## 🧩 Problem to Solve

본 논문은 온디바이스(On-device) 자동 음성 인식(Automatic Speech Recognition, ASR) 시스템에서 발생하는 추론 지연 시간(Inference latency) 문제를 해결하고자 한다. 최근 스마트 스피커와 스마트폰 등 하드웨어 자원이 제한된 환경에서 ASR을 구동하려는 수요가 증가함에 따라, 계산 복잡도를 낮추면서도 실시간성을 확보하는 것이 매우 중요하다.

특히, 음성 비서 서비스에서는 사용자가 호출어(Wake Word, WW)를 말하기 전과 그 순간까지의 오디오 구간인 '리드인(Lead-in)' 세그먼트가 존재한다. 시스템은 보통 호출어가 감지된 시점부터 ASR 디코딩을 시작하지만, 자연스러운 사용자 경험을 위해 버퍼에 저장된 리드인 구간을 함께 처리해야 한다. 이 리드인 구간은 전체 오디오의 약 30%를 차지하며, 이 백로그 데이터를 빠르게 처리하지 못하면 전체 추론 과정에서 지속적인 지연(Lag)이 발생하게 된다. 따라서 본 논문의 목표는 호출어 감지 신호를 활용하여 추론 경로를 동적으로 변경함으로써, 정확도를 유지하면서도 계산 비용을 효율적으로 낮추는 Bifocal RNN-T 아키텍처를 제안하는 것이다.

## ✨ Key Contributions

본 연구의 핵심 아이디어는 호출어(Wake Word)를 기점으로 음성 데이터의 특성이 달라진다는 점에 착안하여, 서로 다른 용량의 두 가지 인코더를 사용하는 'Bifocal(이초점)' 구조를 설계한 것이다.

1. **동적 추론 경로 설계**: 호출어 이전의 리드인 구간은 상대적으로 언어적 복잡도(Perplexity)가 낮으므로 가벼운 소형 인코더($F_s$)를 사용하고, 호출어 이후의 본문 구간은 복잡도가 높으므로 고성능의 대형 인코더($F_\ell$)를 사용하도록 설계하였다.
2. **Bifocal LSTM (BF-LSTM) 제안**: 서로 다른 차원을 가진 두 LSTM 셀 사이를 매끄럽게 전환하기 위해, 학습 가능한 상태 투영(State Projection) 메커니즘이 포함된 새로운 RNN 셀을 제안하였다.
3. **추론 최적화**: 인코더는 매 프레임마다 실행되어 병목 현상을 일으키는 주된 원인이 되는데, 이를 동적으로 제어함으로써 전체 계산 비용(FLOPs)을 획기적으로 줄였다.

## 📎 Related Works

기존의 온디바이스 ASR 최적화 연구는 주로 다음과 같은 방향으로 진행되었다.

- **모델 압축 및 양자화**: 32비트 가중치를 8비트나 4비트로 낮추는 양자화(Quantization), 저차원 행렬 분해(Low-rank matrix factorization), 네트워크 희소화(Sparsification) 등이 연구되었다.
- **아키텍처 변경**: 파라미터 수를 줄인 CIFG-LSTM이나 병렬 처리가 용이한 Simple Recurrent Units (SRU)와 같은 대안적 재귀 레이어가 제안되었다.
- **시간 축 감소**: Time-reduction 레이어를 통해 디코딩 프레임 레이트를 낮추어 연산량을 줄이는 방법이 사용되었다.

본 논문의 접근 방식은 이러한 정적인 모델 압축이나 단순한 구조 변경과 달리, 실행 시점(Runtime)에 입력 신호(Keyword Spotting 결과)에 따라 계산 경로를 동적으로 선택하는 '조건부 실행' 전략을 취한다는 점에서 차별점을 가진다.

## 🛠️ Methodology

### 전체 시스템 구조
본 모델은 RNN-T(Recurrent Neural Network Transducer) 구조를 기반으로 한다. RNN-T는 크게 세 가지 구성 요소로 이루어진다.
1. **Encoder (Transcription Network, $F$)**: 입력 오디오 특징 벡터 $x_{1:T}$를 처리하여 벡터 $h^{enc}_t$를 출력한다.
2. **Decoder (Prediction Network, $G$)**: 출력된 라벨 시퀀스 $y_{1:M}$을 처리하여 벡터 $h^{dec}_m$을 출력한다.
3. **Joint Network ($J$)**: 인코더와 디코더의 출력을 결합하여 다음 출력 라벨의 확률 분포를 생성한다.

Bifocal RNN-T는 여기서 인코더 $F$를 두 개의 경로($F_s, F_\ell$)로 분리하고, Keyword Spotter로부터 전달받은 프레임 인덱스(호출어 종료 시점)를 기준으로 어느 경로를 실행할지 결정한다.

### Bifocal LSTM (BF-LSTM)
BF-LSTM은 런타임에 추론 경로를 동적으로 변경할 수 있게 하는 학습 가능한 RNN 셀이다. 이 셀 내부에는 소형(small) 셀과 대형(large) 셀이 공존하며, 스위칭 변수 $z_t \in \{0, 1\}$에 의해 실행 경로가 결정된다. ($z_t=0$이면 소형, $z_t=1$이면 대형)

#### 1. 표준 LSTM 연산
소형 셀과 대형 셀은 각각 표준 LSTM 방정식을 따른다. (편의상 바이어스 생략)
- **소형 셀**:
$$f^s_t = \sigma(W^s_f x^s_t + U^s_f h^s_{t-1})$$
$$i^s_t = \sigma(W^s_i x^s_t + U^s_i h^s_{t-1})$$
$$o^s_t = \sigma(W^s_o x^s_t + U^s_o h^s_{t-1})$$
$$\tilde{c}^s_t = \tanh(W^s_c x^s_t + U^s_c h^s_{t-1})$$
$$c^s_t = f^s_t \circ c^s_{t-1} + i^s_t \circ \tilde{c}^s_t$$
$$h^s_t = o^s_t \circ \sigma(c^s_t)$$

- **대형 셀**: 위와 동일한 구조를 가지며, 가중치 행렬과 상태 벡터가 $s$ 대신 $\ell$로 표기된다.

#### 2. 상태 투영 (State Projections)
두 셀의 은닉 상태 차원이 서로 다르기 때문에, 경로 전환 시 상태를 변환해줄 학습 가능한 투영 행렬 $P$를 사용한다.
$$\hat{c}^\ell_t = P^s_c c^s_t, \quad \hat{c}^s_t = P^\ell_c c^\ell_t$$
$$\hat{h}^\ell_t = P^s_h h^s_t, \quad \hat{h}^s_t = P^\ell_h h^\ell_t$$

#### 3. 상태 업데이트 및 스위칭
스위칭 변수 $z_t$를 이용하여 최종 상태를 결정한다.
$$c^s_t := c^s_t(1-z_t) + \hat{c}^s_t z_t$$
$$c^\ell_t := \hat{c}^\ell_t z_t + c^\ell_t(1-z_t)$$
$$h^s_t := h^s_t(1-z_t) + \hat{h}^s_t z_t$$
$$h^\ell_t := \hat{h}^\ell_t z_t + h^\ell_t(1-z_t)$$

### Interleaving (Trifocal)
논문은 이 개념을 확장하여 3개 이상의 인코더를 사용하는 'Interleaving' 전략을 실험하였다. 예를 들어, 호출어 이후 구간에서도 대형 인코더($F^\ell_l$)와 소형 인코더($F^\ell_s$)를 정해진 스케줄(예: 2프레임 사용 후 교체)에 따라 번갈아 가며 사용하는 Trifocal 구조를 제안하였다.

## 📊 Results

### 실험 설정
- **데이터셋**: 42,000시간의 영어 가상 비서 작업 오디오 데이터. (호출어 포함 데이터만 사용)
- **특징 추출**: 64차원 Log-Filterbank Energies (LBFE), 3배 다운샘플링 및 스트라이드 2 적용 (최종 프레임 크기 30ms).
- **베이스라인 모델**: 5개 층의 LSTM 인코더(층당 1024 유닛), 2개 층의 LSTM 디코더(층당 1024 유닛). 총 파라미터 63.5M.
- **Bifocal 모델**: 소형 인코더(층당 256 유닛) $\to$ 대형 인코더(층당 1024 유닛) 구조.
- **평가 지표**: Word Error Rate (WER) 및 인코더의 부동 소수점 연산량 (FLOPs).

### 정량적 결과
| 모델 | WER (상대값) | 파라미터 (Encoder) | FLOPs | 비용 감소율 |
| :--- | :---: | :---: | :---: | :---: |
| Baseline | - | 42.7M | 11.1B | - |
| Baseline Small | +3.4% | 30.2M | 7.86B | 29.1% |
| **Bifocal** | **-1.9%** | **48.9M** | **7.86B** | **29.1%** |
| Bifocal (No Proj.) | +3.4% | 46.3M | 7.85B | 29.2% |
| Trifocal A | +23.6% | 55.1M | 4.74B | 57.3% |
| Trifocal B | +27.0% | 55.1M | 3.70B | 66.7% |
| Trifocal C | +28.1% | 55.1M | 3.47B | 68.7% |

### 결과 분석
1. **효율성 및 성능**: Bifocal 모델은 베이스라인보다 모델 크기는 약 10% 커졌지만, 연산량(FLOPs)은 29.1% 감소하였다. 특히, 동일한 연산량을 가진 `Baseline Small` 모델보다 WER이 5% 이상 우수하여, 성능 저하 없이 추론 비용을 낮출 수 있음을 증명하였다.
2. **상태 투영의 중요성**: 상태 투영을 제거한 `Bifocal (No Proj.)` 모델의 WER이 급격히 상승한 것을 통해, 서로 다른 인코더 간의 상태 전이가 예측 성능에 결정적인 역할을 함을 알 수 있다.
3. **Trifocal의 한계**: Trifocal 모델들은 연산량을 더욱 낮췄으나 WER이 매우 크게 상승하였다. 이는 임의의 시점에서 인코더를 스위칭하는 것이 모델의 예측 성능을 심각하게 저하시킴을 의미한다.

## 🧠 Insights & Discussion

본 논문은 호출어 감지라는 도메인 특화 지식을 신경망 아키텍처에 직접 통합하여 실질적인 추론 최적화를 달성하였다. 특히 다음의 통찰이 중요하다.

- **전략적 스위칭의 유효성**: Trifocal 실험 결과, 단순히 연산량을 줄이기 위해 빈번하게 스위칭하는 것보다, 리드인 구간과 본문 구간이라는 명확한 구분점을 기준으로 스위칭하는 Bifocal 설계가 훨씬 효과적이다. 이는 ASR 모델에서 어떤 구간에 계산 자원을 집중해야 하는지에 대한 가이드라인을 제공한다.
- **컴퓨팅 병목 지점의 정확한 타격**: 디코더는 캐싱이 가능하고 쿼리 횟수가 적은 반면, 인코더는 매 프레임 실행되어야 하므로 실제 지연 시간의 주범이다. 본 연구가 인코더의 동적 경로 선택에 집중한 것은 매우 실용적인 선택이었다.
- **한계점**: 제안된 방식은 호출어 감지기(KWS)의 정확도에 의존한다. 만약 KWS가 잘못된 시점에 신호를 준다면, 잘못된 인코더가 선택되어 WER이 상승할 가능성이 있다. 또한, 스위칭 스케줄이 고정되어 있어 이를 학습 가능하게 만들 수 있다면 더 높은 최적화 가능성이 있을 것으로 보인다.

## 📌 TL;DR

이 논문은 호출어(Wake Word) 감지 신호를 이용해 **소형 인코더(리드인 구간) $\to$ 대형 인코더(본문 구간)**로 동적 전환하는 **Bifocal RNN-T** 아키텍처를 제안한다. 이를 통해 정확도 손실 없이 인코더 추론 비용을 **약 29.1% 절감**하였으며, 이는 자원이 제한된 온디바이스 환경에서 실시간 음성 인식의 지연 시간을 줄이는 데 매우 중요한 기여를 한다.
