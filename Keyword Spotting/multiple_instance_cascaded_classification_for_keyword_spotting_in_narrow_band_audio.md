# Multiple-Instance, Cascaded Classification for Keyword Spotting in Narrow-Band Audio

Ahmad Abdulkader, Kareem Nassar, Mohamed Mahmoud, Daniel Galvez, Chetan Patil (2017)

## 🧩 Problem to Solve

본 논문은 좁은 대역폭(Narrow-Band, NB)의 8kHz 오디오 환경에서 특정 키워드를 탐지하는 Keyword Spotting (KWS) 문제를 해결하고자 한다. 특히, 이 시스템은 기업용 음성 비서인 'Eva'의 웨이크 워드(Wake word)인 "Okay Eva"를 인식하기 위해 설계되었다.

이 연구가 해결하려는 주요 문제는 다음과 같다:

1. **Narrow-Band 오디오의 한계**: 일반적인 광대역(Wide-band) 오디오와 달리, G.711과 같은 8kHz 샘플링 레이트의 NB 오디오는 정보 손실이 커서 인간과 자동 음성 인식 시스템 모두에서 정확도가 크게 떨어진다.
2. **Non-IID 환경**: 마이크 종류, 주변 환경, 화자, 소음 프로필이 극심하게 변하는 환경에서 입력 신호가 독립 동일 분포(IID)를 따르지 않아 모델의 일반화가 어렵다.
3. **극심한 클래스 불균형**: 실시간 스트림에서 키워드가 등장하는 빈도는 매우 낮다. 따라서 오탐지(False Positive)를 최소화하면서도 미탐지(False Negative)를 줄여야 하는 상충 관계(Trade-off)가 존재한다.
4. **자원 제약**: 계산 능력이 제한된 장치에서 동작해야 하므로 전력 소비를 줄이기 위한 효율적인 계산 구조가 필요하다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **다중 특징 표현(Multiple-Feature Representations)**, **계층적 분류기(Cascaded Classifiers)**, 그리고 **다중 인스턴스 학습(Multiple-Instance Learning, MIL)**을 결합하여 NB 오디오 환경에서도 강건한 KWS 시스템을 구축하는 것이다.

- **Multi-Representation Ensemble**: 서로 다른 특성을 가진 MFCC와 PLP 특징을 동시에 사용하여, 학습 데이터에서 보지 못한 새로운 소리가 키워드로 오인되는 'Novelty Detection' 문제를 억제한다.
- **Cascading Architecture**: 분류기를 계층적으로 배치하여 쉬운 음성 샘플은 빠르게 걸러내고(Early Termination), 후속 단계에서는 이전 단계에서 틀린 '어려운 부정 샘플(Hard Negative)'에 집중하게 함으로써 클래스 불균형 문제를 해결하고 전력 소모를 줄인다.
- **MIL 기반의 Noisy Or 구조**: 키워드의 발화 길이가 사람마다 다르다는 점에 착안하여, 단일 윈도우가 아닌 '백(Bag)' 단위로 학습하고 이를 Noisy Or 연산으로 통합하여 위치 불확실성을 해결한다.

## 📎 Related Works

기존의 KWS 시스템인 Google의 CNN 기반 모델이나 Apple의 DNN-HMM 결합 모델 등이 언급된다.

- **기존 연구의 한계**: Google과 Apple의 시스템은 하드웨어 제어권을 가지고 있어 고품질의 광대역(Wide-band) 오디오를 사용할 수 있었으나, 본 논문이 다루는 NB 오디오 환경에서는 성능 저하가 발생한다.
- **차별점**: 본 논문은 전력 효율을 위해 Apple과 유사하게 캐스케이드 구조를 채택하였으나, NB 오디오라는 제약 조건과 데이터 불균형을 해결하기 위해 다중 특징 표현과 MIL 기법을 구체적으로 통합했다는 점에서 차별성을 가진다.

## 🛠️ Methodology

### 1. 전체 파이프라인

시스템은 10ms마다 트리거되며, 과거 500ms의 오디오 윈도우(Example)를 처리한다. 전체 구조는 **특징 추출 $\rightarrow$ 계층적 분류기 $\rightarrow$ Noisy Or 통합** 순으로 진행된다.

### 2. 다중 특징 표현 (Multi-Representation)

학습 데이터에 없는 소리가 키워드로 오분류되는 것을 막기 위해 두 가지 특징 추출 방식을 사용한다.

- **MFCC (Mel Frequency Cepstral Coefficient)** 및 **PLP (Perceptual Linear Prediction)**: 두 특징 모두 30ms 프레임, 10ms 스트라이드(Stride)로 추출되어 500ms 윈도우 내에서 연결(Concatenation)된다.
- **앙상블 방법**: 각 특징으로 학습된 두 DNN 분류기의 출력 확률을 모델 평균화(Model-averaging)하여 최종 확률을 계산한다. 두 모델이 학습 데이터 영역에서는 일치하고 미학습 영역에서는 무작위로 동작하게 하여 오탐지율을 낮춘다.

### 3. 계층적 분류기 (Cascading)

계산 복잡도가 낮은 분류기부터 높은 분류기로 이어지는 구조를 가진다.

- **구조**: 총 3단계의 캐스케이드로 구성되며, 각 단계는 MFCC와 PLP 기반의 두 DNN(은닉층 2개, 각 128개 유닛)으로 이루어져 있다.
- **학습 절차**:
  - **1단계**: 긍정 샘플 대비 부정 샘플의 비율을 $1:100$으로 설정하여 대량의 부정 샘플을 학습시킨다.
  - **2, 3단계**: 이전 단계에서 키워드로 오분류된 'Hard Negative' 샘플을 집중적으로 학습한다. 이 단계들에서는 긍정 및 부정 샘플의 비율을 $1:2$로 맞추어 데이터 분포를 대칭적으로 만든다.
- **추론**: 각 단계의 임계값(Threshold)을 넘지 못하면 즉시 계산을 종료(Early Termination)하여 전력을 절약한다.

### 4. 다중 인스턴스 학습 (Multiple-Instance Learning, MIL)

키워드의 발화 길이가 $300\text{ms}$에서 $900\text{ms}$까지 다양하므로, 단일 윈도우가 아닌 윈도우들의 집합인 '백(Bag)' 단위로 접근한다.

- **정의**:
  - **Positive Bag**: 백 내의 윈도우 중 적어도 하나가 긍정(Keyword)인 경우.
  - **Negative Bag**: 백 내의 모든 윈도우가 부정(Non-keyword)인 경우.
- **Noisy Or 연산**: 백 내의 모든 윈도우 출력값을 "Noisy Or" 방식으로 결합하여 최종 트리거 여부를 결정한다.

## 📊 Results

### 1. 실험 설정

- **데이터셋**: 200명의 화자로부터 수집한 19k 개의 긍정 샘플과 다양한 회의 녹음 데이터에서 추출한 부정 샘플을 사용하였다. 데이터는 화자 기준으로 8:2로 분할하여 학습과 평가에 사용하였다.
- **평가 지표**: 시간당 오탐지율(Hourly False Positive Rate, FPR)과 미탐지율(False Negative Rate, FNR)의 관계를 통해 성능을 측정하였다.

### 2. 주요 결과

- **다중 특징 표현의 효과**: 단일 특징(MFCC 또는 PLP만 사용)을 사용했을 때보다 두 특징을 모두 사용한 앙상블 모델의 FPR이 현저히 낮았다. 예를 들어, $\text{FNR} \approx 7\%$ 지점에서 MFCC는 시간당 $\text{FPR } 1.2$, PLP는 $1.0$이었으나, 다중 표현 모델은 $0.55$를 기록하였다.
- **캐스케이드 단계의 효과**: 분류기의 단계가 증가할수록 ROC 곡선이 개선되었으며, 이는 단계별로 Hard Negative를 학습하는 전략이 유효했음을 보여준다.
- **최종 성능**: 시간당 $\text{FPR } 0.75$에서 $\text{FNR } 6\%$를 달성하였다. 또한 5% FNR 기준, FPR을 8회에서 1.2회로 약 85% 감소시켰다.

## 🧠 Insights & Discussion

본 연구는 실제 상용 서비스(Eva)에서 마주하는 제약 조건인 **저대역폭 오디오, 전력 제한, 극심한 클래스 불균형**을 공학적인 파이프라인 설계(Cascading + Ensemble + MIL)로 효과적으로 해결하였다.

**강점**:

- 단순히 모델의 파라미터를 늘리는 대신, 데이터의 분포 특성(Hard Negative)과 도메인 특성(NB 오디오)을 고려한 구조적 접근을 취하였다.
- 특히 다중 특징 표현을 통해 'Novelty Detection' 문제를 완화한 점은 실용적인 관점에서 매우 중요한 통찰이다.

**한계 및 논의**:

- 캐스케이드 구조에서 하드 샘플을 추출하는 과정이 단계가 진행될수록 더 많은 시간이 소요된다는 점이 명시되어 있으며, 이는 학습 비용의 증가를 의미한다.
- 본문에서 Google의 시스템과 비교하며 성능이 더 좋다고 주장하지만, 사용한 데이터셋이 다르기 때문에 직접적인 비교에는 한계가 있음을 스스로 인정하고 있다.

## 📌 TL;DR

이 논문은 8kHz 저대역폭 오디오 환경에서 전력 효율과 정확도를 동시에 잡기 위한 KWS 시스템을 제안한다. **MFCC/PLP 앙상블**로 오탐지를 줄이고, **3단계 캐스케이드 구조**로 연산 효율과 클래스 불균형을 해결하며, **MIL의 Noisy Or** 구조로 발화 길이의 가변성을 처리하였다. 결과적으로 매우 낮은 시간당 오탐지율(0.75회/hr)을 달성하였으며, 이는 자원 제약이 심한 환경에서의 음성 인터페이스 구현에 중요한 참고 자료가 될 수 있다.
