# Hardware Aware Training for Efficient Keyword Spotting on General Purpose and Specialized Hardware

Peter Blouw, Gurshaant Malik, Benjamin Morcos, Aaron R. Voelker, Chris Eliasmith (2021)

## 🧩 Problem to Solve

본 논문은 모바일, 웨어러블, 자동차 등 다양한 엣지 디바이스에서 필수적인 사용자 인터페이스인 키워드 검출(Keyword Spotting, KWS) 시스템의 효율성을 최적화하는 문제를 해결하고자 한다. KWS 시스템은 특성상 '항상 켜져 있는(Always-on)' 상태를 유지해야 하므로, 높은 정확도를 유지하면서도 전력 소모를 극도로 낮추는 것이 상용화의 핵심이다.

특히 저자는 기존 연구들이 간과하는 실제 배포 환경의 네 가지 제약 조건을 강조한다.

1. **상태 유지(Stateful):** 키워드가 언제 나타날지 알 수 없으므로, 네트워크 상태가 특정 시점에 '0'으로 리셋되지 않고 이전 처리 결과가 유지되어야 한다.
2. **온라인/스트리밍(Online/Streaming):** 지연 시간을 최소화하기 위해 오디오 데이터를 버퍼링하지 않고 실시간으로 처리해야 한다.
3. **양자화(Quantized):** 메모리 풋프린트를 줄이기 위해 8비트 이하의 가중치와 활성화 함수를 사용하는 양자화가 필수적이다.
4. **전력 효율성(Power Efficiency):** 단순한 양자화를 넘어 연산 횟수와 메모리 액세스를 최적화하여 실제 하드웨어에서의 전력 소모를 줄여야 한다.

따라서 본 논문의 목표는 위 제약 조건을 모두 만족하면서도 정확도, 모델 크기, 전력 소모 측면에서 최첨단(State-of-the-art, SotA) 성능을 달성하는 KWS 네트워크와 하드웨어 구현 방법을 제시하는 것이다.

## ✨ Key Contributions

본 연구의 핵심 아이디어는 **Legendre Memory Unit (LMU)** 기반의 신경망 구조와 **Hardware Aware Training (HAT)** 기법을 결합하여 하드웨어 제약 사항을 학습 단계에서부터 직접 반영하는 것이다.

LMU는 입력 시계열 데이터를 압축하는 데 최적화된 선형 메모리 층을 가지고 있어, 적은 파라미터만으로도 긴 시간 범위의 정보를 효율적으로 기억할 수 있다. 여기에 HAT를 적용하여 모델의 비트 정밀도를 실제 배포될 하드웨어 사양과 정확히 일치시켜 학습함으로써, 양자화로 인한 성능 저하를 최소화하고 실제 하드웨어에서 예측 가능한 정확도를 확보하였다.

## 📎 Related Works

저자는 Google, ARM, DarwinAI의 최신 KWS 결과물들을 비교 대상으로 삼는다.

- **Google 및 ARM의 접근 방식:** HAT와 유사하게 양자화 및 스트리밍 제약을 고려하여 모델을 설계하였다.
- **DarwinAI (TinySpeech):** 모델 크기를 매우 작게 줄였으나, 저자는 TinySpeech 모델들이 상태를 유지하지 않는(Stateless) 방식이며 스트리밍이 불가능하여 실시간 배포에 부적절하다고 지적한다.
- **기존 RNN/CNN 모델:** LSTMs, GRUs, CNN 기반 모델들이 널리 사용되지만, LMU에 비해 파라미터 효율성이 떨어지거나(RNN 계열), 실시간 스트리밍 처리 시 지연 시간이 발생하는(CNN 계열) 한계가 있다.

## 🛠️ Methodology

### 1. LMU 기반 아키텍처

본 논문에서 사용된 LMU는 입력 시계열의 압축에 최적화된 선형 메모리 층과 비선형 출력 층으로 구성된다. 저자는 KWS 작업의 성능을 높이기 위해 기존 LMU 구조를 다음과 같이 수정하였다.

- 비선형 층에서 선형 층으로의 연결 제거
- 선형 층에서 중간 입력 $u_t$로의 연결 제거
- 비선형 층의 자기 회귀 연결(Recurrent connection) 제거
- 여러 개의 선형 메모리 층을 배치하고 그 출력을 연결(Concatenate)하여 비선형 층으로 전달

최종적인 단일 LMU 층의 동작은 다음 방정식으로 설명된다.
$$h_t = f(W_x x_t + W_m m_t + b)$$
$$u_t = e^T x_t + e^T h_{t-1}$$
$$m_t = \bar{A} m_{t-1} + \bar{B} u_t$$

여기서 $f$는 ReLU 활성화 함수이며, $m_t$는 메모리 상태를, $h_t$는 출력 상태를 나타낸다. 최종 네트워크는 이러한 LMU 층을 여러 개 쌓고 마지막에 피드포워드 출력 층을 둔 구조이다.

### 2. Hardware Aware Training (HAT) 및 최적화

- **HAT:** 일반적인 양자화 인식 학습(Quantization Aware Training)을 확장하여, 하드웨어에서 실제로 사용되는 비트 정밀도(예: 가중치 4/8비트, 활성화 7비트)를 학습 과정에 정밀하게 반영한다.
- **Pruning (가지치기):** TensorFlow Model Optimization 라이브러리의 `PrunableLayer`를 사용하여 가중치를 제거하였다. LMU-3는 80%, LMU-4는 91%의 프루닝을 적용하였으며, 프루닝 후에는 HAT와 병행하여 미세 조정(Fine-tuning)을 진행하였다.

### 3. 하드웨어 구현

LMU-2 모델을 위해 전용 저전력 디지털 하드웨어를 설계하였다. 22nm 공정을 기준으로 MAC(Multiply-Accumulate) 및 SRAM의 정적/동적 전력 소모를 모델링하였으며, 실시간 제약 조건(40ms 윈도우를 20ms마다 업데이트)을 만족하는 최적의 클록 주파수와 자원 배치를 탐색하였다.

## 📊 Results

### 1. 실험 설정

- **데이터셋:** Speech-Commands 데이터셋 (16kHz 오디오 $\rightarrow$ MFCC 변환).
- **클래스:** 10개의 키워드, 침묵(Silence), 알 수 없음(Unknown)을 포함한 총 12개 라벨.
- **지표:** 정확도(Accuracy), 모델 크기(kbits, 파라미터 수 $\times$ 비트 수), 전력 소모($\mu$W).

### 2. 모델 성능 비교 (정량적 결과)

LMU 모델들은 모델 크기 대비 매우 높은 정확도를 보였다.

- **LMU-1:** Google의 최적 모델과 동일한 정확도를 보이면서도 모델 크기는 41% 더 작다.
- **LMU-2:** CNN 기반 모델과 유사한 정확도를 내면서 모델 크기는 11.7배 더 작다.
- **TinySpeech 대비:** LMU-3는 TinySpeech-A보다 에러를 14% 줄이면서 크기는 17% 더 작고, LMU-4는 TinySpeech-B보다 에러를 19% 줄이면서 크기는 8% 더 작다.

### 3. 하드웨어 전력 효율성

- **전용 하드웨어 결과:** 최적 설계 시 $8.79\mu\text{W}$ (92kHz 클록, 약 8M 트랜지스터)의 전력 소모를 달성하였다.
- **범용 하드웨어 대비:** ARM M4F에서 구동 시 $212\mu\text{W}$가 소모되므로, 전용 하드웨어가 약 24배 더 효율적이다. (이상적인 M4F 기준으로는 14배 효율적)
- **특수 목적 ASIC 대비:** Syntiant NDP10x ($170\mu\text{W}$) 대비 16~19배 더 전력 효율적이며, 정확도 또한 $95.9\%$로 더 높다.

## 🧠 Insights & Discussion

본 논문은 하드웨어와 소프트웨어의 공동 설계(Co-design)가 초저전력 엣지 AI 구현에 얼마나 중요한지를 입증하였다. 특히 LMU의 파라미터 효율성은 매우 강력한 강점으로 작용한다. LMU-2는 Syntiant의 네트워크보다 파라미터 수가 12배 적음에도 더 높은 정확도를 기록하였다.

저자는 이러한 파라미터 효율성이 결국 전용 하드웨어의 필요성을 낮출 수 있다고 주장한다. 실제로 LMU-2를 범용 ARM M4F에서 구동했을 때의 전력 소모($212\mu\text{W}$)가 Syntiant의 전용 칩($170\mu\text{W}$)과 큰 차이가 없으며, 더 작은 LMU-3를 사용한다면 범용 하드웨어에서도 특수 칩 수준의 전력 효율을 달성하면서 더 높은 정확도를 얻을 수 있을 가능성이 크다.

다만, 본 논문에서 제시한 전용 하드웨어의 전력 소모 결과는 실제 실리콘 측정값이 아닌 ABR의 독자적인 모델링 툴을 이용한 추정치라는 점을 유의해야 한다.

## 📌 TL;DR

본 연구는 LMU(Legendre Memory Unit) 구조와 HAT(Hardware Aware Training)를 결합하여, 실시간 스트리밍과 상태 유지가 가능하면서도 극도로 작고 정확한 KWS 모델을 제안하였다. 결과적으로 범용 하드웨어(ARM M4F) 및 전용 하드웨어 모두에서 기존 SotA 모델들보다 우수한 정확도와 전력 효율성을 달성하였으며, 특히 전용 하드웨어 설계 시 전력 소모를 $8.79\mu\text{W}$까지 낮추어 기존 ASIC 대비 최대 19배의 효율 향상을 이루었다. 이는 향후 초저전력 상시 대기형 음성 인식 시스템 구현에 핵심적인 역할을 할 것으로 기대된다.
