# Boosting keyword spotting through on-device learnable user speech characteristics

Cristian Cioflan, Lukas Cavigelli, and Luca Benini (2024)

## 🧩 Problem to Solve

본 논문은 항상 켜져 있는(always-on) TinyML 환경의 키워드 검출(Keyword Spotting, KWS) 시스템이 직면한 **사용자 간 음성 특성 차이로 인한 정확도 저하 문제**를 해결하고자 한다. 

오프라인에서 학습된 분류기는 배포 후 처음 접하는 사용자의 음성 특성(피치, 억양, 성별, 억양 등)이나 주변 환경 변화로 인해 성능이 하락하는 도메인 시프트(domain shift) 문제가 발생한다. 이를 해결하기 위해 온디바이스 학습(on-device learning)을 통한 튜닝이 필요하지만, 기존의 방식들은 다음과 같은 한계를 가진다.

1. **자원 제약**: 배터리로 구동되는 마이크로컨트롤러(MCU) 환경에서 기존의 백본(backbone) 업데이트 방식은 계산량(FLOPs)이 너무 많고 메모리 소모가 극심하여 부적합하다.
2. **데이터 부족**: 실제 환경에서 특정 사용자의 음성 데이터를 대량으로 확보하여 모델을 미세 조정(fine-tuning)하는 것은 현실적으로 어렵다.
3. **치명적 망각(Catastrophic Forgetting)**: 데이터가 부족한 상황에서 전체 모델을 업데이트할 경우, 기존에 학습된 일반적인 특징을 잃어버릴 위험이 있다.

따라서 본 연구의 목표는 **극소량의 데이터만으로도 적은 자원을 사용하여 특정 사용자의 음성 특성에 빠르게 적응할 수 있는 효율적인 온디바이스 학습 아키텍처를 설계**하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **사전 학습된 고정 백본(frozen backbone)과 학습 가능한 사용자 임베딩(learnable user embedding)을 결합한 하이브리드 구조**를 제안하는 것이다.

핵심 직관은 모든 사용자가 공유하는 일반적인 음성 특징은 고정된 백본을 통해 추출하고, 사용자 개별의 고유한 특성만을 별도의 저차원 벡터(embedding)로 학습하여 이를 융합하는 것이다. 이렇게 하면 전체 모델을 재학습시킬 필요 없이 **사용자 임베딩 레이어만 업데이트**함으로써 계산 비용을 획기적으로 줄이면서도 개인화된 성능 향상을 달성할 수 있다.

## 📎 Related Works

### 기존 연구 및 한계
1. **KWS 아키텍처**: CNN, RNN, Transformer 등이 사용되었으며, 특히 TinyML을 위해 Depthwise Separable CNN(DS-CNN)이 효율적인 대안으로 제시되었다.
2. **화자 적응형 오디오 시스템(Speaker-adaptive systems)**: i-vector, x-vector와 같은 벡터 임베딩을 입력 특징에 추가하는 방식이 연구되었다. 그러나 이는 별도의 사전 학습된 DNN을 통해 벡터를 생성해야 하므로 추론 지연 시간(latency)이 증가하여 실시간성이 중요한 엣지 디바이스에는 부적합하다.
3. **백본 미세 조정(Backbone refinement)**: 모델 자체를 사용자의 음성 도메인에 맞게 수정하는 방식이 있으나, 이는 MCU에서 감당하기 어려운 메모리 용량을 요구하며 학습을 위해 수 분에서 수십 분의 레이블링된 데이터가 필요하다는 단점이 있다.

### 차별점
제안 방법은 백본을 완전히 고정(freeze)하여 치명적 망각을 방지하고, 추론 지연 시간을 늘리는 외부 벡터 추출기 대신 내부의 가벼운 **룩업 테이블(look-up table) 형태의 임베딩 레이어**를 사용하여 온디바이스 학습의 효율성을 극대화하였다.

## 🛠️ Methodology

### 전체 시스템 구조
시스템은 크게 두 개의 경로로 구성된다.
1. **Frozen Backbone**: 오디오 입력에서 일반적인 키워드 특징을 추출한다.
2. **User Embedding Layer**: 사용자 ID를 입력받아 해당 사용자의 고유 특성을 나타내는 $n$차원 벡터를 출력하는 룩업 테이블이다.

### 상세 구성 및 절차
- **백본 구조**: DS-CNN을 기반으로 하며, $10 \times 4$ 커널의 2D Convolution 레이어로 시작하여 Depthwise 및 Pointwise Convolution 블록으로 이어진다.
- **사용자 임베딩**: 각 사용자는 고유한 ID를 가지며, 이 ID는 임베딩 행렬의 인덱스로 작용하여 특정 벡터를 추출한다. 새로운 사용자가 추가되면 임베딩 행렬에 새로운 엔트리를 추가하기만 하면 된다.
- **특징 융합(Feature Fusion)**: 백본에서 추출된 잠재 표현(latent representation)과 사용자 임베딩 벡터를 분류기(Classifier) 직전에 융합한다. 본 논문에서는 다음의 융합 연산자를 실험하였다.
    - 덧셈(Addition)
    - 곱셈(Multiplication)
    - 연결(Concatenation): 백본 호환(BC) 및 분류기 호환(CC) 방식 두 가지로 구분하여 구현하였다.
- **학습 절차**:
    1. **사전 학습(Pre-training)**: Google Speech Commands(GSC) 데이터셋을 사용하여 백본과 임베딩 레이어를 함께 학습시킨다.
    2. **온디바이스 적응(Online Learning, OL)**: 모델 배포 후, 백본과 FC 레이어는 고정하고 **사용자 임베딩 레이어의 가중치만** 업데이트한다.
    3. **손실 함수**: 일반적인 Cross-Entropy (CE) loss를 사용하여 새로운 사용자의 음성 특성을 학습한다.

## 📊 Results

### 실험 설정
- **데이터셋**: Google Speech Commands (GSC 10 및 GSC 35 클래스 문제)
- **평가 지표**: 에러율(Error Rate, %)
- **비교 대상**: 사전 학습 모델(Baseline), 사용자 임베딩만 업데이트, 백본만 업데이트, 전체 모델 업데이트

### 주요 결과
1. **사용자 정보의 효과**: 사용자 임베딩을 추가했을 때 에러율이 감소하였다. 특히 **곱셈(multiplicative) 융합** 방식이 가장 낮은 에러율을 기록하였다.
2. **Few-shot 학습 능력**: 클래스당 단 **4개의 샘플**만으로도 효과적인 학습이 가능함을 보였다. GSC 35 문제에서 사전 학습 베이스라인 대비 에러율을 최대 19%까지 감소시켜 $30.1\%$에서 $24.3\%$로 낮추었다.
3. **온디바이스 학습 효율성 (DS-CNN Small 모델 기준)**:
    - **계산량**: 전체 학습 대비 사용자 임베딩 업데이트의 FLOPs가 약 340배 적다.
    - **메모리**: 피크 메모리 사용량이 $16\text{kB}$ 미만으로, 전체 학습 시 필요한 수 $\text{MB}$에 비해 획기적으로 낮다.
    - **에너지**: 에포크당 약 $13\mu\text{J}$의 매우 낮은 에너지 소비를 기록하였다.

### 정량적 비교 요약
| 업데이트 대상 | 에러 감소율 (GSC 10) | 계산 비용 (FLOPs) | 메모리 소모 |
| :--- | :---: | :---: | :---: |
| 전체 학습 (Full) | 최고 | 매우 높음 | 매우 높음 |
| 분류기만 업데이트 | 낮음 | 매우 낮음 | 매우 낮음 |
| **사용자 임베딩만 업데이트** | **높음 (Full의 78% 수준)** | **낮음** | **매우 낮음** |

## 🧠 Insights & Discussion

### 강점 및 해석
본 연구는 정확도와 자원 소모 사이의 최적의 트레이드-오프(trade-off) 지점을 찾아냈다. 전체 모델을 학습시키는 것은 성능은 가장 좋지만 TinyML 환경에서 불가능하며, 단순히 분류기만 학습시키는 것은 자원은 적게 들지만 성능 향상이 미미하다. 반면, **사용자 임베딩만 학습시키는 방식은 전체 학습 성능의 약 78%를 달성하면서도 자원 소모는 극도로 낮추었다**는 점에서 실용성이 매우 높다.

### 한계 및 논의사항
- **사용자 식별 전제**: 본 시스템은 입력되는 음성이 어떤 사용자의 것인지 식별할 수 있는 ID가 제공된다는 가정을 전제로 한다. 실제 환경에서 여러 사용자가 섞여 있을 때 이를 자동으로 식별하여 적절한 임베딩을 적용하는 메커니즘에 대한 추가 연구가 필요할 것으로 보인다.
- **데이터 분포**: 클래스당 샘플 수가 극히 적은(4개 미만) 환경에서의 강건성에 대해서는 명확히 제시되지 않았다.

## 📌 TL;DR

본 논문은 TinyML 기반 키워드 검출 시스템을 위해 **고정된 백본과 학습 가능한 사용자 임베딩을 결합한 가벼운 온디바이스 학습 아키텍처**를 제안한다. 사용자 고유의 음성 특성을 저차원 벡터로 학습하여 융합함으로써, **클래스당 4개의 샘플만으로도 에러율을 최대 19% 낮추는 성능 향상**을 이루었으며, 메모리 사용량을 $16\text{kB}$ 이하로 억제하여 초저전력 MCU에서도 구현 가능하다는 것을 입증하였다. 이는 향후 개인 맞춤형 AI 음성 인터페이스를 엣지 디바이스에 구현하는 데 핵심적인 역할을 할 것으로 기대된다.
