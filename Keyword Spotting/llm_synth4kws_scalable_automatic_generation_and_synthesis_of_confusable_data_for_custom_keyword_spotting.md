# LLM-Synth4KWS: Scalable Automatic Generation and Synthesis of Confusable Data for Custom Keyword Spotting

Pai Zhu, Quan Wang, Dhruuv Agarwal, Kurt Partridge (2025)

## 🧩 Problem to Solve

본 논문은 사용자가 정의한 키워드를 감지하는 Custom Keyword Spotting (KWS) 시스템이 직면한 '혼동하기 쉬운 키워드(confusable keywords)' 식별 문제에 집중한다. Custom KWS는 일반적으로 음성 등록(enrollment) 데이터와 입력 오디오의 embedding을 비교하여 작동하며, 최신 모델들은 contrastive learning을 통해 학습된다.

그러나 기존의 contrastive learning 방식은 학습 데이터셋에서 키워드를 무작위로 샘플링하기 때문에, "blue"와 "glue"처럼 발음이 매우 유사한 키워드들이 동일한 학습 배치(batch)에 포함될 확률이 매우 낮다. 이로 인해 모델은 서로 확연히 다른 발음은 잘 구분하지만, 유사한 발음을 가진 키워드들 사이에서는 높은 False Accept (FA) 비율을 보이는 한계가 있다. 기존에 Levenshtein distance 등을 이용하여 유사 단어를 검색하는 방법이 제안되었으나, 이는 학습 시 계산 복잡도가 $O(N)$으로 증가하여 학습 속도를 저하시키거나, 룩업 테이블(lookup table) 유지로 인해 메모리 오버헤드가 발생하는 등 확장성(scalability) 문제가 존재한다.

따라서 본 연구의 목표는 대규모 언어 모델(LLM)과 텍스트-음성 변환(TTS) 엔진을 활용하여, 인적 자원 소모 없이 확장 가능하게 혼동 가능한 데이터를 자동으로 생성하고 이를 통해 Custom KWS의 성능을 향상시키는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 LLM의 텍스트 생성 능력과 TTS의 고품질 음성 합성 능력을 결합하여, 모델이 학습 과정에서 명시적으로 '어려운 예제(hard negatives)'를 학습하도록 유도하는 것이다. 구체적인 기여 사항은 다음과 같다.

1. **c-AUC (confusable-AUC) 지표 제안**: 기존 지표들이 혼동 가능한 키워드 간의 구분 능력을 충분히 반영하지 못한다는 점에 착안하여, 모음 그룹(vowel group) 기반의 새로운 북스타 지표인 c-AUC를 정의하였다.
2. **자동화된 데이터 증강 프레임워크**: LLM 프롬프팅을 통해 모음 기반의 혼동 가능한 키워드 그룹을 생성하고, TTS를 통해 다양한 억양과 운율이 반영된 음성 데이터를 합성하는 확장 가능한 파이프라인을 설계하였다.
3. **효율적인 학습 스키마**: GE2E(Generalized End-to-End) 배치를 수정하여 모음 그룹 기반의 예제들을 샘플링함으로써, 혼동 가능한 발화 간의 contrastive loss를 최적화하는 학습 구조를 제안하였다.

## 📎 Related Works

기존의 Custom KWS 연구들은 주로 Triplet loss나 GE2E loss와 같은 representation learning 기법을 사용하여 고정 길이의 벡터(embedding)를 생성하고 코사인 유사도를 측정하는 방식을 취했다.

혼동 가능한 예제를 처리하기 위한 이전 연구들은 다음과 같은 한계를 보였다.

- **검색 기반 방식**: Levenshtein distance 등을 이용해 학습 데이터 내에서 유사 단어를 찾는 방식은 학습 시간이 지나치게 길어지거나 메모리 사용량이 과도하여 대규모 데이터셋에 적용하기 어렵다.
- **TTS 기반 증강 방식**: 일부 연구에서 TTS를 사용했으나, 수작업으로 만든 소수의 혼동 단어 리스트(예: 중국어 12개 단어)에 의존했다. 이는 단어의 다양성이 부족하여 모델의 일반화 성능을 높이는 데 한계가 있었다.

본 논문은 LLM을 통해 방대한 양의 혼동 단어 리스트를 자동으로 생성하고, 이를 최신 TTS 엔진(Virtuoso)으로 합성함으로써 데이터의 다양성과 생성 효율성을 동시에 해결하며 기존 방식과 차별화된다.

## 🛠️ Methodology

### 전체 파이프라인 및 시스템 구조

전체 시스템은 [LLM을 통한 키워드 생성] $\rightarrow$ [TTS를 통한 음성 합성] $\rightarrow$ [GE2E 기반 contrastive 학습]의 순서로 구성된다.

### 1. Baseline 모델 및 GE2E Loss

베이스라인으로는 항상 켜져 있는(always-on) 기기에 최적화된 420KB 크기의 양자화된 Conformer 모델을 사용한다. 학습에는 GE2E loss를 사용하며, 구체적인 배치 구성은 다음과 같다.

- 배치 내에 $X$개의 구절(phrase)을 선택하고, 각 구절당 $Y$개의 발화를 샘플링한다.
- $Y/2$개의 발화는 등록(enrollment) embedding의 중심점(centroid)을 만드는 데 사용된다.
- 나머지 $Y/2$개의 발화는 테스트 발화로서, 등록 중심점과의 코사인 유사도를 통해 비교된다.

### 2. LLM 기반 혼동 키워드 그룹 생성

영어의 20가지 모음을 기준으로, 각 모음이 포함된 단순하고 구분 가능한 단어 100개를 생성하도록 Gemini 1.5 Pro에 프롬프팅한다. 예를 들어 $[u:]$ 모음 그룹을 위해 "Can you generate a group of words? The group should have 100 simple but distinguishable words. All words in the group should contain the vowel $[u:]$"와 같은 프롬프트를 사용한다.

### 3. TTS 스타일 샘플링

생성된 키워드들을 Virtuoso TTS 엔진을 통해 음성으로 변환한다. 이때 데이터의 다양성을 확보하기 위해 다음과 같은 설정을 적용한다.

- **화자(Speaker)**: 다양한 인구통계학적 특성과 억양을 가진 726명의 화자 샘플링.
- **운율(Prosody)**: 문장 부호를 이용해 제어되는 5가지의 운율 스타일 적용.
- 각 키워드당 100개의 발화를 생성하여 증강 데이터셋을 구축한다.

### 4. 학습 절차 및 데이터 구성

학습 시 각 배치는 다음 두 가지 데이터 소스에서 확률적으로 선택된다.

- **실제 데이터**: MSWC (Multilingual Spoken Words Corpus) 데이터셋 ($P_{mswc} = 0.5$)
- **합성 데이터**: LLM/TTS로 생성된 혼동 데이터셋 ($P_{tts} = 0.5$)
합성 데이터 배치를 구성할 때는 동일한 모음 그룹 내에서 $X=8$개의 키워드를 무작위로 선택하고 키워드당 $Y=10$개의 발화를 샘플링하여, 모델이 매우 유사한 발음 간의 차이를 학습하도록 강제한다.

## 📊 Results

### 실험 설정

- **데이터셋**: Speech Commands (테스트), LibriPhrase-1s (테스트)
- **기준선**: MSWC 데이터만으로 학습한 Baseline 모델
- **평가 지표**:
  - $\text{EER}$ (Equal Error Rate): FAR과 FRR이 같아지는 지점의 값.
  - $\text{AUC}$ (Area Under the Curve): DET 곡선 하단 면적.
  - $\text{c-AUC}$: 모음 그룹별 AUC의 평균값 (혼동 가능 키워드 전용 지표).
  - $\text{Hard-AUC}$: LibriPhrase-1s에서 발음이 유사한 hard negative pair에 대한 AUC.

### 정량적 결과

Speech Commands 데이터셋에서의 결과는 다음과 같다.

| 모델 | EER (%) | AUC (%) | c-AUC (%) |
| :--- | :---: | :---: | :---: |
| Baseline | 2.49 | 0.490 | 0.915 |
| AugModel | 2.38 | 0.472 | 0.812 |
| **상대적 개선도** | **-4.4%** | **-3.7%** | **-11.3%** |

LibriPhrase-1s에서는 Hard-AUC가 $14.4\%$에서 $12.6\%$로 약 $12.5\%$ 상대적으로 개선되었다.

### 분석 결과

- **전반적 성능**: c-AUC의 11.3% 개선은 제안 방법이 혼동 가능한 키워드를 구분하는 능력을 직접적으로 향상시켰음을 보여준다. 일반 AUC의 개선 폭이 상대적으로 작은 이유는 테스트 세트의 대부분이 혼동 가능하지 않은 키워드 쌍으로 구성되어 있기 때문이다.
- **모음 그룹별 영향**: 대부분의 모음 그룹에서 성능이 향상되었으나, $/i:/$와 $/I/$ 그룹(예: 'three'와 'tree')에서는 약간의 성능 저하가 관찰되었다. 이는 TTS가 이 두 단어를 거의 동일하게 합성하여 모델이 구분 불가능한 정보를 학습했을 가능성이 있다.
- **TTS 바이어스**: 매우 낮은 FAR 영역(FAR < 1%)에서는 AugModel의 성능이 다소 떨어지는 경향이 있는데, 이는 합성 데이터로 인해 모델이 실제 음성보다 합성 음성의 분포에 편향(bias)되었기 때문으로 분석된다.

## 🧠 Insights & Discussion

### 강점

본 연구는 데이터 수집에 드는 인건비와 시간을 획기적으로 줄이면서도, LLM과 TTS라는 최신 도구를 활용해 '학습하기 어려운 예제'를 체계적으로 생성했다는 점이 매우 강력하다. 특히 모음 기반의 그룹핑 전략은 언어학적 특성을 딥러닝 학습 과정에 효율적으로 이식한 사례이다.

### 한계 및 논의사항

1. **TTS의 정밀도 문제**: 'three'와 'tree'처럼 극도로 유사한 단어의 경우, TTS 엔진 자체가 두 단어를 명확히 구분하여 합성하지 못한다면 학습 데이터로서의 가치가 떨어진다. 더 정교한 음운 제어가 가능한 TTS 시스템이 필요하다.
2. **합성 데이터 오버피팅**: TTS 데이터로 인한 분포 전이(distribution shift) 현상이 관찰되었다. 저자는 이를 해결하기 위해 향후 Adversarial training 기법을 도입하여 TTS 오버피팅을 최소화할 필요가 있다고 언급한다.
3. **시퀀스 길이의 일관성**: LibriPhrase 전체 데이터셋(평균 0.63초)에서 성능 향상이 없었던 점은, 학습 데이터(1초)와 추론 데이터의 길이 일관성이 Custom KWS 성능에 결정적인 영향을 미침을 시사한다.

## 📌 TL;DR

본 논문은 Custom KWS 모델이 발음이 유사한 키워드를 구분하지 못하는 문제를 해결하기 위해, **LLM으로 모음 기반 혼동 단어 리스트를 생성하고 TTS로 이를 합성하여 학습에 활용하는 자동화 프레임워크**를 제안하였다. 이를 통해 Speech Commands 데이터셋에서 **c-AUC를 11.3% 향상**시켰으며, 하드 네거티브(hard negative)에 대한 구분 능력을 유의미하게 개선하였다. 이 방법론은 데이터 구축 비용이 거의 없으며, LLM/TTS의 다국어 능력을 활용해 다양한 언어로 쉽게 확장 가능하다는 점에서 실제 산업 적용 가능성이 매우 높다.
