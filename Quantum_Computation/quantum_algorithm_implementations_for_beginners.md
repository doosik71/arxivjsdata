# Quantum Algorithm Implementations for Beginners

Abhijith J., et al. (2022)

## 🧩 Problem to Solve

본 논문은 양자 컴퓨터가 일반 대중에게 공개됨에 따라, 기존의 클래식 컴퓨터 프로그래밍 경험만 가진 컴퓨터 과학자, 엔지니어 및 물리학자들이 양자 프로그래밍을 학습하고 실제 하드웨어에 구현할 수 있도록 돕는 것을 목표로 한다.

현재 가용한 양자 컴퓨터는 큐비트(qubit) 수가 100개 미만으로 제한적이지만, 하드웨어의 확장성과 품질 향상이 기대되는 상황이다. 그러나 양자 프로그래밍 모델은 클래식 프로그래밍과 근본적으로 다르며, 특히 물리학적 배경과 복잡한 대수적 표기법이 진입 장벽으로 작용하고 있다. 따라서 본 논문은 복잡한 양자 역학적 원리에 대한 깊은 지식 없이도 직선적인 대수학만으로 양자 알고리즘의 원리를 이해하고, 이를 실제 하드웨어(IBM Q)에 구현할 수 있는 상세한 청사진(blueprint)을 제공하고자 한다.

## ✨ Key Contributions

본 논문의 핵심 기여는 이론적인 양자 알고리즘을 실제 하드웨어 구현 관점에서 재구성하여 제시했다는 점이다. 구체적인 기여 사항은 다음과 같다.

1. **양자 프로그래밍 모델의 체계적 정리**: Qubit, 중첩(Superposition), 얽힘(Entanglement), 유니타리 변환(Unitary transformations) 등 양자 컴퓨팅의 기초 개념을 컴퓨터 과학자의 관점에서 설명하였다.
2. **20가지 양자 알고리즘의 서베이 및 구현**: Grover, Shor, HHL, QAOA 등 주요 양자 알고리즘 20가지를 선정하여 각 알고리즘의 문제 정의, 작동 원리 및 구현 방법을 간결하게 설명하였다.
3. **실제 하드웨어 실험 및 비교 분석**: IBM의 양자 컴퓨터(ibmqx4, ibmqx5, ibmqx2 등)와 시뮬레이터를 사용하여 알고리즘을 실제로 실행하고, 시뮬레이션 결과와 실제 하드웨어 결과 간의 차이를 분석함으로써 현재 양자 하드웨어의 한계(노이즈, 디코히어런스 등)를 명시적으로 보여주었다.
4. **구현 도구 및 라이브러리 활용**: IBM의 Qiskit 라이브러리를 활용한 구현 방법을 제시하여, 독자가 직접 코드를 작성하고 실행할 수 있는 실무적인 가이드를 제공하였다.

## 📎 Related Works

논문은 기존의 양자 알고리즘 서베이들이 주로 이론적인 설명에 치우쳐 있거나, 실제 하드웨어 구현 사례를 포함하지 않았다는 점을 한계로 지적한다.

- **기존 접근 방식**: 대부분의 양자 알고리즘 문헌은 1990년대부터 발표된 수학적 기초와 프로그래밍 모델에 집중하였으며, 이는 소수의 전문 공동체 내에서만 소비되었다.
- **본 논문의 차별점**: 본 연구는 이론적 설명을 넘어, 클라우드 서비스로 제공되는 실제 양자 프로세서(IBM Q)에서의 실행 결과를 직접 제시한다. 특히, 하드웨어의 제약 조건인 큐비트 연결성(Connectivity), 게이트 충실도(Gate fidelity), 디코히어런스(Decoherence)가 실제 알고리즘 성능에 미치는 영향을 정량적으로 분석하였다는 점에서 차별성을 가진다.

## 🛠️ Methodology

### 1. 양자 프로그래밍 모델 (Quantum Programming Model)

양자 시스템의 상태는 힐베르트 공간(Hilbert space)의 복소 벡터로 표현된다.

- **Qubit**: 기본 정보 단위로, 상태는 $|\psi\rangle = \alpha|0\rangle + \beta|1\rangle$ (단, $|\alpha|^2 + |\beta|^2 = 1$)로 표현된다.
- **System of Qubits**: 여러 큐비트의 결합 상태는 텐서 곱(Tensor product, $\otimes$)을 통해 표현되며, $n$개의 큐비트는 $2^n$ 차원의 벡터 공간을 형성한다.
- **Unitary Transformations**: 양자 게이트는 유니타리 행렬 $U$로 표현되며, $UU^\dagger = I$를 만족해야 한다. 이는 양자 연산이 가역적(Reversible)이어야 함을 의미한다.
- **Measurement**: 상태 $|\psi\rangle$를 측정하면 $|0\rangle$일 확률은 $|\langle 0|\psi\rangle|^2$, $|1\rangle$일 확률은 $|\langle 1|\psi\rangle|^2$가 된다.

### 2. IBM 양자 컴퓨터 구현 환경

- **Gate Set**: 물리적으로 구현된 기본 게이트 세트 $\{U_1(\theta), R_{sx}(\pi/2), CNOT\}$를 사용하며, 사용자가 정의한 고수준 게이트는 컴파일러에 의해 이들로 분해된다.
- **Connectivity**: 모든 큐비트가 서로 연결되어 있지 않으므로, 연결되지 않은 큐비트 간의 CNOT 연산을 위해 SWAP 게이트나 추가적인 게이트 시퀀스를 사용하여 가상 연결을 생성한다.
- **Noise Sources**: 게이트 불충분성(Gate infidelity)과 디코히어런스(Decoherence)가 주요 에러 원인이며, 이는 회로의 깊이(Depth)가 깊어질수록 성능을 저하시킨다.

### 3. 주요 알고리즘 구현 방법 (예시)

논문은 20가지 알고리즘을 다루며, 그 중 핵심적인 방법론은 다음과 같다.

- **Grover's Algorithm**: 정렬되지 않은 데이터베이스에서 특정 항목을 $\mathcal{O}(\sqrt{N})$ 시간 내에 찾는 알고리즘이다. Grover 연산자 $\mathcal{G} = (2|\psi\rangle\langle\psi| - I)\mathcal{O}$를 반복 적용하여 정답 상태의 진폭을 증폭시킨다.
- **Bernstein-Vazirani Algorithm**: 숨겨진 비트 문자열 $s$를 단 한 번의 오라클 쿼리로 찾아낸다. $\text{Hadamard} \rightarrow \text{Oracle} \rightarrow \text{Hadamard}$ 순으로 연산을 수행하여 $|s\rangle$ 상태를 얻는다.
- **HHL Algorithm**: 선형 방정식 $Ax=b$를 풀기 위해 양자 위상 추정(Quantum Phase Estimation, QPE)을 사용하여 $A$의 고유값 $\lambda_j$를 찾고, 이를 역수로 변환하는 회전 연산을 통해 $|x\rangle$를 구한다.
- **QAOA (Quantum Approximate Optimization Algorithm)**: 최적화 문제를 풀기 위한 하이브리드 알고리즘이다. 문제 해밀토니안 $C$와 믹싱 해밀토니안 $B$를 교대로 적용하여 상태 $|\beta, \gamma\rangle$를 준비하고, 클래식 옵티마이저가 $\beta, \gamma$ 파라미터를 최적화한다.

## 📊 Results

### 1. 실험 설정

- **데이터셋 및 하드웨어**: IBM Q의 5-큐비트(ibmqx4) 및 16-큐비트(ibmqx5) 프로세서와 Qiskit 시뮬레이터를 사용하였다.
- **측정 지표**: 정답 상태를 얻을 확률(Probability), 기대값(Expectation value), 하드웨어 결과와 시뮬레이션 결과의 일치도 등을 측정하였다.

### 2. 주요 실험 결과

- **Grover's Algorithm**: 2비트 검색 문제에서 시뮬레이터는 100% 확률로 정답을 찾았으나, 실제 하드웨어(ibmqx4)에서는 약 65%의 확률을 보였다. 이는 하드웨어의 내재적 에러로 인해 발생한 결과이다.
- **Bernstein-Vazirani Algorithm**: 2비트 숨겨진 문자열에 대해 시뮬레이터는 100% 정확도를 보였으나, 실제 하드웨어에서는 노이즈로 인해 확률이 낮아졌으며, 특히 3비트 이상의 경우 회로 깊이가 하드웨어의 코히어런스 시간(Coherence time)을 초과하여 실행이 불가능함을 확인하였다.
- **QAOA (MaxCut)**: 시뮬레이션에서는 라운드 $p$가 증가함에 따라 근사치가 최적값에 수렴하였으나, 하드웨어에서는 $p=2$일 때 $p=1$보다 성능이 떨어지는 현상이 관찰되었다. 이는 회로 깊이가 증가하면서 게이트 에러와 디코히어런스의 영향이 누적되었기 때문이다.
- **Quantum PCA**: 2개 특징의 공분산 행렬을 분석했을 때, 시뮬레이터는 정확한 고유값을 도출하였으나 실제 하드웨어 결과는 물리적으로 불가능한 복소수 값을 출력하는 등 실패하였다.

## 🧠 Insights & Discussion

### 1. 강점 및 성과

본 논문은 이론적 양자 알고리즘을 실제 하드웨어에 어떻게 매핑(Mapping)하고 컴파일(Compile)하는지에 대한 실무적인 가이드를 제공하였다. 특히, QNC(Quantum Netlist Compiler)와 같은 도구를 통해 임의의 유니타리 행렬을 물리적 게이트 세트로 변환하는 과정을 상세히 다룬 점이 돋보인다.

### 2. 한계 및 비판적 해석

- **하드웨어 제약**: 실험 결과에서 나타나듯, 현재의 NISQ(Noisy Intermediate-Scale Quantum) 장치는 매우 낮은 게이트 충실도와 짧은 코히어런스 시간을 가지고 있다. 이로 인해 이론적으로는 효율적인 알고리즘이라 할지라도, 실제 구현 시 회로 깊이가 조금만 깊어져도 결과가 무작위 잡음(Noise)에 가까워지는 한계를 보였다.
- **양자 오류 정정(QEC)의 실효성**: 저자들은 3-큐비트 반복 코드(Repetition code)를 통한 오류 정정을 실험하였으나, 오히려 정정을 위해 추가된 게이트들이 더 많은 에러를 유발하여 최종 정확도가 떨어지는 결과를 얻었다. 이는 현재 수준의 게이트 정밀도로는 QEC가 오히려 독이 될 수 있음을 시사하며, 게이트 에러가 단순한 비트 플립이 아니라 상관관계가 있는(Correlated) 에러임을 암시한다.

### 3. 결론 및 시사점

양자 알고리즘의 실용화를 위해서는 단순한 알고리즘 설계뿐만 아니라, 하드웨어의 토폴로지를 고려한 회로 최적화와 획기적인 게이트 충실도 향상이 필수적이다.

## 📌 TL;DR

본 논문은 양자 컴퓨팅 입문자를 위해 20가지 주요 양자 알고리즘의 원리와 IBM Q 하드웨어 상의 실제 구현 방법을 상세히 분석한 리뷰 보고서이다. 시뮬레이션과 실제 하드웨어 실행 결과를 비교함으로써, 이론적 우위(Quantum Advantage)와 실제 하드웨어 성능 사이의 간극을 명확히 보여주었으며, 특히 회로 깊이 증가에 따른 노이즈 누적 문제와 현재 QEC의 한계를 정량적으로 입증하였다. 이 연구는 향후 실용적인 양자 소프트웨어 개발을 위한 구현 청사진을 제공한다는 점에서 중요한 가치를 가진다.
