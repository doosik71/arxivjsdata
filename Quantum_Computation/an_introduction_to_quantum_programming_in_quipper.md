# An Introduction to Quantum Programming in Quipper

Alexander S. Green, Peter LeFanu Lumsdaine, Neil J. Ross, Peter Selinger, and Benoît Valiron (2013)

## 🧩 Problem to Solve

본 논문은 실제 양자 컴퓨터에서 양자 알고리즘을 구현할 때 필요한 계산 자원을 정확하게 추정하고 이를 줄이기 위한 프로그래밍 언어의 필요성을 다룬다. 기존의 양자 프로그래밍 언어들은 고수준의 추상화를 제공하면서도 동시에 실제 하드웨어 수준의 회로(Circuit)로 효율적으로 변환되어야 하는 상충되는 요구사항을 가지고 있다. 특히, 대규모 양자 알고리즘의 경우 회로의 크기가 기하급수적으로 커질 수 있어, 이를 효율적으로 생성하고 관리하며 자원(Gate 수, Qubit 수 등)을 추정할 수 있는 확장 가능한(Scalable) 언어적 도구가 부족하다는 점이 주요 문제이다. 따라서 본 논문의 목표는 Haskell에 임베디드된 함수형 양자 프로그래밍 언어인 Quipper의 주요 기능과 사용법을 소개하고, 이를 통해 실제 양자 알고리즘을 어떻게 효율적으로 구현할 수 있는지 보여주는 것이다.

## ✨ Key Contributions

Quipper의 핵심적인 설계 아이디어는 **함수형 프로그래밍의 고수준 추상화 능력을 유지하면서도, 이를 실제 양자 회로 모델로 정밀하게 매핑하는 임베디드 언어(Embedded Language) 구조**를 채택한 것이다. 주요 기여 사항은 다음과 같다.

1. **확장 가능한 회로 생성 모델**: 회로를 한 번에 메모리에 적재하는 대신, Lazy evaluation을 통해 필요한 시점에 회로를 생성하고 소비함으로써 매우 거대한 회로도 다룰 수 있게 설계하였다.
2. **강력한 타입 시스템 및 제네릭(Generic) 기능**: `QShape`를 도입하여 양자 데이터 타입, 클래식 데이터 타입, 불리언 데이터 타입 간의 형태(Shape)를 일치시켜, 동일한 로직을 다양한 데이터 구조에 적용할 수 있는 제네릭 함수를 제공한다.
3. **고수준 추상화 도구**: 재귀(Recursion), 박스형 서브회로(Boxed subcircuits), 그리고 클래식 코드로부터의 자동 회로 생성(`build_circuit`) 기능을 통해 프로그래머의 생산성을 높였다.
4. **정밀한 자원 추정**: 실제 회로를 모두 생성하지 않고도 게이트 수와 큐비트 수를 계산할 수 있는 메커니즘을 제공하여 대규모 알고리즘의 비용 분석을 가능하게 하였다.

## 📎 Related Works

논문에서는 기존의 양자 프로그래밍 언어들에 대해 언급하며 Quipper와의 차별점을 제시한다.

- **QCL**: C-style 언어로 양자 시뮬레이션에 최적화되어 있으나, Quipper와 같은 함수형 추상화와 확장성 측면에서 차이가 있다.
- **Quantum IO Monad**: Haskell에 임베디드된 또 다른 언어이지만, Quipper는 IARPA의 QCS 프로젝트의 일환으로 더 광범위한 알고리즘 구현과 자원 추정에 초점을 맞추어 개발되었다.
- **LQPL**: 선형 타입(Linear types)을 가진 함수형 언어이다. Quipper는 현재 Haskell의 타입 시스템을 사용하므로 선형 타입이 기본적으로 제공되지 않아 일부 웰-폼드니스(Well-formedness) 체크를 런타임에 수행해야 한다는 한계가 있으나, 향후 도입을 계획하고 있다.

가장 큰 차별점은 기존 언어들이 대규모 문제 크기에 대해 확장성(Scalability)을 입증하지 못한 반면, Quipper는 조 단위의 게이트를 포함하는 회로의 자원 추정이 가능할 정도로 확장성이 뛰어나다는 점이다.

## 🛠️ Methodology

Quipper는 Haskell을 호스트 언어로 사용하는 임베디드 언어이며, 다음과 같은 구조와 메커니즘을 가진다.

### 1. 회로 모델 및 실행 단계

Quipper는 확장된 회로 모델을 사용하며, 실행 단계를 세 가지로 구분한다.

- **Compile time**: Haskell 컴파일러에 의해 코드가 분석되는 단계이다.
- **Circuit generation time**: 생성 함수가 실행되어 실제 양자 회로 구조가 만들어지는 단계이다. 이 단계에서 결정되는 입력값은 `Bool` 타입의 파라미터로 처리된다.
- **Circuit execution time**: 생성된 회로가 실제 양자 컴퓨터나 시뮬레이터에서 실행되는 단계이다. 이 단계의 입력은 `Bit` 또는 `Qubit` 타입으로 처리된다.

### 2. 주요 타입 시스템

- $\text{Bool}$: 회로 생성 시점에 값이 결정되는 불리언 파라미터이다.
- $\text{Bit}$: 회로 실행 시점에 결정되는 클래식 비트 입력이다.
- $\text{Qubit}$: 회로 실행 시점에 결정되는 양자 큐비트 입력이다.
- $\text{Circ } a$: 양자 회로를 생성하는 계산을 나타내는 모나드(Monad) 타입으로, 물리적인 부수 효과(Side effect)가 발생할 수 있음을 명시한다.

### 3. 핵심 기능 및 연산자

- **제네릭 함수 ($\text{QShape}$)**: `QShape ba qa ca` 제약 조건을 통해 불리언, 양자, 클래식 타입의 셋이 동일한 구조를 가짐을 정의한다. 이를 통해 `mapUnary`, `mapBinary`와 같은 연산자를 사용하여 데이터 구조의 형태에 관계없이 동일한 양자 연산을 적용할 수 있다.
- **재귀 및 회로 레벨 연산**: 파라미터를 기반으로 회로 생성 함수를 재귀적으로 정의할 수 있다. 또한 `reverse_generic_endo`와 같이 전체 회로를 반전시키는 회로 레벨 연산자를 제공한다.
- **$\text{with\_computed}$**: 특정 계산을 수행한 후, 그 역연산(Inverse)을 자동으로 마지막에 배치하는 고수준 연산자이다.
- **Boxed Subcircuits**: 자주 사용되는 서브회로를 `box` 연산자로 묶어 계층적 모델을 구축함으로써 중복 생성을 방지하고 가독성을 높인다.
- **$\text{build\_circuit}$**: Template Haskell을 이용하여 일반 Haskell 함수(클래식 코드)로부터 가역 회로(Reversible circuit)를 자동으로 합성한다.

### 4. 시뮬레이션 및 변환

Quipper는 세 가지 시뮬레이터를 제공한다:

- **Classical simulation**: 클래식 회로의 효율적 시뮬레이션.
- **Stabilizer simulation**: Clifford 그룹 회로의 효율적 시뮬레이션.
- **Quantum simulation**: 모든 회로를 시뮬레이션하나 지수적 오버헤드가 발생한다.
또한 `decompose_generic Binary`와 같은 변환기를 통해 고수준 게이트를 2-큐비트 기본 게이트 셋으로 분해할 수 있다.

## 📊 Results

본 논문은 특정 실험 데이터셋에 대한 성능 측정보다는, Quipper의 기능을 통해 구현된 알고리즘 예제와 자원 추정 능력을 통해 정성적/정량적 결과를 보여준다.

- **구현 사례**: Quantum Teleportation, Quantum Fourier Transform (QFT), Quantum Addition 회로를 성공적으로 구현하고 이를 PDF 형태의 회로도로 출력하여 검증하였다.
- **자동 합성**: 클래식 가산기(Full Adder) 코드를 `build_circuit`을 통해 양자 회로로 변환하고, 이를 다시 가역 회로로 합성하는 과정을 보여주었다.
- **확장성 및 자원 추정**: Triangle finding 알고리즘을 구현한 결과, **30조(30 trillion) 개 이상의 게이트**를 포함하는 거대한 회로의 자원 수를 1.2GHz 랩톱에서 **2분 미만**의 시간 내에 계산해 냈음을 명시하였다. 이는 Quipper의 Lazy generation과 Boxed subcircuit 구조가 매우 효율적으로 작동함을 입증하는 결과이다.

## 🧠 Insights & Discussion

### 강점

Quipper는 함수형 프로그래밍의 추상화 이점을 양자 컴퓨팅에 성공적으로 접목하였다. 특히 `QShape`를 통한 제네릭 프로그래밍과 `build_circuit`을 통한 자동 합성 기능은 복잡한 양자 알고리즘 설계 시 발생하는 반복적인 작업을 획기적으로 줄여준다. 또한, 실제 하드웨어 실행 전 단계인 '회로 생성 단계'를 분리함으로써 매우 거대한 회로의 비용 분석을 가능하게 한 점이 매우 인상적이다.

### 한계 및 비판적 해석

논문에서도 언급되었듯이, Haskell의 타입 시스템을 그대로 사용함에 따라 양자 역학의 핵심 제약인 **No-cloning theorem(복제 불가능성)**을 타입 수준에서 강제하는 선형 타입(Linear types)이나 의존 타입(Dependent types)이 부재하다. 이로 인해 큐비트의 중복 사용과 같은 오류를 컴파일 타임이 아닌 런타임(회로 생성 시점)에서 확인해야 한다는 점은 안정성 측면에서 아쉬운 부분이다.

또한, `build_circuit` 기능은 강력하지만, 사용자가 라이브러리 함수의 퀀텀 템플릿을 직접 제공해야 한다는 제약이 있어, 사용하려는 라이브러리가 많아질수록 사전 작업의 부담이 커질 수 있다.

## 📌 TL;DR

본 논문은 Haskell 기반의 확장 가능한 양자 프로그래밍 언어인 **Quipper**를 소개한다. Quipper는 제네릭 함수, 재귀적 회로 생성, 자동 가역 회로 합성, 그리고 계층적 박스 구조를 통해 고수준의 추상화를 제공하면서도 실제 양자 회로로의 정밀한 변환을 지원한다. 특히 조 단위의 게이트를 가진 거대 회로의 자원 추정이 가능할 정도의 뛰어난 확장성을 보여주어, 향후 복잡한 양자 알고리즘의 구현 및 분석에 중요한 도구가 될 가능성이 높다.
