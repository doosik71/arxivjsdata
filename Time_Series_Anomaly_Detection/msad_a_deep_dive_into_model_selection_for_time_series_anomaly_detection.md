# MSAD: A Deep Dive into Model Selection for Time series Anomaly Detection

Emmanouil Sylligardos, John Paparrizos, Themis Palpanas, Pierre Senellart, Paul Boniol (2025)

## 🧩 Problem to Solve

시계열 이상 탐지(Time Series Anomaly Detection, TSAD)는 다양한 도메인에서 매우 중요한 과제이지만, 데이터셋의 성격이 매우 이질적(heterogeneous)이기 때문에 모든 데이터셋에서 일관되게 최상의 성능을 보이는 단일 알고리즘은 존재하지 않는다. 특정 방법론은 정적인(stationary) 데이터나 점 기반 이상치(point-based anomalies) 탐지에는 유리하지만, 다른 방법론은 비정적 데이터나 시퀀스 기반 이상치(sequence-based anomalies) 탐지에 더 적합한 경향이 있다.

이러한 문제를 해결하기 위해 모든 탐지기의 결과를 평균 내는 앙상블(Ensembling) 방식이 제안되었으나, 이는 모든 모델을 실행해야 하므로 계산 비용이 너무 높다는 치명적인 단점이 있다. 또한 기존의 AutoML 솔루션들은 시계열 데이터의 가변적인 길이, 표준화된 피처 추출 방법의 부재, 그리고 레이블링된 이상치 데이터의 부족으로 인해 TSAD에 직접 적용하기 어렵다. 따라서 본 논문의 목표는 시계열의 특성을 분석하여 해당 데이터에 가장 적합한 이상 탐지 모델을 자동으로 선택하거나, 최적의 모델 조합을 제안하는 모델 선택(Model Selection) 방법론을 구축하고 그 효과를 정밀하게 분석하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **모델 선택 문제를 시계열 분류(Time Series Classification, TSC) 문제로 변환**하여 해결하는 것이다. 즉, 어떤 이상 탐지기가 특정 시계열에서 가장 높은 성능을 낼지를 예측하는 분류기를 학습시키는 방식이다.

주요 기여 사항은 다음과 같다:

- **분류 기반 모델 선택 프레임워크 제안**: 다양한 시계열 분류 알고리즘(Feature-based, ConvNet, Transformer 등)을 모델 선택기로 사용하여 최적의 AD 모델을 선정하는 파이프라인을 구축하였다.
- **다중 모델 조합 전략($k$ 파라미터)**: 단일 모델 선택($k=1$)뿐만 아니라, 상위 $k$개의 모델을 가중치 기반으로 조합하여 성능과 비용 간의 트레이드오프를 분석하였다.
- **광범위한 벤치마크 평가**: TSB-UAD 벤치마크의 1,980개 이상의 시계열과 12개의 AD 방법론을 사용하여 In-distribution 및 Out-Of-Distribution(OOD) 설정에서 실험적 검증을 수행하였다.
- **분석적 통찰 제공**: 윈도우 길이, 조합 방법(Average vs Vote), 그리고 분류 정확도와 이상 탐지 정확도 사이의 상관관계를 심층적으로 분석하였다.

## 📎 Related Works

논문에서는 기존 TSAD 방법론을 크게 네 가지 범주로 구분하여 설명한다:

1. **거리 기반(Distance-based)**: Discord-based, Proximity-based, Clustering-based(예: NormA) 방법 등이 있으며, raw 데이터의 거리를 측정한다.
2. **밀도 기반(Density-based)**: Isolation Forest, Series2Graph 등이 있으며, 표현 공간에서의 밀도를 평가한다.
3. **예측 기반(Prediction-based)**: RNN, CNN 등을 이용한 Forecasting 기반 또는 Autoencoder 기반의 Reconstruction 방법이 있으며, 예측 오차를 이상치 점수로 사용한다.
4. **기타**: OCSVM, HBOS 등이 포함된다.

기존의 AutoML 접근 방식(예: MetaOD)은 표 형식(tabular) 데이터의 이상 탐지 모델 선택에는 적용되었으나, 시계열 데이터의 특수성(가변 길이, 구조적 패턴)으로 인해 적용에 한계가 있었다. 본 연구는 이러한 한계를 극복하기 위해 시계열 전용 분류기를 모델 선택기로 활용한다는 점에서 차별점을 가진다.

## 🛠️ Methodology

### 1. 전체 파이프라인 구조

MSAD 파이프라인은 다음과 같은 단계로 구성된다:

- **전처리 단계(Preprocessing)**: 가변 길이의 시계열 $T$를 고정된 길이 $\ell$의 겹치지 않는(non-overlapping) 서브시퀀스로 분할한다.
- **예측 단계(Prediction)**: 학습된 시계열 분류기 $M$이 각 서브시퀀스에 대해 사용 가능한 AD 모델들에 대한 확률 분포 $P$를 출력한다.
- **조합 단계(Combination)**: 각 서브시퀀스의 확률 분포를 통합하여 전체 시계열에 적용할 가중치 $W^T = \{w_1, w_2, \dots, w_m\}$를 생성한다.
- **최종 점수 계산**: 선택된 상위 $k$개 모델의 결과값에 가중치를 적용하여 최종 이상치 점수를 산출한다.

### 2. 주요 방정식 및 알고리즘

최종 이상치 점수 $\bar{S}_T$는 다음과 같이 정의된다:
$$\bar{S}_T = \sum_{i=1}^{m} w_i D_i(T)$$
여기서 $w_i$는 모델 선택기가 예측한 가중치이며, $\sum w_i = 1, w_i \ge 0$을 만족해야 한다. 또한 $k$ 파라미터에 의해 오직 $k$개의 $w_i$만 0이 아닌 값을 가진다.

가중치를 결정하는 두 가지 전략은 다음과 같다:

- **평균 전략(Average Strategy)**: 모든 서브시퀀스에서 예측된 확률 분포의 평균을 구한다.
  $$\bar{P}_{D_j} = \frac{1}{n} \sum_{i=1}^{n} P^{(i)}_{D_j}$$
- **투표 전략(Vote Strategy)**: 각 서브시퀀스에서 가장 확률이 높은 모델에 투표하고, 그 빈도수를 기반으로 가중치를 정한다.
  $$\bar{V}_{D_j} = \sum_{i=1}^{n} \mathbb{1} \left[ P^{(i)}_{D_j} = \max_{k \in [1, m]} P^{(i)}_{D_k} \right]$$

### 3. 사용된 분류 모델

- **Feature-based**: TSFresh를 통해 피처를 추출한 후 SVC, Random Forest, kNN 등을 적용한다.
- **Raw-based**: MiniRocket을 사용하여 원시 데이터에서 직접 특징을 추출하고 분류한다.
- **Convolutional-based**: ConvNet, ResNet, InceptionTime 등의 딥러닝 아키텍처를 사용한다.
- **Transformer-based**: Signal Transformer(SiT)를 기반으로 다양한 임베딩 방식(conv, linear, stem 등)을 적용한 모델을 사용한다.

## 📊 Results

### 1. 실험 설정

- **데이터셋**: TSB-UAD 벤치마크 내 16개 데이터셋 (총 1,980개 이상의 시계열).
- **비교 대상**: 12개의 개별 AD 방법론, Averaging Ensemble(Avg Ens), 그리고 이론적 상한선인 Oracle.
- **지표**: 분류 정확도(Classification Accuracy), AUC-PR, VUS-PR(Volume Under the Precision-Recall Surface).

### 2. 주요 결과

- **성능 향상**: 대부분의 모델 선택 방법이 단일 AD 모델 및 Avg Ens보다 높은 성능을 보였다. 특히 최적의 선택 모델은 TSB-UAD의 최상위 단일 모델보다 최대 2.3배, Avg Ens보다 1.9배 더 정확했다.
- **다중 모델 조합($k > 1$)의 효과**: 단일 모델($k=1$)을 선택하는 것보다 소수의 모델($k=4 \sim 8$)을 조합했을 때 성능이 향상되었다. 특히 $k=5$일 때 많은 모델이 정점에 도달하는 경향을 보였다.
- **딥러닝 모델의 우위**: ConvNet과 SiT 기반의 선택기가 Feature-based나 Rocket보다 훨씬 정확한 예측을 수행하였으며, 정답 분포(Ground Truth)와 가장 유사한 선택 경향을 보였다.
- **OOD 성능**: 완전히 새로운 도메인의 데이터셋에 적용했을 때, $k=1$인 경우에는 Avg Ens보다 성능이 낮았으나, $k=5$ 정도의 조합을 사용하면 Avg Ens와 유사한 성능을 내면서 실행 시간은 획기적으로 줄일 수 있었다.
- **실행 시간**: 모델 선택 과정이 추가됨에도 불구하고, 전체 실행 시간은 단일 AD 모델과 동일한 규모(order of magnitude)이며, 모든 모델을 실행하는 Avg Ens보다 훨씬 빠르다.

## 🧠 Insights & Discussion

### 1. 강점 및 발견

- **분류 정확도와 탐지 정확도의 상관관계**: 실험을 통해 분류 정확도가 높을수록 이상 탐지 성능(VUS-PR)이 향상됨을 확인하였으며, 이는 분류 정확도를 성능의 대리 지표(proxy)로 사용할 수 있음을 시사한다.
- **특성 추출 능력**: 딥러닝 기반 선택기들은 명시적인 학습 없이도 데이터가 점 기반 이상치인지 시퀀스 기반 이상치인지를 구분하여 적절한 모델(예: CNN vs NormA)을 선택하는 능력을 보였다.

### 2. 한계 및 비판적 해석

- **Oracle과의 간극**: 최상의 모델 선택기조차 이론적 상한선인 Oracle과는 여전히 상당한 격차가 존재한다. 이는 현재의 TSC 모델들이 시계열의 모든 특성을 완벽하게 캡처하지 못하고 있음을 의미하며, 향후 개선의 여지가 크다.
- **높은 분산**: 일부 매우 어려운 시계열의 경우, 어떤 모델을 선택하더라도 성능이 낮게 나타나 모델 선택기들의 성능 분산이 크게 나타났다.
- **순위 예측의 한계**: 단순 분류(정답 모델 하나를 맞추는 것)보다는, 성능 순위(rank)를 정확히 예측하는 학습 방식이 도입된다면 $k$개 모델 조합 시 더 높은 성능을 낼 수 있을 것이다.

## 📌 TL;DR

본 논문은 **시계열 이상 탐지 모델 선택 문제를 시계열 분류 문제로 재정의**하여, 데이터 특성에 맞는 최적의 탐지기 또는 탐지기 조합을 자동으로 선택하는 **MSAD 프레임워크**를 제안한다. 실험 결과, 딥러닝 기반 분류기(ConvNet, SiT)를 이용해 상위 $k=5$개의 모델을 조합하여 사용하는 것이 단일 모델이나 전체 앙상블보다 효율성과 정확도 면에서 모두 우수함을 증명하였다. 특히 OOD 환경에서 적은 수의 모델 조합만으로도 전체 앙상블 수준의 성능을 달성할 수 있다는 점은 실제 AutoML 파이프라인 구축에 매우 중요한 가이드라인을 제공한다.
