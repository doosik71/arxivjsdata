# VizADS-B: Analyzing Sequences of ADS-B Images Using Explainable Convolutional LSTM Encoder-Decoder to Detect Cyber Attacks

Sefi Akerman, Edan Habler, Asaf Shabtai (2019)

## 🧩 Problem to Solve

본 논문은 항공 교통 관제(Air Traffic Control, ATC) 시스템의 핵심 기술인 ADS-B(Automatic Dependent Surveillance-Broadcast) 프로토콜의 심각한 보안 취약성을 해결하고자 한다. ADS-B는 GPS 기반의 항공기 위치 정보를 지상국과 주변 항공기에 방송함으로써 기존 레이더 기반 시스템을 대체하여 안전성과 효율성을 높이는 것을 목표로 한다.

그러나 ADS-B는 설계 당시 정확성과 비용 절감에만 치중하여 보안 메커니즘이 거의 고려되지 않았다. 구체적으로 메시지에 인증 코드나 디지털 서명이 없어 위조 및 조작이 가능하며, 공개된 형식을 사용하므로 도청에 취약하고, 송신 전 인증 절차가 없어 권한 없는 사용자가 가짜 메시지를 주입할 수 있는 구조적 결함을 가지고 있다.

기존의 암호화 도입이나 하드웨어 센서 추가와 같은 해결책은 항공 시스템의 복잡한 규제 절차, 막대한 교체 비용, 이미 전 세계적으로 배포된 시스템이라는 실무적 제약으로 인해 적용이 불가능에 가깝다. 따라서 본 연구의 목표는 현재의 ADS-B 아키텍처를 변경하거나 추가 하드웨어를 도입하지 않고도 소프트웨어만으로 이상 ADS-B 메시지를 탐지할 수 있는 보안 솔루션인 VizADS-B를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 개별 항공기의 메시지 데이터를 분석하는 대신, 특정 지리적 영역 내의 모든 항공기 정보를 시각적 이미지 스트림으로 변환하여 분석하는 것이다.

1. **데이터의 이미지화**: 다수 항공기의 시공간적 정보를 이미지 시퀀스로 표현함으로써, 복잡한 피처 엔지니어링(Feature Engineering) 없이도 항공기 간의 상관관계를 모델이 직접 학습하게 한다.
2. **ConvLSTM Encoder-Decoder 활용**: 시공간적 특징을 효과적으로 추출할 수 있는 Convolutional LSTM 기반의 오토인코더 구조를 사용하여 정상적인 비행 패턴을 학습하고, 재구성 오차를 통해 이상 징후를 탐지한다.
3. **설명 가능한 AI(XAI) 기법 제안**: ConvLSTM Encoder-Decoder 모델의 출력을 해석하여 어떤 영역에서 이상이 발생했는지 시각적으로 표시함으로써, 조종사가 탐지 결과의 신뢰성을 빠르게 판단하고 오탐(False Alarm)을 구분할 수 있도록 돕는 설명 가능성 기술을 도입하였다.

## 📎 Related Works

### 기존 보안 접근 방식 및 한계

* **암호화 및 인증**: 메시지 암호화나 PKI 기반 인증 등을 제안하였으나, 전 세계적인 키 관리의 어려움과 프로토콜 자체를 수정해야 한다는 점에서 실효성이 떨어진다.
* **물리 계층 분석 및 도플러 효과**: 물리적 신호 분석을 통해 위치를 검증하는 방법은 정확하지만, 지상국의 센서 인프라에 의존해야 하므로 모든 지역에 적용하기 어렵다.
* **다변측정(Multilateration, MLAT)**: 여러 지점의 신호 도달 시간 차이(TDOA)를 이용하는 방식은 추가 센서가 필요하며, 정교한 공격자가 여러 송신원을 사용하여 이를 우회할 수 있다는 한계가 있다.

### 기존 머신러닝 접근 방식과의 차별점

기존 연구(Habler et al.)에서는 LSTM Encoder-Decoder를 사용하여 개별 항공기의 비행 경로를 모델링하고 이상치를 탐지하였다. 하지만 이는 항공기 각각을 개별적으로 분석하므로, 동일 공역을 공유하는 항공기들 사이의 시공간적 상관관계(Spatial-temporal correlation)를 고려하지 못한다. 반면, VizADS-B는 공역 전체를 이미지로 표현함으로써 다수 항공기의 상호작용과 패턴을 통합적으로 분석한다는 점에서 차별성을 가진다.

## 🛠️ Methodology

### 1. 이미지 생성 과정 (Constructing the Images)

ADS-B 메시지에 포함된 위도, 경도, 고도, 속도, 방향 정보를 기반으로 시각적 이미지를 생성한다.

* **투영**: Lambert Conformal Conic (LCC) 투영법을 사용하여 지리적 좌표를 2D 평면으로 변환한다.
* **시각적 요소 매핑**:
  * **화살표의 위치**: 항공기의 위도와 경도 ($\text{latitude, longitude}$)
  * **화살표의 길이**: $\Delta t$ 시간 동안 이동한 거리 (속도 반영)
  * **화살표의 방향**: 항공기의 진행 방향 ($\text{heading}$)
  * **화살표의 크기**: 항공기의 고도 ($\text{altitude}$). 이때 고도 값에는 로그 함수 $\log(\text{altitude})$를 적용하여 표현한다.
  * **화살표의 색상**: (옵션) 개별 메시지의 이상 점수 ($\text{anomaly score}$)

### 2. 모델 구조 및 학습

본 모델은 **Convolutional LSTM (ConvLSTM) Encoder-Decoder** 구조를 사용한다.

* **Encoder**: ConvLSTM 레이어를 통해 입력 이미지 시퀀스에서 시공간적 특징을 추출하고 압축된 표현(Hidden State)을 생성한다.
* **Decoder**: Encoder가 추출한 특징을 바탕으로 원래의 입력 이미지 시퀀스를 재구성(Reconstruct)한다.
* **학습 목표**: 정상적인 비행 데이터만을 학습시켜, 모델이 정상 패턴의 이미지를 정확하게 재구성하도록 한다.
* **손실 함수 및 최적화**: Mean Square Error (MSE) 손실 함수를 사용하며, Adam optimizer를 통해 학습한다.

### 3. 이상 탐지 절차

정상 데이터로 학습된 모델은 정상적인 이미지는 잘 재구성하지만, 공격이 주입된 이상 이미지는 제대로 재구성하지 못한다.

* **유사도 측정**: 입력 이미지와 재구성된 출력 이미지 간의 구조적 유사도를 측정하기 위해 **SSIM (Structural Similarity Index)** 지표를 사용한다.
$$\text{Anomaly Score} = \text{SSIM}(\text{Image}_{\text{in}}, \text{Image}_{\text{out}})$$
* **탐지 알고리즘**:
    1. 단일 프레임의 SSIM 값이 임계값 $t_1$보다 낮으면 해당 프레임을 '의심(suspicious)' 상태로 간주한다.
    2. 윈도우 크기 $w$ 내에서 의심 프레임의 개수가 임계값 $t_2$를 초과하면 해당 윈도우 전체를 이상(Anomalous)으로 분류한다.

### 4. 설명 가능성 방법론 (Proposed XAI)

탐지된 이상 징후가 실제 공격인지 오탐인지 판단하기 위해 다음 절차를 수행한다.

1. 이상으로 분류된 윈도우 내에서 SSIM 점수가 가장 낮은(가장 부정확하게 재구성된) 이미지를 선택한다.
2. 해당 이미지를 $n \times n$ 개의 작은 서브 이미지(Square)로 분할한다.
3. 각 서브 이미지 쌍에 대해 SSIM을 계산하여, 재구성 오차가 큰 영역을 색상으로 강조 표시(Highlighting)한다. 이를 통해 조종사는 이미지의 어느 부분(어느 항공기)에서 이상이 발생했는지 즉각적으로 확인할 수 있다.

## 📊 Results

### 실험 설정

* **데이터셋**: OpenSky 플랫폼에서 수집한 5개 주요 공항(파리 CDG, 베를린 TXL, 런던 LHR, 로마 FCO, 암스테르담 AMS) 주변 $10,000 \text{km}^2$ 영역의 7일치 데이터.
* **주입된 공격 유형**:
  * **Flooding**: 무작위 위치에 가짜 항공기 다수 주입.
  * **Ghost**: 다른 지역의 정상 경로 데이터를 가져와 가짜 항공기 주입.
  * **Jamming**: 특정 항공기를 일정 시간 사라지게 함 (DoS 공격).
  * **Reverse**: 항공기의 진행 방향을 180도 변경.
  * **Change altitude**: 고도를 비정상적으로 급격히 변경.

### 주요 결과

* **탐지 성능**: Flooding, Ghost, Change Altitude 공격은 대부분의 데이터셋에서 매우 높은 탐지율(TPR)을 보였다. Jamming 공격 역시 대부분 탐지되었다.
* **공격별 특성**: 항공기를 새로 추가하는 방식의 공격(Flooding, Ghost)이 기존 항공기의 데이터를 미세하게 수정하는 방식보다 탐지가 더 용이했다.
* **Reverse 공격의 한계**: Reverse 공격은 이미지 상에서 변화하는 영역이 매우 작고 미묘하여 TPR이 낮게 나타났다. 저자들은 이를 해결하기 위해 Attention 기반 CNN의 필요성을 언급한다.
* **하이퍼파라미터 영향**: 시간 간격 $\Delta t$가 짧을수록(예: $2\text{s}$) 공격의 세밀한 변화를 포착할 수 있어 탐지 성능이 향상되는 경향을 보였다.
* **XAI 효과**: 도메인 전문가(조종사)가 XAI 모듈의 시각적 표시를 확인했을 때, 평균 5초 이내에 오탐과 실제 공격을 구분할 수 있었다.

## 🧠 Insights & Discussion

### 강점

VizADS-B는 하드웨어 변경 없이 소프트웨어만으로 구현 가능하다는 점에서 실무적 적용 가능성이 매우 높다. 특히, 개별 항공기가 아닌 공역 전체를 이미지로 처리함으로써 시공간적 상관관계를 학습했다는 점이 고무적이다. 또한, 딥러닝 모델의 블랙박스 문제를 해결하기 위해 서브 이미지 SSIM 기반의 XAI 기법을 도입하여 운용자의 신뢰도를 높였다.

### 한계 및 비판적 해석

1. **공격 민감도**: Reverse 공격과 같이 국소적인 변화를 일으키는 공격에 취약하다는 점은 모델이 전역적인 구조 변화에는 강하지만 미세한 국소 변화 포착 능력은 부족함을 시사한다.
2. **데이터셋 의존성**: 공항별 지리적 특성과 비행 경로의 복잡도에 따라 탐지 성능이 달라지는데, 이는 모델이 특정 지역의 '정상 패턴'에 과적합(Overfitting)될 가능성이 있음을 의미한다.
3. **임계값 설정**: SSIM 임계값을 각 데이터셋마다 별도로 설정한 점은 실제 환경에서 새로운 지역에 적용할 때 최적의 임계값을 찾는 과정이 추가적으로 필요함을 뜻한다.

## 📌 TL;DR

본 논문은 보안에 취약한 ADS-B 시스템을 위해 항공기들의 위치, 속도, 고도 정보를 **이미지 시퀀스로 변환**하고, 이를 **ConvLSTM Encoder-Decoder** 모델로 분석하여 사이버 공격을 탐지하는 **VizADS-B**를 제안한다. 정상 패턴을 학습한 모델이 재구성하지 못하는 영역을 **SSIM** 지표로 찾아내며, **XAI 기법**을 통해 이상 영역을 시각화함으로써 조종사의 신속한 의사결정을 돕는다. 이 연구는 하드웨어 수정 없이 소프트웨어만으로 항공 보안을 강화할 수 있는 실용적인 프레임워크를 제시했다는 점에서 향후 실시간 항공 관제 보안 시스템 구축에 중요한 기여를 할 것으로 보인다.
