# Let the Model Decide its Curriculum for Multitask Learning

Neeraj Varshney, Swaroop Mishra, Chitta Baral (2022)

## 🧩 Problem to Solve

본 논문은 Multi-Task Learning (MTL) 환경에서 학습 데이터의 순서를 결정하는 Curriculum Learning 전략의 효율성 문제를 해결하고자 한다. 기존의 Curriculum Learning 방식은 크게 두 가지 접근법을 사용한다. 첫째는 인간의 직관에 의존하여 난이도 계층을 설정하는 방식이고, 둘째는 가능한 모든 조합을 탐색하여 최적의 배열을 찾는 Exhaustive Search 방식이다.

그러나 인간이 인지하는 난이도는 실제 모델(기계)이 학습하는 난이도와 일치하지 않을 수 있으며, 이는 성능 저하로 이어진다. 또한, Exhaustive Search는 데이터셋의 수와 크기가 증가함에 따라 계산 비용이 기하급수적으로 상승하여 현실적으로 적용이 불가능하다는 한계가 있다. 따라서 본 연구의 목표는 모델 스스로가 데이터의 난이도를 해석하고 이를 바탕으로 학습 커리큘럼을 구성하게 함으로써, 계산 효율성을 높이고 MTL 성능을 향상시키는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 모델 기반의 난이도 점수(Difficulty Score)를 계산하여, 이를 통해 학습 데이터를 '쉬운 것에서 어려운 것' 순으로 배열하는 자동화된 커리큘럼 체계를 구축하는 것이다. 특히, 단순히 데이터셋 단위로 순서를 정하는 것을 넘어, 데이터셋의 경계를 허물고 개별 샘플 단위로 난이도를 평가하여 배열하는 방식을 제안한다.

주요 기여 사항은 다음과 같다.
1. **모델 기반 난이도 해석 도입**: 인간의 직관이 아닌, 모델의 예측 확신도(Confidence)를 기반으로 난이도를 측정하는 프레임워크를 제안하였다.
2. **두 가지 수준의 커리큘럼 구성**: 배열의 세밀함에 따라 Dataset-level과 Instance-level의 두 가지 접근 방식을 제시하였다.
3. **성능 향상 및 분석**: 12개의 NLP 데이터셋을 통해 제안 방법론이 기존 베이스라인 대비 평균적으로 유의미한 성능 향상을 가져옴을 입증하였으며, 특히 어려운 샘플(Difficult Instances)에서 그 효과가 극대화됨을 확인하였다.

## 📎 Related Works

기존의 MTL 연구들은 여러 작업을 동시에 학습함으로써 지식 전이 효과를 얻고 과적합을 방지하며 효율성을 높이려 하였다. 커리큘럼 학습 역시 MTL 설정에서 연구되었으나, 앞서 언급한 바와 같이 인간의 지각에 의존하거나 전수 조사를 수행하는 방식에 머물러 있었다.

본 논문은 이러한 한계를 극복하기 위해 모델의 Training Dynamics(학습 역학)와 예측 확신도를 활용하는 최신 연구들을 참고하였다. 특히 인간의 주관적 판단을 배제하고 모델이 직접 데이터의 난이도를 결정하게 함으로써, 기계의 해석 방식에 최적화된 학습 경로를 제공한다는 점에서 기존 방식과 차별화된다.

## 🛠️ Methodology

### 1. Difficulty Score Computation
모델이 데이터의 난이도를 판단하기 위해 다음 두 가지 방법 중 하나를 사용하여 점수 $s_i$를 계산한다. 점수가 높을수록 더 어려운 샘플임을 의미한다.

- **Cross Review Method**: 전체 학습 데이터 $D$를 $N$개의 메타 데이터셋($M_1, \dots, M_N$)으로 나눈다. 각 메타 데이터셋으로 $N$개의 동일한 아키텍처 모델을 학습시킨 후, 특정 샘플 $i$가 속하지 않은 나머지 모델들의 평균 예측 확신도(Confidence)를 계산하여 $1$에서 뺀다.
$$s_i = 1 - \frac{\sum_{j \in (1, \dots, N), j \neq k} c_{ji}}{N-1}$$
여기서 $c_{ji}$는 $M_j$ 모델이 샘플 $i$에 대해 가진 확신도이다.

- **Average Confidence Across Epochs**: 단일 모델을 $E$ 에폭까지 학습시키며, 각 체크포인트에서 정답에 대한 예측 확신도의 평균을 구하여 $1$에서 뺀다.
$$s_i = 1 - \frac{\sum_{j=1}^{E} c_{ji}}{E}$$
이 방법은 Cross Review 방식보다 계산 비용이 훨씬 적으면서도 효율적인 것으로 나타났다.

### 2. Proposed Techniques (Curriculum Arrangement)
계산된 난이도 점수를 바탕으로 학습 데이터를 $K$개의 스플릿($S_1, \dots, S_K$)으로 나누고 순차적으로 학습시킨다.

- **Dataset-level**: 각 데이터셋 $D_k$의 모든 인스턴스의 평균 난이도 점수 $d_k$를 기준으로 데이터셋의 순서를 정한다.
$$d_k = \frac{\sum_{i \in D_k} s_i}{|D_k|}$$
- **Instance-level**: 데이터셋의 경계를 무시하고 모든 개별 인스턴스를 난이도 순으로 정렬한다. 스플릿을 나누는 방식은 두 가지이다.
    - **Uniform Splitting**: 전체 데이터를 균등한 크기의 $K$개 그룹으로 나눈다.
    - **Distribution-based Splitting**: 난이도 점수의 분포에 따라 유사한 점수를 가진 샘플끼리 그룹화한다. 이 경우 그룹 간 크기가 다를 수 있다.

### 3. Training Procedure
Catastrophic Forgetting(치명적 망각)을 방지하기 위해, 현재 스플릿 $S_i$를 학습할 때 이전 스플릿들($S_j, j < i$)에서 일정 비율($frac$)의 샘플을 함께 샘플링하여 학습에 포함한다. 최종적으로는 전체 데이터셋 $D$를 사용하여 한 번 더 학습함으로써 모든 난이도 수준의 데이터를 통합한다.

## 📊 Results

### 실험 설정
- **데이터셋**: SNLI, MNLI, QNLI, Winogrande, PAWS 등 총 12개의 NLP 데이터셋을 사용하였다.
- **조건**: 저자원 환경(Low-resource regime)을 시뮬레이션하기 위해 데이터셋당 학습 샘플을 5,000개로 제한하였다. 모든 작업은 단일 모델로 처리 가능하도록 Span Identification QA 형식으로 통합하였다.
- **베이스라인**: Instance-level의 경우 Heterogeneous Batching(무작위 샘플링)을, Dataset-level의 경우 Random Order(무작위 순서)를 베이스라인으로 설정하였다.

### 정량적 결과
- **성능 향상**: Instance-level 기법은 베이스라인 대비 평균 **4.17%**, Dataset-level 기법은 평균 **3.15%**의 성능 향상을 보였다.
- **방법론 비교**: Average Confidence 방식이 Cross Review 방식보다 성능과 계산 효율성 측면 모두에서 우수하였다.
- **분할 전략**: Instance-level에서는 Distribution-based Splitting이 Uniform Splitting보다 약간 더 높은 성능을 보였는데, 이는 유사한 난이도의 샘플들이 모였을 때 더 강한 Inductive Bias를 제공하기 때문으로 분석된다.

### 분석 결과
- **난이도 분포**: 데이터셋마다 난이도 편차가 컸으며, 특히 Equate와 QNLI는 상대적으로 쉬운 작업으로, PAWS와 Winogrande는 어려운 작업으로 분류되었다.
- **어려운 샘플의 효과**: 테스트 세트 분석 결과, 제안된 커리큘럼 방식은 특히 난이도 점수가 높은(어려운) 샘플들에서 베이스라인 대비 큰 성능 향상을 보였다. 이는 본 기법이 어려운 작업을 해결하는 능력을 크게 개선함을 시사한다.

## 🧠 Insights & Discussion

본 연구는 모델이 스스로 정의한 난이도에 따라 학습 순서를 정하는 것이 MTL 성능을 높이는 데 효과적임을 입증하였다. 특히 Instance-level 접근법이 더 높은 성능을 보인 점은, 데이터셋이라는 인위적인 경계보다 실제 샘플의 난이도가 학습에 더 중요한 요소임을 의미한다.

**강점 및 가능성**:
- 본 기법은 범용적이어서 Dynamic Sampling이나 Pre-finetuning과 같은 다른 MTL 방법론과 결합하여 추가적인 성능 향상을 꾀할 수 있다.
- 어려운 샘플들에 대해 높은 효용성을 보이므로, 실제 정제되지 않은 복잡한 데이터셋이 많은 실무 환경에서 유용할 가능성이 크다.

**한계 및 논의사항**:
- **계산 오버헤드**: 난이도 점수를 계산하기 위해 추가적인 학습 과정이 필요하다. 다만, 이는 학습 단계에서만 발생하며 추론(Inference) 시에는 영향이 없다.
- **노이즈 데이터**: 잘못 라벨링된 노이즈 샘플은 매우 높은 난이도 점수를 갖게 되어 학습 후반부에 배치된다. 노이즈의 양이 적을 때는 문제가 없으나, 데이터셋에 노이즈가 많을 경우 모델 학습에 부정적인 영향을 줄 수 있다는 점이 한계로 지적되었다.

## 📌 TL;DR

본 논문은 MTL에서 인간의 직관이나 전수 조사가 아닌, 모델의 예측 확신도를 이용해 스스로 학습 순서를 결정하는 **Model-based Curriculum Learning**을 제안한다. 데이터셋 단위(Dataset-level)와 개별 샘플 단위(Instance-level)의 두 가지 전략을 제시하였으며, 실험 결과 Instance-level 방식이 평균 4.17%의 성능 향상을 보이며 특히 어려운 샘플 학습에 매우 효과적임을 확인하였다. 이 연구는 향후 더 복잡한 실제 데이터 환경에서 MTL 모델의 강건성을 높이는 데 기여할 것으로 기대된다.
