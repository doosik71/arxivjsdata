# Teacher-Student Curriculum Learning

Tambet Matiisen, Avital Oliver, Taco Cohen, John Schulman (2017)

## 🧩 Problem to Solve

본 논문은 딥러닝, 특히 강화학습(Reinforcement Learning)에서 발생하는 희소 보상(Sparse Rewards) 문제를 해결하기 위해 자동화된 커리큘럼 학습(Automatic Curriculum Learning) 프레임워크를 제안한다. 복잡한 작업의 경우, 보상을 얻기까지 필요한 단계가 많아질수록 무작위 탐색(Random Exploration)을 통해 해결책을 찾을 확률이 지수적으로 감소하는 문제가 발생한다.

이를 해결하기 위해 기존에는 쉬운 작업부터 어려운 작업으로 순차적으로 학습시키는 커리큘럼 학습(Curriculum Learning)이 사용되었다. 그러나 수동으로 커리큘럼을 설계하는 방식은 다음과 같은 한계가 있다. 첫째, 작업의 난이도를 정확히 정의하고 순서를 매기는 것이 어렵다. 둘째, 다음 단계로 넘어가기 위한 '숙달(Mastery)' 임계값을 결정하는 기준이 모호하다. 셋째, 어려운 작업을 배우는 동안 쉬운 작업을 잊어버리는 '망각(Forgetting)' 현상을 방지하기 위해 기존 작업을 적절히 혼합하는 비율을 설계하는 것이 매우 까다롭다. 따라서 본 논문의 목표는 이러한 수동 설계 과정 없이, 학생(Student)의 학습 상태를 모니터링하여 최적의 작업을 자동으로 선택하는 교사(Teacher) 알고리즘을 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 학생이 가장 빠른 진전을 보이고 있는 작업, 즉 학습 곡선의 기울기(Slope of the learning curve)가 가장 가파른 작업을 더 많이 연습하게 한다는 것이다. 이를 통해 학습 효율을 극대화하며, 동시에 학습 곡선의 기울기가 음수(Negative)가 되어 성능이 떨어지는 작업(망각이 발생하는 작업)을 다시 선택하게 함으로써 망각 문제를 자동으로 해결한다.

주요 기여 사항은 다음과 같다.

1. Teacher-Student Curriculum Learning (TSCL) 프레임워크를 부분 관찰 가능 마르코프 결정 과정(Partially Observable Markov Decision Process, POMDP)으로 정형화하였다.
2. 학습 진전(Learning Progress) 개념에 기반하여 망각 문제를 해결할 수 있는 다양한 교사 알고리즘 제품군을 제안하였다.
3. 지도 학습(LSTM을 이용한 십진수 덧셈)과 강화 학습(Minecraft 내비게이션)이라는 서로 다른 두 가지 환경에서 제안 방법론이 수동 설계된 커리큘럼과 대등하거나 이를 능가함을 입증하였다.

## 📎 Related Works

기존의 커리큘럼 학습 연구(Bengio et al., 2009 등)는 주로 사람이 직접 설계한 정적인 커리큘럼에 의존하였다. 반면, 최근의 일부 연구들은 자동 생성된 목표(Goal)를 통해 커리큘럼을 구축하려 시도하였다. 예를 들어, Alice와 Bob이라는 두 에이전트가 서로 궤적을 생성하고 복제하는 방식(Sukhbaatar et al., 2017)이 있으나, 이는 주로 탐색(Exploration)에 도움을 줄 뿐 최종 작업의 학습 성공을 보장하는 구조는 아니다.

또한, 학습 진전을 보상으로 사용하는 개념은 내재적 동기(Intrinsic Motivation) 연구와 맥을 같이 한다. 본 논문은 비정상 다중 팔 Bandit(Non-stationary Multi-armed Bandit) 문제의 알고리즘을 차용하여, 고정된 환경이 아닌 학생의 상태 변화(비정상성)에 대응하며 최적의 작업을 선택하는 방식을 취한다는 점에서 기존의 정적 커리큘럼이나 단순 탐색 보너스 방식과 차별화된다.

## 🛠️ Methodology

### 1. TSCL 프레임워크 및 POMDP 정형화

TSCL은 학생(Student) 모델과 이를 가이드하는 교사(Teacher) 모델의 상호작용으로 구성된다. 교사는 매 단계마다 학생이 연습할 작업을 선택하고, 학생은 그 결과로 점수(Score)를 반환한다. 본 논문은 이를 두 가지 POMDP 형태로 정의한다.

**가. Simple POMDP (강화 학습용)**

- **상태($s_t$):** 학생의 전체 상태(신경망 가중치 및 옵티마이저 상태)이며, 교사에게는 관찰되지 않는다.
- **행동($a_t$):** 교사가 선택한 이산적인 작업(Subtask)의 파라미터이다.
- **관찰($o_t$):** 선택된 작업 $i=a_t$에 대한 학생의 점수 $x_t^{(i)}$ (예: 에피소드 총 보상)이다.
- **보상($r_t$):** 해당 작업의 점수 변화량이다.
  $$r_t = x_t^{(i)} - x_{t'}^{(i)}$$
  여기서 $t'$는 해당 작업을 마지막으로 수행했던 시점이다.

**나. Batch POMDP (지도 학습용)**

- **행동($a_t$):** $N$개의 작업들에 대한 확률 분포 $a_t = (p_t^{(1)}, \dots, p_t^{(N)})$이다.
- **관찰($o_t$):** 학습 단계 후 모든 작업에 대한 점수 벡터 $o_t = (x_t^{(1)}, \dots, x_t^{(N)})$이다.
- **보상($r_t$):** 모든 작업의 점수 변화량의 합이다.
  $$r_t = \sum_{i=1}^{N} (x_t^{(i)} - x_{t-1}^{(i)})$$

교사의 최종 목표는 모든 작업의 최종 점수 합을 최대화하는 것이며, 이는 중간 단계의 작업들을 성공적으로 마스터하는 것이 최종 작업의 성공으로 이어진다는 가정을 바탕으로 한다.

### 2. Teacher 알고리즘

교사는 학습 곡선의 기울기를 추정하여 작업을 선택한다. 특히 망각을 방지하기 위해 보상의 절대값 $|Q|$를 기준으로 작업을 선택하는 전략을 사용한다.

- **Online Algorithm:** 지수 가중 이동 평균(EWMA)을 사용하여 각 작업의 기대 보상 $Q$를 추적한다.
  $$Q_{t+1}(a_t) = \alpha r_t + (1-\alpha)Q_t(a_t)$$
  이후 $\epsilon$-greedy 또는 Boltzmann 분포를 통해 다음 작업을 선택한다.
- **Naive Algorithm:** 각 작업을 $K$번 반복 수행한 후, 그 점수들에 대해 선형 회귀(Linear Regression)를 수행하여 기울기를 계산하고 이를 보상 $r_t$로 사용한다.
- **Window Algorithm:** 각 작업별로 최근 $K$개의 점수와 시간을 저장하는 FIFO 버퍼를 유지한다. 이 버퍼 내의 데이터로 선형 회귀를 수행해 기울기를 추정한다.
- **Sampling Algorithm:** 각 작업의 최근 $K$개 보상 버퍼에서 무작위로 하나를 샘플링하여, 가장 높은 보상을 가진 작업을 선택하는 Thompson Sampling 방식의 변형이다.

## 📊 Results

### 1. 십진수 덧셈 (Decimal Number Addition)

LSTM을 이용해 숫자의 길이에 따른 난이도가 다른 덧셈 문제를 풀게 하였다.

- **1차원 커리큘럼:** 숫자의 최대 길이를 기준으로 난이도를 정의하였다. 실험 결과, TSCL 알고리즘(특히 $|Q|$를 사용한 경우)이 균등 샘플링(Uniform Sampling) 및 수동 설계된 최적 커리큘럼(Combined)보다 더 빠른 학습 속도를 보였다.
- **2차원 커리큘럼:** 두 숫자의 길이를 각각 다르게 설정하여 난이도 순서가 불분명한 환경을 구축하였다. 이 경우 수동 커리큘럼이 매우 강력했으나, TSCL은 여전히 균등 샘플링과 경쟁 가능한 성능을 보였으며, 수동 방식과는 다른 독특한 학습 경로(더 짧은 숫자부터 학습)를 선택하는 경향을 보였다.

### 2. Minecraft 내비게이션

다양한 장애물(용암, 벽)이 포함된 5단계의 미로 탈출 과제를 수행하였다.

- **실험 설정:** PPO(Proximal Policy Optimization) 알고리즘을 학생 모델로 사용하였으며, 교사는 Window 알고리즘을 사용하였다.
- **결과:** 단순히 마지막 단계의 작업만 학습시키거나 균등 샘플링을 했을 때는 거의 진전이 없었다. 반면 TSCL은 정교하게 설계된 수동 커리큘럼과 대등한 성능을 보였으며, 수동 커리큘럼 없이도 복잡한 미로를 해결할 수 있음을 증명하였다.

## 🧠 Insights & Discussion

본 연구의 가장 큰 강점은 학습 진전(Learning Progress)이라는 직관적인 지표를 통해 커리큘럼 생성을 자동화했다는 점이다. 특히 보상의 절대값 $|Q|$를 사용함으로써, 성능이 급격히 향상되는 작업뿐만 아니라 성능이 떨어지고 있는(망각 중인) 작업을 적시에 재학습시키는 메커니즘을 성공적으로 구현하였다.

실험을 통해 도출된 주요 통찰은 다음과 같다.

- **망각 방지의 중요성:** $|Q|$를 통한 작업 선택이 거의 모든 알고리즘에서 성능 향상을 가져왔다.
- **알고리즘의 상황별 효율성:** 모든 상황에 최적인 단일 알고리즘은 없었으며, 1D 과제에서는 Window, 2D 과제에서는 Naive 알고리즘이 우수했다. Sampling 알고리즘은 하이퍼파라미터 튜닝이 적으면서도 경쟁력 있는 성능을 보였다.
- **균등 샘플링의 의외성:** 특히 2D 과제에서 균등 샘플링이 생각보다 효율적이었다는 점은, 작업 간의 관계가 복잡할 때 단순한 접근이 유효할 수 있음을 시사한다.

한계점으로는 현재의 프레임워크가 이산적인 작업 파라미터 설정에만 국한되어 있다는 점이 있다. 또한, 학생 모델의 내부 상태를 직접 관찰하지 않고 오직 외부 점수(Score)에만 의존하므로, 점수 산출에 노이즈가 심할 경우 교사의 결정이 불안정해질 수 있는 가정이 존재한다.

## 📌 TL;DR

본 논문은 학생의 학습 곡선 기울기가 가장 가파른 작업을 우선 선택하고, 성능이 하락하는 작업을 재학습시키는 **Teacher-Student Curriculum Learning (TSCL)** 프레임워크를 제안한다. 이를 통해 사람이 직접 난이도를 설계해야 하는 번거로움을 없애고, LSTM 덧셈 및 Minecraft 내비게이션 과제에서 수동 설계 커리큘럼에 필적하거나 이를 능가하는 성능을 달성하였다. 이 연구는 향후 복잡한 강화 학습 환경에서 보상이 희소한 문제를 해결하기 위한 자동화된 학습 경로 설계에 중요한 기초를 제공한다.
