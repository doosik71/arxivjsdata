# Curriculum Learning for Reinforcement Learning Domains: A Framework and Survey

Sanmit Narvekar, Bei Peng, Matteo Leonetti, Jivko Sinapov, Matthew E. Taylor, and Peter Stone (2020)

## 🧩 Problem to Solve

강화학습(Reinforcement Learning, RL)은 에이전트가 환경과의 상호작용을 통해 보상을 최대화하는 정책을 학습하는 강력한 패러다임이다. 그러나 복잡한 도메인에서 RL 에이전트를 학습시키기 위해서는 방대한 양의 환경 상호작용 데이터가 필요하며, 이는 실제 시나리오에서 비용이 매우 많이 들거나 불가능한 경우가 많다. 특히 보상 신호가 희소(sparse)하거나 상태 표현이 복잡한 경우, 에이전트가 처음부터(from scratch) 최적의 정책을 학습하는 것은 매우 어렵다.

이러한 문제를 해결하기 위해 전이 학습(Transfer Learning)이 도입되었으나, 구체적으로 어떤 순서로 과업(task)이나 데이터를 제공해야 학습 효율을 극대화할 수 있는지에 대한 체계적인 정의와 분류가 부족한 상태였다. 본 논문의 목표는 강화학습 도메인에서의 커리큘럼 학습(Curriculum Learning, CL)에 대한 포괄적인 프레임워크를 제공하고, 기존의 다양한 CL 방법론들을 가정, 능력, 목표에 따라 체계적으로 분류하고 분석하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 강화학습에서의 커리큘럼 학습을 수학적으로 정형화하고, 이를 기반으로 기존 연구들을 분류할 수 있는 다차원적인 분류 체계(Taxonomy)를 제안한 것이다. 

가장 중심적인 아이디어는 커리큘럼을 단순히 '과업의 나열'로 보는 것이 아니라, 경험 샘플들의 부분 집합을 정점으로 하는 **방향성 비순환 그래프(Directed Acyclic Graph, DAG)**로 정의한 것이다. 이를 통해 선형적인 순서뿐만 아니라, 여러 기초 과업이 하나의 상위 과업으로 이어지는 복잡한 학습 경로를 표현할 수 있게 하였다. 또한, CL의 전체 과정을 **과업 생성(Task Generation)**, **순서 결정(Sequencing)**, **전이 학습(Transfer Learning)**이라는 세 가지 핵심 요소로 구분하여 분석함으로써, 연구자들이 자신의 문제에 맞는 CL 구성 요소를 선택하고 설계할 수 있는 가이드라인을 제시하였다.

## 📎 Related Works

논문은 강화학습과 전이 학습의 기본 개념을 먼저 설명한다. 강화학습은 에피소드 기반의 마르코프 결정 과정(Markov Decision Process, MDP)으로 정의되며, 에이전트는 보상의 합인 Return $G_t$를 최대화하는 최적 정책 $\pi^*$를 찾는 것을 목표로 한다.

전이 학습은 소스 과업(source task)에서 얻은 지식을 타겟 과업(target task)으로 옮겨 학습 속도를 높이는 방법이다. 기존의 전이 학습 연구들은 주로 '하나의 소스 $\rightarrow$ 하나의 타겟' 구조에 집중하였으나, 커리큘럼 학습은 이를 확장하여 '여러 중간 과업 $\rightarrow$ 최종 타겟 과업'으로 이어지는 연속적인 전이 과정을 다룬다. 

본 논문은 기존 연구들이 커리큘럼이라는 개념을 암묵적으로 사용(예: AlphaGo의 self-play)했음에도 불구하고, 이를 명시적인 프레임워크로 통합하여 논의한 사례가 적었다는 점을 지적하며 기존 접근 방식과의 차별성을 강조한다.

## 🛠️ Methodology

### 1. 커리큘럼의 정형적 정의
본 논문은 커리큘럼 $\mathcal{C}$를 다음과 같이 정의한다:
$$\mathcal{C} = (V, E, g, \mathcal{T})$$
여기서 $V$는 정점(vertices)의 집합, $E$는 방향성 에지(directed edges)의 집합, $\mathcal{T}$는 과업들의 집합, 그리고 $g: V \rightarrow \mathcal{P}(D_{\mathcal{T}})$는 각 정점을 경험 샘플의 부분 집합에 매핑하는 함수이다. 에지 $\langle v_j, v_k \rangle$는 $v_j$에 연관된 샘플들을 먼저 학습한 후 $v_k$를 학습해야 함을 의미한다.

이 일반적인 정의는 다음과 같이 단순화될 수 있다:
- **단일 과업 커리큘럼(Single-task Curriculum):** 타겟 과업 하나 내에서 샘플의 순서만 조정하는 경우.
- **과업 수준 커리큘럼(Task-level Curriculum):** 각 정점이 하나의 독립된 과업(MDP)에 대응하는 경우.
- **시퀀스 커리큘럼(Sequence Curriculum):** 그래프 구조가 단순한 선형 체인(linear chain) 형태인 경우.

### 2. 커리큘럼 학습의 3대 구성 요소
커리큘럼 학습 방법론은 다음 세 단계의 파이프라인으로 구성된다.

**가. 과업 생성 (Task Generation)**
최종 과업을 해결하는 데 도움이 될만한 중간 과업들을 생성하는 과정이다. 
- **방식:** 도메인 전문가가 직접 설계하거나, MDP의 파라미터를 조정하여 자동으로 생성한다. 
- **예시:** 'Quick Chess'처럼 보드 크기를 줄이거나 기물의 수를 제한하여 쉬운 버전의 게임을 만드는 방식이 이에 해당한다.

**나. 순서 결정 (Sequencing)**
생성된 과업이나 샘플들을 어떤 순서로 배치할지 결정하는 과정이다. 본 논문은 이를 다섯 가지 범주로 분류한다:
1. **샘플 시퀀싱(Sample Sequencing):** 환경은 그대로 두고 데이터 재생 순서만 조정 (예: Prioritized Experience Replay).
2. **공동 학습(Co-learning):** 에이전트 간의 상호작용(경쟁/협력)을 통해 자연스럽게 난이도가 상승 (예: Self-play).
3. **보상 및 상태 분포 변경:** 보상 함수나 시작/종료 상태 분포를 조정하여 난이도 조절.
4. **제한 없는 과업 구성:** MDP의 모든 요소를 자유롭게 변경. 여기서는 **커리큘럼 MDP(CMDP)**라는 개념이 도입된다.
   - **CMDP:** '교사' 에이전트가 '학생' 에이전트의 현재 상태(정책/가중치)를 관찰하고, 다음으로 학습할 최적의 과업을 선택하는 상위 수준의 MDP이다.
5. **인간 참여형(Human-in-the-loop):** 전문가나 일반 사용자가 직접 순서를 설계.

**다. 전이 학습 (Transfer Learning)**
과업 $m_i$에서 학습한 지식을 $m_{i+1}$로 전달하는 방법이다.
- **저수준 지식 전이:** 정책(Policy), 가치 함수(Value Function), 모델(Model)의 파라미터를 직접 초기값으로 사용.
- **고수준 지식 전이:** 부분 정책(Partial Policies), 옵션(Options), 스킬(Skills), 셰이핑 보상(Shaping Rewards) 등을 통해 학습 방향을 가이드.

## 📊 Results

본 논문은 특정 알고리즘의 성능을 측정하는 실험 논문이 아니라, 수많은 기존 연구를 분석한 **서베이(Survey) 논문**이다. 따라서 개별 실험 결과보다는 분류 체계에 따른 연구 경향성을 제시한다.

### 1. 정량적 평가 지표
전이 학습 및 커리큘럼 학습의 효과를 측정하기 위해 다음과 같은 지표들이 사용됨을 정리하였다:
- **Time to Threshold:** 특정 성능 임계값 $\delta$에 도달하기까지 걸리는 시간.
- **Asymptotic Performance:** 학습이 수렴한 후의 최종 성능.
- **Jumpstart:** 전이 직후 초기 성능의 상승 폭.
- **Total Reward Ratio:** 학습 전체 과정에서 누적된 총 보상의 비율.

### 2. 분석 결과 요약 (Table 1, 2, 3 참조)
- **샘플 시퀀싱** 방법(PER, HER 등)은 환경을 수정할 필요가 없어 범용성이 높으며, 특히 Atari 게임과 같은 비디오 게임 도메인에서 효과적이다.
- **공동 학습(Co-learning)** 방식은 AlphaStar와 같은 복잡한 다중 에이전트 환경에서 인간 수준의 성능을 달성하는 데 결정적인 역할을 하였다.
- **자동화된 시퀀싱** 연구는 최근 증가하는 추세이나, 여전히 많은 연구가 도메인 전문가의 수동 설계에 의존하고 있다.
- **전이 방법**으로는 가치 함수 전이와 정책 전이가 가장 흔하게 사용되며, 최근에는 신경망 구조를 확장하는 Progressive Neural Networks와 같은 방법이 망각(forgetting) 문제를 해결하기 위해 사용된다.

## 🧠 Insights & Discussion

### 강점 및 기여
본 논문은 파편화되어 있던 RL 커리큘럼 학습 연구들을 'Task Generation $\rightarrow$ Sequencing $\rightarrow$ Transfer'라는 단일 프레임워크로 통합하였다. 특히 커리큘럼을 DAG로 정의함으로써, 복잡한 전제 조건(prerequisites)이 필요한 학습 경로를 수학적으로 표현할 수 있게 한 점이 매우 탁월하다.

### 한계 및 미해결 과제 (Open Problems)
논문은 다음과 같은 비판적 관점과 향후 연구 방향을 제시한다:
1. **완전 자동화된 과업 생성의 부재:** 대부분의 연구가 과업 풀(pool)이 이미 주어졌다고 가정하거나 단순한 규칙에 의존한다. 인간의 개입 없이 진정으로 유용한 중간 과업을 생성하는 메커니즘이 필요하다.
2. **다양한 지식 전이의 결합:** 현재는 하나의 전이 방법(예: 가치 함수 전이)만 사용하지만, 과업의 특성에 따라 스킬, 모델, 보상을 적절히 섞어서 전이하는 하이브리드 방식이 필요하다.
3. **Sim-to-Real 커리큘럼:** 시뮬레이션에서 학습한 '커리큘럼의 구조(순서)' 자체를 실제 로봇에 적용하는 연구가 유망할 것으로 보인다.
4. **이론적 기반 부족:** 커리큘럼이 왜 작동하는지에 대한 경험적 증거는 많으나, 수렴 속도나 성능 향상을 보장하는 수학적 이론(Theoretical results)은 매우 부족한 상태이다.

## 📌 TL;DR

본 논문은 강화학습에서 학습 효율을 높이기 위한 **커리큘럼 학습(CL)**의 통합 프레임워크를 제안한 서베이 논문이다. 커리큘럼을 경험 샘플들의 **방향성 비순환 그래프(DAG)**로 정의하고, 전체 과정을 **[과업 생성 $\rightarrow$ 순서 결정 $\rightarrow$ 지식 전이]**의 세 단계로 체계화하였다. 수많은 기존 방법론을 7가지 차원으로 분류하여 정리하였으며, 특히 자동화된 과업 생성 및 이론적 분석의 필요성을 강조하였다. 이 연구는 향후 복잡한 RL 문제를 풀기 위한 체계적인 학습 경로 설계의 기초 지도로 활용될 가능성이 크다.
