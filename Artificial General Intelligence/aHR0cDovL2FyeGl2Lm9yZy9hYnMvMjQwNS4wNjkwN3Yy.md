# AIOS Compiler: LLM as Interpreter for Natural Language Programming and Flow Programming of AI Agents

Shuyuan Xu, Zelong Li, Kai Mei, Yongfeng Zhang (2024)

## 🧩 Problem to Solve

본 논문은 프로그래밍 언어가 점차 읽기 쉬운 방향으로 발전해 온 흐름에 주목하여, 자연어(Natural Language)를 일종의 프로그래밍 언어로 활용하여 AI 에이전트를 설계하고자 한다. 자연어 프로그래밍은 전문적인 코딩 지식이 없는 사용자에게도 프로그래밍의 민주화를 제공하고 높은 유연성과 사용성을 부여할 수 있다는 점에서 매우 중요하다.

그러나 자연어는 본질적으로 모호성(Ambiguity), 막연함(Vagueness), 그리고 장황함(Verbosity)이라는 특성을 가지고 있다. 이러한 특성 때문에 자연어로 작성된 프로그래밍 로직을 정확하게 이해하고 실행할 수 있는 인터프리터(Interpreter)를 개발하는 것은 매우 어려운 과제이다. 따라서 본 연구의 목표는 대규모 언어 모델(LLM)의 뛰어난 언어 이해 능력을 활용하여, 자연어 프로그램을 해석하고 실행할 수 있는 새로운 시스템인 CoRE(Code Representation and Execution)를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 LLM을 단순히 솔루션을 생성하는 도구가 아니라, 사람이 설계한 자연어 기반의 로직을 실행하는 **인터프리터(Interpreter)**로 사용하는 것이다. 주요 기여 사항은 다음과 같다.

- **CoRE 언어 설계**: 자연어 프로그래밍, 의사 코드(Pseudo-code) 프로그래밍, 그리고 플로우 프로그래밍(Flow programming)을 하나의 통일된 표현 방식으로 통합하는 CoRE 언어 구문을 설계하였다.
- **CoRE 시스템 제안**: LLM을 인터프리터로 사용하여 자연어 명령어를 단계별로 해석하고 실행하는 시스템을 구축하였다. 이 시스템은 실행 효율을 높이기 위해 외부 메모리(External Memory)의 정보 검색 기능과 외부 도구(External Tools) 호출 능력을 통합하였다.
- **실무적 검증**: OpenAGI 벤치마크 데이터셋을 통해 제안된 시스템이 기존의 Zero-shot, Few-shot, Chain-of-Thought(CoT) 방식보다 우수한 성능을 보임을 정량적으로 입증하였다.

## 📎 Related Works

### 관련 연구 및 한계

- **자연어 프로그래밍(Natural Language Programming)**: 과거의 연구들은 주로 자연어를 기존의 프로그래밍 언어(Python, Bash 등)나 도메인 특화 언어(DSL)로 번역하는 방식에 집중하였다. 하지만 이는 새로운 DSL을 매번 설계해야 하는 확장성의 한계가 있었다.
- **LLM 기반 문제 해결**: Chain-of-Thought(CoT)나 ReAct와 같은 기법들은 LLM이 스스로 추론 체인을 생성하여 문제를 해결하게 한다. 그러나 이러한 방식은 LLM의 생성 능력에 전적으로 의존하므로, 복잡한 논리 구조를 정밀하게 제어하기 어렵다는 단점이 있다.

### 차별점

CoRE 시스템은 LLM이 직접 해결책을 생성하게 하는 기존 방식과 달리, 사람이 설계한 논리적 구조(프로그램)를 LLM이 **해석(Interpret)**하여 실행하게 한다. 이는 인간의 창의적인 솔루션 설계 능력과 LLM의 강력한 실행 능력을 결합하여, 더 복잡하고 정밀한 제어가 가능한 에이전트 프로그래밍을 가능하게 한다.

## 🛠️ Methodology

### CoRE 언어 구문 (CoRE Language Syntax)

자연어 지침을 논리적으로 구조화하기 위해, 각 단계(Step)를 다음 네 가지 구성 요소로 정의하며, 각 요소는 `:::` 구분자로 분리한다.

1. **Step Name**: 각 단계를 식별하는 고유 이름 (함수 식별자와 유사).
2. **Step Type**: 작업의 성격을 규정하며 세 가지 타입으로 나뉜다.
   - `Process`: 특정 작업을 수행하고 다음 단계로 이동하는 절차적 문장.
   - `Decision`: 조건에 따라 실행 경로를 결정하는 조건문 (if-else와 유사).
   - `Terminal`: 프로그램의 종료를 알리는 문장.
3. **Step Instruction**: 해당 단계에서 수행해야 할 구체적인 작업 내용을 자연어로 기술.
4. **Step Connection**: 다음 단계로의 흐름을 정의. `Process`는 단일 경로를, `Decision`은 조건에 따른 다중 경로를 가진다.

이 구문을 통해 프로그래밍의 3대 기본 제어 구조인 **순차(Sequence)**, **선택(Selection)**, **반복(Iteration)**을 모두 구현할 수 있다. 특히 반복문은 `Decision` 타입을 통해 루프 조건을 확인하고, 조건이 충족되지 않았을 때 이전 단계로 연결하여 구현한다.

### LLM 인터프리터 실행 절차

CoRE 시스템은 하나의 단계를 실행하기 위해 다음과 같은 4단계 파이프라인을 거친다.

**1. 메모리 기반 관찰 정보 검색 (Observation Retrieval)**
LLM의 컨텍스트 윈도우 제한 문제를 해결하기 위해 모든 중간 결과를 메모리에 저장한다. 인터프리터는 현재 단계의 지침을 수행하는 데 필요한 관련 기록만을 메모리에서 선택적으로 검색하여 가져온다.

**2. 입력 프롬프트 구성 (Input Prompt Construction)**
검색된 정보와 현재 상태를 결합하여 LLM에게 전달할 프롬프트를 생성한다. 프롬프트는 다음 요소들로 구성된다.
$$\text{Input Prompt} = \text{Task Description} + \text{Current Progress} + \text{Observation} + \text{Current Instruction}$$

**3. 출력 분석 및 도구 호출 (Output Analysis)**
LLM이 생성한 응답을 분석하여, 도메인 특화 지식이나 실시간 정보가 필요한 경우 외부 도구를 호출한다. 도구 사용이 결정되면 적절한 도구 이름과 인자(Arguments)를 결정하여 실행하고, 그 결과(Observation)를 다시 시스템에 통합한다.

**4. 분기 분석 (Branching Analysis)**
현재 단계가 `Decision` 타입인 경우, LLM은 실행 결과가 어떤 분기 조건에 부합하는지 판단한다. 이를 통해 다음으로 실행할 가장 적절한 단계(Step)를 결정하고 프로그램 흐름을 제어한다.

## 📊 Results

### 실험 설정

- **LLM 모델**: 폐쇄형 모델인 GPT-4(gpt-4-1106-preview)와 오픈소스 모델인 Mixtral-8x7B를 사용하였다.
- **비교 대상 (Baselines)**: Zero-shot, Chain-of-Thought(CoT), Few-shot Learning.
- **데이터셋 및 지표**: OpenAGI 벤치마크를 사용하였으며, 작업 성격에 따라 CLIP Score(텍스트-이미지 유사도), BERT Score(텍스트 생성 성능), ViT Score(이미지 유사도)를 측정하였다.

### 주요 결과

- **정량적 성능**: Table 1의 결과에 따르면, CoRE 방식은 Mixtral과 GPT-4 두 모델 모두에서 평균적으로 가장 높은 성능을 기록하였다.
- **모델별 차이**: 오픈소스 모델인 Mixtral을 사용했을 때 CoRE는 Zero-shot과 CoT보다 모든 작업에서 우수했으며, Few-shot보다도 평균 점수와 Task 2에서 더 나은 성능을 보였다.
- **GPT-4 결과**: GPT-4 환경에서도 CoRE는 특히 Task 2(BERT Score)와 평균 점수에서 가장 우수한 성능을 기록하였다. 특히 Few-shot 방식에 출력 예시를 제공하지 않았음에도 불구하고 더 높은 평균 성능을 보였다는 점이 주목할 만하다.
- **유효 계획 비율**: `% of Valid Plans` 지표에서 CoRE는 GPT-4 기준 $92.31\%$라는 매우 높은 수치를 기록하여, 실행 가능한 계획을 세우는 능력이 탁월함을 입증하였다.

## 🧠 Insights & Discussion

### 강점

본 연구는 자연어를 프로그래밍 언어의 인터페이스로 활용함으로써, 비전문가도 AI 에이전트를 설계할 수 있는 '프로그래밍의 민주화' 가능성을 보여주었다. 특히 LLM을 생성기가 아닌 인터프리터로 배치함으로써, 인간이 설계한 논리적 구조를 유지하면서도 LLM의 유연한 언어 처리 능력을 활용할 수 있다는 점이 큰 강점이다.

### 한계 및 비판적 해석

현재의 CoRE 시스템은 사람이 직접 작성한(manually crafted) 프로그램에 의존하고 있다. 자연어의 본질적인 모호성으로 인해, 사람이 작성한 지침 자체가 비효율적이거나 부정확할 경우 시스템 전체의 성능이 저하될 위험이 있다. 또한 실험 결과에서 나타나듯, 동일한 CoRE 프로그램이라도 사용하는 LLM의 자연어 이해 능력에 따라 성능 차이가 발생한다는 점은 인터프리터의 성능이 기반 모델의 역량에 종속적임을 시사한다.

### 향후 방향

논문에서는 자연어 프로그래밍 지침을 자동으로 생성하는 시스템을 개발하여 표준화된 명확성을 확보하고, 다국어 지원 및 실시간 디버깅 기능을 추가하여 교육적 활용도를 높이는 방향을 제시하고 있다.

## 📌 TL;DR

본 논문은 자연어, 의사 코드, 플로우 차트를 통합하는 **CoRE 언어**와 이를 실행하는 **LLM 기반 인터프리터 시스템**을 제안한다. LLM을 인터프리터로 활용하여 외부 메모리와 도구를 결합해 단계별로 지침을 수행함으로써, 전문 코딩 지식 없이도 정밀한 AI 에이전트를 구축할 수 있게 한다. OpenAGI 벤치마크에서 기존의 CoT나 Few-shot 방식보다 우수한 성능을 입증하였으며, 이는 향후 일반 사용자가 직접 AI 에이전트를 설계하고 배포하는 생태계 구축에 중요한 기반이 될 것으로 보인다.
