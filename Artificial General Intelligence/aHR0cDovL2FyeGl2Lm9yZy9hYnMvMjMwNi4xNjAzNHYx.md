# Stone Needle: A General Multimodal Large-scale Model Framework towards Healthcare

Weihua Liu, Yong Zuo (2024)

## 🧩 Problem to Solve

의료 분야의 데이터는 의료 영상, 임상 보고서, 오디오, 비디오 등 다양한 모달리티(modality)로 구성되어 있으며, 정확한 진단을 위해서는 이러한 다중 모달 데이터를 종합적으로 분석하는 것이 필수적이다. 그러나 현재의 대규모 인공지능 모델들은 주로 단일 모달리티의 인지 능력에 집중하고 있으며, 여러 모달리티를 통합적으로 처리하는 능력이 부족하다.

본 논문의 목표는 의료 애플리케이션에 특화된 일반적인 다중 모달 대규모 모델 프레임워크인 **Stone Needle**을 제안하는 것이다. 이를 통해 단일 모달 시스템의 한계를 극복하고, 텍스트, 이미지, 비디오, 오디오를 통합하여 복잡한 의료 정보를 처리함으로써 진단 정확도를 높이고 치료 권장 사항 및 환자 케어의 질을 향상시키고자 한다.

## ✨ Key Contributions

Stone Needle의 핵심 아이디어는 **모듈형 파이프라인**을 통해 다양한 의료 모달리티를 통합하고, 사용자의 의도를 분석하여 최적의 전문 모델을 선택적으로 활용하는 구조를 설계한 것이다. 주요 기여 사항은 다음과 같다.

- **다중 모달리티 통합**: 텍스트, 이미지, 비디오, 오디오를 모두 수용하는 프레임워크를 구축하여 단일 모달 시스템의 정보 제한 문제를 해결하였다.
- **의도 분석 기반의 상호작용**: 다회차 대화(multi-round dialogue) 과정에서 과거 대화 이력과 현재 입력을 분석하여, 해당 태스크에 가장 적합한 의료 기초 모델(Medical Foundation Model, MFM)을 동적으로 선택하는 메커니즘을 구현하였다.
- **범용성 및 확장성**: 특정 태스크에 국한되지 않은 일반적인 프레임워크를 제공함으로써, 의료 전문가나 개발자가 필요에 따라 특정 의료 데이터나 태스크에 맞춰 맞춤형으로 조정할 수 있는 확장성을 확보하였다.

## 📎 Related Works

본 논문은 세 가지 관련 연구 분야를 검토한다.

- **Multimodal Large Models**: Visual ChatGPT, Kosmos-1, LLaVA, AudioGPT 등이 언급된다. 이들은 시각적 또는 청각적 정보를 언어 모델과 결합하여 이해도를 높였으나, 의료 도메인의 특수성과 복잡성을 완전히 해결하지는 못했다.
- **Dialogue Large Language Models**: LLaMA, GPT-3, GPT-4와 같은 모델들은 자연어 이해 및 생성 능력은 뛰어나지만, 기본적으로 단일 모달(텍스트) 중심이며 다른 모달리티의 콘텐츠를 생성하거나 처리하는 능력이 부족하다.
- **Medical Large Model Frameworks**: ChatDoctor, ChatCAD 등이 의료 데이터셋을 활용한 파인튜닝이나 이미지-텍스트 결합을 시도하였다. 하지만 텍스트, 이미지, 비디오, 오디오를 모두 아우르는 포괄적인 다중 모달 의료 프레임워크는 여전히 부족한 상태이다.

Stone Needle은 이러한 기존 연구들의 한계를 넘어, 의료 환경에서 필요한 모든 주요 모달리티를 통합하고 의도 분석 단계를 통해 전문 모델을 유연하게 활용한다는 점에서 차별점을 가진다.

## 🛠️ Methodology

Stone Needle은 입력 데이터 $\mathbf{q}_n$에 대해 의도 분석, 의료 기초 모델 선택, 프롬프트 관리, 그리고 최종 언어 생성을 거치는 파이프라인으로 구성된다. 전체적인 응답 생성 과정은 다음과 같은 함수로 표현된다.

$$\mathbf{a}_n = \text{MMF}(\mathbf{q}_n, \mathcal{C})$$

여기서 $\mathcal{C} = \{(\mathbf{q}_1, \mathbf{a}_1), (\mathbf{q}_2, \mathbf{a}_2), \dots, (\mathbf{q}_{n-1}, \mathbf{a}_{n-1})\}$는 과거의 대화 이력이다.

### 3.1 Intent Analysis (의도 분석)
입력 $\mathbf{q}_n$은 텍스트 부분 $\mathbf{q}_n(t)$와 기타 리소스 부분 $\mathbf{q}_n(s)$ (이미지, 비디오, 오디오)로 나뉜다. 의도 분석 모듈 $\mathcal{T}$는 현재 입력과 대화 이력을 바탕으로 어떤 의료 기초 모델(MFM)을 사용할지 결정하며, 다음과 같이 모델 $M_i$와 관련 확률 $p_{M_i}$를 출력한다.

$$(M_i, p_{M_i}) = \mathcal{T}(\mathbf{q}_n, \mathcal{C})$$

최종적으로 가장 높은 확률을 가진 모델이 선택된다.
$$\text{MFM} = \max p_{M_i} M_i$$
만약 적절한 모델이 없다면 $M_0$가 선택되어 이후 처리 단계로 바로 넘어간다.

### 3.2 Medical Foundation Models (MFM)
MFM은 $m$개의 서로 다른 태스크 모델들의 앙상블 $\{M_1, M_2, \dots, M_m\}$이다. 선택된 MFM은 입력 $\mathbf{q}_k(s)$를 처리하여 결과 $\mathbf{R}_n$ (예: 이미지 설명, 건강 지표 값, 음성 전사 텍스트 등)을 생성한다.

$$\mathbf{R}_n = \begin{cases} \text{MFM}(\mathbf{q}_k(s)), & \text{if } \text{MFM} \neq M_0 \\ \emptyset, & \text{if } \text{MFM} = M_0 \end{cases}$$

특이점은 현재 쿼리에 리소스가 없더라도 과거 쿼리($k \in \{n, n-1, \dots, 1\}$)에서 필요한 리소스를 자동으로 검색하여 활용한다는 점이다.

### 3.3 Prompt Manager (프롬프트 관리자)
프롬프트 관리자 $\mathcal{P}$는 MFM의 결과 $\mathbf{R}_n$, 현재 입력 $\mathbf{q}_n$, 대화 이력 $\mathcal{C}$를 결합하여 최종 프롬프트 $\mathcal{P}_n$을 구성한다.

$$\mathcal{P}_n = \mathcal{P}(\mathbf{R}_n, \mathbf{q}_n, \mathcal{C})$$

이 과정에서 **Structured Retriever**가 포함되어 지식 베이스(Knowledge Base)로부터 질병, 증상, 검사 항목과 관련된 속성을 검색하여 엔티티를 정규화하고 주석을 달아 프롬프트의 문맥 이해도를 높인다.

### 3.4 Medical Language Module (MLM)
최종 응답을 생성하는 MLM은 `gpt-3.5-turbo`를 기반으로 하며, 다음과 같이 작동한다.

$$\mathbf{a}_n = \text{MLM}(\mathcal{P}_n)$$

의료 전문성을 확보하기 위해 MLM은 3단계 학습 과정을 거친다:
1. **General Pre-training**: 대규모 일반 질의응답 데이터셋으로 사전 학습한다.
2. **Domain Fine-tuning**: 전자 건강 기록(EHR), 의료 문헌 등 특정 의료 도메인 데이터로 파인튜닝한다.
3. **Medical QA Fine-tuning**: 실제 의료 쿼리와 전문가 답변으로 구성된 데이터셋으로 최종 파인튜닝하여 전문가 수준의 답변을 생성하게 한다.

## 📊 Results

### 실험 설정
- **데이터셋**: 익명화된 환자 기록을 사용하였다. 훈련 데이터는 텍스트 865,275개, 이미지 428,391개, 비디오 421,785개, 오디오 463,814개로 구성되며, 평가 데이터는 각 모달리티별로 1,000개씩 사용되었다.
- **하드웨어**: Tesla A100 GPU를 사용하였다.
- **평가 방법**: 의료 전문가의 진단 및 치료 권장 사항을 Ground Truth로 설정하고, Accuracy, Precision, Recall, F1-score를 통해 정량 평가를 수행하였다.

### 정량적 결과

### 결과 분석
GPT-4나 LLaMA와 같은 언어 모델보다 텍스트 작업에서 더 높은 정확도를 보였으며, 이미지-텍스트 기반의 Visual ChatGPT나 LLaVA보다 이미지 처리 능력뿐만 아니라 비디오, 오디오 처리 능력 면에서 압도적인 우위를 점하였다. 이는 의료 도메인에 특화된 MFM과 MLM의 파이프라인 구조가 효과적으로 작용했음을 시사한다.

## 🧠 Insights & Discussion

**강점**
- **모듈성(Modularity)**: 의도 분석 단계에서 적절한 MFM을 선택함으로써, 모든 데이터를 하나의 거대 모델이 처리하게 하는 대신 전문 모델을 효율적으로 활용하였다.
- **포괄적 모달리티 지원**: 의료 현장에서 발생하는 거의 모든 데이터 형태(텍스트, 이미지, 비디오, 오디오)를 통합 처리할 수 있는 체계를 구축하였다.
- **지식 베이스 결합**: Prompt Manager 내의 Structured Retriever를 통해 단순 생성 모델의 환각(Hallucination) 문제를 완화하고 의학적 사실에 기반한 응답을 유도하였다.

**한계 및 논의사항**
- **MFM의 세부 구조 미비**: 논문에서 $\text{MFMs} = \{M_1, M_2, \dots, M_m\}$라고 명시하였으나, 각 개별 모델 $M_i$가 구체적으로 어떤 아키텍처를 사용하는지, 혹은 어떻게 학습되었는지에 대한 상세 설명이 부족하다.
- **학습 손실 함수 부재**: MLM의 파인튜닝 과정은 설명되어 있으나, 구체적인 손실 함수(Loss Function)나 최적화 알고리즘에 대한 수학적 명시가 없다.
- **데이터셋의 출처**: 사용된 대규모 익명 데이터셋의 구체적인 출처나 수집 방법이 명확하지 않아 실험의 재현 가능성에 의문이 있을 수 있다.

## 📌 TL;DR

본 논문은 의료 분야의 다중 모달 데이터를 통합 처리하기 위한 프레임워크인 **Stone Needle**을 제안한다. 이 시스템은 **의도 분석 $\rightarrow$ 전문 모델(MFM) 선택 $\rightarrow$ 프롬프트 최적화 $\rightarrow$ 의료 언어 모델(MLM) 생성**의 파이프라인을 통해 텍스트, 이미지, 비디오, 오디오를 모두 처리하며, 실험 결과 GPT-4 및 LLaVA 등 기존 모델보다 의료 진단 및 권장 사항 생성에서 높은 정확도를 보였다. 이 연구는 향후 AI 기반의 정밀 의료 진단 및 환자 케어 시스템 구축에 중요한 기초 프레임워크가 될 가능성이 크다.
