# Surgical Phase and Instrument Recognition: How to identify appropriate Dataset Splits

Georgii Kostiuchik, Lalith Sharan, Benedikt Mayer, Ivo Wolf, Bernhard Preim and Sandy Engelhardt (2023)

## 🧩 Problem to Solve

본 논문은 외과 수술 워크플로우 인식(Surgical Workflow Recognition) 및 기구 인식(Instrument Recognition)을 위한 머신러닝 모델 학습 시, 데이터셋 분할(Dataset Split)의 대표성 문제를 해결하고자 한다.

수술 데이터는 수술 단계(Phase)의 길이 차이와 발생 빈도의 불규칙성으로 인해 심각한 데이터 불균형(Data Imbalance) 문제를 가지고 있다. 특히, 특정 수술 단계나 기구의 동시 발생(Co-occurrence)과 같은 세부 속성(Sub-properties)들이 훈련, 검증, 테스트 세트에 균등하게 분포되지 않고 일부 세트에서 누락되는 경우가 빈번하다. 만약 특정 단계의 전이(Transition)나 기구 조합이 훈련 세트에만 존재하고 테스트 세트에 없다면, 모델의 실제 성능을 정확히 평가할 수 없게 된다.

따라서 본 연구의 목표는 머신러닝 실무자와 챌린지 운영자가 데이터셋 분할의 적절성을 시각적으로 탐색하고, 최적의 분할 방안을 찾을 수 있도록 돕는 인터랙티브 데이터 시각화 도구를 제공하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 외과 수술 단계 및 기구 인식 데이터셋의 분할 상태를 분석할 수 있는 인터랙티브 시각화 프레임워크를 제안한 것이다.

중심적인 설계 아이디어는 단순히 클래스별 빈도수를 확인하는 것을 넘어, **수술 단계 간의 전이(Phase Transition)**와 **기구들의 동시 발생(Instrument Co-occurrence)** 패턴을 시각화하여, 특정 데이터 분할(Split)에서 누락된 케이스를 직관적으로 식별하게 하는 것이다. 이를 통해 사용자는 수술 데이터를 수동으로 재배치하여 더욱 대표성 있는 데이터셋 분할을 구성할 수 있다.

## 📎 Related Works

### 기존 연구의 흐름

수술 단계 및 기구 인식 분야에서는 초기 CNN과 Hidden Markov Model(HMM)의 결합부터 시작하여 RNN, Temporal Convolutional Network(TCN), 그리고 최근의 Transformer 아키텍처에 이르기까지 다양한 딥러닝 모델이 연구되어 왔다.

### 시각화 관련 연구 및 한계

수술 워크플로우 시각화 연구로는 HMM을 이용해 워크플로우 모델을 그래프로 시각화하거나, 수술 데이터 내의 시간적 관계를 분석하고 수술 세트 간을 비교하는 방법 등이 제안된 바 있다. 그러나 기존의 시각화 도구들은 주로 워크플로우 자체의 분석에 집중했을 뿐, 머신러닝 모델 평가의 핵심인 **훈련/검증/테스트 세트 간의 분포 불균형 및 누락된 케이스 분석**이라는 관점에서는 접근하지 않았다.

## 🛠️ Methodology

제안된 시각화 프레임워크는 수술 단계와 기구 주석(Annotation) 데이터를 입력으로 받아, 세 가지 주요 뷰(View)를 통해 데이터를 탐색한다. 모든 뷰에서 훈련(Red), 검증(Green), 테스트(Blue) 세트는 색상으로 구분된다.

### 1. Phase View (단계 뷰)

수술 단계를 수평축을 따라 노드로 배치하며, 다음과 같은 정보를 제공한다.

- **노드(Node):** 각 단계 내의 도넛 차트를 통해 데이터 분할 비율을 표시하며, 중심에는 해당 단계가 등장하는 수술 횟수를 표시한다.
- **아크(Arc):** 노드 사이의 곡선은 단계 간의 전이(Transition)를 나타낸다. 곡선의 너비는 전이 횟수에 비례하며, 상단 아크는 순방향 전이, 하단 아크는 역방향 전이를 의미한다.
- **바 차트(Bar Chart):** 하단에는 전체 단계별 프레임 분포와 각 단계 내에서 사용된 기구의 발생 빈도를 시각화한다.

### 2. Instrument View (기구 뷰)

기구의 사용 빈도와 기구 간의 조합을 분석하는 뷰이다.

- **방사형 바 차트(Radial Bar Charts):** 각 기구가 각 데이터 세트에서 몇 프레임 동안 나타났는지 표시한다. 기구가 하나도 없는 유휴 프레임(Idle frames) 정보도 포함된다.
- **중앙 파이 차트(Central Pie Charts):** 기구들의 조합(Co-occurrence)을 노드로 표현하며, 파이 차트를 통해 해당 조합이 어느 세트에 얼마나 분포하는지 보여준다. 노드의 배치는 Force-directed layout 알고리즘을 사용한다.

### 3. Supplementary Views (보조 뷰)

- **수술 할당 테이블:** 각 수술이 어느 세트에 할당되었는지 표 형태로 보여주며, 드래그 앤 드롭(Drag-and-drop) 방식으로 수술의 세트 할당을 인터랙티브하게 변경할 수 있다.
- **통계 바 차트:** 세트별 총 수술 수와 프레임 수, 그리고 개별 수술의 프레임 길이를 시각화하여 세트 간의 규모 차이를 확인할 수 있게 한다.

## 📊 Results

### User Study (사용자 연구)

데이터 사이언스 배경을 가진 10명의 참가자를 대상으로 10가지 데이터 탐색 과제(T1~T10)를 수행하게 하였다.

- **과제 성공률:** 대부분의 과제가 80% 이상의 성공률을 보였으나, T2(유휴 프레임 비율 확인)와 T6(훈련 세트 내 누락된 전이 확인)는 각각 30%, 40%의 낮은 성공률을 기록하였다.
- **사용성 평가:** SUS(System Usability Scale) 점수 81.25점을 획득하여, 시스템의 사용성이 평균 이상임을 입증하였다.

### Cholec80 데이터셋 분할 분석

널리 사용되는 Cholec80 데이터셋의 세 가지 분할 방식(40/-/40, 32/8/40, 40/8/32)을 분석한 결과, 다음과 같은 문제점을 발견하였다.

- **40/-/40 분할:** 일부 수술이 첫 번째 단계를 건너뛰고 시작하거나 특이한 단계 순서를 가지는데, 이러한 케이스들이 모두 훈련 세트에만 집중되어 있어 테스트 세트에서의 성능 평가가 편향될 수 있음을 확인하였다.
- **32/8/40 분할:** 검증 세트의 평균 프레임 수가 훈련/테스트 세트보다 현저히 적어($\approx 1,900$ vs $\approx 2,500$), 검증 세트의 수술들이 상대적으로 덜 복잡할 가능성이 있으며, 이는 성능 추정치의 왜곡을 초래할 수 있다.
- **40/8/32 분할:** 32/8/40 분할에 비해 검증 세트의 프레임 수가 더 많아 케이스 커버리지가 개선되었으며, 특정 기구 조합(Grasper, Bipolar, Irrigator)이 모든 세트에 골고루 분포함을 확인하였다.

### 분석 요약 (Table 1 기준)

분석 결과, 훈련 세트는 분할 방식에 관계없이 안정적이었으나, 검증 및 테스트 세트에서 누락된 케이스(단계 전이, 기구 조합 등)의 수는 분할 방식에 따라 극적으로 변하는 것을 확인하였다.

## 🧠 Insights & Discussion

### 강점 및 가치

본 연구는 수술 데이터셋 분할 시 단순히 클래스 비율을 맞추는 것을 넘어, **워크플로우의 구조적 특성(전이)과 세부 속성(기구 조합)**을 고려해야 함을 시각적으로 증명하였다. 제안된 도구를 통해 데이터셋의 취약점을 빠르게 파악하고 수동으로 재배치함으로써, 모델 평가의 신뢰성을 높일 수 있는 실질적인 방법을 제시하였다.

### 한계 및 향후 과제

- **분석 범위의 제한:** 현재 도구는 단계와 기구 주석 데이터만 분석할 수 있다. 실제 모델 성능에 큰 영향을 미치는 조명 조건, 가려짐(Occlusion)과 같은 시각적 특징(Visual features)의 분포 분석은 포함되지 않았다.
- **수동 작업의 번거로움:** 현재는 사용자가 직접 데이터를 재배치해야 하므로, 향후에는 최적의 데이터 분할을 자동으로 생성하는 알고리즘 연구가 필요하다.
- **확장 가능성:** 기구 검출(Detection)이나 세그멘테이션(Segmentation)을 위한 바운딩 박스 및 픽셀 레벨 예측 데이터까지 분석 범위를 확장할 필요가 있다.

## 📌 TL;DR

본 논문은 외과 수술 워크플로우 인식 모델의 성능 평가 신뢰도를 높이기 위해, 데이터셋 분할의 대표성을 시각적으로 분석하고 최적화할 수 있는 인터랙티브 툴을 제안한다. 특히 수술 단계 간의 전이 패턴과 기구 동시 발생 분포를 시각화하여, 특정 세트에서 누락된 케이스를 식별하고 수정할 수 있게 한다. 이 연구는 향후 수술 데이터셋 구축 및 챌린지 설계 시 모델 평가의 편향을 줄이는 중요한 도구로 활용될 가능성이 크다.
