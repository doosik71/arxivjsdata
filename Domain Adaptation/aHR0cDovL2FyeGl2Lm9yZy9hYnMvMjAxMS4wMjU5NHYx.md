# Universal Multi-Source Domain Adaptation

Yueming Yin, Zhen Yang, Haifeng Hu, and Xiaofu Wu (2020)

## 🧩 Problem to Solve

본 논문은 **Universal Multi-Source Domain Adaptation (UMDA)**라는 새로운 도메인 적응 설정을 제안하고 이를 해결하고자 한다. 기존의 도메인 적응 연구는 주로 단일 소스 도메인에서 타겟 도메인으로 지식을 전이하거나, 여러 소스 도메인을 사용하더라도 모든 도메인의 라벨 세트가 동일하다는 가정을 전제로 하였다.

하지만 실제 환경에서는 다음과 같은 복잡한 상황이 발생한다:
1. **다양한 소스 도메인의 존재**: 활용 가능한 소스 도메인이 여러 개이며, 각 소스 도메인 간의 라벨 세트가 서로 다를 수 있다(포함, 교차 또는 완전히 분리됨).
2. **타겟 라벨 세트의 미지성**: 타겟 도메인의 라벨 세트가 무엇인지 전혀 알 수 없는 상태에서 학습을 진행해야 한다.

이러한 UMDA 설정에서 해결해야 할 핵심 과제는 **각 소스 도메인과 타겟 도메인 사이의 공통 라벨 세트(Common Label Set)를 정확히 식별**하는 것과, 소스 도메인의 수가 증가하더라도 모델의 복잡도가 급격히 증가하지 않도록 **확장성(Scalability)을 유지**하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 **Universal Multi-Source Adaptation Network (UMAN)**를 제안한 것이다. UMAN의 중심 아이디어는 다음과 같다:

- **Prediction Margin 기반의 신뢰도 측정**: 타겟 샘플이 특정 클래스로 예측될 때의 확신도를 측정하는 `Prediction Margin`을 도입하여, 소스-타겟 간의 공통 클래스를 식별하는 지표로 활용한다.
- **Target Margin Register (TMR) 도입**: 학습 과정에서 누적된 마진 벡터를 저장하는 TMR을 통해 클래스별 신뢰도를 관리하며, 이를 소스 샘플과 타겟 샘플의 가중치로 사용하여 효율적인 분포 정렬을 수행한다.
- **공유 구조를 통한 확장성 확보**: 모든 소스 도메인이 하나의 공유 특징 추출기, 분류기, 판별기를 사용하게 함으로써, 소스 도메인의 개수가 늘어나도 모델의 파라미터 수나 연산 복잡도가 증가하지 않도록 설계하였다.
- **이론적 보장**: 제안한 손실 함수들이 소스 도메인 간의 분포와 소스-타겟 간의 분포를 각각 어떻게 정렬시키는지에 대한 이론적 증명을 제공한다.

## 📎 Related Works

논문은 기존의 도메인 적응 방법을 크게 두 가지 범주로 나누어 설명한다.

1. **Single-Source Domain Adaptation**:
   - **Closed Set**: 소스와 타겟의 라벨 세트가 동일하다고 가정한다.
   - **Partial**: 소스의 라벨 세트가 타겟의 라벨 세트를 포함한다고 가정한다.
   - **Open Set**: 소스와 타겟에 각각 고유한 클래스가 존재한다고 가정하며, 이를 'Unknown'으로 처리한다.
   - **Universal (USDA)**: 라벨 세트의 관계에 대한 사전 지식이 없는 상태에서 적응을 수행한다. 최근 UAN과 같은 방법론이 제안되었으나, 단일 소스 상황에만 국한된다는 한계가 있다.

2. **Multi-Source Domain Adaptation**:
   - 대부분의 기존 연구는 **Closed Set Multi-Source DA (CSMDA)**에 집중되어 있으며, 모든 소스와 타겟의 라벨 세트가 동일하다고 가정한다. 일부 연구가 카테고리 시프트(Category Shift)를 다루었으나, 여전히 모든 소스 라벨의 합집합이 타겟 라벨 세트와 동일하다는 제약이 있다.

**차별점**: UMAN은 타겟 라벨 세트가 완전히 미지수이며, 소스 도메인들 간에도 라벨 세트가 서로 다를 수 있는 가장 일반적인(Universal) 다중 소스 설정을 다룬다는 점에서 기존 연구와 차별화된다.

## 🛠️ Methodology

### 1. Prediction Margin 및 Margin Vector
UMAN은 타겟 샘플 $x$에 대한 분류기의 예측 결과에서 가장 높은 확률과 두 번째로 높은 확률의 차이를 `Prediction Margin`으로 정의한다:
$$m_f(x) = f(x, \hat{y}) - \max_{y \neq \hat{y}} f(x, y)$$
여기서 $\hat{y}$는 예측된 pseudo-label이다. 마진 값이 클수록 해당 샘플이 소스 도메인의 기지 클래스(Known Class)에 속할 가능성이 높다고 판단한다. 이를 통해 데이터 분포 $D$에 대한 경험적 마진 벡터 $M(D, f)$를 산출한다.

### 2. Target Margin Register (TMR) 및 가중치 메커니즘
TMR은 모든 학습 단계에서 계산된 마진 벡터의 평균을 저장하는 벡터 $V_{TMR}$이다.
$$V_{T+1}^{TMR} = \frac{1}{t+1} (t \times V_t^{TMR} + M(D_t^b, f))$$
이 TMR 벡터는 각 클래스가 타겟 도메인에 존재할 확률(신뢰도)을 나타내며, 이를 이용하여 다음과 같이 가중치를 부여한다:
- **소스 샘플 가중치**: $w_s(y_j) = V_{TMR}[y_j]$ (해당 클래스의 TMR 값 적용)
- **타겟 샘플 가중치**: $w_t(x_k, f) = m_f(x_k) \cdot V_{TMR}[\hat{y}_k]$ (샘플 개별 마진과 클래스별 TMR 값을 모두 고려)

### 3. 네트워크 구조 및 학습 목표
UMAN은 공유 특징 추출기 $F$, 공유 분류기 $G$, 공유 판별기 $D$로 구성된다.

- **Joint Cross-Entropy Loss ($E_G$)**: 모든 소스 도메인에 대해 가중치를 적용하지 않은 일반적인 교차 엔트로피 손실을 최소화한다. 이는 이론적으로 소스 도메인들 간의 공통 라벨 세트 내 분포를 정렬시키는 효과가 있다.
  $$E_G = \frac{1}{M} \sum_{i=1}^M \mathbb{E}_{(x,y) \sim p_i} L(y, G(F(x)))$$
- **Joint Adversarial Loss ($E_D$)**: TMR 기반의 가중치 $w_s$와 $w_t$를 적용하여 소스와 타겟의 분포를 정렬시킨다.
  $$E_D = -\frac{1}{M} \sum_{i=1}^M \mathbb{E}_{(x,y) \sim p_i} w_s(y) \log D(F(x)) - \mathbb{E}_{x \sim q} w_t(x, G \circ F) \log (1 - D(F(x)))$$
- **전체 최적화 목표**:
  $$\max_D \min_{F,G} E_G - E_D$$

### 4. 추론 (Inference)
테스트 시에는 타겟 샘플의 마진 $m_{G \circ F}(x)$가 특정 임계값 $w_0$보다 크면 예측된 클래스로 분류하고, 그렇지 않으면 `Unknown` 클래스로 분류하여 배제한다.

## 📊 Results

### 1. 실험 설정
- **데이터셋**: Office-31, Office-Home, VisDA2017+ImageCLEF-DA를 사용하였다.
- **비교 대상**: Source-only(ResNet), 단일 소스 DA(DANN, RTN, IWAN, PADA, OSBP, UAN), 다중 소스 DA(MDAN, MDDA, DCTN) 등 SOTA 방법론들을 포함하였다.
- **평가 지표**: 분류 정확도(Accuracy)를 측정하였으며, 타겟 전용 클래스는 'Unknown'으로 통합하여 처리하였다.

### 2. 주요 결과
- **정량적 성과**: UMAN은 거의 모든 설정에서 기존 방법론들을 큰 차이로 압도하였다. 특히 소스 도메인의 수를 늘렸을 때, 다른 방법들은 성능이 정체되거나 하락하는 반면 UMAN은 **Transfer Gain(전이 이득)**이 증가하는 경향을 보였다.
- **강건성 분석**: 타겟 전용 라벨 세트의 크기나 소스 도메인 간의 라벨 교집합 크기를 변화시켜 실험한 결과, UMAN은 다른 방법(UAN, MDDA)들에 비해 성능 변동이 적고 매우 안정적인 성능을 유지하였다.
- **이론적 검증**: t-SNE 시각화를 통해 소스 도메인 간의 공통 라벨 세트가 잘 정렬되고, 소스-타겟 간의 공통 분포가 효과적으로 일치됨을 확인하여 Theorem 1과 Theorem 2를 입증하였다.

## 🧠 Insights & Discussion

본 논문은 다중 소스 도메인 적응에서 발생할 수 있는 **'부정적 전이(Negative Transfer)'** 문제를 효과적으로 해결하였다. 기존의 UAN과 같은 Universal DA 방법은 단일 소스 기반이기에 다중 소스 환경에서 소스 간의 분포 차이를 고려하지 못해 성능이 저하되는 문제가 있었다. 반면, UMAN은 공유 아키텍처와 TMR 기반 가중치 메커니즘을 통해 소스 간의 정렬과 소스-타겟 간의 정렬을 동시에 달성하였다.

**강점**:
- 모델 복잡도를 유지하면서 다중 소스를 수용할 수 있는 확장성을 확보하였다.
- Prediction Margin과 TMR이라는 단순하지만 강력한 도구를 통해 미지의 타겟 라벨 세트를 효과적으로 추론하였다.

**한계 및 논의**:
- 추론 단계에서 사용되는 임계값 $w_0$에 대한 분석이 수행되었으나, 데이터셋마다 최적의 $w_0$가 다를 수 있으며 이를 자동으로 결정하는 메커니즘에 대한 논의는 부족하다.
- 실험에서 사용된 $w_0=0.5$가 보편적으로 작동한다고 주장하지만, 매우 불균형한 데이터셋에서도 동일한 성능을 낼 수 있을지는 추가 검토가 필요하다.

## 📌 TL;DR

본 논문은 소스 도메인이 여러 개이고 각 라벨 세트가 다르며, 타겟 라벨 세트가 완전히 알려지지 않은 **Universal Multi-Source Domain Adaptation (UMDA)** 문제를 정의하고 이를 해결하는 **UMAN** 네트워크를 제안한다. 예측 마진(Prediction Margin)과 타겟 마진 레지스터(TMR)를 통해 공통 클래스를 식별하고 가중치를 부여함으로써, 모델 복잡도 증가 없이 효율적으로 도메인 간 분포를 정렬한다. 실험 결과, 소스 도메인의 수가 많아질수록 더 많은 지식을 전이하여 성능이 향상됨을 입증하였으며, 이는 향후 복잡한 실제 환경의 다중 소스 도메인 적응 연구에 중요한 기반이 될 것으로 보인다.
