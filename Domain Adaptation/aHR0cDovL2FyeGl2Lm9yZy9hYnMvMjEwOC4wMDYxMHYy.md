# Multiple Classifiers Based Adversarial Training for Unsupervised Domain Adaptation

Yiju Yang, Taejoon Kim, Guanghui Wang (2022)

## 🧩 Problem to Solve

본 논문은 이미지 분류를 위한 비지도 도메인 적응(Unsupervised Domain Adaptation, UDA) 문제를 해결하고자 한다. 딥러닝 모델은 일반적으로 대량의 라벨링된 데이터에 의존하지만, 실제 환경에서는 새로운 도메인(Target Domain)의 데이터를 수집하는 것은 쉽지만 라벨을 생성하는 것은 매우 비용이 많이 든다. 따라서 라벨이 있는 소스 도메인(Source Domain)의 지식을 라벨이 없는 타겟 도메인으로 전이하여 모델의 일반화 성능을 높이는 것이 핵심이다.

특히, 기존의 Maximum Classifier Discrepancy (MCD) 프레임워크가 두 개의 분류기(Classifier) 구조를 사용하여 성공적인 결과를 냈으나, 단 두 개의 분류기만으로는 새로운 도메인의 데이터 특성을 충분히 반영하지 못하거나 결정 경계(Decision Boundary)의 변별력이 부족할 수 있다는 점에 주목한다. 본 연구의 목표는 분류기의 수를 확장하여 타겟 도메인에서의 성능을 더욱 향상시키는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 MCD 프레임워크의 분류기 구조를 두 개에서 여러 개(Multiple Classifiers)로 확장하여 더 정교한 결정 경계를 학습하는 것이다. 주요 기여 사항은 다음과 같다.

1.  **다중 분류기 불일치 손실 함수(Multi-classifier Discrepancy Loss) 제안**: 각 분류기가 서로 다른 특성을 유지해야 한다는 원칙에 기반하여, 임의의 개수 $n$개의 분류기를 추가할 수 있는 새로운 손실 함수를 설계하였다.
2.  **다중 분류기 구조의 성능 우위 입증**: 실험을 통해 분류기를 늘리는 것이 기존의 2-분류기 구조보다 성능 면에서 뚜렷한 이점이 있음을 보였다.
3.  **최적의 분류기 수 제시**: 정확도와 계산 효율성 사이의 트레이드오프(Trade-off)를 분석하여, 실무적으로 3개의 분류기를 사용하는 구조가 가장 효율적이고 우수한 성능을 낸다는 점을 제안하였다.

## 📎 Related Works

논문에서는 도메인 적응을 위한 기존 연구들을 세 가지 범주로 설명한다.

1.  **도메인 불변 특징 학습 (Learn domain invariant features)**: 서로 다른 시나리오에서도 공통적으로 나타나는 특징을 학습하여 도메인 간의 격차를 줄이는 방법이다.
2.  **거리 기반 방법 (Distance-based methods)**: Maximum Mean Discrepancy (MMD)와 같이 소스 도메인과 타겟 도메인 간의 분포 거리를 측정하고 이를 최소화하여 정렬하는 방식이다. (예: DAN, JAN, DeepCORAL 등)
3.  **적대적 학습 방법 (Adversarial methods)**: 도메인 판별기(Discriminator)를 통해 특징 추출기가 어느 도메인에서 온 데이터인지 구분하지 못하게 만들어 도메인 불변 특징을 추출하는 방식이다. (예: DANN, ADDA, MCD 등)

본 논문은 이 중 **Maximum Classifier Discrepancy (MCD)** 방식에 기반을 두고 있다. MCD는 두 분류기 간의 출력 차이를 최대화하고 이를 다시 최소화하는 과정을 통해 타겟 도메인의 데이터를 소스 도메인의 결정 경계 안으로 끌어들이는 전략을 사용한다. 본 연구는 여기서 분류기의 개수를 확장함으로써 MCD가 가진 결정 경계의 한계를 극복하고자 한다.

## 🛠️ Methodology

### 1. 불일치 손실 함수 (Discrepancy Loss)

먼저, 두 분류기 사이의 확률 출력값 차이를 측정하는 기본 손실 함수 $L_2$는 다음과 같이 정의된다.
$$L_2(p_1, p_2) = \frac{1}{K} \sum_{k=1}^{K} |p_{1k} - p_{2k}|$$
여기서 $K$는 클래스의 수이며, $p_{1k}$와 $p_{2k}$는 각 분류기가 예측한 $k$번째 클래스의 확률값이다.

이를 확장하여 $n$개의 분류기를 가진 경우의 손실 함수 $L_n$은 모든 분류기 쌍(Pair) 간의 $L_2$ 손실의 합으로 정의한다. 이는 모든 분류기가 서로 다르게 작동하도록 강제하여 모델 붕괴(Collapse)를 방지하기 위함이다.
- **3개 분류기 시**: $L_3(p_1, p_2, p_3) = L_2(p_1, p_2) + L_2(p_1, p_3) + L_2(p_2, p_3)$
- **$n$개 분류기 시**: $L_n(p_1, \dots, p_n) = \sum_{i=1}^{n-1} \sum_{j=i+1}^{n} L_2(p_i, p_j)$

### 2. 학습 절차 (Training Steps)

전체 학습 과정은 총 3단계로 진행된다.

**Step 1: 소스 도메인 초기 학습**
먼저 라벨이 있는 소스 도메인 데이터 $X_s$를 사용하여 모든 분류기($f_{\theta_1}, \dots, f_{\theta_n}$)를 학습시킨다. 각 분류기는 교차 엔트로피(Cross-Entropy) 손실 함수를 사용하여 학습된다.
$$\text{Loss}_1 = \sum_{i=1}^{n} L_{Ci}(f_{\theta_i}(X_s), Y_s)$$

**Step 2: 분류기 업데이트 (적대적 학습)**
특징 추출기(Feature Extractor)의 파라미터를 고정하고, 분류기의 파라미터만 업데이트한다.
- **소스 도메인**: 기존과 동일하게 분류 정확도를 높이는 방향으로 학습한다.
- **타겟 도메인**: 분류기들 간의 불일치(Discrepancy)를 **최대화**하도록 학습한다. 이때 Gradient Reversal Layer와 유사하게 그래디언트를 반전시켜 최대화를 수행한다.
$$\text{Loss}_2 = \text{Loss}_s - \text{Loss}_t$$
($\text{Loss}_s$는 소스 도메인의 총 손실, $\text{Loss}_t$는 타겟 도메인의 $L_n$ 손실이다.)

**Step 3: 특징 추출기 업데이트**
분류기들의 파라미터를 고정하고, CNN 특징 추출기의 파라미터를 업데이트한다. 타겟 도메인 데이터에 대해 분류기들 간의 불일치 $L_n$을 **최소화**하도록 학습한다. 이렇게 하면 특징 추출기는 타겟 데이터를 소스 도메인의 결정 경계 내부로 밀어 넣어 도메인 간의 분포를 정렬하게 된다.
$$\text{Loss}_3 = L_n(f_{\theta_1}(X_t), \dots, f_{\theta_n}(X_t))$$

## 📊 Results

### 1. 실험 설정
- **Toy Dataset**: 결정 경계의 시각적 비교를 위해 사용.
- **Digit/Sign Datasets**: MNIST, SVHN, USPS, SYN SIGNS, GTSRB 데이터셋을 사용하여 4가지 전이 시나리오(예: SVHN $\rightarrow$ MNIST)를 테스트.
- **VisDA-2017**: 합성 이미지(Synthetic)에서 실제 이미지(Real)로의 전이 성능을 측정하기 위해 ResNet-101 모델을 사용.

### 2. 주요 결과
- **정량적 성능**: 모든 전이 작업에서 제안 방법이 기존 MCD보다 높은 정확도를 보였다.
    - SVHN $\rightarrow$ MNIST: MCD 대비 약 2.7% 향상.
    - USPS $\rightarrow$ MNIST: MCD 대비 약 3.0% 향상.
    - VisDA-2017: 평균 정확도가 71.9%(MCD)에서 78.3%($n=3$)로 크게 상승하였다.
- **분류기 수의 영향**: 분류기 수를 2개에서 3개로 늘릴 때 성능 향상 폭이 가장 컸으며, 4개 이상으로 늘릴 경우 성능 증가율이 둔화되거나 오히려 하락하는 경향을 보였다.
- **Ablation Study**: 손실 함수의 '닫힌 루프(Closed loop)' 구조(모든 쌍의 조합)가 필수적임을 확인하였다. 일부 손실 항을 누락시키거나 중복시킬 경우 성능이 MCD보다 낮게 떨어졌다.
- **효율성 분석**: $n=3$일 때의 학습 시간은 $n=2$일 때와 거의 차이가 없었으나 성능은 대폭 향상되었으므로, $n=3$이 가장 최적의 선택임을 확인하였다.

## 🧠 Insights & Discussion

본 논문은 매우 단순한 구조적 확장(분류기 수 증가)만으로도 비지도 도메인 적응 성능을 유의미하게 끌어올릴 수 있음을 증명하였다. 

**강점 및 해석**: 
단순히 분류기를 늘리는 것이 아니라, 모든 분류기 쌍 간의 불일치를 최대화/최소화하는 제약 조건을 부여함으로써, 특징 추출기가 더 엄격하고 정교한 도메인 불변 특징을 학습하도록 강제한 점이 유효했다. 이는 결정 경계를 더 다각도에서 정의함으로써 타겟 도메인의 데이터 분포를 더 정확하게 감싸 안을 수 있게 된 것으로 해석된다.

**한계 및 논의**:
분류기의 수를 무한히 늘린다고 해서 성능이 계속 좋아지지 않는다는 점은 주목할 만하다. 이는 과적합(Overfitting)이나, 너무 많은 제약 조건이 오히려 학습의 수렴을 방해하는 최적화 문제를 야기할 수 있음을 시사한다. 또한, 본 연구는 분류기의 아키텍처 자체보다는 개수에 집중하였으므로, 서로 다른 구조의 분류기를 사용하는 것이 어떤 영향을 줄지에 대한 추가 연구가 필요해 보인다.

## 📌 TL;DR

- **핵심**: 기존 MCD(Maximum Classifier Discrepancy)의 2-분류기 구조를 $n$-분류기로 확장하고, 모든 쌍의 불일치를 계산하는 새로운 손실 함수를 제안함.
- **결과**: 분류기를 3개 사용할 때 성능과 효율성의 균형이 가장 좋으며, 기존 MCD보다 눈에 띄는 정확도 향상을 달성함.
- **의의**: 복잡한 아키텍처 변경 없이 구조적 확장만으로 UDA 성능을 높일 수 있는 실용적인 가이드를 제공함.
