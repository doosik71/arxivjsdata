# Few-shot Adaptation Works with UnpredicTable Data

Jun Shern Chan, Michael Pieler, Jonathan Jao, Jérémy Scheurer, Ethan Perez (2022)

## 🧩 Problem to Solve

본 논문은 언어 모델(Language Models, LMs)의 Few-shot Learning(FSL) 능력을 향상시키기 위한 학습 데이터의 성격과 규모의 영향력을 분석한다. 기존 연구들은 다양한 태스크를 다수 학습시키는 Multi-task training이 새로운 태스크에 대한 일반화 성능(Generalization)을 높인다고 주장해 왔다. 이에 따라 연구자들은 학습 데이터의 규모와 다양성을 늘리는 방향으로 접근해 왔다.

본 연구의 목표는 인터넷 상의 표(Table) 데이터에서 자동으로 추출한 초거대 규모의 FSL 태스크 데이터셋을 구축하고, 이를 통해 모델을 파인튜닝했을 때 실제로 FSL 성능이 어떻게 변화하는지 확인하는 것이다. 특히, 데이터의 다양성과 품질이 FSL 적응(Adaptation)에 필수적인 요소인지, 아니면 예상치 못한 좁은 범위의 데이터셋만으로도 일반적인 FSL 능력이 향상될 수 있는지를 탐구한다.

## ✨ Key Contributions

본 논문의 핵심 기여는 다음과 같다.

1. **UnpredicTable 데이터셋 구축**: 인터넷의 웹 표(Web Tables)를 활용하여 413,299개의 다양한 Few-shot 태스크를 자동으로 추출한 초거대 데이터셋을 제안한다. 이는 기존 공개 데이터셋보다 수십 배 더 큰 규모이다.
2. **데이터 다양성 및 규모의 역설 발견**: 데이터셋의 규모를 무작정 늘리는 것이 성능 향상으로 직결되지 않음을 보였다. 오히려 소프트웨어 문서(예: `support.google.com`)와 같이 매우 좁은 범위의 데이터셋이 훨씬 더 다양하고 정교하게 큐레이션된 NLP 데이터셋보다 더 높은 FSL 성능 향상을 이끌어낼 수 있음을 발견했다.
3. **Few-shot Adaptation의 도메인 불가지론(Domain-agnostic) 입증**: 특정 좁은 도메인의 데이터로 학습했음에도 불구하고, 테스트 시에는 전혀 다른 도메인의 태스크들에서도 성능 향상이 나타났다. 이는 모델이 특정 도메인 지식을 배운 것이 아니라, '적은 예시로부터 새로운 태스크를 학습하는 방법' 자체, 즉 일반적인 Few-shot 적응 능력을 습득했음을 시사한다.

## 📎 Related Works

본 논문은 다음과 같은 관련 연구들을 바탕으로 한다.

*   **MetaICL**: 모델이 컨텍스트 내에서 제공된 몇 개의 예시(Few-shot examples)를 보고 타겟 입력을 예측하도록 훈련시키는 방법론이다. 본 연구는 이 MetaICL 프레임워크를 채택하여 모델을 학습시켰다.
*   **Multi-task Learning**: 많은 수의 태스크를 학습할수록 Zero-shot 또는 Few-shot 일반화 성능이 향상된다는 기존 연구들이 존재한다. 본 연구는 이러한 아이디어를 극단적으로 확장하여 수십만 개의 태스크를 적용했다.
*   **Domain Adaptation**: 일반적으로는 다운스트림 태스크와 유사한 도메인의 데이터로 학습하는 것이 유리하다고 알려져 있다. 그러나 본 논문은 FSL 적응의 경우 도메인 유사성이 필수적이지 않으며, 직관에 반하는(Counter-intuitive) 데이터셋으로도 충분한 성능 향상이 가능함을 보여줌으로써 기존 상식에 의문을 제기한다.

## 🛠️ Methodology

### 1. 데이터 생성 파이프라인: Tables $\rightarrow$ Tasks
연구진은 WDC Web Table Corpus 2015에서 관계형 표(Relational Tables)를 추출하여 FSL 태스크로 변환하였다.

*   **태스크 변환**: 표의 각 행(Row)을 하나의 인스턴스로 본다. 특정 열(Column) 하나를 정답인 목표 변수 $y$로 설정하고, 나머지 열들을 입력값 $x$로 설정한다. 이때, 각 값 앞에 열 헤더(Column Header)를 붙여 컨텍스트를 제공한다.
*   **다중 태스크 생성**: 하나의 표에서 서로 다른 열을 목표 변수로 설정함으로써, 표 하나당 여러 개의 서로 다른 태스크를 생성할 수 있다. (예: $P(A|B, C), P(B|A, C), P(C|A, B)$)
*   **필터링 절차**: 
    *   유니크한 열 2개 미만 또는 행 6개 미만인 표 제거.
    *   spaCy POS classifier를 사용하여 숫자, 고유명사, 기호 등이 20% 이상인 '정크 데이터' 제거.
    *   정답 공간의 유니크한 답이 2개 미만이거나 클래스 불균형이 심한 태스크(Shannon Diversity Index < 0.7) 제거.
    *   동일한 입력에 대해 서로 다른 출력이 존재하는 경우(단일 정답 원칙 위배) 제거.

### 2. 학습 방법: MetaICL
본 연구는 GPT2-large(774M 파라미터) 모델을 사용하여 MetaICL 방식으로 학습하였다.

*   **훈련 목표**: $k$개의 예시 $\{(x_i, y_i)\}_{i=1}^k$와 새로운 입력 $x_{k+1}$이 주어졌을 때, 정답 $y_{k+1}$을 예측하는 확률을 최대화하는 것이다.
*   **손실 함수**: 다음과 같은 로그 가능도(Log-likelihood)를 최대화하는 방향으로 파인튜닝을 진행한다.
    $$\max \log P(y_{k+1} | x_1, y_1, \dots, x_k, y_k, x_{k+1})$$
*   **추론 절차**: 테스트 시에는 학습되지 않은 새로운 태스크의 예시 $k$개를 컨텍스트에 넣고, 모델이 가장 가능성 높은 $y_{k+1}$을 선택하도록 한다.

## 📊 Results

### 1. 실험 설정
*   **데이터셋**: `UnpredicTable-5k` (자동 추출된 5,000개 태스크) vs. 인간이 큐레이션한 NLP 데이터셋.
*   **평가 지표**: 52개의 NLP 테스트 태스크(Low Resource, NLI, Paraphrase, Classification, QA)에 대한 평균 정확도.
*   **베이스라인**: Pretrained GPT2 (0-shot 및 k-shot).

### 2. 주요 결과
*   **전반적 성능**: `UnpredicTable-5k`로 학습한 모델은 모든 테스트 카테고리에서 pretrained 모델보다 높은 성능을 보였으며, 큐레이션된 NLP 데이터셋으로 학습한 모델과 대등한 수준의 성능을 기록하였다.
*   **좁은 데이터셋의 효율성**: 놀랍게도 `support.google.com`에서 추출한 소프트웨어 문서 데이터셋만으로 학습했을 때, 52개 테스트 태스크에서 평균 $+7.5\%$의 성능 향상을 보였다. 이는 40개의 정교한 NLP 데이터셋으로 학습했을 때의 향상분($+6.7\%$)보다 높았다.
*   **데이터 규모와의 관계**: 태스크의 수를 $T=40$에서 $T=125\text{k}$까지 늘려보았으나, 일정 수준($T=5000$)을 넘어서면 오히려 성능이 저하되는 경향을 보였다. 즉, 단순히 규모를 키우는 것이 정답이 아님을 확인하였다.

## 🧠 Insights & Discussion

### 1. 데이터 품질의 주관성
연구진은 사람이 직접 '고품질'이라고 판단한 태스크들로 데이터셋을 구성해 보았으나, 이는 실제 다운스트림 성능 향상과 강한 상관관계를 보이지 않았다. 이는 모델이 FSL 능력을 습득하는 데 필요한 데이터의 특성이 인간이 생각하는 '논리적' 혹은 '유용한' 지식의 형태와 다를 수 있음을 의미한다.

### 2. 도메인 불가지론적 전이(Domain-agnostic Transfer)
가장 성능 향상이 컸던 데이터셋들은 소프트웨어 문서나 비디오 게임 트리비아(`mmo-champion.com`)와 같이 테스트 태스크와는 아무런 관련이 없는 도메인이었다. 그럼에도 불구하고 뉴스 분류, 감성 분석, 과학 QA 등 광범위한 태스크에서 성능이 올랐다. 이는 모델이 특정 도메인 지식을 배운 것이 아니라, **"컨텍스트에 주어진 몇 개의 예시를 통해 규칙을 빠르게 파악하는 능력"** 자체를 학습했음을 시사한다.

### 3. 한계 및 비판적 해석
본 논문은 어떤 데이터가 FSL 향상에 도움이 되는지에 대한 명확한 패턴을 제시하지 못했다. "왜 특정 데이터는 도움이 되고 다른 데이터는 안 되는가"에 대한 이론적 설명이 부족하며, 이는 향후 연구 과제로 남아 있다. 또한, 사용된 모델이 GPT2-large로 제한적이어서 최신 대규모 언어 모델(LLM)에서도 동일한 현상이 발생하는지에 대한 추가 검증이 필요하다.

## 📌 TL;DR

본 연구는 웹 표에서 자동으로 추출한 41만 개의 태스크 데이터셋 `UnpredicTable`을 구축하고, 이를 통해 언어 모델의 Few-shot Learning 능력을 향상시킬 수 있음을 보였다. 특히, 정교하게 큐레이션된 대규모 데이터셋보다 **특정 좁은 범위의 데이터(예: 소프트웨어 문서)가 오히려 더 강력한 FSL 일반화 성능을 유도**한다는 사실을 발견했다. 이는 FSL 적응이 도메인 지식의 습득이 아니라, **범용적인 '적은 예시로부터의 학습 능력'을 기르는 과정**임을 시사하며, 향후 효율적인 FSL 데이터 구축 전략에 중요한 시사점을 제공한다.
