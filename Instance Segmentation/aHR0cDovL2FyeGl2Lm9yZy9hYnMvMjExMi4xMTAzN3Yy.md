# SOIT: Segmenting Objects with Instance-Aware Transformers

Xiaodong Yu, Dahu Shi, Xing Wei, Ye Ren, Tingqun Ye, Wenming Tan (2022)

## 🧩 Problem to Solve

인스턴스 분할(Instance Segmentation)은 이미지 내의 각 객체에 대해 클래스 라벨과 픽셀 단위의 마스크를 동시에 예측해야 하는 도전적인 작업이다. 기존의 주류 방법론인 Mask R-CNN과 같은 '검출 후 분할(detect-then-segment)' 패러다임은 객체 검출기로 바운딩 박스를 먼저 생성하고, RoIAlign을 통해 특징 맵을 크롭(crop)한 뒤 소규모 영역에서만 마스크를 예측한다. 이러한 방식은 다음과 같은 치명적인 한계를 가진다.

첫째, 분할 결과가 객체 검출기의 성능에 과도하게 의존하므로 복잡한 시나리오에서 성능이 저하된다. 둘째, RoI를 고정된 크기(예: $14 \times 14$)로 리사이징하는 과정에서 큰 객체의 경계 세부 정보가 손실되어 마스크의 품질이 제한된다. 또한, 최근의 단일 단계(single-stage) 방법론들조차 훈련 시의 일대다(one-to-many) 라벨 할당 문제와 테스트 시 중복 제거를 위한 NMS(Non-Maximum Suppression)와 같은 수동 설계된(hand-crafted) 후처리 과정에 의존하며, 이는 전체 파이프라인의 완전한 엔드-투-엔드 최적화를 방해한다.

따라서 본 논문의 목표는 RoI 크롭과 NMS가 필요 없는, 완전한 엔드-투-엔드 형태의 인스턴스 분할 프레임워크를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 인스턴스 분할을 직접적인 '집합 예측(set prediction)' 문제로 재정의하는 것이다. 이를 위해 DETR의 개념을 확장하여, 여러 개의 학습 가능한 쿼리(query)가 글로벌 이미지 컨텍스트 하에서 객체의 카테고리, 바운딩 박스 위치, 그리고 픽셀 단위 마스크를 병렬적으로 추론하도록 설계하였다.

특히, 고차원이며 인스턴스마다 형태가 다른 픽셀 마스크를 효율적으로 표현하기 위해 '인스턴스 인식 트랜스포머(Instance-aware Transformer)'를 제안하였다. 이는 고정된 가중치를 사용하는 대신, 각 객체 쿼리로부터 동적으로 생성된 파라미터를 사용하여 해당 인스턴스에 특화된 트랜스포머를 구축함으로써 RoI 연산 없이도 전체 해상도 특징 맵에서 직접 마스크를 생성할 수 있게 한다. 또한, 객체의 위치 정보를 더욱 정밀하게 반영하기 위해 새로운 상대적 위치 인코딩(Relative Positional Encoding)을 도입하였다.

## 📎 Related Works

기존의 인스턴스 분할 연구는 크게 세 가지로 나뉜다.
1. **Top-down 방식**: Mask R-CNN 계열로, 객체 검출 후 해당 영역을 분할하는 'detect-then-segment' 방식을 따른다. 하지만 앞서 언급한 RoI 기반의 한계가 명확하다.
2. **Bottom-up 방식**: 픽셀 단위의 임베딩을 먼저 학습한 후 이를 클러스터링하여 인스턴스를 구분하는 'label-then-cluster' 방식을 사용한다.
3. **Single-stage 방식**: YOLACT, CondInst, SOLO 등이 있으며, 단일 단계 검출기 위에서 동작하여 속도를 높였으나, 여전히 NMS나 복잡한 라벨 할당 전략에 의존하는 경우가 많다.

최근 QueryInst나 SOLQ와 같은 엔드-투-엔드 시도가 있었으나, 이들은 여전히 인스턴스 분리를 위해 RoI 크롭 과정이 필요하다. 반면, SOIT는 Transformer 아키텍처와 이분 매칭(bipartite matching) 전략을 사용하여 RoI 크롭과 NMS를 완전히 제거함으로써 기존 접근 방식과 차별화된다.

## 🛠️ Methodology

### 전체 시스템 구조
SOIT의 전체 파이프라인은 세 가지 주요 구성 요소로 이루어져 있다.
1. **Backbone Network**: ResNet-50/101 또는 Swin Transformer를 사용하여 다중 스케일 이미지 특징 맵 $F = \{F^3, F^4, F^5, F^6\}$을 추출한다.
2. **Transformer Encoder-Decoder**: Deformable Transformer Encoder를 통해 정제된 특징 메모리 $P$를 생성하고, 이후 Deformable Transformer Decoder가 학습 가능한 $N$개의 객체 쿼리를 사용하여 각 객체의 임베딩을 추론한다.
3. **Multi-task Prediction Network**: 추출된 쿼리 임베딩을 바탕으로 클래스, 바운딩 박스, 마스크 파라미터를 동시에 예측한다.

### 인스턴스 인식 트랜스포머 (Instance-aware Transformer)
마스크를 생성하기 위해, 본 논문은 각 인스턴스마다 동적으로 생성되는 트랜스포머를 사용한다. 마스크 브랜치에서 예측된 $D$-차원 벡터는 인스턴스 인식 트랜스포머의 선형 투영 층(linear projection layers)의 가중치로 변환된다.

입력 특징 맵 $x$에 대해, $n$번째 객체 쿼리의 디포머블 멀티 헤드 어텐션 특징 $H_m^n$은 다음과 같이 계산된다.
$$H_m^n = \sum_{k=1}^{K} A_{mqk}^n \cdot x(p_q + \Delta p_{mqk}^n)$$
여기서 $m$은 어텐션 헤드, $k$는 샘플링된 키를 의미하며, $\Delta p_{mqk}^n$(샘플링 오프셋)과 $A_{mqk}^n$(어텐션 가중치)은 쿼리 특징 $z_q$를 통해 동적으로 생성된 가중치를 사용하여 계산된다. 최종 마스크 $\text{Mask}^n$은 다음과 같이 출력 투영 층 $W^n$을 거쳐 생성된다.
$$\text{Mask}^n = W^n [\text{Concat}(H_1^n, H_2^n, \dots, H_M^n)]$$
이때 $W^n$ 역시 해당 인스턴스에 특화된 동적 파라미터이며, 결과값에 시그모이드(sigmoid) 함수를 적용하여 이진 마스크를 얻는다.

### 상대적 위치 인코딩 (Relative Positional Encoding)
기존의 절대적 위치 인코딩 대신, 객체 쿼리가 예측한 바운딩 박스의 중심 좌표 $pos^q$를 반영한 상대적 위치 인코딩을 제안한다.
$$\text{PE}(\text{pos}, 2i) = \sin((\text{pos} - \text{pos}^q) / 10000^{2i/d_{model}})$$
$$\text{PE}(\text{pos}, 2i+1) = \cos((\text{pos} - \text{pos}^q) / 10000^{2i/d_{model}})$$
이 방식은 각 쿼리가 자신의 예측 위치를 기준으로 특징 맵을 바라보게 함으로써, 외형이 유사한 객체들이 서로 다른 위치에 있을 때 이를 효과적으로 구분할 수 있게 한다.

### 학습 절차 및 손실 함수
모델은 클래스 분류, 위치 추정, 마스크 분할의 세 가지 태스크를 동시에 학습한다. 전체 손실 함수 $\mathcal{L}$은 다음과 같다.
$$\mathcal{L} = \lambda_{cls}\mathcal{L}_{cls} + \lambda_{L1}\mathcal{L}_{L1} + \lambda_{iou}\mathcal{L}_{iou} + \lambda_{dice}\mathcal{L}_{dice} + \lambda_{bce}\mathcal{L}_{bce}$$
분할 작업에는 Dice Loss와 Binary Cross-Entropy(BCE) Loss를 사용하였으며, 모든 디코더 스테이지에서 마스크 손실을 적용하여 학습함으로써 검출 성능까지 동시에 향상시키는 공동 학습(joint learning) 전략을 취했다.

## 📊 Results

### 실험 설정
- **데이터셋**: MS COCO 2017 데이터셋을 사용하여 검증하였다.
- **지표**: Mask AP(분할 성능)와 Box AP(검출 성능)를 측정하였다.
- **구현**: ResNet-50, ResNet-101 및 Swin-L 백본을 사용하였으며, Adam 옵티마이저로 50 에포크 동안 학습하였다.

### 정량적 결과
ResNet-50 백본 기준, SOIT는 **Mask AP 42.5%**, **Box AP 49.1%**를 달성하였다. 이는 기존의 복잡한 튜닝이 적용된 HTC(Hybrid Task Cascade)보다 Mask AP에서 2.8%, Box AP에서 4.2% 높은 수치이다. 또한, 동적 컨볼루션을 사용하는 CondInst보다 Mask AP 기준 4.7% 더 높은 성능을 보였다. Swin-L 백본을 적용했을 때는 Mask AP 49.2%, Box AP 56.9%라는 매우 높은 성능을 기록하며 최신 모델들과 경쟁 가능한 수준임을 입증하였다.

### 주요 분석 (Ablation Study)
1. **어텐션 헤드 수**: 헤드 수를 4개로 설정했을 때 최적의 성능을 보였으며, 8개 이상에서는 파라미터 증가로 인해 최적화가 어려워지며 성능이 포화되는 경향을 보였다.
2. **상대적 위치 인코딩**: 절대적 위치 인코딩 대비 상대적 위치 인코딩을 사용했을 때 Mask AP가 0.8% 향상되어, 위치 큐(location cue)의 중요성이 확인되었다.
3. **마스크 손실 적용 단계**: 모든 디코더 스테이지에서 마스크 손실을 적용했을 때, 순수 객체 검출 모델보다 Box AP가 2.1% 향상되었다. 이는 분할 작업의 학습이 검출 작업에 긍정적인 영향을 주는 상호 보완적 관계임을 시사한다.

## 🧠 Insights & Discussion

SOIT는 인스턴스 분할을 단순한 후처리 작업이 아닌, 트랜스포머의 집합 예측 문제로 치환함으로써 RoI 크롭과 NMS라는 고질적인 제약 조건을 제거하였다. 특히 인스턴스 인식 트랜스포머를 통해 픽셀 마스크를 동적 파라미터의 집합으로 표현한 점은 매우 효율적인 설계이다.

가장 주목할 점은 분할(segmentation)과 검출(detection)의 통합 학습이 단순히 분할 성능을 높이는 것에 그치지 않고, 검출 성능까지 끌어올렸다는 것이다. 이는 객체의 세밀한 픽셀 정보를 학습하는 과정이 객체의 전반적인 위치와 클래스를 파악하는 데 도움을 주는 정규화(regularization) 효과를 가졌음을 의미한다.

다만, 모델의 성능이 다중 스케일 특징 맵과 디포머블 어텐션의 효율성에 크게 의존하고 있으며, 쿼리의 개수 $N$에 따라 계산 복잡도가 결정된다는 점이 잠재적인 한계로 작용할 수 있다. 또한, 매우 작은 객체에 대한 세밀한 분할 성능이 어느 정도까지 향상될 수 있는지에 대한 추가적인 분석이 필요해 보인다.

## 📌 TL;DR

본 논문은 RoI 크롭과 NMS 없이 인스턴스 분할을 수행하는 엔드-투-엔드 프레임워크인 **SOIT**를 제안한다. 핵심은 쿼리 임베딩으로부터 동적으로 가중치를 생성하는 **인스턴스 인식 트랜스포머**와 **상대적 위치 인코딩**을 통해 고해상도 특징 맵에서 직접 마스크를 예측하는 것이다. COCO 데이터셋에서 SOTA 성능을 달성했으며, 분할과 검출의 통합 학습이 서로의 성능을 향상시킨다는 것을 입증하였다. 이 연구는 향후 복잡한 수동 설계 없는 단순하고 강력한 인스턴스 인식 베이스라인으로 활용될 가능성이 크다.
