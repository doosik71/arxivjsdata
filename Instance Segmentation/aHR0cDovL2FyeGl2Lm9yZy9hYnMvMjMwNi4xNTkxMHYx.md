# Incremental Learning on Food Instance Segmentation

Huu-Thanh Nguyen, Yu Cao, Chong-Wah Ngo, and Wing-Kwong Chan (2023)

## 🧩 Problem to Solve

본 논문이 해결하고자 하는 핵심 문제는 식품 영상의 인스턴스 분할(Instance Segmentation) 모델을 학습시키는 데 필요한 막대한 데이터 라벨링 비용과 시간이다. 식품 영상의 인스턴스 분할은 음식의 1인분 양(serving size)을 추정하는 데 필수적인 기술이지만, 복잡한 외형으로 인해 정확한 마스크(mask)나 폴리곤(polygon) 라벨을 생성하는 과정은 매우 많은 비용이 소요된다.

기존의 준지도 학습(Semi-supervised learning) 방식은 일부 샘플만 라벨링하고 나머지는 모델이 생성한 pseudo-label을 사용하는 방식을 취하지만, 생성된 pseudo-label의 품질이 보장되지 않아 오히려 모델의 성능을 저하시킬 위험이 있다. 따라서 본 논문의 목표는 제한된 라벨링 예산 내에서 모델의 성능을 최적화할 수 있도록, 라벨링이 가장 필요한 '어려운' 샘플을 자동으로 식별하고 고품질의 pseudo-label만을 선별하여 학습에 활용하는 점진적 학습(Incremental Learning) 프레임워크를 제안하는 것이다.

## ✨ Key Contributions

본 연구의 중심적인 아이디어는 **난이도 평가 모델(Difficulty Assessment Model)**을 도입하여 데이터의 가치를 정량적으로 판단하는 것이다. 이 모델은 라벨링되지 않은 새로운 샘플이 현재의 인스턴스 분할 모델에게 얼마나 도전적인지를 예측한다.

구체적인 설계 아이디어는 다음과 같다.
1. **데이터 선별의 최적화**: 난이도 평가 모델을 통해 샘플을 'Hard', 'Neutral', 'Easy' 세 그룹으로 분류한다. 'Hard' 샘플은 전문가가 직접 폴리곤 라벨링을 수행하게 하여 학습 효율을 극대화하고, 'Easy' 샘플은 고품질의 pseudo-label을 생성하여 학습 데이터에 포함시킨다.
2. **PQ 기반의 난이도 예측**: 단순히 모델의 불확실성을 측정하는 것이 아니라, 실제 정답이 없을 때도 해당 샘플의 Panoptic Quality (PQ) 점수를 예측하도록 설계하여 라벨링 우선순위를 결정한다.
3. **점진적 파이프라인**: 데이터가 패키지 단위로 수집됨에 따라 모델을 지속적으로 업데이트하며, 라벨링 예산을 전략적으로 배분하는 구조를 제안한다.

## 📎 Related Works

### 인스턴스 분할 기법
인스턴스 분할 기술은 크게 Proposal-based(예: Mask R-CNN, CenterMask)와 Clustering-based(예: Terrace, Watershed) 방식으로 나뉜다. Proposal-based 방식은 바운딩 박스를 먼저 예측한 후 마스크를 생성하지만, 본 논문은 식품 도메인에서 뛰어난 성능을 보인 Clustering-based 방식인 Terrace를 핵심 모델로 사용한다. Terrace는 예측 결과로 Terrace probability distribution을 출력하므로, 이를 통해 모델의 확신도를 분석하고 난이도를 예측하는 데 활용할 수 있다는 차별점이 있다.

### 준지도 학습 및 멤버십 추론
기존의 준지도 학습 방식은 무작위로 샘플을 선택해 라벨링하고 나머지에 pseudo-label을 부여하지만, pseudo-label의 품질 관리 기전이 부족하다. 또한, 멤버십 추론 공격(Membership Inference Attack) 기술은 쿼리 샘플이 학습 세트에 포함되었는지를 판별하는 이진 분류 문제에 집중한다. 본 논문은 이러한 아이디어를 확장하여, 단순한 포함 여부가 아니라 샘플의 난이도(PQ 점수)를 회귀(Regression) 값으로 예측하는 방식으로 발전시켜 데이터 선택의 정밀도를 높였다.

## 🛠️ Methodology

### 전체 시스템 구조
본 프레임워크는 데이터 풀(Data Pool), 지도 학습 라벨(Supervised labels), 어노테이션 모델(Annotation model), 그리고 프로덕션 모델(Production model)의 네 가지 상태를 유지하며 점진적으로 업데이트된다.

1. **Annotation Model**: 지도 학습 라벨로만 학습된 모델로, 새로운 샘플의 난이도를 평가하기 위한 기반 예측(clustering maps)을 생성한다.
2. **Production Model**: 지도 학습 라벨과 선별된 고품질 pseudo-label을 모두 사용하여 학습되며, 실제 서비스에 사용되는 최종 모델이다.
3. **Incremental Learning Engine**: 어노테이션 모델의 출력물을 난이도 평가 모델에 입력하여 샘플을 라벨링 모듈, pseudo-label 생성 모듈, 또는 데이터 풀로 배분한다.

### 난이도 평가 모델 (Difficulty Assessment Model)
난이도 평가 모델은 ImageNet으로 사전 학습된 ResNet-50을 기반으로 하며, 입력값으로는 어노테이션 모델이 생성한 Terrace probability distribution의 스택(clustering layers)을 사용한다. 출력값은 $0$에서 $1$ 사이의 예측된 PQ 점수이며, 값이 낮을수록 모델이 처리하기 어려운 'Hard' 샘플임을 의미한다.

학습을 위한 손실 함수는 다음과 같은 평균 제곱 오차(MSE) 형식을 따른다.
$$L = \frac{\sum_{n=1}^{N} (s_n - \hat{s}_n)^2}{N}$$
여기서 $s_n$은 실제 PQ 점수(Ground-truth)이고, $\hat{s}_n$은 모델이 예측한 PQ 점수이다.

### 학습 데이터 생성 및 절차
난이도 평가 모델을 학습시키기 위해 'Shadow Model' 전략을 사용한다.
1. 라벨링된 데이터를 사용하여 여러 개의 섀도우 학습 세트와 테스트 세트를 구성한다.
2. 각 학습 세트로 섀도우 Terrace 모델을 학습시키고, 테스트 세트에 대해 추론을 수행하여 (Clustering map, 실제 PQ score) 쌍을 생성한다.
3. 이렇게 생성된 약 35,000개의 쌍 중 균등 분포로 추출한 9,100개의 데이터를 사용하여 난이도 평가 모델을 학습시킨다.

### 모델 업데이트 및 가중치
선별된 데이터는 다음과 같은 가중치 비율로 프로덕션 모델 학습에 사용된다.
- **첫 번째 패키지 데이터 : Hard 샘플 : Easy 샘플 = $2 : 4 : 1$**
이러한 가중치 설정은 특히 도전적인 케이스(Hard samples)를 더 집중적으로 학습시키기 위함이다.

## 📊 Results

### 실험 설정
- **데이터셋**: Dimsum, Sushi, Cookie, UECFood-PixComp 등 4개의 식품 데이터셋을 사용하였다.
- **지표**: 인스턴스 분할 성능은 Panoptic Quality (PQ)로 측정하였으며, 난이도 평가 모델의 성능은 실제 PQ 점수와의 평균 절대 오차(MAE)로 측정하였다.
- **비교 대상**: 무작위 샘플링(Random), 단순 준지도 학습(Random*), 멤버십 기반 방법(Membership), 제안 방법(PQ-based, PQ-based*)을 비교하였다.

### 주요 결과
1. **샘플링 전략 비교**: Dimsum 데이터셋 실험 결과, Hard-mining 전략이 Random이나 Easy 샘플링보다 PQ 점수가 $1 \sim 4\%$ 더 높게 나타났다. 이는 한정된 예산 하에서 어려운 샘플을 우선 라벨링하는 것이 효율적임을 입증한다.
2. **난이도 평가 모델의 정확도**: 예측 PQ 점수와 실제 점수 간의 MAE는 $7.8\%$로 낮게 측정되어, 모델이 샘플의 난이도를 효과적으로 예측하고 있음을 보였다.
3. **Pseudo-label 임계값**: Pseudo-label 수용 임계값을 $60\%$ PQ 점수로 설정했을 때, 라벨의 품질과 양 사이의 최적의 균형을 이루어 성능이 가장 높았다($83.34\%$).
4. **점진적 학습 전략 비교**: 제안된 $\text{PQ-based}^*$ 방식이 $\text{Random}^*$ 및 멤버십 기반 방식보다 모든 단계에서 일관되게 높은 성능을 보였다. 특히 $\text{Random}^*$ 대비 $1.5 \sim 2\%$의 PQ 향상을 달성하였다.
5. **라벨링 비용 대비 성능**: $\text{PQ-based}^*$ 방식은 전체 데이터의 $20 \sim 30\%$만 라벨링하고도 $100\%$ 전체를 라벨링한 모델과 거의 대등한 성능을 보였다.

## 🧠 Insights & Discussion

본 논문은 데이터 라벨링이라는 현실적인 비용 문제를 딥러닝 모델의 '예측 능력'을 이용해 해결하려 했다는 점에서 강점이 있다. 특히 단순히 모델의 Entropy나 Confidence를 사용하는 대신, 실제 평가 지표인 PQ 점수를 직접 예측하는 보조 모델을 설계하여 데이터 선택의 객관성을 높였다.

다만, 몇 가지 한계와 가정은 존재한다. 첫째, 난이도 평가 모델을 학습시키기 위해 초기 단계에서 상당량의 라벨링된 데이터와 섀도우 모델 학습 과정이 필요하다는 점이다. 둘째, 본 프레임워크가 Terrace라는 특정 아키텍처의 출력 특성에 의존하고 있어, 다른 인스턴스 분할 모델(예: Mask R-CNN)에 적용하기 위해서는 해당 모델에 맞는 난이도 측정 지표를 재설계해야 한다. 

비판적으로 해석하자면, $\text{PQ-based}^*$의 성능 향상이 pseudo-label의 양보다는 '저품질 데이터의 제거'에서 오는 효과가 더 클 것으로 판단된다. 이는 모델이 잘못된 정보로 인해 오염되는 것을 방지하는 필터링 기전이 점진적 학습에서 핵심적인 역할을 수행했음을 시사한다.

## 📌 TL;DR

본 논문은 식품 인스턴스 분할을 위해 **난이도 평가 모델**을 이용한 점진적 학습 프레임워크를 제안한다. 이 프레임워크는 새로운 데이터 중 모델이 처리하기 어려운 샘플은 전문가가 라벨링하게 하고, 쉬운 샘플은 고품질의 pseudo-label로 활용함으로써 라벨링 비용을 획기적으로 줄인다. 실험 결과, 전체 데이터의 $20 \sim 30\%$만 라벨링하고도 전체 라벨링 모델에 근접하는 성능을 달성하였으며, 이는 실제 산업 현장에서의 식품 데이터셋 구축 및 모델 배포에 매우 효율적인 전략이 될 가능성이 높다.
