# Instance Segmentation for Point Sets

Abhimanyu Talwar and Julien Laasri (2019)

## 🧩 Problem to Solve

본 논문은 3D Point Set의 Instance Segmentation 과정에서 발생하는 과도한 메모리 사용 문제를 해결하고자 한다. 특히, 기존의 Similarity Group Proposal Network (SGPN)는 포인트 세트의 전체 포인트 수 $N$에 대해 $N \times N$ 크기의 Similarity Matrix를 계산해야 하므로, 메모리 요구량이 포인트 수의 제곱에 비례하여 증가하는 Quadratic Memory Complexity 문제를 가지고 있다. 훈련 단계에서는 포인트 수를 고정하여 제어할 수 있으나, 테스트 단계에서 포인트 수가 많은 실제 장면을 처리할 경우 메모리 부족 현상이 발생하며 연산 시간이 급격히 증가한다. 따라서 본 연구의 목표는 성능 저하를 최소화하면서 메모리 사용량과 계산 시간을 줄일 수 있는 효율적인 포인트 샘플링 및 레이블 확장 방법을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 전체 포인트 세트가 아닌 일부의 서브 샘플(Sub-sample)에 대해서만 Instance Segmentation을 수행하고, 그 결과를 나머지 전체 포인트로 확장(Extrapolate)하는 것이다. 이를 위해 저자들은 두 가지 샘플링 전략인 Random Sub-sampling과 Grid-based Sub-sampling을 제안하였다. 핵심 직관은 소수의 랜드마크(Landmark) 포인트들만으로도 객체의 인스턴스 정보를 충분히 추출할 수 있으며, 유클리드 공간에서의 Nearest Neighbour 접근법을 통해 이 정보를 효율적으로 전파할 수 있다는 점이다.

## 📎 Related Works

포인트 세트 데이터에 딥러닝을 적용하기 위한 기존 연구로 VoxelNet이 언급된다. VoxelNet은 포인트 세트를 균등한 크기의 복셀로 나누어 3D Convolution을 적용하지만, 이는 데이터를 불필요하게 방대하게 만드는 단점이 있다. PointNet은 Symmetric Function인 Max Pooling 레이어를 사용하여 포인트의 순서 불변성(Permutation Invariance)을 해결하고 전역 형상 기술자(Global Shape Descriptor)를 학습하여 Semantic Segmentation에서 우수한 성능을 보였다. PointNet++는 전역 풀링의 한계를 극복하기 위해 이웃 영역의 기술자를 학습하는 계층적 구조를 도입하여 국부적 구조(Local Structure)를 캡처할 수 있게 하였다. SGPN은 이러한 PointNet 계열의 아키텍처를 기반으로 인스턴스 분할을 수행하지만, 앞서 언급한 $N \times N$ 유사도 행렬로 인한 메모리 효율성 문제가 한계점으로 지적되었다.

## 🛠️ Methodology

### 1. SGPN의 메모리 문제 원인

SGPN은 PointNet을 통해 각 포인트의 특징 벡터 $F_{sim} \in \mathbb{R}^{N \times f}$를 추출한다. 이후 모든 포인트 쌍 사이의 유클리드 거리를 계산하여 Similarity Matrix $S$를 생성하며, 그 요소는 다음과 같이 정의된다.
$$S_{i j}=\left\|F_{sim}^{i}-F_{sim}^{j}\right\|^{2}$$
이 행렬의 크기가 $N \times N$이므로, $N$이 커질수록 메모리 사용량이 급격히 증가한다.

### 2. 제안 방법: 샘플링 및 레이블 전파

저자들은 전체 포인트 대신 $K$개의 랜드마크 포인트만 사용하여 $K \times K$ 크기의 작은 유사도 행렬을 계산한다. 이후 랜드마크에 대해 예측된 레이블을 전체 포인트로 확장한다. 포인트 $p$에 대해 $K$개의 랜드마크 중 유클리드 거리가 가장 가까운 포인트 $q_{K}(p)$를 찾아 다음과 같이 레이블을 할당한다.
$$\text{Label}(p)=\text{Label}\left(q_{K}(p)\right)$$
이 과정에서 효율적인 탐색을 위해 SciPy의 KDTree를 사용하며, $N$번의 검색에 대해 평균 $O(N \log K)$의 시간 복잡도를 갖는다.

### 3. 샘플링 전략

- **Random Sub-sampling**: 각 블록에서 무작위로 $K$개의 포인트를 선택한다. 구현이 간단하고 속도가 매우 빠르다.
- **Grid-based Sub-sampling**: 장면 블록을 정규 격자(Regular Grid)로 나누어 공간적으로 균등하게 분포된 포인트를 선택한다. 구체적으로, 격자 점들과 가장 가까운 실제 포인트들을 선택하여 $K$개의 랜드마크를 구성한다. 이는 무작위 샘플링이 데이터 밀도가 낮은 지역의 객체를 놓칠 수 있는 문제를 해결하기 위함이다.

## 📊 Results

### 실험 설정

- **데이터셋**: Stanford Indoor 3D Dataset
- **평가 지표**: Mean Average Precision (mAP), 계산 시간(Computation Time)
- **기준선**: 전체 포인트 세트(4096개)를 사용한 기존 SGPN

### 주요 결과

- **정량적 결과**: Random Sub-sampling을 통해 포인트를 절반($K=2048$)으로 줄였을 때, mAP 성능은 전체 포인트를 사용했을 때와 비슷하게 유지되었다. 반면 메모리 사용량은 $\frac{1}{4}$로 감소하였으며, 계산 시간은 201초에서 62초로 약 3분의 1 수준으로 단축되었다.
- **정성적 결과**:
  - 샘플 수를 너무 적게 설정($K=1024$)할 경우, Random 샘플링에서는 일부 포인트들이 잘못된 인스턴스(예: 테이블 내부 포인트가 벽으로 분류)로 할당되는 경향이 발견되었다.
  - Grid-based 샘플링은 Random 샘플링이 놓치기 쉬운 저밀도 영역의 객체들을 더 잘 포착하여 보다 안정적인 세그멘테이션 결과를 보여주었다.
  - 흥미롭게도 일부 사례에서는 mAP 점수는 낮지만, 샘플링을 통해 결과가 평활화(Smoothing)되어 시각적으로 더 나은 인스턴스 분할 결과가 나타나기도 하였다.

## 🧠 Insights & Discussion

본 연구는 계산 복잡도가 높은 딥러닝 모델을 실제 대규모 데이터에 적용할 때, 단순한 샘플링과 보간법(Interpolation)만으로도 상당한 효율성을 얻을 수 있음을 보여주었다.

- **강점 및 트레이드-오프**: Random 샘플링은 속도와 메모리 효율성 면에서 압도적이지만, 공간적 커버리지가 부족하여 작은 객체나 저밀도 영역을 놓칠 위험이 있다. 반면 Grid-based 샘플링은 공간적 분포를 보장하여 정확도를 높일 수 있지만, 격자 밀도를 설정하는 하이퍼파라미터에 민감하며 연산 비용이 약간 더 높다.
- **한계 및 비판적 해석**: 본 논문은 레이블 전파를 단순히 유클리드 공간의 거리 기반으로 수행하였다. 하지만 저자들도 논의했듯이, SGPN이 이미 특징 공간(Feature Space)에서 유사도를 학습했으므로, 실제 공간이 아닌 특징 공간에서의 Nearest Neighbour를 사용하는 것이 더 이론적으로 타당할 수 있다.
- **결론적 해석**: 전체 포인트에 대해 $N \times N$ 행렬을 만드는 대신, $K \times K$ 행렬과 $O(N \log K)$의 전파 과정을 사용하는 것이 실용적인 관점에서 매우 효율적인 대안임을 입증하였다.

## 📌 TL;DR

이 논문은 3D 포인트 클라우드 인스턴스 분할 모델인 SGPN의 과도한 메모리 사용 문제($N \times N$ 유사도 행렬)를 해결하기 위해, **소수의 랜드마크 포인트만 샘플링하여 예측한 뒤 이를 주변으로 전파하는 방식**을 제안한다. Random 및 Grid-based 샘플링을 통해 mAP 성능 저하를 최소화하면서 메모리 사용량을 획기적으로 줄이고 추론 속도를 3배 이상 향상시켰다. 이는 향후 대규모 3D 장면 처리 및 실시간 인스턴스 분할 연구에 있어 효율적인 연산 전략의 가능성을 제시한다.
