# VLM See, Robot Do: Human Demo Video to Robot Action Plan via Vision Language Model

Beichen Wang, Juexiao Zhang, Shuwen Dong, Irving Fang, and Chen Feng (2024)

## 🧩 Problem to Solve

본 논문은 로봇이 인간의 시연 영상(Human Demonstration Videos)으로부터 직접 로봇 작업 계획(Task Plan)을 학습하고 생성하는 문제를 해결하고자 한다.

기존의 로봇 학습 방식은 주로 자연어 지침에 의존하거나 로봇의 시점 데이터(Teleoperation)를 통한 모방 학습(Imitation Learning)을 사용하였다. 그러나 자연어 지침은 복잡하고 긴 단계(Long-horizon)를 가진 작업이나 시공간적 의존성이 강한 작업을 정밀하게 표현하는 데 한계가 있다. 또한, 인간 시연 영상을 직접 사용하는 모방 학습의 경우, 인간과 로봇 사이의 물리적 구조 차이인 Embodiment Gap으로 인해 도메인 간격(Domain Gap)이 크며, 작업이 길어질수록 방대한 양의 시연 데이터가 필요하다는 문제가 있다.

따라서 본 연구의 목표는 Vision Language Model(VLM)의 상식 추론 능력과 일반화 성능을 활용하여, 인간의 시연 영상을 분석하고 이를 로봇이 수행 가능한 단계별 작업 계획으로 변환하는 파이프라인인 **SeeDo**를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 VLM이 비디오의 모든 프레임을 처리하는 데 어려움을 겪고 공간적 관계 파악 능력이 부족하다는 점을 인지하고, 이를 보완하기 위해 **'키프레임 선택 $\rightarrow$ 시각적 인지 $\rightarrow$ VLM 추론'**으로 이어지는 시스템 중심의 파이프라인을 설계한 것이다.

중심적인 설계 아이디어는 다음과 같다.

1. **Hand-speed Heuristics 기반 키프레임 추출**: 모든 프레임을 입력하는 대신, 손의 속도가 느려지는 시점(Troughs)을 포착하여 물체와의 상호작용이 일어나는 핵심 프레임만을 선택함으로써 VLM의 컨텍스트 제한 문제를 해결한다.
2. **시각적 프롬프팅(Visual Prompting) 강화**: SAM2(Segment Anything Model 2)를 이용한 객체 추적 결과를 마스크 외곽선(Contours)과 ID 형태로 이미지에 직접 표시하여, VLM이 유사하게 생긴 물체를 구분하고 정확한 공간 관계를 파악하도록 돕는다.
3. **Chain-of-Thought(CoT) 추론**: 복잡한 작업을 한 번에 생성하지 않고 '객체 식별 $\rightarrow$ 선택된 객체 파악 $\rightarrow$ 참조 객체 설정 $\rightarrow$ 공간 관계 정의'의 단계적 추론 과정을 거쳐 계획의 정확도를 높인다.

## 📎 Related Works

### 관련 연구 및 한계

- **VLM 기반 작업 계획**: 기존 연구들은 주로 자연어 지침이나 이미지를 입력으로 하여 작업 계획을 생성하거나, $Code\ as\ Policies$와 같이 로봇 실행 코드를 직접 생성하는 방식을 취했다. 하지만 이는 여전히 텍스트 기반 지침에 의존하며, 비디오를 통한 직접적인 작업 명세(Task Specification) 학습은 부족했다.
- **인간 비디오 기반 로봇 학습**: 모방 학습(Imitation Learning) 분야에서 인간의 영상을 활용하려는 시도가 많았으나, 앞서 언급한 Embodiment Gap과 환경 변화에 대한 취약성, 그리고 long-horizon 작업 시 데이터 요구량 급증이라는 한계가 있었다.
- **비디오 입력 VLM**: 최근 비디오 분석(QA, Captioning)이 가능한 VLM들이 등장했으나, 로봇 제어에 필요한 정밀한 공간적/시간적 추론을 수행하기에는 여전히 부족함이 많다.

### SeeDo의 차별점

SeeDo는 단순히 비디오를 VLM에 입력하는 것이 아니라, 로봇 공학적 특성(손 속도 기반 키프레임)과 정밀한 시각적 보조 도구(SAM2 추적)를 결합한 파이프라인을 구축함으로써, 기존 VLM의 한계를 시스템적으로 보완하고 이를 실제 로봇 실행 코드(LMPs)와 연결했다는 점에서 차별성을 가진다.

## 🛠️ Methodology

SeeDo의 전체 파이프라인은 크게 세 가지 모듈로 구성되며, 최종적으로 생성된 계획은 Language Model Programs(LMPs)를 통해 로봇 API로 전달된다.

### 1. Keyframe Selection 모듈

VLM의 컨텍스트 길이 제한과 정보 손실을 방지하기 위해, 단순히 균등하게 프레임을 샘플링하는 대신 손의 움직임을 분석한다.

- **절차**: 가벼운 손 검출 모델을 사용하여 시간에 따른 손의 속도를 측정한다.
- **핵심 원리**: 물체를 집거나 놓는 동작(Pick-and-Place) 시 손의 속도가 느려진다는 점에 착안하여, 속도 그래프의 골(Troughs)에 해당하는 프레임을 키프레임으로 선택한다.
- **필터링**: 선택된 프레임 중 VLM이 판단하기에 손-물체 상호작용이 없는 노이즈 프레임은 제거한다.

### 2. Visual Perception 모듈

VLM의 시각적 인지 능력(특히 유사 객체 구분 및 공간 관계 파악)을 보완한다.

- **객체 탐지 및 추적**: VLM이 비디오 내 객체 리스트를 먼저 식별하면, GroundingDINO를 통해 첫 프레임에서 바운딩 박스를 추출하고, 이를 SAM2에 입력하여 비디오 전체에서 객체를 추적한다.
- **시각적 프롬프트 생성**: 추적된 결과(마스크 외곽선 및 ID)를 앞서 선택한 키프레임 위에 덧씌운다. 이때 내부를 채우지 않고 외곽선(Contours)만 표시하여 물체의 외형을 가리지 않도록 한다.
- **공간 정보 추가**: 마스크 중심 좌표와 ID를 텍스트 프롬프트에 함께 제공하여 VLM이 상대적 위치를 더 잘 이해하게 한다.

### 3. VLM Reasoning 모듈

GPT-4o를 기반으로 하며, Chain-of-Thought(CoT) 프롬프팅을 통해 최종 작업 계획을 생성한다. 추론 단계는 다음과 같다.

- **Step 1 (필터링)**: 선택된 키프레임이 실제로 조작 중인 프레임인지 확인한다.
- **Step 2 (객체 식별)**: 'Pick' 프레임에서 어떤 물체가 집혔는지 식별한다.
- **Step 3 (참조 객체 설정)**: 'Place' 프레임에서 물체가 놓일 기준이 되는 참조 객체(Reference Object)를 선택한다.
- **Step 4 (공간 관계 정의)**: 집어 올린 물체와 참조 객체 사이의 지배적인 공간 관계를 다음 6가지 중 하나로 선택한다: $\text{\{in, on top of, at the back of, in front of, to the left, to the right\}}$.

### 4. Plan Execution

생성된 텍스트 기반 작업 계획은 $Code\ as\ Policies$ 접근 방식을 통해 로봇이 실행 가능한 Python 코드 형태의 LMPs로 변환되며, UR10e 로봇 팔의 API를 호출하여 실제 물리 환경이나 시뮬레이션(Pybullet)에서 수행된다.

## 📊 Results

### 실험 설정

- **작업 종류**: Vegetable Organization(채소 정리), Garment Organization(의류 정리), Wooden Block Stacking(나무 블록 쌓기)의 세 가지 long-horizon pick-and-place 작업.
- **평가 지표**:
  - $TSR$ (Task Success Rate): 생성된 계획이 시연 영상의 단계 및 순서와 완전히 일치하는지 측정.
  - $FSR$ (Final-state Success Rate): 순서와 상관없이 최종 상태가 시연 영상과 일치하는지 측정.
  - $SSR$ (Step Success Rate): 전체 정답 단계 중 순서대로 일치하는 단계의 비율을 측정.
- **비교 대상 (Baselines)**: Gemini 1.5 Pro, LLaVA-OneVision, VILA, 그리고 GPT-4o의 변형 모델(초기/최종 프레임만 사용, 균등 샘플링 사용 등).

### 주요 결과

- **정량적 성능**: SeeDo는 모든 작업에서 TSR, FSR, SSR 지표 모두에서 기존 비디오 VLM 베이스라인들을 압도했다. 특히 일상 작업(채소, 의류)의 경우 SSR이 70%를 상회하였다.
- **키프레임 선택의 효과**: 균등 샘플링(Uniform Sampling)을 사용한 GPT-4o는 거의 모든 작업에서 실패했다. 이는 핵심 동작 프레임을 놓치거나 컨텍스트 제한을 초과하기 때문이며, 손 속도 기반 선택의 중요성을 입증한다.
- **시각적 프롬프팅의 효과**: 나무 블록 쌓기 작업(유사 객체 다수 존재)에서 시각적 프롬프팅을 제거했을 때 TSR이 0%로 급락한 반면, SeeDo는 21.62%를 기록했다. 이는 SAM2 기반의 추적 정보가 공간 추론 성능을 비약적으로 향상시킴을 보여준다.
- **에러 분석**: SeeDo의 가장 주된 실패 원인은 **Spatial Error(공간 에러)**였다. 이는 현재 VLM의 공간 지능 한계와 추적 모듈의 불완전함에서 기인한다. 반면, Temporal Error(시간적 순서 에러)는 타 모델 대비 현저히 낮았다.

## 🧠 Insights & Discussion

### 강점

본 연구는 VLM이 비디오 전체를 이해하려 하기보다, 로봇 공학적인 힌트(손 속도)와 보조적인 시각 정보(SAM2 추적)를 통해 정보를 압축하고 정제하여 전달했을 때 훨씬 강력한 성능을 낸다는 것을 보여주었다. 특히 long-horizon 작업에서 시간적 순서를 유지하며 계획을 세우는 능력이 탁월하다.

### 한계 및 미해결 질문

1. **제한된 액션 공간**: 본 연구는 Pick-and-Place 동작에 국한되어 있다. 더 복잡한 동작(예: 붓기, 닦기 등)으로의 확장이 필요하다.
2. **공간 지능의 한계**: 시각적 프롬프팅으로 개선했음에도 불구하고, 다방향의 정밀한 공간 관계가 필요한 나무 블록 쌓기 등에서는 여전히 오류가 발생한다. 이는 VLM 자체의 공간 추론 능력 개선이 필요함을 시사한다.
3. **정밀한 위치 정의 부족**: 현재는 '왼쪽', '위'와 같은 고수준의 상대적 관계로만 정의하고 있으며, 구체적인 좌표값이나 정밀한 조작을 위한 수치적 정보는 추출하지 못하고 있다.

## 📌 TL;DR

SeeDo는 인간의 시연 영상에서 로봇 작업 계획을 추출하기 위해 **손 속도 기반 키프레임 선택, SAM2 기반 시각적 프롬프팅, 그리고 GPT-4o의 CoT 추론**을 결합한 파이프라인이다. 실험 결과, 기존의 비디오 VLM보다 시간적/공간적 추론 성능이 월등히 뛰어남을 확인했으며, 이는 데이터 수집 비용이 높은 로봇 학습 분야에서 인터넷의 방대한 인간 영상을 활용할 수 있는 가능성을 제시한다.
