# Demystifying CLIP Data

Hu Xu, Saining Xie, Xiaoqing Ellen Tan, Po-Yao Huang, Russell Howes, Vasu Sharma, Shang-Wen Li, Gargi Ghosh, Luke Zettlemoyer, Christoph Feichtenhofer (2024)

## 🧩 Problem to Solve

본 논문은 Contrastive Language-Image Pre-training (CLIP)의 성공 요인이 모델 아키텍처나 학습 목표보다는 학습 데이터의 품질과 구성에 있다고 분석한다. 그러나 OpenAI가 공개한 CLIP의 데이터 수집 및 큐레이션(curation) 과정은 매우 제한적인 정보만 제공되어, 연구 커뮤니티에서는 이를 블랙박스로 취급해 왔다.

기존의 CLIP 데이터 복제 시도(예: LAION)는 CLIP 모델 자체를 필터로 사용하여 노이즈를 제거하는 방식을 사용했다. 하지만 이는 결과적으로 CLIP이 이미 학습한 정보를 증류(distillation)하는 것에 불과하며, 모델 필터에 의한 편향(bias)이 개입될 수 있다는 문제가 있다. 따라서 본 연구의 목표는 외부 모델에 의존하지 않고, 투명하고 공개 가능한 방식으로 고품질의 데이터셋을 구축하는 큐레이션 방법론을 제안하고 검증하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 **Metadata-Curated Language-Image Pre-training (MetaCLIP)** 방법론의 제안이다. MetaCLIP의 중심 아이디어는 모델 기반의 필터링 대신, 구조화된 **메타데이터(Metadata)**를 활용하여 데이터의 분포를 인위적으로 균형 있게 조정(balancing)하는 것이다.

저자들은 단순히 데이터의 양을 늘리는 것보다, 메타데이터 분포를 통해 특정 개념이 과하게 지배하는 것을 막고 희소한 개념(tail entries)을 보존하는 것이 모델의 일반화 성능과 태스크 불가지론적(task-agnostic) 특성을 확보하는 데 결정적이라는 직관을 제시한다.

## 📎 Related Works

기존 연구들은 크게 두 가지 방향으로 진행되었다. 첫째는 이미 구축된 데이터셋에서 사전 학습된 모델을 이용해 중요 샘플을 선택하는 데이터 프루닝(Data Pruning) 기법이다. 둘째는 LAION이나 DataComp와 같이 웹에서 대규모 데이터를 수집하고 CLIP 모델을 블랙박스 필터로 사용하여 노이즈를 제거하는 방식이다.

본 논문은 이러한 기존 접근 방식과 다음과 같은 차별점을 가진다.

1. **모델 독립성**: MetaCLIP은 데이터 필터링 과정에서 CLIP과 같은 외부 ML 모델을 전혀 사용하지 않으며, 순수하게 NLP 기반의 메타데이터 매칭과 샘플링만을 이용한다.
2. **분포 제어**: 기존 방식들이 주로 데이터의 '양'을 맞추는 데 집중했다면, MetaCLIP은 메타데이터 분포의 '균형'을 맞추는 것에 집중하여 데이터 편향을 최소화한다.
3. **투명성**: 큐레이션 과정이 명확한 알고리즘으로 정의되어 있어, 누구나 동일한 파이프라인을 통해 데이터를 구축할 수 있다.

## 🛠️ Methodology

MetaCLIP의 전체 파이프라인은 크게 메타데이터 구축, 서브스트링 매칭, 그리고 균형 샘플링의 세 단계로 구성된다.

### 1. 메타데이터 구축 ($M$)

먼저 학습 데이터의 기준이 될 50만 개의 엔트리로 구성된 메타데이터 $M$을 구축한다. 구성 요소는 다음과 같다.

- **WordNet synsets**: 모든 시놉셋(synsets) 포함.
- **Wikipedia uni-grams**: 영어 위키피디아에서 100회 이상 등장한 단어.
- **Wikipedia bi-grams**: Pointwise Mutual Information (PMI) 값이 높은 바이그램.
- **Wikipedia titles**: 조회수가 일정 수준 이상인 위키피디아 문서 제목.

### 2. 서브스트링 매칭 (Sub-string Matching)

수집된 원시 데이터 풀 $D$의 텍스트가 메타데이터 $M$의 엔트리를 포함하고 있는지 확인하는 과정이다. 텍스트 $text$에 대해 $text \to entry$ 매핑을 생성하며, 이 과정에서 메타데이터의 어떤 항목과도 매칭되지 않는 저품질 텍스트는 자동으로 제거된다.

### 3. 균형 샘플링 (Query and Balancing)

본 방법론의 핵심으로, 특정 엔트리가 데이터셋 전체를 지배하는 것을 방지하기 위해 최대 임계값 $t = 20,000$을 적용한다.

- **Tail entries**: 매칭된 샘플 수가 $t$보다 적은 엔트리는 모든 샘플을 유지한다.
- **Head entries**: 매칭된 샘플 수가 $t$보다 많은 엔트리는 $t$개만 남기고 무작위로 서브샘플링한다.

이 과정은 데이터 분포를 평탄하게 만들어, 흔한 단어(예: "photo", "image")에 의한 노이즈를 줄이고 희귀한 시각적 개념을 보존함으로써 모델이 더 다양한 데이터를 학습하게 한다.

### 4. 효율적인 큐레이션 알고리즘

대규모 데이터셋에서 역색인(inverted index)을 구축하는 것은 메모리 비용이 매우 크다. 이를 해결하기 위해 저자들은 각 엔트리별 총 매칭 횟수($entry\_count$)만 저장하고, 독립 샘플링(independent sampling) 방식을 제안한다.

특정 이미지-텍스트 쌍이 선택될 확률은 다음과 같이 결정된다. 각 엔트리 $entry\_id$에 대해 샘플링 확률 $entry\_prob$를 다음과 같이 정의한다.
$$entry\_prob[entry\_id] = \min\left(1, \frac{t}{entry\_count[entry\_id]}\right)$$
데이터 풀의 각 쌍을 순회하며, 해당 쌍이 가진 매칭 엔트리 중 하나라도 선택되면 최종 데이터셋 $D^*$에 포함시킨다.

## 📊 Results

### 실험 설정

- **데이터셋**: CommonCrawl (CC)에서 수집한 데이터 풀을 사용하였다.
- **비교 대상**: OpenAI CLIP (WIT400M 데이터), OpenCLIP (LAION-400M 데이터).
- **평가 지표**: ImageNet zero-shot classification 정확도 및 기타 26개 벤치마크의 평균 정확도.
- **모델 크기**: ViT-B/32, ViT-B/16, ViT-L/14, ViT-H/14, ViT-bigG/14.

### 주요 결과

1. **성능 우위**: 400M 규모의 데이터에서 MetaCLIP은 모든 모델 크기에서 CLIP과 LAION을 압도했다. 특히 ViT-B/16 모델의 ImageNet zero-shot 정확도에서 CLIP(68.3%)보다 높은 **70.8%**를 달성했다.
2. **데이터 스케일링**: 동일한 학습 예산 내에서 데이터 규모를 1B, 2.5B로 확장했을 때 성능이 더욱 향상되었다. 특히 ViT-bigG 모델은 ImageNet에서 **82.1%**라는 높은 정확도를 기록했다.
3. **균형의 중요성 (Ablation Study)**:
    - 데이터의 양을 4배 늘려 1.6B의 불균형한(unbalanced) 데이터를 학습시킨 경우, 오히려 400M의 균형 잡힌 MetaCLIP 데이터보다 ImageNet 정확도가 크게 낮게 나타났다 ($61.9\%$ vs $65.5\%$).
    - 이는 단순히 데이터 양을 늘리는 것보다 분포의 균형을 맞추는 것이 훨씬 더 중요하다는 것을 증명한다.

## 🧠 Insights & Discussion

### 강점 및 해석

본 연구는 CLIP의 성공 비결이 '양질의 데이터'에 있으며, 그 '질'의 핵심은 단순한 노이즈 제거가 아니라 **분포의 균형**에 있음을 정량적으로 보여주었다. 메타데이터를 활용한 큐레이션은 인간이 설계한 복잡한 필터나 블랙박스 모델 없이도 효율적으로 고품질 데이터셋을 구축할 수 있음을 시사한다.

### 한계 및 논의사항

- **메타데이터 의존성**: 본 방법론은 위키피디아와 WordNet이라는 특정 지식 베이스에 의존한다. 만약 메타데이터 자체가 편향되어 있거나 특정 도메인의 지식을 포함하지 않는다면, 큐레이션 결과 또한 해당 한계를 가질 수 있다.
- **임계값 $t$의 설정**: $t=20,000$이라는 수치가 400M 규모에서는 최적이었으나, 데이터 풀의 크기가 변함에 따라 이 최적값 또한 변할 수 있다. 논문에서는 $t$를 조정하며 실험했으나, 이를 일반화할 수 있는 공식은 제시되지 않았다.

## 📌 TL;DR

MetaCLIP은 CLIP 모델의 정체불명인 데이터 큐레이션 과정을 분석하여, 외부 모델 없이 **구조화된 메타데이터 기반의 균형 샘플링**만으로 고품질 데이터셋을 구축할 수 있음을 보였다. 실험 결과, 데이터의 절대적인 양보다 **분포의 균형을 맞추는 것이 모델 성능 향상에 더 결정적**이며, 이를 통해 OpenAI CLIP 및 LAION-400M보다 뛰어난 zero-shot 성능을 달성하였다. 이 연구는 향후 대규모 시각-언어 모델의 데이터 구축 시 모델 필터링에 의존하지 않는 투명하고 효율적인 가이드라인을 제공한다.
