# Vision-Language Model Selection and Reuse for Downstream Adaptation

Hao-Zhe Tan, Zhi Zhou, Yu-Feng Li, Lan-Zhe Guo (2025)

## 🧩 Problem to Solve

본 논문은 사전 학습된 Vision-Language Model (VLM)을 특정 다운스트림 작업(downstream task)에 맞게 효율적으로 선택하고 재사용하는 문제를 다룬다. 최근 CLIP, ALIGN과 같은 VLM이 다양한 시각적 작업에서 뛰어난 제로샷(zero-shot) 성능을 보이며 널리 사용되고 있으며, 수많은 오픈 소스 VLM 변형들이 출시되었다.

그러나 모든 다운스트림 작업에서 일관되게 최고의 성능을 내는 단일 VLM은 존재하지 않는다. 특정 도메인에서는 성능이 낮은 모델이 다른 특정 작업에서는 오히려 더 높은 성능을 보일 수 있으며, 심지어 동일한 작업 내에서도 클래스별로 강점을 가진 모델이 다르다. 그렇다고 하여 가용한 모든 VLM을 매번 평가하여 최적의 모델을 찾는 것은 시간과 데이터, 계산 리소스의 한계로 인해 현실적으로 불가능하다. 따라서 본 논문의 목표는 데이터와 계산 효율성을 유지하면서, 타겟 작업의 요구사항에 가장 적합한 VLM을 식별하고 이를 효과적으로 결합하여 사용할 수 있는 새로운 패러다임을 제시하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 **Model Label Learning (MLL)**이라는 새로운 VLM 선택 및 재사용 패러다임을 제안한 것이다. MLL의 중심 아이디어는 후보 VLM들을 모델 허브(model hub)로 구성하고, 각 모델의 전문성과 유용성을 나타내는 '모델 레이블(model label)'을 사전에 부여하여 관리하는 것이다.

MLL은 크게 세 가지 모듈로 구성된다:

1. **Model Labeling**: 모델의 특성을 설명하는 레이블을 생성한다.
2. **Model Selection**: 타겟 작업의 요구사항과 모델 레이블을 매칭하여 적합한 모델을 선택한다.
3. **Model Reuse**: 선택된 모델들을 앙상블(ensemble) 방식으로 결합하여 타겟 작업에 적용한다.

이 방식은 모델 레이블링 과정이 타겟 작업과 독립적으로 수행되므로 계산 효율성이 매우 높으며, 허브에 추가되는 모델의 수에 따라 시스템의 능력이 확장될 수 있는 확장성(growable)을 가진다.

## 📎 Related Works

### 관련 연구 및 한계

1. **Vision-Language Models**: CLIP, BLIP 등 대규모 이미지-텍스트 쌍으로 학습된 모델들이 제로샷 성능을 높이기 위해 아키텍처 및 데이터셋 개선 연구가 많이 이루어졌다. 하지만 모델 허브(open-clip, HuggingFace 등)는 단순한 저장소 역할만 할 뿐, 사용자가 자신의 작업에 맞는 모델을 선택할 수 있는 효과적인 메커니즘을 제공하지 않는다.
2. **Model Selection**: NCE, LEEP, LogME 등 사전 학습 모델의 전이 가능성(transferability)을 측정하는 연구들이 있었으나, 대부분 단일 모달(single-modal) 모델에 집중되어 있어 VLM에 직접 적용하기 어렵다. VLM 전용 선택 방법인 LOVM(Zohar et al., 2023)은 텍스트 정보만을 활용하며, 다운스트림 작업과 학습 작업 간의 도메인 시프트가 발생할 경우 효율성이 떨어진다는 한계가 있다.
3. **Learnware**: 모델의 능력을 설명하는 사양(specification)을 구축하여 모델을 선택하는 패러다임이다. RKME 등을 통해 모델을 식별하지만, VLM과 같은 대규모 모델의 경우 학습 데이터에 접근하여 사양을 구축하는 것이 리소스 측면에서 비현실적이다.

### 차별점

MLL은 VLM의 학습 데이터에 접근하지 않고도, 공통적인 시각적 개념을 담은 **Semantic Graph**를 통해 모델의 능력을 레이블링함으로써 기존의 전이 학습 기반 선택 방식이나 데이터 중심의 Learnware 방식이 가진 한계를 극복한다.

## 🛠️ Methodology

### 1. 제로샷 시각 작업의 기초

VLM은 이미지 인코더 $I$와 텍스트 인코더 $T$로 구성된다. 이미지 $x$와 클래스 $y$에 대해, 예측값 $\hat{y}$는 다음과 같은 소프트맥스 기반의 코사인 유사도로 결정된다.

$$\hat{y} = \arg \max_{y \in Y^T} \frac{\exp(\text{sim}(I(x), T(y))/\tau)}{\sum_{y' \in Y^T} \exp(\text{sim}(I(x), T(y'))/\tau)}$$

여기서 $\text{sim}(\cdot, \cdot)$는 코사인 유사도이며, $\tau$는 온도 하이퍼파라미터이다.

### 2. Model Labeling (모델 레이블링)

모델의 능력을 정량화하기 위해 WordNet synsets를 기반으로 **Semantic Graph $G$**를 구축한다.

- 각 노드 $v$는 시각적 개념을 나타내며, 각 노드에는 대표 이미지 집합 $X_v$와 캡션 $d_v$가 할당된다.
- 모델 $f_m$이 제출되면, 이미지 인코더 $I_m$과 텍스트 인코더 $T_m$을 사용하여 각 노드 $v$의 이미지 $x \in X_v$와 캡션 $d_v$ 사이의 유사도를 계산한다.

$$s_{m,x}^v = \text{sim}(I_m(x), T_m(d_v)), \quad x \in X_v$$
$$s_m^v = \{s_{m,x}^v \mid x \in X_v\}$$
$$S_m = \{s_m^v \mid v \in V_G\}$$

이렇게 생성된 $S_m$이 해당 모델의 전문성을 나타내는 모델 레이블이 된다.

### 3. Model Selection (모델 선택)

타겟 작업 $T$의 클래스 $Y^T$가 주어지면, 이를 Semantic Graph의 노드 $V_G$와 매칭해야 한다.

- **캡션 생성**: GPT-4를 사용하여 타겟 클래스에 대한 상세 캡션 $D_T$를 생성하고, 이를 그래프 노드의 캡션 $D_G$와 비교하여 유사도가 높은 상위 $k$개의 노드를 선택한다.
- **전이 행렬 $Z$**: 노드 $v$와 클래스 $y$ 간의 캡션 유사도를 기반으로 전이 행렬 $z_{vy}$를 구성한다.
- **정밀도 예측**: 모델 $f_m$의 특정 노드 $v$에서의 정밀도 $p_{m,v}$를 계산하고, 이를 통해 타겟 클래스 $y$에서의 예측 정밀도 $p_{m,y}$를 도출한다.

$$p_{m,y} = \sum_{v \in V_{Selected}} p_{m,v} \cdot z_{vy}$$

- **재사용 지표 $r_{m,y}$**: 특정 클래스 성능과 전반적인 성능의 균형을 맞추기 위해 가중치 $\alpha$를 도입한다.

$$r_{m,y} = \alpha \cdot p_{m,y} + \frac{1-\alpha}{|Y^T|} \sum_{y' \in Y^T} p_{m,y'}$$

### 4. Model Reuse (모델 재사용)

선택된 상위 $k$개의 모델을 앙상블하여 최종 예측을 수행한다.

- **가중치 부여**: VLM의 과잉 확신(overconfidence) 문제를 해결하기 위해 확률 엔트로피 $H$를 기반으로 가중치 $w_{m,y}$를 설정하여, 불확실성이 높은 모델의 영향력을 조절한다.
- **최종 예측**: 각 클래스별 앙상블 예측값 $p_y^k(x)$ 중 가장 높은 값을 가진 클래스를 최종 결과로 선택한다.

$$\hat{y}(x) = \arg \max_{y \in Y^T} p_y^k(x)$$

## 📊 Results

### 실험 설정

- **데이터셋**: 49개의 사전 학습 VLM (open-clip 라이브러리), 5개의 샘플 데이터셋(Semantic Graph 구축용), 17개의 타겟 데이터셋(다운스트림 작업용)을 사용하였다.
- **비교 대상**:
  - **INB (ImageNet Baseline)**: ImageNet 성능이 가장 좋은 단일 모델을 선택.
  - **ModelGPT**: 텍스트 캡션을 통해 VLM의 성능을 예측하는 기존 방법.
- **평가 지표**: 제로샷 분류 정확도(Accuracy)를 사용하였다.

### 주요 결과

1. **제로샷 성능**: 표 1(단일 모델 $k=1$)과 표 2(앙상블 $k=3$) 결과, 제안 방법(Proposal)이 대부분의 작업에서 INB와 ModelGPT보다 높은 성능을 보였다. 특히 ModelGPT는 INB와 유사한 경향을 보였으나, MLL은 도메인 특화 모델을 더 잘 찾아내어 성능을 크게 향상시켰다.
2. **허브 확장성**: 모델 허브의 크기가 커질수록 타겟 작업에 적합한 모델을 발견할 확률이 높아지며, 이에 따라 평균 성능이 지속적으로 상승하는 결과(Figure 3)를 확인하였다.
3. **Ablation Study**:
    - $\alpha$ 값의 변화에 대해 성능이 비교적 견고하게 유지되었으며, $\alpha=0.7$에서 최적의 성능을 보였다.
    - 모델 재사용 수 $k$를 늘렸을 때 성능 향상은 미미한 반면 인터페이스 시간 비용은 증가하였으며, $k=1 \sim 3$ 범위가 효율성과 성능의 최적 균형점임을 확인하였다.

## 🧠 Insights & Discussion

### 강점

본 논문은 VLM 선택 문제를 위해 'Semantic Graph'라는 매개체를 도입함으로써, 타겟 데이터셋 없이도 모델의 능력을 정량화할 수 있는 효율적인 체계를 구축하였다. 특히 모델 레이블링을 사전 수행함으로써 실제 선택 단계에서의 계산 비용을 획기적으로 줄였으며, 앙상블 시 엔트로피 기반 가중치를 적용하여 VLM의 고질적인 문제인 과잉 확신 문제를 완화한 점이 돋보인다.

### 한계 및 논의사항

1. **작업 범위의 국한**: 현재의 제안 방식은 시각적 분류(classification) 작업에만 초점이 맞춰져 있다. 세그멘테이션, 디텍션 등 더 복잡한 비전 작업이나 다른 형태의 멀티모달 작업으로의 확장 가능성에 대해서는 추가 연구가 필요하다.
2. **Semantic Graph의 의존성**: WordNet 기반의 그래프가 실제 세상의 모든 시각적 개념을 완벽하게 대변하는지에 대한 의문이 있을 수 있다. 그래프의 노드 구성이 모델 선택의 상한선을 결정하므로, 더 정교하고 동적인 그래프 구축 방법이 요구된다.
3. **가정**: 사용자가 타겟 작업의 도메인과 클래스 정보를 제공한다는 가정이 필요하며, 이 정보의 정확도가 GPT-4의 캡션 생성 품질과 직결되어 최종 선택에 영향을 줄 수 있다.

## 📌 TL;DR

본 논문은 수많은 VLM 중 타겟 작업에 최적인 모델을 효율적으로 선택하고 재사용하기 위한 **Model Label Learning (MLL)** 패러다임을 제안한다. Semantic Graph를 통해 모델의 전문성을 레이블링하고, GPT-4 기반의 캡션 매칭으로 최적의 모델을 선택하며, 엔트로피 가중치 앙상블을 통해 성능을 극대화한다. 49개 모델과 17개 데이터셋으로 구성된 벤치마크를 통해 기존 방식보다 우수한 성능과 뛰어난 확장성을 입증하였으며, 향후 대규모 VLM 허브 구축을 위한 핵심 메커니즘으로 활용될 가능성이 높다.
