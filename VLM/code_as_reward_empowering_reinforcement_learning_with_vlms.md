# Code as Reward: Empowering Reinforcement Learning with VLMs

David Venuto, Sami Nur Islam, Martin Klissarov, Doina Precup, Sherry Yang, Ankit Anand (2024)

## 🧩 Problem to Solve

강화학습(Reinforcement Learning, RL) 에이전트를 실제 환경에 배치하기 위해서는 목표를 정확히 캡처하면서도 학습 효율성을 높일 수 있는 적절한 보상 함수(Reward Function)를 정의하는 것이 필수적이다. 그러나 보상 함수 설계는 일반적으로 도메인 전문가의 상당한 노력이 필요하며, 특히 보상이 희소한(Sparse Reward) 환경에서는 학습 속도가 매우 느리거나 학습 자체가 불가능한 경우가 많다.

최근 Vision-Language Models(VLMs)가 시각적 개념 이해 및 복잡한 작업 분해 능력을 보여주며 보상 모델로 활용될 가능성이 제기되었다. 하지만 VLM을 직접적으로 쿼리하여 보상을 계산하는 방식은 다음과 같은 세 가지 주요 문제점이 존재한다.

1. **계산 효율성 저하**: VLM의 거대한 모델 크기로 인해 매 스텝마다 쿼리를 수행하는 것은 계산 비용이 너무 크며 학습 속도를 현저히 늦춘다.
2. **신뢰성 부족**: VLM의 출력은 확률적(Stochastic)이므로 일관되지 않은 보상을 제공할 수 있다.
3. **해석 및 검증 불가능**: VLM이 출력하는 보상 값은 블랙박스 형태이므로, 왜 그러한 보상이 주어졌는지 해석하거나 검증하기 어렵다.

본 논문의 목표는 VLM의 추론 능력을 활용하되, 이를 직접 쿼리하는 대신 **실행 가능한 코드(Code) 형태의 보상 함수를 생성**함으로써 효율성, 신뢰성, 해석 가능성을 동시에 확보하는 프레임워크인 **VLM-CaR (Code as Reward)**을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 VLM을 보상 계산기가 아닌 **보상 함수 설계자(Reward Designer)**로 사용하는 것이다.

VLM이 환경의 초기 상태와 목표 상태 이미지를 보고 작업을 이해한 뒤, 이를 달성하기 위한 하위 작업(Sub-tasks)을 정의하고 각 하위 작업의 완료 여부를 판단하는 **Python 코드**를 생성하게 한다. 이렇게 생성된 코드는 한 번 생성되면 이후 RL 학습 과정에서 매우 빠르게 실행될 수 있으며, 사람이 코드를 직접 검토할 수 있어 해석 가능하고, 소량의 데이터를 통해 검증 가능하다.

## 📎 Related Works

### 1. Vision-Language Models (VLMs)

CLIP이나 GPT-4와 같은 VLM은 이미지와 텍스트 간의 복잡한 관계를 학습하여 장면 묘사 및 시각적 질문 답변(VQA) 등에서 뛰어난 성능을 보인다.

### 2. VLMs as Reward Models

기존 연구들은 CLIP의 임베딩 공간에서 목표 설명(Text)과 현재 상태(Image) 간의 코사인 유사도(Cosine Similarity)를 측정하여 보상을 생성하거나, VLM을 직접 파인튜닝하여 성공 여부를 판단하는 success detector로 사용하였다. 그러나 이러한 방식은 매 단계 VLM 쿼리가 필요하여 비용이 많이 들며, 모델의 임베딩 공간에 의존하므로 VLM의 고차원적 추론 능력을 충분히 활용하지 못한다는 한계가 있다.

### 3. Code as Policies and Rewards

LLM을 이용해 로봇 제어 액션을 생성하는 'Code as Policies' 연구가 있었으나, 이는 정책(Policy) 생성에 집중한 것이다. 본 논문은 VLM을 활용해 **보상 함수를 코드로 생성**하는 접근 방식을 제안하며, 이는 시각적 관측치가 포함된 환경에서 VLM을 이용한 보상 코드 생성의 첫 시도라는 점에서 차별점을 가진다.

## 🛠️ Methodology

VLM-CaR 프레임워크는 크게 세 단계의 파이프라인으로 구성된다.

### 1. 프로그램 생성 (Generate Programs)

VLM(GPT-4)에게 순차적인 프롬프팅(Sequential Prompting)을 수행하여 보상 코드를 도출한다.

- **에이전트 식별**: 초기 프레임에서 에이전트의 위치를 반환하는 스크립트를 생성한다.
- **작업 추론**: 목표 상태 이미지를 제공하고 최종 목표가 무엇인지 추론하게 한다.
- **작업 분해**: 최종 목표를 달성하기 위해 필요한 $n$개의 순차적인 하위 작업(Sub-tasks)으로 분해한다.
- **객체 식별 및 완료 체크**: 각 하위 작업에 필요한 객체를 식별하는 코드와, 해당 작업이 완료되었는지 판단하여 `True/False`를 반환하는 검사 스크립트를 생성한다.

### 2. 프로그램 검증 (Verify Programs)

생성된 코드의 신뢰성을 확보하기 위해 자동화된 검증 파이프라인을 거친다.

- **데이터셋**: 2개의 전문가 궤적(Expert Trajectories)과 100개의 무작위 궤적(Random Trajectories)을 사용한다.
- **검증 기준**:
  - 전문가 궤적에서는 모든 하위 작업이 반드시 완료(`True`)되어야 한다.
  - 무작위 궤적에서는 하위 작업 완료율이 특정 임계값 $p\%$ (실험에서는 0.1) 미만이어야 한다.
- **피드백 루프**: 검증을 통과하지 못한 경우, VLM에게 실패 사례를 알리고 코드를 수정하도록 다시 프롬프팅한다.

### 3. RL 학습 루프 (RL Training)

검증된 코드들은 RL 에이전트의 **보조 보상 함수(Auxiliary Reward Functions)**로 활용된다.

- 하위 작업이 완료될 때마다 보조 보상 $r_{aux}$를 제공하여 희소 보상 문제를 해결한다.
- 모든 하위 작업이 완료되면 최종 목표 달성 스크립트를 통해 최종 보상을 부여한다.
- 이는 RL의 Option Framework 관점에서 VLM이 옵션의 종료 조건(Termination Condition)과 보상 함수를 설계한 것으로 해석할 수 있다.

또한, Pandas-Gym과 같은 연속 제어 환경에서는 다음과 같은 거리 기반의 조밀한 보상(Dense Reward) 공식을 VLM이 제안하도록 유도하였다.
$$r = \frac{d_{initial} - d_{current}}{d_{initial}}$$
여기서 $d$는 목표물과 객체 사이의 유클리드 거리(Euclidean Distance)를 의미한다.

## 📊 Results

### 1. 실험 설정

- **환경**: MiniGrid (이산 액션 그리드), Pandas-Gym (연속 로봇 제어), CLIPort (복잡한 로봇 조작).
- **알고리즘**: MiniGrid에서는 PPO(Proximal Policy Optimization)를, Pandas-Gym에서는 TQC(Truncated Quantile Critics)를 사용하였다.
- **지표**: 에피소드당 평균 보상, 작업 성공률(Success Rate), 샘플 효율성.

### 2. 주요 결과

- **MiniGrid**: VLM-CaR이 생성한 조밀한 보상이 학습 속도를 현저히 향상시켰다. 특히 전통적인 PPO로는 해결하지 못한 `DoorKey8x8` 환경을 VLM-CaR 기반 보상을 통해 성공적으로 해결하였다.
- **Pandas-Gym**: `PandasPush` 및 `PandasSlide` 작업에서 환경 기본 보상(Sparse)만 사용했을 때보다 성공률이 높았으며, 동일 성공률에 도달하기 위한 샘플 효율성이 크게 개선되었다.
- **CLIPort**: 사전 학습된 서로 다른 수준의 정책(Random, Novice, Sub-optimal, Expert)에 VLM-CaR 보상 함수를 적용한 결과, 보상 값이 정책의 숙련도에 따라 뚜렷하게 구분됨을 확인하여 보상 함수의 유효성을 입증하였다.

## 🧠 Insights & Discussion

### 강점

- **효율성**: VLM을 학습 중에 직접 쿼리하지 않고 코드 형태로 변환하여 실행하므로 계산 비용이 극도로 낮다.
- **해석 가능성**: 보상 함수가 Python 코드로 작성되어 있어, 에이전트가 왜 보상을 받았는지 사람이 직접 검증하고 수정할 수 있다.
- **희소 보상 해결**: 복잡한 작업을 하위 작업으로 분해하여 조밀한 보상을 제공함으로써 RL의 고질적인 문제인 Sparse Reward 문제를 효과적으로 완화하였다.

### 한계 및 비판적 해석

- **시각적 인식의 단순함**: VLM이 제안하는 코드들이 주로 색상(Color)과 모양(Shape) 기반의 단순한 OpenCV 필터링에 의존하고 있다. 색상이 비슷하거나 조명이 변하는 환경에서는 취약할 가능성이 크다.
- **수동 조립 단계**: VLM이 생성한 개별 코드 블록들을 하나의 클래스로 합치는 과정에 여전히 사람의 개입(Manual Assembly)이 필요하다. 이는 완전한 자동화 파이프라인으로 가기 위한 해결 과제이다.
- **해상도 및 가림 문제**: 객체가 겹치거나 해상도가 낮을 경우 shape/color detection이 실패하는 문제가 있으며, 이를 위해 Convex Hull 등을 이용한 근사 기법을 사용하였으나 완벽하지 않다.

## 📌 TL;DR

본 논문은 VLM을 이용하여 RL의 보상 함수를 **실행 가능한 Python 코드**로 생성하는 **VLM-CaR** 프레임워크를 제안한다. VLM이 작업을 하위 작업으로 분해하고 이를 검증하는 코드를 짜게 함으로써, VLM 직접 쿼리의 높은 비용 문제를 해결하고, 사람이 읽을 수 있는 해석 가능한 보상 함수를 구축하였다. 실험을 통해 sparse reward 환경에서도 효율적인 학습이 가능함을 증명하였으며, 이는 향후 VLM을 이용한 자동화된 RL 환경 설계 및 로봇 제어 연구에 중요한 기초가 될 것으로 보인다.
