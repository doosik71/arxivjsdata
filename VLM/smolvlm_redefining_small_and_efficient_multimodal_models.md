# SmolVLM: Redefining small and efficient multimodal models

Andrés Marafioti, Orr Zohar, Miquel Farré, et al. (2025)

## 🧩 Problem to Solve

최근 Vision-Language Models(VLMs)는 비약적인 성능 향상을 이루었으나, 대다수의 모델이 거대한 파라미터 수와 막대한 계산 자원을 요구한다. 이는 모바일 기기나 엣지 디바이스(edge devices)와 같은 자원 제한적 환경에서의 배포를 어렵게 만든다.

기존의 소형 VLM들은 단순히 대형 모델의 설계 방식을 그대로 축소하여 적용하는 경향이 있었다. 특히 과도한 이미지 토큰화(image tokenization) 전략은 GPU 메모리 사용량을 비효율적으로 증가시키며, 이는 온디바이스(on-device) 애플리케이션의 실용성을 제한하는 핵심 원인이 된다. 따라서 본 논문의 목표는 연산 오버헤드를 최소화하면서도 이미지 및 비디오 작업에서 높은 성능을 유지하는, 자원 효율적인 소형 멀티모달 모델 시리즈인 SmolVLM을 설계하고 최적화하는 것이다.

## ✨ Key Contributions

본 연구의 핵심 직관은 대형 모델의 설계를 단순히 복제하는 것이 아니라, 소형 모델의 특성에 맞는 아키텍처 최적화, 공격적인 토큰 압축, 그리고 정교하게 큐레이션된 데이터셋을 통해 효율성을 극대화하는 것이다. 주요 기여 사항은 다음과 같다.

- **효율적인 소형 모델 제품군 제안**: 256M, 500M, 2.2B 파라미터 규모의 모델을 통해 자원 요구량을 획기적으로 줄이면서도 강력한 성능을 입증하였다.
- **극단적인 메모리 효율성**: 가장 작은 모델인 SmolVLM-256M은 추론 시 GPU 메모리를 1GB 미만으로 사용하며, 이는 온디바이스 배포의 장벽을 크게 낮춘다.
- **체계적인 아키텍처 탐색**: 인코더와 언어 모델(LM) 간의 파라미터 밸런스, 토큰화 전략, positional encoding, 데이터 구성 등이 소형 VLM의 성능에 미치는 영향을 심층 분석하였다.
- **엣지 디바이스에서의 비디오 이해 능력**: 소형 모델임에도 불구하고 Video-MME와 같은 벤치마크에서 경쟁력 있는 성능을 보여, 실시간 온디바이스 비디오 분석 가능성을 제시하였다.
- **완전한 오픈소스화**: 모델 가중치, 데이터셋, 코드 및 스마트폰 추론 시연 앱을 공개하여 재현성을 높였다.

## 📎 Related Works

기존의 VLM 연구는 크게 두 가지 흐름으로 나뉜다. 첫 번째는 Flamingo나 Idefics와 같이 파라미터 규모를 확장하여 성능을 높이는 방식이다. 이러한 모델들은 강력하지만 배포 비용이 매우 높다는 한계가 있다. 두 번째는 BLIP-2나 LLaVA처럼 효율적인 어댑터나 인스트럭션 튜닝을 통해 효율성을 추구하는 방식이다.

최근에는 PaliGemma, Moondream2, MiniCPM-V와 같은 효율성 중심의 모델들이 등장하였다. 그러나 본 논문은 이러한 기존 소형 모델들이 여전히 대형 모델의 설계 관습(예: 과도한 토큰 생성)을 따르고 있어, 실제 하드웨어 메모리 점유율 측면에서는 여전히 최적화 여지가 많다고 지적한다. SmolVLM은 단순한 파라미터 축소를 넘어, Pixel Shuffle과 같은 공격적인 토큰 압축과 소형 모델 전용 데이터 믹싱 전략을 통해 기존 접근 방식과 차별화한다.

## 🛠️ Methodology

### 1. 전체 시스템 아키텍처

SmolVLM은 시각적 특징을 추출하는 Vision Encoder와 이를 처리하는 SmolLM2 백본으로 구성된다. 전체 파이프라인은 다음과 같다:
**이미지 입력 $\rightarrow$ 이미지 분할(Image Splitting) $\rightarrow$ Vision Encoder $\rightarrow$ Pixel Shuffle $\rightarrow$ MLP Projection $\rightarrow$ SmolLM2 $\rightarrow$ 텍스트 출력**

### 2. 주요 구성 요소 및 설계 최적화

- **인코더-LM 밸런스**: 실험 결과, 소형 VLM에서는 거대한 인코더보다 작고 효율적인 인코더(SigLIP-B/16)를 사용하는 것이 LM과의 균형 측면에서 더 유리함을 발견하였다.
- **컨텍스트 길이 확장**: 고해상도 이미지를 처리하기 위해 RoPE(Rotary Positional Embedding)의 base를 $10k$에서 $273k$로 확장하여 컨텍스트 용량을 늘렸다. 이를 통해 2.2B 모델은 최대 $16k$ 토큰까지 안정적으로 처리할 수 있다.
- **Pixel Shuffle (Token Compression)**: 시각적 토큰 수를 줄이기 위해 space-to-depth 방식의 Pixel Shuffle을 적용하였다. 셔플 비율을 $r$이라고 할 때, 토큰 수는 $r^2$ 배로 감소한다. 본 연구에서는 소형 모델의 어텐션 오버헤드를 줄이기 위해 $r=4$라는 공격적인 압축 비율을 채택하였다.
- **이미지 및 비디오 처리**: 고해상도 이미지는 여러 개의 서브 이미지로 분할하여 처리하는 Image Splitting 전략을 사용한다. 반면, 비디오의 경우 프레임 평균화(frame averaging)가 성능을 저하시키는 것을 확인하여 이를 배제하고, 프레임을 이미지 인코더 해상도로 리스케일링하여 입력한다.

### 3. Smol Instruction Tuning

- **Learned Positional Tokens**: 단순 텍스트 기반 위치 토큰(예: `<row_1_col_2>`) 사용 시 발생하는 학습 불안정성(OCR loss plague)을 해결하기 위해, 학습 가능한(learned) 위치 토큰을 도입하여 OCR 성능과 수렴 속도를 개선하였다.
- **구조화된 프롬프트**: 시스템 프롬프트(System Prompts)와 미디어 시작/종료 토큰(Intro/Outro Tokens)을 도입하여 모델이 시각적 콘텐츠와 텍스트 지시사항을 명확히 구분하게 하였다.
- **SFT 최적화**: 지도 학습(SFT) 단계에서 사용자 쿼리를 마스킹(masking)하여 모델이 단순 반복이 아닌 실제 콘텐츠에 기반해 답변하도록 유도함으로써 일반화 성능을 높였다.
- **데이터 큐레이션**:
  - **텍스트 비율**: 텍스트 데이터 비중을 $14\%$로 엄격히 제한하여 시각적 성능 저하를 막았다.
  - **CoT 데이터**: Chain-of-Thought(CoT) 데이터의 과도한 사용은 소형 모델의 용량을 초과하여 오히려 성능을 떨어뜨리므로, 매우 적은 비율($0.02\% \sim 0.05\%$)만 유지하였다.
  - **비디오 길이**: 학습 시 비디오 길이를 약 $3.5$분까지 늘리는 것이 효율성과 성능 사이의 최적 지점임을 확인하였다.

## 📊 Results

### 1. 실험 설정

- **모델 변체**:
  - **SmolVLM-256M**: SigLIP-B/16(93M) + SmolLM2-135M
  - **SmolVLM-500M**: SigLIP-B/16(93M) + SmolLM2-360M
  - **SmolVLM-2.2B**: SigLIP-SO400M(400M) + SmolLM2-1.7B
- **측정 지표**: RAM 사용량(VRAM)을 핵심 지표로 설정하고, OCRBench, MMMU, MathVista, Video-MME 등 다양한 벤치마크를 사용하였다.

### 2. 정량적 결과

- **메모리 효율성**: SmolVLM-256M은 단일 이미지 추론 시 $\mathbf{0.8GB}$의 VRAM만 사용한다. 이는 MolmoE-A1B-7B($27.7GB$)나 Qwen2VL-2B($13.7GB$)에 비해 압도적으로 낮다.
- **성능 분석**:
  - **Small yet Mighty**: 256M 모델이 18개월 전 출시된 80B 규모의 Idefics 모델보다 대부분의 벤치마크에서 우수한 성능을 보였다.
  - **비교 우위**: 2.2B 모델은 MathVista($51.5$)와 ScienceQA($90.0$)에서 매우 강력한 성능을 보였으며, 특히 비디오 이해(Video-MME $52.1$)에서 Qwen2VL-7B와 같은 대형 모델과 경쟁 가능한 수준에 도달하였다.
- **온디바이스 성능**: WebGPU를 통해 브라우저 환경에서 실행 가능하며, M4 Max MacBook Pro에서 256M 모델은 초당 최대 $80$개 토큰을 생성하는 높은 처리량(throughput)을 기록하였다.

## 🧠 Insights & Discussion

본 논문은 소형 VLM 설계 시 단순히 파라미터를 줄이는 것이 아니라, **추론 시의 메모리 풋프린트(Memory Footprint)를 최소화하는 설계**가 핵심임을 시사한다. 특히 다음과 같은 통찰을 제공한다.

첫째, 소형 모델에서는 언어 모델의 용량이 제한적이므로, 너무 강력한 비전 인코더를 사용하는 것보다 인코더와 LM 간의 적절한 파라미터 균형을 맞추는 것이 더 효율적이다.
둘째, 소형 모델은 어텐션 오버헤드에 더 민감하므로, 더 공격적인 토큰 압축(Pixel Shuffle $r=4$)이 오히려 장기 컨텍스트 모델링에 도움이 된다.
셋째, 소형 모델의 용량 한계로 인해 대형 모델에 유효한 전략(예: 방대한 CoT 데이터 학습)이 오히려 독이 될 수 있으며, 매우 정밀하게 큐레이션된 소량의 고품질 데이터가 더 중요하다.

다만, MMMU나 AI2D와 같이 고도의 언어적 추론 능력이 필요한 작업에서는 여전히 대형 언어 백본을 가진 모델에 비해 성능 격차가 존재한다는 점이 한계로 남는다.

## 📌 TL;DR

SmolVLM은 엣지 디바이스 배포를 위해 최적화된 소형 멀티모달 모델 시리즈로, **공격적인 토큰 압축(Pixel Shuffle), 확장된 컨텍스트(RoPE), 그리고 소형 모델 특화 데이터 전략**을 통해 극단적인 메모리 효율성을 달성하였다. 가장 작은 256M 모델은 1GB 미만의 VRAM으로 동작하면서도 과거의 80B 모델을 능가하는 성능을 보이며, 이는 향후 저전력·실시간 온디바이스 멀티모달 AI 구현에 중요한 이정표가 될 것으로 보인다.
