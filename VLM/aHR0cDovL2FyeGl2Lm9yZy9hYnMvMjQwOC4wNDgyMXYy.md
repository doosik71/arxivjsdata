# VLM-MPC: Model Predictive Controller Augmented Vision Language Model for Autonomous Driving

Keke Long, Haotian Shi, Jiaxi Liu, Chaowei Xiao, Xiaopeng Li (2024)

## 🧩 Problem to Solve

자율주행 시스템의 해석 가능성(comprehensibility)과 적응성은 매우 중요한 요소이다. 기존의 모델 기반(model-based) 또는 학습 기반(learning-based) 시스템은 복잡한 규칙이나 보상 함수 설계에 의존하여 다양한 교통 시나리오에 대한 적응력이 떨어지며, 특히 학습 기반 시스템은 데이터셋에 없는 분포 외(out-of-distribution, OOD) 데이터에 직면했을 때 신뢰할 수 없는 의사결정을 내리는 문제가 있다.

최근 Vision Language Models (VLMs)와 같은 파운데이션 모델(Foundation Models, FMs)이 복잡한 환경에서 뛰어난 추론 능력을 보여주며 대안으로 떠오르고 있다. 그러나 VLM을 직접 제어에 활용하려는 시도에는 두 가지 핵심적인 한계가 존재한다. 첫째, 'FM to action' 방식의 end-to-end 제어는 차량의 운동학(kinematics) 및 역학(dynamics)을 충분히 고려하지 못해 실제 적용 가능성이 낮다. 둘째, 자율주행, 특히 충돌 위험 상황에서는 약 $10\text{Hz}$ 이상의 빠른 응답 속도가 필요하지만, 현재의 VLM은 추론 속도가 너무 느려 실시간 제어가 불가능하다. 또한, VLM의 고질적인 문제인 환각(hallucination) 현상은 주행 환경을 잘못 이해하게 하여 일관되지 않거나 위험한 응답을 생성할 위험이 있다.

따라서 본 논문의 목표는 VLM의 고수준 추론 능력과 Model Predictive Control (MPC)의 정밀한 저수준 제어 능력을 결합하여, 실시간 응답성, 차량 역학의 준수, 그리고 안전성을 모두 확보한 폐루프(closed-loop) 자율주행 컨트롤러인 VLM-MPC를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 VLM과 MPC를 **비동기적(asynchronous) 2층 구조**로 설계하여 각 계층의 장점을 극대화하고 단점을 상쇄하는 것이다.

1. **비동기적 계층 구조 설계**: 고수준의 VLM은 낮은 빈도로 주행 파라미터를 생성하고, 저수준의 MPC는 높은 빈도로 실시간 차량 제어를 수행한다. 이를 통해 VLM의 느린 응답 속도 문제를 해결하고, 검증된 MPC 알고리즘을 통해 차량 역학을 보장한다.
2. **신뢰성 향상을 위한 보조 컴포넌트**: VLM의 환각 현상을 줄이고 응답 안정성을 높이기 위해, 과거 주행 데이터의 통계적 정보를 담은 `Reference Memory`와 이미지에서 환경 특성을 추출하는 `Environment Encoder`를 도입하였다.
3. **추론 과정의 체계화**: Chain-of-Thought (CoT) 프롬프팅을 통해 VLM이 환경 이해 $\rightarrow$ 파라미터 평가 $\rightarrow$ 결정의 단계를 거치도록 유도하여 인간과 유사한 논리적 추론을 수행하게 한다.

## 📎 Related Works

논문에서는 FM 기반 자율주행 접근 방식을 크게 두 가지로 분류한다.

1. **FM to action**: VLM/LLM이 직접 웨이포인트, 가속/감속, 조향 명령을 생성하는 방식이다. 하지만 차량의 물리적 제약 조건을 무시하는 경향이 있으며, 실시간 제어에 필요한 높은 응답 속도를 달성하기 어렵다는 한계가 있다.
2. **FM to commands/codes**: FM이 상위 의사결정자로서 하위 제어 시스템에 지침이나 파라미터를 전달하는 방식이다. 이는 상위 계층(전략적 계획)과 하위 계층(실행 제어)을 분리하여 응답 속도 문제를 완화하고 산업계에서 검증된 제어 기법(예: MPC)을 활용할 수 있게 한다.

본 연구는 이러한 'FM to commands' 방식을 채택하되, 특히 MPC가 VLM의 의사결정 성능을 실제로 얼마나 향상시키는지 정량적으로 분석하고, 환경 인코더와 메모리를 통해 VLM의 신뢰성을 높였다는 점에서 기존 연구와 차별화된다.

## 🛠️ Methodology

### 1. 전체 시스템 구조

VLM-MPC는 상위 계층의 VLM Planner와 하위 계층의 MPC Controller로 구성된다. 두 계층은 서로 다른 주기로 동작하며, VLM이 업데이트된 파라미터를 제공하기 전까지 MPC는 이전의 파라미터를 유지하며 고속으로 제어를 수행하는 비동기 구조를 가진다.

### 2. 상위 계층: VLM

VLM은 다음 네 가지 구성 요소를 통해 최종 주행 파라미터 $\theta^t = [P, Q, R, Q_h, v_d, h_d]$를 생성한다.

* **Environment Encoder**: CLIP (ViT-B/32) 모델을 사용하여 전방 카메라 이미지에서 날씨, 조명, 도로 유형, 도로 상태, 정적 장애물 정보를 추출한다. 이미지 특징과 미리 정의된 텍스트 설명 간의 코사인 유사도를 계산하여 가장 적절한 환경 묘사 $E^t$를 생성한다.
* **Scene Encoder**: 자차(ego vehicle)의 과거 상태($s_{ego}^t$), 전방 차량의 상태($s_{pre}^t$), 정지선 위치($s_{SL}^t$) 등 수치적 상태 정보를 추출하여 텍스트 형태로 변환한다.
* **Reference Memory**: nuScenes 데이터셋에서 추출한 다양한 시나리오별 평균 MPC 파라미터 집합이다. 현재 환경 $E^t$에 맞는 평균 파라미터 $M(E^t)$를 제공하여 VLM이 완전히 무에서 시작하지 않고 기준점을 가질 수 있도록 돕는다.
* **Prompt Generator**: 위의 정보들을 수집하여 VLM에 입력할 프롬프트를 구성한다. 특히 CoT(Chain-of-Thought)를 적용하여 "환경 이해 $\rightarrow$ 예측 호라이즌 평가 $\rightarrow$ 비용 가중치 설정 $\rightarrow$ 목표 속도 정의 $\rightarrow$ 목표 차간 거리 결정" 순으로 사고하도록 유도한다.

### 3. 하위 계층: MPC

MPC는 VLM이 제공한 파라미터를 바탕으로 차량의 비선형 종방향 역학 모델을 고려하여 최적의 가속도를 계산한다.

**차량 역학 모델**은 다음과 같이 정의된다:
$$\begin{cases} \frac{dx}{dt} = v \\ \frac{dv}{dt} = a \\ \frac{da}{dt} = f(v, a) + g(v)\eta \end{cases}$$
여기서 $\eta$는 엔진 입력이며, $f(v, a)$와 $g(v)$는 공기 저항, 기계적 저항 및 엔진 지연 시간($\tau_a$)을 포함하는 함수이다. 본 연구는 피드백 선형화(feedback linearization)를 통해 가속도 미분 방정식을 $\dot{a} = \frac{u-a}{\tau_a}$ 형태로 단순화하여 제어한다.

**최적화 목표(Cost Function)**:
전방 차량이나 정지선이 있을 경우, 다음의 비용 함수 $J$를 최소화하는 가속도 $u$의 시퀀스를 찾는다:
$$\min_{u} J = \sum_{k=0}^{P-1} \|\Delta v_{t+k}\|^2 Q + \sum_{k=0}^{P-1} \|u_{t+k}\|^2 R + \sum_{k=0}^{P-1} \|\Delta x_{t+k}\|^2 Q_h + \epsilon^T \rho \epsilon$$

* $\Delta v$: 목표 속도와 현재 속도의 차이
* $u$: 제어 입력(가속도)
* $\Delta x$: 전방 차량과의 원하는 안전 거리($d^t = h_d v^t + d_0$)와 실제 거리의 차이
* $Q, R, Q_h$: VLM이 결정한 가중치 파라미터

전방 차량이 없을 때는 $\Delta x$ 관련 항을 제외하고 속도 유지와 제어 노력 최소화에 집중한다.

## 📊 Results

### 1. 실험 설정

* **데이터셋**: nuScenes (보스턴, 미국, 싱가포르 지역의 303개 시나리오)
* **사용 모델**: Llava 1.6 (기본), GPT-4o, GPT-4mini (민감도 분석용)
* **평가 지표**:
  * **Safety**: Post Encroachment Time (PET). 값이 클수록 안전함.
  * **Smoothness**: 가속도의 제곱평균제곱근 ($\text{RMS}_a$). 값이 작을수록 주행이 부드러움.
  * **Stability**: Completion Rate. VLM이 유효한 형식을 생성하여 작업을 완수한 비율.

### 2. 주요 결과

* **안전성 (Safety)**: VLM-MPC는 모든 시나리오에서 PET 값을 안전 임계값인 $1\text{s}$ 이상으로 유지하였다. 반면, Baseline LLM to Action 방식은 PET가 $1\text{s}$ 미만으로 떨어지는 충돌 위험 상황이 빈번하게 발생하였다.
* **부드러움 (Smoothness)**: VLM-MPC는 $\text{RMS}_a$ 값이 가장 낮게 나타나, 실제 주행 궤적(Real-world)과 유사하거나 더 부드러운 주행을 보였다. 반면 LLM to Action은 차량 역학을 무시한 궤적을 생성하여 매우 높은 $\text{RMS}_a$ 값을 기록하였다.
* **완성도 (Completion Rate)**: GPT 시리즈는 $100\%$의 완벽한 성공률을 보였으며, Llava 1.6은 $99.7\%$를 기록하였다. 이는 모델의 파라미터 크기가 클수록 환각 현상이 적고 형식을 잘 준수함을 시사한다.
* **응답 속도**: Llava 모델은 평균 $3.42\text{s}$로 시스템 요구사항($0.2\text{Hz}$)을 충족하였으나, GPT-4 시리즈는 API 지연 및 모델 크기로 인해 약 $10\text{s}$가 소요되었다.

### 3. 절제 연구 (Ablation Study)

* **Reference Memory 제거 시**: 안전성은 유지되었으나, 전반적으로 속도가 너무 높게 설정되는 경향이 있었으며, 특히 비나 밤 환경에서 주행 부드러움($\text{RMS}_a$)이 크게 저하되었다.
* **Environment Encoder 제거 시**: 성공률(Completion Rate)이 $99.7\%$에서 $87.5\%$로 급격히 하락하였다. 이는 도로 정보 없이는 VLM이 현재 상황을 이해하지 못해 잘못된 형식의 답변을 내놓거나 응답 시간이 비정상적으로 길어지기 때문이다.

## 🧠 Insights & Discussion

### 강점 및 해석

본 연구는 VLM의 '추론 능력'과 MPC의 '물리적 제약 준수 능력'을 비동기적으로 결합함으로써, LLM 기반 자율주행의 가장 큰 약점인 **실시간성 부족**과 **물리적 비현실성**을 동시에 해결하였다. 특히, 단순한 VLM 출력이 아니라 `Reference Memory`와 `Environment Encoder`를 통해 컨텍스트를 보강함으로써, VLM이 겪는 환각 현상을 효과적으로 억제하고 응답의 안정성을 높였다는 점이 돋보인다.

### 한계 및 논의사항

1. **응답 속도의 한계**: Llava 모델이 요구사항을 충족하긴 했으나, 여전히 수 초의 시간이 걸린다. 실제 도로의 초고속 상황에서는 이 정도의 지연 시간도 위험할 수 있으므로, 더 작은 모델의 파인튜닝이나 최적화가 필수적이다.
2. **데이터셋의 제약**: nuScenes와 같은 공개 데이터셋은 동일 장면의 다양한 기상 조건 데이터를 제공하지 않는다. 따라서 저자들은 향후 직접 데이터를 수집하여 OOD 시나리오에 대한 강건성을 테스트하겠다고 언급하였다.
3. **인식 오류의 부재**: 본 연구는 인식 단계(perception phase)에서의 오류가 없다고 가정하고 정답 데이터를 사용하였다. 실제 환경에서는 인식 모델의 오차가 VLM의 판단에 어떤 영향을 줄지에 대한 분석이 추가되어야 한다.

## 📌 TL;DR

VLM-MPC는 고수준의 추론을 담당하는 **VLM**과 저수준의 실시간 제어를 담당하는 **MPC**를 비동기적으로 결합한 자율주행 컨트롤러이다. VLM은 환경 인코더와 참조 메모리를 통해 최적의 주행 파라미터를 결정하고, MPC는 이를 바탕으로 차량 역학을 고려한 안전한 궤적을 생성한다. 실험 결과, 기존 end-to-end LLM 방식보다 훨씬 안전하고 부드러운 주행이 가능함을 확인하였으며, 이는 모델 기반 제어(MPC)가 LLM의 의사결정 능력을 실제 물리 세계에 안전하게 투영하는 가교 역할을 할 수 있음을 시사한다.
