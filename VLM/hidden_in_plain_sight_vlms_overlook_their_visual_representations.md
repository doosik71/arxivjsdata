# Hidden in plain sight: VLMs overlook their visual representations

Stephanie Fu, Tyler Bonnen, Devin Guillory, Trevor Darrell (2025)

## 🧩 Problem to Solve

본 논문은 시각 언어 모델(Vision Language Models, VLMs)이 시각적 정보를 통합하고 활용하는 능력에 존재하는 심각한 결함을 분석한다. 일반적으로 VLMs는 강력한 시각 인코더(Vision Encoder)와 대규모 언어 모델(LLM)을 결합하여 멀티모달 태스크를 수행하도록 설계되었다. 그러나 연구자들은 VLMs가 도메인 지식이나 언어적 사전 지식 없이 오직 시각적 입력에만 의존해야 하는 '시각 중심(vision-centric)' 태스크에서 매우 낮은 성능을 보인다는 점에 주목하였다.

특히, VLM에 내장된 시각 인코더 자체는 해당 태스크를 충분히 해결할 수 있는 고품질의 표현(representation)을 생성함에도 불구하고, 최종 VLM의 출력은 무작위 추측(random chance) 수준으로 떨어지는 현상이 발견되었다. 따라서 본 연구의 목표는 시각 인코더에서 LLM 출력에 이르는 파이프라인 중 어디에서 정보의 손실이나 활용 실패가 발생하는지 진단하고, VLM이 왜 자신의 시각적 표현을 제대로 활용하지 못하는지 분석하는 것이다.

## ✨ Key Contributions

본 논문의 핵심적인 통찰은 VLMs의 성능 저하가 시각 인코더의 표현력 부족이 아니라, **LLM이 이미 존재하는 시각적 정보를 효과적으로 사용하지 못하고 언어적 편향(language priors)에 의존하기 때문에 발생한다**는 점이다.

주요 기여 사항은 다음과 같다:

- **시각적 표현의 보존 확인**: 시각 인코더에서 생성된 정보가 프로젝터(projector)와 LLM의 중간 레이어를 거치는 동안 파괴되지 않고 유지됨을 입증하였다.
- **언어 모델의 병목 현상 규명**: 프롬프트 튜닝이나 프로젝터 수정보다 LLM 자체를 튜닝했을 때 성능 향상이 가장 뚜렷하며, 이는 LLM이 시각적 정보에 주의(attention)를 기울이는 능력이 부족함을 시사한다.
- **Blind Baseline과의 상관관계**: VLM이 이미지를 보았을 때의 답변 분포가 이미지를 제거했을 때(blind)의 답변 분포와 매우 유사함을 보여, 모델이 시각 정보를 무시하고 LLM 내부의 편향에 따라 답을 선택함을 증명하였다.

## 📎 Related Works

연구팀은 VLMs의 시각적 능력을 평가하기 위해 기존의 여러 벤치마크인 CV-Bench, BLINK, MOCHI 등을 활용하였다. 기존 연구들은 VLM의 성능을 높이기 위해 더 강력한 시각 인코더를 사용하거나 여러 인코더를 앙상블하는 전략을 제안해 왔다. 또한, 일부 연구는 시각-언어 사전 학습(CLIP, SigLIP 등)을 거친 인코더가 자기지도 학습(DINOv2 등) 인코더보다 VLM 구성 시 더 유리하다고 주장하였다.

그러나 본 논문은 이러한 결론이 위험할 수 있음을 경고한다. 시각 인코더 단독으로는 DINOv2가 가장 뛰어난 성능을 보임에도 불구하고, VLM으로 구성했을 때 성능이 급락하는 현상이 발생하기 때문이다. 즉, 기존의 VLM 벤치마크 결과가 인코더의 실제 능력을 반영하는 것이 아니라, LLM이 해당 인코더의 표현을 얼마나 잘 활용하느냐에 따라 결정되었을 가능성이 크다는 차별점을 제시한다.

## 🛠️ Methodology

### 1. 평가 체계 (Evaluation Setup)

연구팀은 '시각 중심(vision-centric)' 태스크를 정의하고, 이를 두 가지 방식으로 평가하여 비교하였다.

- **Visual Evaluation (시각적 평가)**: 시각 인코더의 출력값(representation)을 직접 읽어내어(direct readout) 정답을 도출하는 방식이다.
- **VLM Evaluation (VLM 평가)**: 이미지를 입력하고 텍스트 프롬프트를 통해 정답을 선택하게 하는 표준 VQA(Visual Question Answering) 방식이다.

### 2. 세부 태스크 및 구현

- **Depth Estimation (깊이 추정)**: 두 객체 중 무엇이 더 가까운지 판단한다. 시각적 평가는 DPT 헤드를 학습시켜 깊이 맵을 생성한 후 평균 깊이를 비교하며, VLM은 다지선다형 질문에 답한다.
- **Correspondence-based Evaluations (대응 기반 평가)**:
  - **Semantic Correspondence**: 서로 다른 객체 간의 동일 부위(예: 두 동물의 왼쪽 귀)를 찾는다.
  - **Object Affordance**: 기능적으로 유사한 부위(예: 두 서로 다른 도구의 손잡이)를 찾는다.
  - **Low-level Matching**: 조명이나 각도가 다른 동일 장면의 픽셀을 매칭한다.
  - *방법론*: 시각적 평가는 두 지점의 패치 특징(patch features) 간 $\text{cosine similarity}$를 계산하여 가장 높은 값을 선택한다.
- **3D Object Awareness**: 여러 이미지 중 다른 객체가 포함된 이미지를 찾는다. $\text{CLS}$ 토큰 간의 유사도 행렬을 이용해 평균 유사도가 가장 낮은 이미지를 선택한다.
- **Art Style**: 기준 이미지와 스타일이 유사한 이미지를 선택한다. ViT 패치 특징의 내적을 통해 Gram matrix $G$를 계산하고, $\text{MSE}(G_{\text{ref}}, G_{\text{test}})$가 가장 낮은 것을 선택한다.

### 3. 분석 방법론

- **Intermediate Layer Probing**: 프로젝터와 LLM의 각 레이어에서 특징을 추출하여 Visual Evaluation을 수행함으로써, 정보가 어디서 소실되는지 확인한다.
- **Prompt-tuning**: 학습 가능한 프리픽스 임베딩(prefix embeddings)을 추가하여 프롬프트 민감도를 측정한다.
- **Component-wise Finetuning**: LoRA를 사용하여 ViT, Projector, LLM 각각을 동일한 파라미터 수($16.7\text{M}$)로 튜닝하여 어떤 구성 요소가 성능 향상에 핵심적인지 분석한다.

## 📊 Results

### 1. 시각적 능력과 VLM 성능의 괴리

모든 태스크에서 **Visual Evaluation 성능 $\gg$ VLM Evaluation 성능** 결과가 나타났다. 특히 Low-level Matching의 경우, 시각 인코더 단독으로는 높은 정확도를 보였으나 VLM으로 변환 시 성능이 $45.5\%$ 포인트 급락하여 거의 무작위 추측 수준이 되었다.

### 2. 인코더 랭킹의 역전 현상

시각 인코더 단독 성능은 DINOv2가 대부분의 태스크에서 압도적 1위였다. 하지만 VLM으로 구성했을 때 DINOv2 기반 모델이 반드시 최고의 성능을 내지는 않았다. 이는 VLM 벤치마크 결과만으로 특정 인코더가 더 우월하다고 결론 내리는 것이 잘못되었음을 의미한다.

### 3. Blind Baseline과의 일치

이미지 없이 텍스트 프롬프트만 제공했을 때(blind)의 답변 분포가 이미지를 제공했을 때의 답변 분포와 매우 유사하게 나타났다. 이는 VLM이 시각적 입력을 무시하고 LLM 내부에 내재된 언어적 편향(예: 특정 선택지를 선호하는 경향)에 따라 응답하고 있음을 보여준다.

### 4. LLM 튜닝의 효과

ViT나 프로젝터를 튜닝하는 것보다 LLM을 튜닝했을 때 성능 향상이 가장 컸다. 또한, LLM 튜닝 후 Attention Map을 분석한 결과, 모델이 참조 지점(REF)과 선택지(A, B, C, D) 등 중요한 시각적 영역에 더 집중하게 됨을 확인하였다.

## 🧠 Insights & Discussion

본 연구는 VLMs가 '눈은 뜨고 있으나 정작 보지는 않는(Hidden in plain sight)' 상태임을 밝혀냈다. 시각 인코더는 이미 정답을 찾기에 충분한 정보를 생성하여 LLM에 전달하고 있지만, LLM은 이 정보를 처리하는 대신 자신의 언어적 편향에 의존해 답을 내놓는다.

**강점 및 시사점:**

- 단순히 더 강력한 시각 인코더를 추가하거나 앙상블하는 방식으로는 이 문제를 해결할 수 없음을 시사한다. 병목은 '표현의 생성'이 아니라 '표현의 활용'에 있기 때문이다.
- VLM 기반의 벤치마크가 시각 인코더의 실제 능력을 저평가하거나 왜곡할 수 있다는 점을 경고하여, 향후 VLM 평가 방법론에 중요한 관점을 제공한다.

**한계 및 논의사항:**

- 3D Object Awareness 태스크의 경우 LLM 튜닝의 효과가 상대적으로 적었는데, 이는 단순한 지역적 특징 추출을 넘어 더 추상적인 3D 구조 이해가 필요하기 때문으로 추정된다.
- 가벼운 LoRA 튜닝만으로 성능이 올라갔다는 점은, LLM이 시각 정보를 활용하는 '방법'만 배우면 충분하다는 것을 의미하지만, 이것이 모든 일반적인 시각 태스크로 확장될지는 추가 연구가 필요하다.

## 📌 TL;DR

본 논문은 VLMs가 시각 인코더가 제공하는 고품질의 시각적 표현을 제대로 활용하지 못하고, 대신 LLM의 언어적 편향에 의존하여 답을 내놓는다는 것을 증명하였다. 분석 결과, 시각 정보는 모델 내부 레이어까지 잘 전달되고 있으며, 성능 개선의 핵심 키는 시각 인코더의 강화가 아니라 **LLM이 시각적 표현을 효과적으로 사용하도록 만드는 능력**에 있다. 이 연구는 VLM 설계 시 인코더 교체보다 LLM-시각 통합 메커니즘 개선이 더 중요함을 시사한다.
