# SPHINX-X: Scaling Data and Parameters for a Family of Multi-modal Large Language Models

Dongyang Liu et al. (2024)

## 🧩 Problem to Solve

본 논문은 기존의 오픈소스 다중모달 거대 언어 모델(Multi-modal Large Language Models, MLLMs)이 가진 두 가지 주요 한계점을 해결하고자 한다.

첫째는 **작업 도메인 데이터의 제한적 범위**이다. BLIP-2, LLaVA와 같은 기존 모델들은 주로 자연 이미지 도메인의 데이터로 학습되어 일반적인 이미지 이해 능력은 뛰어나지만, 광학 문자 인식(OCR), 표, 차트, 수학적 추론과 같이 도메인 특화 지식이 필수적인 영역에서는 성능이 크게 저하되는 경향이 있다. 반면, 특정 도메인에 특화된 모델들은 일반적인 다중모달 능력이 부족하다는 트레이드-오프 문제가 존재한다.

둘째는 **모델 파라미터 선택지의 제한**이다. 대부분의 MLLM은 LLaMA와 같은 7B 또는 13B 규모의 Dense LLM을 기반으로 한다. 이는 모바일 기기와 같은 엣지 디바이스에 배포하기에는 너무 무겁고, 복잡한 실세계의 다중모달 문제를 해결하기에는 파라미터 규모가 여전히 부족하다는 한계가 있다.

따라서 본 연구의 목표는 데이터의 다양성과 규모를 확장하고, 파라미터 크기를 다양화한 MLLM 제품군인 SPHINX-X를 구축하여 범용성과 전문성을 동시에 확보하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **데이터와 모델 파라미터의 동시 스케일링(Scaling)**을 통해 MLLM의 성능 한계를 돌파하는 것이다. 이를 위해 다음과 같은 설계 전략을 제안한다.

1. **아키텍처 최적화 (Efficiency):** 기존 SPHINX의 복잡한 구조를 단순화하여 연산 효율을 높였다. 불필요한 비전 인코더를 제거하고, 고해상도 이미지 분할 시 발생하는 무의미한 패딩 영역을 처리하기 위한 `skip token`을 도입하였다.
2. **학습 파이프라인 단순화:** 기존의 다단계(multi-stage) 학습 방식을 단일 단계(one-stage)의 'all-in-one' 패러다임으로 전환하여 학습 효율을 극대화하였다.
3. **데이터셋의 광범위한 확장:** 언어, 비전, 비전-언어 작업을 아우르는 대규모 공공 데이터셋을 수집하였으며, 특히 OCR 집중 데이터셋과 Set-of-Mark(SoM) 데이터셋을 직접 구축하여 모델의 세밀한 이해 능력을 강화하였다.
4. **모델 제품군(Family) 구축:** TinyLlama-1.1B부터 Mixtral-8x7B(MoE)에 이르기까지 다양한 LLM 백본을 적용함으로써, 모바일 배포용 모델부터 고성능 추론 모델까지 아우르는 스펙트럼을 제시하였다.

## 📎 Related Works

본 논문은 Transformer 기반의 LLM 발전을 기반으로 하며, 특히 데이터 규모와 파라미터 증가가 모델 성능 향상의 핵심임을 전제로 한다.

기존의 MLLM 연구들은 주로 시각적 토큰을 LLM에 연결하는 방식(예: LLaVA의 linear layer)이나 쿼리 트랜스포머를 이용해 시각 특징을 요약하는 방식(예: BLIP 시리즈)을 사용해 왔다. 또한, Shikra나 mPLUG-DocOwl과 같이 특정 작업(객체 탐지, 문서 분석)에 특화된 모델들이 제안되었으나, 이들은 일반적인 다중모달 성능을 희생하는 경우가 많았다.

SPHINX-X는 이러한 개별적 접근 대신, 다양한 도메인의 데이터를 통합 학습시키고 모델의 규모를 체계적으로 확장함으로써 일반화 능력과 특화 능력을 동시에 잡으려는 차별점을 가진다.

## 🛠️ Methodology

### 1. 아키텍처 개선 사항

SPHINX-X는 기존 SPHINX 프레임워크를 기반으로 세 가지 주요 수정을 가하였다.

- **Mixture of Visual Experts (MoV):** 기존의 4개 비전 인코더 중 중복되는 요소를 제거하고, 서로 다른 학습 방식과 구조를 가진 **CLIP-ConvNeXt**(CNN 기반, 약지도 학습)와 **DINOv2**(ViT 기반, 자기지도 학습) 두 가지만 유지한다. 이를 통해 연산 비용을 줄이면서도 상보적인 시각적 세만틱을 확보한다.
- **Skip Tokens:** 고해상도 이미지를 여러 개의 서브 이미지로 나눌 때, 이미지의 종횡비로 인해 발생하는 완전히 패딩된(zero-filled) 서브 이미지들이 존재한다. 이를 그대로 처리하면 연산 낭비와 노이즈가 발생하므로, 학습 가능한 `skip token`으로 대체하여 시퀀스 길이를 줄이고 상대적 위치 정보만 유지한다.
- **One-Stage All-in-One Training:** 복잡한 다단계 학습 대신, 수집된 모든 데이터를 통합하여 한 번에 학습시킨다. 비전 인코더를 제외한 LLM과 프로젝션 레이어의 모든 파라미터를 동결 해제(unfreeze)하여 학습한다.

### 2. 데이터셋 구축

모델의 잠재력을 끌어올리기 위해 다음과 같은 데이터 전략을 사용한다.

- **OCR-intensive Data:** Common Crawl과 arXiv에서 수집한 대규모 PDF 데이터를 기반으로 약 300만 페이지의 `PaperText` 데이터셋을 구축하였다. 이를 통해 텍스트 탐지 및 문서 레이아웃 분석 능력을 강화하였다.
- **Set-of-Mark (SoM) Data:** 이미지 내 객체에 마크(점, 박스 등)를 표시하고, GPT-4V를 이용해 해당 마크에 대한 정교한 캡션과 관계 분석 데이터를 생성하였다. 실제 학습 시에는 마크가 없는 원본 이미지를 사용하되, 텍스트 프롬프트 내에서 마크의 위치와 의미를 설명함으로써 모델이 세밀한 시각적 접지(grounding) 능력을 갖게 한다.

### 3. 모델 제품군 구성

다양한 LLM 백본을 통해 네 가지 버전의 모델을 개발하였다.

- **SPHINX-Tiny:** TinyLlama-1.1B 기반. 모바일 및 엣지 디바이스 배포 목적.
- **SPHINX-Intern2:** InternLM2-7B 기반. 강력한 이국어(한/중/영 등) 지원 및 추론 목적.
- **SPHINX-Plus:** LLaMA2-13B 기반. 데이터 스케일링의 효과를 검증하기 위한 모델.
- **SPHINX-MoE:** Mixtral-8x7B 기반. MoE 구조를 통해 복잡한 추론 능력 극대화.

## 📊 Results

### 1. 실험 설정 및 지표

MME, Seedbench, MathVista, MM-bench 등 다수의 MLLM 벤치마크를 통해 평가하였다. 특히 OCR 성능 평가를 위해 TextVQA, DocVQA 등을 사용하였으며, 시각적 접지 능력을 위해 RefCOCO 계열의 REC 벤치마크를 사용하였다.

### 2. 주요 결과

- **전반적 성능:** SPHINX-Plus는 원본 SPHINX보다 뛰어난 성능을 보였으며, 이는 데이터의 규모와 다양성이 성능 향상에 직접적인 영향을 미침을 입증한다.
- **파라미터 스케일링 효과:** 1.1B에서 8x7B까지 파라미터 규모가 커질수록 다중모달 이해 능력이 일관되게 상승하는 상관관계를 확인하였다.
- **특수 능력:**
  - **SPHINX-MoE**는 오픈소스 모델 중 MathVerse와 SciVerse에서 최상위권 성능을 기록하며 수학 및 과학적 추론 능력을 입증하였다.
  - **OCR 및 Grounding:** 구축한 특화 데이터셋 덕분에 텍스트 스포팅(Text Spotting)과 객체 위치 탐지에서 매우 정밀한 결과를 보였다.
- **비디오 분석 (Zero-shot):** 비디오 데이터로 학습하지 않았음에도 불구하고, 비디오의 중간 프레임을 샘플링하여 입력하는 방식으로 Video-Bench에서 기존 비디오 특화 모델들을 상회하는 성능을 기록하였다.

## 🧠 Insights & Discussion

본 연구를 통해 MLLM의 성능을 결정짓는 핵심 요소는 **데이터의 다양성(Diversity)**과 **파라미터의 규모(Scale)**임을 확인하였다. 특히, 단순히 양을 늘리는 것이 아니라 OCR-intensive 데이터나 SoM 데이터와 같이 '고품질의 정밀한' 데이터를 추가하는 것이 모델의 전문성을 높이는 데 결정적이다.

**비판적 해석 및 한계:**

- **MoE 전문가 분석:** 부록의 분석에 따르면, MoE의 각 전문가(expert)가 특정 도메인이나 모달리티에 특화되어 작동한다는 뚜렷한 패턴은 발견되지 않았다. 이는 전문가들이 도메인별로 분리되어 학습되기보다 일반적인 특징을 공유하며 협력하고 있음을 시사한다.
- **다학제적 지식의 부족:** MMMU나 CMMMU와 같은 대학 수준의 다학제적 추론 작업에서는 여전히 한계를 보였다. 이는 학습 데이터에 전문 학술 지식이 충분히 포함되지 않았기 때문으로 분석되며, 향후 더 광범위한 전문 지식 데이터의 통합이 필요하다.
- **비디오 모델링:** 비디오 분석에서 좋은 성적을 거두었으나, 이는 단일 프레임의 강력한 인식 능력에 의존한 결과이다. 시간적 흐름(temporal relationship)을 모델링하는 능력은 여전히 부족하며, 이를 위해서는 실제 비디오 시퀀스 데이터 학습이 필수적이다.

## 📌 TL;DR

SPHINX-X는 아키텍처 단순화(MoV, Skip tokens)와 단일 단계 학습, 그리고 대규모 다중 도메인 데이터셋(특히 OCR 및 SoM 특화 데이터)을 통해 구축된 MLLM 제품군이다. 1.1B부터 MoE 구조까지 다양한 규모의 모델을 제공하며, 데이터와 파라미터의 스케일링이 MLLM의 범용성과 전문성을 동시에 향상시킨다는 것을 입증하였다. 이 연구는 특히 모바일 환경을 위한 경량 모델부터 고도의 추론을 위한 대형 모델까지의 가이드라인을 제시하며, 향후 다중모달 AI의 실용적 배포와 전문 지식 확장에 중요한 기여를 할 것으로 보인다.
