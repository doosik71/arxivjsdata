# Linking Points With Labels in 3D: A Review of Point Cloud Semantic Segmentation

Yuxing Xie, Jiaojiao Tian, and Xiao Xiang Zhu (2020)

## 🧩 Problem to Solve

본 논문은 3차원 포인트 클라우드 시맨틱 세그멘테이션(Point Cloud Semantic Segmentation, 이하 PCSS) 분야의 최근 발전 동향을 체계적으로 정리하고 분석하는 것을 목표로 한다. PCSS는 3차원 공간에 분포된 정규 또는 불규칙한 포인트들에 시맨틱 레이블(semantic label)을 부여하는 작업으로, 이는 2차원 이미지의 시맨틱 세그멘테이션을 3차원으로 확장한 개념이다.

이 연구가 해결하고자 하는 주요 문제는 다음과 같다.
첫째, 원격 탐사(Remote Sensing), 컴퓨터 비전(Computer Vision), 로보틱스 등 다양한 분야에서 PCSS가 광범위하게 활용되고 있으나, 기존의 리뷰 논문들은 최신 딥러닝 기법에 대한 상세한 정보가 부족하거나 업데이트가 되지 않았다는 점이다.
둘째, 포인트 클라우드 데이터의 획득 방식(LiDAR, RGB-D, SAR 등)에 따라 데이터의 특성이 매우 다르지만, 이에 대한 통합적인 분석과 알고리즘의 적용 가능성에 대한 논의가 부족하다는 점이다.

결과적으로 본 논문은 데이터 획득부터 벤치마크 데이터셋, 전통적인 세그멘테이션 기법(PCS) 및 최신 시맨틱 세그멘테이션 기법(PCSS)을 망라하여 연구자들에게 최신 기술 현황과 향후 연구 방향을 제시하고자 한다.

## ✨ Key Contributions

본 논문의 핵심 기여는 PCSS 분야의 방대한 문헌을 체계적으로 분류하고 분석한 종합적인 가이드라인을 제공했다는 점에 있다. 주요 기여 사항은 다음과 같다.

1. **포인트 클라우드 데이터 생태계의 정의**: 데이터 획득 방식(Image-derived, LiDAR, RGB-D, SAR)에 따른 특성을 분석하고, 이를 밀도(Sparse, Dense)와 소스(Multi-source) 관점에서 분류하여 각 데이터셋이 갖는 장단점과 적용 분야를 명확히 정의하였다.
2. **PCS와 PCSS의 개념적 구분**: 의미론적 정보 없이 기하학적/분광적 특성으로 포인트를 그룹화하는 포인트 클라우드 세그멘테이션(Point Cloud Segmentation, PCS)과 각 포인트에 클래스 레이블을 부여하는 PCSS의 차이점을 명확히 하여, PCS가 PCSS의 전처리 단계로 활용될 수 있음을 시사하였다.
3. **알고리즘 분류 체계(Taxonomy) 구축**: 전통적인 머신러닝 기반 방법론부터 최신 딥러닝 방법론까지를 계층적으로 분류하였다. 특히 딥러닝 기법을 데이터 처리 방식에 따라 Multiview-based, Voxel-based, Point-based의 세 가지 범주로 나누어 상세히 분석하였다.
4. **학문적 간극 분석**: 컴퓨터 비전 분야와 원격 탐사 분야가 포인트 클라우드를 다루는 관점의 차이(평가 지표, 데이터 소스, 노이즈 처리 등)를 분석하여 두 분야의 융합 필요성을 제기하였다.

## 📎 Related Works

논문에서는 기존의 PCS 및 PCSS 리뷰 논문들(예: [16], [17])을 언급하며, 이들이 전반적인 흐름은 제시하고 있으나 특히 PCSS의 세부적인 방법론과 최신 딥러닝의 급격한 발전을 충분히 반영하지 못하고 있음을 한계로 지적한다.

기존 접근 방식과의 차별점은 다음과 같다.

- **최신성**: 최근 2년간의 딥러닝 기반 비약적 발전과 최신 벤치마크 데이터셋(S3DIS, ScanNet, Paris-Lille-3D 등)을 포함하여 분석하였다.
- **포괄성**: 단순히 알고리즘만 나열하는 것이 아니라, 센서의 물리적 특성(SAR의 4차원 정보, LiDAR의 정밀도 등)과 알고리즘의 상관관계를 심도 있게 다루었다.

## 🛠️ Methodology

본 논문은 리뷰 논문으로서 특정 알고리즘을 제안하는 대신, 기존 연구들을 분석하기 위한 체계적인 분류 방법론을 사용한다.

### 1. Point Cloud Segmentation (PCS) 기법

시맨틱 정보 없이 기하학적 제약과 통계적 규칙을 사용하는 비지도 학습 기반의 기법들이다.

- **Edge-based**: 강도(intensity)의 급격한 변화를 감지하여 경계선을 찾고 포인트를 그룹화한다.
- **Region Growing**: 시드(seed) 포인트를 선정하고, 인접 포인트와의 유사성(법선 벡터, 거리 등)을 기준으로 영역을 확장한다.
- **Model Fitting**: 데이터를 기하학적 프리미티브(평면, 구, 원통 등)에 매칭시킨다.
  - **Hough Transform (HT)**: 파라미터 공간에 투표(voting)하여 최적의 모델을 찾는다. 3차원 평면의 경우 다음과 같은 방정식으로 표현된다.
    $$\rho = x\cos(\theta)\sin(\phi) + y\sin(\theta)\sin(\phi) + z\cos(\phi)$$
  - **RANSAC**: 무작위 샘플링을 통해 가설 모델을 생성하고 검증한다. 최적 모델 $\hat{M}$은 다음의 손실 함수를 최소화하는 방향으로 결정된다.
    $$\hat{M} = \arg \min_{M} \left\{ \sum_{d \in D} \text{Loss}(\text{Err}(d; M)) \right\}$$
- **Clustering-based**: K-means, Mean-shift, Fuzzy clustering 등을 통해 유사한 특성을 가진 포인트들을 그룹화한다.
- **Oversegmentation**: 계산 효율을 높이기 위해 Raw 데이터를 Supervoxel과 같은 작은 단위로 먼저 나누는 전처리 과정이다.

### 2. Point Cloud Semantic Segmentation (PCSS) 기법

지도 학습(Supervised Learning)을 통해 각 포인트에 의미적 레이블을 부여하는 기법들이다.

#### A. 전통적 머신러닝 (Regular Supervised ML)

- **개별 분류 (Individual PCSS)**: 각 포인트의 개별 특성만을 사용하여 SVM, Random Forest 등으로 분류한다.
- **문맥 모델 (Statistical Contextual Models)**: 포인트 간의 관계를 고려하는 CRF(Conditional Random Fields), MRF(Markov Random Fields) 등을 사용하여 결과의 평활화(smoothing)를 수행한다.

#### B. 딥러닝 (Deep Learning)

데이터의 형태에 따라 세 가지 파이프라인으로 구분된다.

- **Multiview-based**: 3D 데이터를 여러 각도의 2D 이미지로 투영하여 2D CNN으로 처리한 후 다시 3D로 복원한다. (예: SnapNet)
- **Voxel-based**: 3D 공간을 격자 형태의 Voxel로 양자화하여 3D CNN을 적용한다. (예: SegCloud)
- **Point-based**: 포인트 세트를 직접 입력으로 사용한다.
  - **PointNet**: 대칭 함수(Symmetric function)인 Max Pooling을 사용하여 포인트의 순서 불변성(Permutation Invariance) 문제를 해결한다.
    $$f(\{x_1, \dots, x_n\}) \approx g(h(x_1), \dots, h(x_n))$$
    여기서 $h$는 MLP를 통한 지역 특징 추출이며, $g$는 글로벌 특징을 집계하는 대칭 함수이다.
  - **PointNet++**: 계층적 구조를 도입하여 지역 기하학적 특징(Local geometric features)을 캡처한다.
  - **DGCNN**: EdgeConv를 통해 포인트 간의 동적 그래프를 생성하여 기하학적 관계를 학습한다.

#### C. 하이브리드 방법 (Hybrid Methods)

전통적인 PCS(전처리)와 PCSS(최종 분류)를 결합한 형태이다. 예를 들어, Supervoxel로 먼저 나누어 데이터 양을 줄인 후 딥러닝 모델에 입력하는 방식이다.

## 📊 Results

본 논문은 직접적인 실험을 수행하지 않고, 기존 벤치마크 데이터셋과 알고리즘의 성과를 분석하였다.

### 1. 주요 벤치마크 데이터셋 분석

- **Semantic3D.net**: 대규모 실외 TLS 데이터셋으로 8개 클래스로 구성된다.
- **S3DIS**: 대규모 실내 RGB-D 데이터셋으로 13개 클래스를 포함하며 딥러닝 모델 평가에 널리 쓰인다.
- **Vaihingen**: 원격 탐사 분야의 대표적인 ALS 데이터셋이다.
- **ScanNet**: 인스턴스 수준의 실내 RGB-D 데이터셋으로 Voxel 기반으로 라벨링 되어 있다.

### 2. 정성적 결과 및 분석

- **전통적 ML vs 딥러닝**: 딥러닝 기반 방법들이 벤치마크 데이터셋에서 압도적으로 높은 정확도를 보이지만, 모델의 해석 가능성(Interpretability)이 낮다는 단점이 있다.
- **데이터 소스별 경향**: LiDAR 데이터는 도시 모델링 및 수목 분석에 가장 많이 쓰이며, RGB-D는 실내 환경 재구성에 특화되어 있다. SAR 데이터는 건물 파사드(facade) 정보 추출에 강점이 있으나 아직 연구가 부족한 상태이다.
- **모델 성능**: PointNet은 현재 대부분의 딥러닝 기반 PCSS 연구에서 기본 베이스라인(baseline)으로 활용되고 있으며, 이후 등장한 DGCNN, PointCNN 등이 지역 특징 추출 능력을 개선하며 성능을 높였다.

## 🧠 Insights & Discussion

### 1. 강점 및 기회

본 논문은 단순한 알고리즘 나열을 넘어, 데이터의 획득-처리-분류-평가로 이어지는 전체 파이프라인을 분석하였다. 특히 원격 탐사와 컴퓨터 비전이라는 서로 다른 두 학문의 관점을 대조함으로써, 실무적인 적용 가능성을 높이는 통찰을 제공하였다.

### 2. 한계 및 미해결 질문

- **해석 가능성 문제**: 딥러닝의 '블랙박스' 특성 때문에 자율주행과 같이 안전성이 극도로 중요한 분야에서는 완전히 신뢰하기 어렵다.
- **데이터의 다양성 부족**: 현재 벤치마크 데이터셋이 LiDAR와 RGB-D에 편중되어 있으며, Image-derived나 SAR 기반의 대규모 라벨링 데이터셋이 부족하다.
- **노이즈 적응성**: 실제 원격 탐사 데이터에는 센서 노이즈와 아웃라이어가 많으나, 현재의 컴퓨터 비전 알고리즘들은 이상적인 데이터셋에서 주로 테스트되어 실제 환경에서의 강건성(Robustness)이 검증되지 않았다.

### 3. 비판적 해석

저자들은 GNN(Graph Neural Networks)의 잠재력을 언급하며 향후 PCSS의 핵심이 될 것이라 주장한다. 하지만 3D 데이터의 방대한 크기를 고려할 때, 그래프 구축에 드는 계산 복잡도를 어떻게 효율적으로 해결할 것인가에 대한 구체적인 논의는 부족한 편이다. 또한, 하이브리드 방법론이 효율적이라고 언급했으나, 전처리 단계(PCS)에서 발생하는 오차가 최종 시맨틱 라벨링에 미치는 영향에 대한 정량적 분석이 더 필요해 보인다.

## 📌 TL;DR

본 논문은 3D 포인트 클라우드 시맨틱 세그멘테이션(PCSS)의 전 과정을 다룬 종합 리뷰 논문이다. 데이터 획득 방식부터 전통적인 기하학적 세그멘테이션(PCS), 그리고 최신 딥러닝 기반의 시맨틱 분류 기법까지 체계적인 분류 체계를 제시하였다. 특히 딥러닝 기법을 Multiview, Voxel, Point-based로 구분하여 분석하고, 컴퓨터 비전과 원격 탐사 분야의 시각 차이를 분석한 점이 돋보인다. 이 연구는 향후 3D 데이터 분석을 위한 모델 설계 시 어떤 데이터 특성에 어떤 알고리즘을 선택해야 하는지에 대한 중요한 이정표 역할을 할 것으로 기대된다.
