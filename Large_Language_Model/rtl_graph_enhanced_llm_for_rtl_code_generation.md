# RTL++: Graph-enhanced LLM for RTL Code Generation

Mohammad Akyash, Kimia Azar, Hadi Kamali (2025)

## 🧩 Problem to Solve

현대 하드웨어 설계의 복잡성이 증가함에 따라 전자 설계 자동화(EDA) 분야에서 고도화된 자동화의 필요성이 절실해지고 있다. 전통적인 Register Transfer Level(RTL) 설계 방식은 수동으로 이루어지며, 시간이 많이 소요될 뿐만 아니라 오류 발생 가능성이 높다. 최근 지시어 튜닝(instruction-tuning)된 상용 대규모 언어 모델(LLM)들이 자동화 가능성을 보여주었으나, 보안 및 개인정보 보호 문제로 인해 기업의 기밀 설계 데이터를 상용 모델에 입력하는 것에는 큰 제약이 따른다.

이에 대한 대안으로 오픈 소스 LLM이 제시되고 있지만, 효과적인 학습과 일반화를 위해 필수적인 고품질의 RTL 코드 데이터가 부족하여 생성된 코드의 품질과 정확성이 떨어지는 한계가 있다. 특히, 기존 LLM들은 RTL의 핵심 특성인 병렬성(concurrency), 계층 구조(hierarchy), 데이터 및 제어 흐름(data and control flow)에 대한 이해도가 낮아, 실제 하드웨어 동작과 일치하지 않는 환각(hallucination) 현상이 빈번하게 발생한다. 본 논문의 목표는 RTL 코드의 구조적 정보를 그래프 형태로 변환하여 LLM에 제공함으로써, 하드웨어 설계의 구조적 이해도를 높이고 고품질의 RTL 코드를 생성하는 RTL++ 모델을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 RTL 코드를 단순한 텍스트로만 처리하지 않고, 이를 구조적으로 분석한 그래프 표현(Graph Representation)을 결합하여 LLM의 문맥 이해 능력을 강화하는 것이다.

구체적으로는 RTL 코드를 텍스트화된 Control Flow Graphs(CFG)와 Data Flow Graphs(DFG)로 인코딩하여, 코드 내에 내재된 계층 구조, 의존성 및 구성 요소 간의 관계를 캡처한다. 이러한 구조적 그래프 기반 접근 방식은 LLM이 하드웨어 설계의 의도를 더 정확하게 파악할 수 있도록 돕는다. 특히, 단순히 기존 코드를 학습시키는 것을 넘어, 그래프 정보를 활용해 더욱 정교한 학습용 지시어(instruction) 데이터를 생성함으로써 데이터의 다양성과 품질 문제를 동시에 해결하고자 하였다.

## 📎 Related Works

### 기존 RTL 생성 LLM의 한계

- **VeriGen**: GitHub 등에서 수집한 대량의 데이터를 사용했으나, 전처리와 조직화가 부족하여 문법 오류가 있는 비기능적 코드를 생성하는 경우가 많았다.
- **RTLCoder**: GPT-3.5를 이용해 코드-지시어 쌍을 합성하였으나, GPT-3.5의 내장 지식에 의존함으로써 생성되는 코드의 다양성이 제한되었다.
- **OriGen & BetterV**: 코드 증강(augmentation) 및 자기 성찰(self-reflection) 메커니즘, 혹은 PPA(Power, Performance, Area) 최적화를 위한 판별기(discriminator)를 도입하여 품질을 높이려 했다.
- **AutoVCoder**: 도메인 특화 RAG(Retrieval-Augmented Generation) 모듈을 도입하여 프롬프트를 강화했다.
- **CraftRTL**: Karnaugh Map이나 상태 전이도와 같은 비텍스트적 표현을 활용하여 정교한 데이터를 구축했다.

### 차별점

기존의 연구들이 주로 텍스트 기반의 데이터 증강이나 RAG, 혹은 단순한 문법 교정에 집중했다면, RTL++는 RTL 코드의 정적 분석 결과인 CFG와 DFG라는 구조적 정보를 직접적으로 활용한다. 이는 하드웨어 설계의 본질적인 특성인 데이터 흐름과 제어 흐름을 모델이 학습 단계에서부터 인지하게 함으로써, 단순한 텍스트 패턴 매칭을 넘어선 구조적 최적화를 가능하게 한다.

## 🛠️ Methodology

RTL++의 전체 파이프라인은 크게 다섯 가지 단계로 구성된다.

### 1. RTL 데이터셋 수집 (Collecting RTL Dataset)

GitHub, Bitbucket, Opencores와 같은 오픈 소스 저장소에서 RTL 설계와 관련된 주요 키워드(Table II 참조)를 기반으로 데이터를 수집한다. 이때 데이터의 품질을 확보하기 위해 다음과 같은 필터링 과정을 거친다.

- **규모 제한**: 너무 짧거나 긴 코드를 배제하고 100~300라인 사이의 모듈을 선택하여 복잡성과 학습 효율의 균형을 맞춘다.
- **품질 검증**: GitHub Star 수가 높은 저장소를 우선적으로 선택하여 커뮤니티에서 검증된 신뢰도 높은 코드를 확보한다.
- **순수 RTL 추출**: 테스트벤치(test-bench)나 넷리스트(netlist)를 제외하고 순수하게 동작을 기술하는 RTL behavioral design 파일만을 추출한다.

### 2. GPT를 이용한 코드 정제 (RTL Code Refinement)

수집된 원본 코드를 GPT-4o를 사용하여 학습에 최적화된 형태로 정제한다.

- **의존성 제거**: 외부 파일이나 서브모듈에 의존하는 로직을 메인 모듈 내부로 직접 통합하여 self-contained 모듈로 만든다.
- **변수 정의 및 초기화**: 정의되지 않은 전역 변수나 헤더 파일의 함수들을 문맥에 맞게 추론하여 값을 할당한다.
- **문법 교정 및 합성 가능성 확인**: 표준 Verilog 문법을 준수하도록 수정하며, 최종적으로 Yosys 합성 도구를 통해 실제 합성이 가능한 코드인지 검증하고 오류가 있는 샘플은 제거한다.

### 3. 고수준 그래프 생성 (High-level Graph Generation)

정제된 RTL 코드를 바탕으로 구조적 정보를 추출하여 텍스트 형태의 그래프로 변환한다.

- **Flattening**: 계층적 설계의 경우 모든 모듈을 메인 그래프로 통합한다.
- **DFG 생성**: 입력에서 출력까지의 데이터 이동 경로를 정의하는 Data Flow Graph를 생성한다.
- **CFG 생성**: 입력에서 출력까지의 제어 신호 흐름을 정의하는 Control Flow Graph를 생성한다.
- **속성 추가**: 각 노드에 타입(gate, module, input 등)과 기능(arith, storage 등) 속성을 부여하고, 순차 회로의 동작 순서 및 피드백 루프(FSM 등)와 같은 시간적 동작(temporal behavior) 정보를 추가한다.

### 4. 그래프 강화 지시어 생성 (Graph-enhanced Instruction Generation)

단순히 코드만 사용하는 것이 아니라, 위에서 생성된 텍스트화된 그래프(CFG, DFG)를 GPT-4o의 프롬프트에 함께 입력한다. 이를 통해 LLM은 코드의 구문적 의미뿐만 아니라 구조적 의존성을 모두 파악하여, 매우 상세하고 정확한 기능 설명(instruction)을 생성하게 된다. 결과적으로 $\text{(RTL 코드, 그래프 기반 지시어)}$ 쌍의 데이터셋이 구축된다.

### 5. 그래프 강화 지시어 튜닝 (Graph-enhanced Instruction-tuning)

최종적으로 구축된 데이터셋을 사용하여 기본 모델인 CodeLlama-7B-Instruct를 파인튜닝한다. 이때 효율적인 학습을 위해 LoRA(Low-Rank Adaptation) 기법을 적용하며, AdamW 옵티마이저와 코사인 학습률 감쇠(cosine decay) 스케줄을 사용한다.

## 📊 Results

### 실험 설정

- **기본 모델**: CodeLlama-7B-Instruct
- **평가 지표**: VerilogEval 및 RTLLM 1.1 벤치마크의 $\text{pass@k}$ (k=1, 5, 10) 지표를 사용한다. $\text{pass@k}$는 $k$번의 시도 중 최소 하나 이상의 정답을 생성할 확률을 의미한다.
- **비교 대상**: GPT-4, VeriGen, RTLCoder, OriGen, BetterV, AutoVCoder, CraftRTL 등.

### 주요 결과

- **데이터 크기별 성능**: 학습 데이터셋의 크기가 증가함에 따라 성능이 일관되게 향상되었다. 특히 100K 샘플 이상으로 학습했을 때 GPT-4의 성능을 상회하는 결과를 보였다 (Table III).
- **SOTA 모델 비교**: $\text{pass@1}$ 기준, RTL++(200K 학습)은 59.9%를 기록하며 기존 SOTA 모델인 CraftRTL(63.1%)과 경쟁 가능한 수준에 도달했으며, RTLLM 1.1 벤치마크에서도 매우 높은 성공률을 보였다.
- **그래프 활용의 효과 (Ablation Study)**: 동일한 데이터셋(5K)에서 그래프 표현(TGR)을 사용하지 않았을 때보다 사용했을 때 $\text{pass@10}$ 기준 약 5%의 성능 향상이 있었으며, 100K 데이터셋으로 확장 시 그 차이는 18%까지 벌어졌다 (Table V).

### 케이스 스터디 (ALU 설계)

ALU 모듈 생성 실험에서, 그래프 표현 없이 생성된 지시어는 단순한 기능 나열에 그쳤으나, 그래프 표현을 통해 생성된 지시어는 $\text{ALUOp}$ 값에 따른 구체적인 연산 매핑과 기본값 처리, Zero 플래그의 생성 조건 등을 매우 상세하게 명시하였다. 이는 모델이 제어 흐름과 데이터 의존성을 더 정확하게 이해하고 코드를 생성하게 함을 시사한다.

## 🧠 Insights & Discussion

### 강점

RTL++의 가장 큰 성과는 하드웨어 설계의 핵심인 '구조(Structure)'를 LLM이 이해할 수 있는 텍스트화된 그래프 형태로 제공했다는 점이다. 이는 단순히 더 많은 데이터를 학습시키는 것보다, 데이터의 '질'과 '맥락'을 개선하는 것이 LLM 기반의 RTL 생성에서 얼마나 중요한지를 입증한다. 특히, 상용 모델에 의존하지 않고 오픈 소스 모델만으로도 고성능을 달성함으로써 보안이 중요한 EDA 분야에 실질적인 대안을 제시하였다.

### 한계 및 논의사항

- **데이터 생성 비용**: 데이터셋 구축 과정에서 GPT-4o를 사용하여 정제 및 지시어 생성을 수행하였으며, 1,000개 샘플당 약 84달러의 비용이 발생하였다. 이는 대규모 데이터셋 구축 시 경제적 부담이 될 수 있다.
- **텍스트화된 그래프의 한계**: 본 논문은 GNN과 같은 직접적인 그래프 임베딩 대신 텍스트화된 그래프를 사용하였다. 이는 해석력은 높으나, 그래프의 규모가 매우 커질 경우 LLM의 컨텍스트 윈도우 제한으로 인해 정보 손실이 발생할 가능성이 있다.
- **합성 가능성**: Yosys를 통해 기본적인 합성을 검증했으나, 실제 물리적 설계(Physical Design) 단계에서의 PPA 최적화나 타이밍 분석 결과까지는 반영되지 않았다.

## 📌 TL;DR

RTL++는 RTL 코드의 구조적 특성을 캡처하기 위해 **Control Flow Graph(CFG)와 Data Flow Graph(DFG)를 텍스트 형태로 변환하여 LLM 학습에 활용**한 최초의 멀티모달 그래프 강화 모델이다. 고품질의 정제된 200K 데이터셋과 그래프 기반 지시어 튜닝을 통해 **CodeLlama-7B 모델을 GPT-4 수준 혹은 그 이상의 RTL 생성 성능**으로 끌어올렸으며, 이는 향후 오픈 소스 기반의 보안성 높은 하드웨어 설계 자동화 도구 개발에 중요한 기여를 할 것으로 평가된다.
