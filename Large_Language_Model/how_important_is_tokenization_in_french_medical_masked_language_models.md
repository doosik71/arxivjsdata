# How Important Is Tokenization in French Medical Masked Language Models?

Yanis Labrak, Adrien Bazoge, Béatrice Daille, Mickael Rouvier, Richard Dufour (2024)

## 🧩 Problem to Solve

본 논문은 프랑스어 의료 분야의 Masked Language Model(MLM)에서 Subword tokenization 전략이 모델의 성능에 미치는 영향을 분석하는 것을 목표로 한다.

최근 NLP 분야에서는 Byte-Pair Encoding(BPE), SentencePiece, WordPiece와 같은 Subword tokenization이 표준으로 자리 잡았으며, 이는 단어 수준이나 문자 수준의 토큰화보다 우수한 성능을 보인다. 그러나 구체적으로 어떤 요인이 이러한 성공을 이끄는지, 특히 의료 용어와 같이 형태론적 규칙이 뚜렷한 특수 도메인에서 최적의 분절 입도(segmentation granularity)가 무엇인지는 충분히 연구되지 않았다.

의료 용어는 그리스어와 라틴어의 어근 및 접사(Morpheme)가 결합된 응집적(agglutinative) 특성을 가진다. 하지만 기존의 통계적 토크나이저는 이러한 언어학적 지식을 반영하지 않고 단순히 빈도 기반으로 단어를 분할하므로, 동일한 의미의 용어가 일관성 없게 분절되는 문제가 발생한다. 따라서 본 연구는 프랑스어 의료 도메인에서 토큰화 전략이 다운스트림 태스크 성능에 미치는 영향을 심층 분석하고, 형태소 정보를 통합한 새로운 토큰화 방법을 제안하여 그 효과를 검증하고자 한다.

## ✨ Key Contributions

본 논문의 핵심 기여는 통계적 토큰화 알고리즘에 언어학적 지식인 형태소(Morpheme)를 명시적으로 통합하여 의료 용어의 분절 정확도를 높이고, 이것이 실제 모델 성능으로 이어지는지 분석한 점이다.

주요 기여 사항은 다음과 같다:

- **Morpheme-enriched Tokenization 제안**: 수동으로 정의된 의료 분야의 주요 형태소 리스트를 통계적 토큰화 알고리즘(BPE, SentencePiece)에 통합하는 전략을 제안하였다.
- **토큰화 입도와 성능의 상관관계 분석**: 토큰화 결과로 생성되는 단어당 평균 서브워드 수(granularity)와 다양한 NLP 태스크 성능 간의 피어슨 상관계수($\rho$)를 계산하여, 분절 입도가 성능에 미치는 영향을 정량적으로 분석하였다.
- **토크나이저 학습 데이터 소스의 영향 평가**: NACHOS(의료 프랑스어), PubMed Central(의료 영어), CC100(다국어 일반), Wikipedia(일반 프랑스어) 등 다양한 데이터 소스로 토크나이저를 학습시켜, 언어 및 도메인 적합성이 성능에 미치는 영향을 비교하였다.
- **광범위한 벤치마크 평가**: 23개의 프랑스어 의료 NLP 태스크(NER, POS, STS, Classification 등)를 통해 제안 방법과 기존 방법의 성능을 종합적으로 검증하였다.

## 📎 Related Works

기존 연구들은 도메인 특화 데이터를 사용하여 언어 모델을 사전 학습(Pre-training)하는 것이 성능 향상에 매우 효과적임을 보여주었다. 특히 의료 분야에서는 WordPiece, SentencePiece, BPE와 같은 통계적 토크나이저를 도메인 내 코퍼스로 학습시키는 방식이 널리 사용되어 왔다.

하지만 이러한 통계적 접근 방식이 모든 태스크나 언어에 최적인지에 대해서는 의문이 제기되어 왔다. 일부 연구에서는 인간이 언어를 학습하는 방식과 유사하게 형태소(Morpheme) 정보를 토큰화 과정에 포함시키는 방법이 제안되었으나, 이는 주로 응집어(agglutinative languages) 연구에 집중되어 있었으며, 인도-유럽어족(특히 프랑스어)의 의료 도메인에 적용한 사례는 매우 부족한 실정이다. 본 논문은 이러한 간극을 메우기 위해 형태소 기반의 보강 전략을 도입하였다.

## 🛠️ Methodology

### 전체 파이프라인

본 연구의 실험 구조는 **[토크나이저 학습 데이터 선정] $\rightarrow$ [토크나이저 구축(통계적 vs 형태소 보강)] $\rightarrow$ [RoBERTa 모델 사전 학습] $\rightarrow$ [다운스트림 태스크 파인튜닝 및 평가]** 순으로 진행된다.

### 토큰화 전략

1. **통계적 토큰화 (Statistical Tokenization)**:
   - **BPE**: 개별 문자에서 시작하여 빈도수가 높은 문자 쌍을 반복적으로 병합한다.
   - **SentencePiece**: Unigram 및 BPE 알고리즘을 사용하며, 더 유연한 분절 입도를 제공한다.
2. **형태소 보강 토큰화 (Morpheme-enriched Tokenization)**:
   - 프랑스어 의료 분야에서 자주 사용되는 약 600개의 어휘 형태소(lexical morphemes, 예: *céphal-*, *-thérapie*) 리스트를 구축하였다.
   - BPE 및 SentencePiece 알고리즘을 수정하여, 학습 과정에서 이 형태소 리스트를 **강제 토큰(enforced selections)**으로 지정하였다.
   - 즉, 텍스트 내에서 해당 형태소가 발견되면 통계적 빈도와 상관없이 우선적으로 해당 단위로 분절하고, 나머지 부분에 대해서만 표준 통계적 토큰화 절차를 적용한다.

### 모델 학습 및 데이터

- **사전 학습 모델**: RoBERTa 아키텍처를 사용하며, Masked Language Modeling(MLM) 목적 함수를 통해 학습하였다.
- **사전 학습 데이터**: NACHOS 코퍼스(약 11억 단어, 7.4GB raw text)를 사용하였다.
- **토크나이저 데이터**: 1GB 규모의 하위 집합을 네 가지 소스(NACHOS, PubMed Central, CC100, Wikipedia)에서 추출하여 사용하였다.
- **실험 설정**: 총 16가지 조합(2가지 알고리즘 $\times$ 2가지 형태소 적용 여부 $\times$ 4가지 데이터 소스)의 토크나이저를 생성하고, 각 토크나이저별로 RoBERTa 모델을 처음부터(from scratch) 학습시켰다.

## 📊 Results

### 실험 설정

- **태스크**: NER, POS Tagging, Semantic Textual Similarity(STS), Classification 등 총 23개 태스크.
- **지표**: SeqEval(NER, POS), MSE(STS), F1-score(Classification).
- **비교군**: 형태소 보강 여부(w/ vs w/o) 및 학습 데이터 소스별 모델 성능 비교.

### 주요 결과

1. **분절 입도와 성능의 상관관계**:
   - 단어당 평균 서브워드 수와 성능 간의 상관계수를 분석한 결과, 평균 $\rho = -0.48$로 나타났다. 이는 **서브워드 수가 적을수록(즉, 더 큼직하게 분절될수록) 전반적인 성능이 향상되는 경향**이 있음을 의미한다.
   - 특히 NER과 STS 같은 태스크는 문맥 의존도가 높아 입도 변화에 민감하게 반응하였다.

2. **형태소 보강의 효과**:
   - **분절 정확도**: 형태소 보강 시 언어학적 정답 분절과 일치하는 비율(Exact Match)이 크게 향상되었다. 특히 SentencePiece NACHOS + Morphemes 조합은 61.3%의 일치율을 보였다.
   - **태스크 성능**: 형태소 보강이 약 25%의 태스크에서 성능 향상을 가져왔으나, 모든 태스크에서 일관된 성능 향상을 보이지는 않았다.

3. **데이터 소스의 영향**:
   - **언어 적합성 $>$ 도메인 적합성**: 도메인이 일치하더라도 언어가 다른 데이터(PubMed Central, 영어)로 토크나이저를 학습시킨 모델보다, 도메인은 다르지만 언어가 일치하는 데이터(Wikipedia, 프랑스어)로 학습시킨 모델의 성능이 더 좋았다.
   - PubMed 기반 토크나이저는 프랑스어 특유의 철자나 형태론적 구조를 반영하지 못해 심각한 **과분절(over-segmentation)** 현상을 보였으며, 이는 성능 저하로 이어졌다.

## 🧠 Insights & Discussion

본 연구는 토큰화 전략이 단순히 전처리 단계에 그치는 것이 아니라, 모델이 언어의 구조적 특징을 이해하는 방식에 직접적인 영향을 준다는 것을 보여준다.

- **분절 입도의 중요성**: 서브워드 수가 너무 많아지면(과분절) 토큰의 희소성(sparsity)이 증가하고, 모델의 입력 윈도우(512 tokens)가 의미 없는 작은 단위들로 빠르게 채워져 효율성이 떨어진다. 이는 본 논문이 발견한 $\rho = -0.48$의 상관관계로 뒷받침된다.
- **모델의 강건성(Robustness)**: 흥미로운 점은 형태소 보강을 통해 분절 정확도를 크게 높였음에도 불구하고, 실제 다운스트림 성능 향상은 제한적이었다는 것이다. 이는 RoBERTa와 같은 대규모 트랜스포머 아키텍처가 어느 정도의 토큰화 노이즈(suboptimal tokenization)를 극복하고 의미를 파악할 수 있는 강건함을 갖추고 있음을 시사한다.
- **언어-도메인 우선순위**: 의료 도메인 모델이라 할지라도, 타겟 언어(프랑스어)의 일반적인 언어 구조를 학습한 토크나이저가 타겟 도메인의 타 언어(영어) 구조를 학습한 것보다 훨씬 유리하다.

## 📌 TL;DR

본 논문은 프랑스어 의료 MLM에서 **형태소 정보를 통합한 토큰화 전략**을 제안하고 그 영향을 분석하였다. 연구 결과, **단어당 서브워드 수가 적을수록(입도가 낮을수록) 성능이 향상되는 경향**이 있으며, 토크나이저 학습 시 **도메인 일치보다 언어 일치가 더 중요하다**는 것을 밝혀냈다. 형태소 보강은 언어학적 분절 정확도를 크게 높이지만, 모델의 강건성으로 인해 실제 태스크 성능 향상으로 이어지는 정도는 제한적이었다. 이 연구는 향후 특수 도메인 언어 모델 구축 시 언어 특성에 맞는 토큰화 설계가 필수적임을 시사한다.
