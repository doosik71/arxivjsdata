# LLM+MAP: Bimanual Robot Task Planning using Large Language Models and Planning Domain Definition Language

Kun Chu, Xufeng Zhao, Cornelius Weber, and Stefan Wermter (2025)

## 🧩 Problem to Solve

본 논문은 양팔 로봇(Bimanual Robot)의 조작 작업에서 발생하는 복잡한 공간적 및 시간적 협응(Spatial and Temporal Coordination) 문제를 해결하고자 한다. 양팔 조작은 높은 범용성을 제공하지만, 두 팔 사이의 정밀한 동기화와 긴 호라이즌(Long-horizon)에 걸친 작업 계획 수립이 매우 어렵다는 고유한 도전 과제가 존재한다.

최근 대규모 언어 모델(LLM)이 로봇 작업 계획에 도입되고 있으나, LLM은 긴 호라이즌의 추론 과정에서 오류를 범하거나 복잡한 로봇 작업 시 환각(Hallucination) 현상을 일으키는 경향이 있다. 특히, 생성된 계획의 논리적 정밀성을 보장할 수 없다는 점이 큰 한계로 지적된다. 따라서 본 연구의 목표는 LLM의 추론 능력과 심볼릭 플래너(Symbolic Planner)의 논리적 엄격함을 결합하여, 양팔 로봇을 위한 효율적이고 논리적으로 올바른 작업 계획 프레임워크를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 LLM을 직접적인 플래너로 사용하는 대신, 자연어 기반의 작업 설명과 환경 정보를 PDDL(Planning Domain Definition Language) 형식으로 변환하는 '번역기'로 활용하고, 실제 계획 생성은 다중 에이전트 플래닝(Multi-Agent Planning) 솔버에 맡기는 것이다.

특히, 양팔 로봇을 두 개의 동질적인 에이전트(왼팔과 오른팔)로 구성된 특수 다중 에이전트 시스템으로 정의함으로써, 두 팔이 독립적으로 수행할 수 있는 작업과 협력하여 수행해야 하는 작업을 명확히 구분한다. 이를 통해 논리적 정밀성이 보장된 부분 순서 계획(Partial-order Plan)을 생성하여, 병렬 실행을 통한 작업 효율성을 극대화하고 논리적 오류를 제거하였다.

## 📎 Related Works

**1. 로봇 작업 계획 방식**

- **심볼릭 플래너(Symbolic Planners):** PDDL이나 ASP(Answer Set Programming) 같은 선언적 언어를 사용하여 논리적 정밀성을 보장한다. 그러나 도메인 특화 언어로 문제를 직접 정의해야 하는 번거로움이 있다.
- **LLM 기반 플래너:** SayCan과 같이 LLM의 상식 지식을 활용해 계획을 세우는 방식이다. 구현이 쉽지만, 긴 호라이즌 추론에서의 취약성과 환각 현상이 주요 한계이다.
- **결합 방식(LLM + Symbolic):** LLM+P나 ViLaIn과 같이 LLM이 자연어를 PDDL로 변환하고 심볼릭 플래너가 이를 해결하는 방식이다. 하지만 기존 연구들은 주로 순차적 계획(Sequential Planning)에 집중했으며, 양팔 로봇과 같은 병렬 실행 및 다중 에이전트 협응 문제는 다루지 않았다.

**2. 양팔 조작 및 다중 에이전트 플래닝**
기존의 양팔 조작 연구들은 주로 특정 기술(예: 옷 접기, 채소 썰기)의 획득에 집중했으며, 고수준의 작업 계획 단계는 간과하는 경향이 있었다. DAG-Plan과 같은 최근 시도는 LLM을 이용해 작업을 유향 비순환 그래프(DAG)로 분해하여 할당하지만, 추상 공간에서의 계산적 탐색을 통한 최적화와 자기 검증 능력이 부족하다는 한계가 있다.

## 🛠️ Methodology

### 1. 양팔 로봇의 공간 장면 이해 (Spatial Scene Understanding)

로봇의 작업 영역을 도달 가능 범위에 따라 다음과 같이 세 가지 영역으로 정의한다.

- **비협응 영역 (Uncoordinated Areas):** 한쪽 팔만 도달할 수 있는 왼쪽/오른쪽 영역이다. 이곳에서는 두 팔이 서로 영향을 주지 않고 독립적으로 병렬 작업을 수행할 수 있다.
- **협응 영역 (Coordinated Area):** 두 팔이 모두 도달 가능한 중앙의 중첩 영역이다. 여기서는 비동기적(한 팔이 다른 팔의 전제 조건을 형성) 또는 동기적(정밀한 상호 의존성)으로 협력한다.

시스템은 OWLv2 객체 탐지 모델을 사용하여 객체의 위치를 파악하고, 미리 정의된 기준선(Black lines)과 바운딩 박스를 비교하는 규칙 기반 인식기를 통해 각 객체가 어느 영역에 속하는지 판단한다.

### 2. LLM+MAP 프레임워크 구조

전체 파이프라인은 다음과 같은 순서로 진행된다.

1. **입력:** 자연어 작업 설명과 비전 모델로부터 얻은 공간 정보가 입력된다.
2. **PDDL 생성:** LLM이 이를 바탕으로 PDDL의 초기 상태(Initial State)와 목표 상태(Goal State)를 정의한다. 이때 LLM은 Python의 `Unified-Planning` 라이브러리를 사용하여 코드를 작성하며, 인터프리터 오류 발생 시 재프롬프팅을 통해 수정한다.
3. **심볼릭 플래닝:** FMAP(Forward Multi-Agent Planning) 솔버를 사용하여 논리적으로 올바른 부분 순서 계획(Partial-order Plan)을 생성한다.
4. **실행:** 생성된 계획을 기반으로 NICOL 양팔 로봇이 작업을 수행한다.

### 3. PDDL 정의 및 다중 에이전트 설정

작업 문제는 다음과 같은 튜플 $T$로 정의된다.
$$T = \langle O, P, A, S, G \rangle$$
여기서 $O$는 객체 집합, $P$는 술어(Predicate) 집합, $A$는 액션 집합, $S$는 초기 상태, $G$는 목표 상태이다.

**양팔 로봇을 위한 특화 설정:**

- **에이전트 정의:** `LEFT`와 `RIGHT`를 독립적인 에이전트로 설정하며, 각 에이전트는 자신의 팔을 제어할 수 있는 전용 술어(`control`)를 가진다.
- **술어(Predicates):** `armat(hand, area)`, `lifting(object, hand)`, `objectatarea(object, area)` 등 공간적 관계와 상태를 정의한다.
- **스킬 설계:**
  - **단일 스킬 (Single Skills):** `grasp`, `moveto`, `release`, `pour` 등 한 팔이 수행하는 기본 동작이다.
  - **결합 스킬 (Joint Skills):** `cohold`, `comoveto`와 같이 두 팔이 동시에 참여하는 대칭 조작 동작이다. 이 스킬들은 특정 팔을 입력으로 받지 않고, 두 팔 모두 `available` 상태여야 한다는 전제 조건을 가진다.

## 📊 Results

### 1. 실험 설정

- **환경:** CoppeliaSim 시뮬레이터 및 NICOL 로봇 사용.
- **작업 도메인:**
  - **ServeWater:** 노란 컵에 물을 따라 사람에게 주고, 파란 컵은 상자에 넣는 작업.
  - **ServeFruit:** 과일들을 그릇에 담아 사람에게 전달 (그릇은 크기가 커서 두 팔로 옮겨야 함).
  - **StackBlock:** 지정된 순서대로 블록을 쌓아 1~2개의 탑을 만드는 작업.
- **비교 대상:** GPT-4o (direct), DeepSeek-V3 (direct), OpenAI-o1 (direct), DeepSeek-R1 (direct).

### 2. 평가 지표

- **Planning Time (PT):** 계획 생성에 걸리는 시간.
- **Success Rate (SR):** 작업 완수 성공률.
- **Group Debits (GD):** 가장 적은 단계로 작업을 완수한 '챔피언' 모델 대비 추가된 단계 수 (낮을수록 효율적).

### 3. 주요 결과

- **성공률 (SR):** LLM+MAP은 모든 도메인에서 가장 높은 성공률을 보였다. 특히 단순 LLM 생성 방식(GPT-4o, V3)은 매우 낮은 성공률을 보인 반면, 추론 모델(o1, R1)은 상대적으로 높았으나 여전히 LLM+MAP에 미치지 못했다.
- **계획 시간 (PT):** o1과 R1 같은 추론 모델은 높은 성공률을 위해 막대한 추론 시간을 소모한다. 반면 LLM+MAP은 GPT-4o를 백엔드로 사용함에도 불구하고 훨씬 빠른 속도로 최적의 계획을 생성했다.
- **효율성 (GD):** LLM+MAP은 특히 쉬운 작업에서 0-debits(최적 경로)에 가까운 분포를 보이며 압도적인 효율성을 입증했다.

### 4. 절제 연구 (Ablation Study)

다중 에이전트 플래닝(MAP) 대신 단일 에이전트로 처리하는 순차적 계획(LLM+P 방식)과 비교한 결과, 계획 단계 감소율(PSRR)이 유의미하게 나타났다. 이는 LLM+MAP이 두 팔의 병렬 작업 할당을 효과적으로 수행함으로써 전체 실행 시간을 단축시켰음을 의미한다.

## 🧠 Insights & Discussion

본 연구는 LLM의 강력한 문맥 이해 능력과 심볼릭 플래너의 논리적 무결성을 결합했을 때, 특히 양팔 로봇과 같은 복잡한 협응 시스템에서 큰 시너지가 발생함을 보여주었다.

**강점 및 통찰:**

- **추론 모델의 '과잉 생각(Overthinking)':** o1, R1과 같은 최신 추론 모델들은 논리적 성능은 높지만, 간단한 작업에서도 불필요하게 많은 추론 시간을 소모하고 장황한 결과물을 내놓는 경향이 있다. 반면, PDDL로의 변환 후 심볼릭 탐색을 거치는 방식은 계산 효율성이 훨씬 뛰어나다.
- **공간 vs 시간 이해도:** 실험 결과, o1 모델은 공간적 이해도가 높아 성공률이 높았으며, R1 모델은 시간적 협응(효율적인 단계 구성) 면에서 더 우수한 성능을 보였다. 이는 LLM마다 강점이 다르며, 이를 구조적인 플래닝 프레임워크로 보완하는 것이 중요함을 시사한다.

**한계 및 향후 과제:**

- **하위 스킬 의존성:** 본 논문은 고수준 계획에 집중하며, 하위 동작(Motion Primitives)이 완벽하게 수행된다고 가정한다. 실제 환경 적용을 위해서는 학습 기반의 정밀 조작 스킬과의 통합이 필요하다.
- **동적 환경 대응:** 현재는 정적인 상태에서 계획을 세우지만, 향후 액션의 비용(Cost)이나 지속 시간을 고려하고, 실행 중 실패 시 적응적으로 재계획(Re-planning)하는 기능이 추가되어야 한다.

## 📌 TL;DR

LLM+MAP은 양팔 로봇의 복잡한 협응 문제를 해결하기 위해 **LLM(PDDL 작성자) $\rightarrow$ 심볼릭 플래너(FMAP 솔버) $\rightarrow$ 부분 순서 계획(Partial-order Plan)** 구조를 제안한 프레임워크이다. 이를 통해 기존 LLM 단독 계획 방식의 환각 문제를 해결하고, 최신 추론 모델(o1, R1)보다 빠른 생성 시간과 더 높은 성공률 및 실행 효율성을 달성하였다. 이 연구는 대규모 언어 모델을 로봇의 고수준 제어 시스템에 논리적으로 접목하는 효과적인 방법론을 제시하였다는 점에서 의의가 있다.
