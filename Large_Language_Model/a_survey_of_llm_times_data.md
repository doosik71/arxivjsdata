# A Survey of LLM$\times$DATA

Xuanhe Zhou, Junxuan He, Wei Zhou, Haodong Chen, Zirui Tang, Haoyu Zhao, Xin Tong, Guoliang Li, Youmin Chen, Jun Zhou, Zhaojun Sun, Binyuan Hui, Shuo Wang, Conghui He, Zhiyuan Liu, Jingren Zhou, Fan Wu (2025)

## 🧩 Problem to Solve

본 논문은 거대 언어 모델(Large Language Model, LLM)과 데이터 관리(Data Management, DATA) 사이의 양방향 관계를 체계적으로 분석하고자 한다. LLM의 성능은 학습 및 추론 단계에서 사용되는 데이터의 품질과 효율적인 관리에 절대적으로 의존하며, 반대로 LLM의 강력한 추론 능력은 전통적인 데이터 관리 작업의 자동화와 지능화를 가능하게 한다.

구체적으로, 논문은 다음 두 가지 핵심 문제를 해결하려 한다. 첫째, **DATA4LLM** 관점에서 LLM의 전생애 주기(사전 학습, 미세 조정, RAG, 에이전트 등) 전반에 걸쳐 데이터를 어떻게 수집, 처리, 저장 및 서빙할 것인가에 대한 체계적인 방법론을 제시하는 것이다. 둘째, **LLM4DATA** 관점에서 LLM을 데이터 조작, 분석, 시스템 최적화라는 데이터 관리의 핵심 작업에 어떻게 적용하여 기존 방식의 한계를 극복할 것인가를 분석하는 것이다.

## ✨ Key Contributions

본 논문의 핵심적인 기여는 LLM과 데이터 관리의 상호작용을 '양방향' 관점에서 정의하고 이를 위한 포괄적인 프레임워크를 제안한 점에 있다.

1. **IaaS 개념 제안**: LLM 데이터셋의 품질을 평가하기 위한 원칙적인 렌즈로서 IaaS(Inclusiveness, Abundance, Articulation, Sanitization) 개념을 제안한다. 이는 데이터의 포괄성, 풍부함, 명료성, 정제 상태라는 네 가지 차원을 통해 데이터 품질을 정량적/정성적으로 평가하는 기준을 제공한다.
2. **DATA4LLM의 생애 주기 기반 분류**: 단순한 데이터 수집을 넘어, 사전 학습(Pre-training)부터 SFT, RLHF, RAG, LLM 에이전트에 이르기까지 각 단계별 데이터의 특성과 이에 필요한 처리, 저장, 서빙 기술을 체계적으로 분류하였다.
3. **LLM4DATA의 통합적 분석**: 데이터 조작(Manipulation), 데이터 분석(Analysis), 시스템 최적화(System Optimization)라는 세 가지 영역에서 LLM이 어떻게 기존의 규칙 기반 또는 전통적 ML 기반 방식의 한계를 극복하고 있는지 상세히 분석하였다.

## 📎 Related Works

기존의 LLM 및 데이터 관리 관련 서베이 논문들은 주로 특정 단계(예: 사전 학습 단계의 데이터 필터링)나 특정 작업(예: 데이터 어노테이션)에 집중하는 경향이 있었다. 일부 연구는 고전적인 머신러닝(ML)을 데이터 관리에 적용하는 방안을 다루었으나, LLM만이 가진 독특한 강점인 '제로샷 추론'이나 '시맨틱 이해 능력'을 충분히 반영하지 못했다.

본 논문은 다음과 같은 점에서 기존 연구와 차별화된다. 우선, 데이터 처리-저장-서빙으로 이어지는 전체 파이프라인을 포괄하며, 특히 SFT, RAG, 에이전트와 같은 최신 LLM 활용 단계까지 분석 범위에 포함하였다. 또한, 데이터가 LLM을 돕는 방식(DATA4LLM)과 LLM이 데이터를 돕는 방식(LLM4DATA)을 동시에 다룸으로써 두 도메인의 상호 호혜적 관계를 명확히 규명하였다.

## 🛠️ Methodology

논문은 LLM$\times$DATA의 관계를 두 개의 큰 축으로 나누어 설명한다.

### 1. DATA4LLM: LLM을 위한 데이터 관리

LLM의 성능을 극대화하기 위해 데이터를 처리하고 제공하는 과정으로, 크게 세 단계로 구분된다.

#### (1) 데이터 처리 (Data Processing)

고품질 데이터셋을 구축하기 위한 일련의 과정이다.

* **데이터 획득(Acquisition)**: 웹 스크래핑, 레이아웃 분석(Layout Analysis) 등을 통해 원천 데이터를 수집한다.
* **중복 제거(Deduplication)**: 정확한 일치(Exact Matching), 해싱 기반 근사 일치(MinHash, SimHash), 임베딩 기반 클러스터링 등을 사용하여 중복 데이터를 제거함으로써 과적합을 방지한다.
* **필터링(Filtering)**: Perplexity 측정이나 모델 스코어링을 통한 샘플 수준 필터링과, 유해 콘텐츠 및 개인정보(PII)를 제거하는 콘텐츠 수준 필터링을 수행한다.
* **데이터 믹싱(Data Mixing)**: 다양한 도메인의 데이터 비율을 최적화한다. 여기에는 휴리스틱 방법뿐만 아니라 Bilevel Optimization이나 DRO(Distributionally Robust Optimization)와 같은 수학적 최적화 기법이 사용된다.
* **데이터 합성(Synthesis)**: 지식 증류(Knowledge Distillation)나 LLM 프롬프팅을 통해 부족한 데이터를 생성한다. 특히 Reasoning data 합성을 위해 CoT(Chain-of-Thought)나 프로그램 기반 검증(Program-aided verification)이 활용된다.

#### (2) 데이터 저장 (Data Storage)

거대한 모델 파라미터와 데이터셋을 효율적으로 관리하는 기술이다.

* **저장 포맷**: TFRecord, Safetensors 등 효율적인 이진 포맷을 사용한다.
* **분산 저장**: 3FS와 같은 분산 파일 시스템을 통해 대규모 데이터를 여러 노드에 분산 배치한다.
* **데이터 조직화**: RAG를 위해 텍스트를 벡터(Vector) 또는 그래프(Graph) 형태로 조직화하여 저장하며, 효율적인 검색을 위해 차원 축소 및 인덱싱 기술을 적용한다.
* **KV Cache 관리**: 추론 속도 향상을 위해 Key-Value 캐시를 효율적으로 배치하고, PagedAttention과 같은 블록 기반 관리 기법을 사용한다.

#### (3) 데이터 서빙 (Data Serving)

저장된 데이터를 모델이 소비하기 좋은 형태로 가공하여 제공하는 단계이다.

* **데이터 셔플링(Shuffling)**: 학습 단계에서 데이터의 순서를 결정하거나, RAG 단계에서 관련 지식을 필터링 및 재순위화(Re-ranking)한다.
* **데이터 압축(Compression)**: 프롬프트 길이를 줄이기 위해 정보 엔트로피 기반의 LLMLingua와 같은 기법으로 토큰을 압축한다.
* **데이터 패킹(Packing)**: 가변 길이의 시퀀스를 효율적으로 배치하여 패딩(Padding) 토큰을 최소화하고 학습 효율을 높인다.

### 2. LLM4DATA: 데이터 관리를 위한 LLM

LLM의 추론 능력을 데이터 관리 작업의 자동화에 활용하는 방법이다.

#### (1) 데이터 조작 (Data Manipulation)

* **데이터 클리닝**: LLM을 통해 비정형 데이터의 형식을 표준화하거나, 오류를 탐지하고 수정하며, 누락된 값을 추론하여 채우는(Imputation) 작업을 수행한다.
* **데이터 통합**: 서로 다른 데이터셋 간의 개체 매칭(Entity Matching) 및 스키마 매칭(Schema Matching)을 수행한다.
* **데이터 발견**: 데이터셋의 특성을 요약하는 프로파일링(Profiling) 및 시맨틱 레이블을 부여하는 어노테이션(Annotation)을 자동화한다.

#### (2) 데이터 분석 (Data Analysis)

* **구조적 데이터 분석**: 자연어 질의를 SQL이나 GQL로 변환하는 NL2SQL/NL2GQL 인터페이스를 구현하며, 복잡한 쿼리를 하위 문제로 분해하는 Multi-step QA 프레임워크를 사용한다.
* **비구조적 데이터 분석**: OCR-free 모델이나 멀티모달 LLM을 사용하여 문서의 레이아웃과 내용을 동시에 분석하며, 프로그램 분석을 통해 코드의 취약점을 탐지한다.

#### (3) 시스템 최적화 (System Optimization)

* **설정 튜닝(Configuration Tuning)**: 데이터베이스의 Knob 설정이나 인덱스 추천을 위해 LLM의 도메인 지식을 활용한다.
* **쿼리 최적화(Query Optimization)**: SQL 쿼리를 효율적인 형태로 재작성(Rewriting)하거나 최적의 실행 계획(Execution Plan)을 선택한다.
* **이상 진단(Anomaly Diagnosis)**: 시스템 메트릭과 로그를 분석하여 이상 현상의 근본 원인을 파악하고 해결책을 제시하는 진단 보고서를 생성한다.

## 📊 Results

본 논문은 서베이 논문이므로 특정 단일 실험 결과보다는, 인용된 수많은 연구의 정량적/정성적 결과들을 종합하여 제시한다.

1. **데이터 처리 파이프라인의 효율성**: FineWeb 파이프라인의 사례에서, 단순한 텍스트 추출보다 정교한 휴리스틱 필터링과 중복 제거를 적용했을 때 모델의 성능이 크게 향상됨을 확인하였다. 하지만 이를 위해 1B 모델 70여 개를 학습시키는 등 막대한 계산 비용(80,000 H100 GPU hours)이 소요됨을 지적하며, 데이터 기반의 최적 구성 예측 필요성을 강조한다.
2. **데이터 믹싱의 영향**: 데이터의 양보다 '최적의 혼합 비율'이 성능에 더 큰 영향을 미친다는 점을 확인하였다. 특히 $\text{L}(R) = \alpha \cdot R^s + \beta$ 와 같은 파워 로(Power-law) 함수를 통해 일반 데이터와 도메인 특화 데이터의 최적 비율을 예측하려는 시도들이 유효함을 보여준다.
3. **LLM 기반 분석 성능**: NL2SQL 및 데이터 클리닝 작업에서 LLM이 전통적인 규칙 기반 시스템보다 훨씬 높은 일반화 성능을 보이며, 특히 CoT 프롬프팅과 RAG를 결합했을 때 복잡한 스키마 환경에서도 높은 정확도를 유지함을 확인하였다.

## 🧠 Insights & Discussion

### 강점 및 시사점

본 논문은 LLM 개발의 핵심이 모델 구조(Architecture)에서 데이터(Data)로 이동하고 있다는 최근의 트렌드를 정확히 짚어내었다. 특히 **IaaS** 개념을 통해 데이터 품질을 다각도로 정의함으로써, 연구자들이 무작정 데이터를 늘리는 것이 아니라 '포괄성'과 '명료성' 등을 고려한 전략적 데이터 구축을 하도록 가이드를 제공한다. 또한, 단순한 툴로서의 LLM이 아니라, 데이터 시스템의 커널 수준 최적화(Query Optimization, Knob Tuning)까지 LLM을 적용하려는 시도를 정리함으로써 데이터베이스(DB)와 AI의 융합 방향을 제시하였다.

### 한계 및 향후 연구 방향

논문은 다음과 같은 미해결 과제들을 제시한다.

1. **데이터 선택의 자동화**: 현재의 데이터 필터링은 여전히 휴리스틱이나 고정된 레이블셋에 의존하고 있다. 모델의 학습 상태에 따라 동적으로 필요한 데이터를 선택하는 적응형(Adaptive) 전략이 필요하다.
2. **통합 분석 시스템의 부재**: 현재는 구조적, 반구조적, 비구조적 데이터를 처리하는 모델이 각각 분리되어 있다. 이를 하나의 프레임워크에서 통합 처리할 수 있는 일반화된 분석 시스템 구축이 필요하다.
3. **비텍스트 데이터의 표현력**: 표(Table)나 그래프(Graph) 데이터를 단순 텍스트로 직렬화(Serialization)하여 LLM에 입력하면 구조적 정보 손실이 발생한다. 이를 방지하기 위해 구조적 세만틱을 보존하는 새로운 인코딩 방식이나 멀티모달 접근법이 요구된다.
4. **비용 효율성**: LLM을 이용한 데이터 분석이나 최적화는 추론 비용과 지연 시간(Latency)이 매우 높다. 이를 위해 전통적인 통계적 방법과 LLM을 결합한 하이브리드 전략이나 효율적인 스케줄링 메커니즘이 필수적이다.

## 📌 TL;DR

본 논문은 **LLM과 데이터 관리의 양방향 시너지(LLM$\times$DATA)**를 분석한 포괄적인 서베이이다. **DATA4LLM**에서는 IaaS 품질 기준을 바탕으로 데이터의 수집-처리-저장-서빙 파이프라인을 체계화하였고, **LLM4DATA**에서는 LLM을 활용한 데이터 클리닝, 분석, 시스템 최적화 방법론을 정리하였다. 결과적으로 LLM의 성능 향상을 위해서는 데이터 공학적 접근이 필수적이며, 데이터 관리의 지능화를 위해서는 LLM의 추론 능력을 시스템 레벨로 통합하는 연구가 중요함을 시사한다.
