# CLIP-Embed-KD: Computationally Efficient Knowledge Distillation Using Embeddings as Teachers

Lakshmi Nair (2024)

## 🧩 Problem to Solve

본 논문은 지식 증류(Knowledge Distillation, KD) 과정에서 발생하는 막대한 계산 비용 문제를 해결하고자 한다. 일반적인 지식 증류 프레임워크는 학습 과정 내내 거대한 교사 모델(Teacher Model)의 순전파(Forward pass)를 수행하여 학생 모델(Student Model)의 가이드를 생성해야 한다. 그러나 최근의 교사 모델들은 파라미터 수가 수십억 개에서 수조 개에 달하기 때문에, 제한된 계산 자원을 가진 환경(예: 모바일 기기)에서는 이러한 방식이 사실상 불가능하거나 매우 비효율적이다.

따라서 본 연구의 목표는 교사 모델의 전체 네트워크를 매번 실행하는 대신, 미리 계산된 임베딩(Pre-computed embeddings)만을 활용하여 계산 효율성을 극대화하면서도 효과적으로 지식을 전수하는 지식 증류 방법을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 CLIP(Contrastive Language-Image Pre-training)의 대조 학습(Contrastive Learning) 구조를 지식 증류에 접목하고, 교사 모델의 역할을 '정적 임베딩'으로 대체하는 것이다.

1. **CLIP-Teacher-KD**: CLIP의 대조 학습 목적 함수를 지식 증류 손실 함수로 사용하여, 학생 모델의 임베딩을 교사 모델의 임베딩 공간에 정렬시킨다.
2. **CLIP-Embed-KD**: 교사 모델을 매번 실행하는 대신, 클래스별로 미리 계산된 평균 임베딩(Averaged embeddings)을 교사로 활용한다. 이를 통해 학습 과정에서 교사 모델의 순전파 연산을 완전히 제거하여 메모리 사용량과 학습 시간을 획기적으로 줄인다.

## 📎 Related Works

지식 증류는 크게 세 가지 방식으로 분류된다.

- **Response-based KD**: 교사와 학생 모델의 최종 분류 층 출력값(Logits)을 비교한다.
- **Feature-based KD**: 최종 층 및 중간 층의 특징 맵(Feature maps)을 비교한다.
- **Instance-based KD**: 교사와 학생 모델의 중간 특징 맵 쌍 간의 유사성을 비교한다.

기존의 모든 방식은 교사와 학생 모델 모두에 대해 순전파를 수행해야 한다는 한계가 있다. CLIP을 이용한 기존의 지식 증류 연구들은 토큰 선택(Token selection), 신뢰도 가중치(Confidence weighting), $L_1$ 차이 손실 함수 또는 KL-divergence 손실 함수 등을 사용하였다. 본 논문은 이러한 기존 방식과 달리, **"미리 계산된 임베딩만으로 학생 모델을 학습시킬 수 있는가?"**라는 질문에 집중하여 차별점을 둔다.

## 🛠️ Methodology

### 1. 전체 지식 증류 구조

기본적인 지식 증류 손실 함수 $L_D$는 학생 모델의 예측값 $\text{z}_s$, 교사 모델의 예측값 $\text{z}_t$, 그리고 정답 라벨 $\hat{y}$를 사용하여 다음과 같이 정의된다.

$$L_D(\text{z}_s, \text{z}_t, \hat{y}) = \alpha_1 L_{CE}(\text{z}_s, \hat{y}) + \alpha_2 L_{KD}(\text{z}_s, \text{z}_t)$$

여기서 $L_{CE}$는 일반적인 교차 엔트로피(Cross-Entropy) 손실이며, $L_{KD}$는 두 모델 간의 차이를 줄이는 증류 손실이다.

### 2. CLIP-Teacher-KD

본 제안 방법에서는 $L_{KD}$ 대신 CLIP의 대조 학습 목적 함수인 $L_{clip}$을 사용한다.

- **절차**:
    1. 학생 모델과 교사 모델에서 각각 $[CLS]$ 토큰의 임베딩 $E_s$와 $E_t$를 추출한다.
    2. 학습 가능한 투영 층(Projection layer) $W_{proj}$를 통해 교사 임베딩의 차원을 학생 임베딩 차원으로 맞춘다: $\hat{E}_t = E_t \cdot W_{proj}^T$.
    3. 정규화된 두 임베딩 간의 내적(Dot product)을 계산하여 유사도를 측정하고, 이를 정답 행렬 $G$(단위 행렬 $I_{B \times B}$)와 비교하여 $L_{clip}$을 계산한다.

$$L_{clip} = L_{CE}(\|E_s\| \cdot \|\hat{E}_t\|^T, G)$$

최종 손실 함수는 다음과 같다:
$$L_D(E_s, \hat{E}_t, \text{z}_s, \hat{y}) = \alpha_1 L_{CE}(\text{z}_s, \hat{y}) + \alpha_2 L_{clip}(E_s, \hat{E}_t)$$

### 3. CLIP-Embed-KD

교사 모델의 반복적인 순전파를 제거하기 위해 제안된 효율적인 방법이다.

- **절차**:
    1. 각 클래스 $c$에 대해 $N$개의 샘플을 무작위로 추출하여 교사 모델의 $[CLS]$ 임베딩을 미리 계산한다.
    2. 각 클래스 임베딩들의 평균을 구해 클래스당 하나의 '평균 임베딩' $E_t \in \mathbb{R}^{N_c \times D_t}$를 생성한다.
    3. 학습 시에는 교사 모델 대신 이 평균 임베딩들을 사용한다. 이때 $L_{clip}$의 정답 행렬 $G$는 단위 행렬이 아니라, 해당 배치의 샘플들에 대한 원-핫 인코딩(One-hot encoding) 행렬이 된다.
    4. 학생 임베딩 $E_s$와 전치된 평균 교사 임베딩 간의 내적을 통해 $B \times N_c$ 차원의 출력을 얻고, 이를 통해 $L_{clip}$을 계산한다.

## 📊 Results

### 실험 설정

- **데이터셋**: CIFAR100
- **모델**: Vision Transformer (ViT). 교사는 ImageNet으로 사전 학습된 ViT-base/large(patch size 16, 32)를 사용하였고, 학생은 ViT-base/large(patch size 4)를 사용하였다.
- **지표**: 검증 정확도(Validation Accuracy), 메모리 사용량, 학습 시간.

### 주요 결과

1. **손실 함수 비교**: 일반적인 KD 손실(Eq. 1)보다 CLIP 기반의 대조 학습 손실(Eq. 5)을 사용했을 때 더 높은 성능을 보였다.
2. **CLIP-Teacher-KD vs. CLIP-Embed-KD**:
    - **정확도**: CLIP-Teacher-KD가 약간 더 높은 정확도를 보였으나(약 2% 차이), 학생 모델의 크기가 커질수록 두 방법의 성능 차이는 줄어들었다.
    - **효율성**: CLIP-Embed-KD는 교사 모델을 메모리에 올릴 필요가 없으므로 극적인 자원 절감을 이뤄냈다. 표 2에 따르면, 교사 모델의 크기에 따라 메모리 사용량이 **17배에서 최대 59배까지 감소**하였다.
    - **확장성**: 교사 모델을 실행하는 데 드는 비용이 없으므로, 절약된 자원을 더 큰 학생 모델이나 더 큰 이미지 사이즈를 학습시키는 데 투입할 수 있으며, 이 경우 CLIP-Embed-KD가 CLIP-Teacher-KD보다 더 높은 최종 정확도를 달성하였다.
3. **가중치 $\alpha_2$의 영향**: $\alpha_1$과 $\alpha_2$ 모두 0이 아닐 때 최적의 성능을 보였으며, $\alpha_2=1$ (증류 손실만 사용)일 경우 과적합(Overfitting)이 발생하였다.

## 🧠 Insights & Discussion

### 강점

본 연구는 교사 모델의 연산을 '평균 임베딩'이라는 정적인 형태로 추상화함으로써, KD의 고질적인 문제인 계산 비용을 획기적으로 낮췄다. 특히, 단순히 효율성만 높인 것이 아니라, 절약된 메모리 자원을 학생 모델의 용량을 키우는 데 사용하여 결과적으로 더 높은 성능을 낼 수 있다는 '자원 재배분'의 관점을 제시한 점이 인상적이다.

### 한계 및 논의사항

- **정보 손실**: 샘플별 임베딩이 아닌 클래스별 평균 임베딩을 사용함으로써 발생하는 정보 손실이 정확도 하락(약 2%)의 원인으로 분석된다.
- **가정**: 본 논문은 클래스별 평균 임베딩이 해당 클래스의 충분한 대표성을 가진다고 가정하고 있다. 하지만 데이터 분포가 복잡한 경우 단순 평균이 효과적이지 않을 수 있다.
- **미해결 과제**: 저자는 향후 연구에서 단순 평균 외에 더 충실한(faithful) 임베딩 표현 방식을 탐색하고, 수십억 개 이상의 파라미터를 가진 NLP 모델 및 다양한 데이터셋으로 확장할 계획임을 명시하였다.

## 📌 TL;DR

본 논문은 거대 모델의 지식 증류 시 발생하는 막대한 계산 비용을 줄이기 위해, 교사 모델의 실시간 실행 대신 **미리 계산된 클래스별 평균 임베딩**을 활용하는 **CLIP-Embed-KD**를 제안한다. 실험 결과, 이 방법은 CLIP-Teacher-KD 대비 정확도는 미세하게 낮지만, **메모리 사용량을 최대 59배까지 줄이며 학습 시간을 8배 단축**시켰다. 특히 절약된 자원을 더 큰 학생 모델 학습에 투자함으로써 실질적인 성능 향상을 꾀할 수 있음을 입증하여, 향후 초거대 모델의 효율적인 지식 전수에 중요한 가능성을 제시하였다.
