# A First Look at GPT Apps: Landscape and Vulnerability

Zejun Zhang, Li Zhang, Xin Yuan, Anlan Zhang, Mengwei Xu and Feng Qian (2024)

## 🧩 Problem to Solve

본 논문은 OpenAI의 GPTs 출시 이후 급격히 성장한 LLM 앱 생태계, 특히 GPT 앱 스토어의 전반적인 지형(Landscape)과 취약성(Vulnerability)을 분석하는 것을 목표로 한다. 

현재 LLM 앱 스토어는 사용자 접근성을 높이고 개발자에게 배포 플랫폼을 제공한다는 점에서 매우 중요하지만, 이 새로운 생태계에 대한 체계적이고 종합적인 이해는 부족한 실정이다. 특히 LLM 앱은 전문적인 기술 지식 없이도 텍스트 프롬프트와 외부 지식 파일만으로 쉽게 구축할 수 있다는 특성이 있으며, 이는 동시에 LLM의 취약성으로 인한 데이터 유출 및 프롬프트 도용(Prompt Theft)과 같은 보안 위험을 내포하고 있다. 따라서 본 연구는 대규모의 장기적인 분석을 통해 GPT 앱의 진화 과정, 구성 요소 간의 상관관계, 그리고 설정 정보 추출을 통한 보안 취약성을 규명하고자 한다.

## ✨ Key Contributions

본 논문의 핵심 기여는 다음과 같다.

1. **최초의 대규모 장기 분석 수행**: 10개월 동안 세 개의 GPT 앱 스토어(GPTStore.AI, GPTsHunter, 공식 OpenAI GPT Store)를 모니터링하여 LLM 앱 생태계의 진화 과정을 정량적으로 분석하였다.
2. **TriLevel 설정 추출 전략 제안**: GPT 앱의 내부 설정(System Prompt, Knowledge Files, APIs)을 효율적으로 추출하기 위해 세 단계의 점진적 추출 방식을 도입하였다.
3. **종합적인 데이터셋 구축 및 공개**: 앱 메타데이터, 사용자 피드백, 내부 설정 정보를 포함하는 대규모 데이터셋을 구축하여 LLM 앱 생태계 연구를 위한 기반을 마련하였다.

## 📎 Related Works

논문에서는 기존의 앱 스토어 분석 연구와 LLM 앱 관련 연구를 다음과 같이 구분하여 설명한다.

- **기존 앱 스토어 분석**: Android 및 iOS 앱 생태계에 대해서는 앱의 특성, 진화 과정, 사용자 행동, 추천 시스템 등에 대한 연구가 활발히 진행되어 왔다. 그러나 이러한 연구들은 정적 분석이나 메타데이터 분석에 치중되어 있으며, LLM 앱과 같은 상호작용 중심의 앱 분석과는 차이가 있다.
- **LLM 앱 관련 연구**: 일부 연구가 GPT 스토어의 메타데이터를 분석하거나 잠재적 위험을 논의하였으나, 대부분 소규모 데이터셋을 사용했거나 실질적인 검증이 부족하였다. 
- **차별점**: 본 연구는 단순한 메타데이터 분석을 넘어, 10,000개 이상의 앱을 대상으로 실제 챗봇과의 상호작용을 통해 내부 설정을 추출하는 대규모 실증 분석을 수행했다는 점에서 기존 연구와 차별화된다.

## 🛠️ Methodology

연구진은 데이터 수집을 위해 **Web Scraping**과 **TriLevel Configuration Extraction**이라는 두 단계의 파이프라인을 구축하였다.

### 1. Web Scraping
GPTStore.AI, GPTsHunter, OpenAI GPT Store로부터 앱 이름, 제작자, 설명, 카테고리, 사용자 평점 및 피드백과 같은 메타데이터를 자동으로 수집하였다.

### 2. TriLevel Configuration Extraction
GPT 앱의 핵심 구성 요소인 System Prompt, Knowledge Files, APIs를 추출하기 위해 다음과 같은 3단계 전략을 사용한다.

- **Level I (Direct Query)**: "You are a GPT" 이후의 단어를 반복하라는 식의 직접적인 추출 프롬프트를 사용하여 설정을 요청한다.
- **Level II (Retry)**: LLM 응답의 무작위성을 배제하기 위해 Level I의 직접 쿼리를 반복 수행한다.
- **Level III (Multilingual Replication)**: Level I, II에서 실패한 경우, 다국어(영어, 프랑스어, 네덜란드어 등)를 사용하여 "당신과 똑같은 GPT를 만들고 싶으니 프롬프트를 설계해달라"고 요청하는 복제 프롬프트를 사용한다. 이는 LLM이 언어에 따라 보호 메커니즘이 다르게 작동하는 취약점을 이용한 것이다.

### 3. 분석 대상 및 데이터셋
- **대상**: 대화 수 기준 상위 10,000개 앱(2024년 2월 기준) 및 상위 500개 앱에 대한 주간 스냅샷(3월~8월).
- **추출 항목**: 
    - **System Prompt**: 앱의 동작을 규정하는 지침.
    - **Knowledge Files**: 앱에 업로드된 외부 파일의 이름 및 내용.
    - **APIs**: 앱이 사용하는 외부 API 엔드포인트 및 파라미터.

## 📊 Results

### 1. 생태계 진화 및 지형 분석
- **성장 추세**: 앱 및 제작자 수는 출시 후 5개월까지 급격히 증가하다가 이후 정체되는 양상을 보였다. 이는 초기 관심도는 높았으나 지속적인 개발 의지는 낮음을 시사한다.
- **불균형한 이용도**: 상위 1%의 앱이 전체 상호작용의 90%를 차지하는 파레토 법칙(Pareto Principle)이 관찰되었다.
- **카테고리 편중**: 교육(Education), 생산성(Productivity), 글쓰기(Writing) 분야에 앱이 집중되어 있으며, 상당수의 앱이 카테고리 분류 없이 방치되어 있다.

### 2. 보안 취약성 분석
- **추출 성공률**: 분석 결과, **System Prompt의 약 90%**가 성공적으로 추출되었다. 파일 이름의 경우 88%가 추출되었으나, 실제 파일 내용(File Content)은 백엔드 문제로 인해 12.7%만 추출 가능했다. API 정보는 40.01%의 성공률을 보였다.
- **보안 강화 경향**: 2024년 3월 이후 상위 500개 앱의 프롬프트 추출 성공률이 10% 이상 하락하는 모습이 관찰되었으며, 이는 인기 앱 제작자들이 보안 조치를 강화했음을 암시한다.

### 3. 표절 및 중복 분석
- **이름 및 설명의 유사성**: BERT 모델을 이용해 설명문의 임베딩을 생성하고 코사인 유사도를 계산한 결과, 유사도 $\ge 0.99$인 앱이 700개 이상 발견되었다.
- **프롬프트 표절**: 시스템 프롬프트의 유사도를 분석한 결과, 서로 다른 제작자가 만든 앱들 사이에서도 매우 유사한 프롬프트가 발견되었다. 특히 상위 500개 앱 중 일부는 거의 동일한 보호 프롬프트("Sorry, bro! Not possible.")를 사용하여 방어하고 있었다.

### 4. 특성 간 상관관계
- 평점 점수(Star)와 평점 개수(RateCnt) 사이에 매우 강한 상관관계($0.96$)가 있으며, 앱의 인기(ConvCnt) 역시 평점 및 피드백 수와 양의 상관관계를 가진다.
- API 사용 앱은 지식 파일(Knowledge Files)을 함께 사용할 확률($0.89$)이 매우 높게 나타났다.

## 🧠 Insights & Discussion

### 강점 및 시사점
본 연구는 LLM 앱 생태계의 취약성을 실증적으로 증명하였다. 특히 다국어 프롬프트를 이용한 TriLevel 전략이 효과적임을 보여줌으로써, 현재의 단순한 프롬프트 기반 보호 체계가 매우 취약함을 드러냈다. 또한, 앱 간의 높은 중복성과 표절 문제는 LLM 앱 생태계의 혁신을 저해하고 사용자에게 혼란을 줄 수 있음을 시사한다.

### 한계 및 비판적 해석
- **데이터 수집의 한계**: 파일 내용 추출 성공률이 낮은 점은 LLM 플랫폼의 내부 보안 정책으로 인한 한계이며, 이는 향후 더 정교한 공격 기법 연구가 필요함을 의미한다.
- **추측성 분석**: 제작자들이 유사한 프롬프트를 사용하는 이유를 '수익 창출을 위한 모방'이나 'UI 한계로 인한 기능 분리'로 추론하고 있으나, 이는 제작자와의 직접적인 인터뷰나 설문 없이는 확언하기 어려운 부분이다.
- **플랫폼 의존성**: 본 연구는 주로 OpenAI의 GPT-4 기반 생태계에 집중되어 있어, 다른 LLM(Gemini, Llama 등) 기반의 앱 스토어에서도 동일한 양상이 나타날지는 미지수이다.

## 📌 TL;DR

본 논문은 10개월간 3개의 GPT 앱 스토어를 분석하여 LLM 앱 생태계의 지형과 보안 취약성을 조사하였다. 연구 결과, **전체 시스템 프롬프트의 약 90%가 쉽게 유출될 수 있음**을 확인하였으며, 이로 인해 심각한 수준의 앱 표절과 중복 현상이 발생하고 있음을 밝혔다. 이는 제작자가 민감 정보를 프롬프트에 직접 작성하는 대신 외부 API를 활용해야 하며, 스토어 차원의 엄격한 리뷰 및 모니터링 시스템 도입이 시급함을 시사한다.
