# AutoML-GPT: Automatic Machine Learning with GPT

Shujian Zhang, Chengyue Gong, Lemengue Wu, Xingchao Liu, Mingyuan Zhou (2023)

## 🧩 Problem to Solve

현대 인공지능(AI) 분야에서는 특정 작업이나 응용 분야를 위해 수많은 모델이 설계되었으나, 각 작업에 최적화된 모델 아키텍처를 선정하고 최적의 최적화 알고리즘 및 하이퍼파라미터(Hyperparameter)를 찾는 과정에 상당한 인간의 노력이 요구된다. 특히 새로운 데이터셋에 직면했을 때, 전문가가 수동으로 실험을 반복하여 최적의 설정을 찾는 과정은 시간과 자원 소모가 매우 크다.

본 논문의 목표는 대규모 언어 모델(LLM), 특히 GPT-4의 추론 및 상호작용 능력을 활용하여 데이터 전처리부터 모델 설계, 하이퍼파라미터 튜닝에 이르는 머신러닝 훈련 파이프라인을 자동화하는 AutoML-GPT 시스템을 구축하는 것이다. 이를 통해 사용자의 요청과 데이터/모델의 설명만으로 최적의 AI 모델을 훈련시킬 수 있는 범용적인 자동화 프레임워크를 제공하고자 한다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 언어를 범용 인터페이스로 활용하여 LLM이 다양한 전문가 모델(Expert Models)과 데이터셋을 연결하는 컨트롤러 역할을 수행하게 하는 것이다. 이를 위해 다음과 같은 설계를 제안한다.

1.  **정형화된 프롬프트 구조(Fixed-format Prompt):** 모델의 상세 정보가 담긴 Model Card와 데이터셋의 특성이 기술된 Data Card를 결합하여 LLM에 입력함으로써, LLM이 해당 작업에 가장 적합한 모델과 설정을 선택할 수 있도록 유도한다.
2.  **예측 훈련 로그(Predicted Training Log) 기반 튜닝:** 실제 하드웨어에서 모든 조합을 실험하는 대신, LLM이 예상되는 훈련 로그를 생성하게 함으로써 하이퍼파라미터를 효율적으로 튜닝하고 제안한다.
3.  **미학습 데이터셋으로의 전이:** 새로운 데이터셋(Unseen Dataset)의 경우, 텍스트 인코더(예: CLIP)를 통해 기존 데이터셋들과의 유사도를 측정하고, 가장 유사한 데이터셋의 하이퍼파라미터 설정을 전이(Transfer)하여 추천하는 메커니즘을 도입한다.

## 📎 Related Works

논문은 크게 세 가지 관련 연구 분야를 언급한다.

1.  **Advanced LLM:** GPT-4, PaLM 등 매개변수가 수천 억 개에 달하는 모델들이 Zero-shot 및 Few-shot 학습에서 뛰어난 성능을 보이며 새로운 창발적 능력(Emergent abilities)을 나타내고 있음을 설명한다.
2.  **Chain of Thought (CoT):** 모델이 최종 정답을 내놓기 전 중간 추론 단계를 생성하도록 유도하는 CoT 프롬프팅 기법을 언급하며, LLM의 복잡한 추론 능력을 향상시키는 방법론들을 소개한다.
3.  **GPT-based Systems:** HuggingGPT나 VisualGPT와 같이 LLM을 에이전트로 사용하여 외부 도구나 모델을 호출하는 시스템들이 존재한다.

**기존 연구와의 차별점:** 기존의 GPT 기반 시스템들이 주로 추론(Inference) 단계에서 모델을 조합하여 결과를 내는 데 집중했다면, AutoML-GPT는 모델의 '훈련(Training) 파이프라인' 전체를 자동화한다는 점에서 차별성을 가진다. 즉, 데이터 전처리 $\rightarrow$ 아키텍처 설계 $\rightarrow$ 하이퍼파라미터 튜닝 $\rightarrow$ 로그 분석으로 이어지는 전체 학습 사이클을 구축하는 것에 초점을 맞춘다.

## 🛠️ Methodology

AutoML-GPT는 LLM을 컨트롤러로, 다양한 전문가 모델들을 실행기(Executor)로 사용하는 협업 시스템이다. 전체 파이프라인은 다음 네 가지 단계로 구성된다.

### 1. 입력 분해 (Input Decomposition)
LLM이 정확한 판단을 내릴 수 있도록 세 가지 구성 요소로 이루어진 프롬프트를 구성한다.
- **Data Card:** 데이터셋 이름, 입력 데이터 타입(이미지, 텍스트 등), 레이블 공간(클래스 종류, 해상도 등), 기본 평가 지표를 포함한다.
- **Model Card:** 모델 이름, 모델 구조(예: Swin Transformer + UperNet head), 모델 상세 설명, 아키텍처 하이퍼파라미터를 포함한다.
- **평가 지표 및 추가 요청:** 사용자가 원하는 특정 제약 조건(예: "추론 속도 10 FPS 미만")이나 추가 지표를 입력한다.

### 2. 데이터 전처리 (Data Processing)
입력된 Data Card를 바탕으로 LLM은 해당 데이터의 특성에 맞는 전처리 기법을 결정한다. 컴퓨터 비전의 경우 리사이징, 정규화, 증강(Augmentation) 등이 포함되며, NLP의 경우 토큰화, 불용어 제거 등이 포함된다. LLM은 이를 수행하기 위한 구체적인 파이썬 스크립트를 생성한다.

### 3. 모델 아키텍처 설계 (Model Architecture)
제공된 Model Card들의 집합 중에서 현재 작업에 가장 적합한 모델을 동적으로 할당한다. 이는 In-context task-model assignment 메커니즘을 통해 이루어지며, LLM은 모델의 구조적 특징(입출력 레이어, 활성화 함수, 손실 함수 등)을 분석하여 최적의 모델을 선택한다.

### 4. 예측 로그 기반 하이퍼파라미터 튜닝 (Hyperparameter Tuning with Predicted Training Log)
실제 훈련 전, LLM은 선택된 모델과 데이터셋에 대해 특정 하이퍼파라미터 설정 시 예상되는 훈련 로그(에폭별 손실 값, 정확도 등)를 생성한다. 사용자는 이 예측 로그를 확인하고 LLM과 상호작용하며 학습률(Learning Rate), 배치 크기(Batch Size) 등의 하이퍼파라미터를 수정한다.

**미학습 데이터셋(Unseen Datasets) 처리 방법:**
새로운 데이터셋 $\mathcal{D}_{new}$가 주어지면, 텍스트 인코더 $f_{enc}$를 사용하여 Data Card의 임베딩 벡터를 생성하고 기존 데이터셋 $\mathcal{D}_{old}$와의 코사인 유사도를 계산한다.
$$\text{Similarity} = \cos(f_{enc}(\text{Card}_{new}), f_{enc}(\text{Card}_{old}))$$
이 유사도 점수를 바탕으로 가장 관련성이 높은 기존 데이터셋의 하이퍼파라미터 설정을 추천한다.

## 📊 Results

본 연구는 GPT-4를 사용하여 다양한 도메인에서 실험을 진행하였다.

### 1. 미학습 데이터셋 (Unseen Dataset)
- **설정:** ImageNet 데이터셋을 분할하여 일부는 훈련 데이터(Data Card 생성용)로, 일부는 미학습 데이터로 사용하였다. ViT base 모델을 사용하였다.
- **결과:** AutoML-GPT가 추천한 하이퍼파라미터 설정은 Top-1 예측 정확도에서 $98\%$를 달성하였으며, 이는 랜덤하게 선택된 하이퍼파라미터의 평균 정확도인 $80\%$보다 훨씬 높은 수치이다. 또한 추가 데이터 없이 GPT-4의 제안만으로 학습했을 때도 $82\%$의 정확도를 기록하여 유효함을 입증하였다.

### 2. 객체 탐지 (Object Detection)
- **설정:** COCO 데이터셋을 대상으로 DETR 모델과 Swin Transformer 백본을 사용하였다.
- **결과:** LLM이 데이터 전처리를 위한 파이썬 스크립트를 정확히 생성하였으며, 학습률 $10^{-4}$, Weight Decay $10^{-4}$와 같은 구체적인 하이퍼파라미터를 제안하였다. 이는 실제 전문가의 설정과 유사한 수준임을 보였다.

### 3. 질의응답 (Question Answering)
- **설정:** Natural Questions(NQ) 데이터셋과 Dense Passage Retrieval(DPR) 모델을 사용하였다.
- **결과:** 제안된 학습률 $10^{-5}$와 최대 에폭 40회 설정이 실제 DPR 논문의 설정과 매우 밀접하게 일치하였다. 특히 "추론 속도 향상"이라는 추가 요청에 대해 모델 차원을 768에서 256으로 줄이고 시퀀스 길이를 128로 제한하라는 구체적인 최적화 가이드를 제공하였다.

### 4. 분류 (Classification)
- **설정:** UCI Adult 데이터셋과 XGBoost 모델을 사용하였다.
- **결과:** AutoML-GPT가 제안한 하이퍼파라미터(learning_rate: 0.3, max_depth: 6 등)를 적용하여 학습시킨 결과, 최종 검증 손실 $0.277$, 정확도 $85.92\%$를 달성하였다.

## 🧠 Insights & Discussion

**강점:**
AutoML-GPT는 특정 도메인에 국한되지 않고 컴퓨터 비전, NLP, 정형 데이터 분류 등 다양한 작업에 범용적으로 적용 가능하다는 점이 가장 큰 강점이다. 특히 Model Card와 Data Card라는 정형화된 인터페이스를 통해 LLM의 Hallucination을 줄이고, 전문가의 지식을 체계적으로 주입했다는 점이 긍정적이다. 또한 '예측 훈련 로그' 개념을 통해 실제 계산 자원을 소모하기 전 전략을 세울 수 있게 한 점이 효율적이다.

**한계 및 비판적 해석:**
1.  **데이터/모델 카드의 의존성:** 시스템의 성능이 입력으로 제공되는 Model Card와 Data Card의 품질에 완전히 의존한다. 만약 정교하게 작성된 카드가 없다면 LLM의 성능이 급격히 저하될 가능성이 크다.
2.  **예측 로그의 신뢰성:** LLM이 생성한 '예측 훈련 로그'가 실제 훈련 결과와 얼마나 일치하는지에 대한 정량적 분석이 부족하다. LLM이 단순히 '그럴듯한' 로그를 생성하는 것인지, 아니면 실제 모델의 거동을 정확히 시뮬레이션하는 것인지에 대한 검증이 더 필요하다.
3.  **실질적 자동화 범위:** 본 논문은 LLM이 설정값을 '제안'하고 스크립트를 '작성'하는 것에 집중하고 있다. 실제로 이 스크립트가 오류 없이 실행되고, 실행 결과가 다시 LLM으로 피드백되어 자동으로 수정되는 Closed-loop 시스템의 완전한 구현 단계까지는 나아가지 않은 것으로 보인다.

## 📌 TL;DR

AutoML-GPT는 LLM(GPT-4)을 컨트롤러로 활용하여 데이터 전처리, 모델 선정, 하이퍼파라미터 튜닝까지의 ML 훈련 과정을 자동화하는 시스템이다. Model Card와 Data Card를 프롬프트로 사용하여 범용성을 확보하였으며, 특히 미학습 데이터셋에 대해 유사도 기반의 하이퍼파라미터 전이 학습을 제안한다. 비전, NLP, 분류 작업 전반에서 효과적인 설정값을 제안함으로써 ML 실험의 효율성을 극대화할 수 있는 가능성을 보여주었으며, 향후 LLM 기반의 자동화된 AI 개발 인터페이스 구축에 중요한 기여를 할 것으로 평가된다.
