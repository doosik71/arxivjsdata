# RefGPT: Dialogue Generation of GPT, by GPT, and for GPT

Dongjie Yang, Ruifeng Yuan, Yuantao Fan, Yifei Yang, Zili Wang, Shusen Wang, Hai Zhao (2023)

## 🧩 Problem to Solve

최근 대규모 언어 모델(Large Language Models, LLMs)은 고품질의 지시어 데이터(Instruction data)를 통한 미세 조정(Fine-tuning)을 통해 다양한 NLP 태스크에서 뛰어난 성능을 보여주고 있다. 특히 다회차 대화(Multi-turn dialogues) 데이터는 모델의 대화 능력을 향상시키는 데 필수적이지만, 전문 지식을 갖춘 인력을 통해 고품질의 데이터를 수집하는 것은 비용이 매우 높고 현실적으로 어렵다.

이를 해결하기 위해 기존 연구들은 LLM을 이용해 자동으로 대화 데이터를 생성하는 방식을 채택하였다. 그러나 이러한 방식은 모델의 환각(Hallucination) 현상으로 인해 사실과 다른 거짓 정보를 생성하는 Truthfulness 문제가 빈번하게 발생한다. 또한, 생성되는 대화의 구조, 스타일, 내용에 대한 세밀한 제어가 불가능하여 데이터의 다양성과 품질을 확보하는 데 한계가 있다.

본 논문의 목표는 모델의 환각 문제를 해결하여 사실 관계가 정확(Truthful)하며, 사용자가 세밀하게 제어(Customized)할 수 있는 다회차 대화 데이터를 대량으로 생성하는 방법론인 RefGPT를 제안하는 것이다.

## ✨ Key Contributions

RefGPT의 핵심 아이디어는 LLM이 자신의 내부 지식에 의존하여 답변을 생성하게 두는 것이 아니라, 외부에서 제공된 신뢰할 수 있는 참조 문서(Reference)를 기반으로 대화를 구성하도록 강제하는 것이다. 이를 통해 모델이 근거 없는 내용을 지어내는 환각 현상을 최소화하고 사실성을 극대화한다.

또한, 단순히 데이터를 생성하는 것에 그치지 않고, 대화의 구조(Structure), 스타일(Style), 내용(Content)을 세부적으로 설정할 수 있는 로컬 커스터마이징(Local Customization) 프롬프팅 프로세스를 도입하여, 학습에 용이한 정형화된 포맷의 고품질 데이터를 생성할 수 있게 하였다.

## 📎 Related Works

### LLM 기반 대화 생성
Self-Instruct, Baize, UltraChat 등의 연구는 LLM을 활용해 지시어 데이터나 다회차 대화를 자동으로 생성하는 프레임워크를 제시하였다. 이러한 방식은 데이터 수집 비용을 획기적으로 낮추었으나, 모델이 학습 데이터 내의 편향이나 잘못된 지식을 바탕으로 답변을 생성하는 환각 문제에 취약하며, 생성 결과물에 대한 제어 능력이 부족하다는 한계가 있다.

### 참조 기반 대화 생성
특정 문서에서 QA 쌍을 생성하거나 Wikipedia 문서를 대화로 변환하는 Dialogue Inpainting과 같은 연구들이 존재한다. RefGPT는 이러한 참조 기반 생성 방식의 아이디어를 계승하되, 최신 LLM의 강력한 프롬프트 준수 능력을 활용하여 더 복잡한 다회차 대화와 세밀한 커스터마이징을 가능하게 함으로써 기존 접근 방식과 차별화된다.

## 🛠️ Methodology

RefGPT의 전체 파이프라인은 크게 **참조 선택(Reference Selection)**, **기본 프롬프트(Basic Prompt)**, **대화 설정(Dialogue Settings)**의 세 단계로 구성된다.

### 1. 참조 선택 (Reference Selection)
생성되는 대화의 진실성은 참조 문서의 품질에 직접적으로 의존한다. 따라서 Wikipedia와 같은 권위 있는 지식 기반 웹사이트나 Github의 실제 코드 저장소와 같이 신뢰도가 높은 외부 문서를 참조 데이터로 선택한다. 이는 모델이 내부 지식을 회상(Reciting)하는 대신 제공된 컨텍스트를 활용하게 함으로써 환각을 방지하는 기초가 된다.

### 2. 기본 프롬프트 (Basic Prompt)
모델이 참조 문서를 효과적으로 활용하도록 다음과 같은 기본 지침을 제공한다.
- 제공된 참조 문서만을 기반으로 다회차 대화를 생성할 것.
- 생성 언어를 명시하며, 참조 문서의 언어와 일치하도록 설정한다.
- 부적절하거나 불법적인 요청에 대해서는 거절하고 적절한 조언을 제공하도록 하여 인간의 선호도(Human preference)에 부합하게 한다.
- 시스템 역할(System role) 프롬프트를 통해 챗봇의 정체성(Persona)을 설정한다.

### 3. 대화 설정 및 로컬 커스터마이징 (Dialogue Settings & Local Customization)
대화의 다양성과 정밀한 제어를 위해 다음과 같은 세부 설정을 적용한다.

#### 🛠️ 대화 구조 (Dialogue Structure)
대화의 시작과 끝을 `<chat>` 및 `</chat>` 태그로 명시하고, 각 발화자를 `<user>`와 `<assistant>`로 구분한다.
- **턴 수 제어**: LLM이 숫자를 정확히 세지 못하는 문제를 해결하기 위해, 원하는 턴 수만큼 `<user i>`와 `<assistant i>` 형태의 플레이스홀더를 명시적으로 제공하여 모델이 이를 채우도록 유도한다.
- **발화 길이 제어**: 각 턴의 발화 전에 `(word count: x words)`와 같은 길이 요구사항을 명시한다. 단, 환각을 방지하기 위해 참조 문서의 길이가 요구되는 전체 대화 길이의 80%보다 짧은 경우 해당 참조를 필터링한다.

#### 🎨 대화 스타일 및 내용 (Dialogue Style & Content)
- **스타일(Style)**: '어린아이 같은 말투로 질문' 또는 '어린이가 이해하기 쉽게 답변'과 같은 스타일 풀(Style pool)에서 샘플링하여 각 발화에 적용한다.
- **내용(Content)**: 태스크(사실 지식, 코드 등)에 따라 질문의 관점(예: 코드의 버그 수정, 실제 사례 적용 등)을 정의한 내용 풀(Content pool)에서 샘플링한다.

이 모든 설정은 최종적으로 **대화 템플릿(Dialogue Template)**으로 통합되어 LLM에 입력되며, 결과적으로 하나의 API 호출만으로 구조와 내용이 제어된 전체 다회차 대화가 생성된다.

## 📊 Results

### 생성 데이터셋
RefGPT를 통해 GPT-4를 이용하여 두 가지 대규모 데이터셋을 구축하였다.
- **RefGPT-Fact**: Wikipedia 및 Baidu Baike를 참조로 하여 생성한 100k 규모의 사실 기반 다회차 대화 데이터셋 (영어 50k, 중국어 50k).
- **RefGPT-Code**: Github 저장소를 참조로 하여 생성한 76k 규모의 프로그래밍 대화 데이터셋 (영어 37k, 중국어 39k). 이는 코드 논의(Discussion), 코드 생성(Creation), 버그 수정(Bug fixing)의 세 가지 시나리오를 포함한다.

### 정량적 평가 (Truthfulness)
Self-Instruct 및 Baize와 같은 기존 자동 생성 방식과 진실성(Truthfulness)을 비교 평가하였다. 평가는 인간 평가와 GPT-4를 이용한 자동 평가(참조 문서 제공 방식)로 진행되었다.
- **결과**: RefGPT는 GPT-4 평가 기준 **97.5%**의 정확도를 기록하며, Self-Instruct(50.2%)와 Baize(47.2%) 대비 압도적인 진실성을 보여주었다.

### 추가 분석 (Robustness & Control)
- **참조 길이 및 노이즈 영향**: 참조 문서의 길이를 25%까지 줄이거나, 20%의 HTML 노이즈를 추가하더라도 진실성 점수가 크게 떨어지지 않아 강건함(Robustness)을 입증하였다.
- **템플릿 준수율**: 생성해야 할 단어 수가 많아지거나 턴 수가 증가할수록 템플릿 준수율(Success rate)이 낮아지는 경향이 확인되었다. 특히 종료 태그 `</chat>`가 정확히 생성될 때 턴 수 제어 성공률이 높았다.

## 🧠 Insights & Discussion

### 강점
본 연구는 LLM의 고질적인 문제인 환각 현상을 '외부 참조 강제'라는 단순하지만 강력한 제약 조건을 통해 효과적으로 해결하였다. 또한, API 호출 횟수를 최소화하면서도(대화당 1회) 발화 길이, 턴 수, 스타일을 세밀하게 조정할 수 있는 프롬프팅 전략을 제시하였다는 점이 큰 성과이다.

### 한계 및 비판적 해석
논문에서 명시했듯, RefGPT는 참조 문서에 전적으로 의존하므로 **참조 문서 자체에 오류가 있을 경우** 이를 그대로 생성하게 된다. 즉, 생성 데이터의 상한선(Upper bound)은 참조 문서의 정확도에 종속된다. 또한, 참조 문서가 가진 편향(Bias)이 생성된 대화에 그대로 반영될 위험이 있다.

학술적인 관점에서 볼 때, 본 방법론은 LLM의 '추론 능력'보다는 '문맥 준수 능력(Context following)'을 극대화하여 데이터를 생성하는 방식이다. 따라서 이는 새로운 지식을 창조하는 것이 아니라 기존 지식을 대화 형태로 재구성하는 것에 가깝다. 하지만 지시어 튜닝(Instruction tuning)을 위한 데이터셋 구축이라는 목적하에서는 매우 효율적이고 실용적인 접근법이라고 판단된다.

## 📌 TL;DR

RefGPT는 LLM이 자신의 내부 지식이 아닌 **신뢰할 수 있는 외부 참조 문서(Reference)만을 활용**하게 하여, 환각 현상이 제거된 **진실성 높은 다회차 대화 데이터를 생성**하는 방법론이다. 이를 통해 사실 기반의 `RefGPT-Fact`와 코드 기반의 `RefGPT-Code` 데이터셋을 구축하였으며, 기존 자동 생성 방식 대비 월등한 진실성(97.5%)을 확보하였다. 이 연구는 특정 도메인의 지식 베이스만 있다면 누구나 해당 분야의 전문 챗봇 학습을 위한 고품질 대화 데이터를 구축할 수 있는 가능성을 제시한다.
