# Beyond Generating Code: Evaluating GPT on a Data Visualization Course

Chen Zhu-Tian, Chenyang Zhang, Qianwen Wang, Jakob Troidl, Simon Warchol, Johanna Beyer, Nils Gehlenborg, Hanspeter Pfister (2024)

## 🧩 Problem to Solve

본 논문은 데이터 시각화 분야에서 거대 언어 모델(LLM)인 GPT의 능력을 단순히 시각화 코드 생성 능력에 한정 짓지 않고, 보다 포괄적인 시각화 작업 수행 능력 관점에서 평가하고자 한다. 기존의 많은 연구들이 자연어 쿼리를 시각화 코드로 변환하는 능력에 집중해 왔으나, 실제 데이터 시각화 프로세스는 데이터 정제, 시각화 설계, 시각적 데이터 탐색, 그리고 통찰(insight)의 전달이라는 복잡한 파이프라인으로 구성된다.

따라서 본 연구의 목표는 하버드 대학교의 데이터 시각화 입문 과정인 CS171 강의의 과제와 시험을 벤치마크로 활용하여, GPT-3.5와 GPT-4가 데이터 해석, 시각화 설계, 상호작용 탐색 및 통찰 전달과 같은 다양한 시각화 작업에서 어느 정도의 숙련도를 보이는지 정량적 및 정성적으로 분석하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 LLM의 시각화 능력을 평가하기 위해 실제 대학 교육 과정의 커리큘럼을 평가 프레임워크로 도입했다는 점이다. 특히 다음과 같은 설계 아이디어를 통해 평가 범위를 확장하였다.

1. **코드 생성을 넘어서는 평가 범위**: 단순히 차트를 그리는 코드를 짜는 것을 넘어, SVG 형식을 통해 시각화 결과물을 '읽고' 해석하며, JavaScript 이벤트를 발생시켜 시각화 도구와 '상호작용'하고, 대상 청중에 맞게 통찰을 '전달'하는 능력을 테스트하였다.
2. **블라인드 테스트를 통한 인간-AI 구분 능력 분석**: GPT가 생성한 결과물을 실제 학생들의 제출물과 섞어 숙련된 조교(Teaching Fellow, TF)들이 평가하게 함으로써, AI 생성물의 특징과 한계를 정성적으로 도출하였다.
3. **시각화 파이프라인의 end-to-end 검증**: 데이터 수집 및 정제부터 최종 스토리텔링 웹페이지(scrollytelling) 제작까지의 전 과정을 탐색적으로 수행하여 GPT-4의 잠재적 활용 가능성을 확인하였다.

## 📎 Related Works

최근 GPT-3, GPT-4와 같은 모델들이 법률 시험(Bar exam)이나 의학 시험(USMLE) 등 전문 분야에서 뛰어난 성능을 보임에 따라, 이를 시각화 분야에 적용하려는 시도가 증가하고 있다. LIDA, Chat2VIS, VizGPT, ChartGPT 등 기존 연구들은 주로 사용자의 자연어 설명을 바탕으로 시각화 코드를 자동 생성하는 기능에 초점을 맞추어 왔다.

그러나 이러한 기존 접근 방식은 시각화의 '생성' 단계에만 치우쳐 있으며, 생성된 시각화 결과물을 다시 분석하거나, 복잡한 설계 원칙을 적용하고, 사용자에게 통찰을 효과적으로 전달하는 '해석 및 커뮤니케이션' 단계에 대한 연구는 부족했다는 점이 본 논문과 기존 연구의 차별점이다.

## 🛠️ Methodology

본 연구는 하버드 대학교의 CS171 강의 구성 요소를 활용하여 GPT-3.5와 GPT-4의 성능을 Zero-shot 방식으로 평가하였다.

### 1. 평가 구성 요소
- **Quizzes (객관식)**: 설계 원칙, 웹 프로그래밍 등 시각화 지식을 묻는 91개의 문항으로 구성된다. 각 문항을 10번 반복 수행하여 평균 점수를 산출하였다.
- **Homework (주관식/코딩)**: 에세이 작성 및 상호작용 시각화 구현 과제이다. 토큰 길이 제한을 고려하여 단계별 지침을 순차적으로 입력하였으며, 실행 에러 발생 시 최대 30회까지 재시도하였다.
- **Final Project (탐색적 연구)**: 팀 프로젝트 성격상 직접 평가 대신, 데이터 정제 $\rightarrow$ 설계 $\rightarrow$ SVG 읽기 $\rightarrow$ JS 이벤트 제어 $\rightarrow$ 통찰 전달로 이어지는 개별 태스크를 수행하는 오픈엔드(open-ended) 실험을 진행하였다.

### 2. 평가 및 채점 절차
- **블라인드 채점**: 세 명의 TF가 GPT 생성물과 과거 학생 제출물을 구분하지 못한 채 기존 루브릭(rubric)에 따라 채점하였다.
- **구분 지표**: TF들은 5점 척도(Likert scale)를 통해 해당 과제가 인간에 의해 작성되었는지 AI에 의해 작성되었는지 판단하였다.

### 3. 상호작용 탐색 방법론
GPT가 시각화 결과물과 상호작용하도록 하기 위해 다음과 같은 절차를 사용하였다.
- 시각화의 SVG 내용과 JavaScript 코드를 GPT에 제공한다.
- GPT가 특정 동작(예: 드롭다운 메뉴 변경, 줌 인/아웃)을 수행하기 위한 JavaScript 이벤트 디스패칭(event dispatching) 코드를 생성하게 한다.
- 실행 결과로 생성된 새로운 SVG를 다시 GPT에게 입력하여 시각적 변화에 따른 통찰을 도출하게 한다.

## 📊 Results

### 1. 정량적 결과
- **Quiz 점수**: GPT-4가 $86.4\%$로 GPT-3.5($68.8\%$)보다 우수한 성적을 거두었다.
- **Homework 점수**: GPT-4는 $79.7\%$, GPT-3.5는 $75.2\%$를 기록하였다.
- **인간-AI 구분 정확도**: TF들은 약 $70\%$의 정확도로 GPT 생성물을 구분해 냈으며, 이는 일반 텍스트 생성물보다 구분하기 쉬웠음을 시사한다.

### 2. 정성적 결과 및 한계
TF들이 GPT 생성물을 구분해 낸 주요 휴리스틱(heuristics)은 다음과 같다.
- **창의성 부족**: 결과물이 너무 일반적(generic)이며, 학생들의 제출물에서 보이는 독특한 색상 선택이나 세심한 애니메이션 효과가 부족하다.
- **비전형적인 라이브러리 사용**: D3.js와 vanilla JS를 섞어 쓰는 방식이 부자연스러운 경우가 많았다.
- **시각적 해석 오류**: 이미지 입력이 불가능한 모델 특성상, 존재하지 않는 차트를 언급하거나(Hallucination), 요소의 위치를 잘못 배치(예: 지도가 화면 밖으로 나감)하는 경향이 있었다.

### 3. 탐색적 연구 결과 (GPT-4)
- **데이터 정제**: Python(pandas, numpy)을 이용한 데이터 클리닝 및 병합 작업을 성공적으로 수행하였다.
- **SVG 읽기 및 상호작용**: 이미지 파일이 아닌 SVG 텍스트 형식을 통해 차트 타입, 마크 모양, 인코딩 등을 정확히 파악하였으며, JS 이벤트를 발생시켜 차트의 축을 바꾸거나 줌을 조절하는 코드를 생성할 수 있었다.
- **통찰 전달**: 동일한 데이터 통찰이라도 대상(UN 관계자 vs 초등학생)에 따라 톤앤매너를 다르게 하여 보고서를 작성하거나, D3.js와 Scrollama를 이용한 스크롤리텔링(scrollytelling) 페이지를 구현하였다.

## 🧠 Insights & Discussion

### 강점 및 잠재력
GPT는 시각화 설계 원칙에 대한 방대한 지식을 보유하고 있으며, 특히 자연어 입력을 통해 복잡한 시각화 파이프라인을 자동화할 수 있는 잠재력을 보여주었다. 특히 SVG 형식을 통해 시각적 구조를 이해하고 JS 이벤트를 통해 인터랙션을 모사하는 능력은 LLM이 단순한 코드 생성기를 넘어 '시각화 전문가(Visualization Expert)'로서 보조 도구로 활용될 수 있음을 시사한다.

### 한계 및 비판적 해석
- **컨텍스트 유지의 어려움**: 여러 코드 파일(HTML, JS, CSS) 간의 관계를 추론하는 능력이 부족하여, 파일 간 라이브러리 버전이 불일치하는 문제가 발생하였다.
- **시각적 환각(Hallucination)**: 이미지 입력을 처리하지 못하는 상태에서 URL만 제공되었을 때, 실제 이미지를 보지 않고도 내용을 '상상'하여 비평하는 심각한 환각 현상이 관찰되었다.
- **토큰 제한**: 대규모 프로젝트의 경우 토큰 길이 제한으로 인해 전체 코드를 한 번에 처리하지 못하는 물리적 한계가 존재한다.

### 교육적 시사점
본 연구는 LLM이 교육 과정의 과제를 쉽게 해결할 수 있음을 보여줌으로써, 기존의 시각화 교육 방식에 대한 재고를 촉구한다. 단순 구현보다는 인간의 판단과 미묘한 해석이 필요한 영역(예: 스케칭, 수업 내 특수 정보 활용, 팀 프로젝트)에 집중하거나, AI 도구를 커리큘럼에 통합하여 AI와 협업하는 능력을 가르치는 방향으로의 전환이 필요하다.

## 📌 TL;DR

본 논문은 하버드 대학교의 CS171 시각화 코스를 벤치마크로 하여 GPT-3.5와 GPT-4의 능력을 평가하였다. 실험 결과, GPT-4는 퀴즈와 과제에서 높은 성적을 거두었으며, 특히 코드 생성을 넘어 SVG 해석, JS 이벤트 기반의 상호작용 탐색, 대상 맞춤형 통찰 전달 등 광범위한 시각화 태스크를 수행할 수 있음을 입증하였다. 다만, 창의성 부족과 시각적 환각, 파일 간 관계 추론 능력 부족이라는 한계가 명확히 드러났다. 이 연구는 LLM이 시각화 전문가로서의 가능성을 가졌음을 보여주는 동시에, AI 시대에 맞는 시각화 교육의 재설계가 시급함을 시사한다.
