# ALoRA: Allocating Low-Rank Adaptation for Fine-tuning Large Language Models

Zequan Liu, Jiawen Lyn, Wei Zhu, Xing Tian, Yvette Graham (2024)

## 🧩 Problem to Solve

본 논문은 대규모 언어 모델(LLM)의 효율적인 미세 조정(Fine-tuning)을 위한 Parameter-Efficient Fine-Tuning(PEFT) 기법 중 하나인 LoRA(Low-Rank Adaptation)의 한계를 해결하고자 한다.

LoRA는 모델의 가중치 변화가 낮은 내재적 차원(Intrinsic Dimension)을 가진다는 가정을 바탕으로 저차원 행렬 분해를 통해 파라미터를 최적화한다. 그러나 기존의 LoRA는 모든 Transformer 모듈에 대해 **고정된 랭크(Fixed Rank)**를 적용한다. 이는 다음과 같은 두 가지 핵심 문제를 야기한다.

1. **비효율적인 자원 배분**: 다운스트림 태스크에 따라 각 모듈(Query, Key, Value, FFN 등)이 필요로 하는 최적의 랭크가 다를 수 있음에도 불구하고, 동일한 랭크를 적용함으로써 일부 모듈에서는 자원이 낭비되고 일부에서는 부족한 현상이 발생한다.
2. **최적 랭크 결정의 어려움**: 각 가중치 행렬에 대해 최적의 랭크 $r$을 수동으로 결정하는 것은 매우 어려우며, 과도하게 큰 랭크는 계산 비용을 증가시키고 너무 작은 랭크는 성능 저하를 초래한다.

따라서 본 연구의 목표는 학습 과정에서 각 모듈의 중요도에 따라 LoRA 랭크를 동적으로 조정하고 재배치하는 **ALoRA(Allocating LoRA)** 프레임워크를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 모든 모듈에 동일한 랭크를 초기화한 후, **중요도가 낮은 랭크를 제거(Pruning)하고 그 여유분(Budget)을 중요도가 높은 모듈에 재할당(Allocation)**하는 동적 메커니즘을 구축하는 것이다.

이를 위해 다음과 같은 핵심 기여를 제시한다.

- **AB-LoRA (Ablation-Based LoRA)**: 각 LoRA 랭크의 중요도 점수를 정확하게 측정하기 위해 제안된 새로운 방법론이다.
- **동적 랭크 재배치 프레임워크**: AB-LoRA를 통해 측정된 점수를 기반으로 랭크를 점진적으로 가지치기하고, 이를 필요한 모듈에 다시 할당하여 모델의 표현력을 최적화한다.
- **자원 효율성**: 초기부터 매우 큰 랭크를 설정하고 깎아내는 기존 방식(AdaLoRA 등)과 달리, 목표 파라미터 예산 내에서 시작하여 재배치하므로 GPU 메모리 소비를 줄인다.

## 📎 Related Works

### 기존 PEFT 및 LoRA 변형 연구

- **Adapter-based/Prompt-based Tuning**: 모델 내부에 작은 모듈을 삽입하거나 입력 프롬프트를 최적화하는 방식이다.
- **LoRA 및 변형 모델**: LoRA는 가중치 업데이트를 저차원 행렬의 곱으로 표현한다. 이후 **AdaLoRA**, **SoRA**, **SaLoRA**와 같은 연구들이 가변적인 랭크를 탐색하려는 시도를 하였다.

### 기존 접근 방식과의 차별점

기존의 가변 랭크 LoRA 방식들은 일반적으로 다음과 같은 한계가 있다.

- **초기 오버헤드**: 대부분의 기존 연구는 처음에 매우 큰 랭크($r_{max}$)를 설정한 뒤 휴리스틱한 방법으로 가지치기를 수행한다. 이는 학습 초기 단계에서 불필요하게 많은 GPU 메모리를 점유한다.
- **신뢰할 수 없는 중요도 지표**: 기존 방식들이 사용하는 중요도 점수(Sensitivity-based 등)가 실제 각 랭크의 기여도를 정확히 반영하지 못할 수 있다.

ALoRA는 목표 예산($R_{target}$) 내에서 시작하여 랭크를 이동시키는 방식을 취하며, Ablation 기반의 새로운 지표를 통해 더 정확한 중요도를 측정함으로써 차별성을 갖는다.

## 🛠️ Methodology

### 1. 전체 구조 및 수식화

ALoRA는 Transformer의 각 가중치 행렬 $W$에 대해 다음과 같은 구조로 업데이트를 수행한다.

$$z = x W_A^m G_m W_B^m$$

여기서 $W_A^m \in \mathbb{R}^{d \times r_m}$와 $W_B^m \in \mathbb{R}^{r_m \times d}$는 LoRA의 저차원 행렬이며, $G_m$은 각 랭크의 활성화 여부를 결정하는 대각 행렬(Diagonal Matrix)이다.
$$G_m = \text{diag}(\alpha_{m,1}, \dots, \alpha_{m,r_m})$$
게이트 유닛 $\alpha_{m,i} \in \{0, 1\}$는 해당 랭크의 사용 여부를 결정한다. 미분 가능성을 위해 $\alpha$는 Sigmoid 함수를 통해 $(0, 1)$ 사이의 연속적인 값으로 완화되어 학습된다.

### 2. AB-LoRA: 중요도 측정 방법

본 논문의 핵심인 **AB-LoRA**는 특정 랭크 $r$이 모델 성능에 미치는 영향력을 측정하기 위해 '제거 시의 성능 저하'와 '단독 사용 시의 성능'을 모두 고려한다.

중요도 점수 $IS(r)$는 다음과 같이 정의된다.
$$IS(r) = S(M) - S(M \setminus r) + S(M_r)$$

- $S(M)$: 전체 슈퍼 네트워크(Super-network)의 성능 점수 (여기서는 $-\text{Cross-Entropy Loss}$ 사용).
- $S(M \setminus r)$: 해당 랭크 $r$만 제외하고 나머지 모든 랭크를 유지했을 때의 성능.
- $S(M_r)$: 해당 랭크 $r$만 유지하고 나머지 모든 랭크를 제거했을 때의 성능.

직관적으로, 랭크 $r$을 제거했을 때 성능이 크게 떨어진다면($S(M) - S(M \setminus r)$ 값이 크다면) 해당 랭크는 매우 중요한 것이며, 단독으로 존재할 때도 어느 정도 성능을 유지한다면($S(M_r)$ 값이 크다면) 중요한 지식을 담고 있다고 판단한다.

### 3. ALoRA 학습 절차 (Workflow)

ALoRA의 전체 프로세스는 다음과 같은 단계로 반복된다 (Algorithm 1 참조).

1. **초기화**: 모든 Transformer 모듈에 목표 예산 $R_{target}$을 균등하게 분배하여 초기 랭크 $r_{init}$을 설정한다.
2. **슈퍼 네트워크 학습**: 학습 세트에서 모델을 $K_1$ 에포크 동안 학습시킨다.
3. **중요도 평가**: 검증 데이터셋 $B_{val}$을 사용하여 AB-LoRA 방식으로 모든 랭크의 중요도 점수 $IS(r)$를 계산한다.
4. **가지치기(Pruning)**: 점수가 가장 낮은 $n_0$개의 랭크를 선택하여 게이트 $\alpha$를 0으로 설정하고 제거한다.
5. **재할당(Allocation)**: 제거된 랭크 예산을 가지치기가 발생하지 않은(즉, 중요한) 모듈들에 다시 할당한다. 이때 평균 중요도 점수가 높은 모듈에 우선적으로 할당한다.
6. **성능 회복 학습**: 구조가 변경된 모델을 $K_2$ 에포크 동안 추가 학습하여 손실된 성능을 회복한다.
7. **반복**: 위 과정을 $N_A$번 반복하여 최적의 랭크 배분을 찾는다.

## 📊 Results

### 실험 설정

- **백본 모델**: Llama-2 7B (주 실험), RoBERTa-large, GPT2-large (Ablation).
- **데이터셋**:
  - 분류 및 QA: GLUE (SST-2, RTE, QNLI), SuperGLUE (BoolQ, COPA, ReCoRD), SQuAD.
  - 생성 및 지시 튜닝: E2E benchmark, Alpaca dataset (MT-Bench 평가).
- **비교 대상**: LoRA, AdaLoRA, SoRA, SaLoRA, various Adapters 등.

### 주요 결과

1. **정량적 성능 향상**: Table 1에서 볼 수 있듯이, ALoRA는 7가지 벤치마크 태스크 모두에서 기존 PEFT 베이스라인보다 우수한 성능을 보였다. 특히 동일하거나 더 적은 파라미터 예산 하에서 AdaLoRA, SoRA, SaLoRA를 능가하였다.
2. **지시 튜닝(Instruction Tuning)**: Alpaca 데이터셋으로 튜닝 후 MT-Bench에서 GPT-4 평가 결과, ALoRA가 SoRA보다 더 높은 점수(7.47 vs 7.16)와 ROUGE-L 점수를 기록하였다.
3. **효율성**: Table 4에 따르면, ALoRA는 초기 랭크를 크게 잡지 않으므로 SoRA보다 GPU 메모리 사용량이 적으며, 전체 학습 시간은 LoRA/SoRA와 유사한 수준이다.
4. **랭크 할당 분석**: 시각화 결과(Figure 3), ALoRA는 **Attention 모듈의 Query(Q)와 Key(K)**에 더 많은 랭크를 할당하고, FFN 모듈에는 상대적으로 적은 랭크를 할당하는 경향을 보였다. 이는 태스크 특화 지식이 주로 Attention 메커니즘에 저장됨을 시사한다.

## 🧠 Insights & Discussion

### 강점 및 해석

- **합리적인 랭크 배분**: 모든 모듈에 동일한 랭크를 주는 대신, 태스크에 따라 필요한 모듈에 자원을 집중시키는 전략이 유효함을 입증하였다. 특히 Attention 모듈이 FFN보다 더 많은 적응 능력이 필요하다는 발견은 중요한 통찰이다.
- **안정적인 지표**: 기존의 DNAS 방식(건축 파라미터 $\alpha$ 값 자체를 중요도로 보는 방식)보다 AB-LoRA의 Ablation 기반 지표가 실제 성능과 더 밀접하게 연관되어 있음을 실험적으로 보여주었다.

### 한계 및 비판적 논의

- **계산 비용의 Trade-off**: 랭크를 재할당한 후 매번 성능 회복을 위해 추가 학습($K_2$ 에포크)을 수행해야 하므로, 단순 LoRA보다는 학습 시간이 더 소요된다.
- **모델 확장성**: Llama-2 7B까지는 검증되었으나, 13B나 70B와 같은 더 거대한 모델에서의 효율성과 성능 유지 여부는 계산 자원 문제로 인해 확인되지 않았다.
- **태스크 범위**: 주로 NLP 태스크에 집중되어 있어, 정보 추출(Information Extraction) 등 다른 유형의 NLP 작업에서도 동일한 랭크 배분 패턴이 나타날지는 미지수이다.

## 📌 TL;DR

본 논문은 LoRA의 고정 랭크 문제를 해결하기 위해, 중요도가 낮은 랭크를 제거하고 이를 필요한 모듈에 동적으로 재할당하는 **ALoRA** 프레임워크를 제안한다. 특히 **AB-LoRA**라는 새로운 중요도 측정 방식을 도입하여, 추가적인 메모리 오버헤드 없이 각 Transformer 모듈에 최적의 랭크를 배분함으로써 기존 PEFT 방법론들보다 우수한 성능을 달성하였다. 이 연구는 LLM 미세 조정 시 모든 모듈이 동일한 중요도를 가지지 않는다는 점을 활용하여 효율적인 파라미터 활용 방안을 제시했다는 점에서 향후 적응형 PEFT 연구에 중요한 기여를 할 것으로 보인다.
