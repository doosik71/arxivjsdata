# Client Adaptation improves Federated Learning with Simulated Non-IID Clients

L. Rieger, R. M. Th. Høegh, L. K. Hansen (2020)

## 🧩 Problem to Solve

본 논문은 연합 학습(Federated Learning, FL) 환경에서 클라이언트 간 데이터가 비독립 동일 분포(non-IID, non-identically and independently distributed)를 가질 때 발생하는 성능 저하 문제를 해결하고자 한다. 일반적인 연합 학습에서는 데이터가 여러 클라이언트에 분산되어 있으며, 각 클라이언트가 보유한 데이터의 클래스 분포뿐만 아니라, 동일한 클래스 내에서도 특성(feature) 분포가 서로 다를 수 있다.

이러한 데이터의 이질성은 모델의 수렴 속도를 늦추고 전역 모델(global model)의 성능을 떨어뜨린다. 특히 사용자가 사용하는 기기가 다르거나 환경이 다른 경우, 동일한 대상(예: 기침 소리)이라도 수집되는 신호의 특성이 다르게 나타나는 '클라이언트별 변이'가 발생한다. 따라서 본 연구의 목표는 이러한 클라이언트별 특성 차이를 적응적으로 학습할 수 있는 모델 구조를 제안하여, Non-IID 데이터 환경에서도 강건하고 높은 성능을 내는 연합 학습 프레임워크를 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 클라이언트별 맞춤형 적응을 가능하게 하는 **Conditional Gated Activation Unit (CGAU)** 구조를 제안한 것이다. CGAU의 중심 아이디어는 전역적으로 공유되는 특징(global features) 외에, 각 클라이언트가 가진 고유한 특성 표현 방식을 학습할 수 있도록 조건부 입력(conditioning)을 추가하는 것이다.

또한, 실제 환경과 유사한 Non-IID 상황을 재현하기 위해, 단순한 클래스 비율 조정을 넘어 사전 학습된 네트워크의 임베딩 공간에서 클러스터링을 통해 클라이언트를 시뮬레이션하는 원칙적인 방법을 제시하였다. 이를 통해 특성 분포의 이질성이 모델 성능에 미치는 영향을 정밀하게 분석할 수 있게 하였다.

## 📎 Related Works

기존의 연합 학습 연구들은 주로 클래스 분포의 불균형(class imbalance) 문제에 집중해 왔다. 예를 들어, Zhao et al. (2018)은 각 클라이언트가 소수의 클래스만 보유했을 때 발생하는 가중치 발산 문제를 다루었으며, 일부 데이터를 전역적으로 공유함으로써 이를 완화하려 하였다. Peng et al. (2019)은 적대적 재구성을 통해 도메인 불변 특징(domain-invariant features)을 학습하여 도메인 시프트 문제를 해결하고자 하였다.

본 논문과의 차별점은 기존 연구들이 주로 '클래스 분포'나 '도메인 불변성'에 집중한 반면, 본 연구는 **'클라이언트별 특성 변이(feature distribution variance)'**에 주목한다는 점이다. 또한, Ghosh et al. (2019)이 클라이언트들을 클러스터링하여 그룹별 모델을 학습시킨 것과 달리, 본 논문은 신경망 구조 내에 조건부 게이트 유닛을 통합하여 전역 모델과 클라이언트별 적응 파라미터를 동시에 학습하는 방식을 취한다.

## 🛠️ Methodology

### 1. 전체 시스템 구조

본 논문은 계산 자원이 제한된 환경을 고려하여, 사전 학습된 네트워크를 **고정된 특징 추출기(frozen feature extractor)**로 사용한다. 이 추출기는 중앙에서 학습되거나 기존 모델을 가져와 사용하며, 연합 학습 과정에서는 그 위에 얹혀진 작은 크기의 분류기(classifier)만을 학습시킨다. 이를 통해 통신 비용을 줄이고 학습 효율을 높인다.

### 2. Conditional Gated Activation Unit (CGAU)

분류기의 핵심 구성 요소인 CGAU는 입력 데이터 $x$를 처리할 때, 해당 데이터가 어느 클라이언트에서 왔는지를 나타내는 조건부 정보 $h$(one-hot encoding)를 함께 입력받는다. CGAU는 크게 필터링(filtering) 부분과 게이팅(gating) 부분으로 나뉜다.

- **Filtering**: 특성이 무엇인지(what the feature is)를 결정하며, 클라이언트 조건에 따라 이 특징이 어떻게 변조(modulation)되는지를 학습한다.
- **Gating**: 해당 특성이 활성화될지 여부(whether this feature is active)를 결정하며, 클라이언트별로 특징의 발현(expression) 정도를 조절한다.

전체 연산 과정은 다음과 같은 방정식으로 표현된다.

$$z = \tanh(W_f^T x + V_f^T h) \odot \sigma(W_g^T x + V_g^T h)$$

여기서:

- $x$는 입력 벡터, $h$는 클라이언트 ID의 원-핫 인코딩 벡터이다.
- $W_f, W_g$는 전역적으로 공유되는 필터 및 게이트 가중치이다.
- $V_f, V_g$는 클라이언트별로 학습되는 조건부 가중치이다.
- $\odot$은 요소별 곱(Hadamard product)을 의미하며, $\tanh$와 $\sigma$(시그모이드)는 각각 활성화 함수이다.

실제 연합 학습 환경에서 $V_f, V_g$는 클라이언트 로컬에 저장되어 관리되므로, 서버와 공유할 필요 없이 로컬의 학습 가능한 변이(translation) 값으로 작동한다.

### 3. Non-IID 클라이언트 시뮬레이션

데이터셋에 클라이언트 식별자가 없는 문제를 해결하기 위해, 다음과 같은 절차로 Non-IID 데이터를 생성한다.

1. 사전 학습된 네트워크를 통해 데이터를 임베딩 공간으로 투영한다.
2. PCA(주성분 분석)를 통해 차원을 축소한 뒤, 각 클래스별로 K-means 클러스터링을 수행하여 $C \times K$ 개의 중심점을 찾는다.
3. 각 샘플을 가장 가까운 중심점을 가진 클라이언트에게 할당함으로써, 임베딩 공간 상에서 지역적으로 밀집된(locally clustered) Non-IID 데이터를 생성한다.

### 4. 이질성 정량화 (Heterogeneity Quantification)

클라이언트 간의 이질성 $\Gamma$를 측정하기 위해, 각 클라이언트의 임베딩 분포와 나머지 모든 클라이언트의 통합 분포 사이의 **Fréchet Distance** (2nd Wasserstein distance)를 사용한다. 두 다변량 가우시안 분포 $D_1 = \mathcal{N}(\mu_1, \Sigma_1)$과 $D_2 = \mathcal{N}(\mu_2, \Sigma_2)$ 사이의 거리는 다음과 같다.

$$d^2(D_1, D_2) = \|\mu_1 - \mu_2\|_2^2 + \text{Tr}(\Sigma_1 + \Sigma_2 - 2(\Sigma_1 \Sigma_2)^{1/2})$$

최종 이질성 지표 $\Gamma$는 모든 클라이언트에 대해 이 거리의 평균을 내어 산출한다.

## 📊 Results

### 실험 설정

- **데이터셋**:
  - Experiment I: Freesound Database (FSD) - 오디오 데이터, 기침 소리 탐지(이진 분류), 클래스 불균형 심함.
  - Experiment II: CIFAR-10 - 이미지 데이터, 10개 클래스 분류, 클래스 균형 잡힘.
- **기준선(Baseline)**: 조건부 입력이 없는 표준 피드포워드 신경망(ReLU 사용).
- **학습 알고리즘**: FedAvg를 사용하여 10명의 시뮬레이션 클라이언트와 학습.
- **지표**: 오디오 데이터는 AUC(Area Under ROC), 이미지 데이터는 Accuracy를 측정.

### 주요 결과

- **오디오 데이터 (FSD)**: 데이터가 매우 이질적일 때(shuffling proportion 0.0) CGAU를 적용한 모델이 Baseline(AUC $\approx 0.994$)보다 높은 AUC($\approx 0.998$)를 기록하였다. 데이터가 균일해질수록 두 모델의 성능 차이는 사라졌다.
- **이미지 데이터 (CIFAR-10)**: 모든 셔플링 수준에서 CGAU가 Baseline보다 우수한 성능을 보였다. 특히 데이터 이질성이 높을 때 성능 향상 폭이 뚜렷했다.
- **정량적 분석**: 셔플링 비율이 낮을수록 Fréchet Distance $\Gamma$가 증가함을 확인하여, 제안한 시뮬레이션 방법이 데이터 이질성을 적절히 제어하고 있음을 입증하였다.

## 🧠 Insights & Discussion

본 연구는 CGAU가 특히 **데이터의 이질성이 높을 때** 효과적임을 보여주었다. 흥미로운 점은 일반적으로 Non-IID 데이터는 모델의 성능을 떨어뜨리지만, CGAU 모델은 오히려 이질성이 높을 때(셔플링 비율이 낮을 때) 더 높은 성능을 보였다는 것이다. 저자들은 이를 모델이 각 클라이언트의 고유한 데이터 분포를 명시적으로 학습했기 때문이라고 해석한다.

또한, 특징 추출기가 타겟 데이터셋이 아닌 다른 데이터셋(예: CIFAR-10이 아닌 ImageNet)으로 사전 학습되었을 때 CGAU의 효과가 더 컸다. 이는 사전 학습 모델이 타겟 도메인의 불변 특징을 완벽하게 추출하지 못해 '불필요한 특성(spurious features)'이 임베딩에 포함되어 있으며, CGAU가 이러한 클라이언트별 변이를 효과적으로 처리해 주기 때문인 것으로 분석된다.

**한계점 및 논의사항**:

1. **계산 비용**: CGAU는 필터와 게이트 가중치를 추가하므로 파라미터 수가 증가하며, 시뮬레이션 환경에서 에포크당 학습 시간이 약 85% 증가하였다. 실제 환경에서 이 비용이 성능 이득보다 큰지 검토가 필요하다.
2. **프라이버시**: 클라이언트별 조건부 가중치를 로컬에 유지하더라도, 이러한 구조 자체가 프라이버시 공격에 취약할 가능성이 있으므로 이에 대한 보안성 평가가 필요하다.
3. **클라이언트 수**: 본 실험은 10명의 적은 수의 클라이언트를 대상으로 하였으므로, 실제 수천 명의 클라이언트가 참여하는 대규모 환경에서의 확장성 검증이 필요하다.

## 📌 TL;DR

본 논문은 연합 학습에서 클라이언트 간 특성 분포가 서로 다른 Non-IID 문제 해결을 위해, 클라이언트별 고유 특성을 학습하는 **Conditional Gated Activation Unit (CGAU)**를 제안한다. 실험을 통해 CGAU가 특히 데이터 이질성이 높고 전이 학습(Transfer Learning)을 사용할 때 성능을 유의미하게 향상시킴을 입증하였다. 이는 향후 사용자 맞춤형 개인화 연합 학습 모델 설계에 중요한 기초가 될 수 있다.
