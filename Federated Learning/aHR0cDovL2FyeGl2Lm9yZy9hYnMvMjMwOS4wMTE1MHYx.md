# FedFwd: Federated Learning without Backpropagation

Seonghwan Park, Dahun Shin, Jinseok Chung, Namhoon Lee (2023)

## 🧩 Problem to Solve

본 논문은 연합 학습(Federated Learning, FL) 환경에서 로컬 클라이언트들이 겪는 계산 및 메모리 자원의 제한 문제를 해결하고자 한다. 전통적인 딥러닝 모델 학습의 핵심인 역전파(Backpropagation, BP) 알고리즘은 모든 층의 중간 활성화 값(intermediate activation values)을 메모리에 저장해야 하며, 이는 네트워크의 깊이와 크기에 비례하여 메모리 사용량을 선형적으로 증가시킨다.

이러한 BP의 특성은 자원이 제한된 엣지 디바이스에서 대규모 고성능 모델을 배포하고 학습시키는 데 근본적인 제약이 된다. 기존의 FL 효율성 연구들은 주로 통신 오버헤드(communication overhead)를 줄이는 데 집중해 왔으나, 로컬 학습 단계에서의 계산 비용과 메모리 부하 문제는 충분히 다루어지지 않았다. 따라서 본 논문의 목표는 BP를 대체하는 새로운 학습 절차를 FL에 도입하여 로컬 클라이언트의 부담을 줄이는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 Hinton(2022)이 제안한 **Forward-Forward(FF) 알고리즘**을 연합 학습 프레임워크에 결합한 **FedFwd**를 제안하는 것이다. 

FedFwd의 중심적인 직관은 역전파를 통한 그래디언트 계산 대신, 두 번의 순방향 패스(forward pass)만을 사용하여 각 층을 독립적으로 최적화하는 것이다. 이를 통해 모든 중간 활성화 값을 저장할 필요가 없어져 메모리 사용량을 획기적으로 줄일 수 있으며, 층별(layer-wise) 로컬 업데이트를 통해 계산 효율성을 높일 수 있다.

## 📎 Related Works

논문에서는 다음과 같은 관련 연구들을 언급한다.

- **Federated Learning (FL):** 데이터 프라이버시를 유지하며 분산된 디바이스에서 모델을 학습시키는 패러다임이다. 기존 연구들은 수렴 속도 가속화, 모델 압축, 지식 증류(knowledge distillation) 등을 통해 효율성을 높이려 했으나, 로컬 학습의 근본적인 메모리/시간 제약 문제를 완전히 해결하지는 못했다.
- **Backpropagation (BP):** 신경망 학습의 표준 방법론이지만, 순차적인 그래디언트 계산 구조로 인해 병렬 계산이 어렵고 메모리 소모가 크다는 한계가 있다.
- **Federated Averaging (FedAvg):** 로컬에서 SGD를 수행한 후 서버에서 모델 파라미터를 평균 내는 표준 FL 알고리즘이다. 본 논문은 FedAvg의 전체적인 통신 구조는 유지하되, 로컬 학습 방식만을 BP에서 FF로 교체한다.

## 🛠️ Methodology

### 전체 시스템 구조
FedFwd는 기본적으로 FedAvg의 3단계 구조(클라이언트 선택 $\rightarrow$ 로컬 파라미터 업데이트 $\rightarrow$ 서버 집계)를 따른다. 차이점은 로컬 업데이트 과정에서 BP 대신 FF 알고리즘을 사용한다는 점이다.

### Forward-Forward (FF) 학습 절차
FF 알고리즘은 탐욕적인 층별 학습(greedy layer-wise learning) 방식을 취하며, 각 층은 '적합도(goodness)'라는 척도를 통해 최적화된다.

1. **적합도(Goodness) 정의**: 본 논문에서는 활동 벡터(activity vector)의 제곱 합을 적합도로 정의한다.
   $$\text{goodness} = \sum_j y_j^2$$
   여기서 $y_j$는 은닉 유닛 $j$의 활동 값이다.

2. **학습 목표**: 
   - **양성 샘플(Positive samples)**: 데이터의 일부 픽셀을 해당 레이블의 one-hot 벡터로 대체한 데이터이다. 이 샘플의 적합도가 임계값 $\theta$보다 커지도록 최대화한다.
   - **음성 샘플(Negative samples)**: 잘못된 레이블의 one-hot 벡터를 삽입한 데이터이다. 이 샘플의 적합도가 $\theta$보다 작아지도록 최소화한다.

3. **목적 함수**: 시그모이드 함수 $\sigma$를 사용하여 확률 형태로 정의한다.
   $$p(\text{positive}) = \sigma(\text{goodness}_{x_{\text{pos}}} - \theta)$$
   $$p(\text{negative}) = \sigma(-\text{goodness}_{x_{\text{neg}}} + \theta)$$

4. **층간 전달 및 정규화**: 
   각 층의 활동 벡터를 다음 층으로 전달하기 전에 **Layer Normalization**을 수행한다. 이는 이전 층의 활동 벡터 크기(goodness)만으로 샘플을 구분하는 것을 방지하고, 다음 층이 새로운 특징(feature)을 학습하도록 강제하기 위함이다.

### 추론(Prediction) 절차
예측 단계에서는 학습 때와 마찬가지로 이미지에 레이블 정보를 삽입한다. 모든 가능한 레이블에 대해 이미지를 생성하여 모델에 입력하고, **모든 층에서 계산된 적합도 값의 총합**이 가장 높은 레이블을 최종 예측값으로 선택한다.

## 📊 Results

### 실험 설정
- **데이터셋**: MNIST, CIFAR-10
- **모델**: MLP (Multi-Layer Perceptron) 기반 아키텍처 (2층 및 3층)
- **설정**: i.i.d. (데이터 균등 분배) 및 non-i.i.d. (데이터 불균등 분배) 설정
- **비교 대상**: FedAvg (BP 기반)
- **지표**: 테스트 정확도(Test Accuracy), 학습 시간(Wall Clock Time)

### 주요 결과
1. **정확도 분석 (MNIST)**:
   - i.i.d. 설정에서 FedFwd는 FedAvg와 유사한 성능을 보였다. (3층 MLP 기준 FedAvg 98.25%, FedFwd 96.78%로 약 1.5% 차이)
   - non-i.i.d. 설정에서는 성능 격차가 벌어졌다. (3층 MLP 기준 FedAvg 97.46%, FedFwd 92.57%) 이는 FedFwd가 데이터 분포 변화에 더 민감하며 수렴 속도가 느리기 때문이다.

2. **정확도 분석 (CIFAR-10)**:
   - 두 방법 모두 성능이 낮았으나, FedAvg가 FedFwd보다 약 13-14% 더 높은 성능을 기록했다. 저자들은 이미지를 단순 평탄화(flattening)하여 처리한 것이 복잡한 CIFAR-10 이미지 특징 학습에 한계가 있었기 때문이라고 분석했다.

3. **학습 시간 분석**:
   - mini-batch 크기가 1일 때는 FedFwd가 FedAvg보다 약 7배 느렸다. 이는 FedAvg가 라이브러리 최적화가 잘 되어 있는 반면, FedFwd는 그렇지 않기 때문이다.
   - 그러나 **mini-batch 크기가 증가할수록 두 방법 간의 시간 차이는 급격히 줄어들며**, 일부 경우에서는 FedFwd가 더 빠른 속도를 보였다.

## 🧠 Insights & Discussion

### 강점 및 가능성
FedFwd는 BP 없이도 MNIST와 같은 데이터셋에서 경쟁력 있는 정확도를 달성할 수 있음을 보여주었다. 특히 배치 사이즈가 커질수록 학습 시간이 효율적으로 변한다는 점은, 향후 소프트웨어 및 하드웨어 최적화가 이루어질 경우 BP 기반 FL보다 훨씬 적은 메모리로 빠른 학습이 가능함을 시사한다.

### 한계 및 비판적 해석
- **데이터 분포 민감도**: non-i.i.d. 설정에서 성능이 크게 하락하는 점은 FedFwd가 각 층을 개별적으로 최적화하는 탐욕적(greedy) 방식의 한계임을 보여준다. 전역적인 최적화보다는 국소적인 적합도에 의존하기 때문에 데이터 불균형에 취약하다.
- **아키텍처의 단순함**: CIFAR-10에서의 낮은 성능은 단순 MLP 구조의 한계이다. FF 알고리즘이 실제 효용성을 가지려면 CNN과 같은 더 복잡한 구조로의 확장이 필수적이다.
- **최적화 부족**: Wall Clock Time 측정 시 라이브러리 최적화 차이로 인해 초기 결과가 불리하게 나왔다. 이는 알고리즘 자체의 효율성보다는 구현상의 문제일 가능성이 크다.

### 향후 발전 방향
논문의 부록(Appendix B.2)에서는 **SymBa** 알고리즘을 결합하여 양성과 음성 손실(loss)의 균형을 맞춤으로써 학습 안정성과 수렴 속도를 개선할 수 있음을 제시했다. 이는 FF 기반 FL이 목적 함수 수정이나 아키텍처 개선을 통해 충분히 발전 가능성이 있음을 의미한다.

## 📌 TL;DR

본 논문은 연합 학습(FL)의 로컬 학습 단계에서 메모리 소모가 큰 역전파(BP) 대신, 층별로 적합도를 최적화하는 **Forward-Forward(FF) 알고리즘**을 도입한 **FedFwd**를 제안한다. MNIST 데이터셋의 i.i.d. 설정에서 BP 기반의 FedAvg와 대등한 성능을 보였으며, 배치 사이즈가 커질수록 학습 시간 효율성이 증가함을 확인했다. 비록 non-i.i.d. 데이터나 복잡한 데이터셋(CIFAR-10)에서는 한계를 보였으나, 메모리 제약이 심한 엣지 디바이스 환경에서 BP를 대체할 수 있는 새로운 가능성을 제시한 연구이다.
