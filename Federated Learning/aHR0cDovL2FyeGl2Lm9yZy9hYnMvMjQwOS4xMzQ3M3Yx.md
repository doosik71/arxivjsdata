# Flotta: a Secure and Flexible Spark-inspired Federated Learning Framework

Claudio Bonesana, Daniele Malpetti, Sandra Mitrović, Francesca Mangili, Laura Azzimonti (2024)

## 🧩 Problem to Solve

본 논문은 특히 생의학(biomedical) 분야와 같이 매우 높은 수준의 보안이 요구되는 다자간 컨소시엄 환경에서, 분산된 민감 데이터에 대해 머신러닝 모델을 학습시키기 위한 프레임워크의 필요성을 다룬다.

일반적으로 Federated Learning(FL)은 데이터를 중앙으로 이동시키지 않고 모델 파라미터만을 공유하여 프라이버시를 보호한다. 그러나 실제 의료 연구 환경에서는 다음과 같은 추가적인 제약 사항과 문제점이 존재한다.

- **데이터 및 노드 노출 제한**: 데이터의 이동뿐만 아니라, 데이터를 보유한 당사자(party) 자체가 외부 네트워크에 노출되는 것을 금지하는 엄격한 내부 규정이 존재할 수 있다.
- **외부 서버 의존성 문제**: 기존의 많은 FL 프레임워크가 외부 클라우드 서비스나 중앙 서버에 의존하지만, 보안 규정상 컨소시엄 내부의 머신만을 사용해야 하는 경우가 많다.
- **유연성과 보안의 트레이드오프**: 분석가에게 자유로운 코드 실행 권한을 주면 유연성은 높아지나 민감 데이터가 유출될 위험이 있고, 반대로 권한을 너무 제한하면 연구의 유연성이 떨어진다.

따라서 본 논문의 목표는 컨소시엄 내부 자원만을 활용하면서도, 보안성과 연구의 유연성 사이의 적절한 균형을 갖춘 Secure하고 Flexible한 FL 프레임워크인 **Flotta**를 제안하는 것이다.

## ✨ Key Contributions

Flotta의 핵심 아이디어는 Apache Spark의 분산 컴퓨팅 아키텍처에서 영감을 받아, **분석 설계(High-level decision)**와 **작업 할당(Low-level task assignment)**을 완전히 분리하는 것이다.

중심적인 설계 전략은 다음과 같다.

- **분석-진입점 분리 (Disentangling Analyst and Entry Point)**: 분석가(Analyst)가 분석 파이프라인을 설계하면, 이를 진입점 노드(Entry Point Node)가 수신하여 구체적인 작업 그래프로 변환하고 각 노드에 배분하는 구조를 가진다.
- **허가된 명령 세트 (Granted Instructions)**: 임의의 코드를 실행하는 대신, 프레임워크에서 미리 정의되고 검증된 '명령(Instruction)'들의 조합으로만 분석 파이프라인을 구성하게 함으로써, 악의적인 사용자가 민감 데이터를 추출하는 것을 방지하고 보안을 강화한다.
- **유연한 토폴로지 지원**: 중앙 집중형(Centralized), 분산형(Decentralized), 그리고 그 중간 형태의 다양한 네트워크 구조를 자연스럽게 지원하도록 설계되었다.

## 📎 Related Works

논문에서는 다양한 FL 프레임워크를 언급하며 Flotta와의 차별점을 제시한다.

- **Deep Learning 중심**: TensorFlow Federated, PySyft 등은 딥러닝에 특화되어 있다.
- **Cross-device 시나리오 중심**: Felicitas, FairFed 등은 수많은 엣지 디바이스를 대상으로 하는 시나리오에 집중한다.
- **분산 토폴로지 중심**: Substra는 탈중앙화된 토폴로지를 구현한다.
- **범용 프레임워크**: Flower, FedML, FATE 등이 이에 해당한다.
- **생의학 특화 프레임워크**: FeatureCloud와 Sfkit은 사용자 친화적인 웹 인터페이스를 제공하며 의료 분야에 특화되어 있다.

**FeatureCloud와의 차별점**:
FeatureCloud는 편리하지만 전용 서버를 사용해야 하므로 외부 서버 이용을 금지하는 기관의 규정을 충족하기 어렵다. 반면 Flotta는 컨소시엄 내부의 머신만으로 운영이 가능하다. 또한 FeatureCloud는 매 분석마다 데이터 보유자의 승인이 필요하지만, Flotta는 '프로젝트(Project)' 단위로 권한을 관리하여 승인된 분석가가 해당 프로젝트 범위 내에서 여러 분석을 수행할 수 있도록 하여 효율성을 높였다.

## 🛠️ Methodology

### 전체 시스템 구조

Flotta는 크게 세 가지 주요 구성 요소인 **Nodes**, **Workbench**, **Artifact**로 이루어져 있다.

1. **Nodes (노드)**: 데이터 저장 및 실제 계산을 수행하는 백본이다.
    - **기능적 관점**:
        - **Scheduler (스케줄러)**: 진입점 노드(Entry Point Node)가 수행하며, Artifact를 작업 그래프(Graph of tasks)로 변환하고 노드들에 배분하며 생명주기를 관리한다.
        - **Executor (실행기)**: 할당된 작업을 실제로 수행한다.
    - **통신 관점**:
        - **Default mode**: 모든 노드와 양방향 통신이 가능하다.
        - **Client-mode**: 특정 참조 노드(보통 진입점 노드)와만 단방향으로 통신하며, 주기적으로 업데이트와 작업을 요청(Polling)한다. 이는 외부 인터넷 접속이 차단된 병원 환경 등에 적합하다.

2. **Workbench (워크벤치)**: 분석가가 Python 코드를 통해 분석 내용을 설계하는 인터페이스이다. Apache Spark의 PySpark와 유사한 사용자 경험을 제공한다.

3. **Artifact (아티팩트)**: 워크벤치에서 작성되어 진입점 노드로 전송되는 지침서이다. 이는 미리 정의된 보안 명령(Instructions)들의 집합으로 구성되며, 일종의 ETL 파이프라인과 유사하게 작동한다.

### 구현 세부 사항

- **웹 서비스**: `FastAPI`를 사용하여 노드 간 스케줄링 및 통신을 처리한다.
- **병렬 실행**: `Ray` 패키지를 사용하여 다중 프로세스를 생성하고 병렬로 작업을 실행한다.
- **데이터 직렬화**: `Pydantic`을 사용하여 Python 객체와 JSON 간의 변환을 수행하며, 모델 파라미터와 같은 대용량 데이터는 바이너리 형태로 교환한다.
- **보안**: 모든 통신 객체는 비대칭 암호화(Asymmetric Encryption) 알고리즘을 통해 암호화되며, 서명 시스템을 통해 노드와 워크벤치의 신원을 확인한다.

### 실행 절차 (Example Process)

논문에서 제시한 Random Forest 모델 학습의 흐름은 다음과 같다.

1. **Artifact 제출**: 분석가가 워크벤치에서 설계한 `instructions`를 진입점 노드에 제출한다.
2. **작업 분배**: 진입점 노드가 이를 작업 그래프로 변환하여 데이터 보유 노드들에 배분한다.
3. **로컬 학습**: 각 노드는 로컬 데이터를 추출하고 전처리한 뒤, 모델을 개별적으로 학습시킨다.
4. **모델 수집 및 집계**: 학습된 로컬 모델들이 진입점 노드로 전송되면, 진입점 노드는 이를 집계(Aggregation, 예: Merge)하여 글로벌 모델을 생성한다.
5. **결과 반환**: 최종 생성된 글로벌 모델이 분석가에게 전달된다.

## 📊 Results

본 논문에서는 정량적인 벤치마크 결과보다는 프레임워크의 기능적 검증과 실용적 사례(Use Case)를 중심으로 설명한다.

- **실험 시나리오**: 5개 당사자가 참여하는 컨소시엄 환경에서 글로벌 Random Forest 모델을 학습시키는 사례를 시연하였다.
- **작업 내용**:
  - `Parallel` 명령을 통해 여러 노드에서 동시에 `TrainTest` 및 `Train` 작업을 수행하였다.
  - `FederatedSplitter`를 사용하여 로컬 데이터를 훈련 세트와 테스트 세트로 분리하였다.
  - `StrategyRandomForestClassifier.MERGE` 전략을 사용하여 로컬 모델들을 병합하는 집계 과정을 거쳤다.
- **결과**: 워크벤치에서 작성된 Python 코드가 Artifact 형태로 전달되어, 실제 분산 환경의 노드들에서 계획된 순서대로 작업이 수행되고 최종 모델이 분석가에게 성공적으로 반환됨을 확인하였다. 이는 Flotta가 설계한 대로 보안성과 유연성을 유지하며 동작함을 입증한다.

## 🧠 Insights & Discussion

### 강점

- **인프라 독립성**: 외부 클라우드나 제3자 서버 없이 컨소시엄 내부 자원만으로 FL 환경을 구축할 수 있어, 보안 규정이 극도로 엄격한 의료 기관에 최적화되어 있다.
- **보안과 유연성의 조화**: 임의 코드 실행을 금지하고 '허가된 명령'만 사용하게 함으로써 데이터 유출 가능성을 원천적으로 차단하는 동시에, 명령들의 조합을 통해 복잡한 분석 파이프라인을 구축할 수 있게 하였다.
- **실무적 배포 고려**: `Client-mode`와 `Polling` 메커니즘을 통해 방화벽 설정이 까다로운 환경에서도 최소한의 설정으로 노드를 배치할 수 있도록 설계되었다.

### 한계 및 논의사항

- **명령 세트의 확장성**: 현재는 미리 정의된 명령만 사용할 수 있다. 커스텀 명령을 플러그인 형태로 추가할 수 있으나, 이 경우 프레임워크 차원의 보안 보장이 어려우며 개발자가 직접 보안성을 검증해야 한다는 책임이 따른다.
- **성능 평가 부재**: 본 논문에서는 시스템의 구조와 기능적 구현에 집중하였으며, 대규모 데이터셋이나 많은 수의 노드 환경에서의 통신 오버헤드, 학습 수렴 속도 등에 대한 정량적 성능 분석은 제시되지 않았다.

## 📌 TL;DR

본 논문은 보안이 매우 중요한 생의학 연구 컨소시엄을 위해 Apache Spark에서 영감을 받은 FL 프레임워크 **Flotta**를 제안한다. Flotta는 외부 서버 의존성을 없애고 컨소시엄 내부 자원만으로 운영 가능하며, '허가된 명령(Granted Instructions)' 기반의 아티팩트 구조를 통해 임의 코드 실행으로 인한 데이터 유출을 방지하고 보안성을 극대화하였다. 이 연구는 특히 규제가 엄격한 의료 데이터 환경에서 실질적으로 적용 가능한 안전한 분산 학습 인프라를 제공한다는 점에서 가치가 크다.
