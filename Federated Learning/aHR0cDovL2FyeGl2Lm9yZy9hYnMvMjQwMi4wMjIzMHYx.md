# Federated Learning with Differential Privacy

Adrien Banse, Jan Kreischer, Xavier Oliva i Jürgens (2024)

## 🧩 Problem to Solve

본 논문은 분산 머신러닝의 한 형태인 Federated Learning(FL) 환경에서 데이터 프라이버시를 더욱 강화하기 위해 Differential Privacy(DP)를 적용했을 때, 모델의 성능과 수렴 속도에 어떠한 영향이 있는지를 분석하는 것을 목표로 한다.

일반적인 Federated Learning은 클라이언트의 원본 데이터를 서버로 전송하지 않으므로 프라이버시를 보호한다고 알려져 있다. 그러나 서버가 수집하는 모델의 파라미터 가중치(parameter weights)를 분석함으로써 역으로 클라이언트의 개인 정보가 유출될 수 있는 위험이 존재한다. 특히 의료 데이터와 같이 민감한 정보를 다루는 분야에서는 GDPR과 같은 엄격한 법적 규제를 준수해야 하므로, 모델 업데이트 과정에서도 강력한 프라이버시 보장 기법이 필수적이다.

따라서 본 연구는 다음과 같은 구체적인 질문에 답하고자 한다.
1. 데이터 분포의 수준(i.i.d vs non-i.i.d)이 모델의 정확도와 최적화 과정의 수렴에 어떤 영향을 미치는가?
2. Differential Privacy의 적용이 모델의 정확도에 어떤 영향을 미치는가?
3. 이러한 기법이 실제 상황과 유사한 소규모 데이터셋에서도 적용 가능한가?

## ✨ Key Contributions

본 논문의 핵심 기여는 Federated Learning 환경에서 Differential Privacy를 적용했을 때 발생하는 **프라이버시와 유용성 간의 트레이드-오프(Privacy-Utility Trade-off)**를 다양한 데이터 시나리오(i.i.d, non-i.i.d, 소규모 데이터셋)를 통해 실증적으로 벤치마킹한 것이다. 

연구의 중심 아이디어는 데이터의 분포 상태와 데이터셋의 크기가 DP 메커니즘에 의한 성능 저하에 얼마나 민감하게 반응하는지를 정량적으로 분석하는 것이다. 특히, non-i.i.d 데이터와 소규모 데이터셋이 분산 및 DP 설정 환경에서 가장 심각한 성능 하락을 겪는다는 점을 밝혀냈다.

## 📎 Related Works

본 논문에서 다루는 관련 연구는 크게 두 가지 방향으로 나뉜다.

1. **Federated Learning (FL):** 데이터를 소유한 클라이언트가 데이터를 유지한 채 모델의 업데이트 값만을 서버와 교환하여 학습하는 비암호화 방식의 프라이버시 보존 학습 기법이다. 대표적으로 FedAvg 알고리즘이 사용된다.
2. **Differential Privacy (DP):** 데이터셋에 특정 개별 레코드가 포함되었는지 여부를 알 수 없도록 노이즈를 추가하여 프라이버시 손실의 상한선을 설정하는 기법이다.

기존 연구들은 FL과 DP를 결합하여 정직하지만 호기심 많은(honest-but-curious) 서버로부터 데이터를 보호하려 했다. 본 논문은 이러한 이론적 배경을 바탕으로, 실제 다양한 데이터 특성(데이터 분포 및 크기)이 결합되었을 때 발생하는 실질적인 성능 저하 문제를 집중적으로 분석한다는 점에서 차별점을 가진다.

## 🛠️ Methodology

### 1. 전체 시스템 구조 및 파이프라인
본 연구는 **Cross-silo Federated Learning** 설정을 따른다. 이는 병원이나 은행과 같이 높은 계산 자원을 가진 조직 간의 협업 학습 환경을 의미한다. 전체 학습 과정은 다음과 같은 흐름으로 진행된다.
- **중앙 서버:** 글로벌 모델을 관리하며, 각 클라이언트에게 모델을 배포하고 수집된 업데이트를 평균 내어 글로벌 모델을 갱신한다.
- **클라이언트:** 로컬 데이터를 사용하여 모델을 학습시키고, DP 메커니즘이 적용된 모델 업데이트 값을 서버로 전송한다.

### 2. 주요 구성 요소 및 알고리즘

#### Federated Averaging (FedAvg)
글로벌 모델의 파라미터를 $w_t$라고 할 때, $K$개의 클라이언트가 각각 로컬 업데이트를 수행하여 $w_{t+1}^k$를 생성하면, 서버는 이를 다음과 같이 가중 평균하여 다음 라운드의 글로벌 모델을 생성한다.
$$w_{t+1} = \sum_{k=1}^{K} \frac{n_k}{n} w_{t+1}^k$$
여기서 $n_k$는 클라이언트 $k$의 데이터 크기이며, $n$은 전체 데이터의 총합이다.

#### Differential Privacy ($\epsilon, \delta$-DP)
본 논문은 $(\epsilon, \delta)$-DP 기준을 사용하여 프라이버시를 정의한다. 인접한 두 데이터셋 $D$와 $D^{-r}$(한 개의 레코드 $r$이 차이 나는 데이터셋)에 대해, 알고리즘 $A$의 출력값 $O$에 대한 확률 분포가 다음과 같은 범위를 유지해야 한다.
$$\log \frac{P[A(D) = O]}{P[A(D^{-r}) = O]} < \epsilon \text{ (확률 } 1-\delta \text{ 내에서)}$$

#### Gradient Perturbation (경사도 섭동)
딥러닝 모델에 DP를 적용하기 위해 본 연구는 PyTorch의 **Opacus** 라이브러리를 사용하여 Gradient Perturbation 방식을 채택하였다. 학습 과정은 다음과 같은 절차로 이루어진다.
1. **Gradient Clipping:** 개별 데이터 포인트의 경사도 $g$를 계산하고, $L_2$ 노름(norm)이 임계값 $C$를 넘지 않도록 클리핑하여 $\bar{g}$를 생성한다.
   $$\bar{g} = \frac{g}{\max(1, \frac{\|g\|_2}{C})}$$
2. **Noise Addition:** 클리핑된 경사도들의 평균에 가우시안 노이즈를 추가하여 최종 경사도 $\tilde{g}$를 계산한다.
   $$\tilde{g} = \frac{1}{B} \left( \sum_{i=1}^{B} \bar{g}_i + \mathcal{N}(0, \sigma^2 C^2 \mathbf{I}) \right)$$
   여기서 $B$는 배치 크기이며, $\sigma$는 노이즈 스케일이다.

## 📊 Results

### 1. 실험 설정
- **데이터셋 및 모델:**
    - **MNIST:** i.i.d 분포, CNN 모델 사용.
    - **FEMNIST:** non-i.i.d 분포(필기자 기준 분할), CNN 모델 사용.
    - **Medical Dataset:** 소규모 데이터(120명), Logistic Regression 모델 사용.
- **실험 변수:**
    - **실험 1:** 클라이언트 수 ($nr\_clients \in \{1, 5, 10\}$)에 따른 영향 분석.
    - **실험 2:** 프라이버시 예산 ($\epsilon \in \{10, 50, 100\}$)에 따른 영향 분석. (단, $\delta = 1/2n$ 고정)

### 2. 주요 결과
- **클라이언트 수의 영향:** 모든 데이터셋에서 클라이언트 수가 증가할수록 학습 속도가 느려지는 경향이 관찰되었다. 이는 각 클라이언트가 서로 다른 로컬 최적점(local minima)으로 수렴하려 하기 때문이다.
- **DP 적용의 영향 (MNIST):** DP를 적용하지 않았을 때 95% 이상의 정확도를 보였으나, $\epsilon=100$이라는 매우 느슨한 프라이버시 설정에서도 정확도가 약 75%까지 급격히 하락하였다. $\epsilon$ 값이 커질수록(프라이버시 보호 수준이 낮아질수록) 수렴 속도는 빨라지지만, 최종 정확도는 비슷하게 유지되었다.
- **데이터 분포의 영향 (FEMNIST):** non-i.i.d 데이터셋인 FEMNIST에서는 DP 적용 시 학습 프로세스가 사실상 수렴하지 않았으며, 실용적인 모델을 생성하는 데 실패하였다.
- **소규모 데이터셋의 영향 (Medical):** DP 메커니즘에 매우 민감하게 반응하였으며, 학습 중 분산(variance)이 매우 높게 나타났다. $\epsilon=10$인 경우에만 손실 함수가 감소하며 모델이 일부 개선되는 모습을 보였다.

## 🧠 Insights & Discussion

### 1. 프라이버시와 유용성의 상충 관계
실험 결과, 프라이버시를 강화하기 위해 노이즈를 추가하고 경사도를 클리핑하는 행위가 모델의 유용성(utility)을 심각하게 훼손함을 확인하였다. i.i.d 데이터인 MNIST에서도 성능이 20%가량 하락한 점은 DP가 딥러닝 모델의 성능에 매우 치명적일 수 있음을 시사한다.

### 2. 데이터 특성에 따른 취약성
- **Non-i.i.d 데이터:** 데이터가 실로(silo)별로 분산되어 있을 때 글로벌 모델의 성능이 저하되는데, 여기에 DP 노이즈까지 추가되면 모델이 수렴하기 매우 어려워진다. 저자는 이에 대한 대안으로 중앙 집중식 FL 대신 d-clique 토폴로지를 이용한 분산 학습(decentralized learning)을 제안한다.
- **소규모 데이터셋:** 데이터 양이 적은 실제 의료 데이터의 경우 DP 메커니즘에 의한 영향이 극대화된다. 특히 표준적인 $\epsilon=0.1$ 수준의 강한 프라이버시는 단 한 번의 미니배치 SGD 반복조차 불가능하게 만들 정도로 제한적이어서, 훨씬 큰 $\epsilon$ 값을 사용해야만 했다.

### 3. 한계점 및 비판적 해석
본 연구는 하이퍼파라미터 튜닝을 정밀하게 수행하지 않고 고정된 파라미터를 사용하였다는 한계가 있다. 따라서 관찰된 성능 저하가 DP 자체의 문제인지, 혹은 DP 환경에 최적화되지 않은 하이퍼파라미터 때문인지에 대한 추가 분석이 필요하다. 또한, DP 외에 모델 자체를 암호화하는 동형 암호(Fully Homomorphic Encryption, FHE)와 같은 다른 대안들과의 비교 분석이 부족하다.

## 📌 TL;DR

본 논문은 Federated Learning에 Differential Privacy(DP)를 결합했을 때, 데이터의 분포(i.i.d vs non-i.i.d)와 데이터셋의 크기가 모델 성능에 미치는 영향을 분석하였다. 실험 결과, **DP 적용은 전반적인 모델 성능을 크게 하락시키며, 특히 데이터가 불균형하게 분포되어 있거나 데이터셋의 규모가 작은 실제 의료 데이터 환경에서 그 피해가 극심함**을 보였다. 이는 향후 프라이버시 보존 학습 연구가 단순히 이론적인 $\epsilon$ 값 설정에 그치지 않고, 실제 데이터의 특성을 고려한 적응형 DP 메커니즘이나 대체 암호화 기술의 도입이 필요함을 시사한다.
