# Towards Interpretable Federated Learning

Anran Li, Rui Liu, Ming Hu, Luu Anh Tuan, and Han Yu (2023)

## 🧩 Problem to Solve

연합 학습(Federated Learning, FL)은 여러 데이터 소유자가 자신의 개인 데이터를 외부에 노출하지 않고 협력적으로 머신러닝 모델을 구축할 수 있게 한다. 그러나 현재의 FL 모델들은 심층 신경망(Deep Neural Networks, DNNs)과 같이 복잡한 비선형 기반 모델을 사용하는 경우가 많아, 모델의 내부 작동 메커니즘이나 의사결정 과정을 이해하기 어려운 '블랙박스' 문제가 발생한다.

이러한 해석 가능성의 부재는 특히 금융이나 의료와 같이 신뢰성과 안전성이 필수적인 미션 크리티컬(mission critical) 애플리케이션에서 기술 도입을 저해하는 주요 원인이 된다. 따라서 모델의 예측 근거를 설명하거나 훈련 과정의 메커니즘을 제공하는 해석 가능한 연합 학습(Interpretable Federated Learning, IFL)의 필요성이 대두되었다. 특히 IFL은 중앙 집중식 학습의 설명 가능한 AI(XAI)와 달리, 데이터의 비가시성(invisibility)과 로컬 계산 및 통신 자원의 제약이라는 추가적인 제약 조건이 있어 구현이 더욱 어렵다. 본 논문의 목표는 이러한 IFL 분야의 연구 흐름을 체계화하고, 이를 분석하기 위한 독자적인 분류 체계(Taxonomy)를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 학계와 산업계에서 빠르게 발전하고 있는 IFL 분야의 첫 번째 종합 서베이(Survey)를 제공한다는 점이다. 저자들은 IFL을 단순히 예측 결과의 설명에 국한시키지 않고, 모델 디버깅 및 개별 데이터 소유자의 기여도 평가까지 확장하여 다음과 같은 통합적인 IFL 분류 체계를 제안한다.

1. **이해관계자(Stakeholders) 분석**: FL 서버, 클라이언트, 그리고 규제 기관과 같은 간접 이해관계자의 서로 다른 요구사항을 정의한다.
2. **프라이버시 보호 분석**: IFL 구현 시 고려해야 할 보호 대상(원시 데이터, 분포, 레이블), 위협 모델(Semi-honest, Malicious), 그리고 보호 기술(DP, HE, MPC)을 체계화한다.
3. **IFL 단계별 접근법 제시**: 클라이언트 선택, 샘플 선택, 특징 선택, 모델 최적화, 기여도 평가라는 5가지 주요 단계로 IFL 기술을 분류하고 각 단계별 대표적 방법론을 분석한다.
4. **평가 지표 정의**: IFL의 효과성을 측정하기 위한 성능 및 신뢰성 지표와 효율성 지표를 제시한다.

## 📎 Related Works

본 논문은 기존의 중앙 집중식 머신러닝을 위한 XAI 방법론과 IFL의 차이점을 강조한다. 중앙 집중식 XAI는 모든 데이터에 접근할 수 있다는 가정하에 작동하지만, IFL은 데이터 프라이버시 보존이라는 제약이 있어 기존 방식을 그대로 적용할 수 없다.

논문에서 언급된 관련 연구들은 주로 개별 단계(예: 단순한 클라이언트 선택이나 특정 특징 추출)에 집중되어 있으며, IFL 전반을 아우르는 체계적인 프레임워크나 분류 체계가 부족했다는 점을 지적한다. 또한, 기존의 많은 IFL 연구들이 단순한 Semi-honest 위협 모델을 가정하고 있어, 실제 환경의 복잡한 적대적 공격이나 공모(collusion) 상황에 취약하다는 한계를 가지고 있다.

## 🛠️ Methodology

본 논문은 IFL의 전체 파이프라인을 5가지 핵심 단계로 나누어 분석한다.

### 1. 해석 가능한 클라이언트 선택 (Interpretable Client Selection)
전역 모델의 성능은 로컬 데이터의 품질에 의존하므로, 고품질 데이터를 보유한 클라이언트를 식별하는 것이 중요하다.
- **중요도 기반(Importance-based)**: 모델 업데이트의 편차(Model Deviation), 로컬 손실(Loss), 그래디언트 노름(Gradient Norm), 또는 어텐션 메커니즘(Attention)을 통해 클라이언트의 중요도를 정량화한다.
- **영향력 기반(Influence-based)**: 특정 클라이언트를 제거하고 모델을 재학습시켜 성능 변화를 측정하거나, 영향력 함수(Influence Function)를 사용한다. 특히 Hessian 행렬 계산의 비용을 줄이기 위해 Hessian-Vector Product (HVP) 근사치를 사용하여 계산 복잡도를 $O(p^3n)$에서 $O(np)$로 낮추는 방식이 제안되었다. (여기서 $p$는 파라미터 수, $n$은 샘플 수이다.)

### 2. 해석 가능한 샘플 선택 (Interpretable Sample Selection)
모든 로컬 샘플이 학습에 유용한 것은 아니므로, 유효한 샘플만을 선택하여 효율성을 높인다.
- **관련성 기반(Relevance-based)**: 벤치마크 모델을 통해 작업과 관련성이 높은 샘플을 사전에 선택한다.
- **중요도 기반(Importance-based)**: 손실 값이 임계치를 넘는 샘플이나 그래디언트 노름 상한선이 높은 샘플을 우선적으로 선택한다.
- **영향력 기반(Influence-based)**: HVP 근사를 통해 개별 샘플이 예측 결과에 미치는 영향을 분석하여 선택한다.

### 3. 해석 가능한 특징 선택 (Interpretable Feature Selection)
불필요하거나 노이즈가 섞인 특징(Feature)을 제거하여 모델의 복잡도를 낮추고 해석력을 높인다.
- **모델 불가지론적(Model-Agnostic)**: 모델 내부 구조를 보지 않고 상호 정보량(Mutual Information), Gini 불순도, F-통계량 등을 활용해 특징의 관련성을 측정한다.
- **모델 특화적(Model-Specific)**: LASSO와 같은 $l_1$ 제약 조건이나, 확률적 이중 게이트(Stochastic Dual-Gate) 기반의 $l_0$ 제약 조건을 활용하여 중요한 특징을 직접 선택한다.

### 4. 해석 가능한 모델 최적화 (Interpretable Model Optimization)
- **내재적 해석 가능 모델**: 의사결정 나무(Decision Tree)나 선형 모델과 같이 구조적으로 해석이 쉬운 모델을 사용하거나, 희소성(Sparsity) 제약 조건을 추가하여 학습한다.
- **강건한 집계(Robust Aggregation)**: 비잔틴(Byzantine) 클라이언트의 영향을 줄이기 위해 영향력 값을 기반으로 가중치를 조절하는 집계 방식을 사용한다.

### 5. 해석 가능한 기여도 평가 (Interpretable Contribution Evaluation)
- **클라이언트 기여도**: 협력 게임 이론의 샤플리 값(Shapley Value, SV)을 주로 사용한다. SV의 지수적 계산 복잡도 $O(2^K)$를 해결하기 위해 Monte Carlo 샘플링이나 Truncated MR(TMR) 방식을 통해 계산 효율을 $O(m \log m)$ 수준으로 낮춘다.
- **특징 기여도**: 어텐션 메커니즘을 통해 모델이 어떤 입력 특징에 집중하는지 시각화하거나, 뉴런을 강하게 활성화하는 입력 서브시퀀스를 추출하는 활성화 기반(Activation-based) 방식을 사용한다.

## 📊 Results

본 논문은 서베이 논문이므로 새로운 실험 결과보다는 기존 IFL 연구들에서 사용되는 평가 지표를 분석하여 제시한다.

### 효과성 지표 (Effectiveness Metrics)
- **해석 후 성능 (Post-Interpretation Performance)**: IFL을 통해 식별된 중요한 클라이언트/샘플/특징만을 사용하여 모델을 조정했을 때, 정확도나 에러율이 얼마나 개선되었는지를 측정한다.
- **신뢰성 (Faithfulness)**: 'Leave-some-out' 재학습 방식을 사용한다. 즉, 해석 결과에서 중요하다고 판단된 요소를 제거했을 때 모델 성능이 유의미하게 하락한다면, 해당 해석이 신뢰할 수 있다고 판단한다.

### 효율성 지표 (Efficiency Metrics)
- **계산 비용 (Computation Cost)**: 해석을 생성하는 데 필요한 연산 시간(Elementary operations)과 메모리 사용량을 측정한다.
- **통신 비용 (Communication Cost)**: IFL 모델 학습 및 해석 과정에서 전송되는 모델 파라미터의 바이트(Bytes) 수를 측정한다.

## 🧠 Insights & Discussion

저자들은 현재 IFL 연구의 한계점과 향후 발전 방향에 대해 다음과 같은 통찰을 제시한다.

첫째, **정확도와 해석 가능성의 트레이드오프** 문제이다. 내재적으로 해석 가능한 모델(예: 의사결정 나무)은 복잡한 DNN보다 정확도가 떨어지는 경향이 있다. 이를 해결하기 위해 복잡한 모델을 단순한 모델로 모방하는 '모델 근사(Model Approximation)' 또는 '미믹 러닝(Mimic Learning)'이 유망한 대안이 될 수 있다.

둘째, **노이즈에 대한 강건성** 부족이다. 현재의 샘플 선택 방식은 단순히 손실 값이 큰 샘플을 중요하다고 판단하는데, 이는 실제 유용한 'Hard Sample'과 잘못된 'Noisy Sample'을 구분하지 못한다는 치명적인 약점이 있다. 따라서 노이즈에 강건하면서도 중요한 샘플을 식별하는 IFL 기법이 필요하다.

셋째, **현실적인 위협 모델의 부재**이다. 대부분의 연구가 Semi-honest 모델에 의존하고 있어, 실제 환경의 악의적인 공격자나 공모자 상황에 취약하다. 또한, IFL이 제공하는 '해석 정보' 자체가 역으로 공격자에게 이용되어 프라이버시를 침해할 위험성에 대한 연구가 부족하다.

마지막으로, **프라이버시와 효율성의 상충 관계**이다. HE나 MPC 같은 강력한 보호 기술은 계산 및 통신 오버헤드가 매우 커서 AIoT와 같은 자원 제한적 환경에서는 적용이 어렵다.

## 📌 TL;DR

본 논문은 프라이버시 보존과 모델 해석 가능성을 동시에 달성하려는 **해석 가능한 연합 학습(IFL)** 분야의 첫 번째 종합 서베이 보고서이다. 저자들은 IFL을 클라이언트/샘플/특징 선택, 모델 최적화, 기여도 평가의 5단계로 분류하는 **새로운 Taxonomy를 제안**하고, 각 단계별 방법론과 평가 지표를 체계적으로 분석하였다. 이 연구는 향후 IFL 시스템이 단순한 성능 향상을 넘어, 의료/금융 등 고신뢰 분야에 적용되기 위해 해결해야 할 핵심 과제(노이즈 강건성, 실제 위협 모델 대응, 효율적 프라이버시 보호)를 명확히 제시했다는 점에서 중요한 가치를 가진다.
