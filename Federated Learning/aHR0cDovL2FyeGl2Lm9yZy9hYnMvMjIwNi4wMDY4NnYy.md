# Federated Learning in Non-IID Settings Aided by Differentially Private Synthetic Data

Huancheng Chen, Haris Vikalo (2023)

## 🧩 Problem to Solve

본 논문은 연합 학습(Federated Learning, FL) 환경에서 데이터가 독립 동일 분포(Independent and Identically Distributed, IID)를 따르지 않는 **Non-IID 설정**일 때 발생하는 성능 저하 문제를 해결하고자 한다.

일반적인 FL 시스템에서는 각 클라이언트가 보유한 데이터의 분포가 서로 다르며, 특히 일부 클라이언트가 특정 클래스의 데이터를 전혀 가지고 있지 않은 데이터 희소성(Data Scarcity) 문제가 발생한다. 이러한 데이터의 불균형은 로컬 모델의 편향을 유발하고, 결과적으로 서버에서 집계된 글로벌 모델의 성능을 크게 떨어뜨리는 '모델 드리프트(Model Drift)' 현상을 초래한다. 본 연구의 목표는 프라이버시를 보호하면서도 클라이언트 간의 데이터 다양성을 높여, Non-IID 환경에서도 강건한 글로벌 모델을 학습시키는 알고리즘을 개발하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **FedDPMS(Federated Differentially Private Means Sharing)**라는 프레임워크를 통해 클라이언트가 자신의 로컬 데이터셋을 합성 데이터(Synthetic Data)로 보강하는 것이다.

중심적인 직관은 모든 클라이언트가 원본 데이터를 공유하는 대신, 신뢰할 수 있는 서버가 매개체가 되어 **잠재 표현(Latent Representation)의 평균값**을 차분 프라이버시(Differential Privacy, DP)가 적용된 상태로 공유하게 하는 것이다. 이를 통해 데이터를 많이 가진 클라이언트(Assisting Client)의 정보가 데이터를 적게 가진 클라이언트(Benefiting Client)에게 전달되며, 수신 측은 Variational Auto-Encoder(VAE)의 디코더를 이용해 부족한 클래스의 데이터를 합성하여 로컬 데이터셋의 균형을 맞춘다.

## 📎 Related Works

논문에서는 Non-IID 문제를 해결하기 위한 기존 연구를 두 가지 범주로 분류한다.

1. **모델 집계 및 로컬 드리프트 감소 방법**:
    * PFNM, FedMA, FedNova 등은 서버의 집계 단계를 개선하려 했으며, FedProx, SCAFFOLD, Moon, FedDyn 등은 정규화 항(Regularization term)이나 대조 학습(Contrastive learning)을 통해 로컬 학습 시의 드리프트를 줄이려 했다.
    * **한계**: 이러한 방법들은 데이터 분포 차이가 극심하여 특정 클래스가 완전히 누락된 경우(Imbalanced partition)에는 효과가 제한적이다.

2. **도메인 일반화 및 데이터 증강 방법**:
    * FedDG는 진폭 스펙트럼을 공유하고, FedMix는 Mixup을 통해 평균화된 데이터를 공유하며, FedDA는 조건부 VAE(CVAE)를 사용한다.
    * **한계**: FedDG는 위상 스펙트럼 공유의 프라이버시 문제로 클래스 다양성을 높이지 못하며, FedMix는 원본 데이터의 평균을 사용하므로 프라이버시 침해 위험이 크다. FedDA는 Non-IID가 심할 때 로컬 지식이 글로벌 기대치에서 벗어나는 문제가 있다.

FedDPMS는 VAE 기반의 잠재 공간 평균 공유와 차분 프라이버시 메커니즘을 결합하여, 기존 방식보다 프라이버시 보호 수준을 높이면서도 클래스 다양성을 효과적으로 확보한다는 점에서 차별점을 가진다.

## 🛠️ Methodology

### 1. 네트워크 아키텍처 및 학습 목표

모델은 **인코더(Encoder), 디코더(Decoder), 분류기(Classifier)**의 세 가지 구성 요소로 이루어져 있다.

* **인코더**: 입력 데이터를 잠재 변수 $z \sim \mathcal{N}(\mu, \sigma^2_z)$로 인코딩한다.
* **디코더**: 잠재 변수 $z$를 다시 원래의 데이터 공간으로 복원한다.
* **분류기**: 잠재 변수 $z$를 입력받아 각 클래스에 속할 확률인 범주형 벡터 $p$를 출력한다.

학습 손실 함수 $\ell$은 다음과 같이 정의된다.
$$\ell = \ell_1 + \lambda(\ell_2 + \ell_3)$$
여기서 각 항의 의미는 다음과 같다.

* $\ell_1$: 예측값과 실제 정답 사이의 교차 엔트로피(Cross-Entropy) 손실이다.
* $\ell_2$: 사전 분포 $p(z)$와 근사 분포 $q(z)$ 사이의 KL 발산(Kullback-Leibler Divergence)이다.
* $\ell_3$: 원본 데이터 $x$와 복원된 데이터 $\hat{x}$ 사이의 평균 제곱 오차(MSE) 손실이다.
$\lambda$는 하이퍼파라미터이며, 초기 단계에서는 세 손실을 모두 최적화하고 이후 단계에서는 $\ell_1$에 집중하여 학습한다.

### 2. FedDPMS 학습 절차

학습은 크게 두 단계로 진행된다.

#### 단계 I: 예비 학습 (Preliminary Training)

각 클라이언트는 로컬에서 VAE를 학습시키고, 인코더와 분류기의 가중치를 서버에 전송한다. 서버는 이를 집계하여 글로벌 모델을 생성하고 다시 배포한다. 마지막 라운드에서는 디코더 가중치까지 공유하여 글로벌 디코더 $w_d$를 구축한다.

#### 단계 II: 2차 학습 (Secondary Training)

다음의 4가지 과정이 반복된다.

1. **클라이언트 매칭 (Client Matching)**: 서버는 각 클라이언트의 데이터 분포 정보(가장 많은 클래스와 적은 클래스의 인덱스)를 바탕으로, 데이터가 부족한 클라이언트(Benefiting)와 보완적인 데이터를 가진 클라이언트(Assisting)를 매칭한다.
2. **데이터 합성 (Data Synthesis)**: 매칭된 Benefiting 클라이언트는 서버로부터 전달받은 (노이즈가 섞인) 잠재 평균값과 글로벌 디코더 $w_d$를 사용하여 부족한 클래스의 합성 데이터를 생성하고, 이를 로컬 데이터셋에 추가한다.
3. **모델 학습 (Model Training)**: 증강된 데이터셋을 사용하여 인코더와 분류기를 로컬 학습시키고 업데이트된 모델을 서버에 전송한다.
4. **DPMS (Differentially Private Means Sharing)**: 아직 정보를 공유하지 않은 클라이언트는 자신의 데이터 중 가장 풍부한 클래스들에 대해 잠재 표현의 평균 $\bar{z}$를 계산하고, 여기에 가우시안 노이즈 $\delta \sim \mathcal{N}(0, \sigma^2)$를 추가하여 $\tilde{z}$를 생성한다.
    * **검증 단계**: 노이즈가 너무 심해 합성 데이터의 품질이 떨어지는 것을 방지하기 위해, 생성된 $\tilde{z}$를 글로벌 디코더로 복원한 뒤 로컬 분류기로 예측했을 때 정답 클래스와 일치하는 경우에만 서버에 전송한다.

### 3. 프라이버시 보호 메커니즘

본 논문은 가우시안 노이즈 메커니즘을 통해 $(\epsilon, \delta)$-차분 프라이버시를 달성한다. 잠재 표현의 평균을 계산하는 함수 $f$의 민감도 $S_f$는 활성화 함수로 시그모이드를 사용함에 따라 $1/m$ (여기서 $m$은 샘플 수)으로 제한된다. 노이즈의 표준편차 $\sigma$를 적절히 설정함으로써 개별 데이터 포인트의 잠재 표현이 유출될 위험을 수학적으로 억제한다.

## 📊 Results

### 1. 실험 설정

* **데이터셋**: Fashion-MNIST, CIFAR-10, CIFAR-100.
* **Non-IID 설정**: 디리클레 분포(Dirichlet distribution)의 집중 파라미터 $\beta$를 조절하여 데이터 불균형 정도를 설정하였다 ($\beta=0.1$일 때 가장 불균형함).
* **비교 대상**: FedAvg, FedProx, FedMix, Moon.
* **평가 지표**: 테스트 정확도(Test Accuracy).

### 2. 정량적 결과

* **정확도 향상**: $\beta=0.5$ 설정에서 FedDPMS는 FedAvg 대비 FMNIST에서 3.3%, CIFAR-10에서 4.5%, CIFAR-100에서 6.6%의 성능 향상을 보였으며, 모든 비교 대상 중 가장 높은 성능을 기록하였다.
* **데이터 불균형에 대한 강건성**: $\beta$ 값이 작아져 데이터 불균형이 심해질수록 다른 방법론들의 성능은 급격히 하락하는 반면, FedDPMS는 상대적으로 높은 정확도를 유지하였다.
* **확장성(Scalability)**: 클라이언트 수를 10명에서 50명까지 늘렸을 때도 FedDPMS는 일관되게 타 방법론보다 우수한 성능을 보였으며, 특히 클라이언트 수가 적을 때 FedAvg 대비 최대 22.3% 이상의 높은 성능 향상을 기록하였다.
* **수렴 속도**: FedDPMS는 타 방법론보다 빠른 수렴 속도를 보였으며, 특히 합성 데이터를 통한 데이터 증강이 이루어지는 시점에서 정확도가 급격히 상승하는 양상을 보였다.

## 🧠 Insights & Discussion

### 강점

FedDPMS는 단순한 모델 가중치 집계나 정규화 방식에서 벗어나, **데이터 분포 자체를 보완**하는 접근 방식을 취했다. 특히 VAE의 잠재 공간에서 평균값을 공유함으로써 원본 데이터의 직접적인 노출을 피하고, 차분 프라이버시(DP)와 검증 단계를 통해 프라이버시와 데이터 품질이라는 두 마리 토끼를 잡은 점이 돋보인다.

### 한계 및 고려사항

* **자원 소모**: VAE 구조를 추가로 사용하고 합성 데이터를 생성해야 하므로, 기존 FedAvg 대비 계산 비용과 메모리 사용량이 증가한다.
* **가정**: 서버가 클라이언트의 클래스 분포(어떤 클래스가 많고 적은지)에 대한 부분적인 정보를 가지고 있어야 매칭이 가능하다는 가정이 필요하다.
* **하이퍼파라미터 의존성**: 노이즈의 표준편차 $\sigma$와 데이터 생성 쿼터 $\alpha$, 예비 학습 라운드 수 등에 따라 성능이 달라질 수 있다.

### 비판적 해석

본 논문은 합성 데이터가 Non-IID 문제를 해결하는 강력한 도구가 될 수 있음을 입증하였다. 다만, 합성 데이터의 질이 실제 데이터의 분포를 얼마나 정확하게 모사하는지에 대한 정성적 분석보다는 최종 분류 정확도라는 정량적 지표에 의존하고 있다. 향후 연구에서는 생성된 합성 데이터가 모델의 결정 경계(Decision Boundary)에 구체적으로 어떤 영향을 주는지에 대한 분석이 추가될 필요가 있다.

## 📌 TL;DR

본 논문은 Non-IID 데이터 환경의 연합 학습에서 발생하는 성능 저하를 해결하기 위해, **차분 프라이버시가 적용된 잠재 표현 평균 공유(FedDPMS)** 방식을 제안한다. 클라이언트는 VAE를 통해 부족한 클래스의 데이터를 합성하여 로컬 데이터셋을 보강하며, 이는 프라이버시를 보호하면서도 모델 드리프트를 효과적으로 억제한다. 실험 결과, CIFAR-100 등 복잡한 데이터셋과 극심한 불균형 설정에서도 기존 SOTA 방법론들을 상회하는 성능과 빠른 수렴 속도를 보여, 향후 실질적인 Non-IID 연합 학습 시스템 구축에 중요한 기여를 할 것으로 평가된다.
