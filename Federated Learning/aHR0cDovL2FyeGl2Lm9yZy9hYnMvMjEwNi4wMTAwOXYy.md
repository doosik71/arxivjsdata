# FedHealth 2: Weighted Federated Transfer Learning via Batch Normalization for Personalized Healthcare

Yiqiang Chen, Wang Lu, Jindong Wang and Xin Qin (2021)

## 🧩 Problem to Solve

본 논문은 의료 헬스케어 분야의 연합 학습(Federated Learning, FL)에서 발생하는 **도메인 시프트(Domain Shift)** 문제를 해결하고자 한다. 의료 데이터는 환자나 의료 기관마다 데이터 분포가 서로 다른 Non-IID(Not Identically and Independently Distributed) 특성이 강하며, 이는 기존의 표준 연합 학습 알고리즘인 FedAvg의 성능 저하나 발산으로 이어진다.

특히, 의료 분야에서는 데이터 프라이버시와 보안 규정으로 인해 데이터를 직접 공유하는 것이 불가능하며, 각 클라이언트(사용자 또는 병원)에게 최적화된 **개인화된 모델(Personalized Model)**을 제공하는 것이 필수적이다. 이전 연구인 FedHealth는 이러한 문제를 해결하려 했으나, 클라이언트 간에 상당한 양의 데이터를 공유해야 한다는 비현실적인 제약이 있었다. 따라서 본 논문의 목표는 공통 데이터를 공유하지 않고도 도메인 시프트를 극복하고 각 클라이언트에 최적화된 개인화 모델을 생성하는 FedHealth 2 알고리즘을 제안하는 것이다.

## ✨ Key Contributions

FedHealth 2의 핵심 아이디어는 **클라이언트 간의 유사도(Similarity)를 기반으로 한 가중치 적용 모델 집계(Weighted Aggregation)**와 **로컬 배치 정규화(Local Batch Normalization, BN)의 유지**를 결합하는 것이다.

1.  **유사도 기반 가중치 집계**: 사전 학습된 모델(Pretrained Model)을 통해 각 클라이언트 데이터의 통계적 특성을 추출하고, 이를 바탕으로 클라이언트 간의 유사도를 계산하여 모델 파라미터를 가중 평균한다. 이는 Label Shift와 같은 분포 차이를 해결하는 데 도움을 준다.
2.  **로컬 BN 유지**: 모델 집계 시 BN 레이어의 파라미터는 서버로 전송하거나 업데이트하지 않고 각 클라이언트가 로컬하게 유지하도록 한다. 이는 Feature Shift 문제를 해결하여 개인화를 달성하는 핵심 기법이다.
3.  **데이터 비공유 개인화**: 공통 데이터셋 없이도 사전 학습된 모델의 출력 통계값만을 활용해 유사도를 측정함으로써 프라이버시를 보호하면서도 높은 개인화 성능을 달성한다.

## 📎 Related Works

본 논문은 다음과 같은 기존 연구들의 한계를 지적하며 차별점을 제시한다.

-   **FedAvg**: 가장 기초적인 FL 알고리즘이나, Non-IID 데이터 상황에서 성능이 크게 저하되는 문제가 있다.
-   **FedProx & FedPer**: Non-IID 문제를 해결하기 위해 근사 항(Proximal term)을 추가하거나 개인화 레이어를 유지하는 방식을 제안했으나, 헬스케어 특유의 도메인 시프트를 정밀하게 해결하지 못한다.
-   **FedHealth**: 헬스케어를 위한 연합 전이 학습 프레임워크를 제안했지만, 많은 양의 데이터를 공유해야 한다는 현실적인 한계가 있다.
-   **FedBN**: 로컬 BN을 유지하여 Feature Shift를 완화하지만, 클라이언트 간의 유사도를 고려하지 않아 Label Shift 문제에 취약하며 모든 클라이언트에게 동일한 가중치를 적용한다.

## 🛠️ Methodology

### 전체 파이프라인
FedHealth 2의 전체 프로세스는 다음과 같은 순서로 진행된다.
1.  서버가 모든 클라이언트에게 사전 학습된 모델 $f$를 배포한다.
2.  각 클라이언트는 로컬 데이터를 통해 특정 레이어의 출력 통계값$(\mu_i, \sigma_i)$을 계산하여 서버로 전송한다.
3.  서버는 전송받은 통계값을 바탕으로 클라이언트 간 유사도 행렬 $W$를 생성한다.
4.  각 클라이언트는 로컬 데이터를 통해 모델을 학습시키고 파라미터를 서버에 전송한다.
5.  서버는 유사도 행렬 $W$를 이용하여 가중 평균된 개인화 모델을 생성하고 이를 다시 각 클라이언트에게 배포한다.

### 모델 파라미터 분리 및 집계
모델의 파라미터 $\theta_i$를 BN 레이어의 파라미터 $\phi_i$와 그 외 레이어의 파라미터 $\psi_i$로 분리한다. 
-   **BN 파라미터 ($\phi_i$)**: 로컬에 고정하며 업데이트하지 않는다.
-   **기타 파라미터 ($\psi_i$)**: 서버에서 다음과 같이 가중 평균하여 업데이트한다.

$$\phi_{t+1}^i = \phi_t^i$$
$$\psi_{t+1}^i = \sum_{j=1}^N w_{ij} \psi_t^j$$

여기서 $w_{ij}$는 클라이언트 $i$와 $j$ 사이의 유사도를 나타내는 가중치이다.

### 유사도 행렬 $W$ 계산 방법
사전 학습된 모델의 각 BN 레이어 입력 $z_{i,l}$을 가우시안 분포 $N(\mu_{i,l}, \sigma_{i,l})$로 가정한다. 두 클라이언트 간의 거리를 측정하기 위해 **Wasserstein 거리**를 사용하며, 계산 효율성을 위해 각 채널이 독립적이라고 가정하여 다음과 같이 근사한다.

$$W_2^2(N(\mu_{i,l}, \sigma_{i,l}), N(\mu_{j,l}, \sigma_{j,l})) = ||\mu_{i,l} - \mu_{j,l}||^2 + ||\sqrt{r_{i,l}} - \sqrt{r_{j,l}}||^2$$

전체 레이어 $L$에 대한 거리 합 $d_{i,j}$를 구한 뒤, 가중치 $w_{i,j}$는 거리의 역수에 비례하도록 설정한다. 하이퍼파라미터 $\lambda$는 자기 자신의 모델에 부여하는 가중치이다.

$$w_{i,j} = 
\begin{cases} 
\lambda, & j=i \\
(1-\lambda) \times \frac{1/d_{i,j}}{\sum_{k=1, k \neq i}^N 1/d_{i,k}}, & j \neq i 
\end{cases}$$

### 확장 모델
-   **d-FedHealth 2**: BN 레이어 대신 분류기(Classify layer)의 입력 통계값을 사용하여 유사도를 측정한다.
-   **f-FedHealth 2**: 사전 학습된 모델이 없는 경우, FedBN을 몇 라운드 수행하여 얻은 BN 레이어의 Running Mean과 Running Variance를 통계값으로 활용한다.

## 📊 Results

### 실험 설정
-   **데이터셋**: 
    -   **PAMAP**: 인간 활동 인식 데이터셋 (18개 활동, 27개 채널). Dirichlet 분포를 통해 Non-IID하게 분할.
    -   **COVID-19**: 흉부 X-ray 이미지 데이터셋 (4개 클래스).
-   **비교 대상**: Base(로컬 학습), FedAvg, FedBN, FedProx, FedPer.
-   **지표**: 분류 정확도(Accuracy) 및 수렴 속도.

### 주요 결과
1.  **정확도 향상**: PAMAP 데이터셋에서 FedHealth 2는 다른 모든 베이스라인보다 월등한 성능을 보였으며, 평균적으로 **10% 이상의 정확도 향상**을 달성했다. COVID-19 데이터셋에서도 소폭(약 2%) 향상된 결과를 보였다.
2.  **Label Shift 대응**: Ablation Study 결과, 가중치 부여(Weighting)를 제거하면 FedBN으로 퇴화하며 성능이 급격히 하락한다. 이는 본 제안 방식이 Label Shift 문제를 효과적으로 해결하고 있음을 입증한다.
3.  **Feature Shift 대응**: 로컬 BN을 유지하는 LBN 방식이 공통 BN을 사용하는 SBN 방식보다 성능이 높게 나타나, Feature Shift 해결 능력을 확인했다.
4.  **수렴 속도**: FedHealth 2는 약 10~20 라운드 내에 빠르게 수렴하는 반면, FedBN은 400 라운드 이상이 소요되어 학습 효율성이 극도로 높음을 보였다.

## 🧠 Insights & Discussion

본 논문은 헬스케어 데이터의 특성인 Non-IID 문제를 두 가지 관점에서 동시에 접근했다. **로컬 BN 유지**를 통해 개별 클라이언트의 특성(Feature Shift)을 보존하고, **유사도 기반 가중치 집계**를 통해 유사한 특성을 가진 클라이언트 간의 지식을 효율적으로 전이(Label Shift 해결)함으로써 개인화된 모델을 성공적으로 구축했다.

특히, 사전 학습된 모델이 없는 환경에서도 `f-FedHealth 2`라는 대안을 제시함으로써 실용성을 높였다. 다만, COVID-19 데이터셋에서 성능 향상 폭이 적었던 이유는 클래스 수가 적어 태스크 자체가 상대적으로 쉬웠기 때문으로 분석된다. 

비판적으로 살펴보면, 유사도 계산을 위해 사용되는 $\lambda$ 값이나 사전 학습 모델의 품질이 최종 성능에 영향을 줄 가능성이 크다. 또한, Wasserstein 거리의 근사치 사용이 실제 데이터 분포의 복잡성을 충분히 반영했는지에 대한 추가적인 분석이 필요할 수 있다.

## 📌 TL;DR

FedHealth 2는 헬스케어 분야의 Non-IID 데이터 문제를 해결하기 위해 **사전 학습된 모델 기반의 클라이언트 유사도 측정**과 **로컬 배치 정규화(BN) 유지**를 결합한 가중 연합 전이 학습 알고리즘이다. 이를 통해 데이터 프라이버시를 침해하지 않고도 각 사용자에게 최적화된 개인화 모델을 제공하며, 기존 FedAvg나 FedBN 대비 월등한 정확도 향상과 빠른 수렴 속도를 달성했다. 이 연구는 데이터 고립(Data Islanding) 문제가 심한 의료 AI 시스템의 실용적인 배포 모델로 활용될 가능성이 매우 높다.
