# Evaluating Multi-Global Server Architecture for Federated Learning

Asfia Kawnine, Hung Cao, Atah Nuh Mih, Monica Wachowicz (2023)

## 🧩 Problem to Solve

본 논문은 현재 Federated Learning (FL)의 주류 패러다임인 단일 글로벌 서버(Single Global Server) 구조가 가진 치명적인 약점인 단일 장애점(Single Point of Failure) 문제를 해결하고자 한다. 일반적인 FL 프레임워크에서는 모든 클라이언트의 모델 업데이트를 하나의 중앙 서버에서 집계하는데, 만약 이 중앙 서버에 통신 장애가 발생하거나 서버 자체가 다운될 경우 전체 시스템이 완전히 붕괴되는 리스크가 존재한다.

따라서 본 연구의 목표는 여러 개의 글로벌 서버를 배치하는 Multi-Global Server 아키텍처를 설계하고 구현하여, 서버 장애에 대한 결함 허용(Fault Tolerance) 능력을 높이고, 로컬 협업 및 지식 집계를 통해 학습 효율성을 향상시킬 수 있는지 평가하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 글로벌 서버의 역할을 다중화하여 시스템의 안정성과 유연성을 확보하는 것이다. 주요 기여 사항은 다음과 같다.

1. **Multi-Global Server 아키텍처 제안**: 단일 서버에 의존하지 않고 여러 글로벌 서버가 협력하여 모델 파라미터를 집계하고 교환하는 구조를 설계하였다.
2. **통신 결함 허용 메커니즘 도입**: 특정 글로벌 서버에 접속할 수 없는 경우, 다른 가용한 서버로 전환하여 통신을 유지하는 알고리즘을 통해 시스템의 강건성을 확보하였다.
3. **정형 데이터(Tabular Data) 기반의 FL 검증**: 이미지 데이터 중심의 기존 연구와 달리, 전기차(EV) 충전 에너지 소비 데이터라는 정형 데이터를 활용하여 제안하는 아키텍처의 실효성을 실험적으로 분석하였다.

## 📎 Related Works

논문에서는 Federated Learning의 기본 개념과 더불어 다양한 집계 전략 및 엣지 컴퓨팅과의 결합에 대해 설명한다.

- **Federated Learning (FL)**: 데이터 프라이버시를 유지하면서 분산된 장치에서 모델을 학습시키는 접근법이다.
- **집계 전략(Aggregate Strategies)**: 가장 기본이 되는 FedAvg 외에도, 모멘텀을 활용한 FedAvgM, 적응형 학습률을 사용하는 FedAdaGrad, FedYogi, FedAdam 등이 제안되었다. 본 논문에서는 데이터셋의 특성상 추가 최적화가 필요하지 않다고 판단하여 가장 보편적인 FedAvg를 채택하였다.
- **계층적 연합 학습(Hierarchical FL, HFL)**: 클라우드 서버와 클라이언트 사이에 엣지 서버와 같은 중간 집계자를 두어 통신 오버헤드를 줄이는 방식이다.
- **기존 연구의 한계**: 많은 연구가 FL의 성능이나 효율성 향상에 집중하고 있지만, 여전히 단일 중앙 서버에 의존하고 있어 서버 장애 시 전체 시스템이 중단되는 문제는 여전히 해결되지 않은 과제로 남아 있다.

## 🛠️ Methodology

### 전체 시스템 구조

본 논문이 제안하는 아키텍처는 여러 대의 글로벌 서버와 다수의 엣지 클라이언트로 구성된다. 각 클라이언트는 자신에게 할당된 글로벌 서버와 통신하며, 서버들은 서로 협력하여 글로벌 모델을 업데이트한다.

### 모델 업데이트 방정식

글로벌 서버에서의 파라미터 업데이트 과정은 다음과 같은 수식으로 표현된다.

$$\theta_i(t+1) = \theta_i(t) + \eta \sum (w_i * \Delta \mu_j)$$

여기서 각 변수의 의미는 다음과 같다.

- $\theta_i(t+1)$: 시간 단계 $t+1$에서의 업데이트된 글로벌 모델 파라미터이다.
- $\theta_i(t)$: 현재 시간 단계 $t$에서의 글로벌 모델 파라미터이다.
- $\eta$: 파라미터 업데이트의 스텝 크기를 조절하는 학습률(Learning Rate)이다.
- $\sum (w_i * \Delta \mu_j)$: 각 클라이언트 $j$로부터 전달된 로컬 모델 업데이트 $\Delta \mu_j$에 가중치 $w_i$를 곱한 값들의 합이다. 가중치는 데이터의 크기나 클라이언트의 신뢰도에 따라 결정될 수 있다.

### 통신 결함 허용 알고리즘

서버 장애 시 대응을 위해 'try-except' 블록 기반의 서버 전환 알고리즘을 구현하였다. 클라이언트는 서버 주소 리스트 $S[1..n]$을 가지고 있으며, 첫 번째 서버 접속에 실패할 경우 순차적으로 다음 서버에 접속을 시도하여 가용한 서버를 찾아 로컬 파라미터를 전송한다.

### 학습 절차 및 로컬 모델

- **로컬 모델**: 정형 데이터 학습에 최적화된 TabNet을 사용하였다. TabNet은 Attentive Transformer와 Feature Transformer를 통해 정형 데이터의 특성을 효과적으로 학습한다.
- **집계 방법**: 각 글로벌 서버는 클라이언트로부터 받은 파라미터를 FedAvg 알고리즘을 통해 가중 평균하여 글로벌 모델을 생성한다.

## 📊 Results

### 실험 환경

- **데이터셋**: NB Power에서 제공한 2019년 4월~2022년 6월 사이의 전기차(EV) 충전 이벤트 데이터셋을 사용하였다.
- **데이터 분할**: GPS 좌표를 기반으로 데이터를 9개 지역으로 나누어 엣지 클라이언트에 분산 배치하였다.
- **하드웨어 구성**: 글로벌 서버 1(Ubuntu Cloud Server), 글로벌 서버 2(Intel Core i7-4790 CPU), 엣지 클라이언트(reComputer-Edge AI Device)를 사용하여 환경을 구축하였다.

### 주요 결과

- **로컬 학습 결과**: 모든 엣지 클라이언트의 학습 손실(Training Loss)이 25 에포크(epoch) 이후 약 $0.1\%$ 수준으로 수렴하는 것을 확인하였다.
- **글로벌 모델 비교**: 두 대의 글로벌 서버에서 생성된 모델의 손실 함수를 비교한 결과, 두 모델 모두 3회차 집계(aggregation round) 이후 매우 유사한 손실 값으로 수렴하였다.
- **성능 차이**: 단일 서버 방식 대비 다중 서버 방식의 성능 차이는 $1\%$ 미만으로 나타났다. 즉, 서버를 다중화하더라도 모델의 예측 성능에는 큰 손실이 없음을 확인하였다.

## 🧠 Insights & Discussion

### 강점 및 성과

본 연구는 FL 시스템의 최대 약점 중 하나인 서버 가용성 문제를 해결하기 위해 Multi-Global Server 구조를 제안하고, 이를 실제 하드웨어와 정형 데이터를 통해 검증하였다. 특히, 서버 다중화가 모델의 정확도를 저하시키지 않으면서도 통신 장애에 대한 강력한 결함 허용 능력을 제공한다는 점을 입증하였다.

### 한계 및 비판적 해석

- **효율성 가설의 불일치**: 저자들은 다중 서버 배치가 학습 효율성(속도 및 정확도)을 높일 것이라고 가설을 세웠으나, 실제 결과에서는 성능 향상이 뚜렷하게 나타나지 않았다. 이는 단순히 서버를 늘리는 것만으로는 병렬 처리의 이득보다 통신 및 동기화 비용이 더 클 수 있음을 시사한다.
- **실험 규모의 제한**: 2대의 글로벌 서버와 소수의 클라이언트를 대상으로 한 예비 실험(Preliminary results) 단계이므로, 서버 수가 대폭 늘어났을 때의 확장성(Scalability)이나 서버 간의 파라미터 동기화 오버헤드에 대한 분석이 부족하다.
- **단순한 결함 허용**: 제안된 'try-except' 방식의 서버 전환은 매우 기본적인 수준이며, 실제 복잡한 네트워크 환경에서 서버 간의 일관성(Consistency)을 어떻게 유지할 것인지에 대한 깊이 있는 논의가 부족하다.

## 📌 TL;DR

본 논문은 단일 서버 기반 연합 학습(FL)의 장애 취약성을 해결하기 위해 **다중 글로벌 서버 아키텍처**를 제안하였다. 전기차 충전 에너지 소비 데이터를 활용한 실험 결과, 서버 다중화가 모델 성능에 거의 영향을 주지 않으면서($1\%$ 미만 차이) 서버 장애 시 다른 서버로 전환하는 **결함 허용(Fault Tolerance)** 능력을 제공함을 확인하였다. 이는 향후 고신뢰성이 요구되는 산업용 분산 학습 시스템 구축에 기여할 수 있을 것으로 보인다.
