# Efficient Vertical Federated Learning with Secure Aggregation

Xinchi Qiu, Heng Pan, Wanru Zhao, Chenyang Ma, Pedro Porto Buarque de Gusmão, Nicholas D. Lane (2023)

## 🧩 Problem to Solve

본 논문은 수직 연합 학습(Vertical Federated Learning, VFL) 환경에서 데이터 프라이버시를 보호하면서도 효율적으로 모델을 학습시키는 방법을 해결하고자 한다. 일반적으로 연합 학습은 동일한 특성(feature)을 공유하고 서로 다른 샘플을 가진 수평적 분할(Horizontal Partitioning) 데이터셋에 집중되어 왔다. 그러나 금융 사기 탐지나 질병 진단과 같은 실제 사례에서는 동일한 데이터 포인트에 대해 서로 다른 기관이 서로 다른 특성을 보유하는 수직적 분할 상황이 빈번하게 발생한다.

수직 연합 학습의 핵심 문제는 학습 과정에서 참여자 간에 중간 활성화 값(intermediate activations)이나 그래디언트(gradients)를 교환해야 한다는 점이다. 이러한 정보들은 역으로 원본 데이터를 재구성(reconstruction)하는 데 사용될 수 있어 심각한 프라이버시 유출 위험을 초래한다. 기존의 동형 암호(Homomorphic Encryption, HE) 기반 솔루션들은 보안성은 높지만, 막대한 계산 및 통신 오버헤드가 발생하여 실용성이 떨어진다는 한계가 있다. 따라서 본 연구의 목표는 학습 성능을 유지하면서도 계산 효율성을 극대화한 보안 집계(Secure Aggregation) 기반의 VFL 프레임워크를 설계하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 수평 연합 학습에서 제안되었던 보안 집계(Secure Aggregation) 기법을 수직 연합 학습 구조에 맞게 재설계하여 적용하는 것이다. 

가장 중심적인 직관은 모든 데이터를 암호화하여 연산하는 대신, 각 참여자가 생성한 가벼운 마스킹 노이즈(masking noise)를 데이터에 더해 전송하고, 최종 집계 단계에서 이 노이즈들의 합이 0이 되어 상쇄되도록 설계하는 것이다. 이를 통해 무거운 동형 암호 연산을 제거하면서도, 집계 서버(aggregator)가 개별 참여자의 원본 데이터를 알 수 없게 만들어 프라이버시를 보호한다.

## 📎 Related Works

기존의 VFL 연구들은 주로 모델의 성능 향상에 집중해 왔으며, 보안 측면에서는 다음과 같은 접근 방식들이 있었다.
- **ID 보호 중심:** 일부 연구는 샘플 ID만을 보호하려 했으며, 원본 데이터 전체에 대한 보호는 부족했다.
- **임베딩 섭동(Perturbation):** 로컬 임베딩에 노이즈를 추가하여 프라이버시를 보호하려 했으나, 이는 임베딩 설계에 엄격한 제약이 따르며 전체적인 모델 성능 저하를 야기할 수 있다.
- **동형 암호(HE) 기반:** BlindFL, ACML, PrADA 등이 대표적이며, 데이터가 암호화된 상태에서 연산을 수행한다. 하지만 이 방식은 계산 비용이 매우 높고 통신 오버헤드가 크며, 다수 참여자 시나리오로 확장하는 데 어려움이 있다.

본 논문은 이러한 HE 기반 방식의 고비용 문제를 해결하기 위해, 가벼운 마스킹 기반의 보안 집계를 도입함으로써 성능 저하 없이 획기적인 속도 향상을 달성했다는 점에서 차별점을 갖는다.

## 🛠️ Methodology

### 전체 파이프라인 및 시스템 구조
본 시스템은 하나의 **Active Party**(정답 라벨과 일부 특성을 보유)와 여러 개의 **Passive Parties**(라벨 없이 보완적인 특성만을 보유)로 구성되며, 이들 사이에서 학습을 조율하는 **Central Aggregator**가 존재한다. 전체 과정은 Setup, Training, Testing의 세 단계로 나뉜다.

### 1. Setup Phase (키 교환)
참여자들이 안전하지 않은 채널을 통해 공유 비밀값(shared secrets)을 생성하는 단계이다.
- **ECDH(Elliptic-curve Diffie-Hellman)** 프로토콜을 사용하여 모든 클라이언트 쌍 사이에 공유 비밀값 $ss_{ij}$를 생성한다.
- 이 비밀값은 이후 데이터 전송 시 사용될 대칭 암호화 및 보안 집계를 위한 마스크 생성의 기초가 된다.

### 2. Training Phase (학습 절차)
학습은 다음과 같은 세부 절차로 진행된다.

**A. Mini-batch Selection**
Active Party가 학습할 배치를 선택한다. 이때 샘플 ID를 각 Passive Party와의 공유 비밀값 $ss_{0i}$로 암호화하여 Aggregator에게 보내고, Aggregator는 이를 다시 Passive Party들에게 브로드캐스트한다. 이를 통해 Passive Party는 자신이 보유한 샘플만 복호화할 수 있어, 다른 참여자의 데이터 구성 정보를 알 수 없다.

**B. Forward Pass (순전파)**
각 참여자는 로컬 특성을 이용해 임베딩을 생성하고, 이를 마스킹하여 전송한다. Passive Party $p$가 전송하는 마스크 벡터는 다음과 같다.
$$\left( \sum_{j=1}^{B} \sum_{i=k}^{l} \mathbb{1}(f_i^{(j)} \in D_p) w_i f_i^{(j)} \right) + n_p$$
여기서 $n_p$는 무작위 노이즈 벡터이며, 모든 참여자의 노이즈 합 $\sum_p n_p = 0$이 되도록 설계된다. 구체적으로 의사 난수 생성기(PRG)를 이용하여 다음과 같이 생성한다.
$$n_i = -\sum_{j<i} \text{PRG}(ss_{ij}) + \sum_{j>i} \text{PRG}(ss_{ij})$$
Aggregator는 모든 참여자의 마스크 벡터를 합산하여 $n_p$를 상쇄시키고, 최종 결과값 $z^{(k)}$를 정확하게 계산한다.

**C. Backward Pass (역전파)**
각 참여자는 손실 함수 $L$에 대한 로컬 특성의 부분 그래디언트를 계산한다. 이때도 순전파와 동일하게 마스크 $n_p$를 더해 전송한다.
$$\left( \sum_{k=1}^{B} \mathbb{1}(f_i^{(k)} \in D_p) \nabla L(f_i^{(k)}) \right) + n_p$$
Aggregator는 이를 합산하여 Active Party에게 전달하며, 최종적인 모델 파라미터 업데이트는 오직 Active Party만이 수행할 수 있다.

### 3. Testing Phase (추론 절차)
학습된 모델 가중치 $w$를 사용하여, Active Party와 Passive Party들이 각각 마스킹된 활성화 값을 생성하여 Aggregator에게 보내고, Aggregator가 이를 합산하여 최종 예측값을 도출한다.

## 📊 Results

### 실험 설정
- **데이터셋:** Banking, Adult Income, Taobao (대규모 데이터셋) 등 3종을 사용하였다.
- **네트워크 구조:** 데이터셋별 특성 수에 따라 Linear Layer를 구성하였으며, ReLU 활성화 함수를 적용하였다.
- **비교 지표:** CPU 시간(ms) 및 통신 전송량(bytes)을 측정하여 unsecured VFL 대비 오버헤드를 분석하였고, 동형 암호(HE) 방식과의 속도를 비교하였다.

### 주요 결과
- **계산 및 통신 오버헤드:** 표 1과 2에 따르면, 보안 집계를 적용했을 때 발생하는 추가 CPU 시간과 통신량은 전체 비용에서 매우 작은 비중을 차지한다. 이는 마스크 합산 과정이 매우 효율적이기 때문이다.
- **HE 대비 효율성:** 가장 핵심적인 결과로, 본 논문의 보안 집계 방식은 동형 암호(HE) 방식 대비 **$9.1 \times 10^2$에서 $3.8 \times 10^4$배의 속도 향상**을 보였다. 
- **학습 성능:** 보안 모듈을 추가했음에도 불구하고 모델의 예측 정확도나 학습 성능에는 영향을 주지 않았음을 확인하였다.

## 🧠 Insights & Discussion

### 강점 및 프라이버시 보장
본 연구는 **Honest-but-curious** 위협 모델(프로토콜은 준수하지만 가능한 많은 정보를 얻으려는 공격자) 하에서 강력한 보안성을 제공한다. 마스킹된 그래디언트를 사용함으로써 Aggregator가 개별 참여자의 데이터를 재구성하거나 멤버십 추론 공격(membership inference attack)을 수행하는 것을 방지한다. 또한, 제안된 방식은 참여자 수나 데이터 분할 방식에 관계없이 적용 가능한 확장성(Scalability)을 가진다.

### 한계 및 고려사항
- **키 유출 위험:** 이론적으로는 안전하지만 실제 환경에서 비밀 키가 유출될 위험이 있다. 이를 방지하기 위해 저자들은 $K$번의 반복 학습마다 키를 주기적으로 재생성(regenerate)할 것을 권장한다.
- **능동적 공격자:** 현재 구현은 정직한 참여자를 가정하고 있으며, 악의적인 공격자(malicious adversaries)를 방어하기 위해서는 PKI(공개키 기반 구조) 등의 추가적인 인증 메커니즘이 필요하다.

### 비판적 해석
본 논문은 VFL에서 보안과 효율성이라는 두 마리 토끼를 잡기 위해 매우 실용적인 접근을 취했다. 특히 HE의 비효율성을 수치적으로 극명하게 보여줌으로써, 실제 산업 현장에서 VFL을 적용할 때 어떤 보안 모듈을 선택해야 하는지에 대한 명확한 가이드를 제시한다. 다만, 실험에서 사용한 모델이 단순한 Linear Layer 위주라는 점은 향후 더 복잡한 딥러닝 구조에서도 동일한 효율성이 유지되는지에 대한 추가 검증이 필요함을 시사한다.

## 📌 TL;DR

본 논문은 수직 연합 학습(VFL)의 프라이버시 문제를 해결하기 위해, 무거운 동형 암호(HE) 대신 가벼운 **보안 집계(Secure Aggregation)** 기반의 마스킹 기법을 도입하였다. 이를 통해 학습 성능 손실 없이 HE 대비 최대 **$3.8 \times 10^4$배의 속도 향상**을 달성하였으며, 실용적인 수준의 낮은 오버헤드로 데이터 프라이버시를 보장하는 VFL 프레임워크를 제안하였다. 이 연구는 향후 대규모 산업 데이터 기반의 보안 협력 학습 시스템 구축에 중요한 기반이 될 가능성이 높다.
