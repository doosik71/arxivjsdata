# SplitFedZip: Learned Compression for Data Transfer Reduction in Split-Federated Learning

Chamani Shiranthika, Hadi Hadizadeh, Parvaneh Saeedi, Ivan V. Bajić (2024)

## 🧩 Problem to Solve

본 논문은 Split-Federated Learning (SplitFed) 환경에서 발생하는 막대한 데이터 전송량으로 인한 통신 병목 현상을 해결하고자 한다. Split-Federated Learning은 Federated Learning (FL)의 데이터 프라이버시 보호 능력과 Split Learning (SL)의 클라이언트 계산 부담 완화라는 장점을 결합한 프레임워크이다.

그러나 SplitFed 네트워크는 학습 과정에서 클라이언트와 서버 간에 수많은 특징 맵(features)과 기울기(gradients)를 주고받아야 하므로, 네트워크 지연(latency), 대역폭 제한, 동기화 오버헤드와 같은 심각한 통신 문제에 직면하게 된다. 특히 데이터 프라이버시가 매우 중요한 의료 영상 분석 분야에서 SplitFed의 활용도를 높이기 위해서는 이러한 데이터 전송량을 획기적으로 줄이는 기술이 필수적이다. 따라서 본 연구의 목표는 학습된 압축(learned compression) 기법을 통해 모델의 정확도를 유지하면서 데이터 전송량을 최소화하는 SplitFedZip 방법론을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **Rate-Distortion (RD) 최적화 기반의 학습된 압축(learned compression)을 SplitFed의 통신 파이프라인에 통합**하는 것이다.

기존의 압축 방식들이 단순한 양자화(quantization)나 희소화(sparsification)에 의존했던 것과 달리, SplitFedZip은 데이터의 통계적 특성에 적응할 수 있는 신경망 기반의 코덱(codec)을 사용한다. 특히 특징(features)과 기울기(gradients)라는 두 가지 서로 다른 성격의 데이터를 모두 압축 대상으로 삼으며, 이를 모델 학습과 동시에 최적화하는 end-to-end RD 최적화 방식을 도입하여 통신 효율성을 극대화하였다.

## 📎 Related Works

기존의 분산 학습 환경에서의 통신 효율화 연구는 크게 두 갈래로 나뉜다. Federated Learning에서는 모델 업데이트의 크기를 줄이기 위한 구조적 업데이트, 양자화, 지식 증류(knowledge distillation) 등이 제안되었으며, Split Learning에서는 원형 컨볼루션(circular convolution)이나 오토인코더(AE)를 이용한 압축이 시도되었다.

특히 SplitFed 환경에서의 압축 연구는 매우 제한적이며, 기존 연구 중에서는 양자화나 단순 AE를 이용한 특징 압축 정도만 수행되었다. 본 논문은 기존 연구들이 정적인 압축 방식을 사용하거나 일부 데이터(특징)만 압축했던 한계를 지적하며, 특징과 기울기를 모두 포함하는 end-to-end RD 최적화 기반의 데이터 압축을 SplitFed에 최초로 적용했다는 점에서 차별성을 가진다.

## 🛠️ Methodology

### 1. 시스템 아키텍처

SplitFedZip은 의료 영상 세그멘테이션을 위해 Split U-Net 구조를 사용한다. 전체 모델은 다음과 같이 세 부분으로 분할된다.

- **Front End (FE):** 클라이언트가 보유하며 이미지 입력 데이터를 처리한다.
- **Server Model:** 서버가 보유하며 모델의 대부분의 파라미터를 담당한다.
- **Back End (BE):** 클라이언트가 보유하며 정답(Ground Truth) 레이블에 접근하여 최종 결과를 도출한다.

이 시스템에는 두 개의 분할 지점(Split point, $S1, S2$)이 존재하며, 각 지점에서 데이터를 압축하고 복원하기 위한 **학습 가능한 코덱(Learned Codecs)**이 배치된다. 전방 전달(forward pass) 시에는 FE에서 생성된 특징이 압축되어 서버로 전송되고, 역전파(backward pass) 시에는 서버에서 계산된 기울기가 압축되어 클라이언트로 전송된다.

### 2. 사용된 코덱 (Codecs)

본 연구에서는 두 가지 종류의 코덱 아키텍처를 실험하였다.

- **Custom AE:** 컨볼루션 층과 Generalized Divisive Normalization (GDN) 층을 사용하여 데이터를 변환하고, 엔트로피 병목(entropy bottleneck)을 통해 양자화 및 엔트로피 코딩을 수행한다.
- **ChengAT:** 어텐션 메커니즘이 포함된 심층 컨볼루션 신경망으로, 데이터의 복잡한 부분에 비트를 더 많이 할당하는 정교한 엔트로피 모델링을 수행한다.

### 3. 훈련 목표 및 손실 함수

각 클라이언트는 Split U-Net과 코덱을 동시에 학습시키며, 이때 사용하는 손실 함수는 다음과 같다.

$$L = L_r + \lambda \cdot \{L_{Dice} + L_{mse}\}$$

- $L_r$: 코덱의 엔트로피 추정기로부터 얻은 비트 전송률(Rate term)으로, 값이 작을수록 압축률이 높다.
- $L_{Dice}$: 세그멘테이션 정확도를 측정하는 Dice coefficient 손실이다.
- $L_{mse}$: 코덱의 입력 데이터와 복원된 데이터 사이의 평균 제곱 오차(Mean Squared Error)로, 복원 품질(Distortion term)을 의미한다.
- $\lambda$: 비트 전송률과 왜곡(정확도) 사이의 가중치를 조절하는 하이퍼파라미터이다. $\lambda$가 클수록 정확도에, 작을수록 압축률에 우선순위를 둔다.

### 4. 압축 스킴 (Compression Schemes)

- **FG scheme:** 전방 전달의 특징($F$)과 역전파의 기울기($G$)를 모두 압축한다.
- **F scheme:** 전방 전달의 특징($F$)만 압축하고 기울기는 압축하지 않는다.

## 📊 Results

### 1. 실험 설정

- **데이터셋:** Blastocyst(다중 클래스 세그멘테이션), HAM10K(이진 클래스 세그멘테이션)
- **평가 지표:** Mean Jaccard Index (MJI), Bits Per Pixel (BPP), 압축률 (Compression Ratio, CR)
- **환경:** 5명의 클라이언트, Adam 옵티마이저, PyTorch 및 CompressAI 라이브러리 사용.

### 2. 주요 결과

- **Rate-Accuracy Trade-off:** $\lambda$ 값에 따라 비트 전송률과 모델 정확도(MJI) 사이에 뚜렷한 상관관계가 나타났다. 특히 HAM10K 데이터셋의 경우 압축이 일종의 정규화(regularizer) 역할을 하여, 특정 압축률에서 오히려 무압축(NC) 상태보다 약간 더 높은 성능을 보이기도 했다.
- **코덱 성능 비교:** ChengAT 코덱이 AE 코덱보다 훨씬 우수한 성능을 보였다. Bjøntegaard Delta (BD) 분석 결과, ChengAT는 F 스킴에서 AE 대비 약 68.7%, FG 스킴에서 약 24.0%의 비트 전송률 절감 효과를 거두었다.
- **데이터 전송량 감소:** HAM10K 데이터셋 기준, 압축률(CR)이 $10^3$에서 $10^{6.5}$ 범위에 달했다. 적절한 $\lambda$를 선택했을 때, 모델 성능 저하 없이 데이터 전송량을 최소 3자리수(1,000배) 이상 줄일 수 있음을 확인하였다.
- **기울기 압축의 효율성:** 실험 결과 기울기의 비트 전송률이 특징의 비트 전송률보다 낮게 나타났다(Blastocyst 약 20%, HAM10K 약 6% 낮음). 이는 기울기가 특징보다 더 높은 압축 가능성을 가지고 있음을 시사한다.

### 3. 기존 방법론과의 비교

Ayad et al. (2021)의 AE 기반 압축 방식과 비교했을 때, SplitFedZip는 동일한 정확도 수준에서 데이터 전송량을 획기적으로 줄였다.

- **Blastocyst 데이터셋:** 전송량을 약 32배 감소시켰다.
- **HAM10K 데이터셋:** 전송량을 약 89배 감소시켰다.
- (일부 설정에서는 최대 5000배까지 전송량을 줄이면서도 유사한 성능을 유지하였다.)

## 🧠 Insights & Discussion

### 강점 및 기여

본 연구는 SplitFed 환경에서 특징과 기울기를 동시에 압축하는 end-to-end RD 최적화 프레임워크를 제안함으로써, 통신 오버헤드라는 고질적인 문제를 매우 효과적으로 해결하였다. 특히 정교한 엔트로피 모델링을 사용하는 ChengAT와 같은 최신 코덱을 통합하여 단순 양자화보다 훨씬 높은 압축 효율을 달성했다는 점이 고무적이다.

### 비판적 해석 및 한계

1. **계산 복잡도 증가:** 데이터 전송량은 획기적으로 줄었으나, 클라이언트와 서버 단에서 압축 및 복원을 위한 인코더/디코더 신경망을 추가로 실행해야 하므로 계산 비용(FLOPs)이 증가한다. 논문에서는 전송 지연 감소가 계산 증가분보다 크다고 가정하고 있으나, 이에 대한 정량적인 계산 시간 분석은 부족하다.
2. **데이터셋 특성 의존성:** HAM10K와 같이 상대적으로 단순하거나 규모가 큰 데이터셋에서는 압축이 정규화 효과를 주어 성능이 향상되었으나, Blastocyst와 같이 복잡한 다중 클래스 문제에서는 $\lambda$ 값에 따른 성능 하락이 더 민감하게 나타났다. 이는 실제 적용 시 데이터 특성에 따른 정밀한 $\lambda$ 튜닝이 필요함을 의미한다.
3. **실제 네트워크 환경 미반영:** 시뮬레이션 환경에서의 BPP 측정 위주로 결과가 제시되었으며, 실제 불안정한 네트워크 환경(패킷 손실, 지터 등)에서 압축된 비트스트림이 어떻게 작동하는지에 대한 실증적 분석은 포함되지 않았다.

## 📌 TL;DR

본 논문은 Split-Federated Learning의 통신 비용을 줄이기 위해 **Rate-Distortion 최적화 기반의 학습된 압축 기법인 SplitFedZip**을 제안한다. 특징 맵과 기울기를 모두 압축 대상으로 하며, 모델 학습과 압축 코덱 학습을 동시에 진행함으로써 **정확도 손실 없이 데이터 전송량을 $10^3$ ~ $10^{6.5}$ 배까지 절감**하였다. 이 연구는 특히 대용량 의료 영상 데이터를 다루는 분산 학습 환경에서 실용적인 통신 효율화 방안을 제시하며, 향후 리소스 제한적인 환경에서의 분산 학습 연구에 중요한 기초가 될 것으로 보인다.
