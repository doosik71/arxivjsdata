# Heterogeneous Data-Aware Federated Learning

Lixuan YANG, Cedric BELIARD, Dario Rossi (2020)

## 🧩 Problem to Solve

본 논문은 연합 학습(Federated Learning, FL)을 실제 산업 현장에 배포할 때 발생하는 데이터의 불균형성과 다양성 문제를 해결하고자 한다. 일반적으로 FL은 데이터를 중앙 서버로 수집하지 않고 분산된 클라이언트에서 모델을 학습시켜 프라이버시를 보호하지만, 다음과 같은 현실적인 제약 조건들이 모델의 수렴을 방해하고 성능을 저하시킨다.

첫째, 데이터가 독립 동일 분포(i.i.d)를 따르지 않는 Non-i.i.d 특성이 존재한다. 둘째, 클라이언트 간에 서로 겹치지 않는 클래스가 존재하는 Disjoint classes 문제가 있다. 셋째, 동일한 클래스 내에서도 데이터셋마다 신호의 특성이 다른 Signal multi-modality 현상이 나타난다. 이러한 특성들은 모델의 발산(divergence)을 초래하며, 결과적으로 FL 모델의 실용성을 제한한다. 따라서 본 논문의 목표는 모델 공유에 대한 제약 조건을 만족하면서도, 위와 같은 데이터의 이질성(heterogeneity)을 효과적으로 처리할 수 있는 새로운 FL 프레임워크를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 '분할 정복(Divide et Impera)' 전략을 신경망 아키텍처에 적용하는 것이다. 모든 모델 파라미터를 동일하게 처리하는 기존 방식과 달리, 신경망의 파라미터를 **범용 파라미터(Generic parameters)**와 **특화 파라미터(Specific parameters)**로 분리하여 서로 다르게 처리한다.

구체적으로, 신경망의 앞부분에 위치하여 특징 추출(Feature extraction)을 담당하는 범용 파라미터는 서버에서 집계하여 모든 클라이언트가 공유하게 함으로써 전역적인 학습 이득을 얻게 한다. 반면, 마지막 단계에서 분류를 수행하는 특화 파라미터(Classifier)는 각 클라이언트가 개별적으로 보유하고 최적화하게 함으로써, 각자의 로컬 데이터 분포에 특화된 맞춤형 모델을 유지하도록 설계하였다.

## 📎 Related Works

기존의 FL 연구들은 주로 통신 복잡도를 줄이거나(Model compression, Knowledge distillation), 시스템의 이질성(Computational bottleneck)을 해결하는 데 집중해 왔다. 한편, Non-i.i.d 데이터 문제를 해결하기 위해 클라이언트를 유사도 기반으로 그룹화하는 Clustered FL이나, 글로벌 손실 함수를 최적화하는 방식 등이 제안되었다.

특히 본 논문과 가장 유사한 연구는 LG-FedAvg(Local Global FL)이다. LG-FedAvg 역시 모델의 일부만 공유하는 방식을 취하지만, 공유하는 부분에서 결정적인 차이가 있다. LG-FedAvg는 로컬에서 특징 추출기를 학습하고 예측기(Predictor) 부분만 서버에서 공유한다. 그러나 저자들은 각 클라이언트가 로컬 특징 추출기를 충분히 학습시킬 만큼의 데이터를 가지고 있지 않을 수 있다고 주장한다. 이에 반해, 본 논문의 HDAFL은 특징 추출기(Generic parameters)를 공유하고 분류기(Specific parameters)를 로컬에 유지함으로써, 적은 데이터로도 풍부한 특징 추출 능력을 갖추게 하고 데이터 이질성에는 로컬 분류기로 대응하는 전략을 취한다.

## 🛠️ Methodology

### 시스템 구조 및 파이프라인

HDAFL은 합성곱 신경망(CNN) 구조를 기반으로 한다. CNN의 앞단에 위치한 합성곱 층(Convolutional layers)과 일부 완전 연결 층(Fully connected layers)을 범용 파라미터 $w_b$로 정의하고, 최종 출력층인 분류기(Classification layer)를 특화 파라미터 $w_l$로 정의한다.

### 학습 절차 및 알고리즘

전체 학습 프로세스는 다음과 같은 단계로 진행된다.

1. **초기화**: 서버는 $w_b^0$와 $w_l^0$를 초기화하여 모든 클라이언트에게 브로드캐스트한다.
2. **로컬 업데이트**: 각 라운드 $t$에서 선택된 클라이언트 $k$는 서버로부터 받은 $w_b^t$를 복사하고, 자신의 로컬 데이터로 $w_b^k$와 $w_l^k$를 $E$ 에포크 동안 학습시킨다.
3. **파라미터 전송**: 학습이 끝나면 클라이언트는 오직 범용 파라미터 $w_b^k$만을 서버로 전송한다. 특화 파라미터 $w_l^k$는 클라이언트 내부에 비밀리에 유지된다.
4. **서버 집계**: 서버는 수신된 범용 파라미터들을 각 클라이언트의 샘플 수 $n_k$에 따라 가중 평균하여 새로운 전역 모델 $w_b^{t+1}$을 생성한다.
   $$w_b^{t+1} = \sum_{k=1}^{K_t} \frac{n_k}{\sum_{j=1}^{K_t} n_j} w_{b,k}^t$$
5. **반복**: 위 과정을 목표 정확도에 도달하거나 최대 라운드 $T$가 될 때까지 반복한다.

이 구조를 통해 클라이언트는 전체 데이터셋의 특성이 반영된 강력한 특징 추출 능력을 공유하면서도, 자신의 로컬 데이터 분포에 맞는 최적의 분류기를 유지할 수 있게 된다.

## 📊 Results

### 실험 설정

- **데이터셋**: 공개 데이터셋인 FEMNIST와 화웨이의 독자적인 TCP/IP 트래픽 분류 데이터셋을 사용하였다.
- **시나리오**: FEMNIST의 경우 데이터를 i.i.d, non-i.i.d, disjoint 세 가지 방식으로 샘플링하여 실험하였다.
- **비교 대상**: 표준 알고리즘인 FedAvg와 성능을 비교하였다.

### 주요 결과

1. **FEMNIST 정확도 분석**:
   - **i.i.d 설정**: FedAvg가 HDAFL보다 약간 더 높은 성능을 보였다. 이는 모든 데이터 분포가 동일할 때 모델 전체를 공유하는 것이 변동성을 더 잘 학습할 수 있기 때문이다.
   - **non-i.i.d 설정**: HDAFL이 FedAvg보다 우수한 성능을 기록하였다. 로컬 분류기가 각 클라이언트의 데이터 분포를 더 잘 캡처했기 때문이다.
   - **disjoint 설정**: HDAFL이 압도적인 성능 차이(HDAFL $\approx 97\%$ vs FedAvg $\approx 19\%$)를 보였다. FedAvg는 클래스가 겹치지 않을 때 가중치 발산이 심해지지만, HDAFL은 로컬 분류기를 유지하므로 이 문제에서 자유롭다.

2. **통신 효율성**:
   - non-i.i.d 설정에서 78% 정확도에 도달하기 위한 통신 비용을 측정했을 때, HDAFL은 FedAvg 대비 최소 2배 이상 낮은 통신 비용을 소모하였다. 이는 모델 전체가 아닌 일부 파라미터($w_b$)만 전송하기 때문이며, 수렴 속도 또한 더 빨랐다.

3. **TCP/IP 트래픽 분류**:
   - 중앙 집중식 학습 모델과 비교했을 때, HDAFL은 단 4라운드 만에 정확도 손실을 1% 이내로 좁혔으며, 통신 비용을 33배나 절감하였다. 반면 FedAvg는 라운드 수와 상관없이 정확도 목표치를 달성하지 못했다.

## 🧠 Insights & Discussion

본 논문은 FL에서 모델의 모든 부분을 공유하는 것이 항상 최선은 아니라는 점을 시사한다. 특히 데이터의 이질성이 극심한 환경(Disjoint classes)에서는 전역 모델의 평균화 과정이 오히려 개별 클라이언트의 성능을 파괴하는 '독'이 될 수 있다. HDAFL은 범용적인 특징 추출 능력은 공유하되, 최종 판단 단계인 분류기를 로컬화함으로써 이 트레이드오프를 성공적으로 해결하였다.

다만, i.i.d 상황에서 FedAvg보다 성능이 약간 낮게 나타난 점은 HDAFL의 한계로 볼 수 있다. 이는 분류기 층이 오직 로컬 데이터만으로 학습되어 전역 데이터의 다양성을 충분히 반영하지 못하기 때문이다(Over-adaptation 가능성). 그러나 실제 산업 현장의 데이터는 대부분 non-i.i.d 특성을 가지므로, 본 연구의 접근 방식이 훨씬 더 실용적이라고 판단된다.

## 📌 TL;DR

본 논문은 Non-i.i.d 데이터와 클래스 불일치 문제로 인해 발생하는 FL의 모델 발산 문제를 해결하기 위해, 모델을 범용 특징 추출기($w_b$)와 로컬 특화 분류기($w_l$)로 분리하여 학습하는 **HDAFL** 방식을 제안한다. 범용 파라미터만 서버에서 공유하고 분류기는 로컬에 유지함으로써, 통신 비용을 줄이는 동시에 데이터 이질성이 강한 환경에서 FedAvg 대비 월등한 정확도와 빠른 수렴 속도를 달성하였다. 이 연구는 실제 산업 현장의 복잡한 데이터 환경에서 FL을 적용하는 데 중요한 가이드를 제공한다.
