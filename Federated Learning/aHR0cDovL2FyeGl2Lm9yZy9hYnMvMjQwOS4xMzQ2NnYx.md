# Global Outlier Detection in a Federated Learning Setting with Isolation Forest

Daniele Malpetti, Laura Azzimonti (2024)

## 🧩 Problem to Solve

본 논문은 연합 학습(Federated Learning, FL) 환경에서 데이터의 프라이버시를 보호하면서 **전역 이상치(Global Outliers)**를 탐지하는 문제를 해결하고자 한다.

일반적인 머신러닝 모델과 마찬가지로 연합 학습 모델 또한 데이터 내의 이상치나 어노말리에 의해 성능이 저하될 수 있다. 특히 연합 학습에서는 이상치를 두 가지 유형으로 구분할 수 있는데, 특정 클라이언트 내에서만 이상치로 간주되는 '지역 이상치(Local Outliers)'와 전체 데이터셋 관점에서 이상치로 간주되는 '전역 이상치'이다. 예를 들어, 특정 질병이 어떤 지역에서는 흔하지만 다른 지역에서는 매우 드문 경우, 드문 지역의 데이터는 지역적으로는 이상치일 수 있으나 전역적으로는 정상 데이터일 수 있다. 따라서 모델의 품질을 높이기 위해서는 지역 이상치는 유지하되, 전역 이상치만을 식별하여 제거하는 전처리 과정이 필수적이다.

기존의 이상치 탐지 솔루션은 주로 중앙 집중식 환경에 최적화되어 있거나, IoT 환경과 같은 'cross-device' 시나리오(다수의 엣지 디바이스 참여)에 치중되어 있다. 반면, 의료 데이터와 같이 규제가 엄격한 도메인에서 사용되는 'cross-silo' 시나리오(소수의 기관이 협력)를 위한 프라이버시 보존형 전역 이상치 탐지 방법론은 부족한 실정이다. 본 논문의 목표는 데이터의 민감한 정보를 공개하지 않으면서도 중앙 집중식 방식과 유사한 수준의 성능을 내는 전역 이상치 탐지 프레임워크를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **두 개의 서버(Principal Server $P$, Auxiliary Server $A$)와 데이터 마스킹(Masking) 기법**을 결합하여, 서버가 개별 데이터의 원본 값이나 소유자를 알 수 없게 하면서도 이상치 특성인 '고립성(Isolationness)'은 유지하도록 하는 것이다.

구체적으로는 다음과 같은 설계 아이디어를 적용하였다.

1. **마스킹 변환(Masking Transformation)**: 데이터에 회전(Rotation)과 스케일링(Scaling)을 적용한 가역 행렬 $M$을 곱하여 데이터를 변형한다. 이는 데이터의 절대적 거리나 밀도는 변화시키지만, 이상치가 일반 데이터보다 쉽게 고립된다는 특성은 유지한다.
2. **가산 마스킹(Additive Masking)**: 가우시안 노이즈 행렬을 사용하여 데이터를 한 번 더 숨기고, 두 서버가 각각 노이즈와 데이터-노이즈 합산본을 나누어 가짐으로써 단일 서버가 원본 데이터를 복원할 수 없게 한다.
3. **순열 메커니즘(Permutation Mechanism)**: 보조 서버 $A$를 통해 데이터 포인트의 인덱스를 무작위로 섞어, 주 서버 $P$가 특정 데이터 포인트가 어떤 클라이언트의 것인지 알 수 없도록 익명성을 보장한다.

## 📎 Related Works

논문에서는 기존의 이상치 탐지 방법론과 연합 학습 기반의 접근 방식을 검토한다.

- **중앙 집중식 방법**: $z$-score, $k$-nearest neighbors, Local Outlier Factor (LOF), Isolation Forest (IF) 등이 널리 사용되고 있다.
- **연합 학습 기반 방법**: 주로 IoT 도메인의 cross-device 설정에서 침입 탐지나 장치 고장 예측을 위해 LSTM, GRU, CNN 기반의 딥러닝 기법들이 제안되었다. 또한, Differential Privacy를 적용한 연합 버전의 IF 연구가 존재한다.
- **차별점**: 기존 연구들은 주로 실시간 모니터링(Anomaly Detection)에 초점을 맞춘 반면, 본 연구는 후속 연합 학습 모델의 성능 향상을 위한 **전처리 단계로서의 이상치 제거(Outlier Removal)**에 집중하며, 특히 cross-silo 시나리오를 타겟팅한다는 점에서 차별성을 갖는다.

## 🛠️ Methodology

본 논문이 제안하는 방법론은 크게 준비 단계(Preliminary steps)와 메인 프로세스(Main process)로 나뉜다. 전체 시스템은 주 서버 $P$, 보조 서버 $A$, 그리고 $m$개의 클라이언트 $C^i$로 구성된다.

### 1. 준비 단계 (Preliminary Steps)

**정수 및 인덱스 생성**:

- 클라이언트들은 Paillier 동형 암호 시스템(Homomorphic Encryption)을 사용하여 서버들이 알 수 없는 전역 시드(Global Seed) $\Xi$에 합의한다.
- 전체 데이터 포인트의 수 $N$을 보안 합계 프로토콜을 통해 계산한다.
- 보조 서버 $A$의 도움을 받아 각 클라이언트는 $Z_N$ 집합 내에서 서로 겹치지 않는 고유한 정수 집합 $\tilde{Z}_i$를 할당받는다. 이때 순열 함수를 사용하여 클라이언트가 연속적인 행을 점유하지 않도록 하여 서버가 데이터 패턴을 통해 클라이언트를 추측하는 것을 방지한다.

**행렬 생성**:

- 모든 클라이언트는 공유된 시드 $\Xi$를 사용하여 동일한 가역 행렬 $M$을 로컬에서 생성한다. $M$은 두 개의 직교 행렬 $Q, Q'$와 대각 행렬 $S$의 곱으로 정의된다:
  $$M = QSQ'$$
  여기서 $S$의 요소는 $(1, T)$ 범위에서 생성되어 행렬이 특이 행렬(Singular matrix)이 되지 않도록 보장한다.
- 각 클라이언트는 가우시안 분포 $\mathcal{N}(0, \sigma^2)$를 따르는 노이즈 행렬 $R^i$를 생성한다.

### 2. 메인 프로세스 (Main Process)

**데이터 마스킹 및 전송**:

- 클라이언트는 자신의 로컬 데이터 $X^i$에 마스킹 행렬 $M$을 곱해 $\tilde{X}^i = X^i M$을 계산한다.
- 이후, 할당받은 인덱스 $\tilde{Z}_i$에 맞춰 $\tilde{X}^i$의 행들을 노이즈 행렬 $R^i$에 더해 $W^i$를 생성한다.
- 클라이언트는 $R^i$는 보조 서버 $A$에게, $W^i$는 주 서버 $P$에게 전송한다.

**데이터 복원 및 이상치 탐지**:

- 보조 서버 $A$는 모든 클라이언트의 노이즈를 합산하여 $R = \sum R^i$를 계산하고 이를 주 서버 $P$에게 보낸다.
- 주 서버 $P$는 다음과 같이 마스킹된 데이터 행렬 $X_{\text{masked}}$를 복원한다:
  $$X_{\text{masked}} = W - R = \left( \sum W^i \right) - \left( \sum R^i \right)$$
- $P$는 복원된 $X_{\text{masked}}$에 대해 **Isolation Forest (IF)** 또는 **Extended Isolation Forest (EIF)**를 적용하여 각 데이터 포인트의 이상치 점수를 계산한다.

**결과 전달**:

- $P$는 계산된 점수를 클라이언트에게 다시 보내거나, 이상치가 존재하는 '마스킹 공간'의 하이퍼플레인 방정식을 전달한다. 클라이언트는 이를 통해 자신의 데이터 중 어떤 점이 전역 이상치인지 판단하여 제거한다.

## 📊 Results

### 실험 설정

- **데이터셋**: Arrhythmia, Mnist, Satellite 등 총 14개의 벤치마크 데이터셋을 사용하였다. 데이터셋마다 변수 개수와 이상치 비율이 다양하게 구성되었다.
- **비교 대상**:
  - Standard Approach: 마스킹 없이 중앙에서 IF/EIF를 수행한 경우.
  - Multiparty Approach: 제안된 프레임워크를 통해 마스킹된 데이터를 사용한 경우.
- **지표**: AUROC (Area Under the Receiver Operating Characteristic curve)를 사용하여 성능을 측정하였다.
- **하이퍼파라미터**: 트리 개수 $t=100$, 샘플링 크기 $\psi=256$을 사용하였으며, 스케일링 파라미터 $T$의 다양한 값을 테스트하였다.

### 주요 결과

- **성능 비교**: 실험 결과, 제안된 Multiparty 방식의 AUROC 성능은 중앙 집중식 Standard 방식과 비교했을 때 전반적으로 **비슷한 수준(Comparable)**임을 확인하였다. 일부 데이터셋(Glass, Mnist)에서는 오히려 제안 방식이 더 높은 성능을 보이기도 했다.
- **알고리즘 간 차이**: 마스킹된 데이터 상에서는 IF와 EIF의 성능 차이가 거의 없었다. 이는 마스킹 행렬 $M$이 수행하는 회전 및 스케일링 변환이 IF의 분할 방향을 일반화하여 EIF와 유사한 효과를 내기 때문으로 분석된다.
- **파라미터 영향**: 스케일링 계수 $T$의 변화는 결과에 유의미한 영향을 주지 않았다. 다만, Standard 방식보다 결과의 변동성(Variability)이 다소 크게 나타났는데, 이는 전역 시드 $\Xi$에 따라 마스킹 결과가 달라지기 때문이며, 여러 번 실행하여 다수결 방식을 취함으로써 강건성을 높일 수 있다.

## 🧠 Insights & Discussion

### 마스킹 행렬 $M$의 기하학적 의미

$M$은 두 번의 회전과 한 번의 스케일링을 결합한 변환이다. 이러한 변환은 데이터의 밀도, 상관관계, 노름(Norm) 등은 변화시키지만, 이상치가 일반 데이터보다 적은 횟수의 분할만으로도 쉽게 고립된다는 **'고립성'**은 보존한다. 이 때문에 IF 알고리즘을 마스킹된 공간에서 적용해도 원래 공간에서의 이상치를 정확하게 찾아낼 수 있다.

### 보안 및 공모(Collusion) 분석

- **클라이언트 공모**: $m-2$명의 클라이언트가 공모하더라도 특정 데이터의 소유자를 정확히 식별할 수 없다. 다만 $m-1$명이 공모하면 나머지 한 명의 데이터를 추적할 수 있는 취약점이 있다.
- **서버-클라이언트 공모**: 주 서버 $P$와 클라이언트가 공모할 경우 전체 데이터 행렬을 재구성할 위험이 있다. 따라서 본 방법론의 가장 중요한 전제 조건은 **주 서버 $P$가 신뢰할 수 있는 기관(Trusted Party)**이어야 한다는 점이다.
- **보조 서버의 역할**: 보조 서버 $A$는 노이즈 합산과 인덱스 할당만을 담당하며, $P$와 공모하더라도 원본 데이터를 복원할 수 없도록 설계되었다.

### 한계 및 향후 방향

본 연구는 IF 계열의 알고리즘에 최적화되어 있다. 밀도 기반(Density-based) 알고리즘은 마스킹 변환 시 밀도가 변하므로 적용하기 어렵겠지만, 다른 클래스의 알고리즘에 대해서는 추가 검증이 필요하다. 또한, 배치 효과(Batch effect) 탐지와 같은 다른 작업으로의 확장 가능성이 언급되었다.

## 📌 TL;DR

본 논문은 연합 학습의 cross-silo 환경에서 프라이버시를 보호하며 전역 이상치를 제거할 수 있는 프레임워크를 제안한다. **두 개의 서버와 가역 마스킹 행렬($M=QSQ'$), 가우시안 노이즈, 인덱스 순열**을 이용하여 서버가 원본 데이터를 알 수 없게 하면서도 **Isolation Forest**를 통해 이상치를 탐지한다. 실험 결과, 중앙 집중식 방식과 대등한 수준의 AUROC 성능을 보였으며, 이는 데이터의 프라이버시를 유지하면서도 학습 데이터의 품질을 높이는 전처리 단계로 활용될 가능성이 높다.
