# StackNet: Stacking feature maps for Continual learning

Jangho Kim, Jeesoo Kim, Nojun Kwak (2020)

## 🧩 Problem to Solve

본 논문이 해결하고자 하는 핵심 문제는 신경망 학습에서 발생하는 **Catastrophic Forgetting**(파괴적 망각)이다. 일반적인 신경망 학습은 모든 데이터가 처음부터 제공된다고 가정하지만, 실제 환경에서는 데이터가 점진적으로 축적되며, 새로운 데이터를 학습할 때 이전 학습 데이터에 접근하지 못하는 경우가 많다. 이때 새로운 작업을 학습시키면 기존에 학습된 가중치가 덮어씌워지면서 이전 작업에 대한 성능이 급격히 저하되는 현상이 발생한다.

이 문제의 중요성은 인공지능이 인간의 뇌처럼 평생에 걸쳐 지식을 축적하고 조직화하는 능력을 갖추기 위해 필수적이라는 점에 있다. 따라서 본 논문의 목표는 이전 작업의 성능을 유지하면서 새로운 작업을 추가로 학습할 수 있는 **Continual Learning**(지속 학습) 방법론을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 중심 아이디어는 네트워크의 파라미터를 '스택(Stack)' 방식으로 쌓아 올려 각 작업에 할당하는 것이다. 이를 위해 **Index Module**과 **StackNet**이라는 두 개의 상호 보완적인 구성 요소를 제안한다.

핵심 직관은 네트워크의 고정된 용량 내에서 필터를 분할하여 각 작업에 할당하고, 새로운 작업이 추가될 때마다 기존 필터는 동결(freeze)시킨 채 새로운 필터를 추가로 쌓아 올리는 것이다. 이때, 입력 데이터가 어떤 작업에 속하는지 판단하는 Index Module을 통해 적절한 필터 조합과 클래스 라벨을 선택함으로써 이전 지식의 손실 없이 새로운 지식을 확장할 수 있게 한다.

## 📎 Related Works

논문에서는 기존의 지속 학습 접근 방식을 크게 두 가지로 분류하여 설명한다.

1.  **Regularization-based methods**: LwF(Learning without forgetting)와 같이 출력이나 특징(feature)을 규제하거나, 이전 파라미터와 새 파라미터 사이의 거리(quadratic distance)에 패널티를 주는 방식이다. 이러한 방식은 기존 네트워크의 변형을 억제하는 데 집중하므로, 학습할 작업이 많아질수록 이전 작업의 성능이 저하되며 새 작업의 성능 또한 최적화되지 않는 한계가 있다.
2.  **Dynamic architectures**: Progressive Network와 같이 새로운 작업이 추가될 때마다 네트워크 용량을 확장하는 방식이다. 하지만 Progressive Network는 각 작업마다 추가적인 adapter를 학습시켜야 하므로 전형적인 CNN 구조에서 벗어나며 용량이 급격히 증가한다. 또한, PackNet은 가중치 가지치기(weight pruning)를 통해 특정 필터를 작업에 할당하지만, 입력 데이터가 어떤 작업인지에 대한 사전 정보(prior knowledge)가 반드시 필요하며, 전체 용량이 꽉 차면 더 이상 작업을 추가할 수 없는 확장성의 한계가 있다.

StackNet은 PackNet과 유사하게 필터를 분할하여 사용하지만, 복잡한 pruning 과정 없이 파라미터를 단순히 쌓는 방식을 취하며, Index Module을 통해 사전 정보 없이도 작업 인덱스를 추정할 수 있다는 점에서 차별점을 가진다.

## 🛠️ Methodology

### 1. StackNet 구조 및 학습 절차

StackNet은 네트워크의 용량을 여러 파티션으로 나누어 다중 작업(multi-task)을 처리하며, 새로운 작업이 주어지면 추가적인 필터를 부착하는 구조이다. VGG나 ResNet과 같은 일반적인 CNN 아키텍처에 적용 가능하다.

**필터 인덱스 및 가중치 할당:**
$J$번째 작업에 대해 필터 인덱스 $I_J$를 정의하며, 이는 각 레이어에서 사용할 필터의 범위를 결정한다. $k$번째 레이어에서 이전 작업의 가중치를 $W_P^k$, 새로운 작업의 가중치를 $W_J^k$라고 할 때, 가중치 행렬의 차원은 다음과 같다.
$$W_J^k \in \mathbb{R}^{x \times x \times I_{J}^{k-1} \times I_{J}^k}$$
여기서 $x$는 커널 크기이며, 이전 레이어의 출력 채널 수($I_{J}^{k-1}$)가 현재 레이어의 입력 채널 수가 된다.

**학습 및 추론 과정:**
새로운 작업을 학습할 때, 이전 작업의 가중치 $W_P^k$는 고정(freeze)시키고, 새로 추가된 파라미터 $W_T^k$ (단, $W_T^k = W_J^k \setminus W_P^k$)만을 업데이트한다.
학습 효율을 높이기 위해 $W_T$를 초기화할 때, 이전 작업의 가중치 $W_P$를 복제하고 여기에 가우시안 노이즈(Gaussian noise)를 추가하는 방식을 사용한다. 추론 시에는 해당 작업 $J$에 할당된 모든 필터(이전 작업 필터 + 현재 작업 필터)를 함께 사용하여 예측을 수행한다.

### 2. Index Module

추론 단계에서 입력 데이터 $x$가 어떤 작업 $J$에 속하는지 알아야 적절한 필터와 클래스 라벨을 선택할 수 있다. 이를 위해 본 논문은 GAN(Generative Adversarial Networks) 기반의 Index Module을 제안한다.

**학습 단계:**
1.  각 작업 $J$에 대해 가상 샘플을 생성하는 생성기 $G_J$를 학습시킨다. 이때 손실 함수는 다음과 같다.
    $$\min_{G_J} \max_{D_J} V(D_J, G_J) = \mathbb{E}_{x \sim P_{data}^J} [\log(D_J(x))] + \mathbb{E}_{z \sim p_z(z)} [\log(1 - D_J(G_J(z)))]$$
2.  학습된 생성기들을 이용하여 각 작업별 가상 데이터를 생성하고, 이를 통해 이진 분류기(Binary Classifier) $B_J$를 학습시킨다. $B_J$는 입력 데이터가 작업 $J$에서 왔는지 여부(1 또는 0)를 판별한다.

**추론 단계:**
입력 데이터 $x$에 대해 모든 이진 분류기 $B_i$를 통과시켜 가장 높은 확률을 가진 인덱스 $J$를 선택한다.
$$J = \arg \max_i B_i(x)$$
선택된 $J$는 StackNet에 전달되어 사용할 필터 범위와 클래스 라벨을 결정한다.

## 📊 Results

### 1. 실험 설정
- **데이터셋**: MNIST, SVHN, CIFAR-10 (기초 데이터셋) 및 ImageNet-A, ImageNet-B (실제 환경 데이터셋).
- **비교 대상**: LwF, PackNet, 단일 작업 네트워크(Single network).
- **지표**: 테스트 정확도(Accuracy) 및 평균 정확도(Avg. %).

### 2. 주요 결과
- **기초 데이터셋 (MNIST $\to$ SVHN $\to$ CIFAR-10)**: 
    - StackNet은 이전 작업의 성능 저하를 거의 완벽하게 막으면서 새로운 작업을 학습했다.
    - 특히 3개 이상의 작업을 순차적으로 학습하는 상황에서 LwF는 급격한 성능 저하를 보였고, PackNet보다 StackNet이 세 번째 작업에서 더 높은 성능을 기록하여 다중 작업 지속 학습에 더 적합함을 입증했다.
    - Index Module이 포함된 $\text{StackNet}^*$ 모델 또한 사전 정보 없이도 PackNet과 경쟁하거나 이를 능가하는 성능을 보였다.

- **실제 데이터셋 (ImageNet-A $\to$ ImageNet-B)**: 
    - ResNet-50 구조를 사용한 실험에서 StackNet은 PackNet과 유사하거나 약간 높은 평균 정확도를 보였다.
    - PackNet은 모든 필터에 대한 마스크(mask)를 저장해야 하므로 메모리 비용이 크지만, StackNet은 레이어당 정수(인덱스) 하나만 저장하면 되므로 훨씬 효율적이다.

- **Ablation Study**:
    - **초기화 방법**: 단순 랜덤 초기화보다 이전 작업의 가중치에 노이즈를 섞어 초기화했을 때 새로운 작업의 정확도가 더 높게 나타났다.
    - **파라미터 공유**: 이전 작업의 지식을 공유하여 학습했을 때, 공유하지 않았을 때보다 수렴 속도가 훨씬 빠르고 최종 에러율이 낮았다.

## 🧠 Insights & Discussion

### 강점 및 성과
본 논문은 파라미터를 쌓는(Stacking) 직관적인 구조를 통해 파괴적 망각 문제를 효과적으로 해결했다. 특히 기존의 PackNet이 가졌던 '사전 정보 필요' 문제와 '확장성 제한' 문제를 Index Module과 동적 필터 추가 방식을 통해 해결한 점이 돋보인다. 또한, 파라미터 공유와 최적의 초기화 전략을 통해 모델의 콤팩트함을 유지하면서도 학습 효율을 극대화했다.

### 한계 및 비판적 해석
1.  **Index Module의 의존성**: 제안된 GAN 기반 Index Module은 저해상도 이미지(MNIST, CIFAR 등)에서는 매우 높은 정확도를 보였으나, 저자 스스로 언급했듯이 고해상도 이미지(ImageNet 등)에서는 생성 모델의 한계로 인해 적용이 어렵다는 점이 명시되어 있다. 이는 실제 고해상도 데이터셋에 대한 Index Module의 검증이 부족함을 의미한다.
2.  **유사 분포 데이터**: 입력 데이터들의 분포가 매우 유사할 경우, Index Module이 작업 인덱스를 정확히 구분해내지 못할 가능성이 크며, 이는 전체 시스템의 성능 저하로 직결될 수 있다.
3.  **용량 증가**: 비록 효율적으로 쌓는다고 하였으나, 작업이 계속 추가될수록 네트워크의 물리적 크기는 결국 증가하게 된다. 물리적 메모리 한계에 도달했을 때의 전략에 대해서는 명확한 대안이 제시되지 않았다.

## 📌 TL;DR

본 논문은 지속 학습에서 발생하는 파괴적 망각을 해결하기 위해, 새로운 작업의 파라미터를 기존 파라미터 위에 쌓아 올리는 **StackNet**과 입력 데이터의 출처를 판별하는 **Index Module**을 제안한다. 이 방식은 기존의 가중치 가지치기(Pruning) 방식보다 효율적이며, 특히 작업의 개수가 늘어나는 다중 작업 환경에서 뛰어난 확장성과 성능을 보인다. 실제 서비스 환경에서 작업 ID가 주어지지 않는 상황을 GAN 기반의 인덱스 추정기로 해결하려 했다는 점에서 실용적인 가치가 높다.
