# Evolutionary Generalized Zero-Shot Learning

Dubing Chen, Chenyi Jiang, Haofeng Zhang (2024)

## 🧩 Problem to Solve

본 논문은 기존의 속성 기반 제로샷 학습(Attribute-based Zero-Shot Learning, ZSL) 모델들이 가진 정적인 특성을 극복하고, 실제 배포 환경에서 모델이 자율적으로 진화할 수 있는 새로운 설정인 **Evolutionary Generalized Zero-Shot Learning (EGZSL)** 문제를 제안한다.

기존의 ZSL 및 일반화 제로샷 학습(GZSL) 모델은 훈련 단계에서 학습한 정적인 지식에 의존하므로, 테스트 단계에서 발생하는 도메인 시프트(Domain Shift) 문제나 동적인 데이터 변화에 대응하기 어렵다. 특히, 모델이 배포된 이후 레이블이 없는(unlabeled) 데이터 스트림을 통해 지속적으로 성능을 개선하고 전문가 영역으로 진화해야 한다는 요구사항이 증가하고 있다.

따라서 본 연구의 목표는 초기 훈련 이후, 레이블이 없는 테스트 데이터 스트림을 통해 스스로 적응하고 진화함으로써, 본 적 없는 클래스(unseen classes)에 대한 인식 능력을 향상시키면서도 기존의 핵심 능력을 유지하는 모델을 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 기여는 다음과 같다.

1. **EGZSL 설정 정의**: 초기 훈련 후 레이블이 없는 데이터 스트림으로부터 온라인으로 지속적인 학습 및 진화가 가능한 새로운 학습 패러다임을 정의하였다.
2. **핵심 도전 과제 분석 및 해결책 제시**: EGZSL 수행 시 발생하는 세 가지 주요 문제인 **치명적 망각(Catastrophic Forgetting)**, **초기 예측 편향(Initial Prediction Bias)**, **진화 데이터 클래스 편향(Evolutionary Data Class Bias)**을 분석하고 이에 대한 구체적인 방법론을 제안하였다.
3. **진화적 학습 프레임워크 제안**: 지식 증류(Knowledge Distillation), 모멘텀 모델(Momentum Model), 클래스 선택적 업데이트 및 적응형 임계값 기반의 데이터 필터링을 결합하여 모델의 안정적인 진화를 가능케 하였다.
4. **평가 프로토콜 수립**: EGZSL을 평가하기 위한 새로운 벤치마크 프로토콜을 제안하고, 세 가지 공개 데이터셋을 통해 제안 방법론의 유효성을 입증하였다.

## 📎 Related Works

논문에서는 EGZSL과 유사한 기존 연구들을 다음과 같이 분류하고 차별점을 설명한다.

- **ZSL 및 GZSL**: 전통적인 ZSL은 정적인 모델을 학습시키며, GZSL은 테스트 시 본 클래스(seen)와 보지 못한 클래스(unseen)를 모두 분류해야 한다. 이들은 도메인 시프트 문제에 취약하다.
- **Transductive ZSL (TZSL)**: 훈련 단계에서 레이블이 없는 unseen 클래스 데이터를 사용할 수 있다고 가정한다. 이는 도메인 시프트를 완화하지만, 훈련 시 테스트 데이터에 접근할 수 있다는 가정이 너무 강력하여 실제 적용 가능성이 낮다.
- **Continual ZSL (CZSL)**: 클래스 증분 학습(Class-incremental learning) 방식으로 확장한 형태이다. 하지만 EGZSL과 달리 주로 새로운 클래스를 순차적으로 배우는 것에 집중하며, 배포 후 레이블 없는 스트림을 통한 자율 진화와는 차이가 있다.
- **Test-Time Adaptation (TTA)**: 테스트 단계에서 자기지도 학습(SSL) 등을 통해 도메인에 적응하는 방식이다. TTA는 주로 분포 변화(Distribution Shift) 해결에 집중하며, EGZSL처럼 클래스 공간을 확장하거나 ZSL의 특수한 편향 문제를 다루지 않는다.

**차별점**: EGZSL은 TGZSL의 엄격한 버전으로 볼 수 있으며, 데이터에 대한 사전 지식이나 전체 테스트 셋에 대한 접근 없이 오직 현재 유입되는 데이터 스트림만을 이용하여 온라인으로 진화한다는 점에서 실용성이 더 높다.

## 🛠️ Methodology

본 논문은 초기 IGZSL 모델을 기반으로 테스트 단계에서 진화적 학습을 수행한다. 전체 파이프라인은 pseudo-label 학습, 글로벌 정보 유지, 클래스 및 데이터 선택 단계로 구성된다.

### 1. Pseudo-label 기반 학습

각 타임스텝 $t$에서, 이전 단계의 모델 $f_{t-1}$을 사용하여 현재 데이터 $x$의 pseudo-label $\hat{y}_x$를 예측한다.
$$\hat{y}_x = f_{t-1}(x) = \arg\max_y F_{t-1}(x, y; W_{t-1})$$
이 예측된 레이블을 기반으로 Cross-Entropy 손실 함수 $\ell_{ce}$를 통해 모델 파라미터 $W_{t-1}$을 최적화한다.

### 2. 글로벌 정보 유지 및 망각 방지

데이터 스트림 학습 시 발생하는 치명적 망각을 방지하기 위해, MoCo와 유사한 **모멘텀 모델(Momentum Model)** $f_{ema}$를 유지한다.
$$W_{ema} = m_1 \cdot W_{ema} + (1 - m_1) \cdot W_t$$
여기서 $m_1$은 smoothing factor이다. 현재 모델 $f_t$가 과거의 지식을 잊지 않도록, 모멘텀 모델과 현재 모델 간의 KL-Divergence를 통한 지식 증류(Distillation) 손실 $\ell_{kl}$을 적용한다.
$$\ell_{kl}(x) = \sum_{y \in Y} p_t(x, y; Y) \log \frac{p_t(x, y; Y)}{p_{ema}(x, y; Y)}$$

### 3. 안정적 학습을 위한 클래스 선택 (Class Selection)

데이터 스트림 내의 클래스 불균형으로 인해 특정 클래스로 예측이 쏠리는 현상을 막기 위해, 현재 배치에서 pseudo-label로 선택된 클래스들($Y_{sel}^t$)에 대해서만 파라미터를 업데이트한다.
$$Y_{sel}^t = \text{unique}(\{\hat{y}_x\}_{x \in D_{te}^t})$$
이를 통해 샘플이 없는 클래스의 가중치가 급격하게 변하여 발생하는 오류 누적을 방지한다.

### 4. 효율적 학습을 위한 데이터 선택 (Data Selection)

신뢰도가 낮은 pseudo-label로 인한 확인 편향(Confirmation Bias)을 막기 위해 적응형 임계값(Adaptive Threshold) 필터링을 사용한다.
단순 고정 임계값은 seen/unseen 클래스 간의 신뢰도 차이로 인해 특정 클래스가 과도하게 필터링되는 문제가 발생한다. 이를 해결하기 위해 각 클래스의 과거 예측 신뢰도 통계 $\delta_{ema}(y)$를 유지하고 이를 반영한 마스크 $M_{scl}$을 생성한다.
$$M_{scl}(x) = \mathbb{1}((x, f_{t-1}(x)) > \delta_{t-1}(f_{t-1}(x)) \cdot \tau)$$
최종적으로 이 마스크는 $\ell_{sel}^{ce}$와 $\ell_{kl}$ 모두에 가중치로 적용된다.

### 5. 최종 목적 함수

매 타임스텝 $t$에서 모델은 다음의 통합 손실 함수를 최소화한다.
$$L_{all} = L_{sel}^{ce} + \lambda L_{kl}$$
여기서 $\lambda$는 두 손실의 균형을 맞추는 하이퍼파라미터이다.

## 📊 Results

### 실험 설정

- **데이터셋**: AWA2, APY, CUB 세 가지 ZSL 벤치마크를 사용하였다.
- **평가 지표**: Seen 클래스 정확도($A_s$)와 Unseen 클래스 정확도($A_u$)의 조화 평균(Harmonic Mean, $H$)을 측정하였다.
- **비교 대상**: IGZSL(하한선), TGZSL(상한선), 그리고 단순 pseudo-labeling 기반의 ERM(Empirical Risk Minimization) 방식을 비교하였다.

### 주요 결과

- **성능 향상**: 제안 방법은 IGZSL 베이스라인보다 일관되게 높은 성능을 보였으며, 단순 ERM 방식은 오히려 성능이 하락하는 경향을 보였다. 이는 치명적 망각과 편향 문제를 해결하는 것이 필수적임을 시사한다.
- **데이터셋 특성**: CUB와 같은 세밀한(fine-grained) 데이터셋보다 AWA2, APY와 같은 거친(coarse-grained) 데이터셋에서 성능 향상 폭이 더 컸다. 이는 coarse-grained 데이터셋에서 시각적 편향(visual bias)이 더 강하게 나타나며, 진화적 학습이 이를 효과적으로 완화했기 때문이다.
- **배치 크기의 영향**: 한 번에 학습하는 데이터 양이 많을수록(10 $\to$ 100) 더 안정적인 그래디언트를 얻어 성능이 소폭 상승하였다.
- **한계**: TGZSL보다는 성능이 낮다. 이는 TGZSL이 전체 테스트 데이터를 반복 학습할 수 있고, seen/unseen 분리에 대한 더 완화된 제약 조건을 갖기 때문이다.

## 🧠 Insights & Discussion

### 강점 및 해석

본 연구는 ZSL 모델을 단순히 학습-테스트의 이분법적 구조에서 벗어나, 배포 후에도 지속적으로 성장하는 시스템으로 확장했다는 점에서 큰 의의가 있다. 특히 모멘텀 모델을 통한 지식 유지와 적응형 임계값을 통한 데이터 필터링의 조합이 실시간 스트림 데이터 학습의 불안정성을 효과적으로 제어했음을 확인하였다.

### 한계 및 논의사항

1. **TGZSL과의 간극**: 여전히 TGZSL 성능에 못 미치는 점은, 온라인 학습의 제약(데이터를 한 번만 보거나 소량만 본다는 점)이 매우 크다는 것을 의미한다. 향후 리플레이 버퍼(Replay Buffer) 등을 도입하여 과거 데이터를 일부 저장해 활용하는 방안이 논의될 수 있다.
2. **하이퍼파라미터 민감도**: 실험 결과 $\lambda$와 $m_1$ (망각 방지 관련 파라미터)에 대한 민감도가 높게 나타났다. 이는 자율 진화 시스템에서 망각과 학습의 균형을 맞추는 것이 매우 정교한 작업임을 보여준다.
3. **가정의 현실성**: 본 논문은 base model의 가중치를 그대로 가져와 사용한다고 가정한다. 실제 환경에서는 base model의 구조가 매우 크거나 업데이트가 어려운 경우에 대한 효율적인 어댑터(Adapter) 방식의 연구가 필요할 것이다.

## 📌 TL;DR

본 논문은 배포 후 레이블 없는 데이터 스트림을 통해 스스로 진화하는 **Evolutionary Generalized Zero-Shot Learning (EGZSL)**이라는 새로운 과제를 정의하고 해결책을 제시하였다. **모멘텀 모델 기반의 지식 증류**로 치명적 망각을 막고, **적응형 임계값 필터링과 클래스 선택적 업데이트**를 통해 예측 편향 문제를 해결하였다. 실험 결과, 제안 방법은 정적인 IGZSL 모델보다 우수한 성능을 보였으며, 특히 도메인 시프트와 시각적 편향이 심한 환경에서 실질적인 성능 향상을 이끌어내어 실제 환경에서의 ZSL 모델 배포 가능성을 높였다.
