# Large Language Models Lack Understanding of Character Composition of Words

Andrew Shin, Kunitake Kaneko (2024)

## 🧩 Problem to Solve

본 논문은 거대 언어 모델(Large Language Models, LLMs)이 단어의 최소 단위인 문자(Character)의 구성 방식을 얼마나 이해하고 있는지를 분석한다. 현재의 LLM들은 단어, 문장, 문서 단위의 작업에서는 인간을 능가하는 놀라운 성능을 보이지만, 정작 그 단어를 구성하는 개별 문자에 대한 이해도는 매우 낮을 것이라는 가설에서 출발한다.

문제의 핵심은 LLM이 텍스트를 처리할 때 문자가 아닌 토큰(Token) 단위로 학습하고 인식한다는 점에 있다. 인간에게는 매우 단순한 문자 수준의 작업(예: 특정 문자가 포함된 단어 찾기, 문자 삽입 및 삭제)이 LLM에게는 매우 어렵게 다가오며, 이는 모델의 신뢰성과 언어적 정밀함을 저해하는 요소가 된다. 따라서 본 연구의 목표는 다양한 문자 구성 작업 및 다국어 환경에서 LLM의 한계를 정량적으로 측정하고, 토큰 수준의 성능과 문자 수준의 성능 사이의 괴리를 분석하는 것이다.

## ✨ Key Contributions

본 논문의 중심적인 직관은 LLM이 단어를 더 이상 쪼갤 수 없는 불가분(indivisible)의 단위인 토큰으로 처리하기 때문에, 단어 내부의 세밀한 문자 관계와 형태론적 구조(Morphological structure)를 파악하지 못한다는 것이다.

주요 기여 사항은 다음과 같다.

- 문자 수준의 이해도를 측정하기 위한 5가지 핵심 작업(Retrieval, Insertion, Deletion, Replacement, Reordering, Counting)을 설계하고 이를 통해 LLM의 취약성을 입증하였다.
- 동일한 유형의 작업을 토큰 수준(Token-level)으로 확장하여 비교 실험함으로써, LLM의 성능 저하가 단순한 작업의 난이도가 아닌 '문자 단위 인식'의 근본적인 결함에서 기인함을 증명하였다.
- 영어뿐만 아니라 중국어, 한국어, 일본어 등 서로 다른 문자 체계를 가진 언어들에 대해 실험을 수행하여, 언어적 특성이 모델의 성능에 미치는 영향을 분석하였다.

## 📎 Related Works

기존의 신경망 언어 모델 연구 중에는 문자 수준의 정보를 통합하려는 시도가 있었다.

- **Character-aware models**: Kim et al. (2015)은 문자 임베딩을 사용하여 형태론적 특징을 캡처하였고, Wieting et al. (2016)의 Charagram은 문자 n-gram을 통해 미등록 단어(Out-of-vocabulary) 문제를 해결하려 하였다.
- **Sub-word information**: FastText (Bojanowski et al., 2016)는 서브워드 정보를 활용해 형태론적으로 풍부한 언어의 표현력을 높였다.
- **Contextual representations**: ELMo (Peters et al., 2018)는 문자 수준의 합성곱(Convolution)을 통해 단어 내부 구조를 반영하였으며, Flair (Akbik et al., 2019)는 문자 임베딩과 문맥 문자열 임베딩을 결합하였다.

그러나 이러한 기존 연구들의 한계는 주로 텍스트 생성이나 일반적인 NLP 벤치마크(예: Penn Treebank)의 성능 향상에 집중했을 뿐, 모델이 실제로 '문자 구성을 이해하고 있는가'를 직접적으로 평가하지 않았다는 점이다. 또한, 최근의 LLM들은 다시금 토큰 중심의 학습으로 회귀하면서 이러한 문자 수준의 이해도가 더욱 결여되었을 가능성이 크다.

## 🛠️ Methodology

### 전체 파이프라인 및 시스템 구조

연구진은 4개의 주요 LLM(GPT-4, Claude, Gemini 1.5, Mistral 7B)을 대상으로, 위키피디아 코퍼스에서 무작위로 추출한 텍스트를 사용하여 다음의 문자 수준 작업을 수행하게 한다.

### 주요 구성 요소 및 평가 작업

1. **Word Retrieval (단어 검색)**: 텍스트 내에서 특정 문자가 포함된 모든 단어를 찾는 작업이다.
2. **Character Insertion / Deletion / Replacement (삽입/삭제/교체)**: 지정된 위치에 문자를 삽입하거나, 특정 문자를 삭제하거나, 다른 문자로 교체하는 작업이다.
3. **Character Reordering (문자 재배열)**: 아나그램(Anagram)과 같이 단어 내 문자의 순서를 바꾸어 새로운 단어를 만드는 작업이다.
4. **Character Counting (문자 개수 세기)**: 특정 문자나 모음/자음의 개수를 정확히 세는 작업이다.

### 비교 실험 설계 (Token-level tasks)

문자 수준의 실패가 '토큰화' 때문인지 확인하기 위해, 위 작업들을 토큰 단위로 변환하여 대조군으로 설정하였다.

- **Sentence Retrieval**: 특정 단어가 포함된 문장을 찾는 작업.
- **Word Insertion / Deletion / Replacement**: 문장 내에서 단어를 삽입, 삭제, 교체하는 작업.
- **Word Reordering / Counting**: 단어의 순서를 바꾸거나 단어의 개수를 세는 작업.

### 성능 측정 지표

정밀도(Precision), 재현율(Recall), 그리고 $F\text{-score}$를 사용하여 성능을 측정하며, 재배열 및 개수 세기 작업의 경우 정확도(Accuracy)를 보고한다.

## 📊 Results

### 실험 환경

- **대상 모델**: GPT-4, Claude, Gemini 1.5, Mistral 7B.
- **데이터**: Wikipedia corpus에서 무작위 샘플링.
- **비교군**: 인간 작업자(Human annotators).

### 주요 정량적 결과

- **문자 수준 vs 토큰 수준**: 모든 모델에서 문자 수준의 작업 성능이 토큰 수준보다 현저히 낮게 나타났다. 예를 들어, 단어 교체(Replacement) 작업의 경우 토큰 수준에서는 거의 완벽에 가까운 성능을 보였으나, 문자 수준에서는 매우 낮은 $F\text{-score}$를 기록하였다.
- **인간과의 격차**: 인간은 거의 모든 작업에서 완벽에 가까운 성능을 보인 반면, LLM은 매우 단순한 작업에서도 실패하였다.
- **특이 사항**: Character Reordering(재배열) 작업에서는 다른 작업보다 상대적으로 높은 성능을 보였는데, 이는 학습 데이터에 아나그램 관련 데이터가 많이 포함되어 있어 패턴을 암기했을 가능성이 크다고 분석된다.
- **수치적 제약의 한계**: '세 번째 문자'와 같이 숫자로 위치를 지정하는 작업에서 성능이 급격히 저하되었으며, 이는 토큰 수준의 작업에서도 일부 관찰되었다.

### 다국어 실험 결과

- **중국어**: 영어보다 약간 더 나은 성능을 보였다. 이는 한자(Logographic) 특성상 문자가 곧 단어인 경우가 많아 모델이 서로 다른 수준(문자 vs 단어)을 구분할 필요가 적기 때문으로 추정된다.
- **한국어**: 가장 낮은 성능을 보였다. 특히 문자를 자음과 모음(Letter/Jamo) 단위로 분해하여 분석했을 때, 모델은 거의 아무런 이해도를 보이지 못했으며 정상적인 출력조차 생성하지 못하는 경우가 많았다.
- **일본어**: 로고그래픽(Kanji)과 표음문자(Hiragana, Katakana)가 혼용되는 복잡성으로 인해 낮은 성능을 기록하였다.

## 🧠 Insights & Discussion

### 강점 및 한계

본 논문은 LLM의 고질적인 약점인 '토큰 맹목성(Token-blindness)'을 체계적인 실험을 통해 입증하였다. 특히 토큰 수준의 작업과 문자 수준의 작업을 대조시킨 설계는 LLM의 한계가 단순한 추론 능력 부족이 아니라 데이터 표현 방식(Representation)의 근본적인 문제임을 명확히 보여준다.

### 비판적 해석 및 논의

LLM이 토큰 단위로 작동한다는 점은 효율적인 연산을 가능하게 하지만, 언어의 미세 구조를 무시하게 만든다. 저자들은 이를 해결하기 위해 다음과 같은 방향을 제시한다.

1. **Multi-level Embedding**: BERT의 세그먼트나 포지션 임베딩처럼, 문자가 단어 내에서 어느 위치에 있는지에 대한 정보를 담은 문자 임베딩을 직접 도입하는 방식이다.
2. **Visual Features**: 인간이 글자를 시각적으로 인식하듯, 컴퓨터 비전 기술을 통합하여 텍스트를 시각적 이미지로 인식하게 함으로써 문자 구성을 파악하는 방법이다.

특히 한국어의 경우, 단순한 문자-토큰 매핑을 넘어 초성-중성-종성의 결합 구조라는 특수한 특성이 있어, 기존의 토큰화 방식으로는 한계가 명확하며 이를 해결하기 위한 별도의 아키텍처적 접근이 필요함을 시사한다.

## 📌 TL;DR

본 연구는 최신 LLM들이 단어의 구성 성분인 **문자 수준의 이해도가 심각하게 결여**되어 있음을 밝혀냈다. 토큰 단위 작업에서는 인간 수준의 성능을 보이지만, 문자 단위의 단순 조작 작업에서는 처참한 실패를 보이며, 이는 모델이 언어를 '의미 단위의 덩어리(Token)'로만 인식하고 '문자의 조합'으로 인식하지 못하기 때문이다. 이 연구는 향후 LLM이 진정한 언어적 정밀함을 갖추기 위해 문자 임베딩이나 시각적 인식 기능을 통합해야 한다는 필요성을 제기한다.
