# Learning to Plan by Updating Natural Language

Yiduo Guo, Yaobo Liang, Chenfei Wu, Wenshan Wu, Dongyan Zhao, Nan Duan (2023)

## 🧩 Problem to Solve

대규모 언어 모델(LLM)은 다양한 기본 자연어 처리 작업에서 뛰어난 성능을 보이지만, 긴 소설 작성, 대규모 프로젝트 코딩, 복잡한 수학 문제 해결과 같은 고난도 작업(complex tasks)을 수행하는 데에는 여전히 어려움을 겪는다. 이러한 복잡한 작업을 완수하기 위해서는 단순한 기본 능력을 넘어, 단계별 해결책과 실수 방지를 위한 행동 지침이 포함된 '작업 계획(task plan)'이 필수적이다.

전문가가 직접 고품질의 작업 계획을 작성하는 것은 비용과 시간이 많이 소요되며, LLM에게 직접 계획 생성을 요청할 경우 피드백 과정이 없다면 사실적 오류(factual errors)가 포함되거나 단계가 불완전한 계획이 생성될 가능성이 높다. 따라서 본 논문의 목표는 LLM이 훈련 데이터의 오류 피드백을 통해 스스로 작업 계획을 반복적으로 업데이트하고 학습함으로써, 복잡한 문제를 해결할 수 있는 고품질의 자연어 작업 계획을 자동으로 획득하는 방법을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 **'오류로부터 학습하여 자연어 계획을 업데이트한다'**는 직관에 기반한다. 구체적인 기여 사항은 다음과 같다.

1.  **Learning to Plan 방법론 제안**: 현재의 계획으로 해결하지 못한 오답 샘플을 찾고, 이를 해결하기 위한 새로운 단계별 솔루션을 생성하며, 검증 세트를 통해 최적의 업데이트 내용을 선택하여 계획에 추가하는 반복적 학습 루프를 설계하였다.
2.  **계획 압축(Plan Compression) 도입**: 계획이 무분별하게 길어짐에 따라 발생하는 토큰 제한 문제와 일반화 성능 저하를 막기 위해, 필수 정보만 유지하도록 LLM을 이용해 계획을 압축하는 단계를 포함하였다.
3.  **Task Plan Transfer 가능성 확인**: 한 모델(예: ChatGPT)이 학습한 자연어 작업 계획이 다른 모델(예: GPT-4)의 성능을 직접적으로 향상시킬 수 있음을 보여줌으로써, 자연어 기반의 새로운 전이 학습(transfer learning) 패러다임을 제시하였다.

## 📎 Related Works

본 논문은 다음과 같은 기존 연구들과의 차별점을 가진다.

-   **Prompt Engineering**: 이산적 프롬프트(discrete prompts)나 연속적 프롬프트(continuous prompts) 연구들이 존재하지만, 이들은 기존 프롬프트를 기반으로 내용을 업데이트하거나 모델의 임베딩 공간에서 튜닝하므로 인간이 이해하기 어렵거나 작업의 구체적인 정보를 유도하는 능력이 부족하다.
-   **Task Planning Methods**: 
    -   **피드백 없는 방식**: Chain of Thought(CoT)나 Least-to-most prompting은 문제를 하위 문제로 분해하지만, 생성된 과정의 오류를 수정하는 피드백 메커니즘이 없다.
    -   **피드백 기반 방식**: Self-refine이나 Tree of Thoughts는 개별 솔루션의 품질을 높이지만, 모든 샘플에 적용 가능한 '일반적인 계획'을 학습하여 저장하는 방식은 아니다.
    -   **APE 및 Instruction Induction**: 작업 지침(instruction)을 최적화하지만, 본 논문처럼 구체적인 단계별 해결책(step-by-step solutions)과 행동 지침을 학습하여 계획을 구축하는 것과는 차이가 있다.

## 🛠️ Methodology

본 방법론은 크게 **학습 단계(Learning Phase)**와 **테스트 단계(Test Phase)**로 나뉜다.

### 1. 전체 파이프라인
학습 단계에서는 다음과 같은 세 가지 단계가 하나의 반복 주기로 수행된다.

#### ① 현재 계획으로 오답 샘플 찾기 (Finding Wrong Samples)
$t$번째 반복에서 현재 작업 계획 $p_{t-1}$을 프롬프트에 넣어 데이터 배치 $D_t = \{x_i, y_i\}$에 대해 예측 $H(x_i, p_{t-1})$을 수행한다. 예측값과 실제 정답 $y_i$를 비교하여 틀린 샘플들만 모아 오답 집합 $\{x_i, H(x_i, p_{t-1}), y_i\}$을 구성한다.

#### ② 계획 업데이트 내용 계산 (Computing Plan Update)
오답 집합에서 무작위로 $m$개의 샘플을 선택하고, 기존 계획 $p_{t-1}$과 함께 LLM에 입력하여 오답을 해결할 수 있는 새로운 단계별 솔루션과 지침 $\nabla p$ 후보군을 $K$개 생성한다.
-   **최적 후보 선택**: 각 $\nabla p$ 후보를 기존 계획에 임시로 추가한 후, 검증 세트 $D_{valid}$에서 성능을 측정한다. 최근 기록된 평균 성능보다 일정 임계치 이상 향상된 후보 중 가장 성능이 좋은 것을 $\nabla p_t$로 선택한다.
-   **정보 압축**: 선택된 $\nabla p_t$ 내의 중복 내용을 제거하고 일반성을 높이기 위해 LLM에게 압축 프롬프트를 제공하여 핵심 정보만 남긴 압축된 업데이트 내용을 생성한다.

#### ③ 계획 업데이트 (Updating the Plan)
최종 선택되고 압축된 $\nabla p_t$를 기존 계획 $p_{t-1}$의 끝에 직접 추가하여 새로운 계획 $p_t$를 생성한다. 더 이상 성능 향상이 없는 경우 학습을 중단한다.

### 2. 추론 절차 (Test Phase)
학습이 완료된 최종 작업 계획 $p$를 테스트 샘플의 프롬프트에 삽입한다. LLM은 이 계획에 명시된 단계별 솔루션을 가이드 삼아 문제를 해결한다.

## 📊 Results

### 실험 설정
-   **데이터셋**: 총 5가지 추론 유형의 8개 데이터셋을 사용하였다.
    -   수학적 추론: AMPS (10개 작업), Math (7개 작업)
    -   인과 추론: Causal Judgment, CRASS
    -   논리 추론: LSAT, Date Understanding
    -   심볼릭 추론: Last Letter Concatenation (LLC)
    -   조합 추론: SayCan
-   **모델**: GPT-3.5-turbo (ChatGPT)를 기본 모델로 사용하였다.
-   **비교 대상(Baselines)**: Zero-shot/Few-shot CoT, Self-Plan (단순 생성), Self-Plan with selection, kNN selection, APE.

### 주요 결과
-   **정량적 성능 향상**: AMPS 수학 데이터셋의 10개 작업에서 Zero-shot CoT 대비 평균 성능이 $18.3\%$ 향상되었으며, Few-shot 설정에서도 $7\%$의 향상을 보였다.
-   **비수학적 추론 작업**: LLC 작업의 경우 성능이 $44.6\%$에서 $75.2\%$로 대폭 상승하는 등 다양한 추론 영역에서 효과를 입증하였다.
-   **전이 학습 효과**: ChatGPT가 학습한 작업 계획을 GPT-4에 적용했을 때, GPT-4의 Zero-shot 성능이 향상됨을 확인하였다 (예: Algebra 작업 $57.8 \rightarrow 58.9$, SayCan 작업 $72.0 \rightarrow 80.0$).
-   **오픈소스 모델 적용**: Vicuna-13b-v1.5 모델에서도 동일한 방법론으로 계획을 학습시켜 성능을 높일 수 있음을 확인하여, 본 방법론이 모델 종류에 상관없이 일반적인 알고리즘임을 증명하였다.

## 🧠 Insights & Discussion

### 강점 및 의의
본 연구는 LLM이 단순히 프롬프트를 잘 받는 것을 넘어, **'경험(오류)을 통해 지식(작업 계획)을 축적하고 이를 자연어로 저장'**할 수 있음을 보여주었다. 특히, 학습된 계획이 모델 간에 전이될 수 있다는 점은 자연어가 LLM 사이의 지식 전송 매개체(portable knowledge)가 될 수 있음을 시사한다.

### 한계 및 비판적 해석
1.  **도메인 제약**: 본 방법론은 작업이 명확하게 정의된(well-defined) 환경에서 작동한다. 따라서 정답이 모호하거나 작업 정의가 어려운 Open-domain 설정에서는 적용이 어려울 것으로 보인다.
2.  **기초 인지 능력의 한계**: 단순한 단어 오해, 숫자 오기입과 같은 기초적인 지각 오류는 계획을 업데이트하는 것만으로는 해결할 수 없으며, 이는 모델 자체의 파라미터 업데이트가 필요함을 의미한다.
3.  **수렴성 보장 부재**: 반복적 업데이트 과정이 이론적으로 최적의 계획에 수렴한다는 수학적 보장이 없으며, 실험적인 결과에 의존하고 있다.

## 📌 TL;DR

본 논문은 LLM이 복잡한 과제를 수행하기 위해 필요한 **'작업 계획(Task Plan)'을 오류 피드백을 통해 반복적으로 학습하고 업데이트하는 방법론**을 제안한다. 8개의 추론 데이터셋에서 성능 향상을 입증하였으며, 특히 한 모델이 학습한 자연어 계획이 다른 모델의 성능을 높이는 **'계획 전이(Plan Transfer)'**가 가능하다는 점을 발견하였다. 이는 LLM의 성능 향상을 위해 모델 재학습 없이도 자연어 기반의 경험 축적만으로 복잡한 추론 능력을 강화할 수 있는 실용적인 방향을 제시한다.
