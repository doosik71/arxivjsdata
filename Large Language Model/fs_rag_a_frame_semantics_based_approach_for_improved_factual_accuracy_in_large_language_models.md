# FS-RAG: A Frame Semantics Based Approach for Improved Factual Accuracy in Large Language Models

Harish Tayyar Madabushi (2024)

## 🧩 Problem to Solve

본 논문은 거대 언어 모델(Large Language Models, LLMs)의 고질적인 문제인 **사실적 환각(Factual Hallucinations)** 현상을 해결하고자 한다. LLM은 매끄러운 문장을 생성하지만, 사실과 다른 내용을 마치 진실인 양 출력하는 경향이 있다. 이는 모델의 파라미터 메모리가 사전 학습 데이터의 양에 비해 현저히 작아, 정보를 압축하는 과정에서 통계적 패턴에 의존하게 되기 때문이다.

이를 해결하기 위해 외부 지식을 참조하는 **RAG(Retrieval Augmented Generation)** 방식이 널리 사용되고 있으나, 기존의 RAG는 다음과 같은 한계가 있다.

1. **의미적 유사성의 한계**: 기존의 키워드 기반 또는 시맨틱 유사성(Semantic Similarity) 기반 검색은 쿼리와 사실(Fact) 간의 의미적 거리만 측정한다. 하지만 논리적으로 연결된 정보가 반드시 의미적으로 유사한 것은 아니므로, 복잡한 추론이 필요한 질문의 경우 관련 정보를 제대로 검색하지 못하는 경우가 많다.
2. **불투명성**: 최근 LLM을 이용해 쿼리를 분해하거나 Chain-of-Thought(CoT)를 생성하여 검색 성능을 높이려는 시도가 있으나, 이러한 방식은 내부 동작 과정이 불투명하여 오류 수정이 어렵다.

따라서 본 논문의 목표는 인지언어학의 **프레임 의미론(Frame Semantics)**을 도입하여, 투명하고 수정 가능하며 논리적 연결성을 고려한 새로운 사실 정보 저장 및 검색 시스템인 **FS-RAG**를 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 단어의 의미가 단순한 정의가 아니라, 그 단어가 작동하는 **개념적 '프레임(Frame)' 또는 '스키마(Schema)'** 내에서 이해된다는 프레임 의미론을 RAG의 인덱싱과 검색 과정에 적용한 것이다.

- **프레임 기반 인덱싱 및 검색**: 텍스트를 단순 벡터나 키워드가 아닌, 해당 텍스트가 호출하는 '프레임' 단위로 저장하고 검색한다.
- **논리적 경로의 근사화**: 질문 프레임과 연관된 다른 프레임 간의 관계를 활용함으로써, 의미적으로는 멀지만 논리적으로 연결된 사실들을 효과적으로 검색할 수 있도록 한다.
- **투명성과 해석 가능성**: 검색 과정에서 어떤 프레임이 사용되었는지가 명시적으로 드러나므로, 시스템의 오류를 분석하고 사람이 직접 수정할 수 있는 해석 가능성을 제공한다.

## 📎 Related Works

논문에서는 기존의 RAG 개선 방식들을 다음과 같이 언급하며 차별점을 제시한다.

- **쿼리 분해 및 CoT 기반 검색**: LLM을 사용하여 질문을 하위 질문으로 나누거나(Query Decomposition), 중간 추론 단계(Chain-of-Thought)를 생성하여 검색어를 도출하는 방식이다. 하지만 저자는 이러한 방식들이 '불투명한 메커니즘(Opaque Mechanism)'이며, 파인튜닝을 통해 성능을 높이려 할수록 내부 과정이 더욱 모호해지는 문제가 있다고 지적한다.
- **기존 검색 시스템**: 키워드 매칭이나 밀집 벡터(Dense Vector) 기반 검색은 의미적 유사성에 의존하므로, 논리적 연결 고리가 필요한 지식 추출 작업에는 부적합하다고 설명한다.

FS-RAG는 인지언어학적 이론에 기반하여 검색 과정을 구조화함으로써, LLM의 블랙박스적 특성에 의존하지 않고 명시적인 프레임 관계를 통해 정보를 탐색한다는 점에서 차별화된다.

## 🛠️ Methodology

FS-RAG의 전체 파이프라인은 크게 세 단계의 프레임 처리 과정으로 구성된다.

### 1. 프레임 식별 및 인덱싱 (Frame Identification & Indexing)

사전 처리 단계에서 모든 개별 사실(Factoids)을 그들이 호출하는 2~4개의 주요 프레임으로 인덱싱한다.

- **프레임 생성**: FrameNet과 같은 기존 데이터베이스는 도메인 제한과 '트리거 단어' 중심의 한계가 있어, GPT-4를 사용하여 데이터셋에 적합한 프레임을 동적으로 생성한다.
- **프레임 세트 관리**: 새롭게 생성된 프레임이 기존 프레임 세트와 중복되는지 확인하기 위해 Sentence-BERT를 사용하여 의미적 유사성을 측정하고, GPT-4가 최종적으로 새로운 프레임의 추가 여부를 결정하는 부트스트래핑(Bootstrapping) 과정을 거친다.

### 2. 추론 시 프레임 추출 (Inference Time Frame Extraction)

사용자의 질문이 입력되면 다음과 같은 절차로 검색을 수행한다.

- **질문 프레임(Question Frame) 식별**: 질문에서 가장 중요한 단일 프레임을 추출한다. (예: "별자리의 위치가 왜 변하는가?" $\rightarrow$ `CELESTIAL_MOTION`)
- **프레임 관계 확장**: 질문 프레임과 논리적으로 연관된 추가 프레임들을 식별한다. 이는 1차적인 프레임 중첩(First-order overlap)만으로는 부족한 정보를 보완하기 위함이며, 추론 단계의 프록시(Proxy) 역할을 한다.

### 3. 사실 검색 (Fact Retrieval)

최종적으로 식별된 '질문 프레임'과 '연관 프레임'들에 태깅된 사실들을 저장소에서 추출하여 LLM의 컨텍스트로 제공한다.

**주요 프로세스 요약:**
$$\text{Question} \rightarrow \text{Question Frame} \rightarrow \text{Related Frames} \rightarrow \text{Relevant Facts}$$

## 📊 Results

### 실험 설정

- **데이터셋**: 과학 질문과 사실, 추론 트리가 포함된 **Entailment Bank**를 사용하였다.
- **평가 지표**: $\text{Retrieval@k}$ (여기서 $k \in \{35, 40, 45\}$)를 사용하여, 전체 코퍼스에서 정답에 필요한 사실들을 얼마나 잘 추출하는지 측정하였다.
- **비교 대상(Baselines)**:
    1. **RAKE Search**: 전통적인 키워드 추출 도구인 RAKE를 사용하여 검색어를 생성하고 단순 문자열 매칭을 수행하는 방식.
    2. **GPT-4 Search**: LLM(GPT-4)이 직접 질문에 적합한 검색어를 생성하여 검색하는 방식.

### 결과 분석

실험 결과, FS-RAG가 모든 $k$ 값에서 baseline들을 유의미하게 앞서는 성능을 보였다.

| Metric | RAKE Search (B1) | GPT-4 Search (B2) | FS-RAG (Ours) |
| :--- | :---: | :---: | :---: |
| Recall@35 | 0.330 | 0.385 | **0.439** |
| Recall@40 | 0.333 | 0.390 | **0.464** |
| Recall@45 | 0.338 | 0.396 | **0.473** |

이 결과는 프레임 기반의 인덱싱과 검색이 단순 키워드 매칭이나 LLM 기반의 검색어 생성보다 관련 사실을 더 정확하게 찾아낼 수 있음을 시사한다. 특히, 이는 복잡한 추론이 필요한 Task 3(전체 코퍼스에서 사실 추출)를 Task 2(제한된 사실들 사이에서 추론) 수준으로 단순화할 수 있는 가능성을 보여준다.

## 🧠 Insights & Discussion

### 강점 및 의의

- **해석 가능성(Interpretability)**: 각 단계에서 어떤 프레임이 생성되고 연결되었는지가 텍스트로 출력되므로, 검색 실패 시 어떤 프레임 관계가 누락되었는지 분석하고 수정할 수 있다.
- **이론적 토대**: 단순한 휴리스틱이 아닌 인지언어학의 프레임 의미론을 LLM의 검색 메커니즘에 결합하여 새로운 접근 방식을 제시하였다.

### 한계 및 미해결 과제

- **단일 도메인 실험**: 본 연구는 Entailment Bank라는 특정 과학 도메인에서만 검증되었다. 다양한 도메인에서의 일반화 성능에 대한 추가 검증이 필요하다.
- **최종 생성물 평가 부재**: 본 논문은 '사실 검색(Retrieval)' 단계의 성능 향상에 집중하였으며, 이것이 실제로 최종 답변의 환각 현상을 얼마나 줄였는지에 대한 엔드-투-엔드(End-to-End) 정량 평가는 수행하지 않았다. (저자는 기존 연구를 근거로 검색 성능 향상이 환각 감소로 이어질 것이라 주장한다.)
- **프레임 생성의 최적화**: 현재는 GPT-4 프롬프팅에 의존하고 있으나, 프레임 식별 및 관계 정의를 위한 전용 모델의 파인튜닝이 필요함을 언급하고 있다.

## 📌 TL;DR

본 논문은 LLM의 사실적 환각을 줄이기 위해 인지언어학의 **프레임 의미론(Frame Semantics)**을 도입한 **FS-RAG** 시스템을 제안한다. 정보를 단순 벡터가 아닌 '프레임' 단위로 인덱싱하고 프레임 간의 관계를 통해 검색함으로써, 의미적으로는 멀지만 논리적으로 연결된 정보를 효과적으로 찾을 수 있게 한다. 실험 결과, 기존 키워드 및 LLM 기반 검색보다 높은 Recall 성능을 보였으며, 검색 과정이 투명하여 수정 및 분석이 가능하다는 이점이 있다. 이 연구는 향후 인지언어학과 LLM 연구를 통합하여 더 정확하고 해석 가능한 지식 검색 시스템을 구축하는 데 기여할 것으로 보인다.
