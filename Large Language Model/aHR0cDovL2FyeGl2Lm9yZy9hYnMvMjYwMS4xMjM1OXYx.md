# Zero-Shot Embedding Drift Detection: A Lightweight Defense Against Prompt Injections in LLMs

Anirudh Sekar, Mrinal Agarwal, Rachel Sharma, Akitsugu Tanaka, Jasmine Zhang, Arjun Damerla, Kevin Zhu (2026)

## 🧩 Problem to Solve

본 논문이 해결하고자 하는 핵심 문제는 대규모 언어 모델(LLM) 애플리케이션에서 발생하는 **Prompt Injection(프롬프트 주입)** 공격이다. 특히 이메일이나 사용자 생성 콘텐츠와 같은 간접적인 입력 채널을 통해 모델의 정렬(Alignment) 안전장치를 우회하고, 유해하거나 의도하지 않은 출력을 유도하는 Indirect Prompt Injection(IPI)의 위험성이 증대되고 있다.

기존의 대응 방식인 RLHF(인간 피드백 기반 강화 학습)나 미세 조정(Fine-tuning)만으로는 정교한 적대적 프롬프트를 완전히 차단하기 어려우며, 최근의 탐지 모델들은 계산 비용이 너무 높거나 특정 모델에 종속적인 패치 형태라는 한계가 있다. 따라서 본 연구의 목표는 다양한 LLM 아키텍처에 범용적으로 적용 가능하면서도, 계산 오버헤드가 적고 효율적인 **경량화된 제로샷(Zero-shot) 탐지 메커니즘**을 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 적대적 프롬프트가 겉으로 보기에는 정상적인 텍스트처럼 보일지라도, **임베딩 공간(Embedding Space)에서는 정상 프롬프트와 비교해 미세한 의미적 변화(Semantic Shift)를 일으킨다**는 점에 착안한 것이다.

주요 기여 사항은 다음과 같다:
1. **ZEDD(Zero-Shot Embedding Drift Detection) 프레임워크 제안**: 모델 내부 구조에 대한 접근 권한이나 공격 유형에 대한 사전 지식 없이도, 임베딩의 '드리프트(Drift, 편차)'를 측정하여 주입 공격을 탐지하는 경량화된 방어 체계를 제안한다.
2. **GMM 및 KDE 기반의 플래깅(Flagging) 기법**: Gaussian Mixture Modeling(GMM)과 Kernel Density Estimation(KDE)을 활용하여 임베딩 분포를 분석함으로써, 오탐지율(False Positive Rate)을 최소화하면서 주입된 프롬프트를 효과적으로 식별하는 임계값 설정 방법을 도입하였다.
3. **범용적 성능 검증**: Llama 3, Qwen 2, Mistral 등 다양한 LLM 아키텍처에서 실험을 진행하여, 임베딩 드리프트가 모델 종류에 상관없이 강력하고 전이 가능한(Transferable) 신호임을 입증하였다.

## 📎 Related Works

기존의 프롬프트 주입 방어 연구들은 주로 다음과 같은 접근 방식을 취해왔다:
- **정렬 및 미세 조정**: RLHF 등을 통해 모델 자체의 안전성을 높이려 했으나, gradient-based 최적화 등을 통한 적대적 프롬프트에 여전히 취약하다.
- **임베딩 기반 분류기**: 일부 연구에서 임베딩을 활용한 탐지를 시도했으나, 주로 Logistic Regression, XGBoost, Random Forest와 같은 외부 분류기를 사용하여 최적화했으며, LLM의 임베딩 공간 자체를 최적화하여 분류하는 방식은 부족했다.
- **대형 모델 및 규칙 기반 필터**: 별도의 대형 모델을 사용하여 입력을 검사하거나 복잡한 규칙 기반 필터를 적용하는 방식은 계산 비용이 높고 지연 시간(Latency)이 발생하여 실시간 서비스 적용에 한계가 있다.

ZEDD는 이러한 무거운 탐지 모델 대신 단순한 벡터 간 거리 측정 방식을 채택하여 계산 효율성을 극대화하고, 모델에 구애받지 않는(Model-agnostic) 특성을 가진다는 점에서 차별화된다.

## 🛠️ Methodology

ZEDD의 전체 파이프라인은 크게 세 단계로 구성된다: **임베딩 추출 $\rightarrow$ 의미적 드리프트 계산 $\rightarrow$ 의심 프롬프트 플래깅**.

### 1. 임베딩 추출 (Embedding Extraction)
먼저, 입력 프롬프트를 벡터 표현으로 변환한다. 본 연구에서는 Sentence BERT(All-MPNET-Base-V2), Llama 3 8B, Mistral 7B, Qwen 2 7B 등의 인코더를 사용하였다. 특이점은 인코더를 한 번 미세 조정(Fine-tuning)하여 정상-주입 프롬프트 쌍의 차이를 임베딩 공간에서 더 잘 구분할 수 있도록 최적화한 후, 실제 탐지 단계에서는 추가 학습 없이 제로샷으로 사용하는 방식을 취했다는 점이다.

### 2. 드리프트 측정 (Drift Measurement)
정상 프롬프트 $x$와 주입이 의심되는 프롬프트 $x'$ 사이의 의미적 변화량을 **코사인 유사도(Cosine Similarity)**를 통해 측정한다. 두 벡터 사이의 거리가 멀수록(유사도가 낮을수록) 의미적 변화가 크다고 판단하며, 이를 '드리프트 스코어'로 정의한다.

수식은 다음과 같다:
$$\text{Drift}(x, x') = 1 - \frac{f(x) \cdot f(x')}{\|f(x)\| \cdot \|f(x')\|}$$
여기서 $f(x)$는 모델의 인코더를 통해 추출된 임베딩 벡터를 의미한다.

### 3. 드리프트 탐지 프레임워크 (Drift Detection Framework)
라벨이 없는 실제 환경에서 주입 여부를 판단하기 위해 앙상블 플래깅 접근 방식을 사용한다.

- **Gaussian Mixture Modeling (GMM)**: 드리프트 스코어의 분포를 두 개의 가우시안 성분(정상군과 주입군)으로 모델링한다. 정상 프롬프트 쌍은 낮은 드리프트 값을, 주입 프롬프트 쌍은 높은 드리프트 값을 가진다는 가정을 바탕으로 한다.
- **임계값(Threshold) 결정**: 두 가우시안 밀도 함수 $f_{\text{clean}}(x)$와 $f_{\text{injected}}(x)$ 및 각각의 혼합 가중치 $w$를 이용하여 최적의 결정 임계값을 계산한다:
$$f_{\text{clean}}(x) \cdot w_{\text{clean}} = f_{\text{injected}}(x) \cdot w_{\text{injected}}$$
- **KDE Fallback**: GMM이 수렴하지 않거나 불안정할 경우, 커널 밀도 추정(Kernel Density Estimation)을 통해 분포의 피크(Peak)와 밸리(Valley)를 찾아내어 임계값을 설정하는 대체 메커니즘을 사용한다.
- **최적화**: 오탐지율(FPR)을 3% 이내로 제한하고, 전체 플래깅 비율을 약 50%로 유지하도록 이진 탐색(Binary Search)을 통해 임계값을 미세 조정한다.

## 📊 Results

### 실험 설정
- **데이터셋**: LLMail-Inject 데이터셋을 재주석(Re-annotate)하여 사용하였다.
- **공격 카테고리**: Jailbreak, System Leak, Task Override, Encoding Manipulation, Prompt Confusion의 5가지 유형으로 분류하였다.
- **평가 데이터**: 총 51,603개의 정렬된 쌍(정상-정상 25,801개, 주입-정상 25,802개)을 사용하였다.
- **하드웨어**: NVIDIA B200 GPU를 사용하였으며, 모델별 미세 조정 시간은 약 15~18분으로 매우 짧았다.

### 정량적 결과
- **정확도 및 FPR**: 대부분의 모델에서 93% 이상의 탐지 정확도를 보였으며, 평균 오탐지율(Clean FPR)은 2.93%로 매우 낮게 유지되었다.
- **모델별 성능 (Table 2 기준)**:
    - **Llama 3 8B**: 정확도 95.32%, F1 스코어 95.30%, FPR 5.5%
    - **Mistral 7B**: 정확도 95.55%, F1 스코어 95.50%, FPR 2.3%
    - **Qwen 2 7B**: 정확도 95.46%, F1 스코어 95.38%, FPR 2.2%
    - **SBERT**: 정확도 90.75%, F1 스코어 89.84%, FPR 1.7%
- **카테고리별 분석**: System Leak 및 Encoding Manipulation에서 매우 높은 탐지율을 보였으나, Jailbreak 유형에서는 상대적으로 성능이 약간 낮게 나타났다. 특히 SBERT 모델이 다른 LLM 기반 인코더에 비해 탐지 성능이 낮았다.

### 결론적 성과
ZEDD는 기존의 무거운 분류 모델들에 비해 매우 빠른 처리 속도(약 8분 내외의 학습 및 신속한 추론)를 가지면서도, Precision과 F1 스코어 면에서 우수한 성능을 입증하였다.

## 🧠 Insights & Discussion

### 강점
ZEDD의 가장 큰 강점은 **극도의 경량성**과 **범용성**이다. 특정 공격 패턴을 학습하는 것이 아니라 '의미적 편차'라는 일반적인 신호를 포착하기 때문에, 새로운 형태의 공격이 등장하더라도 유연하게 대응할 수 있다. 또한, 모델 내부 가중치에 접근할 필요가 없는 블랙박스 형태의 방어가 가능하여 상용 API 환경에서도 적용 가능하다.

### 한계 및 비판적 해석
1. **임베딩 모델 의존성**: 탐지 성능이 선택한 임베딩 모델의 품질에 직접적으로 의존한다. SBERT의 성능이 LLM 기반 인코더보다 낮게 나온 점은, 더 정교한 시맨틱 캡처 능력이 탐지 성능의 핵심임을 시사한다.
2. **적대적 우회 가능성**: 공격자가 ZEDD의 메커니즘을 알고 있다면, 임베딩 공간에서의 드리프트를 최소화하면서도 공격 의도를 유지하는 '임베딩 수준의 조작'을 시도할 수 있다.
3. **데이터셋 편향**: 사용된 LLMail-Inject 데이터셋이 주로 이메일 형식에 특화되어 있어, 다른 도메인(예: 채팅, 코드 생성)에서도 동일한 성능이 나올지는 추가 검증이 필요하다.

본 논문은 ZEDD를 완벽한 단독 방어 체계보다는, LLM 파이프라인의 최전방에서 빠르게 1차 필터링을 수행하는 **'강력한 첫 번째 방어선(Strong first defense)'**으로 정의하고 있다.

## 📌 TL;DR

본 연구는 LLM의 프롬프트 주입 공격을 탐지하기 위해 정상 프롬프트와 의심 프롬프트 간의 임베딩 거리(Drift)를 측정하는 **ZEDD 프레임워크**를 제안한다. GMM과 KDE를 이용한 통계적 임계값 설정 방식을 통해 **오탐지율 3% 미만, 정확도 93% 이상**의 고성능 탐지가 가능함을 보였으며, 특히 계산 비용이 매우 낮아 실시간 LLM 시스템에 즉시 통합 가능한 경량 방어 계층으로서의 가치가 높다. 향후 적대적 임베딩 조작에 대한 대응 및 다양한 도메인으로의 확장 연구가 필요할 것으로 보인다.
