# C2HLSC: Can LLMs Bridge the Software-to-Hardware Design Gap?

Luca Collini, Siddharth Garg, Ramesh Karri (2024)

## 🧩 Problem to Solve

본 논문은 소프트웨어 설계(C 코드)와 하드웨어 설계(RTL) 사이의 간극을 줄이는 문제를 다룬다. 일반적으로 High Level Synthesis (HLS) 도구는 C/C++ 코드를 Register Transfer Level (RTL) 기술로 변환하여 하드웨어 설계 속도를 높여주지만, 모든 C 언어 구문을 지원하는 것은 아니다. HLS 도구는 하드웨어 구조로 매핑 가능한 C 언어의 특정 서브셋(subset)만을 지원하며, 동적 메모리 할당, 재귀 함수(recursion), 포인터 사용 등은 지원하지 않거나 제한적이다.

따라서 설계자는 일반적인 C 코드를 HLS 도구가 인식할 수 있는 형태로 직접 수정하는 '리팩토링(refactoring)' 과정을 거쳐야 한다. 그러나 이러한 수동 리팩토링 작업은 시간이 많이 소요될 뿐만 아니라 인간의 실수로 인한 오류가 발생하기 쉽다는 문제점이 있다. 본 연구의 목표는 Large Language Models (LLMs)를 활용하여 일반적인 C 코드를 HLS-synthesizable C 코드로 효율적으로 변환함으로써 하드웨어 설계 공정을 자동화하고 가속화하는 가능성을 탐색하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 LLM이 HDL(Hardware Description Language)을 직접 생성하는 것보다, 이미 방대한 학습 데이터를 보유하고 있는 C 언어 코드의 조작 및 리팩토링에 더 능숙하다는 점을 이용하는 것이다. 즉, LLM을 통해 '소프트웨어 C $\rightarrow$ HLS 호환 C'로의 변환을 수행하고, 이후 기존의 HLS 도구를 통해 하드웨어를 생성하는 우회 경로를 제안한다.

주요 기여 사항은 다음과 같다.

1. **LLM 기반 리팩토링 사례 연구**: NIST 800-22 랜덤성 테스트, QuickSort, AES-128 알고리즘을 대상으로 일반 C 코드를 HLS 호환 코드로 변환하는 과정을 검증하였다.
2. **개발 시간 및 효율성 입증**: 수동 리팩토링 대비 LLM을 이용한 설계 방식이 개발 시간을 획기적으로 단축함을 보여주었다.
3. **Hands-free C2HLSC 프로토타입 제안**: 인간의 개입을 최소화하기 위해 GCC 컴파일러와 Catapult HLS 도구의 피드백 루프를 결합한 자동화 툴의 구조를 제시하였다.

## 📎 Related Works

기존 연구들은 LLM을 사용하여 Verilog나 VHDL과 같은 HDL 코드를 직접 생성하려는 시도를 해왔다. 예를 들어, VeriGen은 Verilog 생성 능력을 향상시키기 위해 LLM을 미세 조정(fine-tuning)하였으며, ChipChat은 AI 모델이 작성한 설계로 실제 테이프아웃(tapeout)까지 진행하였다.

하지만 이러한 접근 방식은 다음과 같은 한계가 있다.

- **학습 데이터의 부족**: C/C++이나 Python에 비해 HDL 데이터의 양이 매우 적어, LLM이 생성한 HDL 코드의 정확도가 낮고 단일 샷(single-shot) 성능이 떨어진다.
- **인터페이스 생성의 어려움**: 하드웨어의 복잡한 인터페이스나 동기화 로직을 정확히 생성하는 데 한계가 있다.

본 논문은 이러한 한계를 극복하기 위해 HDL을 직접 생성하는 대신, LLM이 더 잘 수행하는 C 코드 조작 능력을 활용하여 HLS 도구의 입력값(HLS-compatible C)을 생성하는 전략을 취함으로써 차별점을 둔다.

## 🛠️ Methodology

### 전체 파이프라인 및 프로세스

본 연구에서는 Gemini 및 ChatGPT-3.5 Turbo를 사용하여 일반 C 코드를 HLS 호환 코드로 변환하는 반복적 프롬프팅(iterative prompting) 과정을 설계하였다.

#### 1. 리팩토링 단계 (Case Study 기준)

LLM에게 한 번에 모든 수정을 요구하는 대신, 다음과 같은 단계적 가이드를 제공한다.

- **기초 정리**: 출력문(`printf`) 제거 및 기본적인 작업 정의.
- **인터페이스 변경**: 메모리 기반의 데이터 처리 방식을 하드웨어의 Streaming Interface(데이터를 비트 단위로 실시간 처리하는 방식)로 변경하도록 요청한다.
- **하드웨어 제약 사항 적용**: 재귀 함수를 스택 기반의 반복문으로 변경하고, 포인터를 제거하며, 배열 크기를 정적으로 고정한다.
- **최적화**: `HLSLIBS`를 사용하여 임의 정밀도 정수(arbitrary width integers) 및 고정 소수점 연산(fixed point arithmetic)으로 데이터 타입을 최적화한다.
- **검증 및 수정**: HLS 도구에서 발생한 에러 메시지를 LLM에게 다시 입력하여 코드를 수정하게 한다.

#### 2. Hands-free C2HLSC 프로토타입 구조

인간의 개입을 줄이기 위해 다음과 같은 이중 피드백 루프(Double Feedback Loop) 시스템을 구축하였다.

- **Loop 1 (GCC)**: 생성된 코드가 표준 C 문법에 맞는지 확인하고, 레퍼런스 테스트를 통과하여 기능적 정답성(functional correctness)을 확보한다.
- **Loop 2 (Catapult HLS)**: 코드가 실제로 하드웨어로 합성 가능한지(synthesizable) 확인한다. 합성 에러 발생 시 해당 로그를 다시 LLM의 프롬프트로 입력하여 수정안을 도출한다.

## 📊 Results

### 실험 설정

- **대상 알고리즘**: NIST 800-22 (Frequency, Block Frequency, Cumulative Sums, Overlapping Template Matching), QuickSort, AES-128.
- **HLS 도구**: Catapult HLS (NanGate 45nm 라이브러리, 50 MHz 타겟).
- **비교 지표**: Area Score (면적), Latency (지연 시간), 개발 소요 시간.

### 주요 결과

1. **정량적 성능**: LLM이 생성한 C 코드를 통해 합성된 하드웨어의 Area Score와 Latency는 수동으로 구현한 설계와 유사한 수준을 보였다. (Table I 참조)
2. **개발 시간 단축**: 수동 리팩토링 시 알고리즘당 약 4시간이 소요되었으나, LLM 기반 방식(C2HLSC)은 30분에서 60분 사이로 시간을 대폭 단축하였다.
3. **자동화 가능성**: Hands-free 프로토타입을 통해 인간의 개입 없이도 QuickSort와 AES의 개별 함수들을 성공적으로 리팩토링할 수 있음을 확인하였다.

### 오류 분석

LLM이 생성한 코드의 오류는 두 가지로 분류되었다.

- **컴파일/합성 에러**: 도구의 에러 메시지를 전달하면 LLM이 쉽게 수정할 수 있었다.
- **기능적 에러**: 테스트 케이스 실패로 발견되는 오류로, LLM이 스스로 원인을 찾기 어려워하며 인간이 구체적인 힌트를 제공해야 수정이 가능했다.

## 🧠 Insights & Discussion

### 강점 및 가능성

본 연구는 LLM이 단순한 코드 생성을 넘어, 소프트웨어의 추상적 설계를 하드웨어의 제약 조건에 맞게 변환하는 '중간 매개체' 역할을 수행할 수 있음을 입증하였다. 특히 메모리 기반 처리를 스트리밍 구조로 변경하거나, 재귀를 반복문으로 바꾸는 것과 같은 복잡한 구조적 변경을 성공적으로 수행했다는 점이 고무적이다.

### 한계 및 비판적 해석

- **기능적 정답성의 의존도**: 합성 에러는 도구의 피드백으로 자동 수정이 가능하지만, 논리적 오류(functional error)는 여전히 인간 전문가의 개입이 필요하다는 점이 완전 자동화의 걸림돌이다.
- **표본의 제한**: 테스트한 알고리즘의 수가 적어, 모든 종류의 C 코드에 대해 일반화할 수 있는지에 대한 추가 검증이 필요하다.
- **아키텍처 최적화 미비**: 본 연구는 '합성 가능성'에 초점을 맞추었으며, HLS의 핵심인 Pragma 및 Directive를 통한 세부 아키텍처 최적화(예: 루프 언롤링, 파이프라이닝 최적화)는 다루지 않았다.

## 📌 TL;DR

본 논문은 LLM을 활용하여 일반 C 코드를 HLS(High Level Synthesis) 도구가 인식할 수 있는 특수 C 코드로 리팩토링하는 **C2HLSC** 방법론을 제안한다. 실험 결과, LLM은 포인터 제거, 재귀 함수 변환, 스트리밍 인터페이스 구축 등의 작업을 효과적으로 수행하였으며, 하드웨어 설계 시간을 약 75% 이상 단축시켰다. 이는 LLM이 HDL을 직접 생성하는 한계를 극복하고, 소프트웨어 설계와 하드웨어 구현 사이의 간극을 메우는 강력한 도구가 될 수 있음을 시사한다.
