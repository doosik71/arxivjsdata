# Self-Evaluating LLMs for Multi-Step Tasks: Stepwise Confidence Estimation for Failure Detection

Vaibhav Mavi, Shubh Jaroria, Weiqi Sun (2025)

## 🧩 Problem to Solve

본 논문은 대규모 언어 모델(LLM)이 복잡한 다단계 추론(multi-step reasoning) 작업을 수행할 때 발생하는 오류를 어떻게 효과적으로 탐지할 것인가에 대한 문제를 다룬다. LLM 에이전트가 태스크 플래닝, 대화 시스템, 다단계 질의응답(multi-hop QA)과 같은 고위험(high-stakes) 환경에 배치될 때, 모델의 응답이 정확한지 판단하는 신뢰도 추정(confidence estimation)과 실패 탐지(failure detection)는 시스템의 신뢰성을 보장하는 데 필수적이다.

기존의 신뢰도 추정 연구들은 주로 단일 단계 출력(single-step outputs)에 집중되어 있었으며, 다단계 추론에서 발생하는 특수한 문제들—즉, 추론 체인의 길이가 가변적이라는 점, 어느 단계에서나 오류가 발생할 수 있다는 점, 그리고 후속 단계가 이전 단계의 결과에 의존한다는 점—을 간과하였다. 따라서 본 연구의 목표는 다단계 작업에서 신뢰도 추정의 최적의 입도(granularity)를 조사하고, 단계별 평가가 전체 응답 평가보다 오류 탐지에 얼마나 효과적인지 검증하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 다단계 추론 과정에서 신뢰도 점수를 부여하는 방식을 **전체 응답 기반 스코어링(Response-level scoring)**과 **단계별 스코어링(Step-level scoring)**으로 구분하여 비교 분석하는 것이다.

연구진은 모델이 최종 답변을 내놓은 후 전체 문맥을 보고 한 번에 판단하는 것보다, 각 추론 단계가 진행될 때마다 즉각적으로 신뢰도를 평가하는 것이 오류 탐지 능력을 크게 향상시킨다는 직관을 제시한다. 특히, 최종 결과는 맞았더라도 중간 추론 과정에 오류가 있는 경우(Correct answer through flawed reasoning)를 잡아내기 위해서는 단계별 평가가 필수적임을 입증하였다.

## 📎 Related Works

신뢰도 추정 방식은 크게 두 가지 접근법으로 나뉜다.

1.  **Black-box 접근법**: 모델 내부 파라미터에 접근하지 않고 프롬프팅 기반의 자기 성찰(self-reflection), 자기 일관성(self-consistency), 혹은 외부 평가 모델을 사용하여 사후적으로 응답을 평가하는 방식이다.
2.  **White-box 접근법**: 오픈소스 모델의 파라미터와 활성화 값(activations)에 접근하는 방식이다. 여기에는 로그 확률(log-probabilities)을 이용한 캘리브레이션, 은닉 상태(hidden states)를 이용해 정답 여부를 예측하는 회귀 모델 학습 등이 포함된다.

기존 연구들의 한계는 대부분의 벤치마크가 단일 단계 작업에 국한되어 있어, 다단계 추론의 복잡한 의존성을 고려한 신뢰도 추정 방법론이 부족했다는 점이다. 본 논문은 이러한 공백을 메우기 위해 다단계 작업으로의 확장을 시도하였다.

## 🛠️ Methodology

### 1. 문제 정의 및 수식화
실패 탐지는 입력 $I$에 대한 에이전트의 응답 $R$이 틀렸을 확률 $p \in [0, 1]$를 추정하는 작업으로 정의된다: $p = F(R | I)$. 다단계 상호작용에서는 입력이 초기 문맥 $C$와 쿼리 시퀀스 $Q$로 구성되며, 응답 $R$ 또한 시퀀스로 생성된다.

*   **Response-level scoring (전체 응답 기반)**: 모든 단계의 쿼리와 응답을 하나의 시퀀스로 처리하여 단일 신뢰도 점수를 부여한다.
    $$p = S_{\text{whole}}(R^{[1:n]} | C, Q^{[1:n]})$$
*   **Step-level scoring (단계별 기반)**: 각 단계 $i$의 응답 $R_i$에 대해, 이전까지의 문맥과 응답들을 조건으로 하여 개별 점수를 부여한다.
    $$p_i = F_{\text{step}}(R_i | C, Q^{[1:i]}, R^{[1:i-1]})$$
    최종적으로 모든 $p_i$ 중 하나라도 임계값을 넘으면 전체 응답을 오류로 간주하며, 이는 $p = \min(\{p_i\}_{i=1}^n)$ (신뢰도 기준일 때) 또는 최대 실패 확률을 찾는 방식으로 구현된다.

### 2. 학습 절차 (Training)
학습이 필요한 평가 모델의 경우, **Teacher Forcing** 기법을 채택한다. 모델은 다음 응답을 평가할 때 정답 히스토리($\hat{R}$)를 제공받으며, 학습 목표는 다음과 같다.
$$p_i = F(R_i, | C, Q^{[1:i]}, \hat{R}^{[1:i-1]}) = I\{R_i \neq \hat{R}_i\}$$
여기서 $I\{\cdot\}$는 지시 함수(indicator function)로, 응답 $R_i$가 정답 $\hat{R}_i$와 다를 때 1(실패)을 부여한다. 추론 시에는 정답 없이 모델의 예측값만을 사용한다.

### 3. 평가 방법론 (Evaluated Methods)
*   **Self-verbalized**: LLM이 스스로 자신의 응답에 대한 신뢰도를 텍스트로 출력하게 한다.
*   **Auxiliary evaluators**: Llama-3.2-11B나 GPT-4.1-mini 같은 외부 모델이 응답을 평가하게 한다.
*   **Regression model**: Llama-3.2-11B의 생성 헤드를 회귀 헤드로 교체하여 $[0, 1]$ 범위의 점수를 직접 예측하도록 파인튜닝한다.
*   **Self-certainty (Logits)**: 출력 로짓(logits)과 균등 분포 간의 KL-divergence를 계산하여 확실성을 측정한다.
*   **Activations**: 모델의 최종 레이어 은닉 상태(hidden states)를 입력으로 하는 5층 MLP 분류기를 학습시켜 정답 여부를 예측한다.
*   **PRM (Preference-based Reward Model)**: 정답 응답을 'chosen', 오답을 'rejected'로 하여 학습시킨 보상 모델을 사용한다(본 논문에서는 Response-level에서만 평가).

## 📊 Results

### 1. 실험 설정
*   **데이터셋**: GSM8K(다단계 수학 문제), CoQA(대화형 질의응답), 그리고 비공개 임상 데이터(clinical notes).
*   **지표**: AUC-ROC(정확도와 오류의 구분 능력), FPR@0.9 Recall(재현율 0.9를 달성할 때의 오탐지율, 낮을수록 좋음).

### 2. 주요 결과
*   **입도에 따른 성능**: CoQA 데이터셋에서 단계별 스코어링(step-level)이 전체 응답 스코어링(response-level)보다 모든 방법론에서 월등히 높은 성능을 보였다. GSM8K에서는 그 차이가 상대적으로 적었으나, 여전히 단계별 평가가 유리한 경향을 보였다.
*   **최적 방법론**: **Regression model**이 두 작업 모두에서 가장 높은 AUC-ROC를 기록하며 최상의 성능을 보였다.
*   **특이 사항**:
    *   PRM은 정답 라벨링 작업보다 응답 랭킹에 더 적합함을 확인하였다.
    *   Self-certainty 방식은 GSM8K의 단계별 평가에서 성능이 급격히 떨어졌는데, 이는 도구(tool) 사용으로 인해 로짓(logits)이 왜곡되기 때문으로 분석된다. 반면 Activations 기반 방식은 최종 토큰의 은닉 상태만 사용하므로 이 문제에서 자유로웠다.
*   **추론 오류 탐지**: 최종 답은 맞았지만 중간 과정이 틀린 경우, 단계별 스코어링이 전체 응답 스코어링보다 훨씬 더 효과적으로 오류를 탐지하였다 (Table 2 참조).
*   **실제 데이터 적용**: 비공개 임상 데이터셋에서도 단계별 회귀 모델이 $\text{AUC-ROC} = 0.940$, $\text{FPR@0.9 rec} = 0.152$라는 높은 성능을 기록하여 실무 적용 가능성을 입증하였다.

## 🧠 Insights & Discussion

본 연구는 다단계 추론 작업에서 **'언제' 평가하느냐**가 **'어떻게' 평가하느냐**만큼 중요하다는 것을 보여준다. 

**강점 및 발견**:
단계별 스코어링은 단순한 오류 탐지를 넘어, 모델이 "운 좋게" 맞춘 정답(flawed reasoning but correct answer)을 걸러낼 수 있는 유일한 방법이다. 이는 높은 신뢰성이 요구되는 의료나 법률 분야에서 LLM을 배치할 때 매우 중요한 안전장치가 될 수 있다.

**한계 및 논의**:
1.  **구현 복잡도**: 단계별로 점수를 매기는 것은 전체 응답을 한 번에 평가하는 것보다 계산 비용이 높고 구현이 복잡하다.
2.  **작업별 특성**: 수학 문제(GSM8K)가 대화형 QA(CoQA)보다 평가가 더 어려웠는데, 이는 수학적 추론의 엄격함이 평가 모델에게도 더 높은 난이도를 요구하기 때문으로 보인다.
3.  **PRM의 확장성**: 본 논문에서는 PRM을 Response-level에서만 평가하였는데, 다단계 작업에서 효과적인 단계별 PRM을 구축하는 방법은 향후 연구 과제로 남아있다.

## 📌 TL;DR

이 논문은 LLM의 다단계 추론 실패를 탐지하기 위해 **전체 응답 기반 평가**와 **단계별 평가**를 비교 분석하였다. 실험 결과, 각 단계마다 신뢰도를 측정하는 **Step-level scoring**이 특히 복잡한 대화나 추론 과정에서 오류 탐지 능력을 크게 향상시키며, 특히 중간 과정이 틀렸음에도 최종 답만 맞은 경우를 잡아내는 데 결정적인 역할을 한다. 가장 성능이 좋은 방법은 Llama-3.2 기반의 **회귀 모델(Regression model)**이었으며, 이는 향후 고신뢰성 LLM 시스템 구축을 위한 실질적인 프레임워크를 제공한다.
