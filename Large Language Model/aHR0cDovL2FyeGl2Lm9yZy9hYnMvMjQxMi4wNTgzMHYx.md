# Large Language Models Merging for Enhancing the Link Stealing Attack on Graph Neural Networks

Faqian Guan, Tianqing Zhu, Wenhan Chang, Wei Ren, and Wanlei Zhou (2024)

## 🧩 Problem to Solve

본 논문은 그래프 신경망(Graph Neural Networks, GNNs)을 대상으로 하는 **Link Stealing Attack**(링크 탈취 공격)의 효율성과 일반화 능력을 높이는 문제를 해결하고자 한다. Link Stealing Attack은 공격자가 타겟 모델을 쿼리하여 노드 간의 민감한 관계(엣지)를 추론하는 프라이버시 공격이다. 이는 금융 거래 네트워크에서의 사기 체인 식별이나 사회 관계망에서의 비공개 관계 추론 등 심각한 보안 위협을 초래할 수 있다.

기존의 연구들은 주로 단일 데이터셋 내에서의 공격에 집중하였으며, 다음과 같은 두 가지 핵심적인 한계점을 가지고 있었다. 첫째, 데이터셋마다 서로 다른 특성 차원과 구조를 가지기 때문에, 하나의 공격 모델이 여러 데이터셋에 적용되는 **Cross-dataset attack**(교차 데이터셋 공격)에 대한 연구가 부족했다. 둘째, 여러 공격자가 협력하여 더 강력한 모델을 만들고 싶어도, 데이터 수집 비용과 프라이버시 문제로 인해 로컬 데이터를 서로 공유하지 못하는 상황에서의 협력 학습 방법이 제시되지 않았다.

따라서 본 논문의 목표는 대규모 언어 모델(Large Language Models, LLMs)과 모델 병합(Model Merging) 기술을 결합하여, 데이터를 공유하지 않고도 여러 공격자의 지식을 통합하고, 학습되지 않은 외부 데이터셋(Out-of-domain)에서도 효과적으로 작동하는 범용적인 Link Stealing Attack 모델을 구축하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 LLM의 유연한 입력 처리 능력과 모델 병합의 지식 통합 능력을 결합하여 범용 공격 모델을 생성하는 것이다.

1. **LLM을 이용한 Cross-dataset 공격**: Transformer 아키텍처의 Attention 메커니즘을 활용하면 가변 길이의 입력을 처리할 수 있으므로, 데이터셋마다 다른 특성 차원과 길이를 가진 그래프 데이터를 통합적으로 처리할 수 있다.
2. **데이터 공유 없는 협력 학습**: 각 공격자가 자신의 데이터로 LLM을 개별적으로 Fine-tuning한 후, 데이터가 아닌 모델 파라미터만을 공유하여 병합함으로써 프라이버시를 유지하며 지식을 통합한다.
3. **가중치 기반 모델 병합**: 단순 평균이 아니라, 각 데이터셋의 특이성(Uniqueness)을 반영한 가중치를 부여하여 병합함으로써, 더 어렵고 복잡한 데이터셋에 대한 일반화 성능을 극대화한다.

## 📎 Related Works

### GNN 프라이버시 공격

GNN은 노드 특성과 관계 정보를 학습하므로 다양한 프라이버시 공격의 대상이 된다. 모델 추출 공격(Model Extraction), 멤버십 추론 공격(Membership Inference), 속성 추론 공격(Attribute Inference) 등이 연구되어 왔다. 특히 Link Stealing Attack은 속성 추론 공격의 일종으로, 타겟 모델의 응답(Posterior Probabilities)을 분석하여 엣지의 존재 여부를 판단한다. 기존 방식들은 주로 사후 확률의 유사성을 비교하거나 데이터에 섭동(Perturbation)을 주어 변화를 관찰하는 방식을 사용했으나, 이는 단일 데이터셋에 종속적이라는 한계가 있다.

### LLM 및 모델 병합

최근 LLM은 방대한 데이터 학습을 통해 뛰어난 일반화 능력을 보여주며, GNN과 결합하여 노드 임베딩을 개선하거나 그래프 구조를 자연어로 표현하여 처리하는 연구가 진행되고 있다. 또한, 서로 다른 태스크로 학습된 LLM들을 병합하여 다재다능한 모델을 만드는 Model Merging 기술(Ties-merging, DARE 등)이 등장하였다. 본 논문은 이러한 LLM의 일반화 능력과 병합 기술을 GNN 프라이버시 공격이라는 새로운 영역에 적용하여 차별점을 둔다.

## 🛠️ Methodology

### 전체 파이프라인

전체 공격 과정은 크게 두 단계로 나뉜다. 첫 번째 단계에서는 각 공격자가 LLM을 도입하여 개별적인 공격 모델을 학습시키고, 두 번째 단계에서는 이 모델들을 병합하여 범용 공격 모델을 생성한다.

### 1단계: LLM 기반 공격 모델 구축 (Introduce LLM)

공격자는 타겟 모델 $T$에 노드 쌍 $\{v, u\}$를 쿼리하여 각 노드의 **Posterior Probabilities**(사후 확률) $p_v, p_u$를 획득한다.

- **Prompt Design**: LLM이 링크 여부를 판단할 수 있도록 다음과 같이 프롬프트를 구성한다.
  - **노드 쌍 정보**: 노드의 텍스트 설명(제목, 초록 등)과 타겟 모델에서 얻은 사후 확률 $p$.
  - **인간의 질문**: "제공된 정보와 GNN 예측 사후 확률을 바탕으로 두 논문 사이에 링크가 존재하는가? Yes 또는 No로 답하라."
  - **LLM 응답**: $Yes$ (링크 존재) 또는 $No$ (링크 없음).
- **Fine-tuning**: 사전 학습된 LLM을 각자의 데이터셋으로 미세 조정한다. 손실 함수로는 Cross-Entropy Loss $L_{CE}$를 사용하며, 예측값 $\hat{y}$와 실제 라벨 $y$ 사이의 오차를 최소화한다.
    $$\hat{y} = \text{LLM}(\text{Prompt}\{(x_u, p_u), (x_v, p_v)\})$$
    $$L_{CE}(y, \hat{y}) = -[y \log(\hat{y}) + (1-y) \log(1-\hat{y})]$$

### 2단계: 모델 병합 (Model Merging)

여러 공격자가 학습시킨 모델 파라미터 $\{\theta_1, \theta_2, \dots, \theta_n\}$와 기본 모델 $\theta$를 통합한다. 여기서 $\delta_t = \theta_t - \theta$는 파라미터의 변화량(Delta)이다.

1. **Drop (가지치기)**: 파라미터 중 영향력이 작은(Magnitude가 낮은) 중복 파라미터를 제거하여 일반화 성능을 높인다. 델타 값의 크기에 따라 드롭 확률 $d_i$를 다르게 부여하며, 제거 후에는 예측값의 일관성을 유지하기 위해 역수 $\frac{1}{1-d_i}$로 스케일링한다.
2. **Elect (선택)**: 모든 모델의 델타 합의 부호($\text{sgn}$)를 계산하여 지배적인 방향을 결정한다. 각 모델의 델타 부호가 전체 합의 부호와 일치하는 경우에만 마스크(Mask) 값 1을 부여하여 비정상적인 파라미터의 간섭을 줄인다.
3. **Merge (병합)**: 각 모델의 성능(정확도)에 따라 가중치 $\mu_t$를 부여하여 합산한다. 가중치는 해당 데이터셋의 '독특함'을 기준으로 설정한다.
    - **가중치 계산**: 특정 데이터셋에 대해 (최대 정확도 - 최소 정확도)를 계산한다. 이 차이가 클수록 다른 모델이 예측하기 어려운 독특한 데이터셋임을 의미하며, 더 높은 가중치를 부여한다.
    $$\mu_t = \frac{\exp(\text{Max}(\text{Acc}_t) - \text{Min}(\text{Acc}_t))}{\sum \exp(\text{Max}(\text{Acc}) - \text{Min}(\text{Acc}))}$$
    - **최종 파라미터**: $\theta_{merge} = \theta + \sum_{t=1}^n \mu_t \cdot \delta'_t$

## 📊 Results

### 실험 설정

- **데이터셋**: Cora, Citeseer, Pubmed, Ogbn-arxiv 등 4종의 그래프 데이터셋을 사용하였다.
- **타겟 모델**: GCN, GAT, GraphSAGE 등 다양한 GNN 아키텍처를 대상으로 공격을 수행하였다.
- **공격 모델**: Vicuna-7B, Vicuna-13B, LongChat 등 다양한 LLM을 사용하였다.
- **지표**: Accuracy(정확도)와 F1-score를 측정 지표로 사용하였다.

### 주요 결과

1. **단일 데이터셋 성능**: LLM 기반 공격 방식이 기존의 특성 기반(Feature), 사후 확률 기반(PP) 방식보다 우수했다. 특히 Pubmed 데이터셋에서 정확도와 F1-score가 약 7% 향상되는 결과를 보였다.
2. **Cross-dataset 공격 가능성**: 단일 데이터셋으로 학습된 LLM 공격 모델이 다른 데이터셋에서도 일정 수준 이상의 공격 성능을 보였다. 이는 기존 MLP 기반 모델이 불가능했던 영역이다.
3. **모델 병합의 효과**: 제안한 가중치 기반 병합 방식(Our)이 Mean, Ties, DARE, Della 등 기존 병합 방식보다 뛰어난 평균 성능을 보였다. 특히 모든 데이터셋에 대해 균형 잡힌 높은 성능(평균 Accuracy 93.33%)을 달성하였다.
4. **Out-of-Domain (OOD) 공격**: 공격자가 전혀 보지 못한 데이터셋에 대해서도, 나머지 3개 데이터셋의 모델을 병합하여 공격했을 때 개별 모델보다 더 높은 성능을 유지함을 확인하였다. 이는 실제 공격 시나리오에서 매우 강력한 위협이 될 수 있음을 시사한다.

## 🧠 Insights & Discussion

### 강점 및 기여

본 연구는 LLM의 강력한 시퀀스 처리 능력을 프라이버시 공격에 접목하여, 데이터셋 간의 구조적 차이라는 기술적 장벽을 극복하였다. 또한, 데이터 공유 없이 모델 파라미터만으로 지식을 통합하는 Model Merging 프레임워크를 통해 현실적인 협력 공격 시나리오를 구현하였다. 특히 가중치 $\mu_t$를 데이터셋의 난이도(Max-Min Acc)에 따라 동적으로 설정한 점은 매우 영리한 설계이다.

### 한계 및 논의사항

- **사후 확률 의존성**: 본 공격은 타겟 모델로부터 사후 확률(Posterior Probabilities)을 얻을 수 있다는 전제하에 작동한다. 만약 타겟 모델이 응답을 제한하거나 노이즈를 추가하는 방어 기제를 적용한다면 공격 효율이 급감할 가능성이 있다.
- **계산 비용**: LLM의 Fine-tuning과 병합 과정은 상당한 컴퓨팅 자원을 필요로 한다. 비록 파라미터 병합 자체는 비용이 적으나, 초기 개별 모델 구축 단계에서의 오버헤드는 고려되어야 한다.
- **LLM 선택의 영향**: 실험 결과 Vicuna-7B 외의 다른 모델에서도 작동함을 보였으나, 기본 LLM의 성능과 크기에 따라 공격 성공률의 편차가 존재할 수 있다.

## 📌 TL;DR

본 논문은 GNN의 민감한 링크 정보를 탈취하는 **Link Stealing Attack**을 강화하기 위해 **LLM과 모델 병합 기술을 결합한 새로운 프레임워크**를 제안한다. LLM을 통해 데이터셋 간의 구조적 차이를 극복하여 **Cross-dataset 공격**을 가능케 했으며, 데이터 공유 없이 모델 파라미터를 병합하는 방식을 통해 여러 공격자의 지식을 통합한 **범용 공격 모델**을 구축하였다. 실험 결과, 제안 방법은 기존 공격 방식보다 우수한 성능을 보였으며, 특히 학습되지 않은 외부 데이터셋(OOD)에 대해서도 강력한 공격 능력을 갖추어 실제 환경에서의 심각한 프라이버시 위협 가능성을 입증하였다.
